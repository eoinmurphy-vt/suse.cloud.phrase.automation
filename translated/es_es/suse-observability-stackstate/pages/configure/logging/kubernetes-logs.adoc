== Visión general

En una configuración de Kubernetes, las funciones de SUSE Observability se distribuyen en diferentes pods y los registros de cada función se almacenan por pod y contenedor. Puedes acceder a los registros recientes a través de `+kubectl+`, aunque para un almacenamiento a largo plazo se recomienda configurar la agregación de registros.

== Pods Kubernetes para el registro

Los registros de SUSE Observability se almacenan por pod y contenedor. La siguiente tabla muestra el pod al que acceder para obtener registros relacionados con funciones específicas de SUSE Observability. Tenga en cuenta que los nombres de pod reales incluirán un número o un sufijo de cadena aleatorio (por ejemplo, `+stackstate-receiver-5b9d79db86-h2hkz+`) y también pueden incluir el nombre de versión especificado cuando se implantó SUSE Observability como prefijo.

[NOTE]
====
Tenga en cuenta que los registros almacenados en los pods se eliminarán periódicamente. Para un acceso a largo plazo a los registros, se recomienda que configure <<_log_aggregation,la agregación de registros>> para su clúster Kubernetes.
====


|===
| Función de observabilidad de SUSE | Registros en la vaina

| API (incluida la topología, los gráficos y la configuración)
| `+stackstate-api+`

| Indexación de datos en Elasticsearch
| `+stackstate-e2es+` (acontecimientos)

| Ingesta de datos
| `+stackstate-receiver+`

| Controladores de eventos

Puede acceder a los registros de un pod específico utilizando el comando `+kubectl logs+`.

Por ejemplo:

[,sh]
----
$ kubectl logs stackstate-api-0
----

== Acceder a los registros recientes

=== Troncos de vaina o contenedor

Los registros más recientes pueden recuperarse de Kubernetes mediante el comando `+kubectl logs+`. Marque el <<_kubernetes_pods_for_logging,pod que necesita supervisar>> para recuperar un registro específico.

Por ejemplo:

[,sh]
----
# Snapshot of logs for all containers of <pod-name>
$ kubectl logs <pod-name> --all-containers=true

# Stream logs for all containers of <pod-name>
$ kubectl logs -f <pod-name> --all-containers=true

# Snapshot of logs for a specific container of <pod-name>
$ kubectl logs -c <container-name> <pod-name>

# Snapshot of logs for previous terminated container of <pod-name>
$ kubectl logs -p -c <container-name> <pod-name>
----

=== Registros de sincronización

Todos los registros de sincronización se pueden encontrar en un pod `+stackstate-sync-<suffix>+`. Puede utilizar el nombre de sincronización para localizar información de registro específica en una instantánea de registro.

Por ejemplo:

[,sh]
----
# Logs of the synchronization for a specific Kubernetes cluster
$ kubectl logs stackstate-sync-0 | grep "Kubernetes - \<cluster-name\>"

# Logs of the Agent synchronization
$ kubectl logs stackstate-sync-0 | grep "Agent"
----

== Agregación de registros

Para el almacenamiento a largo plazo de los datos de registro de SUSE Observability, se recomienda configurar la agregación de registros en el clúster de Kubernetes. Esto se puede hacer utilizando un sistema de terceros para el almacenamiento como Elasticsearch, Splunk o Logz.io y un cargador de logs como Logstash o Fluentd.

Para más detalles sobre cómo hacerlo, consulte:

* Envío de registros con https://docs.fluentd.org/container-deployment/kubernetes[Fluentd (fluentd.org)]
* Una visión completa de la configuración de https://docs.bitnami.com/tutorials/integrate-logging-kubernetes-kibana-elasticsearch-fluentd/[log aggregation into Elasticsearch (bitnami.com)]
