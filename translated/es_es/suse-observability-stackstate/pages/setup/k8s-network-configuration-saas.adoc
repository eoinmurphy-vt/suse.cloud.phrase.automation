= Configuración de red
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: Observabilidad de SUSE

SUSE Observability es una oferta SaaS alojada en la nube. Para poder comunicarse desde sus instalaciones/nube a SUSE Observability SaaS, el agente de SUSE Observability debe poder conectarse a la API de receptor de SUSE Observability SaaS.
Si el clúster se ejecuta en una red privada, es posible que tenga que configurar la red para permitir que el agente de SUSE Observability se conecte a la API de SUSE Observability Receiver, ya que la configuración de la red puede no permitir el tráfico de salida a Internet. En esta página se describe cómo configurar la red para permitir la instalación del Agente de SUSE Observability, así como para permitir que el Agente de SUSE Observability se comunique con la API de SUSE Observability Receiver.

[NOTE]
====
El tráfico entre el Agente de SUSE Observability y la API de SUSE Observability Receiver siempre lo inicia el Agente de SUSE Observability. La API de SUSE Observability Receiver no inicia ningún tráfico hacia el Agente de SUSE Observability.
====


== Instalación del agente de SUSE Observability

La instalación del Agente de SUSE Observability se realiza a través de Helm. Por defecto, Helm Chart está configurado para extraer las imágenes de contenedor del Agente de SUSE Observability del registro de Dockers de Quay.io. Si la configuración de su red no permite el tráfico de salida a Internet, tiene varias opciones para instalar el Agente de SUSE Observability:

. Configure su red para permitir el tráfico de salida al registro de contenedores Quay.io desde su clúster Kubernetes.
. Proxy del registro de contenedores de Quay.io a través de su propio registro de contenedores.
. Extraiga las imágenes Docker en su propio registro de contenedores.

Para las opciones 2 y 3, es necesario configurar el Cuadro de Helm para que extraiga las imágenes de contenedor del Agente de SUSE Observability de su propio registro de contenedores. Puede encontrar una guía para configurar SUSE Observability Agent Helm Chart para extraer imágenes de su propio registro de contenedores xref:/k8s-suse-rancher-prime-agent-air-gapped.adoc[aquí].

== Comunicación con el agente de SUSE Observability

El agente de SUSE Observability se comunica con la API de SUSE Observability Receiver a través de HTTPS. Las diferentes partes del Agente de SUSE Observability se conectan a la API de SUSE Observability Receiver, alojada en su inquilino en, véase el siguiente diagrama:

image::k8s/k8s-agent-communication.png[Comunicación con el agente de SUSE Observability]

Toda la comunicación se realiza a través de HTTPS, utilizando el puerto HTTPS estándar 443. El agente de SUSE Observability utiliza los siguientes puntos finales para comunicarse con la API de SUSE Observability Receiver:

* https:// *<tenant>.app.stackstate.io/receiver/stsAgent*: el agente de SUSE Observability envía métricas, eventos y datos de topología a la API de SUSE Observability Receiver.

Para permitir que el Agente de SUSE Observability se comunique con la API de SUSE Observability Receiver, es necesario configurar la red para permitir el tráfico de salida a la API de SUSE Observability Receiver. La API de SUSE Observability Receiver está alojada en la nube y tiene una IP específica para su inquilino. Debe permitir el tráfico de salida a Internet. Para obtener las direcciones IP correctas a las que permitir el tráfico de salida, puede utilizar el siguiente comando:

[,bash]
----
$ dig +short <tenant>.app.stackstate.io
----

También puede visitar la siguiente URL en su navegador: `+https://www.nslookup.io/domains/<tenant>.app.stackstate.io/dns-records/+`
