== ComponentPresentation

El concepto central del modelo UI Presentation es el *ComponentPresentation*.

En `+ComponentPresentation+` se define cómo se presentan en la interfaz de usuario uno o varios componentes coincidentes, en función del contexto activo.
Se centra exclusivamente en *cuestiones de presentación*, como la nomenclatura, los iconos, las tablas, las páginas de resumen y destacadas, los filtros y la colocación de los elementos de menú.

A alto nivel, un ComponentPresentation:

* Selecciona componentes utilizando un enlace (una simple consulta STQL).
* Se aplica dentro de un modo específico (por ejemplo, observabilidad).
* Define uno o varios aspectos de la presentación (como cuadros sinópticos o destacados).
* Puede definir parcialmente las propiedades de presentación y componerlas con otras definiciones coincidentes.

Se pueden aplicar varias ComponentPresentations al mismo componente. En estos casos, sus definiciones se combinan para determinar la presentación final.
