= Configuración del agente de SUSE Observability Kubernetes para conexiones proxy
:revdate: 2025-07-10
:page-revdate: {revdate}

El agente de SUSE Observability Kubernetes permite configurar los ajustes de proxy HTTP o HTTPS para las conexiones que inicia.

== Proxy para la comunicación con SUSE Observability

Para configurar el agente para que proxy conexiones al backend de SUSE Observability, puede utilizar la configuración de Helm.

=== Configuración del timón

==== A través de `+values.yaml+` File

. Abra su fichero Helm chart `+values.yaml+`.
. Localice la configuración de `+global.proxy.url+` y especifique la URL del proxy:
+
[,yaml]
----
 global:
   proxy:
     url: "https://proxy.example.com:8080"
----

. Opcionalmente, si el proxy no tiene un certificado firmado, desactive la verificación SSL estableciendo `+global.skipSslValidation+` en `+true+`:
+
[,yaml]
----
 global:
   skipSslValidation: true
----

==== Mediante indicador de línea de comandos

. Durante la instalación del gráfico Helm, utilice la bandera `+--set+` para especificar la URL del proxy:
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.proxy.url="https://proxy.example.com:8080"
----

. Para desactivar la validación SSL a través de la línea de comandos, utilice:
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.skipSslValidation=true
----
