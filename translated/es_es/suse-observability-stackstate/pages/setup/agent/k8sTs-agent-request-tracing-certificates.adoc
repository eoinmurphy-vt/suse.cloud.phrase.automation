= Certificados para la inyección sidecar de rastreo de solicitudes
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: Observabilidad de SUSE

El xref:/setup/agent/k8sTs-agent-request-tracing.adoc#_enabling_the_trace_header_injection_sidecar[mecanismo de inyección sidecar], que se activa cuando se utiliza `+--set httpHeaderInjectorWebhook.enabled=true+` al instalar el agente, crea un certificado autofirmado y utiliza un `+ClusterRole+` que concede acceso de escritura a los objetos `+Secret+` y `+MutatingWebhookConfiguration+` en el clúster Kubernetes.

Si por motivos de seguridad no es deseable crear `+ClusterRoles+` que conceda derechos de escritura en todo el clúster, o existen formas alternativas de proporcionar un certificado:

. Generar un certificado autofirmado <<_generate_a_certificate_locally,localmente>>.
. Utilice el k8s https://cert-manager.io/[cert-manager] (si ya está en el clúster) <<_generate_a_certificate_using_the_cert_manager,con un `+ClusterIssuer+`>>.

== Generar un certificado localmente

Para generar un certificado localmente, siga los siguientes pasos:

. Descargue el script de generación de certificados y ejecútelo para generar un archivo de valores de timón (`+tls_values.yaml+`) con el certificado correcto:
+
----
wget https://raw.githubusercontent.com/StackVista/http-header-injector/main/scripts/generate_ca_cert.sh
chmod +x generate_ca_cert.sh
./generate_ca_cert.sh <helm-agent-release-name> <helm-agent-namespace>
----
+
Asegúrese de utilizar el nombre de versión que se utilizará en el comando helm y el espacio de nombres, de lo contrario el certificado no será válido.

. Instale el agente añadiendo la configuración adicional añadiendo `+--set httpHeaderInjectorWebhook.enabled=true -f tls_values.yaml+` al comando de invocación de helm

== Generar un certificado utilizando el cert-manager

Si su cluster tiene el https://cert-manager.io/[cert-manager] instalado, y un `+ClusterIssuer+` configurado, es posible utilizar el certificado emitido por el `+ClusterIssuer+` en el agente para el inyector sidecar. Para ello, añada los siguientes argumentos de línea de comandos para instalar el agente: `+--set httpHeaderInjectorWebhook.enabled=true --set-string httpHeaderInjectorWebhook.webhook.tls.mode="cert-manager" --set-string httpHeaderInjectorWebhook.webhook.tls.certManager.issuer="<my-cluster-issuer>"+`. Asegúrese de sustituir mi-emisor-cluster por el nombre del emisor de su clúster.
