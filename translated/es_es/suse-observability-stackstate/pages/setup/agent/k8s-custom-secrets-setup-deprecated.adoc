= k8s Custom Secrets Setup (obsoleto)
:revdate: 2025-10-27
:page-revdate: {revdate}

== Visión general

El agente de SUSE Observability permite especificar el nombre de un secreto personalizado que contiene la clave de API y el token de autorización de clúster. Esta función es útil para los usuarios que deseen gestionar sus propios secretos y evitar la creación automática de secretos por parte del gráfico de Helm.

[CAUTION]
====
*Este método de personalización de secretos está obsoleto y es incompatible con la función RBAC.*

Utilice xref:/setup/agent/k8s-custom-secrets-setup.adoc[el nuevo método] para utilizar secretos externos.
====


== El cuadro de mando

=== Opciones de configuración

* `+stackstate.manageOwnSecrets+`: Un indicador booleano que determina si el usuario desea gestionar sus propios secretos. El valor por defecto es `+false+`.
* `+stackstate.customSecretName+`: (Opcional) Nombre del secreto personalizado que debe crear el usuario. Obligatorio si `+stackstate.manageOwnSecrets+` está configurado como `+true+`.
* `+stackstate.customApiKeySecretKey+`: (Opcional) Nombre de la clave API dentro del secreto personalizado. Obligatorio si `+stackstate.manageOwnSecrets+` está configurado como `+true+`.
* `+stackstate.customClusterAuthTokenSecretKey+`: (Opcional) Nombre de la clave para el token de autorización del clúster dentro del secreto personalizado. Obligatorio si `+stackstate.manageOwnSecrets+` está configurado como `+true+`.

=== Descripción del comportamiento

* *Creación automática de secretos*: Por defecto, el gráfico sigue creando secretos automáticamente como antes si `+stackstate.manageOwnSecrets+` está configurado como `+false+`.
* *Gestión de secretos personalizada*: Si `+stackstate.manageOwnSecrets+` se establece en `+true+`, el gráfico espera que el usuario proporcione el nombre del secreto personalizado (`+stackstate.customSecretName+`) junto con las claves para la clave API y el token de autorización (`+stackstate.customApiKeySecretKey+` y `+stackstate.customClusterAuthTokenSecretKey+`, respectivamente).
* *Omisión implícita*: Al especificar que desea gestionar sus propios secretos, el gráfico ignorará los valores de `+stackstate.apiKey+` y `+stackstate.cluster.authToken+`.
+
== Cómo utilizar en values.yaml

. *Uso de la creación automática de secretos (por defecto)*:
+
[,yaml]
----
 stackstate:
   manageOwnSecrets: false
   apiKey: "<your api key>"
----

. *Gestión de secretos propios*:
+
[,yaml]
----
 stackstate:
   manageOwnSecrets: true
   customSecretName: my-custom-secret
   customApiKeySecretKey: api-key
   customClusterAuthTokenSecretKey: auth-token
----
