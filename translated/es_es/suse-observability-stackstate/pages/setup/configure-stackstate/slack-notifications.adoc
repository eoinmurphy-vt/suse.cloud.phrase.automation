= Configuración de SUSE Observability para las notificaciones de Slack
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability autoalojado

[CAUTION]
====
Los usuarios SaaS de SUSE Observability pueden utilizar las notificaciones de Slack sin necesidad de configuración adicional. Esta guía sólo es aplicable a instalaciones de SUSE Observability autoalojadas que tengan previsto utilizar el canal de notificaciones de Slack.
====


Antes de poder utilizar el canal de notificaciones de Slack en SUSE Observability, debe seguir los pasos siguientes para configurar Slack y SUSE Observability:

. Cree una aplicación Slack para SUSE Observability en su espacio de trabajo
. Configure SUSE Observability con las credenciales de esa aplicación de Slack.

== Creación de una aplicación Slack para SUSE Observability

[NOTE]
====
Necesitas tener los permisos en Slack para gestionar las aplicaciones de Slack para tu espacio de trabajo.
====


Vaya a la https://api.slack.com/apps[página] y haga clic en el botón *Create New App*.

* Seleccione la opción "Desde el manifiesto de una aplicación" en el cuadro de diálogo que se abre.
* Seleccione el área de trabajo a la que desea enviar notificaciones y haga clic en Siguiente.
* Copia el contenido del manifiesto de la aplicación Slack que aparece a continuación y pégalo en el área de texto. Asegúrese de sustituir los valores de `+redirect_urls+` por la(s) URL(s) de su instancia de SUSE Observability. Haz clic en Siguiente.
* Compruebe que la URL es correcta y que los "ámbitos de bots" que aparecen en la lista son `+channels:join, channels:read, chat:write, groups:read+` y haga clic en el botón Crear para crear la aplicación.
* En la página "Información básica" de la App es posible cambiar el icono (en la sección Información de pantalla), puede sustituirlo, por ejemplo, por el logotipo de SUSE Observability image:logo/stackstate-logo.png[Observability logo,25,25] SUSE.

.Manifiesto de la aplicación Slack para SUSE Observability
[,json]
----
{
    "display_information": {
        "name": "SUSE Observability",
        "description": "Receive notification messages from SUSE Observability",
        "background_color": "#000000"
    },
    "features": {
        "bot_user": {
            "display_name": "SUSE Observability",
            "always_online": true
        }
    },
    "oauth_config": {
        "redirect_urls": [
            "https://the.url.of.your.stackstate.installation"
        ],
        "scopes": {
            "bot": [
                "channels:join",
                "channels:read",
                "chat:write",
                "groups:read"
            ]
        }
    },
    "settings": {
        "org_deploy_enabled": false,
        "socket_mode_enabled": false,
        "token_rotation_enabled": false
    }
}
----


== Configure SUSE Observability con las credenciales de esa aplicación de Slack

SUSE Observability debe configurarse con las credenciales de la aplicación Slack que ha creado. Puede hacerlo añadiendo lo siguiente al archivo `+values.yaml+` de su instalación de SUSE Observability:

[,yaml]
----
stackstate:
  components:
    all:
      extraEnv:
        open:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientId: "<app client id>"
        secret:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientSecret: "<app client secret>"
----

Los valores `+<app client id>+` y `+<app client secret>+` se encuentran en la sección "Credenciales de la aplicación" de la página "Información básica" de la aplicación Slack que has creado. Aplique estos cambios de configuración ejecutando el mismo comando Helm utilizado durante la instalación de SUSE Observabilityxref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[(para Kubernetes] u xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_deploy_suse_observability_with_helm[OpenShift]).

Ya puedes utilizar el canal de notificaciones de Slack.
