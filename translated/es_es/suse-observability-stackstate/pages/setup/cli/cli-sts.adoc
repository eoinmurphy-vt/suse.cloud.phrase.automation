== Visión general

La CLI {stackstate-product-name} `+sts+` facilita el acceso a la funcionalidad proporcionada por las API de {stackstate-product-name}. Puede utilizarse para la automatización utilizando datos de {stackstate-product-name}, para configurar {stackstate-product-name} y para desarrollar StackPacks.

== Requisitos previos

* **{stackstate-product-name} URL** - La URL de su instancia {stackstate-product-name} (la misma URL utilizada para acceder a la interfaz web).
* **Token** de API: un token de API válido para la autenticación, que puede recuperarse de la página *CLI* en su instancia de {stackstate-product-name}.

Para obtener su token de API:

. Conéctese a su instancia {stackstate-product-name}.
. Acceda al *menú principal* > *CLI*.
. Copie el token de API que aparece en la página.

== Inicio rápido

La forma más rápida de instalar y configurar la CLI de `+sts+` es utilizar los comandos preconfigurados disponibles en su instancia de {stackstate-product-name}:

. Conéctese a su instancia {stackstate-product-name}.
. Acceda al *menú principal* > *CLI*.
. Copia el comando para tu sistema operativo y ejecútalo en un terminal.

Estos comandos se rellenan previamente con la URL y el token de API correctos para su instancia específica de {stackstate-product-name}, por lo que la instalación y la configuración se realizan en un solo paso.

== Descargar la CLI

[tabs]
====
Windows::
+
--
Abra un terminal *Powershell* (versión 5.1 o posterior) y ejecute los pasos que se indican a continuación. Esto se puede hacer paso a paso, o unidos como un solo script. Tras la instalación, el comando `+sts+` estará disponible para el usuario actual tanto en el terminal Powershell como en el símbolo del sistema (cmd.exe).

. Establezca la versión de origen y la ruta de destino para la CLI:
+
[,powershell]
----
$CLI_PATH = $env:USERPROFILE +"\stackstate-cli"
If (!(test-path $CLI_PATH)) { md $CLI_PATH }
Invoke-WebRequest https://dl.stackstate.com/stackstate-cli/LATEST_VERSION -OutFile $CLI_PATH\VERSION
$VERSION=type $CLI_PATH\VERSION
$VERSION=$VERSION -replace "[v]"
$CLI_DL = "https://dl.stackstate.com/stackstate-cli/v$VERSION/stackstate-cli-$VERSION.windows-x86_64.zip"
echo "Installing {stackstate-product-name} CLI v$VERSION to: $CLI_PATH"
----

. Descargue y descomprima la CLI en la ruta CLI de destino. Eliminar los artefactos restantes:
+
[,powershell]
----
Invoke-WebRequest $CLI_DL -OutFile $CLI_PATH\stackstate-cli.zip
Expand-Archive -Path "$CLI_PATH\stackstate-cli.zip" -DestinationPath $CLI_PATH -Force
rm $CLI_PATH\stackstate-cli.zip, $CLI_PATH\VERSION
----

. Registra la ruta CLI en el PATH del usuario actual. De este modo, el comando `+sts+` estará disponible en todas partes:
+
[,powershell]
----
$PATH = (Get-ItemProperty -Path "Registry::HKEY_CURRENT_USER\Environment" -Name PATH).Path
if ( $PATH -notlike "*$CLI_PATH*" ) {
  $PATH = "$PATH;$CLI_PATH"
  (Set-ItemProperty -Path "Registry::HKEY_CURRENT_USER\Environment" -Name PATH –Value $PATH)
  $MACHINE_PATH = (Get-ItemProperty -Path "Registry::HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\Session Manager\Environment" -Name PATH).path
  $env:Path = "$PATH;$MACHINE_PATH"
}
----

. Compruebe que la CLI funciona:
+
[,powershell]
----
sts version
----

== Configurar la CLI

=== Autenticación

==== Token API

Por defecto, la CLI se autenticará utilizando el token de API que proporcionó cuando se guardó la configuración de la CLI.

[CAUTION]
====
La forma más segura de utilizar su token de API es a través de una variable de entorno. Puedes almacenar el token de la API con un gestor de secretos e inyectarlo como variable de entorno en tu shell.
====

[tabs]
====
Windows / macOS / Linux::
+
--
. En la interfaz de usuario {stackstate-product-name}, vaya al *menú principal* > *CLI* y copie su token de API.
. Ejecute el siguiente comando, donde `+<URL>+` es la URL de su instancia de {stackstate-product-name} y `+<API-TOKEN>+` es el token de API que copió de la página CLI en la interfaz de usuario de {stackstate-product-name}:
+
[,bash]
----
sts context save --name <NAME> --url <URL> --api-token <API-TOKEN>
----

. Se probará la conexión con su instancia de {stackstate-product-name} y se almacenará un archivo de configuración en `+~/.config/stackstate-cli/config.yaml+`.
--

Docker::
+
--
La versión Docker de la CLI no se puede configurar con un archivo de configuración. Especifique la configuración de su instancia {stackstate-product-name} utilizando variables de entorno y páselas a Docker:

* `+STS_CLI_URL+` - la URL de su instancia {stackstate-product-name}.
* `+STS_CLI_API_TOKEN+` - el token de API tomado de la página {stackstate-product-name} UI *Main menu* > *CLI*.
* `+STS_CA_CERT_PATH+` - ruta a un archivo de certificado CA codificado en PEM para servidores que utilicen certificados autofirmados. El directorio que contiene el certificado debe montarse en el contenedor Docker.
* `+STS_CA_CERT_BASE64_DATA+` - Datos del certificado CA codificados en base64 para servidores que utilicen certificados autofirmados (se ignoran si se especifica `+STS_CA_CERT_PATH+` ).
* `+STS_SKIP_SSL+` - desactiva la validación de certificados SSL (ignora las configuraciones de certificados, utilícelo con precaución).

Por ejemplo:

[,bash]
----
docker run \
   -v /path/to/certs:/certs \
   -e STS_CLI_URL \
   -e STS_CLI_API_TOKEN \
   -e STS_CA_CERT_PATH=/certs/ca.crt \
   stackstate/stackstate-cli2 settings list --type Layer
----
--

==== Fichas de servicio

Opcionalmente, puede utilizar la CLI para crear uno o varios tokens de servicio para autenticarse con las API de base y administración de {stackstate-product-name}. Por ejemplo, un token de servicio se puede utilizar para autenticar en escenarios CI (Integración Continua) en los que ningún usuario real está realizando las operaciones en la instancia {stackstate-product-name}.

Para crear un token de servicio, ejecute el siguiente comando:

[,bash]
----
sts service-token create --name <NAME> --roles <ROLE(s)> [--expiration <yyyy-MM-dd>]
----

Esto creará un nuevo token de servicio y lo imprimirá. El parámetro `+--expiration+` es opcional y se puede utilizar para establecer la fecha de caducidad del token de servicio.

Una vez que tenga esto, puede configurar la CLI para usarlo:

[,bash]
----
sts context save --name <NAME> --service-token <TOKEN> --url <URL>
----

=== Gestionar múltiples contextos

La CLI `+sts+` permite configurar y gestionar diferentes contextos (de autenticación). Esto le permite cambiar fácilmente entre un usuario administrativo y un usuario normal, o cambiar entre diferentes instancias de {stackstate-product-name}. Por ejemplo, podría utilizar un contexto diferente para una instancia de prueba y de producción de {stackstate-product-name}. Puede listar, guardar, borrar, establecer y validar contextos en la CLI `+sts+`. Ejecute `+sts context -h+` para obtener información detallada sobre los comandos disponibles y su uso.

=== Opciones de configuración

No necesita un archivo de configuración para ejecutar la CLI de `+sts+`. También puede configurar la CLI mediante una combinación de variables de entorno y flags.

Si se presentan varios tipos de configuración a la CLI, el orden de procesamiento será:

. Banderas
. Variables de entorno
. Archivo de configuración

|===
| Variable de entorno | Bandera | Descripción

| `+STS_CLI_URL+`
| `+--url+`
| URL de su instancia {stackstate-product-name}.

| `+STS_CLI_API_TOKEN+`
| `+--api-token+`
| API a su instancia {stackstate-product-name}. La forma más segura de utilizar su token de API es a través de una variable de entorno. Puedes almacenar el token de la API con un gestor de secretos e inyectarlo como variable de entorno en tu shell.

| `+STS_CLI_SERVICE_TOKEN+`
| `+--service-token+`
| Un token de servicio para su instancia {stackstate-product-name}. La forma más segura de utilizar su token de servicio es a través de una variable de entorno. Puedes almacenar el token de servicio con un gestor de secretos e inyectarlo como variable de entorno en tu shell.

| `+STS_CA_CERT_PATH+`
| `+--ca-cert-path+`
| Ruta a un archivo de certificado de CA codificado en PEM para servidores que utilicen certificados autofirmados o certificados de una CA privada.

| `+STS_CA_CERT_BASE64_DATA+`
| `+--ca-cert-base64-data+`
| Datos de certificado de CA codificados en Base64 para servidores que utilicen certificados autofirmados o certificados de una CA privada. Se ignora si se especifica `+STS_CA_CERT_PATH+`.

| `+STS_SKIP_SSL+`
| `+--skip-ssl+`
| Desactiva la validación de certificados SSL. Cuando se establece en `+true+`, la CLI no validará certificados SSL e ignorará las configuraciones `+STS_CA_CERT_PATH+` y `+STS_CA_CERT_BASE64_DATA+`. Utilícelo con precaución, ya que reduce la seguridad.

| `+STS_CLI_API_PATH+`
| n/a
| La ruta añadida al final de la URL para obtener el punto final de la API. (Por defecto `+/api+`)

| `+STS_CLI_CONTEXT+`
| `+--context+`
| El nombre del contexto a utilizar.
|===

Además de anular partes específicas del archivo de configuración, también es posible anular la ubicación predeterminada del archivo de configuración. Esto se hace a través de la bandera `+--config <PATH>+`.

== Desinstalar

[tabs]
====
Windows::
+
--
[discrete]
=== Desinstalación rápida

Abra un terminal *Powershell* y ejecútelo:

[,powershell]
----
. { iwr -useb https://dl.stackstate.com/stackstate-cli/install.ps1 } | iex; uninstall
----

La CLI `+sts+` y toda la configuración asociada se eliminan para el usuario actual.

[discrete]
=== Desinstalación manual

Abra un terminal *Powershell* y ejecute cada paso uno a uno o todos a la vez. La CLI `+sts+` y toda la configuración asociada se eliminarán para el usuario actual.

. Eliminar binario:
+
[,powershell]
----
$CLI_PATH = $env:USERPROFILE+"\stackstate-cli"
rm -R $CLI_PATH 2>1  > $null
----

. Quitar config:
+
[,powershell]
----
rm -R $env:USERPROFILE+"\.config\stackstate-cli" 2>1  > $null
----

. Elimina la CLI de la ruta del entorno:
+
[,powershell]
----
$PATH = (Get-ItemProperty -Path 'Registry::HKEY_CURRENT_USER\Environment' -Name PATH).Path
$i = $PATH.IndexOf(";$CLI_PATH")
if ($i -ne -1) {
  $PATH = $PATH.Remove($i, $CLI_PATH.Length+1)
  (Set-ItemProperty -Path 'Registry::HKEY_CURRENT_USER\Environment' -Name PATH –Value $PATH)
}
----
--
