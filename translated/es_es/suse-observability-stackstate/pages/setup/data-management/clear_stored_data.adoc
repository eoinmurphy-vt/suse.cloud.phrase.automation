= Borrar datos almacenados
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability autoalojado

Los datos de SUSE Observability se dividen en cuatro conjuntos diferentes:

* Datos de Elasticsearch
* Datos de Kafka Topic
* Datos de StackGraph
* Datos métricos

Con tantos datos que almacenar, es importante disponer de medios para gestionarlos. En SUSE Observability existe un período estándar de conservación de datos de 30 días. Esto puede configurarse según sus necesidades mediante la CLI de SUSE Observability. Obtenga más información sobre xref:/setup/data-management/data_retention.adoc[la retención de datos de SUSE Observability].

== Borrar datos manualmente

Para borrar los datos almacenados en SUSE Observability que se ejecuta en Kubernetes, se recomienda ejecutar una instalación limpia:

. xref:/setup/install-stackstate/kubernetes_openshift/uninstall.adoc#_un_install_the_helm_chart[Desinstalar SUSE Observability]
. xref:/setup/install-stackstate/kubernetes_openshift/uninstall.adoc#_remove_remaining_resources[Retire todos los PVC]
. Instale SUSE Observability de nuevo utilizando la misma configuración que antes, en xref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[Kubernetes] u xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_deploy_suse_observability_with_helm[OpenShift].
