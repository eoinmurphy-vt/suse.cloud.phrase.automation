Cree un archivo `+values.yaml+` con la estructura de configuración `+global.suseObservability+`:

[source,yaml]
----
global:
  # Optional: Override image registry (defaults to registry.rancher.com)
  # Only needed for air-gapped environments or custom registries
  # imageRegistry: "your-private-registry.example.com"

  # Optional: Override storage class (uses cluster default if not set)
  # storageClass: "your-storage-class"

  suseObservability:
    # Required: Your {stackstate-product-name} license key
    license: "YOUR-LICENSE-KEY"

    # Required: Base URL for {stackstate-product-name}
    # Use http://localhost:8080 if ingress is not configured yet
    baseUrl: "https://observability.example.com"

    # Required: Sizing profile
    # Available: trial, 10-nonha, 20-nonha, 50-nonha, 100-nonha,
    #            150-ha, 250-ha, 500-ha, 4000-ha
    sizing:
      profile: "150-ha"

    # Required: Plain text Admin password
    adminPassword: "your-password"
    # Instead of adminPassword you can provide a bcrypt hashed password with adminPasswordBcrypt
    # Generate with: htpasswd -bnBC 10 "" "your-password" | tr -d ':\n'
    # adminPasswordBcrypt: "$2a$10$..."

    # Optional: Receiver API key (auto-generated if not provided)
    # receiverApiKey: "your-receiver-api-key"

    # Optional: Pull secret for private registry
    # pullSecret:
    #   username: "registry-user"
    #   password: "registry-pass"
----

TIP: Para generar un hash de contraseña bcrypt, utilice: `+htpasswd -bnBC 10 "" "your-password" | tr -d ':\n'+`

Para obtener más opciones de configuración, incluidos los ajustes de afinidad, consulte xref:/setup/install-stackstate/kubernetes_openshift/affinity.adoc[Configurar afinidades de Kubernetes].

Puede generar los archivos {stackstate-product-name} `+baseConfig_values.yaml+`, `+sizing_values.yaml+`, y `+affinity_values.yaml+` ejecutando un Helm Chart por separado. El gráfico `+suse-observability/suse-observability-values+`. Un ejemplo de línea de comandos es:

[,text]
----
export VALUES_DIR=.
helm template \
  --set license='<your license>' \
  --set baseUrl='<suse-observability-base-url>' \
  --set sizing.profile='<sizing.profile>' \
  suse-observability-values \
  suse-observability/suse-observability-values --output-dir $VALUES_DIR
----

Este comando genera los ficheros `+$VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml+`, `+$VALUES_DIR/suse-observability-values/templates/sizing_values.yaml+`, y `+$VALUES_DIR/suse-observability-values/templates/affinity_values.yaml+` que contienen la configuración necesaria para instalar el Cuadro de Mando {stackstate-product-name}.

[NOTE]
=====
Las contraseñas de administrador de {stackstate-product-name} serán autogeneradas por el comando anterior y se mostrarán como comentarios en el archivo `+basicConfig.yaml+` generado. Los valores reales contienen los hashes `+bcrypt+` de esas contraseñas para que se almacenen de forma segura en la versión de Helm en el cluster.
=====


Los valores que se pueden pasar a este gráfico son:

[cols="1,1,2"]
|===
| Configuración | Valor | Descripción

| Generar configuración básica
| `+basicConfig.generate+`
| Activar o desactivar la generación del archivo `+basicConfig_values.yaml+`.

| Clave API del receptor
| `+receiverApiKey+`
| La clave API utilizada por {stackstate-product-name} para recibir datos de los agentes. Se trata de una clave secreta que debe mantenerse en privado. Si lo omite, se generará una clave aleatoria para usted.

| URL base
| `+baseUrl+`
| La página `+<STACKSTATE_BASE_URL>+`. La URL externa de {stackstate-product-name} que los usuarios y agentes utilizarán para conectarse. Por ejemplo `+https://suse-observability.internal+`. Si aún no se ha decidido por una configuración Ingress, utilice `+http://localhost:8080+`. Esto puede actualizarse posteriormente en el archivo generado.

| Nombre de usuario y contraseña**
| `+-u+` `+-p+`
| El nombre de usuario y la contraseña utilizados por {stackstate-product-name} para extraer imágenes. Para los entornos con air-gap, estos deben ser el nombre de usuario y la contraseña para el registro local de Docker.

| Clave de licencia
| `+license+`
| La clave de licencia {stackstate-product-name}.

| Contraseña por defecto
| `+adminPassword+`
| La contraseña del usuario por defecto (`+admin+`) para acceder a {stackstate-product-name}'s UI. También puede proporcionar una contraseña con hash bcrypt utilizando `+adminPasswordBcrypt+`. Sólo se puede utilizar una de las dos opciones al mismo tiempo.

| Registro de imágenes
| `+imageRegistry+`
| El registro donde se alojan las imágenes de {stackstate-product-name}. Si no se indica, el valor por defecto será "quay.io".

| Tirar nombre de usuario secreto
| `+pullSecret.username+`
| El nombre de usuario utilizado para extraer imágenes del registro Docker donde se alojan las imágenes {stackstate-product-name}.

| Tire de la contraseña secreta
| `+pullSecret.password+`
| La contraseña utilizada para extraer imágenes del registro Docker donde se alojan las imágenes {stackstate-product-name}.

| Generar tamaño
| `+sizing.generate+`
| Activar o desactivar la generación del archivo `+sizing_values.yaml+`.

| Perfil de tamaño
| `+sizing.profile+`
| OneOf trial, 10-nonha, 20-nonha, 50-nonha, 100-nonha, 150-ha, 250-ha, 500-ha. Basándose en estos perfiles, puede generar el archivo `+sizing_values.yaml+` que contiene los tamaños predeterminados para los recursos {stackstate-product-name} y la configuración que se desplegará en modo HA o Non-HA. Por ejemplo, 10-nonha produce un `+sizing_values.yaml+` para desplegar una instancia Non-HA {stackstate-product-name} para observar un cluster de 10 nodos en modo Non High availability. Actualmente, no es posible pasar de un entorno No-HA a un entorno HA. Si espera que su entorno requiera la observación de unos 150 nodos, entonces es mejor optar inmediatamente por HA.

| Pod AntiAffinity
| `+affinity.podAntiAffinity.requiredDuringSchedulingIgnoredDuringExecution+`
| Imponga que las réplicas de los mismos servicios de datos se programen en diferentes nodos Kubernetes. Sólo aplicable a los perfiles `+HA+`. O `+true+` o `+false+`. El valor por defecto es `+true+` - se impone la programación a diferentes nodos.
|===

[NOTE]
=====
Guarde los archivos `+basicConfig.yaml+` y `+sizing_values.yaml+` generados de forma segura. Puede reutilizar estos archivos para las actualizaciones, lo que ahorra tiempo y garantiza que {stackstate-product-name} siga utilizando la misma clave API. Esto es deseable, ya que significa que los Agentes y otros proveedores de datos para {stackstate-product-name} no necesitarán actualización.
Puede regenerar los archivos de forma independiente utilizando los conmutadores `+basicConfig.generate=false+` y `+sizing.generate=false+` para desactivar cualquiera de ellos sin perder la versión del archivo generada anteriormente en `+output-dir+`.

=== Cree `+openshift-values.yaml+`

Dado que OpenShift tiene un modelo de seguridad más estricto que Kubernetes, todos los contextos de seguridad estándar de la implementación deben desactivarse.

Cree un archivo de valores Helm `+openshift-values.yaml+` con el siguiente contenido y guárdelo junto al archivo `+values.yaml+` generado. Contiene los valores necesarios para la implantación de OpenShift.

[,yaml]
----
elasticsearch:
  prometheus-elasticsearch-exporter:
    podSecurityContext: ""
  sysctlInitContainer:
    enabled: false
scc:
  enabled: true
----

=== Acceda a la interfaz de usuario {stackstate-product-name} 

Una vez desplegado {stackstate-product-name}, puede comprobar si todos los pods están en funcionamiento:

[,bash]
----
kubectl get pods --namespace suse-observability
----

Cuando todos los pods estén en funcionamiento, puede activar un reenvío de puerto:

[,bash]
----
kubectl port-forward service/suse-observability-router 8080:8080 --namespace suse-observability
----

{stackstate-product-name} estará ahora disponible en su navegador en `+https://localhost:8080+`. Inicie sesión con el nombre de usuario `+admin+` y la contraseña predeterminada que se proporciona en el archivo `+values.yaml+`.

== Creación manual de objetos `+SecurityContextConfiguration+` 

Si no puede utilizar una cuenta de administrador para instalar {stackstate-product-name} en OpenShift, pida a su administrador que aplique los siguientes objetos de `+SecurityContextConfiguration+`.

[,yaml]
----
apiVersion: security.openshift.io/v1
kind: SecurityContextConstraints
metadata:
  name: {{ template "common.fullname.short" . }}-{{ .Release.Namespace }}
  labels:
    {{- include "common.labels.standard" . | nindent 4 }}
  annotations:
    helm.sh/hook: pre-install
    suse-observability.io/note: "Ignored by helm uninstall, has to be deleted manually"
fsGroup:
  type: RunAsAny
groups:
- system:serviceaccounts:{{ .Release.Namespace }}
runAsUser:
  type: RunAsAny
seLinuxContext:
  type: MustRunAs
supplementalGroups:
  type: RunAsAny
volumes:
- configMap
- downwardAPI
- emptyDir
- ephemeral
- persistentVolumeClaim
- projected
- secret
allowHostDirVolumePlugin: false
allowHostIPC: false
allowHostNetwork: false
allowHostPID: false
allowHostPorts: false
allowPrivilegeEscalation: true
allowPrivilegedContainer: false
readOnlyRootFilesystem: false
seccompProfiles:
  - runtime/default
----
