= Instalación del agente de SUSE Observability en modo Air-Gapped
:revdate: 2025-07-18
:page-revdate: {revdate}
:description: Observabilidad de SUSE

Este documento proporciona una guía paso a paso para la instalación del Agente de SUSE Observability utilizando las cartas Helm en un entorno air-gapped. El proceso de instalación implica preparar las imágenes Docker y el gráfico Helm necesarios en un host con acceso a Internet, transferirlos a un host dentro de una red privada, subir las imágenes Docker a un registro privado y desplegar los gráficos Helm. Los privilegios necesarios para instalar el agente se encuentran xref:/k8s-suse-rancher-prime.adoc#_required_privileges[aquí]

== Requisitos previos

=== En el host local (acceso a Internet)

* *Sistema operativo*: Linux o MacOS
* *Herramientas instaladas*:
 ** https://www.docker.com/products/docker-desktop/[Docker]
 ** https://helm.sh/docs/intro/install/[Timón cli]
 ** Scripts para descargar imágenes Docker del registro de fuentes (los enlaces se proporcionarán más adelante en la guía).
* *Acceso a Internet*: Necesario para extraer imágenes Docker de Quay.io y gráficos Helm de ChartMuseum.

=== En el host de red privada

* *Acceso*: Acceso SSH al host.
* *Herramientas instaladas*:
 ** https://www.docker.com/products/docker-desktop/[Docker]
 ** https://helm.sh/docs/intro/install/[Timón cli]
 ** Scripts para descargar imágenes Docker del registro de fuentes (los enlaces se proporcionarán más adelante en la guía).
 ** Acceso a la red y credenciales para cargar imágenes en un registro Docker privado.
 ** Un Kubeconfig configurado para instalar las cartas Helm en los clusters de destino.

== Preparación de las imágenes Docker y el gráfico Helm

Ejecute los siguientes comandos en el host local para obtener las imágenes Docker y los gráficos Helm necesarios:

*Añadir repositorios Helm a la caché local de Helm:*

[,bash]
----
# Adding the Helm repository for the SUSE Observability Agent
helm repo add suse-observability https://charts.rancher.com/server-charts/prime/suse-observability
helm repo update
----

*Obtener la última versión del gráfico*

El siguiente comando descargará un archivo TGZ del gráfico desde el repositorio Helm:

[,bash]
----
# Downloading the chart for the SUSE Observability Agent
# The file will be named stackstate-agent-X.Y.Z.tgz
helm fetch suse-observability/suse-observability-agent
----

*Obtención de los scripts Bash para guardar imágenes Docker.*

[,bash]
----
# o11y-agent-get-images.sh
curl -LO https://raw.githubusercontent.com/StackVista/helm-charts/master/stable/suse-observability-agent/installation/o11y-agent-get-images.sh
# o11y-agent-save-images.sh
curl -LO https://raw.githubusercontent.com/StackVista/helm-charts/master/stable/suse-observability-agent/installation/o11y-agent-save-images.sh

# Make the scripts executable
chmod a+x o11y-agent-get-images.sh o11y-agent-save-images.sh
----

*Extracción y almacenamiento de imágenes Docker*

[,bash]
----
# Extract the list of images from the Helm chart and save it to a file.
./o11y-agent-get-images.sh -f suse-observability-agent-X.Y.Z.tgz > o11y-agent-images.txt
----

[NOTE]
====
Sustituya `+suse-observability-agent-X.Y.Z.tgz+` por el nombre real del archivo de gráficos descargado anteriormente.*
====


[,bash]
----
# Save Docker images to an archive.
# The script expects the file o11y-agent-images.txt to contain the list of images used by the SUSE Observability Agent.
# The Docker images will be saved to o11y-agent-images.tar.gz.
./o11y-agent-save-images.sh -i o11y-agent-images.txt -f o11y-agent-images.tar.gz
----

== Copia de los archivos necesarios en el host remoto

Los siguientes archivos deben copiarse del host local al host de la red privada:

* o11y-agent-images.txt (Lista de imágenes requeridas por la tabla del Agente de SUSE Observability)
* o11y-agent-images.tar.gz (Un archivo con las imágenes Docker del Agente de SUSE Observability)
* https://raw.githubusercontent.com/StackVista/helm-charts/master/stable/suse-observability-agent/installation/o11y-agent-load-images.sh[o11y-agent-load-images.sh] (secuencia de comandos Bash para cargar imágenes Docker en un registro)
* Cartas de timón descargadas anteriormente:
 ** suse-observability-agent-X.Y.Z.tgz

== Restauración de imágenes Docker del archivo al registro privado

*Carga de las imágenes en el registro privado:*

[,bash]
----
# Load Docker images from the archive and push them to the private registry.
# Replace <private-registry> with your private registry's URL.
export DST_REGISTRY_USERNAME="..."
export DST_REGISTRY_PASSWORD="..."
./o11y-agent-load-images.sh -d registry.example.com:5043 -i o11y-agent-images.txt -f o11y-agent-images.tar.gz
----

[NOTE]
====
_Nota: si el registro de destino no utiliza autenticación, las variables de entorno `+DST_REGISTRY_USERNAME+` y `+DST_REGISTRY_PASSWORD+` no deben estar configuradas o deben tener valores vacíos._
====


== Instalación del agente de SUSE Observability

El comando para instalar el agente de SUSE Observability debe recibirse desde la interfaz de usuario de SUSE Observability.
Acceda a su instancia y, en el menú de la izquierda, seleccione `+Stackpacks+`. Pulse `+ADD NEW INSTANCE+` e introduzca el nombre del cluster. Para una correcta integración con Rancher tiene que ser el mismo que el nombre del cluster en el Rancher UI.

image::rancher-prime-agent-airgap-01.png[Añadir nuevo agente]

Cuando se añada una instancia, la interfaz de usuario proporcionará instrucciones sobre cómo desplegar el gráfico Helm. Desplácese hasta la sección `+Self-hosted+` y copie el comando `+helm upgrade ...+` 

image::rancher-prime-agent-airgap-02.png[Obtención del comando de instalación de Helm]

El comando tiene que ser actualizado para la instalación air-gapped:

* Anulación del registro de imágenes con el valor `+all.image.registry+`.
* Usando el archivo con el gráfico de Helm en lugar del repositorio de Helm. `+suse-observability/suse-observability-agent+` \-> `+./suse-observability-agent-X.Y.Z.tgz+`

Ejecute el comando para instalar el agente de SUSE Observability

[,bash]
----
helm upgrade --install \
--namespace suse-observability \
--create-namespace \
--set-string 'stackstate.apiKey'='<api-key>' \
--set-string 'stackstate.cluster.name'='<cluster-name>' \
--set-string 'stackstate.url'='https://...' \
--set 'nodeAgent.skipKubeletTLSVerify'=true \
--set-string 'all.image.registry'='registry.acme.com:5000' \
--set-string 'global.imageRegistry'='registry.acme.com:5000' \
--set-string 'global.skipSslValidation'=true \
suse-observability-agent ./suse-observability-agent-X.Y.Z.tgz
----

*Validación de la implantación*

[,bash]
----
kubectl get pod -n suse-observability
----
