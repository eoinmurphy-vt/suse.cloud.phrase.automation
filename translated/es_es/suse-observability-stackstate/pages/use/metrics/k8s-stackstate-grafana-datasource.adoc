= SUSE Observability como fuente de datos de Grafana
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: Utilizar SUSE Observability como fuente de datos de Grafana

SUSE Observability puede utilizarse como fuente de datos para Grafana. Esto le permitirá utilizar Grafana como herramienta de visualización para sus datos de SUSE Observability. Esto es útil si ya tiene algunos cuadros de mando que desea seguir utilizando. Dado que SUSE Observability expone una API compatible con Prometheus, puede utilizar la https://grafana.com/docs/grafana/latest/datasources/prometheus[fuente de datos] en Grafana para conectarse a SUSE Observability. Esto también hace que SUSE Observability pueda utilizarse con otras soluciones compatibles con Prometheus.

== Requisitos previos

Antes de añadir SUSE Observability como fuente de datos en Grafana, debe configurar un ServiceToken para autenticarse con SUSE Observability. SUSE Observability recomienda crear una función dedicada con permisos para este fin.

Puede hacerlo a través de la CLI de SUSE Observability:

[,sh]
----
> sts rbac create-subject --subject grafana
✅ Created subject 'grafana'
> sts rbac grant --subject grafana --permission get-metrics
✅ Granted permission 'get-metrics' on 'system' to subject 'grafana'
PERMISSION  | RESOURCE
get-metrics | system
----

Esto creará un nuevo rol en SUSE Observability llamado `+grafana+` y le otorgará el permiso `+get-metrics+`. A continuación, puede crear un ServiceToken para esta función:

[,sh]
----
> sts service-token create --name grafana --roles grafana
✅ Service token created: svctok-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
----

Más información sobre la xref:/setup/security/authentication/service_tokens.adoc[gestión de ServiceTokens].

El ServiceToken devuelto puede utilizarse para autenticarse con SUSE Observability. Ahora puede añadir SUSE Observability como fuente de datos en Grafana.

== Crear un nuevo origen de datos de SUSE Observability en Grafana

Con el ServiceToken creado, ya puede añadir SUSE Observability como fuente de datos en Grafana. Para ello, vaya a la interfaz de usuario de Grafana y navegue hasta la página de configuración del origen de datos. Haga clic en el botón `+Add data source+` y seleccione `+Prometheus+` en la lista de fuentes de datos.

image::k8s/k8s-grafana-new-datasource.png[Nuevo origen de datos de Grafana]

En la página de configuración del origen de datos, introduzca los siguientes detalles de configuración:

* *Nombre*: Observabilidad de SUSE
* *URL*: `+https://<tenant-name>.app.stackstate.io/prometheus+`
* *Cabeceras HTTP personalizadas*
 ** *Cabecera*: `+X-API-Key+`
 ** *Valor*: `+<service-token>+`

image::k8s/k8s-grafana-datasource.png[Configuración de la fuente de datos de Grafana]

Haga clic en el botón `+Save & Test+` para guardar la fuente de datos. Si la configuración es correcta, debería ver un mensaje verde `+Data source is working+`.

== Restringir el acceso a las métricas

Cuando se utiliza xref:/setup/security/rbac/rbac_rancher.adoc[Rancher RBAC], es posible conceder acceso a las métricas a nivel de clúster o espacio de nombres.

Para conceder acceso a todas las métricas de un clúster:
[,sh]
----
> sts rbac grant --subject grafana --permission get-metrics --resource k8s:YOUR_CLUSTER:__any__
✅ Granted permission 'get-metrics' on 'k8s:YOUR_CLUSTER:__any__' to subject 'grafana'
PERMISSION  | RESOURCE
get-metrics | k8s:YOUR_CLUSTER:__any__
----

Para conceder acceso a todas las métricas de un espacio de nombres:
[,sh]
----
> sts rbac grant --subject grafana --permission get-metrics --resource k8s:OTHER_CLUSTER:SINGLE_NAMESPACE
✅ Granted permission 'get-metrics' on 'k8s:OTHER_CLUSTER:SINGLE_NAMESPACE' to subject 'grafana'
PERMISSION  | RESOURCE
get-metrics | k8s:OTHER_CLUSTER:SINGLE_NAMESPACE
----

Inspeccione todos los permisos que se han concedido al sujeto:
[,sh]
----
> sts rbac describe-permissions --subject grafana
Got subject from the following subject sources: Observability

PERMISSION  | RESOURCE
get-metrics | k8s:OTHER_CLUSTER:YOUR_NAMESPACE
get-metrics | k8s:YOUR_CLUSTER:__any__
----
