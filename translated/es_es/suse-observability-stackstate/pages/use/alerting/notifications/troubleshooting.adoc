= Resolución de problemas con las notificaciones
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: Observabilidad de SUSE

Cuando un canal de notificación no puede enviar notificaciones, el resumen de notificaciones mostrará un estado de error. Para inspeccionar los errores:

. Haga clic en la notificación
. La parte superior de la página muestra un banner con un resumen de los errores
. Desplácese hasta los canales, haga clic en el enlace "error" del canal o canales afectados para abrir los detalles del error.

== Errores comunes

=== Slack

Los errores de Slack suelen contener un código de error:

* `+token_revoked+`, `+token_expired+` o `+missing_scope+`: Todos estos códigos de error indican un problema con la autorización de SUSE Observability para publicar mensajes en Slack. Para solucionarlo, vuelva a crear el canal para volver a autorizar SUSE Observability con Slack de forma que se concedan todos los permisos necesarios.
* Para otros códigos de error, consulte https://api.slack.com/automation/cli/errors[Slack API error codes].

=== HTTP

La mayoría de los canales, específicamente el webhook, esperan una respuesta HTTP con estado 200 OK. Otras respuestas se consideran un error. Los errores HTTP contienen un mensaje, normalmente breve, y el código de estado que se recibió en lugar de 200 OK.

Compruebe que las URL y los tokens configurados son correctos y, en el caso del webhook, que las solicitudes POST se gestionan correctamente y devuelven una respuesta 200 OK.

=== Otros

En caso de que el error sea del tipo "Otro", lo más probable es que SUSE Observability ni siquiera pueda conectarse al servicio externo o webhook. Compruebe que la URL es correcta, puede resolverse y SUSE Observability puede acceder a ella.
