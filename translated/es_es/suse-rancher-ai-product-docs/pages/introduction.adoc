= Conoce a {project-name}
:revdate: 2026-01-23
:page-revdate: {revdate}
:description: Presentando a {project-name}, un producto de SUSE® Rancher Prime.

{short-project-name} es un agente de IA consciente del contexto integrado en el entorno de SUSE Rancher Prime. A diferencia de la IA de propósito general, {short-project-name} está diseñado para entender el estado en vivo de tus clústeres, espacios de nombres y cargas de trabajo.

Disponible como una extensión, {short-project-name} reside dentro de tu espacio de trabajo para proporcionar orientación en tiempo real donde la necesites. Ya sea que estés investigando un despliegue fallido o tratando de entender un nuevo concepto de Kubernetes, {short-project-name} utiliza el Protocolo de Contexto del Modelo (MCP) para cerrar la brecha entre tus datos operativos y las ideas accionables.

Ya sea que necesites la opinión de un ingeniero de fiabilidad del sitio (SRE) o la ayuda de un experto en Rancher, {short-project-name} puede proporcionarte información para ayudarte a tomar la decisión correcta, más rápido. El poder de {short-project-name} radica en su integración con la UI de Rancher y su capacidad para adaptarse a los requisitos de seguridad empresarial.

== Componentes clave

* **Agente de IA**: El puente específico de dominio entre el motor de razonamiento LLM y los servidores MCP de Rancher. 
  Descompone y dirige solicitudes complejas en llamadas a herramientas seguras, aplicando salvaguardas.
  Asegura una ejecución que tiene en cuenta el contexto y sintetiza datos técnicos en respuestas claras y comprensibles.
* **Rancher MCP**: El Protocolo de Contexto del Modelo de Rancher (MCP) es el componente encargado de interactuar con la API de Rancher. Utiliza el mecanismo de API pública de Rancher y lo expone utilizando herramientas personalizadas para proporcionar la mejor experiencia de usuario.
* **Extensión de UI del Asistente de Rancher**: Este componente extiende la UI de Rancher para {short-project-name}. Va más allá de una simple ventana de chat al incrustar {short-project-name} donde tiene sentido en la UI. Integrando al asistente detrás de mensajes de error y distintivos de estado para soporte inmediato y contextual.

=== Soberanía del modelo flexible

Reconociendo que la seguridad es primordial, {short-project-name} ofrece libertad de elección respecto al motor de IA subyacente.

* Conéctate a LLMs locales o en entornos aislados para máxima privacidad de datos utilizando Ollama.
* Utiliza motores públicos para un razonamiento de alto rendimiento, como:
** ChatGPT de OpenAI
** Google’s Gemini
** AWS Bedrock

== ¿Por qué un Asistente de IA?

A medida que los entornos de Kubernetes escalan, la "carga cognitiva" sobre los operadores aumenta exponencialmente. {short-project-name} aborda tres desafíos críticos en las operaciones modernas de plataformas:

. **Cerrando la Brecha de Habilidades**: Kubernetes tiene una curva de aprendizaje notoriamente empinada. {short-project-name} proporciona soporte inmediato a nivel experto a ingenieros que pueden no estar familiarizados con recursos específicos o tipos de errores, reduciendo la dependencia de ingenieros senior "héroes".
. **Acelerando el Tiempo de Resolución**: En lugar de examinar manualmente los registros o buscar documentación externa, los operadores pueden pedir a {short-project-name} que resuma el estado de un espacio de nombres o explique un fallo específico, reduciendo el tiempo medio de resolución (MTTR).
. **Conciencia Proactiva del Sistema**: {short-project-name} permite una postura operativa proactiva. Al hacer preguntas como "¿Qué despliegues están actualmente en mal estado?" o "Resume la salud de este clúster," los equipos pueden identificar y solucionar problemas antes de que se conviertan en interrupciones.
