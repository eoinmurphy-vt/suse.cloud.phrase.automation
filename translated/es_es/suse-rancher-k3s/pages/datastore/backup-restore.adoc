= Copia de seguridad y restauración
:page-languages: [en, ja, ko, zh]
:revdate: 2024-09-11
:page-revdate: {revdate}

La forma en que se realiza la copia de seguridad y la restauración de K3s depende del tipo de almacén de datos que se utilice.

[CAUTION]
====
Además de hacer una copia de seguridad del almacén de datos en sí, también debe hacer una copia de seguridad del archivo token del servidor en `+/var/lib/rancher/k3s/server/token+`.
Debe restaurar este archivo, o pasar su valor a la opción `+--token+`, al restaurar a partir de una copia de seguridad.
Si no se utiliza el mismo valor de token al restaurar, la instantánea quedará inutilizable, ya que el token se utiliza para cifrar datos confidenciales dentro del propio almacén de datos.
====


== Copia de seguridad y restauración con SQLite

No se necesitan comandos especiales para hacer copias de seguridad o restaurar el almacén de datos SQLite.

* Para hacer una copia de seguridad del almacén de datos SQLite, haga una copia de `+/var/lib/rancher/k3s/server/db/+`.
* Para restaurar el almacén de datos SQLite, restaure el contenido de `+/var/lib/rancher/k3s/server/db+` (y el token, como se ha comentado anteriormente).

== Copia de seguridad y restauración con almacén de datos externo

Cuando se utiliza un almacén de datos externo, las operaciones de copia de seguridad y restauración se gestionan fuera de K3s. El administrador de la base de datos deberá realizar una copia de seguridad de la base de datos externa o restaurarla a partir de una instantánea o un volcado.

Se recomienda configurar la base de datos para que realice instantáneas periódicas.

Para obtener más información sobre cómo tomar instantáneas de la base de datos y restaurarla a partir de ellas, consulte la documentación oficial de la base de datos:

* https://dev.mysql.com/doc/refman/8.0/en/replication-snapshot-method.html[Documentación oficial de MySQL]
* https://www.postgresql.org/docs/8.3/backup-dump.html[Documentación oficial de PostgreSQL]
* https://etcd.io/docs/latest/op-guide/recovery/[Documentación oficial de etcd]

== Copia de seguridad y restauración con el almacén de datos etcd integrado

Consulte la xref:cli/etcd-snapshot.adoc[documentación del comando`+k3s etcd-snapshot+` ] para obtener información sobre la realización de operaciones de copia de seguridad y restauración en el almacén de datos etcd integrado.
