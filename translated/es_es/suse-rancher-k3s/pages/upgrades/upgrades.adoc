== Actualización del clúster K3s

xref:upgrades/manual.adoc[Actualizaciones] manuales describe varias técnicas para actualizar su cluster manualmente. También puede utilizarse como base para la actualización mediante herramientas de infraestructura como código de terceros, como https://www.terraform.io/[Terraform].

xref:upgrades/automated.adoc[Actualizaciones automatizadas] describe cómo realizar actualizaciones automatizadas nativas de Kubernetes utilizando el https://github.com/rancher/system-upgrade-controller[controlador de actualizaciones del sistema] de Rancher.

== Advertencias específicas de la versión

* *Traefik:* Si Traefik no está desactivado, K3s versiones 1.20 y anteriores instalarán Traefik v1, mientras que K3s versiones 1.21 y posteriores instalarán Traefik v2, si v1 no está ya presente. Para actualizar Traefik v1 a Traefik v2, consulte la https://doc.traefik.io/traefik/migrate/v1-to-v2/[documentación de] y utilice la https://github.com/traefik/traefik-migration-tool[Traefik].
* *Datos bootstrap K3s:* Si utiliza K3s en una configuración de HA con un almacén de datos SQL externo y sus nodos de servidor (plano de control) no se iniciaron con el indicador CLI `+--token+`, ya no podrá añadir servidores K3s adicionales al clúster sin especificar el token. Asegúrese de que conserva una copia de este testigo, ya que es necesario cuando se restaura a partir de una copia de seguridad. Anteriormente, K3s no exigía el uso de un token cuando se utilizaban almacenes de datos SQL externos.
 ** Las versiones afectadas son <= v1.19.12+k3s1, v1.20.8+k3s1, v1.21.2+k3s1; las versiones parcheadas son v1.19.13+k3s1, v1.20.9+k3s1, v1.21.3+k3s1.
 ** Puede recuperar el valor del token de cualquier servidor ya unido al clúster de la siguiente manera:
+
[,bash]
----
cat /var/lib/rancher/k3s/server/token
----
