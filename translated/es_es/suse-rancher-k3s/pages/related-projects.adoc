= Proyectos relacionados
:page-languages: [en, ja, ko, zh]
:revdate: 2025-03-31
:page-revdate: {revdate}

Los proyectos que apliquen la distribución de K3s son bienvenidos para ayudar a ampliar la comunidad. Estos proyectos ayudan a:

* Explore las capacidades y aplicaciones potenciales del K3.
* Amplíe la funcionalidad de K3 a diferentes plataformas y entornos.
* Simplifique el proceso de creación de grandes clústeres K3s de alta disponibilidad (HA).

== k3s-ansible

Los usuarios que deseen arrancar un clúster K3s multinodo y estén familiarizados con ansible, pueden echar un vistazo al repositorio https://github.com/k3s-io/k3s-ansible[] k3s-io/k3s-ansible. Este conjunto de playbooks de ansible proporciona una forma cómoda de instalar K3 en sus nodos, lo que le permite centrarse en la configuración de su clúster en lugar del proceso de instalación.

== k3sup

Otro proyecto que simplifica el proceso de configuración de un clúster K3s es https://github.com/alexellis/k3sup[] k3sup. Este proyecto, escrito en golang, sólo requiere acceso ssh a sus nodos. También proporciona una forma cómoda de desplegar K3 con almacenes de datos externos, no sólo el etcd integrado.

== autok3s

Otra herramienta de aprovisionamiento, https://github.com/cnrancher/autok3s[autok3s], proporciona una interfaz gráfica de usuario para aprovisionar el clúster k3s a través de una serie de proveedores de nube, máquinas virtuales y máquinas locales. Esta herramienta es útil para los usuarios que prefieren una interfaz gráfica para el aprovisionamiento de clusters K3s.

== hetzner-k3s

Para los usuarios que despliegan en Hetzner Cloud, echa un vistazo a link:https://github.com/vitobotta/hetzner-k3s[hetzner-k3s]. Esta herramienta CLI, escrita en Crystal, gestiona los pasos adicionales necesarios para configurar un clúster K3s en Hetzner Cloud.
