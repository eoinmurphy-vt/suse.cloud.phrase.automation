= Solución de problemas de restauración
:revdate: 2025-07-03
:page-revdate: {revdate}

[WARNING]
====
Cuando se realice una restauración, no reinicie el `+rancher-system-agent+` en los nodos elementales, ya que puede anquilosarse y terminar con el siguiente error:

`+panic: error while connecting to Kubernetes cluster: the server has asked for the client to provide credentials+`

Si te encuentras con este problema, sigue el procedimiento que se indica a continuación.
====


Antes de iniciar una restauración, necesita copiar `+/var/lib/rancher/agent/rancher2_connection_info.json+` desde el nodo elemental a un lugar donde tenga acceso con Rancher UI.

Una vez copiado el archivo, descargue el script `+rancher-agent-token-update.sh+` del https://github.com/rancher/elemental[repositorio]

[,shell]
----
wget -q https://raw.githubusercontent.com/rancher/elemental/main/scripts/rancher-agent-token-update && chmod +x rancher-agent-token-update
----

Ejecuta el script sin opciones adicionales:

[,shell]
----
./rancher-agent-token-update
----

Una vez completada con éxito la restauración, copie `+rancher2_connection_info.json+` de nuevo en el nodo elemental a la ruta`+/var/lib/rancher/agent/rancher2_connection_info.json+`. Por último, reinicie el servicio `+rancher-system-agent+`:

[,shell]
----
systemctl restart rancher-system-agent
----
