= Referencia a ManagedOSVersionChannel
:revdate: 2025-09-08
:page-revdate: {revdate}

El recurso `+ManagedOSVersionChannel+` se encarga de definir el canal de imagen del sistema operativo.

Hay varias claves que pueden configurarse en una especificación de recursos `+ManagedOSVersionChannel+`.

.managedosversionchannel-example.yaml
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: ManagedOSVersionChannel
metadata:
  name: elemental-channel
  namespace: fleet-default
spec:
  options:
    image: registry.suse.com/rancher/elemental-channel:1.4.2
  syncInterval: 1h
  type: custom
----

== `+ManagedOSVersionChannelSpec+` referencia

|===
| Clave | Tipo | Valor por defecto | Descripción

| opciones
| objeto
| null
| Define la información opcional que puede añadirse en un canal OS.

| syncInterval
| cadena
| 1h
| Define cuándo sincronizar el canal OS

| tipo
| cadena
| vacío
| Define el tipo de canal, ahora sólo se admite `+custom+` 

| upgradeContainer
| objeto
| null
| Un contenedor de actualización que se puede definir. Ver <<_upgradecontainer,referencia>>

| deleteNoLongerInSyncVersions
| bool
| falso
| Eliminar automáticamente las versiones de SO obsoletas que ya no se incluyen en el canal.

| habilitado
| bool
| verdadero
| Activa este canal. Permitir la sincronización de versiones del sistema operativo.
|===

== `+upgradeContainer+`

Permite sobrescribir el campo `+upgrade+` predeterminado de los planes del controlador de actualización del sistema (consulte xref:rancher-os-management/upgrade-lifecycle.adoc#_components[los componentes de actualización]) en función de esta ManagedOSVersion.
El controlador de actualización del sistema traduce estas claves a una especificación de Kubernetes https://kubernetes.io/docs/reference/kubernetes-api/workload-resources/pod-v1/#Container[contenedor].
Este es el contenedor responsable de ejecutar una actualización del sistema operativo.
