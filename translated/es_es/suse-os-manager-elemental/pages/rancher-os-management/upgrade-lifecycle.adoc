== Componentes

* https://documentation.suse.com/cloudnative/os-manager/[{elemental-product-name} Operador]

El Operador {elemental-product-name} soporta varias formas de configurar la versión del SO a nivel de Cluster. +
Cada vez que se define un nuevo `+ManagedOSIMage+`, el operador {elemental-product-name} genera también una flota `+Bundle+`.

* https://documentation.suse.com/cloudnative/continuous-delivery/[Fleet]

En este contexto, Fleet se utiliza para aplicar un `+upgrade.cattle.io/v1+` `+Plan+` en cada Cluster al que apunta el `+ManagedOSImage+`. +
A continuación, `+Plan+` es ejecutado por `+System Upgrade Controller+` en el clúster {elemental-product-name}.

* https://github.com/rancher/system-upgrade-controller[Controlador de actualización del sistema]

El controlador de actualización del sistema es una herramienta de actualización utilizada por los clusters https://documentation.suse.com/cloudnative/k3s/latest/en/upgrades/automated.html[] K3s y https://documentation.suse.com/cloudnative/rke2/latest/en/upgrade/automated_upgrade.html[] RKE2. Siempre debe instalarse automáticamente en cada clúster aprovisionado de {elemental-product-name}. En este contexto, el controlador de actualización del sistema orquesta la aplicación de `+upgrade.cattle.io/v1+` `+Plan+` a los nodos {elemental-product-name}.

== Versionado y ciclo de vida de los componentes

Las versiones https://documentation.suse.com/cloudnative/continuous-delivery/[Fleet] y https://github.com/rancher/system-upgrade-controller[System Upgrade Controller] están controladas por Rancher.

En el clúster Rancher, puede ejecutar:

[,bash]
----
kubectl get settings fleet-version system-upgrade-controller-chart-version
----

Para más información sobre las versiones de los gráficos, puede visitar el https://github.com/rancher/charts[repositorio] o leer la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-admin/helm-charts-in-rancher/helm-charts-in-rancher.html[].

La versión de los gráficos la determina Rancher. +
Dependiendo de cómo se instale Rancher, se pueden establecer las siguientes variables de entorno, por ejemplo cuando se instala la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/references/helm-chart-options.html#_setting_extra_environment_variables[tabla]:

[,bash]
----
CATTLE_SYSTEM_UPGRADE_CONTROLLER_CHART_VERSION
CATTLE_FLEET_VERSION
----

En general, no se recomienda actualizar manualmente las versiones. +
También tenga en cuenta que un nuevo `+CATTLE_SYSTEM_UPGRADE_CONTROLLER_CHART_VERSION+` desencadenará una actualización de la tabla `+system-upgrade-controller+` en todos los clústeres posteriores aprovisionados.

== Verificación del éxito de las actualizaciones

Verificar la correcta ejecución de los trabajos de actualización es una parte fundamental para garantizar que el proceso de actualización de {elemental-product-name} se ha completado.

Para solucionar los problemas de cada paso del proceso de actualización, consulte el xref:troubleshooting/troubleshooting-upgrade.adoc[documento correspondiente]. +
Esta sección sólo describe el último paso necesario para verificar la correcta aplicación del Plan de actualización en un clúster objetivo.

* Versiones superiores o iguales a la *v0.13.4* de `+system-upgrade-controller+`:
+
[,shell]
----
kubectl -n cattle-system describe plan os-upgrader-my-upgrade
----
+
Una vez aplicado el Plan a todos los nodos, tendrá una condición de estado `+Complete+`.

* Versiones anteriores:
+
Cada Plan tiene un valor de estado `+latestHash+` que debe coincidir con la etiqueta del Plan en cada nodo. +
Se puede utilizar un sencillo script para listar todos los nodos en los que aún no se ha aplicado el plan:
+
[,shell]
----
#!/bin/bash

# This script prints a list of all nodes where an upgrade plan was not applied.
# To further determine the cause of failures, you can analyze the Plan status and the related jobs.
# For ex: kubectl -n cattle-system get plans,jobs

# Edit this variable according to your ManagedOSImage name.
MANAGED_OS_IMAGE_NAME=my-upgrade

PLAN_NAME=os-upgrader-$MANAGED_OS_IMAGE_NAME
LATEST_HASH=$(kubectl -n cattle-system get plan $PLAN_NAME -o=jsonpath='{.status.latestHash}')

printf "Plan '$PLAN_NAME' latest hash is: '$LATEST_HASH'\n"

PLAN_LABEL=plan.upgrade.cattle.io/$PLAN_NAME
printf "Listing all nodes with mismatching or missing label '$PLAN_LABEL':\n"
kubectl get nodes -l $PLAN_LABEL!=$LATEST_HASH
----
