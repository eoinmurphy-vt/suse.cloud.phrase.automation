Las siguientes instrucciones necesitan Rancher 2.9.x como mínimo.

Esta guía rápida le mostrará cómo implementar el plugin {elemental-product-name} y el operador en una instancia existente de Rancher Manager.

Una vez instalado, podrá aprovisionar un nuevo clúster {elemental-product-name} basado en RKE2 o K3s.

Sin embargo, si desea instalar el operador de staging o dev, sólo puede hacerlo en xref:quickstarts/quickstart-cli.adoc#_non_stable_installations[modo CLI].

== Añadir el repositorio oficial de extensiones de Rancher

Si la extensión Elemental no está disponible, deberá añadir la extensión `+Official Rancher Extensions Repository+`:

image::quickstart-ui-extension-repository.png[Añadir repositorio de extensiones de Rancher Manager]

Si este repositorio no puede añadirse a través de la configuración de la interfaz de usuario de extensiones, puede gestionarse manualmente https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-admin/helm-charts-in-rancher/helm-charts-in-rancher.html#_manage_repositories[añadiendo] al repositorio `+https://github.com/rancher/ui-plugin-charts+` `+git+`, o bien aplicando el siguiente recurso:

[source,yaml]
----
apiVersion: catalog.cattle.io/v1
kind: ClusterRepo
metadata:
  name: rancher-ui-charts
spec:
  gitBranch: main
  gitRepo: https://github.com/rancher/ui-plugin-charts
----

== Instalar el plugin elemental

Una vez habilitado el soporte para las extensiones de Rancher Manager, puede instalar el plugin `+elemental+` de la siguiente manera:

* En la pestaña `+Available+` verá `+elemental+` plugin disponible

image::quickstart-ui-extensions-available.png[Rancher Manager Plugins disponibles]

[NOTE]
====
Si la pestaña `+Available+` no muestra ninguna entrada, actualice la página. A continuación, aparecerá el plugin `+elemental+`.
====

* Haga clic en el botón `+Install+`, aparecerá una ventana emergente y vuelva a hacer clic en `+Install+` para continuar.

image::quickstart-ui-elemental-plugin-install.png[{elemental-product-name} instalación del plugin]

* En la pestaña `+Installed+`, aparece ahora el plugin `+elemental+`.

[NOTE]
====
Si el plugin `+elemental+` aparece en la lista y el estado permanece en `+Installing...+`, actualice la página. El plugin `+elemental+` se mostrará correctamente.
====

Una vez instalado el plugin `+elemental+`, podrá ver la opción `+OS Management+` en el menú Rancher Manager, actualice la página si no la ve.

image::quickstart-ui-elemental-plugin-menu.png[Menú de gestión del SO Rancher Manager]

== Añadir un punto final de registro de máquina

En el panel de gestión del sistema operativo, haga clic en el botón `+Create Registration Endpoint+`.

image::quickstart-ui-registration-endpoint-create.png[Puntos finales de registro de gestión del SO]

Ahora aquí puede introducir cada detalle en sus respectivos lugares o puede editar esto como YAML y crear el punto final de una sola vez. Aquí editaremos todos los campos.

image::quickstart-ui-registration-endpoint-create-details.png[Crear un punto final de registro con interfaz de usuario]

[IMPORTANT]
.Principales opciones
====
`+name: elemental-cluster1+`Modifíquelo según sus necesidades

`+device-selector+`: El campo xref:references/machineregistration-reference.adoc#_config_elemental_install_device_selector[selector de] dispositivo se puede utilizar para elegir dinámicamente el dispositivo durante la instalación. El campo contiene una lista de reglas para seleccionar el dispositivo deseado.

`+snapshotter+`: Tipo de dispositivo utilizado para gestionar instantáneas en imágenes del sistema operativo.
====

Una vez creado el punto final de registro de la máquina, debería aparecer como activo.

image::quickstart-ui-registration-endpoint-complete.png[Máquina registrada en Puntos finales de registro]

== Preparación de la imagen de instalación (semilla)

Ahora este es el último paso, necesitas preparar una imagen semilla que incluya la configuración de registro inicial, para que pueda ser auto registrada, instalada y completamente desplegada como parte de tu cluster. El contenido del archivo no es más que la URL de registro que el nodo necesita para registrarse y el certificado de servidor adecuado, para que pueda conectarse de forma segura.

Esta imagen semilla puede utilizarse para aprovisionar un número infinito de máquinas.

La imagen semilla se crea como un recurso Kubernetes arriba y se puede construir utilizando el botón `+Build Media+`, pero primero, usted tiene que seleccionar ISO o imagen RAW.

A diferencia de la ISO que necesita dos dispositivos (un dispositivo con la ISO y otro disco donde instalar {elemental-product-name}), la imagen RAW permite arrancar desde un único dispositivo e instalar directamente el sistema operativo en el dispositivo.
La imagen RAW sólo contiene una partición de arranque y otra de recuperación y arranca primero en modo recuperación para instalar {elemental-product-name} (a título informativo, el proceso es similar al de xref:node-operational-tasks/reset.adoc#_reset_workflow[reinicio] ).

image::quickstart-ui-registration-endpoint-build-media.png[Crear medios en puntos finales de registro]

Una vez finalizada la compilación, se pueden descargar los archivos multimedia con el botón `+Download Media+`:

image::quickstart-ui-registration-endpoint-download-media.png[Descargar medios en puntos finales de registro]

Ahora puede arrancar sus nodos con esta imagen y lo harán:

* Regístrese con la URL de registro indicada y cree una cuenta por máquina `+MachineInventory+`
* Instala SLE Micro en el dispositivo indicado
* Reiniciar

== Inventario de máquinas

Cuando los nodos arrancan por primera vez, se conectan a Rancher Manager y se crea un xref:references/machineinventory-reference.adoc[`+Machine Inventory+`] para cada nodo.

image::quickstart-ui-machine-inventory-menu.png[Menú Inventario de máquinas]

Las columnas personalizadas se basan en `+Machine Inventory Labels+` que puede añadir al crear su `+Machine Registration Endpoint+`:

image::quickstart-ui-registration-endpoint-hardware-labels.png[Registro de máquinas Etiquetas de hardware de punto final]

En la siguiente captura de pantalla, `+Hardware Labels+` se utiliza como columnas personalizadas:

También puede añadir columnas personalizadas haciendo clic en el menú de los tres puntos.

image::quickstart-ui-machine-inventory-custom-columns.png[Columnas personalizadas de Inventario de máquinas]

Por último, también puede filtrar su `+Machine Inventory+` utilizando esas etiquetas.

Por ejemplo, si sólo quieres ver tus máquinas AMD, puedes filtrar en `+CPUModel+` como se muestra a continuación:

image::quickstart-ui-machine-inventory-filtering.png[Filtrado del inventario de máquinas]

== Cree su primer clúster {elemental-product-name} 

Ahora vamos a utilizar esos `+Machine Inventory+` para crear un clúster haciendo clic en `+Create {elemental-product-name} Cluster+`:

image::quickstart-ui-create-cluster-button.png[Botón Crear clúster {elemental-product-name} ]

Para su clúster {elemental-product-name}, puede elegir K3s o RKE2 para Kubernetes.

image::quickstart-ui-create-cluster-standard-screen-.png[{elemental-product-name} Pantalla de creación de clusters]

La mayoría de las opciones provienen de Rancher, por eso no detallaremos todas las posibilidades.
No dude en consultar la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/k3s.html[documentación de configuración de la distribución] o la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/rke2.html[Rancher] si desea obtener más información.

Sin embargo, es importante destacar la sección `+Inventory of Machines Selector Template+`.

Le permite elegir qué `+Machine Inventory+` desea utilizar para crear su clúster {elemental-product-name} utilizando el `+Machine Inventory Labels+` definido previamente:

image::quickstart-ui-create-cluster-machine-selector-template.png[Utilizar plantilla de selección de inventario de máquinas]

Como nuestros tres inventarios de máquinas contienen la etiqueta `+CPUVendor+` con la clave `+AuthenticAMD+`, las tres máquinas se utilizarán para crear el clúster {elemental-product-name}.
