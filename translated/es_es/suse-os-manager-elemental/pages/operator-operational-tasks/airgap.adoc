= Instale {elemental-product-name} en un entorno sin ventilación
:revdate: 2025-11-06
:page-revdate: {revdate}

== Supuestos

Una instalación air-gapped debe estar ya configurada según la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/air-gapped/air-gapped.html[Rancher].
En particular, debería existir un registro privado en la infraestructura con cobertura aérea.

== Visión general

Para ejecutar {elemental-product-name} en un entorno air-gapped se necesitan los siguientes artefactos:

* los gráficos de {elemental-product-name} Operator
* las imágenes contenedoras a las que se hace referencia en los gráficos (las imágenes _elemental-operator_ y _seedimage-builder_ )
* las imágenes del SO en contenedores

Además, podría ser útil crear una _imagen de canal_ que haga referencia a las imágenes de SO en contenedor disponibles.
La imagen del canal oficial (el _canal elemental_ ) hace referencia a las URL absolutas de las imágenes del sistema operativo en el registro oficial de suse, por lo que no se puede utilizar tal cual en un escenario de air-gapped.

== {elemental-product-name} Instalación de Air-Gapped desde la línea de comandos

Todos los pasos necesarios pueden realizarse ejecutando elhttps://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[script]

Los gráficos de {elemental-product-name} son un parámetro necesario para el script y pueden proporcionarse como archivos descargados, URL o como una de las palabras clave `+stable+`, `+staging+` y `+dev+`, para que el script recupere la versión correcta del gráfico.

`+elemental-airgap.sh+` inspecciona el gráfico {elemental-product-name} Operator, identifica todas las imágenes de contenedor necesarias, las descarga y las guarda en un único archivo docker.
También construye una nueva imagen de canal del SO con las URL de la imagen del SO apuntando al registro privado pasado como argumento (que también es un argumento obligatorio).

La última versión del script elemental puede descargarse fácilmente del repositorio oficial de github:

[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh
chmod 755 elemental-airgap.sh
----

Descarguemos ahora todos los artefactos y creemos un canal personalizado a partir de la última versión estable de {elemental-product-name}:

[tabs]
====
Crear un archivo Docker::
+
======
[,shell]
----
./elemental-airgap.sh stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

una vez completado (el script puede tardar un rato) los siguientes archivos estarán disponibles en el directorio actual:

* `+elemental-operator-crds-chart-<*VERSION*>.tgz+`
* `+elemental-operator-chart-<*VERSION*>.tgz+`
* `+elemental-images.txt+`
* `+elemental-images.tar.gz+`
======

Crear un archivo de transportistas::
+
======
[,shell]
----
./elemental-airgap.sh -ha stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

una vez completado (el script puede tardar un poco) tanto las cartas como las imágenes de los contenedores se empaquetarán en el archivo del transportista llamado `+elemental-haul.tar.zst+`.
======
====

=== {elemental-product-name} instalación 

Los ficheros y archivos creados por el script deben copiarse en un host que: 

* Tiene acceso al registro privado. 
* Tiene el binario kubectl instalado y configurado para acceder al clúster Rancher. 
* Tiene instalado el binario del timón. 

[tabs]
========
Instalar desde un archivo Docker::
+
======
Si el registro privado requiere autenticación, debe iniciar sesión en él con Docker: 

[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

Para realizar la instalación de {elemental-product-name} son necesarios dos pasos: 

* cargar el archivo con todas las imágenes de contenedor necesarias en el registro privado: esto se puede hacer utilizando el script `+rancher-load-images.sh+` distribuido con la versión de Rancher y ya utilizado para el despliegue air-gapped de Rancher: 

[,shell]
----
rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

* instalar los gráficos elementales descargados configurando el registro local y el canal recién creado: 

[,shell]
----
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ 
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM>
----
======

Instalar desde un archivo Hauler::
+
======
Para instalar desde un archivo https://rancherfederal.github.io/hauler-docs/[Hauler] (`+-ha+` opción en `+elemental-airgap.sh+`) La instalación Hauler también es un requisito en el host desde el que se realiza la instalación. 

Si el registro privado requiere autenticación, debe iniciar sesión en él con Hauler: 

[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD
----

Para realizar la instalación de {elemental-product-name} son necesarios tres pasos: 


* Carga el archivo Haul "elemental-haul.tar.zst" en la instancia Hauler de la infraestructura airgapped: 

[,shell]
----
hauler store load 'elemental-haul.tar.zst' 
----

* Si el registro local en el entorno air-gapped no es servidor por Hauler, cargue el archivo Haul en el registro local: 

[,shell]
----
hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

[NOTE]
.El transportista también puede servir de registro
====
En el caso de que el registro local de air-gapped esté servido por una instancia de Hauler, simplemente cargue el archivo Haul directamente allí (como se muestra en el paso (1)) y sáltese el paso (2).
====

* Extrae las cartas elementales de la tienda Hauler e instálalas: 

[,shell]
----
hauler store extract elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
hauler store extract elemental-operator-chart-<ELEMENTAL-VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<ELEMENTAL-VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ -
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM:PORT>
----
======
========

[NOTE]
.El script elemental airgap emite los comandos necesarios 
====
Los scripts de `+elemental-airgap.sh+` imprimen los comandos necesarios que se muestran arriba, pero utilizando la versión real del gráfico y la URL del registro proporcionada para permitir copiar y pegar fácilmente los comandos exactos.
====

== {elemental-product-name} Instalación Air-Gapped del Mercado Ranchero 

Una instalación Rancher air-gapped incluye también los gráficos {elemental-product-name} Operator y las imágenes del operador y del contenedor seedimage. 

Para recopilar las imágenes de SO que faltan y construir una imagen de canal de SO para su registro privado ejecute el https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[script] `+-co+`. 

A modo de ejemplo, utilicemos la imagen `+elemental-channel+` de la última versión estable de {elemental-product-name}. El script se encargará de descargar el gráfico de operador de {elemental-product-name} (si es necesario), extraer la URL de la imagen del canal OS, descargarla, inspeccionar todas las imágenes OS referenciadas, descargarlas todas y crear un nuevo canal OS con enlaces al registro privado del escenario air-gapped. 

[tabs]
========
Crear un archivo Docker::
+
======
[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh 
chmod 755 elemental-airgap.sh 
./elemental-airgap.sh stable -co -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

una vez completado (el script puede tardar un rato) los siguientes archivos estarán disponibles en el directorio actual: 

- `+elemental-operator-crds-chart-<*VERSION*>.tgz+` 
- `+elemental-operator-chart-<*VERSION*>.tgz+` 
- `+elemental-images.txt+` 
- `+elemental-images.tar.gz+`
======

Crear un archivo de transportistas::
+
======
[,shell]
----
./elemental-airgap.sh -ha -co stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

una vez completado (el script puede tardar un poco) las imágenes del contenedor se empaquetarán en el archivo del transportista llamado `+elemental-haul.tar.zst+`.
======
========

=== {elemental-product-name} instalación 

El archivo generado debe cargarse en el registro privado de air-gapped. 

[tabs]
========
Instalar desde un archivo Docker::
+
======
Si el registro privado requiere autenticación, debe iniciar sesión en él con Docker: 
[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

El script imprimirá los comandos necesarios para cargar las imágenes a través de la herramienta de Rancher `+rancher-load-images.sh+`, utilizada para las instalaciones de air-gapped de Rancher. Debería ser algo como: 

[,shell]
----
NEXT STEPS: 

1) Load the 'elemental-images.tar.gz' to the local registry (<REGISTRY.YOURDOMAIN.COM:PORT>) available in the airgapped infrastructure: 

./rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

Una vez cargadas las imágenes del sistema operativo y del canal, debe omitir el punto (2) de la salida del script (que instalará los gráficos de {elemental-product-name} desde los archivos descargados) y, en su lugar, realizar la instalación de {elemental-product-name} Operator desde la interfaz de usuario de Rancher.
======

Instalar desde un archivo Hauler::
+
======
Si el registro privado requiere autenticación, debe iniciar sesión en él con Hauler: 
[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD 
----

El script imprimirá los comandos necesarios para cargar las imágenes. Debería ser algo como: 

[,shell]
----
NEXT STEPS: 

* Load the 'elemental-haul.tar.zst' Haul archive in the Hauler instance in the airgapped infrastructure: 

hauler store load 'elemental-haul.tar.zst' 

* If the local registry in the air-gapped environment is not server by Hauler, load the Haul archive in the local registry: 

hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

Una vez cargadas las imágenes del sistema operativo y del canal, debe omitir el punto (3) de la salida del script (que instalará los gráficos de {elemental-product-name} desde los archivos descargados) y, en su lugar, realizar la instalación de {elemental-product-name} Operator desde la interfaz de usuario de Rancher.
======
========

== {elemental-product-name} Extensión de la interfaz de usuario 

Rancher 2.7.x no es compatible con el plugin de extensiones de interfaz de usuario en entornos air-gapped, por lo que la interfaz de usuario {elemental-product-name} no está disponible en Rancher 2.7.x. 

El plugin {elemental-product-name} UI estará presente en las extensiones UI disponibles en Rancher 2.8.0.
