= Cómo añadir un certificado personalizado
:revdate: 2024-09-24
:page-revdate: {revdate}

Requisito previo: Un certificado en formato `+.pem+` 

Objetivo: Poner un certificado personalizado a disposición de todo el sistema

[NOTE]
.Esto es para los certificados utilizados por los servicios a nivel de sistema.
====
Las cargas de trabajo de Kubernetes deben traer sus certificados dentro de la imagen del contenedor en su lugar.
====

Para instalar un certificado personalizado necesitamos

* copie el archivo `+.pem+` en `+/etc/pki/trust/anchors/+`
* ejecute `+update-ca-certificates+`

El fragmento correspondiente de `+cloud-config+` tiene el siguiente aspecto:

[,yaml]
----
write_files:
  - path: /etc/pki/trust/anchors/my-custom-certificate.pem
    permission: 0444
    content: |-
      -----BEGIN CERTIFICATE-----
      ...
      -----END CERTIFICATE-----
runcmd:
  - update-ca-certificates
----

(se omite el contenido real del certificado por razones de brevedad)
