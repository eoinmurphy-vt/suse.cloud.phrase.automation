= Cómo crear imágenes de disco en bruto para Raspberry Pi
:revdate: 2024-09-24
:page-revdate: {revdate}

Esta guía mostrará cómo podemos construir una imagen de disco sin formato que se puede escribir en una tarjeta SD y arrancar sin ningún otro medio de instalación.

[CAUTION]
====
Cualquier dato en la tarjeta SD se borrará, por favor utilice sólo una tarjeta SD sin nada importante en ella.

La tarjeta SD debe ser razonablemente grande (32 GB o más) y *rápida* (!!).
====

.Recurso SeedImage
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: fire-img
  namespace: fleet-default
spec:
  type: raw
  baseImage: registry.opensuse.org/isv/rancher/elemental/staging/containers/suse/sl-micro/6.0/baremetal-os-container:latest
  targetPlatform: linux/arm64
  registrationRef:
    apiVersion: elemental.cattle.io/v1beta1
    kind: MachineRegistration
    name: fire-nodes
    namespace: fleet-default
----

Compruebe los registros para el pod de construcción utilizando:

[,shell]
----
kubectl logs -n fleet-default fire-img -f -c build
----

Una vez finalizada la compilación, podemos descargar el archivo de imagen utilizando wget:

[,shell]
----
wget --no-check-certificate $(kubectl get seedimage -n fleet-default fire-img -o jsonpath="{.status.downloadURL}") -O sle-micro.arm64.raw
----

Ahora podemos escribir la imagen `+.raw+` en la tarjeta SD. Esto se puede hacer con `+dd+` en la línea de comandos de Linux si se siente cómodo con este comando.https://www.opensuse.org[openSUSE] tiene buenas instrucciones sobre cómo escribir una imagen en un medio de almacenamiento para https://en.opensuse.org/SDB:Live_USB_stick[Linux],https://en.opensuse.org/SDB:Create_a_Live_USB_stick_using_Windows[Windows], y https://en.opensuse.org/SDB:Create_a_Live_USB_stick_using_macOS[OS X].

== Puesta en marcha de la máquina

La imagen de disco raw sólo incluirá la partición EFI, la partición OEM y la partición de recuperación. En el primer arranque, el sistema arrancará en el sistema de recuperación para ampliar y añadir las particiones que faltan. Después de la expansión se registrará con rancher y se reiniciará.

Si se produce un error durante la fase de registro, el diario puede encontrarse utilizando`+journalctl -u elemental-register-reset+`.
