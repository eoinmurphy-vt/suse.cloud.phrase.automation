== Componentes

A continuación se muestra una representación visual de los componentes de la arquitectura de Rancher Turtles. Este diagrama ilustra los elementos clave y sus relaciones dentro del sistema {product_name}. Entender estos componentes es esencial para comprender cómo Rancher aprovecha la API de clúster (CAPI) para la gestión de clústeres.

image::30000ft_view.png[visión general,640,480]

=== Administrador de ranchos

Este es el componente central de Rancher y los usuarios pueden aprovechar la función Explorer existente en el panel de control para acceder a los detalles de la carga de trabajo del clúster.

=== Agente Rancher Cluster

El agente se despliega dentro de los clusters hijos, permitiendo a Rancher importar y establecer una conexión con estos clusters. Esta conexión permite a Rancher gestionar eficazmente los clusters hijos desde su plataforma.

=== {product_name} - Ganadero Extensión CAPI

Proporciona integración entre CAPI y Rancher, soportando actualmente las siguientes funcionalidades:

* *Importación de clusters CAPI en Rancher:* instalación del Agente de Cluster Rancher en clusters provisionados CAPI.
* *Configuración del operador CAPI:* Soporte de configuración para el Operador CAPI.

== Topología de despliegue

=== Gestión combinada de ganadero y CAPI

Esta es la única topología soportada actualmente, en la que tanto Rancher Manager como {product_name} se despliegan en el mismo clúster Kubernetes, y actúa como un clúster de gestión centralizado.

Esta arquitectura ofrece un despliegue simplificado de componentes y proporciona una vista única de todos los clústeres. Por otro lado, es importante tener en cuenta que el número de clústeres que se pueden gestionar eficazmente mediante Cluster API (CAPI) está limitado por los recursos disponibles en el clúster de gestión único.
