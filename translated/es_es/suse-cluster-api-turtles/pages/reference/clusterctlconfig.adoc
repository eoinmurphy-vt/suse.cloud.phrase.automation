= ClusterctlConfig
:revdate: 2025-11-14
:page-revdate: {revdate}

El recurso `+ClusterctlConfig+` permite gestionar overrides para configuraciones de clusterctl (Operador CAPI) de forma declarativa. Se utiliza para configurar los proveedores de clusterctl y sus urls, así como las restricciones de versión.

`+ClusterctlConfig+` sigue un modelo GitOps: los campos de especificaciones son entradas de usuario declarativas. Turtles no crea ni actualiza el recurso, sino que es el usuario quien debe especificar las anulaciones de la url del proveedor y mantener su estado. Tiene prioridad sobre los valores predeterminados incrustados o el conjunto predeterminado de definiciones de proveedores de `+clusterctl+`.

https://github.com/rancher/turtles/blob/main/docs/adr/0012-clusterctl-provider.md[ADR]

== Utilización

Para utilizar el recurso `+ClusterctlConfig+`:

. Cree un recurso `+ClusterctlConfig+` con el nombre `+clusterctl-config+` en el espacio de nombres `+turtles+`.
. El controlador `+ClusterctlConfig+` gestionará las actualizaciones para el `+ConfigMap+` montado en el `+cluster-api-operator+` con la configuración clusterctl requerida basada en la especificación `+ClusterctlConfig+`.
. Gestione el objeto `+ClusterctlConfig+` de forma declarativa para aplicar cambios a las configuraciones de proveedores generadas. Es posible que los cambios tarden algún tiempo en surtir efecto, ya que `+kubelet+` se encarga de actualizar el punto montado en función del estado de `+ConfigMap+`.

He aquí un ejemplo de manifiesto `+ClusterctlConfig+`:

[,yaml]
----
apiVersion: turtles-capi.cattle.io/v1alpha1
kind: ClusterctlConfig
metadata:
  name: clusterctl-config
  namespace: cattle-turtles-system
spec:
  providers:
  - name: metal3
    url: https://github.com/metal3-io/cluster-api-provider-metal3/releases/v1.10.1/infrastructure-components.yaml
    type: InfrastructureProvider
  images:
  - name: metal3
    repository: "my.registry.mirror"
    tag: customTag
----

Este ejemplo generará una configuración clusterctl para el proveedor `+metal3+` con la URL y el tipo especificados.

== Especificación

Los campos clave de la especificación `+ClusterctlConfig+` son:

* `+providers[].name+` - Nombre del proveedor (por ejemplo, metal3)
* `+providers[].url+` - URL de la configuración del proveedor (por ejemplo
https://github.com/metal3-io/cluster-api-provider-metal3/releases/v1.10.1/infrastructure-components.yaml). Puede utilizar la versión `+latest+`, si es compatible, o fijar la versión máxima en `+v1.10.1+`, por ejemplo.
* `+providers[].type+` - Tipo de proveedor (por ejemplo, InfrastructureProvider)

* `+images[].name+` - El nombre de la imagen que se va a sustituir (por ejemplo, infrastructure-metal3, cluster-api, bootstrap-rke2, control-plane-rke2).
* `+images[].repository+` - Establece el registro de contenedores del que se extraerán las imágenes
* `+images[].tag+` - Permite especificar una etiqueta para las imágenes

== Anular una versión de proveedor certificado

{product_name} incorpora una configuración validada en la que se fijan todas las versiones certificadas del proveedor. +
Normalmente las versiones se actualizan con cualquier relese de {product_name}.

[WARNING]
====
- {product_name} sólo valida la configuración integrada para los proveedores certificados. Cualquier personalización de la configuración integrada puede dar lugar a incompatibilidades y situaciones no compatibles. 
====

Para anular una versión del proveedor, o para utilizar un registro diferente para extraer imágenes, se puede definir un `+ClusterctlConfig+`. +
Para más información sobre la anulación de imágenes, consulte la link:https://cluster-api.sigs.k8s.io/clusterctl/configuration#image-overrides[documentación de].

[source,yaml]
----
apiVersion: turtles-capi.cattle.io/v1alpha1
kind: ClusterctlConfig
metadata:
  name: clusterctl-config
  namespace: cattle-turtles-system
spec:
  providers:
  - name: rke2
    url: "https://github.com/rancher/cluster-api-provider-rke2/releases/v0.20.1/control-plane-components.yaml"
    type: ControlPlaneProvider
  - name: rke2
    url: "https://github.com/rancher/cluster-api-provider-rke2/releases/v0.20.1/bootstrap-components.yaml"
    type: BootstrapProvider
  images:
  - name: control-plane-rke2
    repository: "ghcr.io/rancher"
  - name: bootstrap-rke2
    repository: "ghcr.io/rancher"
----

== Supresión

Cuando se elimina un recurso de `+ClusterctlConfig+`, el mapa de configuración vuelve a su estado original, gestionado por tortugas.
