Como se indica en el https://swift.org/[sitio web oficial]

____
Swift es un lenguaje de programación de uso general, accesible para los recién llegados y potente para los expertos.
Es rápido, moderno, seguro y un placer escribir.
____

El compilador swift aún no tiene soporte para WebAssembly, sin embargo el proyectohttps://swiftwasm.org/[SwiftWasm] proporciona un compilador parcheado con esta capacidad.

El equipo de SwiftWasm está trabajando para integrar estos cambios en el proyecto Swift.
Mientras tanto, puede utilizar la cadena de herramientas proporcionada por el proyecto SwiftWasm para crear políticas {project-name}.

[NOTE]
====

No necesitas un sistema Apple para escribir o ejecutar código Swift. Todo se puede hacer también en una máquina Linux o en Windows (utilizando Docker para Windows).
====


== Estado actual

Los autores de las políticas pueden utilizar los siguientes recursos:

* https://github.com/kubewarden/policy-sdk-swift[{short-project-name} Swift SDK]: proporciona un conjunto de ``+estructuras+``y funciones que simplifican el proceso de escritura de políticas.
* https://github.com/kubewarden/swift-policy-template[{short-project-name} Proyecto de plantilla Sw]ift: utilice esta plantilla para crear una política basada en Swift.
La plantilla incluye una política de trabajo y un conjunto de acciones de GitHub para automatizar su ciclo de vida.

Con Swift no hay limitaciones graves, sólo algunos problemas menores:

* Es fundamental realizar algunas optimizaciones posteriores a la compilación antes de utilizar la política _"en producción":_
 .. Desmonte el módulo Wasm a través de `+wasm-strip+` para reducir su tamaño
 .. Optimizar el módulo Wasm mediante `+wasm-opt+`

La Acción GitHub proporcionada por el repositorio de plantillas ya se encarga de ello.
