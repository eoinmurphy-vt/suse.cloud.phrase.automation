El compilador Go oficial puede producir binarios WebAssembly, para su ejecución fuera de un navegador, desde la v1.21.

Existe otro compilador Go que puede generar binarios WebAssembly utilizables por {project-name}.
Este proyecto de compilador es https://tinygo.org/[] TinyGo:

____
TinyGo lleva el lenguaje de programación Go a los sistemas embebidos y a la web moderna mediante la creación de un nuevo compilador basado en LLVM.

Puedes compilar y ejecutar programas TinyGo en más de 94 placas microcontroladoras diferentes, como la BBC micro:bit y la Arduino Uno.

TinyGo también puede producir código WebAssembly (Wasm) de tamaño muy compacto.
Puede compilar programas para navegadores web, así como para servidores y entornos informáticos periféricos compatibles con la familia de interfaces WebAssembly System Interface (WASI).
____

El proyecto {short-project-name} sugiere actualmente el uso de TinyGo por dos razones:

* los binarios son más pequeños
* compatibilidad con https://wapc.io[waPC] mediante la posibilidad de exportar funciones al tiempo de ejecución

== Limitaciones de TinyGo

TinyGo aún no es compatible con todas las funciones de Go, consulta la página de compatibilidad de idiomas de TinyGo https://tinygo.org/lang-support/[] para ver el estado actual del proyecto.
Actualmente, su mayor limitación es la falta de un paquete `+reflect+` totalmente compatible.
Esto significa que los tipos de API oficiales de Kubernetes Go (por ejemplo: `+k8s.io/api/core/v1+`) no compilan.

{short-project-name} Las políticas necesitan procesar datos JSON como la configuración de políticas y la solicitud recibida por Kubernetes.

A pesar de las limitaciones actuales de TinyGo, sigue siendo fácil escribir políticas de validación en {short-project-name}.

== Herramientas

Para escribir las políticas de {short-project-name} se requiere una versión de TinyGo superior a `+v0.28.1+`.
No obstante, utilice la última versión para obtener los mejores resultados.

[CAUTION]
====
El uso de versiones anteriores de TinyGo produce errores en tiempo de ejecución debido al soporte limitado para Go reflection.
====


Estas bibliotecas Go son útiles cuando se escribe una política {short-project-name}:

* https://github.com/kubewarden/policy-sdk-go[{short-project-name} Go SDK]:
Proporciona estructuras y funciones que reducen la cantidad de código necesario.
También proporciona ayudantes de prueba.
* https://github.com/kubewarden/k8s-objects[Tipos de Kubernetes Go]:
Los https://github.com/kubernetes/kubernetes/tree/master/staging/src/k8s.io[tipos oficiales de Kubernetes Go de]no pueden utilizarse con TinyGo.
Este módulo proporciona todos los tipos de Kubernetes de una manera amigable para TinyGo.
* https://github.com/tidwall/gjson[gjson]:
Proporciona un lenguaje de consulta para navegar rápidamente por documentos JSON y recuperar datos.
Esta biblioteca no utiliza el paquete `+encoding/json+` proporcionado por Go `+stdlib+`, por lo tanto es utilizable con TinyGo.
* https://github.com/deckarep/golang-set[mapset]:
Proporciona una implementación en Go de la estructura de datoshttps://en.wikipedia.org/wiki/Set_(abstract_data_type)[Set].
Esta biblioteca reduce la cantidad de código a escribir ya que operaciones como Set `+union+`, `+intersection+`, `+difference+` son operaciones comunes en las políticas {short-project-name}.

Por último, el proyecto {short-project-name} proporciona unahttps://github.com/kubewarden/go-policy-template[plantilla de proyecto de política Go]que puede utilizar para crear políticas Go {short-project-name}.

== Obtener las dependencias de TinyGo

La forma más fácil de obtener TinyGo es utilizando las imágenes de contenedor de aguas arriba.
Las versiones oficiales se publicanhttps://hub.docker.com/r/tinygo/tinygo[en], mientras que las compilaciones de la rama de desarrollo se envían automáticamentehttps://hub.docker.com/r/tinygo/tinygo-dev[].

Si lo necesitas, visita la página deTinyGohttps://tinygo.org/getting-started/[inicio de] para obtener más información.

== Requisitos previos de la tutoría

Durante este tutorial necesitarás estas herramientas en tu máquina de desarrollo:

* docker u otro motor de contenedores: utilizado para construir la política WebAssembly.
Utilizarás el compilador incluido en la imagen oficial del contenedor TinyGo.
* https://github.com/bats-core/bats-core[bats]: se utiliza para escribir las pruebas y automatizar su ejecución.
* https://github.com/kubewarden/kwctl/releases[`+kwctl+`]:
Herramienta CLI proporcionada por {short-project-name} para ejecutar sus políticas fuera de Kubernetes, entre otras acciones.
Se trata en xref:tutorials/testing-policies/index.adoc[esta sección] de la documentación.
