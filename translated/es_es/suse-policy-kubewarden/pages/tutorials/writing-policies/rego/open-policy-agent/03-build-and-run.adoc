En la sección anterior ha redactado su política de Rego.
La estructura se parece:

[subs="+attributes",console]
----
.
├── data
│   ├── default-ns.json
│   └── other-ns.json
├── policy.rego
└── request.rego

1 directory, 4 files
----

== Construya

Para construir:

[subs="+attributes",console]
----
$ opa build -t wasm -e policy/main policy.rego request.rego
----

Esto construye la política de rego, con:

* `+target+`: `+wasm+`.
Desea crear la política para el objetivo `+wasm+`.
* `+entrypoint+`: `+policy/main+`.
El punto de entrada es la regla `+main+` dentro del paquete `+policy+`.
* `+policy.rego+`:
Cree e incluya el archivo `+policy.rego+`.
* `+request.rego+`:
Cree e incluya el archivo `+request.rego+`.

Una vez finalizada la compilación, `+opa build+` ha generado un archivo `+bundle.tar.gz+`.
Puedes extraerlo:

[subs="+attributes",console]
----
$ tar -xf bundle.tar.gz /policy.wasm
----

Ahora el árbol tiene el siguiente aspecto:

[subs="+attributes",console]
----
.
├── bundle.tar.gz
├── data
│   ├── default-ns.json
│   └── other-ns.json
├── policy.rego
├── policy.wasm
└── request.rego

1 directory, 6 file
----

Ya tiene su archivo `+policy.wasm+`:

[subs="+attributes",console]
----
$ file policy.wasm
policy.wasm: WebAssembly (wasm) binary module version 0x1 (MVP)
----

Ahora ejecútalo tú.

== Ejecutar

Utilice `+kwctl+` para ejecutar la política:

[subs="+attributes",console]
----
$ kwctl run -e opa --request-path data/other-ns.json policy.wasm | jq
{
  "uid": "1299d386-525b-4032-98ae-1949f69f9cfc",
  "allowed": true
}
----

Esta solicitud es aceptada por la política, ya que se trata de la solicitud que apunta al espacio de nombres `+other+`.

* `+execution-mode+`: `+opa+`.
Las políticas de Rego pueden tener como objetivo Open Policy Agent o Gatekeeper.
Debe informar a `+kwctl+` del tipo de política que aplica.
* `+request-path+`:
La ubicación de la solicitud registrada a la que `+kwctl+` envía la política para su evaluación.

Ahora intente evaluar la petición que crea el pod dentro del espacio de nombres `+default+`:

[subs="+attributes",console]
----
$ kwctl run -e opa --request-path data/default-ns.json policy.wasm | jq
{
  "uid": "1299d386-525b-4032-98ae-1949f69f9cfc",
  "allowed": false,
  "status": {
    "message": "it is forbidden to use the default namespace"
  }
}
----

La política rechaza la solicitud, dando una razón al servidor API que se devuelve al usuario o consumidor API.
