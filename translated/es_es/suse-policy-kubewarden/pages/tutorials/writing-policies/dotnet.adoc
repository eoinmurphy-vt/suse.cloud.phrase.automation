Actualmente, .NET Core tiene soporte experimental para la plataforma WebAssembly WASI.
Esto es del https://github.com/SteveSandersonMS/dotnet-wasi-sdk[`+dotnet-wasi-sdk+`] proyecto.

[NOTE]
====

No necesita una instalación de Windows para escribir o ejecutar código .NET Core.
Todo se puede hacer también en una máquina Linux o macOS.
====


== Estado actual

Los autores de las políticas pueden utilizar los siguientes recursos:

* https://github.com/kubewarden/policy-sdk-dotnet[{project-name}.NET Core SDK]: proporciona un conjunto de objetos y funciones que simplifican el proceso de redacción de políticas.
* https://github.com/kubewarden/policy-sdk-dotnet/tree/main/example[{short-project-name} ejemplo de] política: este es un ejemplo de una política que funciona.

No se conocen limitaciones.
El SDK permite escribir tanto políticas de validación como de mutación.

Es posible utilizar la bibliotecahttps://www.nuget.org/packages/KubernetesClient.Models[`+KubernetesClient.Models+`]para manejar los objetos de Kubernetes.

== Plantilla de proyecto

Actualmente, no tenemos una plantilla de proyecto que pueda andamiar una política C#.

Por favor, https://github.com/kubewarden/policy-sdk-dotnet/issues[abra una incidencia]si está interesado.
