_{short-project-name} agradece las sugerencias de mejora de todos los colaboradores, ya sean nuevos o experimentados._

Puede contribuir a la documentación de {short-project-name} 

* Mejorar los contenidos existentes
* Creación de nuevos contenidos
* _(Acompañamiento_) Ayuda en la traducción de la documentación

== Cómo empezar

ifeval::["{build-type}" == "community"]

Todo el mundo es bienvenido a sugerir cambios enviando un pull request. También puede informar de un error o errata con la ayuda de una incidencia de GitHub en elhttps://github.com/kubewarden/docs[repositorio]

Participar en la documentación:

* Familiarícese con el https://github.com/kubewarden/docs[repositorio de documentación] y el https://docusaurus.io/[].
* Docusaurus hace uso de la sintaxis Markdown y utiliza https://mdxjs.com/[MDX]como analizador sintáctico.
* Entender cómo xref:howtos/contribution-guide/suggesting-an-improvement.adoc[sugerir una mejora] y abrir un pull request.

endif::[]

ifeval::["{build-type}" == "product"]

Todo el mundo es bienvenido a sugerir cambios enviando un pull request. También puede informar de un error o errata con la ayuda de una incidencia de GitHub en elhttps://github.com/rancher/kubewarden-product-docs[repositorio]

Participar en la documentación:

* Familiarícese con el https://github.com/rancher/kubewarden-product-docs[repositorio de documentación] y el https://antora.org/[].
* Antora utiliza la sintaxis AsciiDoc.
* Entender cómo xref:howtos/contribution-guide/suggesting-an-improvement.adoc[sugerir una mejora] y abrir un pull request.

endif::[]
