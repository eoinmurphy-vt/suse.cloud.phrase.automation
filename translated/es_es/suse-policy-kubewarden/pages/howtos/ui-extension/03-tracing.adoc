El rastreo permite recopilar información detallada sobre las evaluaciones de las políticas.
Es una herramienta útil para depurar problemas en la implantación y las políticas de {project-name}.

{short-project-name} utiliza https://www.jaegertracing.io/[Jaeger] para recibir, almacenar y visualizar eventos de rastreo.

*_Registros de seguimiento de políticas_*image:ui_policy_tracing.png[Registros de seguimiento de políticas] de la interfaz de usuario

== Requisitos previos

Necesita Cert-Manager y OpenTelemetry. Sigaxref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[estas instrucciones] para instalar Cert Manager y el Operador OpenTelemetry.

== Instalar Jaeger

Aplique los pasos de instalación delxref:howtos/telemetry/20-tracing-qs.adoc#_install_jaeger[inicio rápido de rastreo].

Una vez que el operador Jaeger crea todos los recursos, aparece un Servicio en`+my-open-telemetry-collector.jaeger.svc.cluster.local+`.

La interfaz de usuario de Jaeger Query está disponible en la siguiente dirección:

[subs="+attributes",console]
----
https://<CLUSTER_IP>/api/v1/namespaces/jaeger/services/http:my-open-telemetry-query:16686/proxy/search
----

Este punto final puede ser único dependiendo de su configuración. Encontrará el punto final de su recurso Jaeger en la página Servicios.

=== Actualice `+rancher-kubewarden-controller+` con el punto final Jaeger

Tendrá que editar el recurso `+rancher-kubewarden-controller+` para añadir el punto final de Jaeger "my-open-telemetry-collector.jaeger.svc.cluster.local:4317".

Por ejemplo:

[subs="+attributes",yaml]
----
telemetry:
  tracing: true
  mode: sidecar
  sidecar:
    tracing:
      jaeger:
        endpoint: my-open-telemetry-collector.jaeger.svc.cluster.local:4317
        tls:
          insecure: true
----

La instalación del servicio, `+my-open-telemetry-collector+`, se encuentra bajo el espacio de nombres`+jaeger+`.

[CAUTION]
====

Para simplificar, no estamos cifrando la comunicación entre el colector OpenTelemetry y el punto final Jaeger.

Esto *no es adecuado para el despliegue de producción*. Debería leer la de Jaegerhttps://www.jaegertracing.io/docs/latest/kubernetes/[documentación].

====
