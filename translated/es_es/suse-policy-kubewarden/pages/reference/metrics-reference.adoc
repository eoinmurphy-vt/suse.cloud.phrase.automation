== Servidor de políticas

El componente Policy Server inicializa y ejecuta las políticas.
Cuando recibe solicitudes del servidor API de Kubernetes, reenvía la solicitud a la política, devolviendo la respuesta proporcionada por la política al servidor API de Kubernetes.

==== `+kubewarden_policy_evaluations_total+`

===== Equipaje

|===
| Etiqueta | Descripción

| `+policy_name+`
| Nombre de la póliza

| `+resource_name+`
| Nombre del recurso evaluado

| `+resource_kind+`
| Tipo de recurso evaluado

| `+resource_namespace+`
| Espacio de nombres del recurso evaluado. No está presente si el recurso tiene alcance de clúster.

| `+resource_request_operation+`
| Tipo de operación: `+CREATE+`, `+UPDATE+`, `+DELETE+`, `+PATCH+`, `+WATCH+`...

| `+accepted+`
| ¿Se aceptó la solicitud?

| `+mutated+`
| ¿Ha mutado la solicitud?

| `+error_code+`
| Código de error devuelto por la política en caso de rechazo, si existe.
  No presente si la póliza no lo preveía.
|===
