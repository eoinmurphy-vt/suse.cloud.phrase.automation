== Übersicht

Auf dieser Seite finden Sie Tools und Anleitungen zum Sammeln von Protokollen aus einem SUSE Observability-Cluster. Diese Protokolle sind für die Diagnose von Problemen, die Ermittlung der Grundursachen und die Behebung von Problemen in Clustern, die nicht wie erwartet funktionieren, von wesentlicher Bedeutung.
Wenn Sie ein unerwartetes Verhalten, eine verminderte Leistung oder einen Ausfall in Ihrem {stackstate-product-name} Cluster feststellen, hilft Ihnen diese Seite, die notwendigen Daten zu sammeln, um zu verstehen, was falsch gelaufen ist, und Korrekturmaßnahmen zu ergreifen. Neben dem hier beschriebenen Skript gibt es ein aktives Skript xref:/setup/install-stackstate/support-package-performance.adoc[Support Package (Performance)], das aktiv Leistungsmessungen auf dem Cluster und {stackstate-product-name} durchführt.

== Anmerkungen

Das Skript sollte von einem Host aus ausgeführt werden, auf dem kubectl mit Zugriff auf den SUSE Observability-Cluster konfiguriert ist, oder von einem anderen Host mit den erforderlichen Berechtigungen und der Verbindung zum Cluster.
Dies kann direkt auf dem Host mit dem Benutzer root oder über sudo erfolgen.  Wenn kubeconfig nicht gesetzt ist, verwenden Sie den Befehl `+export KUBECONFIG=$PATH-TO-YOUR/kubeconfig+`.

== Verwendung

Das Skript muss heruntergeladen und direkt auf dem Host unter dem Benutzer `+root+` oder `+sudo+` ausgeführt werden.

=== Laden Sie das Skript herunter und führen Sie es aus

Laden Sie das Skript xref::attachment$suse-observability_logs_collector.sh[suse-observability_logs_collector.sh] herunter.

* Speichern Sie das Skript als: `+suse-observability_logs_collector.sh+`
* Führen Sie das Skript mit den folgenden Befehlen aus:
+
[,bash]
----
bash suse-observability_logs_collector.sh
----
+
Dadurch wird eine Datei im aktuellen Verzeichnis mit dem Namen `+suse-observability_logs_<date>.tar.gz+` erzeugt. Bitte laden Sie diese Datei in den Fall hoch.

Zusätzliche Argumente können angegeben werden:
[,bash]
----
> bash suse-observability_logs_collector.sh -h
SUSE Observability support package collector.
Fetches details of a SUSE Observability instance for troubleshooting.

Usage: ./suse-observability_logs_collector.sh [options] [<namespace>]

options:
  -h  Print this help
  -e  Fetch logs for suse-observability pods from Elasticsearch
  -r  Time range for logs to fetch from Elasticsearch,
      when not specified "7d" is used

<namespace>:
  The namespace that is running SUSE Observability, or
  "suse-observability" when not specified
----

Z.B. um 3 Tage Logs von der eingebetteten elasticsearch zu sammeln, wenn {stackstate-product-name} im Namespace `+my-observability+` eingesetzt wird:
[,bash]
----
bash suse-observability_logs_collector.sh -e -r 3d my-observability
----
