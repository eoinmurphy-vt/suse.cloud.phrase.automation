= Konfiguration von Netzwerken
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observabilität

SUSE Observability ist ein SaaS-Angebot, das in der Cloud gehostet wird. Damit der SUSE Observability Agent von Ihren Räumlichkeiten bzw. aus der Cloud mit dem SUSE Observability SaaS kommunizieren kann, muss er eine Verbindung zur SUSE Observability SaaS Receiver API herstellen können.
Wenn Ihr Cluster in einem privaten Netzwerk ausgeführt wird, müssen Sie Ihr Netzwerk möglicherweise so konfigurieren, dass der SUSE Observability-Agent eine Verbindung zur SUSE Observability Receiver-API herstellen kann, da Ihre Netzwerkkonfiguration möglicherweise keinen Datenverkehr in das Internet zulässt. Auf dieser Seite wird beschrieben, wie Sie Ihr Netzwerk so konfigurieren, dass der SUSE Observability-Agent installiert werden kann und dass der SUSE Observability-Agent mit der SUSE Observability Receiver API kommunizieren kann.

[NOTE]
====
Der Datenverkehr zwischen dem SUSE Observability Agent und der SUSE Observability Receiver API wird immer vom SUSE Observability Agent initiiert. Die SUSE Observability Receiver API initiiert keinen Datenverkehr mit dem SUSE Observability Agent.
====


== Installation von SUSE Observability Agent

Die Installation des SUSE Observability Agent erfolgt über Helm. Standardmäßig ist Helm Chart so konfiguriert, dass es die Container-Images von SUSE Observability Agent aus der Quay.io-Docker-Registry bezieht. Wenn Ihre Netzwerkkonfiguration keinen Datenverkehr in das Internet zulässt, haben Sie mehrere Möglichkeiten, den SUSE Observability Agent zu installieren:

. Konfigurieren Sie Ihr Netzwerk so, dass Egress-Traffic zur Quay.io Container-Registry von Ihrem Kubernetes-Cluster zugelassen wird.
. Vermitteln Sie die Quay.io Container-Registry durch Ihre eigene Container-Registry.
. Ziehen Sie die Docker-Images in Ihre eigene Container-Registry.

Für die Optionen 2 und 3 müssen Sie Helm Chart so konfigurieren, dass die SUSE Observability Agent-Container-Images von Ihrer eigenen Container-Registrierung abgerufen werden. Eine Anleitung zur Konfiguration des SUSE Observability Agent Helm Chart, um Images von Ihrer eigenen Container-Registry zu beziehen, finden Sie xref:/k8s-suse-rancher-prime-agent-air-gapped.adoc[hier].

== Kommunikation mit dem SUSE Observability Agent

Der SUSE Observability Agent kommuniziert mit der SUSE Observability Receiver API über HTTPS. Die verschiedenen Teile des SUSE Observability-Agenten stellen eine Verbindung zur SUSE Observability-Receiver-API her, die in Ihrem Mandanten gehostet wird (siehe folgende Abbildung):

image::k8s/k8s-agent-communication.png[Kommunikation mit dem SUSE Observability Agent]

Die gesamte Kommunikation erfolgt über HTTPS unter Verwendung des Standard-HTTPS-Ports 443. Der SUSE Observability Agent verwendet die folgenden Endpunkte zur Kommunikation mit der SUSE Observability Receiver API:

* https:// *<tenant>.app.stackstate.io/receiver/stsAgent* - der SUSE Observability Agent sendet Metriken, Ereignisse und Topologiedaten an die SUSE Observability Receiver API.

Damit der SUSE Observability-Agent mit der SUSE Observability-Receiver-API kommunizieren kann, müssen Sie Ihr Netzwerk so konfigurieren, dass Egress-Datenverkehr zur SUSE Observability-Receiver-API zugelassen wird. Die SUSE Observability Receiver API wird in der Cloud gehostet und verfügt über eine spezifische IP für Ihren Tenant. Sie müssen den ausgehenden Datenverkehr zum Internet zulassen. Um die richtigen IP-Adressen für den Egress-Verkehr zu erhalten, können Sie den folgenden Befehl verwenden:

[,bash]
----
$ dig +short <tenant>.app.stackstate.io
----

Alternativ können Sie auch die folgende URL in Ihrem Browser aufrufen: `+https://www.nslookup.io/domains/<tenant>.app.stackstate.io/dns-records/+`
