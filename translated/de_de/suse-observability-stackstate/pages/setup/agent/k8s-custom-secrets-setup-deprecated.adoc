= k8s Custom Secrets Setup (veraltet)
:revdate: 2025-10-27
:page-revdate: {revdate}

== Übersicht

Der SUSE Observability-Agent unterstützt die Angabe des Namens eines benutzerdefinierten Geheimnisses, das den API-Schlüssel und das Cluster-Autorisierungs-Token enthält. Diese Funktion ist nützlich für Benutzer, die ihre eigenen Geheimnisse verwalten und die automatische Erstellung von Geheimnissen durch das Helm-Diagramm vermeiden möchten.

[CAUTION]
====
*Diese Methode zur Anpassung von Geheimnissen ist veraltet und nicht mit der RBAC-Funktion kompatibel.*

Bitte verwenden Sie xref:/setup/agent/k8s-custom-secrets-setup.adoc[die neue Methode] für die Verwendung externer Geheimnisse.
====


== Die Steuerkarte

=== Konfigurationsoptionen

* `+stackstate.manageOwnSecrets+`: Ein boolesches Flag, das festlegt, ob der Benutzer seine eigenen Geheimnisse verwalten möchte. Der Standardwert ist `+false+`.
* `+stackstate.customSecretName+`: (Optional) Name des benutzerdefinierten Geheimnisses, das vom Benutzer erstellt werden soll. Erforderlich, wenn `+stackstate.manageOwnSecrets+` auf `+true+` eingestellt ist.
* `+stackstate.customApiKeySecretKey+`: (Optional) Schlüsselname für den API-Schlüssel innerhalb des benutzerdefinierten Geheimnisses. Erforderlich, wenn `+stackstate.manageOwnSecrets+` auf `+true+` eingestellt ist.
* `+stackstate.customClusterAuthTokenSecretKey+`: (Optional) Schlüsselname für das Cluster-Autorisierungs-Token innerhalb des benutzerdefinierten Geheimnisses. Erforderlich, wenn `+stackstate.manageOwnSecrets+` auf `+true+` eingestellt ist.

=== Verhaltensbeschreibung

* *Automatische Erstellung von Geheimnissen*: Standardmäßig erstellt das Diagramm weiterhin automatisch Geheimnisse wie bisher, wenn `+stackstate.manageOwnSecrets+` auf `+false+` eingestellt ist.
* *Benutzerdefinierte Verwaltung von Geheimnissen*: Wenn `+stackstate.manageOwnSecrets+` auf `+true+` gesetzt ist, erwartet das Diagramm, dass der Benutzer den Namen des benutzerdefinierten Geheimnisses (`+stackstate.customSecretName+`) zusammen mit den Schlüsseln für den API-Schlüssel und das Autorisierungs-Token (`+stackstate.customApiKeySecretKey+` bzw. `+stackstate.customClusterAuthTokenSecretKey+`) angibt.
* *Stillschweigende Unterlassung*: Wenn Sie angeben, dass Sie Ihre eigenen Geheimnisse verwalten möchten, ignoriert das Diagramm die Werte für `+stackstate.apiKey+` und `+stackstate.cluster.authToken+`.
+
== Wie in values.yaml zu verwenden

. *Automatische Geheimniserstellung verwenden (Standard)*:
+
[,yaml]
----
 stackstate:
   manageOwnSecrets: false
   apiKey: "<your api key>"
----

. *Eigene Geheimnisse verwalten*:
+
[,yaml]
----
 stackstate:
   manageOwnSecrets: true
   customSecretName: my-custom-secret
   customApiKeySecretKey: api-key
   customClusterAuthTokenSecretKey: auth-token
----
