= Zertifikate für die Seitenwageninjektion der Anforderungsverfolgung
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observabilität

Der xref:/setup/agent/k8sTs-agent-request-tracing.adoc#_enabling_the_trace_header_injection_sidecar[Sidecar-Injection-Mechanismus], der bei Verwendung von `+--set httpHeaderInjectorWebhook.enabled=true+` bei der Installation des Agenten aktiviert wird, erstellt ein selbstsigniertes Zertifikat und verwendet ein `+ClusterRole+`, das Schreibzugriff auf `+Secret+` und `+MutatingWebhookConfiguration+` Objekte im Kubernetes-Cluster gewährt.

Wenn es aus Sicherheitsgründen nicht erwünscht ist, `+ClusterRoles+` zu erstellen, das clusterweite Schreibrechte gewährt, oder wenn es alternative Möglichkeiten gibt, ein Zertifikat bereitzustellen:

. Erzeugen Sie <<_generate_a_certificate_locally,lokal>> ein selbstsigniertes Zertifikat.
. Verwenden Sie den k8s https://cert-manager.io/[cert-manager] (falls er bereits auf dem Cluster vorhanden ist) <<_generate_a_certificate_using_the_cert_manager,mit einer `+ClusterIssuer+`>>.

== Ein Zertifikat lokal generieren

Um ein Zertifikat lokal zu erzeugen, gehen Sie wie folgt vor:

. Laden Sie das Skript zur Zertifikatserstellung herunter und führen Sie es aus, um eine Datei mit Steuerwerten (`+tls_values.yaml+`) und dem richtigen Zertifikat zu erstellen:
+
----
wget https://raw.githubusercontent.com/StackVista/http-header-injector/main/scripts/generate_ca_cert.sh
chmod +x generate_ca_cert.sh
./generate_ca_cert.sh <helm-agent-release-name> <helm-agent-namespace>
----
+
Achten Sie darauf, dass Sie den Freigabenamen, der im helm-Befehl verwendet wird, und den Namensraum verwenden, da das Zertifikat sonst ungültig ist.

. Installieren Sie den Agenten und fügen Sie die zusätzliche Konfiguration hinzu, indem Sie dem helm-Aufrufbefehl `+--set httpHeaderInjectorWebhook.enabled=true -f tls_values.yaml+` hinzufügen

== Erzeugen Sie ein Zertifikat mit dem cert-manager

Wenn in Ihrem Cluster der https://cert-manager.io/[cert-manager] installiert und ein `+ClusterIssuer+` konfiguriert ist, können Sie das von `+ClusterIssuer+` ausgestellte Zertifikat im Agenten für den Sidecar-Injektor verwenden. Fügen Sie dazu die folgenden Befehlszeilenargumente hinzu, um den Agenten zu installieren: `+--set httpHeaderInjectorWebhook.enabled=true --set-string httpHeaderInjectorWebhook.webhook.tls.mode="cert-manager" --set-string httpHeaderInjectorWebhook.webhook.tls.certManager.issuer="<my-cluster-issuer>"+`. Ersetzen Sie my-cluster-issuer durch den Namen des Ausstellers in Ihrem Cluster.
