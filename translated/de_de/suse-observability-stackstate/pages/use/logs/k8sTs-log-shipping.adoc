== Agent-Installation

=== Openshift

Protokollverteiler von Drittanbietern werden von Openshift nicht ohne Weiteres unterstützt. Die Plattform verfügt über Optionen für die Protokollweiterleitung, die anstelle der in der SUSE Observability-Steuertabelle enthaltenen Promtail-Konfiguration verwendet wird.  Detaillierte Anweisungen zur Konfiguration finden Sie in der Kubernetes-Stackpack-Dokumentation zu Ihrer laufenden SUSE Observability-Instanz.

=== Kubernetes

In der Standardkonfiguration des SUSE Observability k8s Agent helm chart wird der Protokollversand über die mit dem chart gelieferten helm-Werte aktiviert:

[,yaml]
----
logsAgent:
  # logsAgent.enabled -- Enable / disable k8s pod log collection
  enabled: true
----

Dadurch wird sichergestellt, dass auf jedem Knoten ein Promtail-Container bereitgestellt wird, der Protokolle sammelt und an SUSE Observability sendet.  Für Implementierungen, bei denen es nicht wünschenswert ist, Protokolle an SUSE Observability zu übermitteln, setzen Sie den obigen Wert auf `+false+`.

=== Zusätzliche Promtail-Pods betreiben

SUSE Observability verwendet eine abgestimmte Konfiguration für die Aufnahme von Protokollen, die in der Regel nicht mit den zusätzlichen Anforderungen übereinstimmt.  Es ist daher nicht möglich, eine separate Konfiguration für die Aufnahme von Protokollen an anderen Zielendpunkten auszuführen. Stattdessen wird empfohlen, einen zweiten Promtail-Pod auszuführen, der sich mit diesen Anforderungen befasst, und zwar getrennt von dem Promtail, das vom Agenten-Helm-Diagramm bereitgestellt wird.
