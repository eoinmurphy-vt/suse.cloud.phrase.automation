=== Integration schaffen

Um SUSE Observability in OpsGenie zu integrieren, muss ein globaler API-Schlüssel" erstellt werden.  Die Antwortgeber für eine Benachrichtigung können für jede Benachrichtigung unterschiedlich sein und werden in SUSE Observability konfiguriert.

Gehen Sie in Opsgenie zu "Einstellungen \-> Integrationen".

* Klicken Sie auf die Schaltfläche "Integration hinzufügen".
* Suchen Sie nach dem Integrationstyp "API".
* Wählen Sie einen Namen für diese Integration, aber _wählen Sie kein Zuweisungsteam_.

Wenn die Integration erstellt wurde, erhält sie einen "API-Schlüssel".

=== Erstellen und Testen des Kanals

image::k8s/notifications-opsgenie-channel-configuration.png[Den Kanal erstellen]

Wählen Sie die entsprechende Region für Ihr OpsGenie-Konto und geben Sie den "API-Schlüssel" ein.  Wählen Sie Bearbeiter für die Benachrichtigung aus; hierfür stehen _Benutzer_, _Teams_, _Zeitpläne_und _Eskalationen_ zur Verfügung.

Klicken Sie auf die Schaltfläche "Kanal hinzufügen". Dadurch wird der Kanal zur Liste der Kanäle auf der rechten Seite hinzugefügt. Es wird eine Schaltfläche "Testen" angezeigt. Wenn Sie diese Taste drücken, wird in OpsGenie ein Testalarm erzeugt.

== OpsGenie-Warnungen für Benachrichtigungen

Wenn eine Benachrichtigung geöffnet wird, wird eine neue OpsGenie-Warnung erstellt. Diese Meldung wird bei Änderungen aktualisiert, in der Regel nur bei Änderungen des Gesundheitszustands. Wenn die Benachrichtigung geschlossen wird, wird der OpsGenie-Alarm wieder geschlossen.  Ein Hinweis auf die Ausschreibung enthält den Grund für die Schließung der Ausschreibung.
