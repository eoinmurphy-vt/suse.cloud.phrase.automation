== Eine neue Meldung erstellen

Öffnen Sie die Benachrichtigungsseite über den Link in der unteren Hälfte des Hamburger-Menüs in der SUSE Observability-Benutzeroberfläche. Es öffnet sich eine Übersicht über alle bereits konfigurierten Benachrichtigungen und deren Status.

Sie können prüfen, ob die gewünschte Meldung bereits existiert. Wenn nicht, erstellen Sie eine neue Meldung mit der Schaltfläche "Neue Meldung hinzufügen".

== Konfigurieren Sie, wann Sie benachrichtigt werden

image::k8s/notifications-add-new-notification.png[Hinzufügen einer neuen Meldung]

Konfigurieren Sie die Benachrichtigung:

. Name - Wählen Sie einen Namen, der kurz ist, aber dennoch den Zweck dieser Benachrichtigung beschreibt. Sie dient zu Ihrer eigenen Information in der Meldungsübersicht.
. Status - Die Benachrichtigung kann vorübergehend deaktiviert werden, falls sie noch nicht benötigt wird, sich als zu laut erweist usw.
. Benachrichtigen wenn - Ein kritischer Gesundheitszustand löst immer eine Benachrichtigung aus, optional können aber auch abweichende Zustände einbezogen werden.
. Bereich - Im Beispiel werden die Zustände aller Monitore auf Pods im Standard-Kubernetes-Namensraum gesendet. Verwenden Sie die verfügbaren Filter im Bereich " <<_scope,Umfang">>, um diese Auswahl zu ändern.

=== Umfang

Es gibt 4 mögliche Bereichsfilter. Standardmäßig wird für jeden kritischen (und optional abweichenden) Gesundheitszustand eine Benachrichtigung verschickt. Die Filter dienen dazu, diesen Umfang einzuschränken. Ein Gesundheitszustand führt nur dann zu einer Benachrichtigung, wenn er allen Filtern entspricht.

* Monitore: Wählen Sie einen oder mehrere bestimmte Monitore aus. Benachrichtigungen werden nur für den Gesundheitszustand der ausgewählten Monitore gesendet.
* Monitor-Tags: Wählen Sie ein oder mehrere Monitor-Tags aus. Es werden nur Benachrichtigungen über den Gesundheitszustand von Monitoren gesendet, die mindestens eine der ausgewählten Markierungen aufweisen.
* Komponententypen: Wählen Sie einen oder mehrere Komponententypen aus. Benachrichtigungen werden nur für den Gesundheitszustand von Komponenten der ausgewählten Typen gesendet.
* Komponenten-Tags: Wählen Sie ein oder mehrere Komponenten-Tags aus. Es werden nur Benachrichtigungen über den Gesundheitszustand von Komponenten gesendet, die mindestens einen übereinstimmenden Wert für jeden eindeutigen Schlüssel in den ausgewählten Tags aufweisen.

.Beispiel für Komponenten-Tags
[%collapsible]
====
Angenommen, die ausgewählten Tags sind [ `+k8s-scope:prod-us/checkout+`, `+k8s-scope:prod-emea/checkout+`, `+team:blue+` ].  Die Schlüssel sind in diesem Fall `+k8s-scope+` und `+team+`.

Dann werden diese Komponenten übereinstimmen:

* eine Komponente mit den Tags [ `+k8s-scope:prod-emea/checkout+`, `+team:blue+` ]
* eine Komponente mit den Tags [ `+k8s-scope:prod-us/checkout+`, `+team:blue+` ]

Auch wenn diese Komponenten nicht übereinstimmen werden:

* eine Komponente mit Tags [ `+k8s-scope:prod-emea/checkout+` ]
* eine Komponente mit den Tags [ `+k8s-scope:prod-us/checkout+`, `+team:green+` ]
* eine Komponente mit den Tags [ `+k8s-scope:prod-emea/carts+`, `+team:blue+` ]
====

== Wohin sollen Benachrichtigungen gesendet werden?

SUSE Observability kann über Kanäle Benachrichtigungen an verschiedene externe Systeme senden. Unterstützt werden folgende Kanäle:

* xref:/use/alerting/notifications/channels/slack.adoc[Slack] - Benachrichtigungen an Slack senden
* xref:/use/alerting/notifications/channels/teams.adoc[Teams] - Benachrichtigungen an Microsoft Teams senden
* xref:/use/alerting/notifications/channels/webhook.adoc[Webhook] - Senden Sie Benachrichtigungen an einen Webhook. Der Webhook-Endpunkt kann die SUSE Observability-Nutzdaten in eine beliebige API eines Drittanbieters übersetzen.
* xref:/use/alerting/notifications/channels/opsgenie.adoc[Opsgenie] - Benachrichtigungen an OpsGenie senden

Im Allgemeinen sendet SUSE Observability zwei Arten von Nachrichten für Benachrichtigungen:

. Eine `+open+` Nachricht, wenn ein Gesundheitszustand auf kritisch oder abweichend geht. Diese Meldung kann bei Änderungen des Gesundheitszustands wiederholt werden
. Eine `+close+` Nachricht, wenn der Gesundheitszustand nicht mehr kritisch oder abweichend ist oder wenn aus anderen Gründen (die Komponente ist verschwunden, der Monitor wurde entfernt usw.) die Benachrichtigung nicht mehr aktiv ist.
