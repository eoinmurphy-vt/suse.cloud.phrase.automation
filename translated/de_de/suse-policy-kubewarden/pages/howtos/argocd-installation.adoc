= ArgoCD-Installation
:revdate: 2026-03-18
:page-revdate: {revdate}
:sidebar_label: ArgoCD-Installation
:sidebar_position: 35
:description: Konfigurieren Sie Ihre ArgoCD-Anwendung, um Kubernetes-Ressourcen effektiv zu verwalten, indem Sie spezifische Konfigurationsoptionen für die sichere Erstellung und Erneuerung von Zertifikaten festlegen.
:keywords: ["{project-name}", gitops, argocd]
:doc-persona: [kubewarden-operator]
:doc-type: [howto]
:doc-topic: [argocd-installation]

Wenn Sie ArgoCD zur Verwaltung Ihrer Kubernetes-Ressourcen verwenden, können Sie {project-name} mit ArgoCD installieren. Dies ist möglich, aber es gibt einige Überlegungen. Seit {short-project-name} v1.17.0 wurde die Abhängigkeit von cert-manager entfernt. Es gibt einen neuen Zertifikatsabgleicher, der die Zertifikate automatisch erneuert. Das bedeutet, dass der {short-project-name} Controller die Erstellung und Erneuerung der Zertifikate für Sie übernimmt. Dies gilt sowohl für die Stammzertifizierungsstelle als auch für alle Webhook-Zertifikate.

Die anfängliche Erstellung von Root-CA- und Webhook-Zertifikaten erfolgt während der Installation des Helm-Diagramms unter Verwendung der verfügbaren Helm-Funktionen. Beim Rendern des Helm-Diagramms prüft die Installation also, ob die Zertifikate bereits erstellt wurden, und wenn nicht, werden sie erstellt.

Dies stellt bei der Verwendung von ArgoCD ein Problem dar, da Helm nur zum Rendern der Vorlagen verwendet wird, während die gesamte Verwaltung des Lebenszyklus der Ressourcen von ArgoCD übernommen wird. Immer wenn ArgoCD das Helm-Diagramm rendert, um zu überprüfen, dass die Cluster-Anwendung nicht von der Diagrammdefinition abweicht, versucht es erneut, ein Zertifikat zu erstellen. Infolgedessen werden die Geheimnisse, in denen die Zertifikate und die Webhook-Konfiguration gespeichert sind, als unsynchronisiert gekennzeichnet.

Um dieses Problem zu lösen, ist es notwendig, die ArgoCD-Anwendung so zu konfigurieren, dass sie es ignoriert:

* das `+data+` Feld der Geheimnisse
* das Feld `+caBundle+` in der Datei `+MutatingWebhookConfiguration+`
* `+ValidatingWebhookConfiguration+` Ressourcen.

Dazu fügen Sie das Feld `+ignoreDifferences+` in der ArgoCD-Anwendung hinzu:

[subs="+attributes", yaml]
----
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: kubewarden-crds
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://charts.kubewarden.io
    chart: kubewarden-crds
    targetRevision: 1.13.0
  destination:
    server: https://kubernetes.default.svc
    namespace: kubewarden-system
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
---
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: kubewarden-controller
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://charts.kubewarden.io
    chart: kubewarden-controller
    targetRevision: 4.1.0
  destination:
    server: https://kubernetes.default.svc
    namespace: kubewarden-system
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:     # Sync options which modifies sync behavior
      # When syncing changes, respect fields ignored by the ignoreDifferences configuration
      - RespectIgnoreDifferences=true
  # ignoreDifferences is the important configuration to avoid ArgoCD marking
  # the secrets and # webhook configurations as unsynchronized
  ignoreDifferences:
  - group: ""
    kind: "Secret"
    name: kubewarden-ca
    namespace: kubewarden-system
    jsonPointers:
      - /data
  - group: ""
    kind: "Secret"
    name: kubewarden-webhook-server-cert
    namespace: kubewarden-system
    jsonPointers:
      - /data
  - group: ""
    kind: "Secret"
    name: kubewarden-audit-scanner-client-cert
    namespace: kubewarden-system
    jsonPointers:
      - /data
  - group: "admissionregistration.k8s.io"
    kind: "MutatingWebhookConfiguration"
    jqPathExpressions:
      - '.webhooks[]?.clientConfig.caBundle'
  - group: "admissionregistration.k8s.io"
    kind: "ValidatingWebhookConfiguration"
    jqPathExpressions:
      - '.webhooks[]?.clientConfig.caBundle'
---
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: kubewarden-defaults
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://charts.kubewarden.io
    chart: kubewarden-defaults
    targetRevision: 2.8.0
  destination:
    server: https://kubernetes.default.svc
    namespace: kubewarden-system
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
----

[NOTE]
====

In diesem Beispiel ist der Schlüsselaspekt die `+ignoreDifferences+` Konfiguration für die{short-project-name} Controller-Anwendung. Der Rest des Inhalts beschreibt, wie die ArgoCD-Anwendung zu konfigurieren ist und liefert ein vollständiges Beispiel.
