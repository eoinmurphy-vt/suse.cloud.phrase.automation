=== `+kwctl+` CLI-Tool

`+kwctl+` ist das CLI-Tool von {short-project-name} für Richtlinienautoren und Cluster-Administratoren, um Richtlinien vor der Anwendung auf einen Kubernetes-Cluster zu testen.

Dieses Tool hat eine ähnliche Schnittstelle wie das `+docker+` CLI-Tool.

==== Anwendungsfälle

In diesen Szenarien können Sie `+kwctl+` verwenden:

_Als politischer Autor_

* _End-to-End-Tests Ihrer Police_: Testen Sie Ihre Richtlinie anhand von erstellten Kubernetes-Anfragen und stellen Sie sicher, dass sich Ihre Richtlinie so verhält, wie Sie es erwarten. Sie können sogar kontextabhängige Richtlinien testen, die den Zugriff auf einen laufenden Cluster erfordern.
* _Betten Sie Metadaten in Ihr Wasm-Modul ein_: Die Binärdatei enthält Anmerkungen zu den Berechtigungen, die sie zur Ausführung benötigt. Sie können diese Anmerkungen mit `+kwctl+` überprüfen und ändern.
* _Veröffentlichung von Richtlinien in OCI-Registern_: Die Binärdatei ist ein vollständig konformes OCI-Objekt und kann in OCI-Registern gespeichert werden.

_Als Cluster-Administrator_

* _Überprüfen Sie entfernte Richtlinien_: Zeigen Sie bei einer Richtlinie in einer OCI-Registrierung oder in einem HTTP-Server alle statischen Informationen über die Richtlinie an.
* _Trockenlauf einer Richtlinie in Ihrem Cluster_: Testen Sie die Richtlinie anhand von erstellten Kubernetes-Anfragen und stellen Sie sicher, dass sich die Richtlinie angesichts der von Ihnen bereitgestellten Eingabedaten so verhält, wie Sie es erwarten. Sie können kontextabhängige Richtlinien, die den Zugriff auf einen laufenden Cluster erfordern, auch in einem Trockenlaufmodus testen.
* _Erstellen Sie ein erstes `+ClusterAdmissionPolicy+` Gerüst für Ihre Police_:
  Erzeugen Sie eine `+YAML+` Datei mit allen erforderlichen Einstellungen, die Sie mit `+kubectl+` auf Ihren Kubernetes-Cluster anwenden können.

==== Installation

`+kwctl+` Binärdateien für die stabilen Versionen sind imhttps://github.com/kubewarden/kwctl/releases[GitHub-Repository] verfügbar. Um`+kwctl+` aus dem GitHub https://github.com/kubewarden/kwctl[Repo] zu bauen, benötigen Sie eine https://www.rust-lang.org/tools/install[] Rust Entwicklungsumgebung.

== Policies durchsetzen

Sie setzen eine Richtlinie durch, indem Sie eine `+ClusterAdmissionPolicy+` definieren und sie dann mit `+kubectl+` in Ihrem Cluster bereitstellen.

`+kwctl+` hilft bei der Erstellung einer `+ClusterAdmissionPolicy+` aus der Richtlinie, die Sie durchsetzen wollen.

Nachdem Sie die `+ClusterAdmissionPolicy+` generiert und auf Ihren Cluster angewendet haben, können Sie die im folgendenxref:quick-start.adoc#_example_enforce_your_first_policy[Schnellstart] beschriebenen Schritte ausführen:

* Erzeugen Sie die `+ClusterAdmissionPolicy+` aus der Richtlinie `+manifest+` und speichern Sie sie in einer Datei
 ** Befehl: `+kwctl scaffold manifest -t ClusterAdmissionPolicy <policy URI> > <"policy name".yaml>+`
+
[subs="+attributes",shell]
----
$ kwctl scaffold manifest -t ClusterAdmissionPolicy registry://ghcr.io/kubewarden/policies/pod-privileged:v0.1.9
---
apiVersion: policies.kubewarden.io/v1alpha2
kind: ClusterAdmissionPolicy
metadata:
  name: privileged-pods
spec:
  module: "registry://ghcr.io/kubewarden/policies/pod-privileged:v0.1.9"
  settings: {}
  rules:
    - apiGroups:
        - ""
      apiVersions:
        - v1
      resources:
        - pods
      operations:
        - CREATE
  mutating: false
----

+

[TIP]
====
Standardmäßig ist der Wert `+name+` auf `+generated-policy+` eingestellt.
Möglicherweise möchten Sie sie bearbeiten, bevor Sie die `+ClusterAdmissionPolicy+` bereitstellen.
Der Name im vorherigen Beispiel lautet `+privileged-pods+`.
====

* Stellen Sie die `+ClusterAdmissionPolicy+` in Ihrem Kubernetes-Cluster bereit.
 ** Befehl: `+kubectl apply -f <"policy name".yaml>+`
+
[subs="+attributes",shell]
----
$ kubectl apply -f pod-privileged-policy.yaml
clusteradmissionpolicy.policies.kubewarden.io/privileged-pods created
----

Nach der Bereitstellung von `+ClusterAdmissionPolicy+` werden alle an Ihren Cluster gesendeten Anfragen von der Richtlinie ausgewertet, wenn sie in den Geltungsbereich der Richtlinie fallen.
