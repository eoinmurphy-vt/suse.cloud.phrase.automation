== {short-project-name} UI-Erweiterung installieren

Sie installieren die {short-project-name} UI als globale Erweiterung, aber Sie installieren den {short-project-name} Controller über die Rancher UI als cluster-scoped Ressource.

[NOTE]
====

Bei Installationen mit Luftanschluss gehen Sie wie folgt xref:howtos/airgap/02-install.adoc[vor].

====


Wählen Sie auf der Seite "Erweiterungen" die Schaltfläche "Aktivieren" und wählen Sie die Option zum Hinzufügen des Rancher Extensions Repository. Wenn diese Funktion aktiviert ist, erscheint automatisch die Erweiterung "{short-project-name}". Wählen Sie diesen Punkt, um die Erweiterung zu installieren. Nach der Installation können Sie {short-project-name} in dem gewünschten Cluster installieren.

=== Installieren Sie {short-project-name}

Nach den vorangegangenen Schritten erscheint in Ihrem Cluster ein neuer Eintrag im Seitenmenü für {short-project-name}. Diese Dashboard-Seite führt Sie durch den Installationsprozess und vervollständigt die Voraussetzungen.

[NOTE]
====

Während des Schritts "App-Installation" des Installationsassistenten kann die Schaltfläche "{short-project-name} installieren" ausgegraut bleiben. Wenn dies der Fall ist, aktualisieren Sie die Seite und kehren Sie zu diesem Schritt zurück.

====


== Post-Installation

Nach Abschluss der Installation enthalten die Dashboard-Seite und das Seitenmenü nun neue Elemente, nämlich Policy Server, Cluster Admission Policies und Admission Policies. Von hier aus können Sie Policy Server und Policies erstellen, um das Verhalten innerhalb Ihres Clusters zu steuern.

*_Dashboard-Ansicht_*image:ui_dashboard.png[] UI

=== Aktivieren des Standard-Richtlinienservers und der Richtlinien

Auf der Dashboard-Seite können Sie die Schaltfläche "Chart installieren" auswählen, um dashttps://github.com/kubewarden/helm-charts/tree/main/charts/kubewarden-defaults[`+kubewarden-defaults+`]Helm-Diagramm zu installieren. Dieses Diagramm enthält den Standard-Richtlinienserver und einige kuratierte Richtlinien.

Nach der Installation des Diagramms können Sie die Standardrichtlinienserver-Details mit den zugehörigen Richtlinien in einer sortierbaren Tabelle anzeigen.

*_Policy Server-Detailansicht_*image:ui_policyserver_detail.png[] UI

=== Richtlinien erstellen

Beim Erstellen von Richtlinien erhalten Sie zunächst die Option "Benutzerdefinierte Richtlinie" aus dem Richtlinienraster. Geben Sie die erforderlichen Informationen für den Namen, das Modul und die Regeln Ihrer Police an. Es wird empfohlen, das Repository des Richtlinienkatalogs hinzuzufügen, um auf die offiziellen Richtlinien von {short-project-name} zuzugreifen.

*_Erstellen einer benutzerdefinierten Richtlinie_*image:ui_policy_custom.png[UI-Richtlinie Whitelist]

== Luftdurchlässige Installation

Da {short-project-name} eine offizielle Rancher-Erweiterung ist, stellt das Rancher-Team einen Mechanismus zur Verfügung, um automatisch ein Image des Erweiterungskatalogs zu erzeugen. Dies wird der Datei `+rancher-images.txt+` hinzugefügt, wennhttps://ranchermanager.docs.rancher.com/getting-started/installation-and-upgrade/other-installation-methods/air-gapped-helm-cli-install/publish-images#1-find-the-required-assets-for-your-rancher-version[Installation von Rancher Manager] für Air-Gapped-Instanzen.

Sobald dieses Image in eine Registry gespiegelt wurde, auf die Ihr Air-Gapped-Cluster zugreifen kann, können Sie das Image über die Rancher-Benutzeroberfläche importieren. Dadurch wird ein lokales Helm-Repository mit dem {short-project-name} UI-Diagramm für die Installation erstellt.

=== Installationsschritte

. https://ranchermanager.docs.rancher.com/how-to-guides/new-user-guides/kubernetes-resources-setup/secrets[Erstellen Sie]ein Registrierungsgeheimnis innerhalb des Namensraums `+cattle-ui-plugin-system+`. Geben Sie die Domäne der Bildadresse in das Feld *Registry Domain Name* ein.
. Gehen Sie zurück zur Seite " *Erweiterungen"* (z. B. `+https://cluster-ip/dashboard/c/local/uiplugins+`).
. Wählen Sie oben rechts die Option *Erweiterungskataloge verwalten*. image:ui_airgap_01.png[Kataloge verwalten]
. Wählen Sie die Schaltfläche *Erweiterungskatalog importieren*. \image:ui_airgap_02.png[Kataloge importieren]
. Geben Sie die Bildadresse in das Feld *Katalogbildreferenz* ein.
. Wählen Sie das soeben erstellte Geheimnis aus dem Dropdown-Menü *Pull Secrets* aus. image:ui_airgap_03.png[Kataloginformationen eingeben]
. Klicken Sie auf *Laden*. Die Erweiterung wird nun auf " *Ausstehend"* gesetzt.
. Kehren Sie zur Seite " *Erweiterungen"* zurück.
. Wählen Sie die Registerkarte *Verfügbar* und klicken Sie auf die Schaltfläche *Neu laden*, um sicherzustellen, dass die Liste der Erweiterungen auf dem neuesten Stand ist. image:ui_airgap_04.png[Install {short-project-name}]
. Suchen Sie die soeben hinzugefügte Erweiterung {short-project-name}, und klicken Sie auf die Schaltfläche *Installieren*.
