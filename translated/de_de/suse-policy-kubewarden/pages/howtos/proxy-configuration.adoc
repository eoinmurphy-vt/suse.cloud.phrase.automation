= Proxy-Konfiguration
:description: Verwendung von Proxys mit {project-name}.
:keywords: ["{project-name}", "kubewarden", "kubernetes", "proxy", "proxies"]
:doc-persona: ["kubewarden-user", "kubewarden-operator", "kubewarden-distributor", "kubewarden-integrator"]
:doc-type: ["howto"]
:doc-topic: ["operator-manual", "proxy"]

Sowohl für `+kwctl+` als auch für `+policy-server+` können Sie HTTP- und HTTPS-Proxys konfigurieren sowie Domains von dieser Proxy-Konfiguration ausnehmen.

Sie können dies auf zwei Arten tun:* Indem Sie die Umgebungsvariablen `+HTTP_PROXY+`, `+HTTPS_PROXY+`, `+NO_PROXY+`(und ihre Gegenstücke in Kleinbuchstaben) entweder für `+kwctl+` oder `+policy-server+` setzen.
  Um einen PolicyServer CR zu konfigurieren, setzen Sie sein `+spec.env+` Array.* Durch Bereitstellung einer `+sources.yaml+` Datei über das `+--sources-path+` Argument.
  Die Datei `+sources.yaml+` hat Vorrang vor den Umgebungsvariablen.
  Die Übernahme dieser Konfiguration über das Feld `+spec+` ist in PolicyServer CRs noch nicht vorgesehen.

Einstellung einer Proxy-Konfiguration beeinflusst:

* Policy Pull, Push und Pull-and-Run von und zu dem OCI-Register.
* Host-Funktionen von kontextbezogenen Aufrufen, wie z. B.:
** Containerregister, z. B. Erhalt eines OCI-Manifests oder einer Manifestübersicht.
   Anstatt direkt auf die OCI-Registrierung zuzugreifen, wird der Datenverkehr über die konfigurierten Proxys geleitet.
** Sigstore-Funktionen, wie die Überprüfung einer schlüssellosen Signatur.
   Anstatt auf die von Sigstore definierten Dienste zuzugreifen, wird der Datenverkehr über die konfigurierten Proxys geleitet.

== Beispiel

Sie können die folgenden Umgebungsvariablen setzen, entweder für `+kwctl+` oder`+policy-server+` (über den PolicyServer `+spec.env+`):

[source,bash]
----
HTTP_PROXY="http://proxy.corp:3128"
https_proxy="http://proxy.corp:3129"
NO_PROXY="localhost,.corp"
----

Mit dieser Konfiguration:* Ungesicherter Datenverkehr über HTTP, z. B. von und zu einer unsicheren OCI-Registrierung, wird über `+\http://proxy.corp:3128+` geleitet.
  Dies wirkt sich auf Pulling Policies und Policies aus, die kontextabhängige Fähigkeiten aufrufen, die OCI Image Manifest Digests für Container in diesem Repository haben.* Verschlüsselter Datenverkehr über HTTPS wird über `+\https://proxy.corp:3129+` geroutet.
  Dazu gehören z. B. das Ziehen und Schieben von Richtlinien zu jeder sicheren OCI-Registrierung und die erwähnten kontextabhängigen Fähigkeiten von Richtlinien.* Der Verkehr von und zu localhost oder einer Domäne unter `+.corp+` ist von den vorherigen Konfigurationen ausgenommen.
