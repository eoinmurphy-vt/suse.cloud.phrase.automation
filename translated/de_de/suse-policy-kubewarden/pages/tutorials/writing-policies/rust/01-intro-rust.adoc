https://www.rust-lang.org/[Rust] ist die ausgereifteste Programmiersprache, die WebAssembly-Module erzeugen kann.
WebAssembly ist in der Rust-Welt ein Bürger erster Klasse, so dass viele der Werkzeuge und Kisten aus dem Rust-Ökosystem sofort funktionieren.

{project-name} bietet ein https://crates.io/crates/kubewarden-policy-sdk[SDK] Rust, das das Schreiben von Richtlinien vereinfacht.
Es gibt auch ein https://github.com/kubewarden/rust-policy-template[Vorlagenprojekt], das als Gerüst für ein Richtlinienprojekt dient, das das https://github.com/cargo-generate/cargo-generate[`+cargo-generate+`] Dienstprogramm.

Diese Dokumentation zeigt, wie man diese Projekte verwendet, um {short-project-name} Policies mit Rust zu schreiben.
Die Details des Rust-SDKs von {short-project-name} werden nicht behandelt.
Die Einzelheiten finden Sie in der https://docs.rs/kubewarden-policy-sdk/0.1.0[Kistendokumentation].

== Abrufen der Rust-Abhängigkeiten

Installieren Sie den Rust-Compiler und seine Werkzeuge mithttps://github.com/rust-lang/rustup[rustup].
Siehe die rustup https://rust-lang.github.io/rustup/installation/index.html[Dokumentation zur Installation von].

Sobald Sie `+rustup+` installiert haben, fügen Sie das Ziel WebAssembly System Interface (WASI) hinzu:

[subs="+attributes",console]
----
rustup target add wasm32-wasip1
----

== OSX-Abhängigkeiten

Um `+cargo-generate+` zu verwenden, müssen Sie den Xcode-Werkzeugsatz hinzufügen.
Wenn es nicht über Xcode installiert wird, erhalten Sie mit dem folgenden Befehl die benötigten Abhängigkeiten:

[subs="+attributes",console]
----
xcode-select --install
----
