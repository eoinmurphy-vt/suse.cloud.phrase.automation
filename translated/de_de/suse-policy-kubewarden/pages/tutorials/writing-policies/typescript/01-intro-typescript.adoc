Wie auf der https://www.typescriptlang.org/[offiziellen Website] angegeben:

[quote]
TypeScript erweitert JavaScript durch das Hinzufügen von Typen.

Durch das Verständnis von JavaScript spart TypeScript Zeit beim Abfangen von Fehlern und bei der Bereitstellung von Korrekturen, bevor Sie den Code ausführen.

{project-name} verwendet https://github.com/bytecodealliance/javy[] Javy (ein Projekt der Bytecode Alliance), um WebAssembly-Binärdateien aus JavaScript und TypeScript zu erstellen.

[quote]
Javy übernimmt Ihren JavaScript-Code und führt ihn in einem WebAssembly-Kontext aus.

Es verfügt über eine eingebettete QuickJS-Engine, die in WebAssembly kompiliert ist und JavaScript ausführen kann.

Das Projekt bietet sowohl eine Befehlszeilenschnittstelle (CLI) als auch eine Reihe von APIs zum Einbetten und Anpassen des Verhaltens bei der Ausführung von JavaScript in WebAssembly.

Das Projekt {short-project-name} verwendet aus diesen Gründen derzeit Javy:

* Ausgereifte JavaScript-Engine (QuickJS), kompiliert in WebAssembly.
* Unterstützung für die xref:/tutorials/writing-policies/wasi/01-intro-wasi.adoc[WASI-Schnittstelle] durch benutzerdefinierte Hostfunktionen.
* Kleinere Binärgrößen im Vergleich zu anderen JavaScript-to-WebAssembly-Lösungen.
* Aktive Entwicklung und Pflege durch die Bytecode Alliance.

== Javy Einschränkungen

Javy führt JavaScript in einer sandboxed WebAssembly-Umgebung mit bestimmten Einschränkungen aus:

* *Nur WASI-Umgebung*: Der Zugriff ist auf stdin/stdout/stderr und explizit bereitgestellte Host-Funktionen beschränkt.
* *No Node.js APIs*: Standard-Node.js-Module wie `+fs+`, `+http+` oder `+crypto+` sind nicht verfügbar.
* *Begrenzte Standardbibliothek*: Nur grundlegende JavaScript-Funktionen und explizit aktivierte APIs sind zugänglich.
* *Ausführung in einem einzigen Thread*: Keine Unterstützung für Web Workers oder Multithreading.

Trotz dieser Einschränkungen bietet Javy ausreichende Möglichkeiten für die Erstellung effektiver {short-project-name} Validierungsrichtlinien durch das Hosts Capabilities System.

[WARNING]
====
Das Schreiben nach STDOUT verstößt gegen die Richtlinien - verwenden Sie stattdessen STDERR für die Protokollierung.
====

== Werkzeugbau

Das Verfassen von {short-project-name} Richtlinien erfordert:

* *Node.js*: JavaScript-Laufzeit.
* *npm*: Für die Verwaltung von Abhängigkeiten.
* *TypeScript*: Empfohlen für die Typensicherheit (optional).

[WARNING]
====
Stellen Sie sicher, dass Sie Node.js 18 oder höher verwenden. Ältere Versionen sind möglicherweise nicht mit der Kompilierungs-Toolchain kompatibel.
====

Diese TypeScript/JavaScript-Bibliotheken sind beim Schreiben einer {short-project-name} -Richtlinie nützlich:

* https://github.com/kubewarden/policy-sdk-js[{short-project-name} JavaScript SDK]: Bietet Strukturen und Funktionen, die den Umfang des erforderlichen Codes reduzieren. Es bietet auch Testhilfen und Zugang zu allen Host-Funktionen.
* https://github.com/silverlyra/kubernetes-types[Kubernetes TypeScript-Typen]: Stellt TypeScript-Definitionen für alle Kubernetes-Ressourcen bereit und ermöglicht die Entwicklung typsicherer Richtlinien.

Das Projekt {short-project-name} bietet ein https://github.com/kubewarden/js-policy-template[Vorlagenprojekt für JavaScript/TypeScript-Richtlinien], das Sie zum Erstellen von {short-project-name} Richtlinien verwenden können.

== Die Toolchain erhalten

Der einfachste Weg, die Toolchain zu erhalten, ist die Verwendung des {short-project-name} JavaScript SDK, das das Javy-Kompilierungs-Plug-in enthält:

[source,console]
----
npm install kubewarden-policy-sdk
----

Die Javy-Plugin-Binärdatei ist automatisch enthalten und Sie finden sie unter:

[source,console]
----
node_modules/kubewarden-policy-sdk/plugin/javy-plugin-kubewarden.wasm
----
