= Erstellen einer neuen Validierungsrichtlinie
:revdate: 2026-03-18
:page-revdate: {revdate}

:doc-type: Lehrgang
:doc-topic: {project-name}, kubewarden, writing-policies, typescript, Erstellen einer neuen Validierungsrichtlinie
:doc-persona: kubewarden-policy-developer
:current-version: {page-origin-branch}

Dieses Tutorial behandelt die Erstellung einer Richtlinie, die die Hostnamen von Pod-Objekten validiert.

Die Richtlinie besteht darin, alle Pods abzulehnen, die einen oder mehrere Hostnamen in der Ablehnungsliste verwenden.
Die Richtlinienkonfiguration erfolgt über Laufzeiteinstellungen.

Zusammengefasst sollten die Richtlinieneinstellungen wie folgt aussehen:

[source,yaml]
----
denied_hostnames:
  - bad-host
  - forbidden-host
----

Die Politik lehnt die Schaffung dieses Pods ab:

[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  hostname: bad-host
  containers:
    - name: nginx
      image: nginx:latest
----

Sie akzeptiert jedoch die Einrichtung dieses Pods:

[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  hostname: allowed-host
  containers:
    - name: nginx
      image: nginx:latest
----

== Das Gerüst für ein neues politisches Projekt

Sie können ein neues Richtlinienprojekt mit Hilfe des https://github.com/kubewarden/js-policy-template[Vorlagen-Repositorys] erstellen. Wählen Sie die grüne Schaltfläche "Diese Vorlage verwenden" am oberen Rand der Seite und folgen Sie dem Assistenten von GitHub.
Klonen Sie das Repository lokal und aktualisieren Sie die Datei `+package.json+`, um die Details Ihrer Richtlinie wiederzugeben:

[source,json]
----
{
  "name": "your-policy-name",
  "version": "1.0.0",
  "description": "Your policy description",
  "repository": {
    "type": "git",
    "url": "https://github.com/your-username/your-policy-name"
  }
}
----

Achten Sie darauf, einen Repository-Pfad zu verwenden, der mit Ihrem tatsächlichen GitHub-Repository übereinstimmt.
