Wie auf der https://swift.org/[offiziellen Website] angegeben:

____
Swift ist eine Allzweck-Programmiersprache, die für Einsteiger zugänglich und für Experten leistungsstark ist.
Es ist schnell, modern, sicher und macht Spaß zu schreiben.
____

Der Swift-Compiler hat noch keine WebAssembly-Unterstützung, aber dashttps://swiftwasm.org/[SwiftWasm-Projekt] bietet einen gepatchten Compiler mit dieser Fähigkeit.

Das SwiftWasm-Team arbeitet daran, diese Änderungen in das Swift-Projekt einzubringen.
In der Zwischenzeit können Sie die vom SwiftWasm-Projekt bereitgestellte Toolkette verwenden, um {project-name} Richtlinien zu erstellen.

[NOTE]
====

Sie brauchen kein Apple-System, um Swift-Code zu schreiben oder auszuführen. Alles kann auch auf einem Linux-Rechner oder unter Windows (unter Verwendung von Docker für Windows) durchgeführt werden.
====


== Aktueller Stand

Die Verfasser von Richtlinien können die folgenden Ressourcen nutzen:

* https://github.com/kubewarden/policy-sdk-swift[{short-project-name} Swift SDK]: Es bietet eine Reihe von ``+struct+s``und Funktionen, die das Schreiben von Richtlinien vereinfachen.
* https://github.com/kubewarden/swift-policy-template[{short-project-name} Swift-Vorlagenprojekt]: Verwenden Sie diese Vorlage, um eine Swift-basierte Richtlinie zu erstellen.
Die Vorlage wird mit einer funktionierenden Richtlinie und einer Reihe von GitHub-Aktionen geliefert, um ihren Lebenszyklus zu automatisieren.

Es gibt keine schwerwiegenden Einschränkungen bei Swift, nur einige kleinere Probleme:

* Es ist wichtig, einige Optimierungen nach der Erstellung vorzunehmen, bevor die Richtlinie _"in der Produktion"_ verwendet wird _:_
 .. Strippen Sie das Wasm-Modul über `+wasm-strip+`, um seine Größe zu reduzieren.
 .. Optimieren Sie das Wasm-Modul über `+wasm-opt+`

Die vom Vorlagen-Repository bereitgestellte GitHub-Aktion kümmert sich bereits um diese Aufgabe.
