=== Add-on-Anbieter für Flotten
Diese Funktion installiert den Cluster API Add-on Provider for Fleet (CAAPF), um die Integration mit https://github.com/rancher/fleet[Fleet] und die einfache Bereitstellung von Anwendungen auf CAPI-bereitgestellten Clustern zu ermöglichen. CAAPF installiert Fleet automatisch im Management-Cluster und registriert neu bereitgestellte CAPI-Cluster bei Fleet, so dass Anwendungen über GitOps, `+Bundle+` oder `+HelmOp+` Ressourcen auf diesen Clustern bereitgestellt werden können. CAAPF unterstützt auch die Gruppierung von CAPI-Clustern, die auf dieselbe ClusterClass verweisen, durch die ClusterGroup von Fleet, was die Bereitstellung von Anwendungen für große Gruppen von Clustern wesentlich vereinfacht. Diese Funktion befindet sich im GA-Stadium und ist standardmäßig aktiviert.

=== Agent TLS-Modus
Dieses Feature-Gate wird verwendet, um Turtles zu instruieren, ob es Ranchers `+agent-tls-mode+` Einstellung folgen soll, wenn es Cluster-Registrierungsmanifeste für den Import von Clustern abruft. Wenn diese Funktion aktiviert ist, wird Turtles den Wert der Rancher-Einstellung `+agent-tls-mode+` nachschlagen, um festzustellen, ob er auf `+system-store+` oder `+strict+` gesetzt wurde. Wenn diese Option auf `+system-store+` gesetzt ist, vertraut Turtles jedem Zertifikat, das von einer öffentlichen Zertifizierungsstelle im Vertrauensspeicher des Betriebssystems generiert wurde, einschließlich solcher, die von Stellen wie Let's Encrypt signiert wurden. Wenn diese Option auf `+strict+` gesetzt ist, vertraut Turtles nur Zertifikaten, die von der Zertifizierungsstelle in Ranchers Einstellung `+cacerts+` generiert wurden. Wenn diese Einstellung deaktiviert ist, arbeitet Turtles wieder im Modus `+system-store+`. Diese Funktion befindet sich im Beta-Stadium und ist standardmäßig aktiviert.

=== Kein Cert Manager
Dieses Feature-Gate konvertiert die CAPI-Provider-Manifeste, um die Verwendung von cert-manager durch https://github.com/rancher/wrangler/[] wrangler zu ersetzen.

Insbesondere werden die Webhooks für die Validierung und Änderung von Zulassungen aktualisiert, um Zertifikate zu verwenden, die von Wrangler anstelle von Cert-Manager generiert wurden, was eine Cert-Manager-freie Umgebung ermöglicht.

Beachten Sie, dass die Verwendung der Rancher `+CATTLE_SYNC_ONLY_CHANGED_OBJECTS+` https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/best-practices/tuning-rancher-at-scale.html#_reducing_event_handler_executions[-Funktionalität] dazu führen kann, dass Zertifikate nach der standardmäßigen Ablaufzeit von einem Jahr nicht mehr erneuert werden.

Es ist möglich, diese Funktion zu deaktivieren, um alle Provider-Manifeste auf die ursprüngliche cert-manager-Verwendung zurückzusetzen.

=== Rancher-Standardregistrierung verwenden

Dieses Feature-Gate weist {product_name} an, die Rancher-Verwaltungseinstellung `+system-default-registry+` zu nutzen, wenn Bildreferenzen für CAPI-Anbieterkomponenten erstellt werden. Wenn diese Option aktiviert ist, liest {product_name} diese Einstellung und schreibt, falls eine Registrierung konfiguriert ist, das Repository für jedes Anbieter-Image neu, um diese Registrierung zu verwenden.
Wenn die Einstellung leer ist, greift {product_name} auf die Standard-Bildreferenzen zurück. Bildüberschreibungen, die in der Ressource `+ClusterctlConfig+` definiert sind, haben immer Vorrang. Ausführliche Informationen zur Verwendung in luftgeschützten Umgebungen finden Sie unter xref:operator/airgapped.adoc[Luftgeschützte Umgebung]. Diese Funktion befindet sich im Alpha-Stadium und ist standardmäßig aktiviert.

== Feature-Stufen

{product_name} folgt dem Prozess der Kubernetes https://kubernetes.io/docs/reference/command-line-tools-reference/feature-gates/#feature-stages[Funktionsstufen]. Ein Merkmal kann sich in einem der folgenden Stadien befinden: Alpha, Beta, oder Allgemeine Verfügbarkeit (GA).

=== Alpha-Stufe
- Standardmäßig deaktiviert.
- Kann fehlerhaft sein und hat keinen langfristigen Support.
- Die API kann sich ohne Vorankündigung ändern.
- Nur für kurzlebige Testcluster empfohlen.

=== Beta-Phase
- Standardmäßig immer aktiviert.
- Gut getestet und als sicher eingestuft.
- Die Unterstützung ist stabil, aber das Schema kann sich in zukünftigen Versionen ändern.
- Empfohlen für nicht geschäftskritische Anwendungen aufgrund möglicher inkompatibler Änderungen.

[NOTE]
====
Bitte geben Sie Feedback zu den Alpha- und Beta-Funktionen!
====

=== Phase der allgemeinen Verfügbarkeit (GA)
- Immer aktiviert, kann nicht deaktiviert werden.
- Es wird kein Feature Gate benötigt.
- Stabil, mit langfristiger Unterstützung in zukünftigen Versionen.
