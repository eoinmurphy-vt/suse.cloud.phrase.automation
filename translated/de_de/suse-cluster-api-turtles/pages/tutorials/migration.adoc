:revdate: 2025-11-14

== Context

Ab Rancher v2.13 wird {product_name} als Teil von Rancher installiert, so dass keine separate Bereitstellung mehr erforderlich ist. Vor dem Upgrade auf Rancher v2.13 müssen die Benutzer diesen Leitfaden befolgen, um einen reibungslosen Upgrade-Prozess zu gewährleisten.

== Migration

Führen Sie die folgenden Schritte der Reihe nach aus, _bevor Sie_ auf Rancher v2.13 aktualisieren:

. Stellen Sie sicher, dass Sie {product_name} auf v0.24.3 aktualisieren. Dieser Schritt ist wichtig, da alle Patch-Releases für v0.24 den Cluster für das Upgrade auf Rancher v2.13 vorbereiten.
. Deinstallieren Sie {product_name}. Dies ist notwendig, weil {product_name} ab Rancher v2.13 nicht mehr ein opt-in Addon ist, sondern eine Rancher-Systemkomponente. Daher installiert Rancher sie in einem anderen Namensraum als zuvor, und die zuvor installierte Version muss daher zuerst manuell deinstalliert werden. Bei der Deinstallation von v0.24.3 von {product_name} wird der CAPI-Provider selbst entfernt, die CAPI-CRDs bleiben jedoch erhalten. Während der Migration wird der zentrale CAPI-Controller nicht auf dem Management-Cluster laufen, aber alle nachgelagerten CAPI-Cluster sollten wie gewohnt weiterlaufen und nicht beeinträchtigt werden.
. Führen Sie die folgenden kubectl-Befehle aus:
+
[source,bash]
----
kubectl patch crd capiproviders.turtles-capi.cattle.io --type=json -p='[{"op": "add", "path": "/metadata/annotations/meta.helm.sh~1release-namespace", "value": "cattle-turtles-system"}]'
kubectl patch crd clusterctlconfigs.turtles-capi.cattle.io --type=json -p='[{"op": "add", "path": "/metadata/annotations/meta.helm.sh~1release-namespace", "value": "cattle-turtles-system"}]'
----

Diese Befehle werden die CRDs `+capiprovider+` und `+clusterctlconfig+` patchen, so dass sie von {product_name} übernommen werden, wenn sie als Teil von Rancher v2.13 installiert werden. 

Beachten Sie, dass die oben beschriebene Migration für Benutzer erforderlich ist, die {product_name} Versionen &le; v0.24.3 verwenden. Neue Benutzer, die Rancher v2.13 installieren, müssen keine Migration durchführen.
