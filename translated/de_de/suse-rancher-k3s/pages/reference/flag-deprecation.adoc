= Flaggenverfall
:page-languages: [en, ja, ko, zh]
:revdate: 2024-09-09
:page-revdate: {revdate}

K3s ist ein sich schnell entwickelndes Projekt, und als solches brauchen wir eine Möglichkeit, Flags und Konfigurationsoptionen zu verwerfen. Auf dieser Seite wird der Prozess zur Abschaffung von Flags und Konfigurationsoptionen beschrieben. Um sicherzustellen, dass die Benutzer nicht von der Entfernung der Flaggen überrascht werden, ist der Prozess ähnlich wie die https://kubernetes.io/docs/reference/using-api/deprecation-policy/[Deprecation Policy] Kubernetes.

== Prozess

. Flaggen können jederzeit als "zu verwerfen" deklariert werden.
. Flaggen, die "veraltet" sind, müssen im nächsten Patch für alle derzeit unterstützten Versionen als solche gekennzeichnet werden. Zusätzlich wird das Flag die Benutzer warnen, dass es in der nächsten Version veraltet sein wird.
. In der nächsten kleineren Version wird ein Flag in der Dokumentation als veraltet gekennzeichnet und im Code in ein verstecktes Flag umgewandelt. Das Kennzeichen wird weiterhin funktionieren und Warnungen an die Benutzer ausgeben.
. Im folgenden Minor-Release-Zweig werden veraltete Flags "nonoperational" und verursachen einen fatalen Fehler, wenn sie verwendet werden. Dieser Fehler muss dem Benutzer alle neuen Flaggen oder Konfigurationen erklären, die diese Flagge ersetzen.
. In der nächsten kleineren Version werden die nicht-operativen Flags aus der Dokumentation und dem Code entfernt.

== Beispiel

Ein Beispiel für diesen Prozess:

* `+--foo+` existiert in v1.22.14, v1.23.10 und v1.24.2.
* Nach der Veröffentlichung von v1.24.2 wurde beschlossen, `+--foo+` zugunsten von `+--new-foo+` zu verwerfen.
* In v1.22.15, v1.23.11 und v1.24.3 existiert `+--foo+` weiterhin, warnt aber die Benutzer:
+
----
[Warning] --foo will be deprecated in v1.25.0, use `+--new-foo+` instead
----
+
`+--foo+` wird während der gesamten Laufzeit von v1.22, v1.23 und v1.24 als operatives Kennzeichen weiterbestehen.

* In v1.25.0 ist `+--foo+` in der Dokumentation als veraltet gekennzeichnet und wird im Code ausgeblendet. Es wird weiterhin funktionieren und die Benutzer warnen, zu `+--new-foo+` zu wechseln.
* In v1.26.0 verursacht `+--foo+` einen fatalen Fehler, wenn es verwendet wird. Die Fehlermeldung lautet:
+
----
[Fatal] exit 1: --foo is no longer supported, use --new-foo instead
----

* In v1.27.0 wird `+--foo+` vollständig aus dem gesamten Code und der Dokumentation entfernt.
