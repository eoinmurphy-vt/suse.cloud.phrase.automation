:revdate: 2024-09-26

Um eine hohe Verfügbarkeit bei Upgrades zu gewährleisten, laufen die K3s-Container weiter, wenn der K3s-Dienst angehalten wird.

== K3s Dienstleistung

Das Stoppen und Neustarten von K3s wird durch das Installationsskript für systemd und OpenRC unterstützt.

[tabs]
======
systemd::
+
--
Zum Anhalten von Servern:

[,sh]
----
sudo systemctl stop k3s
----

Zum Neustart von Servern:

[,sh]
----
sudo systemctl start k3s
----

Um Agenten zu stoppen:

[,sh]
----
sudo systemctl stop k3s-agent
----

Zum Neustart von Agenten:

[,sh]
----
sudo systemctl start k3s-agent
----
--

OpenRC::
+
--
Zum Anhalten von Servern:

[,sh]
----
sudo rc-service k3s stop
----

Zum Neustart von Servern:

[,sh]
----
sudo rc-service k3s restart
----

Um Agenten zu stoppen:

[,sh]
----
sudo rc-service k3s-agent stop
----

Zum Neustart von Agenten:

[,sh]
----
sudo rc-service k3s-agent restart
----
--
======

== Killall-Skript

Um alle K3s-Container zu stoppen und den containerd-Status zurückzusetzen, kann das Skript `+k3s-killall.sh+` verwendet werden.

Das killall-Skript bereinigt Container, K3s-Verzeichnisse und Netzwerkkomponenten und entfernt dabei auch die iptables-Kette mit allen zugehörigen Regeln. Die Daten des Clusters werden nicht gelöscht.

Um das Killall-Skript von einem Serverknoten aus zu starten, führen Sie aus:

[,bash]
----
/usr/local/bin/k3s-killall.sh
----
