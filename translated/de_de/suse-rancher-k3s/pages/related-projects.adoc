= Verwandte Projekte
:page-languages: [en, ja, ko, zh]
:revdate: 2025-03-31
:page-revdate: {revdate}

Projekte, die die K3s-Verteilung umsetzen, sind willkommene Ergänzungen, um die Gemeinschaft zu erweitern. Diese Projekte tragen dazu bei:

* Erkunden Sie die Möglichkeiten und potenziellen Anwendungen von K3.
* Erweitern Sie die Funktionalität von K3 auf verschiedene Plattformen und Umgebungen.
* Vereinfachen Sie den Prozess der Erstellung großer High Availability (HA) K3s-Cluster.

== k3s-ansprechbar

Benutzer, die einen K3s-Cluster mit mehreren Knoten booten wollen und mit Ansible vertraut sind, sollten einen Blick auf das https://github.com/k3s-io/k3s-ansible[] k3s-io/k3s-ansible-Repository werfen. Diese Ansible-Playbooks bieten eine bequeme Möglichkeit zur Installation von K3 auf Ihren Knoten, so dass Sie sich auf die Konfiguration Ihres Clusters konzentrieren können und nicht auf den Installationsprozess.

== k3sup

Ein weiteres Projekt, das die Einrichtung eines K3s-Clusters vereinfacht, ist https://github.com/alexellis/k3sup[] k3sup. Dieses in Golang geschriebene Projekt benötigt nur ssh-Zugang zu Ihren Knoten. Es bietet auch eine bequeme Möglichkeit, K3s mit externen Datenspeichern einzusetzen, nicht nur mit dem eingebetteten etcd.

== autok3s

Ein weiteres Provisioning-Tool, https://github.com/cnrancher/autok3s[autok3s], bietet eine grafische Benutzeroberfläche für die Bereitstellung von k3s-Clustern über eine Reihe von Cloud-Anbietern, VMs und lokalen Maschinen. Dieses Tool ist nützlich für Benutzer, die eine grafische Oberfläche für die Prüfung von K3s-Clustern bevorzugen.

== hetzner-k3s

Für Benutzer, die auf der Hetzner Cloud arbeiten, schauen Sie sich link:https://github.com/vitobotta/hetzner-k3s[hetzner-k3s] an. Dieses in Crystal geschriebene CLI-Tool übernimmt die zusätzlichen Schritte, die zur Einrichtung eines K3s-Clusters auf der Hetzner Cloud erforderlich sind.
