[#_prerequisite]
== Voraussetzung

Sie benötigen mindestens die Berechtigungen eines Cluster-Mitglieds.

[#_brand_configuration]
== Branding-Konfiguration

Um die Branding-Einstellungen zu konfigurieren,

. Klicken Sie auf *☰ > Globale Einstellungen*.
. Klicken Sie auf *Branding*.

[#_private_label_company_name]
=== Name des Private-Label-Unternehmens

Diese Option ersetzt "Rancher" in den meisten Fällen durch den von Ihnen angegebenen Wert. Dateien, die 'Rancher' im Namen enthalten, wie `+rancher-compose.yml+`, werden nicht geändert.

[#_support_links]
=== Support-Links

Verwenden Sie eine URL-Adresse, um neue "Problem melden"-Berichte zu senden, anstatt die Benutzer zur GitHub-Issues-Seite zu leiten. Optional Rancher-Community-Support-Links anzeigen.

[#_logo]
=== Logo

Laden Sie helle und dunkle Logos hoch, um das Rancher-Logo in der obersten Navigationsleiste zu ersetzen.

[#_primary_color]
=== Primärfarbe

Sie können die in der gesamten Benutzeroberfläche verwendete Primärfarbe mit einer benutzerdefinierten Farbe Ihrer Wahl überschreiben.

[#_fixed_banners]
=== Feste Banner

Ein benutzerdefiniertes festes Banner im Kopf- oder Fußbereich oder in beiden anzeigen.

Um die Banner-Einstellungen zu konfigurieren,

. Klicken Sie auf *☰ > Globale Einstellungen*.
. Klicken Sie auf *Banner*.

[#_custom_navigation_links]
== Benutzerdefinierte Navigationslinks

In diesem Abschnitt erfahren Sie, wie Sie die Links in der linken Navigationsleiste des *Cluster-Dashboards* konfigurieren. Um zum Cluster-Dashboard zu gelangen,

. Klicken Sie in der oberen linken Ecke auf *☰ > Clusterverwaltung*.
. Gehen Sie auf der *Clusters*-Seite zu dem Cluster, in dem Sie benutzerdefinierte Navigationslinks hinzufügen möchten, und klicken Sie auf *Erkunden*.

Es kann nützlich sein, einen Link für den schnellen Zugriff auf einem Cluster installierte Dienste hinzuzufügen. Sie könnten beispielsweise einen Link zur Kiali-Benutzeroberfläche für Cluster mit installiertem Istio hinzufügen oder einen Link zur Grafana-Benutzeroberfläche für Cluster mit installiertem Rancher-Monitoring hinzufügen.

Die benutzerdefinierten Links beeinflussen nicht, wer Zugriff auf jeden Dienst hat.

Links können auf der obersten Ebene erstellt werden, und mehrere Links können zusammengefasst werden.

[#_adding_a_custom_navigation_link]
=== Hinzufügen eines benutzerdefinierten Navigationslinks

[NOTE]
.Voraussetzung:
====

Sie müssen mindestens die Berechtigungen eines Clustermitglieds oder Projektmitglieds haben.
====


. Klicken Sie auf *☰ > Clusterverwaltung*.
. Gehen Sie auf der *Clusters*-Seite zu dem Cluster, in dem Sie benutzerdefinierte Navigationslinks hinzufügen möchten, und klicken Sie auf *Erkunden*.
. Klicken Sie im oberen Navigationsmenü auf *🔍 (Ressourcensuche)*.
. Geben Sie *Nav* ein und klicken Sie auf *Nav-Links*.
. Klicken Sie auf *Erstellen aus YAML*.
. Der einfachste Weg, einen Navigationslink zu erstellen, besteht darin, diese Felder hinzuzufügen:

  name: linkname
  toURL: https://example.com
+
For more details on setting up links, including optional fields, see <<_link_configuration,Link Configuration.>>

. Klicken Sie auf *Erstellen*.

[#_link_configuration]
== Verbindungskonfiguration

[#__name]
=== `+name+`

Anzeigename für den Link. Erforderlich.

[#__group]
=== `+group+`

Name einer Linkgruppe, die sich beim Klicken erweitert.

Optional. Wenn nicht angegeben, erscheint der Link eigenständig.

Gruppen werden separat von eigenständigen Links angezeigt, wie unten gezeigt:

image::grouped-vs-standalone-links.png[Screenshot von Gruppe und eigenständigem Link]

[#__iconsrc]
=== `+iconSrc+`

Iconquelle im Base64-Format.

Nachfolgend ein Beispiel des Grafana-Logos im Base64-Format:



[#__annotation]
=== `+annotation+`

Eine Kubernetes-Annotation für die NavLink-Custom-Ressource.

[#__label]
=== `+label+`

Ein Kubernetes-Label für die NavLink-Custom-Ressource.

[#__sidelabel]
=== `+sideLabel+`

Label, das in der linken Navigationsleiste erscheint

[#__target]
=== `+target+`

Setzt die Ziel-Eigenschaft des Anker-Tags des Links (`+<a>+`), die (je nach Browser) bestimmt, ob es in einem neuen Fenster oder in einem bestehenden Tab geöffnet wird.

Der Standardwert ist `+_self+`, der den Link im aktuellen Tab öffnet. Um den Link in einem neuen Fenster oder Tab zu öffnen, setzen Sie das Ziel auf `+_blank+`.

Für weitere Informationen zur Ziel-Eigenschaft siehe https://www.w3schools.com/tags/att_a_target.asp[diese Seite.]

[#__toservice]
=== `+toService+`

Hat fünf Felder, die konstruiert sind, um eine URL wie folgt zu erstellen: `+\https://<rancher_server_url>/k8s/clusters/<CLUSTER>/k8s/namespace/<NAMESPACE>/service/<SCHEME>:<NAME>:<PORT>/proxy/<PATH>+`

Zum Beispiel kann ein Link zu einem Überwachungsdienst wie folgt eingerichtet werden:

* Name: `+rancher-monitoring-grafana+`
* Namespace: `+cattle-monitoring-system+`
* Pfad: `+proxy/?orgId=1+`
* Port: `+"80"+`
* scheme: `+http+`

Es ist erforderlich, entweder die `+toService+`-Direktive oder die `+toURL+`-Direktive bereitzustellen.

[#__tourl]
=== `+toUrl+`

Kann jeder Link sein, sogar zu Links außerhalb des Clusters.

Es ist erforderlich, entweder die `+toService+`-Direktive oder die `+toURL+`-Direktive bereitzustellen.

[#_link_examples]
== Link-Beispiele

[#_example_of_link_with_tourl]
=== Beispiel eines Links mit `+toUrl+`

Dieses Beispiel für NavLink-YAML zeigt ein Beispiel zur Konfiguration eines NavLink zu einem Grafana-Dashboard:

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  name: grafana
spec:
  group: "Monitoring Dashboards"
  toURL: https://<RANCHER_SERVER_URL>/api/v1/namespaces/cattle-monitoring-system/services/http:rancher-monitoring-grafana:80/proxy/?orgId=1
----

Das Hinzufügen des obigen YAML führt dazu, dass ein Link zu Grafana erstellt wird, wie im folgenden Screenshot gezeigt:

image::example-grafana-link.png[Screenshot des Grafana-Links]

[#_example_of_link_with_toservice]
=== Beispiel eines Links mit `+toService+`

Dieses Beispiel-YAML zeigt ein Beispiel für `+toService+`, das als Linkziel verwendet wird:

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  annotations:
    key: annotation
    labels:
      key: label
  name: navlinkname
spec:
  description: This is a description field # Optional.
  group: "group1" # Optional. If not provided, the links appear standalone.
  iconSrc: data:image/jpeg;base64,[icon source string is clipped for brevity]
  label: This is a label # Optional.
  sideLabel: A side label. # Optional.
  target: _blank #Optional. _blank opens the link in a new tab or window.
  toService: # toService or #toUrl needs to be provided.
    name: rancher-monitoring-grafana
    namespace: cattle-monitoring-system
    path: proxy/?orgId=1
    port: "80"
    scheme: http
----

Das Hinzufügen der oben genannten `+toService+`-Parameter führt dazu, dass ein Link zu Grafana erstellt wird, wie im folgenden Screenshot gezeigt:

image::example-service-link.png[Screenshot des Grafana-Links]
