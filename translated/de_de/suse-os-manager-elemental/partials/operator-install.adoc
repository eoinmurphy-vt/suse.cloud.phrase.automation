== Installieren {elemental-product-name} Bediener
:revdate: 2025-09-08
:page-revdate: {revdate}

`+elemental-operator+` ist der Verwaltungsendpunkt, der den Verwaltungscluster betreibt und sich um die Erstellung von Inventaren, die Registrierung von Maschinen und vieles mehr kümmert.

Wir werden den Helm-Paketmanager verwenden, um die elemental-operator-Karte in unserem Cluster zu installieren.

[,console]
----
helm upgrade --create-namespace -n cattle-elemental-system --install elemental-operator-crds oci://registry.suse.com/rancher/elemental-operator-crds-chart

helm upgrade --create-namespace -n cattle-elemental-system --install elemental-operator oci://registry.suse.com/rancher/elemental-operator-chart
----

Nach ein paar Sekunden sollte der Operator-Pod im Namensraum `+cattle-elemental-system+` erscheinen:

[,console]
----
kubectl get pods -n cattle-elemental-system

NAME                                  READY   STATUS    RESTARTS   AGE
elemental-operator-64f88fc695-b8qhn   1/1     Running   0          16s
----

[IMPORTANT]
.Helm v3.8.0+ erforderlich
----
Die {elemental-product-name} Operator-Karte wird über ein OCI-Register verteilt: Helm unterstützt ab der Version 3.8.0 korrekt OCI-basierte Registries.
----


[CAUTION]
.Vertauschen der Reihenfolge der Karteninstallation beim Upgrade von elemental-operator < 1.2.4
----
Beim Upgrade von einer elemental-operator-Version, die die {elemental-product-name} CRDs (Version < 1.2.4) einbettet, schlägt die Installation der elemental-operator-crds-Karte fehl.
Sie müssen zuerst die Karte elemental-operator aktualisieren und erst dann die Karte elemental-operator-crds installieren.
----

=== Nicht stabile Anlagen

Neben den oben aufgeführten Helm-Karten sind zwei weitere Versionen von `+non-stable+`erhältlich.

* *Staging:* bezieht sich auf die letzte getaggte Version von Github. Dies ist auf den xref:quickstarts/quickstart-ui.adoc[nächsten] Seiten dokumentiert.
* *Entwicklung:* bezieht sich auf den "Tip of HEAD" von Github. Dies ist die aktuelle Entwicklungsversion, die sich ständig ändert.

[tabs]
=====
Staging-Version (x86-64, ARM64 (Raspberry Pi 4))::
+
--
[,console]
----
helm upgrade --create-namespace -n cattle-elemental-system --install elemental-operator-crds oci://registry.opensuse.org/isv/rancher/elemental/staging/charts/rancher/elemental-operator-crds-chart 
helm upgrade --create-namespace -n cattle-elemental-system --install elemental-operator oci://registry.opensuse.org/isv/rancher/elemental/staging/charts/rancher/elemental-operator-chart
----
--

Entwicklungsversion (x86-64, ARM64 (Raspberry Pi 4))::
+
--
[WARNING]
.Erinnerung
====
Die Entwicklungsversion wird nicht für Produktionsumgebungen empfohlen. Wir freuen uns über Feedback über Slack oder Github Issues, aber es könnte instabil sein und experimentelle Funktionen enthalten, die ohne Vorankündigung entfernt werden können.
====

[,console]
----
helm upgrade --create-namespace -n cattle-elemental-system --install elemental-operator-crds oci://registry.opensuse.org/isv/rancher/elemental/dev/charts/rancher/elemental-operator-crds-chart
helm upgrade --create-namespace -n cattle-elemental-system --install --set image.imagePullPolicy=Always elemental-operator oci://registry.opensuse.org/isv/rancher/elemental/dev/charts/rancher/elemental-operator-chart
----
--
=====

=== Installationsmöglichkeiten

Es gibt einige Optionen, die bei der Installation des Diagramms eingestellt werden können, aber das würde den Rahmen dieses Dokuments sprengen. Sie können alle Werte in der Tabelle https://github.com/rancher/elemental-operator/blob/main/.obs/chartfile/elemental-operator-helm/values.yaml[values.yaml] sehen.
