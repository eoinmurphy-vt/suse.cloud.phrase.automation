= MachineInventorySelectorTemplate-Referenz
:revdate: 2025-09-08
:page-revdate: {revdate}

Das MachineInventorySelectorTemplate ist eine benutzerdefinierte Ressource, die als Blaupause für die Erstellung der erforderlichen xref:references/machineinventoryselector-reference.adoc[MachineInventorySelectors] verwendet wird.

Es ist die Ressource, die für die Definition der Abgleichskriterien verantwortlich ist, um eine inventarisierte Maschine mit einer Cluster-Ressource zu verbinden.

Der relevante Schlüssel ist `+selector+`, der die Ausdrücke des Etikettenselektors enthält.

.MachineInventorySelectorTemplate
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineInventorySelectorTemplate
metadata:
  name: my-machine-selector
  namespace: fleet-default
spec:
  template:
    spec:
      selector:
        ...
----

`+template.spec.selector+` können die Tasten `+matchLabels+` und oder `+matchExpressions+` enthalten.

== `+template.spec.selector.matchLabels+`

Es ist eine Karte mit `+{key,value}+` Paaren `+(map[string]string)+`. Wenn mehrere Etiketten angegeben werden, müssen alle Etiketten übereinstimmen.

.Klicken Sie hier für Details
[%collapsible]
====
[yaml]
----
...
spec: template: spec: selector: matchLabels: element: fire Hersteller: somevalue
----
====

Ein mit dem obigen Selektor definierter Cluster wird nur versuchen, Knoten bereitzustellen, die mit diesen beiden Bezeichnungen inventarisiert sind.

== `+template.spec.selector.matchExpressions+`

Es handelt sich um eine Liste von Label-Selektoren, wobei jeder Label-Selektor wie folgt definiert werden kann:

|===
| Schlüssel | Typ | Beschreibung

| Schlüssel
| String
| Dies ist der Beschriftungsschlüssel, auf den der Selektor angewendet wird

| operator
| String
| Stellt die Beziehung des Schlüssels zu einer Gruppe von Werten dar. Gültige Operatoren sind 'In', 'NotIn', 'Exists' und 'DoesNotExist'.

| Werte
| []string
| Values ist ein Array von String-Werten. Wenn der Operator "In" oder "NotIn" ist, darf das Wertefeld nicht leer sein. Wenn der Operator 'Exists' oder 'DoesNotExist' ist, muss das Wertefeld leer sein
|===

.Klicken Sie hier für Details
[%collapsible]
====
[,yaml]
----
...
spec:
  template:
    spec:
      selector:
        matchExpressions:
        - key: element
          operator: In
          values: [ 'fire' ]
        - key: manufacturer
          operator: Exists
----
====

Ein mit dem obigen Selektor definierter Cluster wird nur versuchen, Knoten bereitzustellen, die mit dem Label `+element=fire+` inventarisiert sind und ein Label `+manufacturer+` mit einem beliebigen Wert enthalten.
