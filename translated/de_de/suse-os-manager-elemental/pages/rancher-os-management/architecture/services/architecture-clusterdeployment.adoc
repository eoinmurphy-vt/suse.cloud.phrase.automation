= Bereitstellung von Kubernetes-Clustern
:revdate: 2025-11-06
:page-revdate: {revdate}

Das Ziel der Kubernetes-Cluster-Bereitstellungsphase ist die Erstellung eines neuen RKE2- oder K3s-Clusters unter Verwendung der verfügbaren xref:references/machineinventory-reference.adoc[MachineInventories], d. h. der Hosts, die die xref:rancher-os-management/architecture/services/architecture-machineonboarding.adoc[Machine-Onboarding-Phase] erfolgreich abgeschlossen haben.

Die Bereitstellung des {elemental-product-name} Kubernetes-Clusters umfasst die folgenden Schritte:

* Der Benutzer erstellt eine xref:references/machineinventoryselectortemplate-reference.adoc[MachineInventorySelectorTemplate-Ressource]: Sie ermöglicht die Definition eines _Selektors_ zur Identifizierung einer Teilmenge der verfügbaren xref:references/machineinventory-reference.adoc[MachineInventories] auf der Grundlage des Werts ihrer Bezeichnungen.
* Der Benutzer definiert einen xref:references/cluster-reference.adoc[Rancher-Cluster] und fügt der `+machinePools+` -Definition einen Verweis auf das im vorherigen Schritt erstellte xref:references/machineinventoryselectortemplate-reference.adoc[MachineInventorySelectorTemplate] hinzu.
* Die https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/launch-kubernetes-with-rancher.html[RKE2/K3s Cluster-Bereitstellung] Rancher reagiert auf die Erstellung von Rancher-Cluster-Ressourcen, indem sie eine Anzahl von xref:references/machineinventoryselector-reference.adoc[MachineInventorySelectors-Ressourcen] erzeugt, die der in den _machinePools_ angegebenen _Menge_ entspricht.
* Der {elemental-product-name} Operator paart jede erzeugte xref:references/machineinventoryselector-reference.adoc[MachineInventorySelector-Ressource] mit einem verfügbaren xref:references/machineinventory-reference.adoc[MachineInventory] und installiert den https://github.com/rancher/system-agent[rancher-system-agent-Daemon] auf dem vom xref:references/machineinventory-reference.adoc[MachineInventory] verfolgten Host.
Das https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/launch-kubernetes-with-rancher.html[RKE2/K3s Cluster Provisioning] Rancher übernimmt das K3s/RKE2 Provisioning mit https://github.com/rancher/system-agent[rancher-system-agent] _plans_: es installiert die benötigten Komponenten (z.B. containerd, K3s, ...) und erstellt die Konfigurationsdateien bis zum erfolgreichen Deployment des neuen Kubernetes-Clusters.
