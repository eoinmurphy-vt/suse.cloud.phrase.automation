= Wie man {elemental-product-name} mit Rancher und VMware verwendet
:revdate: 2025-09-08
:page-revdate: {revdate}

== Excerpt

In diesem Dokument wird gezeigt, wie man eine virtuelle Maschine in VMware Workstation so einrichtet, dass Elemental-Knoten gebootet werden.

== Voraussetzungen

Rancher 2.8 oder höher ist installiert und läuft. Siehe die Kurzanleitungen.

== Schritt 1: Erstellen Sie den Registrierungsendpunkt

Wir müssen ein ISO-Image erstellen, um die Knoten zu booten und bei der Rancher-Instanz zu registrieren. Hierfür ist ein Registrierungsendpunktxref:references/machineregistration-reference.adoc[(] Ressourcexref:references/machineregistration-reference.adoc[MachineRegistration] ) erforderlich.

Siehe dieses Beispiel von `+MachineRegistration+`:

[,yaml]
.registration.yaml
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: fire-nodes
  namespace: fleet-default
spec:
  config:
    cloud-config:
      users:
        - name: root
          passwd: root
    elemental:
      install:
        reboot: true
        device: /dev/sda
        debug: true
  machineInventoryLabels:
    element: fire
    manufacturer: "${Product/Vendor}"
    productName: "${Product/Name}"
    serialNumber: "${Product/Serial Number}"
    machineUUID: "${Product/UUID}"
----

Bei der obigen Maschinenregistrierung wird davon ausgegangen, dass die Knoten über ein TPM 2.0 verfügen. Für den Fall, dass die virtualisierte Zielmaschine kein virtuelles TPM-Gerät enthält, kann im Abschnitt `+config.elemental.registration+` eine Software-Emulation konfiguriert werden.

Betrachten Sie das folgende Beispiel:

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  ...
spec:
  config:
    cloud-config:
      ...
    elemental:
      install:
        ...
      registration:
        emulate-tpm: true
        emulated-tpm-seed: -1
  machineInventoryLabels:
    ...
----

`+emulated-tpm-seed: -1+` legt fest, dass der Client einen zufälligen Seed zur Berechnung des TPM-Hashes verwendet. Dies ist nützlich, um dieselbe Definition des Registrierungsendpunkts für mehrere Rechner wiederverwenden zu können. Siehe weitere xref:operator-operational-tasks/tpm.adoc[TPM-Dokumentation].

== Schritt 2: Erstellen Sie die Installations-ISO

Das Installationsmedium muss an einen bestimmten Registrierungsendpunkt gebunden sein. Diese wird mit der Ressource xref:references/seedimage-reference.adoc[SeedImage] erstellt und behandelt.

Betrachten Sie das folgende Beispiel:

[,yaml]
.seedimage.yaml
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: fire-img
  namespace: fleet-default
spec:
  baseImage: registry.suse.com/suse/sle-micro-iso/5.5:2.0.2
  registrationRef:
    apiVersion: elemental.cattle.io/v1beta1
    kind: MachineRegistration
    name: fire-nodes
    namespace: fleet-default
----

Sobald die SeedImage-Ressource erstellt ist, beginnt sie mit der Erstellung einer ISO-Datei mit dem bereitgestellten Betriebssystem-Image und der Verknüpfung mit dem angegebenen Registrierungsendpunkt. Sobald dies geschehen ist, wird eine Download-URL im SeedImage-Ressourcenstatus verfügbar sein.

Sie können es mit herunterladen:

[,shell]
----
wget --no-check-certificate --content-disposition $(kubectl get seedimages.elemental.cattle.io -n fleet-default fire-img -o jsonpath="{.status.downloadURL}")
----

== Schritt 3: Booten Sie das Zielgerät

Idealerweise brennen Sie die ISO-Datei auf ein USB-Laufwerk und booten Ihr Edge-Gerät über das USB-Gerät. Sobald es hochgefahren und in Rancher unter Maschineninventar aktiv ist, können Sie es auswählen und einen Cluster erstellen.

=== 3.1 Vorbereiten der VM zur Emulation des TPM

Erstellen Sie in VMware Workstation eine vm, wie Sie es normalerweise tun würden, und stellen Sie sicher, dass die Festplattengröße mindestens 40 GB beträgt.

Bearbeiten Sie nun die Maschineneinstellungen und gehen Sie auf die Registerkarte "Optionen". Die allerletzte Option wäre "Erweitert".

Klicken Sie auf "Erweitert" und ändern Sie im rechten Fensterbereich den Firmware-Typ von "BIOS" auf "UEFI" und aktivieren Sie die Option "Sicheres Booten aktivieren" wie folgt:

* Standardeinstellungen bei ausgewähltem BIOS

image::rancher-vmware-vm-boot-bios.png[VM-Boot-Optionen mit BIOS]

* Aktualisierte Einstellungen mit ausgewähltem UEFI und aktiviertem Secure Boot

image::rancher-vmware-vm-boot-uefi.png[VM-Boot-Optionen mit UEFI]

Klicken Sie nun auf der gleichen Registerkarte "Optionen" auf die Option "Zugriffskontrolle" und auf der rechten Seite auf "Verschlüsseln".

image::rancher-vmware-access-control-menu.png[Menü Zugangskontrolle]

Sie werden aufgefordert, ein Passwort einzugeben, um den Rechner zu verschlüsseln. Geben Sie ein Passwort ein und klicken Sie auf "Verschlüsseln".

image::rancher-vmware-access-control-encrypt.png[Zugangskontrolle Verschlüsselungsnachweise]

Dies ist wichtig, um die TPM-Hardware hinzuzufügen. Gehen Sie dann zurück zu den Hardware-Optionen und klicken Sie auf "Hinzufügen".

Fügen Sie die TPM-Hardware (Trusted Platform Module) hinzu und klicken Sie auf "Fertig stellen".

Nach Abschluss dieses Schritts ist unsere VM nun bereit.

=== 3.2 Booten Sie die VM mit der elementaren ISO

Als Nächstes fügen Sie die zuvor erstellte ISO-Datei in die VM ein und starten sie.

Es sollte mit der ISO hochfahren und mit der Installation von Elemental beginnen:

image::rancher-vmware-elemental-install-grub.png[{elemental-product-name} OS-Installationsmenü grub]

image::rancher-vmware-elemental-install-logs.png[{elemental-product-name} OS-Installationsprotokolle]

Sobald der Vorgang abgeschlossen ist, wird die VM neu gestartet und im Maschineninventar in Rancher als aktiv angezeigt.
