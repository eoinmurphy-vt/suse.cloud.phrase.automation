= Zufällige Template-Variablen
:revdate: 2025-09-08
:page-revdate: {revdate}

Zufällige Template-Variablen sind in den Elemental Operator eingebaut.

Sie ermöglichen die Aufnahme von zufälligen Werten `+Int+`, `+Hex+` oder `+UUID+` in benutzerdefinierte xref:label-templates/label-templates.adoc[Etikettenvorlagen].

Die Werte werden während des Renderings von `+label template variables+` im laufenden Betrieb berechnet.

[IMPORTANT]
.Zufällige Etikettenvorlagen werden nur einmal gerendert
====

Eine Etikettenvorlage, die eine Zufallsvariable enthält, wird nur gerendert, wenn das xref:references/machineinventory-reference.adoc[MachineInventory] des registrierenden Hosts noch keinen Wert für dieses Etikett hat (ein Etikett mit demselben Schlüssel fehlt oder sein Wert ist leer).

Es gibt also drei Fälle, in denen eine Etikettenvorlage mit einer Zufallsvariablen wiedergegeben wird:

. der Host registriert sich zum ersten Mal und das xref:references/machineinventory-reference.adoc[MachineInventory] wird neu erstellt.
. die Etikettenvorlage wurde der MachineRegistration hinzugefügt, nachdem sich der Host das letzte Mal (erneut) registriert hat.
. das xref:references/machineinventory-reference.adoc[MachineInventory-Etikett], das der Etikettenvorlage entspricht (gleicher Etikettenschlüssel), wurde manuell entfernt oder sein Wert wurde gelöscht.
====


|===
| Variabel | Beschreibung | von

| `+${Random/UUID}+`
| zufällige UUID (z. B. fd95324a-c26b-4e28-8727-1dcec293a0ec)
| 1.7.0

| `+${Random/Hex/[1-32]}+`
| zufällige hexadezimale Zeichenkette mit der angegebenen Länge (mindestens 1, höchstens 32)
| 1.7.0

| `+${Random/Int/[MAXINT]+`
| zufällige Ganzzahl (min 0, max MAXINT-1)
| 1.7.0
|===

[NOTE]
.Rendering-Beispiele
====
|===
| Vorlagenwert | Beispiel für einen gerenderten Wert

| `+${Random/UUID}+`
| fd95324a-c26b-4e28-8727-1dcec293a0ec

| `+${Random/Hex/12}+`
| acd231f222b8

| `+${Random/Int/10000}+` 
| 9432

|===
====


Die Zufallsvariablen können nützlich sein, um benutzerdefinierte Hostnamen zu erzeugen, die dem registrierenden Host zugewiesen werden.

Da der Hostname eindeutig sein muss und über das Feld xref:references/machineregistration-reference.adoc[MachineRegistration] `+spec.machineName+` zugewiesen wird, können Zufallsvariablen verwendet werden, um die Eindeutigkeit einer Gruppe von Hosts mit demselben benutzerdefinierten Präfix und/oder Suffix zu gewährleisten.

Weitere Informationen finden Sie im Abschnitt xref:installation/hostname.adoc[HowTo/Customize hostname].
