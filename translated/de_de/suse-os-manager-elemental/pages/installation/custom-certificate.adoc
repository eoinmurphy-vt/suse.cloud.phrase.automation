= So fügen Sie ein benutzerdefiniertes Zertifikat hinzu
:revdate: 2024-09-24
:page-revdate: {revdate}

Voraussetzung: Eine Bescheinigung im Format `+.pem+` 

Das Ziel: Ein benutzerdefiniertes Zertifikat systemweit verfügbar machen

[NOTE]
.Dies gilt für Zertifikate, die von Diensten auf Systemebene verwendet werden.
====
Kubernetes-Workloads sollten ihre Zertifikate stattdessen in das Container-Image einbringen.
====

Um ein benutzerdefiniertes Zertifikat zu installieren, müssen wir

* Kopieren Sie die Datei `+.pem+` nach `+/etc/pki/trust/anchors/+`
* run `+update-ca-certificates+`

Das entsprechende `+cloud-config+` Snippet sieht wie folgt aus:

[,yaml]
----
write_files:
  - path: /etc/pki/trust/anchors/my-custom-certificate.pem
    permission: 0444
    content: |-
      -----BEGIN CERTIFICATE-----
      ...
      -----END CERTIFICATE-----
runcmd:
  - update-ca-certificates
----

(aus Gründen der Kürze wurde der tatsächliche Inhalt des Zertifikats weggelassen)
