[NOTE]
====
Die folgenden Anweisungen benötigen mindestens Rancher 2.9.x.
====

Dieser Schnellstart zeigt Ihnen, wie Sie das {elemental-product-name} Plugin und den Operator in einer bestehenden Rancher Manager-Instanz einsetzen.

Nach der Installation können Sie einen neuen {elemental-product-name} -Cluster auf der Basis von RKE2 oder K3 bereitstellen.

Wenn Sie jedoch einen Staging- oder Dev-Operator installieren möchten, können Sie dies nur im xref:quickstarts/quickstart-cli.adoc#_non_stable_installations[CLI-Modus] tun.

== Das offizielle Rancher-Erweiterungs-Repository hinzufügen

Wenn die Elemental-Erweiterung nicht verfügbar ist, müssen Sie die `+Official Rancher Extensions Repository+` hinzufügen:

image::quickstart-ui-extension-repository.png[Repository für Rancher-Manager-Erweiterungen hinzufügen]

Wenn dieses Repository nicht über die Einstellungen der Benutzeroberfläche für Erweiterungen hinzugefügt werden kann, kann es manuell verwaltet werden, indem https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-admin/helm-charts-in-rancher/helm-charts-in-rancher.html#_manage_repositories[hinzufügt] das Repository `+https://github.com/rancher/ui-plugin-charts+` `+git+`  oder alternativ die folgende Ressource verwendet wird:

[source,yaml]
----
apiVersion: catalog.cattle.io/v1
kind: ClusterRepo
metadata:
  name: rancher-ui-charts
spec:
  gitBranch: main
  gitRepo: https://github.com/rancher/ui-plugin-charts
----

== Installieren Sie das Elementar-Plugin

Nachdem die Unterstützung für Rancher Manager Extensions aktiviert wurde, können Sie das Plugin `+elemental+` wie folgt installieren:

* Unter der Registerkarte `+Available+` finden Sie das Plugin `+elemental+`.

image::quickstart-ui-extensions-available.png[Rancher Manager Verfügbare Plugins]

[NOTE]
====
Wenn die Registerkarte `+Available+` keine Einträge enthält, aktualisieren Sie die Seite. Daraufhin wird das Plugin `+elemental+` angezeigt.
====

* Klicken Sie auf die Schaltfläche `+Install+`. Es erscheint ein Popup-Fenster, und klicken Sie erneut auf `+Install+`, um fortzufahren.

image::quickstart-ui-elemental-plugin-install.png[{elemental-product-name} Plugin-Installation]

* Auf der Registerkarte `+Installed+` ist nun das Plugin `+elemental+` aufgeführt.

[NOTE]
====
Wenn das Plugin `+elemental+` aufgelistet ist und der Status auf `+Installing...+` bleibt, aktualisieren Sie die Seite. Das Plugin `+elemental+` wird korrekt angezeigt.
====

Sobald das Plugin `+elemental+` installiert ist, können Sie die Option `+OS Management+` im Menü des Rancher Managers sehen; aktualisieren Sie die Seite, wenn Sie sie nicht sehen.

image::quickstart-ui-elemental-plugin-menu.png[Menü Rancher Manager OS-Verwaltung]

== Installieren Sie den Elementaroperator

[NOTE]
====
Die folgende Anleitung zeigt Ihnen, wie Sie den Operator über die Benutzeroberfläche {elemental-product-name} installieren. Sie können es aber auch direkt vom Marketplace installieren.
====

Klicken Sie im Navigationsmenü auf die Schaltfläche OS Management.

Wenn der Operator noch nicht installiert ist, können Sie ihn über die elementare Benutzeroberfläche einrichten, indem Sie auf die Schaltfläche `+Install {elemental-product-name} Operator+` klicken:

image::quickstart-ui-extension-operator-button.png[Schaltfläche für den Einsatz des Elementaroperators]

Sie werden dann zum Rancher Marketplace weitergeleitet, um den Operator zu installieren.

Klicken Sie auf die Schaltfläche `+Next+`:

image::quickstart-ui-extension-operator-install-1.png[Installieren {elemental-product-name} operator bildschirmfoto 1]

In diesem Bildschirm können Sie die Werte anpassen oder die Standardwerte verwenden. Klicken Sie auf `+Install+`, um fortzufahren:

image::quickstart-ui-extension-operator-install-2.png[Installieren Sie {elemental-product-name} operator screenshot 2]

Sie sollten sehen, dass ``+elemental-operator-crds++ und`` +`elemental-operator`+ im Namespace `+cattle-elemental-system` eingesetzt werden:

image::quickstart-ui-extension-operator-install-3.png[Installieren Sie {elemental-product-name} operator screenshot 3]

[CAUTION]
====
Wenn Sie diese nicht sehen, stellen Sie sicher, dass Sie den richtigen Namensraum oben auf der Seite auswählen.
====

== Hinzufügen eines Endpunkts für die Maschinenregistrierung

Klicken Sie im OS Management Dashboard auf die Schaltfläche `+Create Registration Endpoint+`.

image::quickstart-ui-registration-endpoint-create.png[OS Management Registrierungsendpunkte]

Hier können Sie nun entweder jedes Detail an den entsprechenden Stellen eintragen oder Sie können dies als YAML bearbeiten und den Endpunkt in einem Rutsch erstellen. Hier werden wir alle Felder bearbeiten.

image::quickstart-ui-registration-endpoint-create-details.png[Erstellen eines Registrierungsendpunkts mit UI]

[IMPORTANT]
.Wichtigste Optionen
====
`+name: elemental-cluster1+`: Ändern Sie dies nach Ihrem Bedarf

`+device-selector+`: Das Feld für xref:references/machineregistration-reference.adoc#_config_elemental_install_device_selector[die Geräteauswahl] kann verwendet werden, um das Gerät während der Installation dynamisch auszuwählen. Das Feld enthält eine Liste von Regeln zur Auswahl des gewünschten Geräts.

`+snapshotter+`: Gerätetyp, der zur Verwaltung von Snapshots in Betriebssystem-Images verwendet wird.
====

Sobald Sie den Endpunkt der Maschinenregistrierung erstellt haben, sollte er als aktiv angezeigt werden.

image::quickstart-ui-registration-endpoint-complete.png[Maschine registriert in Registration Endpoints]

== Vorbereitung des Installationsabbilds (Seed)

Dies ist der letzte Schritt. Sie müssen ein Seed-Image vorbereiten, das die anfängliche Registrierungskonfiguration enthält, damit es automatisch registriert, installiert und vollständig als Teil Ihres Clusters bereitgestellt werden kann. Der Inhalt der Datei ist nichts anderes als die Registrierungs-URL, die der Knoten zur Registrierung benötigt, und das richtige Serverzertifikat, damit er eine sichere Verbindung herstellen kann.

Dieses Seed-Image kann dann für die Bereitstellung einer unbegrenzten Anzahl von Rechnern verwendet werden.

Das Seed-Image wird oben als Kubernetes-Ressource erstellt und kann über die Schaltfläche `+Build Media+` erstellt werden, aber zunächst müssen Sie ISO- oder RAW-Image auswählen.

Im Gegensatz zu ISO, bei dem zwei Geräte benötigt werden (ein Gerät mit ISO und ein weiteres, auf dem {elemental-product-name} installiert wird), kann das RAW-Image von einem einzigen Gerät gebootet und das Betriebssystem direkt auf dem Gerät installiert werden.
Das RAW-Image enthält nur eine Boot- und eine Wiederherstellungspartition und bootet zuerst in den Wiederherstellungsmodus, um {elemental-product-name} zu installieren (zur Information: der Vorgang ist ähnlich wie beim xref:node-operational-tasks/reset.adoc#_reset_workflow[Zurücksetzen] ).

image::quickstart-ui-registration-endpoint-build-media.png[Medien in Registrierungsendpunkten erstellen]

Sobald die Erstellung abgeschlossen ist, können die Medien über die Schaltfläche `+Download Media+` heruntergeladen werden:

image::quickstart-ui-registration-endpoint-download-media.png[Medien in Registrierungsendpunkten herunterladen]

Sie können nun Ihre Nodes mit diesem Image booten und sie werden es tun:

* Registrieren Sie sich mit der angegebenen registrationURL und erstellen Sie eine Pro-Maschine `+MachineInventory+`
* SLE Micro auf dem angegebenen Gerät installieren
* Neustart

== Maschineninventar

Wenn die Knoten zum ersten Mal hochfahren, verbinden sie sich mit dem Rancher Manager und für jeden Knoten wird ein xref:references/machineinventory-reference.adoc[`+Machine Inventory+`] wird für jeden Knoten erstellt.

image::quickstart-ui-machine-inventory-menu.png[Menü Maschineninventur]

Benutzerdefinierte Spalten basieren auf `+Machine Inventory Labels+`, die Sie beim Erstellen Ihrer `+Machine Registration Endpoint+` hinzufügen können:

image::quickstart-ui-registration-endpoint-hardware-labels.png[Maschinenregistrierung Endpunkt Hardware-Etiketten]

Im folgenden Screenshot werden `+Hardware Labels+` als benutzerdefinierte Spalten verwendet:

Sie können auch benutzerdefinierte Spalten hinzufügen, indem Sie auf das Menü mit den drei Punkten klicken.

image::quickstart-ui-machine-inventory-custom-columns.png[Benutzerdefinierte Spalten des Maschinenbestands]

Schließlich können Sie Ihre `+Machine Inventory+` auch nach diesen Bezeichnungen filtern.

Wenn Sie z. B. nur Ihre AMD-Rechner sehen möchten, können Sie nach `+CPUModel+` filtern, wie unten dargestellt:

image::quickstart-ui-machine-inventory-filtering.png[Filterung des Maschinenbestands]

== Erstellen Sie Ihren ersten {elemental-product-name} Cluster

Verwenden wir nun diese `+Machine Inventory+`, um einen Cluster zu erstellen, indem wir auf `+Create {elemental-product-name} Cluster+` klicken:

image::quickstart-ui-create-cluster-button.png[Schaltfläche {elemental-product-name} Cluster erstellen]

Für Ihren {elemental-product-name} -Cluster können Sie entweder K3s oder RKE2 für Kubernetes wählen.

image::quickstart-ui-create-cluster-standard-screen-.png[{elemental-product-name} Bildschirm "Cluster-Erstellung]

Die meisten Optionen kommen von Rancher, deshalb werden wir nicht alle Möglichkeiten im Detail aufzählen.
Weitere Informationen finden Sie in der Rancher https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/k3s.html[Dokumentation zur Konfiguration der] oder in der Rancher unter https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/rke2.html[Manager K3s-Verteilung unter].

Es ist jedoch wichtig, den Abschnitt `+Inventory of Machines Selector Template+` hervorzuheben.

Sie können wählen, welche `+Machine Inventory+` Sie verwenden möchten, um Ihren {elemental-product-name} Cluster unter Verwendung der zuvor definierten `+Machine Inventory Labels+` zu erstellen:

image::quickstart-ui-create-cluster-machine-selector-template.png[Auswahlvorlage für das Maschineninventar verwenden]

Da unsere drei Maschineninventare die Bezeichnung `+CPUVendor+` mit dem Schlüssel `+AuthenticAMD+` enthalten, werden die drei Maschinen zur Erstellung des Clusters {elemental-product-name} verwendet.
