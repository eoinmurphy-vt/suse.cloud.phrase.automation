== スクリプトのインストール

{k3s-product-name} は、systemdまたはopenrcベースのシステムにサービスとしてインストールする便利な方法であるインストールスクリプトを提供している。このスクリプトはhttps://get.k3s.io. で入手できる。この方法で{k3s-product-name} をインストールするには、以下のようにする：

インストーラーを実行してください。`+INSTALL_K3S_ARTIFACT_URL+` はhttps://scc.suse.com/rancher-docs/rancherprime/latest/en/reference-guide.html#prime-artifacts-url[Prime Artifacts URL]、`+INSTALL_K3S_CHANNEL+` は購読できるリリースチャンネルで、デフォルトは`+stable+` です。この例では、`+INSTALL_K3S_CHANNEL="latest"+` が{k3s-product-name} の最新版を提供している。

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_CHANNEL="latest" sh -
----
バージョンを指定したい場合は、`+INSTALL_RKE2_VERSION+` 環境変数を設定する：

[,bash]
----
curl -sfL https://get.k3s.io/ | sudo INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_VERSION="<VERSION>" sh -
----

このインストールを実行した後

* K3sサービスは、ノードがリブートした後、またはプロセスがクラッシュしたり、強制終了された後に自動的に再起動するように設定されます。
* `+kubectl+` 、`+crictl+` 、`+ctr+` 、`+k3s-killall.sh+` を含む追加のユーティリティがインストールされる。 `+k3s-uninstall.sh+`
* https://kubernetes.io/docs/concepts/configuration/organize-cluster-access-kubeconfig/[]kubeconfigファイルが`+/etc/rancher/k3s/k3s.yaml+` に書き込まれ、{k3s-product-name} でインストールされたkubectlが自動的に使用されます。

シングルノードサーバーのインストールは、ワークロードPodをホストするのに必要なすべてのデータストア、コントロールプレーン、kubelet、およびコンテナランタイムコンポーネントを含む、完全に機能するKubernetesクラスタです。サーバやエージェントのノードを追加する必要はありませんが、クラスタに容量や冗長性を追加するために追加する場合があります。

追加のエージェント・ノードをインストールしてクラスタに追加するには、`+K3S_URL+` および`+K3S_TOKEN+` 環境変数を指定してインストール・スクリプトを実行します。以下は、エージェントに参加する方法の例です：

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s K3S_URL=https://myserver:6443 K3S_TOKEN=mynodetoken sh -
----

`+K3S_URL+` パラメータを設定すると、インストーラはK3sをサーバーではなくエージェントとして設定します。K3sエージェントは、指定されたURLでリッスンしているK3sサーバーに登録します。`+K3S_TOKEN+` 、サーバー・ノードの`+/var/lib/rancher/k3s/server/node-token+` 。

[NOTE]
====
各マシンは固有のホスト名を持っていなければならない。マシンに固有のホスト名がない場合は、`+K3S_NODE_NAME+` 環境変数を渡し、各ノードに有効で固有のホスト名を指定する。
====


より多くのサーバーノードを持つことに興味がある場合は、xref:datastore/ha-embedded.adoc[High Availability Embedded etcdと] xref:datastore/ha.adoc[High Availability External DBの]ページを参照してください。
