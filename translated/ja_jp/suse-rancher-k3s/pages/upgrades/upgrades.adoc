== K3sクラスターのアップグレード

xref:upgrades/manual.adoc[手動アップグレードでは]、クラスタを手動でアップグレードするためのいくつかのテクニックを説明します。また、https://www.terraform.io/[Terraformの]ようなサードパーティのInfrastructure-as-Codeツールによるアップグレードの基礎としても利用できる。

xref:upgrades/automated.adoc[自動アップグレードでは]、Rancherのhttps://github.com/rancher/system-upgrade-controller[system-upgrade-controllerを]使用してKubernetesネイティブの自動アップグレードを実行する方法について説明します。

== バージョン固有の注意事項

* *トラフィック*Traefikが無効化されていない場合、K3sバージョン1.20以前はTraefik v1がインストールされ、K3sバージョン1.21以降は、v1がまだ存在しない場合、Traefik v2がインストールされます。旧 Traefik v1 から Traefik v2 へアップグレードするには、https://doc.traefik.io/traefik/migrate/v1-to-v2/[Traefik ドキュメントを]参照し、https://github.com/traefik/traefik-migration-tool[移行ツールを]使用してください。
* *K3sブートストラップデータ：*外部SQLデータストアを使用したHA構成でK3sを使用しており、サーバー（コントロールプレーン）ノードが`+--token+` CLIフラグで起動されていない場合、トークンを指定せずにクラスタにK3sサーバーを追加することができなくなります。このトークンはバックアップからリストアする際に必要となるため、必ずコピーを保管してください。以前は、K3sは外部SQLデータストアを使用する際にトークンの使用を強制しませんでした。
 ** 影響を受けるバージョンは、<= v1.19.12+k3s1、v1.20.8+k3s1、v1.21.2+k3s1で、パッチを当てたバージョンはv1.19.13+k3s1、v1.20.9+k3s1、v1.21.3+k3s1です。
 ** 以下のようにして、すでにクラスタに参加しているサーバからトークン値を取得できます：
+
[,bash]
----
cat /var/lib/rancher/k3s/server/token
----
