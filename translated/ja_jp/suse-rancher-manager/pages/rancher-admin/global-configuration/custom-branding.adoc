[#_prerequisite]
== 前提条件

クラスター メンバーの権限を少なくとも持っている必要があります。

[#_brand_configuration]
== ブランド設定

ブランド設定を構成するには、

. *☰ > グローバル設定*をクリックします。
. *ブランディング*をクリックします。

[#_private_label_company_name]
=== プライベートラベル会社名

このオプションは、提供した値で「Rancher」をほとんどの場所で置き換えます。名前にRancherを含む必要があるファイル（例：`+rancher-compose.yml+`）は変更されません。

[#_support_links]
=== サポートリンク

新しい「問題を報告する」レポートを送信するためにURLアドレスを使用し、ユーザーをGitHubの問題ページに送信しないでください。オプションでRancherコミュニティサポートリンクを表示します。

[#_logo]
=== ロゴ

ライトとダークのロゴをアップロードして、トップレベルのナビゲーションヘッダーにあるRancherロゴを置き換えます。

[#_primary_color]
=== プライマリカラー

UI全体で使用されるプライマリカラーをお好みのカスタムカラーで上書きできます。

[#_fixed_banners]
=== 固定バナー

ヘッダー、フッター、またはその両方にカスタム固定バナーを表示します。

バナー設定を構成するには、

. *☰ > グローバル設定*をクリックします。
. *バナー*をクリックします。

[#_custom_navigation_links]
== カスタムナビゲーションリンク

このセクションでは、*Cluster Dashboard*の左側のナビゲーションバーにリンクを設定する方法を学びます。クラスターのダッシュボードにアクセスするには、

. 左上隅で、*☰ > クラスター管理*をクリックします。
. *Clusters*ページで、カスタムナビゲーションリンクを追加したいクラスターに移動し、*Explore*をクリックします。

クラスターにインストールされたサービスに迅速にアクセスするためのリンクを追加することは便利です。例えば、IstioがインストールされたクラスターにはKiali UIへのリンクを追加したり、Rancher監視がインストールされたクラスターにはGrafana UIへのリンクを追加したりできます。

カスタムリンクは、各サービスへのアクセス権に影響を与えません。

リンクは最上位レベルで作成でき、複数のリンクをグループ化することができます。

[#_adding_a_custom_navigation_link]
=== カスタムナビゲーションリンクの追加

[NOTE]
.前提条件:
====

クラスターのメンバーまたはプロジェクトのメンバー権限を少なくとも持っている必要があります。
====


. *☰ > クラスター管理*をクリックします。
. *Clusters*ページで、カスタムナビゲーションリンクを追加したいクラスターに移動し、*Explore*をクリックします。
. 上部のナビゲーションメニューで、*🔍 (リソース検索)*をクリックします。
. *Nav*と入力し、*Nav Links*をクリックします。
. *Create from YAML*をクリックします。
. ナビゲーションリンクを作成する最も簡単な方法は、これらのフィールドを追加することです：

  name: linkname
  toURL: https://example.com
+
For more details on setting up links, including optional fields, see <<_link_configuration,Link Configuration.>>

. ［*作成*］をクリックします。

[#_link_configuration]
== リンク構成

[#__name]
=== `+name+`

リンクの表示名。入力必須パラメータです。

[#__group]
=== `+group+`

クリックすると開くリンクのグループ名。

オプション。提供されない場合、リンクは単独で表示されます。

グループは、単独のリンクとは別に表示されます。以下に示します。

image::grouped-vs-standalone-links.png[グループおよびスタンドアロンリンクのスクリーンショット]

[#__iconsrc]
=== `+iconSrc+`

アイコンソースはbase64形式です。

以下は、base64形式のGrafanaロゴの例です。



[#__annotation]
=== `+annotation+`

NavLinkカスタムリソースのためのKubernetesアノテーション。

[#__label]
=== `+label+`

NavLinkカスタムリソースのためのKubernetesラベル。

[#__sidelabel]
=== `+sideLabel+`

左側のナビゲーションバーに表示されるラベル

[#__target]
=== `+target+`

リンクのアンカー (`+<a>+`) のtargetプロパティを設定します。これにより、ブラウザによっては新しいウィンドウまたは既存のタブで開くかどうかが決まります。

デフォルト値は`+_self+`で、現在のタブでリンクを開きます。リンクを新しいウィンドウまたはタブで開くには、targetを`+_blank+`に設定します。

targetプロパティに関する詳細は、https://www.w3schools.com/tags/att_a_target.asp[このページをご覧ください。]

[#__toservice]
=== `+toService+`

次のようなURLを作成するために構成された5つのフィールドがあります。`+\https://<rancher_server_url>/k8s/clusters/<CLUSTER>/k8s/namespace/<NAMESPACE>/service/<SCHEME>:<NAME>:<PORT>/proxy/<PATH>+`

例えば、監視サービスへのリンクは次のように設定できます。

* 名前：`+rancher-monitoring-grafana+`
* namespace: `+cattle-monitoring-system+`
* パス：`+proxy/?orgId=1+`
* port: `+"80"+`
* scheme: `+http+`

`+toService+`ディレクティブまたは`+toURL+`ディレクティブのいずれかを提供する必要があります。

[#__tourl]
=== `+toUrl+`

クラスター外のリンクでも、任意のリンクを使用できます。

`+toService+`ディレクティブまたは`+toURL+`ディレクティブのいずれかを提供する必要があります。

[#_link_examples]
== リンクの例

[#_example_of_link_with_tourl]
=== リンクの例 `+toUrl+`

この例のNavLink YAMLは、GrafanaダッシュボードへのNavLinkの設定例を示しています。

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  name: grafana
spec:
  group: "Monitoring Dashboards"
  toURL: https://<RANCHER_SERVER_URL>/api/v1/namespaces/cattle-monitoring-system/services/http:rancher-monitoring-grafana:80/proxy/?orgId=1
----

上記のYAMLを追加すると、以下のスクリーンショットに示すようにGrafanaへのリンクが作成されます。

image::example-grafana-link.png[Grafanaリンクのスクリーンショット]

[#_example_of_link_with_toservice]
=== リンクの例 `+toService+`

この例のYAMLは、リンクターゲットとして使用される`+toService+`の例を示しています。

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  annotations:
    key: annotation
    labels:
      key: label
  name: navlinkname
spec:
  description: This is a description field # Optional.
  group: "group1" # Optional. If not provided, the links appear standalone.
  iconSrc: data:image/jpeg;base64,[icon source string is clipped for brevity]
  label: This is a label # Optional.
  sideLabel: A side label. # Optional.
  target: _blank #Optional. _blank opens the link in a new tab or window.
  toService: # toService or #toUrl needs to be provided.
    name: rancher-monitoring-grafana
    namespace: cattle-monitoring-system
    path: proxy/?orgId=1
    port: "80"
    scheme: http
----

上記の`+toService+`パラメータを追加すると、以下のスクリーンショットに示すようにGrafanaへのリンクが作成されます。

image::example-service-link.png[Grafanaリンクのスクリーンショット]
