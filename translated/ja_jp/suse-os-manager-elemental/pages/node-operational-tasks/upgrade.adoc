= アップグレード
:revdate: 2025-09-08
:page-revdate: {revdate}

{elemental-product-name} 、すべてのコンポーネントはKubernetesを使って管理される。以下は、Kubernetesのアプローチを使ってコンポーネントをアップグレードする方法だ。

== {elemental-product-name} ノードアップグレード

{elemental-product-name} ノードは{elemental_operator_name} でアップグレードされる。詳細はxref:references/operatorchart-reference.adoc[{elemental_operator_name}]ドキュメントを参照してください。

アップグレードはCLIでもUIでも可能だ：

* <<_upgrade_via_command_line_interface,コマンドラインインターフェイス>>
* <<_upgrade_via_user_interface,ユーザーインターフェース>>

== コマンドラインインターフェイスによるアップグレード

アップグレードするノードを選択する方法は2つあります。クラスターターゲットでは、セレクタにマッチするクラスター内のすべてのノードにマッチし、ノードセレクタでは、ノードラベルに基づいてノードにマッチする。クラスター・セレクタはマッチしたクラスター内のすべてのノードを選択するのに対し、ノード・セレクタはアップグレードの対象を絞ることができる。

既存の`+ManagedOSImage+` を更新すると、新しいアップグレードサイクルが起動し、設定されたイメージ（またはイメージバージョン）が対象となるすべてのノードに照合されます。

=== 利用可能なバージョンの管理

xref:references/managedosversion-reference.adoc[ManagedOSVersionsを]直接作成することも、xref:references/managedosversionchannel-reference.adoc[ManagedOSVersionChannelsを]購読してそこから`+ManagedOSVersions+` を自動的に同期することもできる。

== ユーザーインターフェースによるアップグレード

UI からアップグレードするには、{elemental-product-name} Advanced メニューから`+Update Groups+` をクリックします。

名前を選択し、対象とするクラスタを選択し、2つのアップグレード方法から選択します：

image::upgrade-ui-menu.png[{elemental-product-name} アップグレードメニュー]

[tabs]
======
マネージドOSバージョン経由::
+
====
この場合、`+OS Versions+` リソースの自動入力に`+OS Version Channels+` が使われる。

下記のチャンネルは弊社が提供するものですが、ご自身のチャンネルをお持ちいただいても結構です。

`+ManagedOSVersion+` の管理方法については、<<_managing_available_versions,利用可能なバージョンの管理>>セクションを参照のこと。

image:upgrade-ui-version-channel.png[OSバージョンチャンネルの作成]

しばらく同期すると、`+OS Versions+` メニューに`+OS Versions+` が表示されます。

image:upgrade-ui-os-version.png[{elemental-product-name} OSバージョンメニュー]

最後に、以下のスクリーンショットに従って、`+Upgrade Group+` を作成する際に、`+OS Versions+` を選択することができます：

image:upgrade-ui-upgrade-group-channel.png[アップグレードグループでOSバージョンを選択]
====

レジストリから画像を経由してタブ::
+
====
アップグレード先のOCIイメージを`+Image path+` フィールドに指定するだけです：image:upgrade-ui-image-registry.png[イメージ・レジストリによるアップグレード]
====
======

複数のノードがある場合は、ノードごとに順次アップグレードが行われます。`+Create+` ボタンをクリックして、アップグレード処理を開始します。
