= マシン・リセット
:revdate: 2025-09-08
:page-revdate: {revdate}

{elemental-product-name} マシンを元の状態にリセットする方法と、廃止する方法がある：

. クラスタを削除すると、関連するすべてのマシンがリセットされます。
+
image::reset-cluster-deletion.png[クラスタを削除してすべてのマシンをリセットする]

. クラスタを管理する場合は、リセットが必要なノードを削除するだけです。
+
image::reset-single-node-deletion.png[ノードを1つ削除してリセットする]

== ワークフローのリセット

関連する`+MachineInventory+` に削除フラグが立てられると、マシン上で実行されている`+elemental-system-agent+` によってリセット計画が実行される。

マシンがまだ動いていれば、このプランは実行される：

. リカバリーモードでマシンを再起動する。
. 実行`+systemctl start elemental-register-reset+`. +
これでリモートの`+MachineRegistration+` 、`+spec.config.elemental.reset+` オプションを適用してマシンをリセットする。 +
新しい`+MachineInventory+` が作成され、`+MachineRegistration+` で定義された`+spec.config.cloud-config+` が再度適用されます。

`+MachineRegistration+` リファレンスは変更さ*れず*、マシンは再インストールさ*れず*、`+COS_PERSISTENT+` および`+COS_OEM+` パーティションは、リセットが`+enabled+` の場合、デフォルトでクリアされることに注意してください。詳しくはxref:installation/installation.adoc#_deployed_partition_table[パーティションテーブルを]参照してください。

`+cloud-config+` はリセット・ワークフロー中に再適用されるため、マシンをリセットして`+MachineRegistration+` 定義からの更新を適用することができる。たとえば、`+users+` 認証情報および認証キーをローテーションする。以前に設定したcloud-configとのコンフリクトを避けるため、`+reset-oem+` オプションを有効にすることを強くお勧めします。

マシンを別の`+MachineRegistration+` にバインドし、新しいフルインストールを起動する必要がある場合、新しいイメージを使用して再度ビジョニングする必要があります。

== マシンリセットを有効にする

マシンが自動的にリセットされるようにするには、`+MachineRegistration+` の`+spec.config.elemental.reset.enabled+` フラグを切り替える必要がある。 +
デフォルトではオフになっていますが、一度有効にすると、新しく作成されたすべての`+MachineInventory+` 、この設定が自動的に継承されます。 +
例えば、こうだ：

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: fire-nodes
  namespace: fleet-default
spec:
  config:
    elemental:
      reset:
        enabled: true
        reset-persistent: true
        reset-oem: true
        # These cloud-init configs will be created during reset and will persist on the system after
        config-urls:
          - "https://my.cloud.init/reset-plan-1.yaml"
          - "https://my.cloud.init/reset-plan-2.yaml"
        # You can select a different image to run the reset.
        # Note that this image will not be installed on the system.
        system-uri: "my.oci.registry/reset-image:latest"
        power-off: false
        reboot: true
----

例えば、一部のマシンが物理的で自動リセットの恩恵を受け、他のマシンが仮想的で必要に応じて単純に破棄して再ビジョニングできるようなシナリオでは、`+MachineInventory+` レベルでリセットを有効にすることも可能である。 +
単一の`+MachineInventory+` 、リセットを許可するフラグを立てるには、`+elemental.cattle.io/resettable: true+` アノテーションを使用します。 +
例えば、こうだ：

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineInventory
metadata:
  annotations:
    elemental.cattle.io/resettable: "true"
----
