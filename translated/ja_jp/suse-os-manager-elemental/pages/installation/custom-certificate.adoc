= カスタム証明書の追加方法
:revdate: 2024-09-24
:page-revdate: {revdate}

前提条件`+.pem+` 形式の証明書

ゴールだ：カスタム証明書をシステム全体で利用可能にする

[NOTE]
.これは、システム・レベル・サービスで使用される証明書用である。
====
Kubernetesワークロードは、代わりにコンテナ・イメージ内に証明書を持ち込むべきである。
====

カスタム証明書をインストールするには、以下が必要です。

* `+.pem+` ファイルを以下にコピーする。 `+/etc/pki/trust/anchors/+`
* 走る `+update-ca-certificates+`

それぞれの`+cloud-config+` スニペットは次のようになる：

[,yaml]
----
write_files:
  - path: /etc/pki/trust/anchors/my-custom-certificate.pem
    permission: 0444
    content: |-
      -----BEGIN CERTIFICATE-----
      ...
      -----END CERTIFICATE-----
runcmd:
  - update-ca-certificates
----

(簡潔にするため、実際の証明書の内容は省略されている）
