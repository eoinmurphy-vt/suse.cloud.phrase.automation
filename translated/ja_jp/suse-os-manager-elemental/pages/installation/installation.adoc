= インストール
:revdate: 2025-09-08
:page-revdate: {revdate}

== 概要

{elemental-product-name} スタックは、OCIコンテナとKubernetesを使用したOS管理を提供する。{elemental-product-name}スタックのインストールには、管理クラスタへの{elemental_operator_name} のインストールと、クラスタノードに OS を提供するためのインストールメディアの作成と使用が含まれます。各コンポーネントの相互作用については、「xref:rancher-os-management/architecture/architecture.adoc[アーキテクチャ]」のセクションを参照のこと。

インストール設定のほとんどは、登録プロセスの一部として適用され、設定されます。
登録プロセスは、Rancher管理クラスタにノードを登録し、インストール設定を取得する責任を負う`+elemental-register+` （{elemental_operator_name} クライアント部分）によって行われる。

簡単なステップバイステップの展開手順については、xref:quickstarts/quickstart-cli.adoc[クイックスタート]ガイドを参照してください。

== {elemental-product-name} オペレーターの設置

{elemental_operator_name} は、{elemental-product-name} バージョンを管理し、エッジまたはベアメタルのインストールを支援するためにマシンインベントリを維持する責任を負う。{elemental_operator_name}は、Rancher Manager を含むクラスタを必要とし、helm chart でインストールできる。

インストール、アンインストール、アップグレード、設定の詳細については、{elemental_operator_name} xref:references/operatorchart-reference.adoc[ヘルムチャートリファレンスを]参照してください。

== Kubernetesリソースを準備する

管理クラスター内で{elemental_operator_name} が稼働すると、{elemental-product-name} ベースのクラスター展開を準備するために、いくつかのkubernetesリソースが必要になる。

* xref:references/machineinventoryselectortemplate-reference.adoc[MachineInventorySelectorTemplate]:
このリソースは、登録されたボックス（MachineInventoryの一部としてリストされている）を利用可能なRancher 2.6クラスタと照合するための基準を特定します。一致するとすぐに、選択されたkubernetesクラスタが登録されたボックスの所有権を取得する。
* xref:references/machineregistration-reference.adoc[マシン登録]：
このリソースは、登録しようとするマシンのOS展開の詳細を定義する。マシン登録は{elemental-product-name} ノードの入り口であり、認証（TPM に基づく）、OS のデプロイ、MachineInventory へのノードの組み込みを処理するため、MachineInventorySelectorTemplate に基づいて一致した場合にクラスタに追加できる。MachineRegistrationオブジェクトには、ノードがそれに対して登録するために使用するマシン登録URLが含まれる。

{elemental-product-name} をデプロイするには Rancher Cluster リソースも必要で、xref:quickstarts/quickstart-cli.adoc[クイックスタート]ガイドに例示されているように手動で作成するか、Rancher 2.6 UI から作成することができる。

== インストールメディアの準備

インストールメディアは、OSのデプロイを開始するために使用されるメディアである。現在サポートされているメディアはライブISOです。ライブISOは登録設定yamlを含んでいなければならないので、MachineRegistrationが作成されると、それは作られなければならない。インストールメディアは、xref:references/seedimage-reference.adoc[シードイメージリソースを]作成することで生成されます（xref:quickstarts/quickstart-cli.adoc#_preparing_the_installation_seed_image[クイックスタートと] xref:operator-operational-tasks/custom-images.adoc#_create_a_custom_bootable_installation_iso[カスタムイメージを]参照）。

ライブISOは、https://documentation.suse.com/suma/4.3/en/suse-manager/client-configuration/autoinst-distributions.html#based-on-iso-image[SUSE Managerと]直接統合するためのPXEブートをサポートしています。

MachineRegistration内では、OSインストール・パラメーターのサブセットのみを構成することができます。利用可能なパラメーターはすべて、xref:references/machineregistration-reference.adoc[MachineRegistration]リファレンス・ページに記載されています。

で提供されている一般的なオプショ ンを超えてインストールを設定するには、 設定ファイルを ISO に含めることができます。xref:references/machineregistration-reference.adoc#_config_elemental_install[`+elemental.install+`]セクションで提供されている一般的なオプションを超えてインストールを設定するために、`+config.yaml+`設定ファイルを ISO に含めることができます (xref:installation/customize-installation/custom-install.adoc#_custom_suse_rancher_prime_os_manager_client_configuration_file[Custom Images] を参照)。
MachineRegistrationの`+elemental.install+` セクションの一部として適用される設定は、カスタム`+config.yaml+` ファイルに含まれる設定の上に適用されることに注意すること。

ほとんどの場合、起動時にデプロイされたノードの設定と設定を行うにはcloud-init設定で十分ですが、何らかの理由でファーストブート時のアクションやスクリプトが必要な場合は、Rancher System Agentのプランをインストールメディアに含めることも可能です。プランの詳細と例については、xref:rancher-os-management/plans.adoc[プランの]セクションを参照してください。`+/var/lib/elemental/agent/plans+`フォルダにある潰されたrootfsにプランを入れることができ、最初の起動時にシステムエージェントがそれを見ることができる。

== インストールの開始

インストールは、ノード上でインストール・メディアを起動することから始まる。インストール・メディアが起動すると、`+elemental-register+` コマンドを呼び出して管理クラスターにコンタクトし、登録しようとする。
登録yamlコンフィギュレーションはすでにISOに含まれているので、`+elemental-register+` 、登録URLと登録に必要なその他のデータを知っています。

登録が成功すると、インストールメディアは、すでにメディアに含まれている設定と MachineRegistration パラメータに基づいて、ホストへのインストールを開始します。インストールが終わると、ノードはすぐに再起動できる。デプロイされたOSには、選択したkubernetesバージョンをインストールするための通常のrancherプロビジョニングプロセスをキックスタートさせるシステムエージェントプランが含まれており、一度起動すると、数分後にノードのインストールが確定し、ノードがクラスタに含まれ、Rancher UIから見えるようになります。

== 展開されたパーティションテーブル

オペレーティング・システムがインストールされると、OSのパーティション・テーブルはデフォルト値に従って次のようになる。

|===
| ラベル | デフォルトサイズ | 内容

| COS_GRUB
| 64 MiB
| UEFIブートパーティション

| COS_STATE
| 8 GiB
| OCIイメージから構築されたA/Bブータブルファイルシステムイメージ

| COS_OEM
| 64 MiB
| OEMクラウド設定ファイルとその他のデータ

| COS_RECOVERY
| 4 GiB
| COS_STATEが破壊された場合のリカバリーファイルシステムイメージ

| COS_PERSISTENT
| 残りスペース
| 永続フォルダのすべての内容
|===

これは、{elemental_toolkit_url}[{elemental_toolkit_name}]によって構築されたOSの基本構造であることに注意されたい。
