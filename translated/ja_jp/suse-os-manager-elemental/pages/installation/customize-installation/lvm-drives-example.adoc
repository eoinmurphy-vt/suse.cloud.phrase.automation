= インストール中にLVMボリューム・グループ・ディスクを追加する方法
:revdate: 2025-09-08
:page-revdate: {revdate}

この例では、LVMセットアップの一部として使用される複数のディスクを持つホストのセットアップを取り上げる。

例として、3つのディスク（`+/dev/sda+` 、`+/dev/sdb+` 、`+/dev/sdc+` ）を持つホストがあるとする。

最初のディスクは通常のElementalインストールに使用され、残りの2つのディスクは、任意の論理ボリュームが作成され、フォーマットされ、拡張`+fstab+` ファイルを介してブート時にマウントされるLVMグループの一部として使用されます。

この例では、cloud-configのステップが2つの異なるステージで必要となる。まず、xref:references/cloud-config-reference.adoc#_elemental_client_cloud_config_hooks[インストール]中に LVM ボリュームを準備し、処理するために、いくつかのxref:references/cloud-config-reference.adoc#_elemental_client_cloud_config_hooks[インストー ルフックが]必要です。第二に、作成されたLVMボリュームが`+/etc/fstab+` 、結果的にマウントされるようにするために、ブート時にcloud-configが必要となる。

設置用フックは、`+SeedImage.spec.cloud-config+` ：

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: custom-seed
  namespace: fleet-default
spec:
  ...
  cloud-config:
    name: "Create LVM logic volumes over some physical disks"
    stages:
      post-install:
      - name: "Create physical volume, volume group and logical volumes"
        if: '[ -e "/dev/sdb" ] && [ -e "/dev/sdc" ]'
        commands:
        - | 
          # Create the physical volume, volume group and logical volumes
          pvcreate /dev/sdb /dev/sdc
          vgcreate elementalLVM /dev/sdb /dev/sdc
          lvcreate -L 8G -n elementalVol1 elementalLVM
          lvcreate -l 100%FREE -n elementalVol2 elementalLVM
          # Trigger udev detection
          if [ ! -e "/dev/elementalLVM/elementalVol1" ] || [ ! -e "/dev/elementalLVM/elementalVol2" ]; then
            sleep 10
            udevadm settle
          fi
          # Ensure devices are already available
          [ -e "/dev/elementalLVM/elementalVol1" ] || exit 1
          [ -e "/dev/elementalLVM/elementalVol2" ] || exit 1
          # Format logical volumes with a known label for later use in fstab
          mkfs.xfs -L eVol1 /dev/elementalLVM/elementalVol1
          mkfs.xfs -L eVol2 /dev/elementalLVM/elementalVol2
----

LVMデバイスが作成され、希望通りにフォーマットされる。このセットアップはインストール時に一度だけ必要だからだ。別の方法として、同じ動作を初回起動時に行うこともできますが、初回起動時に一度だけ適用されるようにするため、より高度なロジックが必要になります。最後に、ブート時のcloud-configデータには、マウントをトリガーするためのマウントポイント設定が含まれている。Elemental OS`+fstab+` ファイルはエフェメラルで、ブート時に動的に作成される。そのため、インストール中には存在せず、インストールフックで使用することもできない。

MachineRegistrationリソースの`+cloud-config+` セクションを使用して、`+/etc/fstab+` ファイルをカスタマイズする次の例を考えてみましょう：

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: my-nodes
  namespace: fleet-default
spec:
  ...
  config:
    ...
    cloud-config:
      stages:
        initramfs:
        - name: "Extend fstab to mount LVM logical volumes at boot"
          commands:
          - |
            echo "LABEL=eVol1 /run/elemental/eVol1  xfs defaults  0 0" >> /etc/fstab
            echo "LABEL=eVol2 /run/elemental/eVol2  xfs defaults  0 0" >> /etc/fstab
----

[NOTE]
====
`+initramfs+` xref:references/cloud-config-reference.adoc[ステージは]、実際のルートツリーに切り替える前の最後のステージである。この段階では、`+/etc/fstab+` ファイルがすでに存在しており、root を切り替える前に適合させることができる。最終的なルート・ツリーで実行されると、SystemDが残りの初期化を処理し、適用する。
====
