= {elemental-product-name} 空気を遮断した環境に設置する
:revdate: 2025-11-06
:page-revdate: {revdate}

== 前提条件

Rancherのエアギャップインストールは、https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/air-gapped/air-gapped.html[Rancherの公式ドキュメントに従って]すでに設定されている必要があります。
特に、エアギャップのあるインフラでは、プライベートレジストリを利用できるようにすべきである。

== 概要

エアギャップ環境で{elemental-product-name} ：

* {elemental-product-name} オペレーターチャート
* チャートで参照されているコンテナイメージ（_elemental-operator_イメージと_seedimage-builder_イメージ）。
* コンテナ化されたOSイメージ

さらに、利用可能なコンテナ化されたOSイメージを参照する_チャンネル・イメージを_作成するのも便利だろう。
公式チャンネルイメージ（_elemental-channel の_もの）は、公式 suse レジストリにある OS イメージの絶対 URL を参照しているため、エアギャップシナリオでそのまま使用することはできません。

== {elemental-product-name} コマンドラインからのAir-Gappedインストール

必要な手順はすべて、インターネットにアクセスできるホストからhttps://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[`+elemental-airgap.sh+` スクリプトを]実行することで達成できる。

{elemental-product-name} チャートはスクリプトの必須パラメータであり、ダウンロードしたアーカイブ、URL、または`+stable+` 、`+staging+` 、`+dev+` のいずれかのキーワードとして提供することで、スクリプトに正しいチャート・バージョンを取得させることができる。

`+elemental-airgap.sh+` は、{elemental-product-name} オペレータ・チャートを検査し、必要なコンテナ・イメージをすべて特定し、それらをダウンロードして単一の docker アーカイブに保存します。
また、引数として渡されたプライベートレジストリ（これも必須引数である）を指すOSイメージURLで、新しいOSチャンネルイメージを構築する。

エレメンタル・スクリプトの最新バージョンは、公式githubリポジトリから簡単にダウンロードできる：

[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh
chmod 755 elemental-airgap.sh
----

それでは、すべてのアーティファクトをダウンロードし、{elemental-product-name} の最新安定版からカスタム・チャンネルを構築してみよう：

[tabs]
====
Dockerアーカイブの作成::
+
======
[,shell]
----
./elemental-airgap.sh stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

スクリプトが完了すると（しばらく時間がかかるかもしれない）、以下のファイルがカレント・ディレクトリで利用可能になる：

* `+elemental-operator-crds-chart-<*VERSION*>.tgz+`
* `+elemental-operator-chart-<*VERSION*>.tgz+`
* `+elemental-images.txt+`
* `+elemental-images.tar.gz+`
======

運搬船アーカイブの作成::
+
======
[,shell]
----
./elemental-airgap.sh -ha stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

スクリプトが完了すると（しばらく時間がかかるかもしれない）、チャートとコンテナ・イメージの両方が`+elemental-haul.tar.zst+` という名前の運搬用アーカイブに格納される。
======
====

=== {elemental-product-name} インストール 

スクリプトによって作成されたファイルやアーカイブは、次のようなホストにコピーする必要があります：

* 個人登録にアクセスできる。
* kubectlバイナリがインストールされ、air-gapped Rancherクラスタにアクセスするように設定されている。
* ヘルムバイナリーを搭載。

[tabs]
========
Dockerアーカイブからインストールする::
+
======
プライベートレジストリに認証が必要な場合は、dockerでログインする必要がある：

[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

{elemental-product-name} のインストールには2つのステップが必要です：

* プライベートレジストリに必要なすべてのコンテナイメージを含むアーカイブをロードする。これは、Rancherのリリースで配布され、Rancherのエアギャップデプロイメントで既に使用されている`+rancher-load-images.sh+` スクリプトを使用して行うことができる：

[,shell]
----
rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

* ダウンロードしたエレメンタルチャートをインストールし、ローカルレジストリと新しく作成したチャネルを設定する：

[,shell]
----
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ 
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM>
----
======

Haulerアーカイブからのインストール::
+
======
https://rancherfederal.github.io/hauler-docs/[Hauler]archive (`+elemental-airgap.sh+` の`+-ha+` オプション) からインストールするには、Hauler のインストールも必要です。

プライベートレジストリに認証が必要な場合は、Haulerでログインする必要がある：

[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD
----

{elemental-product-name} のインストールには3つのステップが必要である：


* airgappedインフラのHaulerインスタンスに「elemental-haul.tar.zst」Haulアーカイブをロードする：

[,shell]
----
hauler store load 'elemental-haul.tar.zst' 
----

* エアギャップ環境のローカルレジストリがHaulerのサーバーでない場合は、ローカルレジストリにHaulアーカイブをロードします：

[,shell]
----
hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

[NOTE]
.搬器はレジストリーの役割も果たす
====
エアギャップされたローカルレジストリがHaulerインスタンスによって提供されている場合、（ステップ(1)に示すように）Haulアーカイブを直接そこにロードするだけで、ステップ(2)をスキップします。
====

* Haulerストアから元素表を取り出し、インストールする：

[,shell]
----
hauler store extract elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
hauler store extract elemental-operator-chart-<ELEMENTAL-VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<ELEMENTAL-VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ -
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM:PORT>
----
======
========

[NOTE]
.エレメンタルエアギャップスクリプトは、必要なコマンドを出力する。 
====
`+elemental-airgap.sh+` スクリプトは、上記の必要なコマンドを出力するが、実際のチャート・バージョンを使用し、提供されたレジストリ URL を使用して、正確なコマンドを簡単にコピー・アンド・ペーストできるようにする。
====

== {elemental-product-name} ランチャーマーケットからのエアギャップ設置 

ランチャーのエアギャップインストールには、{elemental-product-name} オペレーターチャート、オペレーターとシードイメージコンテナーイメージも含まれます。

不足しているOSイメージを収集し、プライベートレジストリ用のOSチャネルイメー ジを構築するには、インターネットにアクセスできるホストから、`+-co+` オプションを使用して、https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[`+elemental-airgap.sh+` スクリプトを]実行します。

例として、{elemental-product-name} の最新安定リリースの`+elemental-channel+` イメージをターゲットにしてみよう。このスクリプトは、（必要であれば）{elemental-product-name} オペレータ・チャートをダウンロードし、OS チャンネル・イメージの URL を抽出し、それをダウンロードし、参照される OS イメージをすべて検査し、それらをすべてダウンロードし、エアギャップ・シナリオのプライベート・レジストリへのリンクを持つ新しい OS チャンネルを作成します。

[tabs]
========
Dockerアーカイブの作成::
+
======
[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh 
chmod 755 elemental-airgap.sh 
./elemental-airgap.sh stable -co -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

スクリプトが完了すると（しばらく時間がかかるかもしれない）、以下のファイルがカレント・ディレクトリで利用可能になる：

- `+elemental-operator-crds-chart-<*VERSION*>.tgz+` 
- `+elemental-operator-chart-<*VERSION*>.tgz+` 
- `+elemental-images.txt+` 
- `+elemental-images.tar.gz+`
======

運搬船アーカイブの作成::
+
======
[,shell]
----
./elemental-airgap.sh -ha -co stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

スクリプトが完了すると（しばらく時間がかかるかもしれません）、コンテナ・イメージは`+elemental-haul.tar.zst+` という名前の運搬用アーカイブに梱包されます。
======
========

=== {elemental-product-name} インストール 

生成されたアーカイブは、エアギャップされたプライベートレジストリにロードされなければならない。

[tabs]
========
Dockerアーカイブからインストールする::
+
======
プライベートレジストリに認証が必要な場合は、dockerでログインする必要がある：
[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

このスクリプトは、Rancher`+rancher-load-images.sh+` ツールを使ってイメージをロードするために必要なコマンドを出力します。というようなものだろう：

[,shell]
----
NEXT STEPS: 

1) Load the 'elemental-images.tar.gz' to the local registry (<REGISTRY.YOURDOMAIN.COM:PORT>) available in the airgapped infrastructure: 

./rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

OSとチャネルイメージがロードされたら、スクリプト出力から(2)のポイント（ダウンロードしたアーカイブから{elemental-product-name} チャートをインストールする）をスキップし、代わりにRancher UIから{elemental-product-name} Operatorのインストールを実行する必要があります。
======

Haulerアーカイブからのインストール::
+
======
プライベートレジストリに認証が必要な場合は、Haulerでログインする必要がある：
[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD 
----

スクリプトは画像をロードするために必要なコマンドを出力します。というようなものだろう：

[,shell]
----
NEXT STEPS: 

* Load the 'elemental-haul.tar.zst' Haul archive in the Hauler instance in the airgapped infrastructure: 

hauler store load 'elemental-haul.tar.zst' 

* If the local registry in the air-gapped environment is not server by Hauler, load the Haul archive in the local registry: 

hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

OSとチャネルイメージがロードされたら、スクリプトの出力から(3)のポイント（ダウンロードしたアーカイブから{elemental-product-name} チャートをインストールする）をスキップし、代わりにRancher UIから{elemental-product-name} Operatorのインストールを実行する。
======
========

== {elemental-product-name} UIエクステンション 

Rancher 2.7.xはエアギャップ環境でのUI拡張プラグインをサポートしていないため、{elemental-product-name} UIはRancher 2.7.xでは利用できません。

{elemental-product-name} UIプラグインはRancher 2.8.0で利用可能なUIエクステンションに含まれます。
