= トラステッド・プラットフォーム・モジュール2.0（TPM）
:revdate: 2024-09-24
:page-revdate: {revdate}

トラステッド・プラットフォーム・モジュール（TPM、ISO/IEC 11889としても知られる）は、セキュアな暗号プロセッサーの国際規格であり、統合された暗号鍵によってハードウェアを保護するように設計された専用マイクロコントローラーである。また、この規格に準拠したチップを指すこともある。

== 仮想マシンにTPMモジュールを追加する

仮想マシンにTPMを追加する簡単な方法は、LibvirtとVirt-managerを使用することです。

=== 仮想マシンの作成

virt-managerの起動後、新しい仮想マシンを作成します。

image::tpm1.png[新規VMの作成]

=== ハードウェア・モジュール・リストの確認と編集

ハードウェア設定画面で、モジュールのリストを確認し *_ハードウェアの追加_*ボタンをクリックします。

image::tpm2.png[デバイス一覧]

=== VMにTPMモジュールを追加する

エミュレートされたデバイスのリストからTPMモジュールを選択し、VMに追加する。

image::tpm3.png[TPMモジュールの追加]

=== VMの設定を終了する

最後の画面で、TPMモジュールが正しく追加されているかもう一度確認する。

image::tpm4.png[TPMの検証]

== ベアメタルマシンにTPMエミュレーションを追加する

`+#!yaml MachineRegistration+` 、yamlに以下のキーを追加する。 `+config:elemental:registration:emulate-tpm: true+`

[IMPORTANT]
====
TPMエミュレーションを搭載したマシンを1台以上導入する場合は、`+config:elemental:registration:emulated-tpm-seed: -1+`、TPMエミュレーションに使用するシードがマシンごとにランダムになるように設定してください。そうでなければ、配置されたすべてのマシンで同じTPMハッシュが取得され、最後に登録されたものだけが有効になる。
====
