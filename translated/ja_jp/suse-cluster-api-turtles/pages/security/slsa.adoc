== プラットフォーム構築

* {product_name} プロジェクトでは、ソースコード管理にGitを使用している。
* すべての{product_name} メンテナーは二要素認証を有効にし、すべてのコントリビューションに署名し、サインオフすることが義務付けられている。
* {product_name} プロジェクトでは、すべてのリリース成果物のビルドに GitHub Actions と GitHub Runners を使用しています。
* ビルドとリリースのプロセスは、GitHubがホストするランナーによって提供されるエフェメラルな環境上で分離して実行される。

== ビルド・プロセス

* ビルドとリリースのプロセスはコードで定義され、バージョン管理下に置かれる。
* GitHubワークフローは、特定のバージョンにピン留めされたGitHub Actionsを利用し、GitHub Dependabotを使って最新の状態に保つ。
* ビルドとリリースプロセスに対するすべての変更は、少なくとも一人の{product_name} メンテナによって承認されなければならない Pull Requests によって行われます。
* リリースプロセスは、{product_name} メンテナがsemver形式のGitタグをプッシュすることでのみ開始できます。

== プロヴァンス

* {product_name} プロジェクトは、https://github.com/slsa-framework/slsa-github-generator[]SLSA公式プロジェクトを利用して、実績の生成と配布を行っています。
* GitHub Container RegistryとRancher Prime Registryに公開されたリリース成果物の実績は、https://github.com/slsa-framework/slsa-github-generator[]SLSAプロジェクトが提供するgenerator_container_slsa3 GitHub Workflowを使用して生成されます。
* 実績は、SHA-256 形式のダイジェストを使用して{product_name} コンテナ・イメージを識別する。
* 実績は、GitHub OIDC IDを使用してSigstore Cosignによって署名され、実績を検証するための公開鍵は、公開されているhttps://docs.sigstore.dev/logging/overview/[Rekorの透明性ログに]保存されます。
* リリースプロセスと実績生成は、GitHubがホストするランナーによって提供されるエフェメラルな環境上で分離して実行される。
* {product_name} コンテナイメージの出所は、公式のhttps://github.com/slsa-framework/slsa-verifier[検証ツールを使って] SLSA検証することができる。
* 実績生成ワークフローは、エフェメラルで隔離された仮想マシン上で実行され、それらはGitHubによって完全に管理されている。
* 証明書署名秘密はエフェメラルで、Sigstoreのhttps://github.com/sigstore/cosign/blob/main/KEYLESS.md[キーレス]署名手順によって生成される。
* https://github.com/slsa-framework/slsa-github-generator[]SLSAは、ビルドやリリースのプロセスとは別の仮想マシン上で実行されるため、{product_name} のビルドスクリプトが署名秘密にアクセスすることはありません。

== 孤立

* リリースプロセスと実績生成は、GitHubがホストするランナーによって提供されるエフェメラルな環境上で分離して実行される。
* https://github.com/slsa-framework/slsa-github-generator[]SLSAは、GitHub が完全に管理する別の仮想マシン上で実行されます。
* プロボナンス・ジェネレーターは、GitHubがホストする別のランナー上で分離して実行されるため、リリース・プロセスはプロボナンス署名キーにアクセスできない。
