= ClusterClassの有効化
:revdate: 2025-06-30
:page-revdate: {revdate}

このセクションではhttps://cluster-api.sigs.k8s.io/tasks/experimental-features/cluster-class/[*クラスタクラス*]機能の有効化について説明します。

[TIP]
====
ClusterClass機能を使用する前に、プロバイダのドキュメントを調べ、その機能がサポートされていることを確認してください。このドキュメントには、xref:../overview/certified.adoc#_clusterclass_support_for_certified_providers[クラスタ・トポロジーのサポート・ステータスが]記載されたxref:../overview/certified.adoc[認定プロバイダ・セクションの]マトリックスが含まれています。
====

== クラスタクラスを有効にする

ClusterClass機能を使用するには、コア・クラスタAPIとそれをサポートするプロバイダで有効にする必要があります。これは、`+CLUSTER_TOPOLOGY+` 変数を`+true+` に設定することによって行われる。

CAPI をインストールすると、{product_name} Helm チャートは自動的にこの機能を有効にします。ただし、追加プロバイダーを有効にする場合は、プロバイダー設定で`+CLUSTER_TOPOLOGY+` 。Turtlesのxref:../reference/capiprovider.adoc[CAPIProvider]リソースは、`+variables+` 、以下のようにインストール・パラメータをプロバイダに渡すことができる：

[source,yaml]
----
apiVersion: turtles-capi.cattle.io/v1alpha1
kind: CAPIProvider
metadata:
  name: azure
  namespace: capz-system
spec:
  type: infrastructure
  name: azure
  configSecret:
    name: azure-variables
  variables:
    CLUSTER_TOPOLOGY: "true"
    EXP_CLUSTER_RESOURCE_SET: "true"
    EXP_MACHINE_POOL: "true"
    EXP_AKS_RESOURCE_HEALTH: "true"
----

このYAMLファイルで定義されたリソースは、`+CLUSTER_TOPOLOGY+` を含む多くの機能をサポートするCAPZをインストールする。
