= クラスター・リソースの関係 {product_name}
:revdate: 2026-01-06
:page-revdate: {revdate}

== 概要

Rancherは、独自の`+Cluster+` カスタムリソースを定義するいくつかのコンポーネントを統合しています。{product_name} は、Cluster APIとFleetを使用してこの基盤の上に構築されます。これは、これらの`+Cluster+` リソースがどのように相互作用し、ユーザーが何を編集するべきかについて混乱を招く可能性があります。

このドキュメントでは、以下のクラスタリソースの目的、動作、および関係について説明します：

* `+clusters.cluster.x-k8s.io+` (CAPIクラスター)
* `+clusters.fleet.cattle.io+` (フリートクラスタ)
* `+clusters.provisioning.cattle.io+` (プロビジョニングクラスター）
* `+clusters.management.cattle.io+` (経営クラスタ）

== リソース概要

|===
| APIグループ | クラスターリソース | 役割

| `+cluster.x-k8s.io+`
| CAPIクラスター
| クラスタのライフサイクルとトポロジーを定義します。タートルズにおける真実の源として機能する。

| `+fleet.cattle.io+`
| フリートクラスター
| GitOpsバンドルターゲットとFleet経由のアプリケーション同期に使用される。

| `+provisioning.cattle.io+`
| プロビジョニング・クラスター
| Rancherの初期の汎用クラスタ抽象化機能。インフラストラクチャに依存しない構成とプロバイダ固有の構成の両方をカプセル化し、クラスタ登録トークンとマニフェストを生成するために必要です。{product_name} 、このリソースは輸入されたCAPIクラスターを包むために使われてきたが、段階的に廃止されつつある。

| `+management.cattle.io+`
| 経営クラスタ
| Rancherの現在のクラスタ表現。`+management.cattle.io/v3+` クラスタは現在、Turtlesによって作成および管理されるデフォルトのリソースです。これはCAPI-managedクラスタと混同してはならない。"managed "はクラスタAPIのライフサイクル管理を意味し、Rancherの`+management.cattle.io+` APIグループを意味しない。
|===

== 人間関係

=== CAPIクラスター → 艦隊クラスター

Cluster API Add-on Provider for Fleet (CAAPF) が有効な場合、CAPI`+Cluster+` から対応する Fleet`+Cluster+` にラベルとアノテーションを伝搬します。これにより、ラベル・セレクタを使ってGitOpsバンドルをターゲットにすることができます。CAAPFは、CAPI`+Cluster+` が`+ControlPlaneInitialized+` 条件に達した後、対応するフリート`+Cluster+` リソースを作成します。

=== CAPIクラスタ → プロビジョニングクラスタ（レガシー）

CAPIクラスタをラップし、ランチャー固有のフィールド（表示名、RKEコンフィグなど）を公開するために、`+provisioning.cattle.io.Cluster+` 。しかし、ほとんどの{product_name} ワークフローでは、このリソースは廃止されつつあり、ユーザーはこのリソースと対話する必要はない。

=== 管理クラスタ → プロビジョニング・クラスタ

レガシークラスターの場合、`+management.cattle.io/v3.Cluster+` から`+provisioning.cattle.io/v1.Cluster+` を生成することができる。このオブジェクトはRancherのUIとAPIで使用され、ユーザーが直接変更することは意図されていません。
タートルだけで管理される新しいクラスタは、`+provisioning.cattle.io+` クラスタを自動的に作成しません。

== 伝搬挙動

|===
| から → へ | 伝播 | 備考

| カピ → フリート
| はい
| CAAPFが推進

| CAPI → プロビジョニング
| いいえ
| 自動的に作成されるのではなく、レガシークラスタ用のTurtlesを介してのみ生成される。

| CAPI → マネジメント
| いいえ
| レガシークラスターの場合はプロビジョニングクラスター経由のみ

| プロビジョニング → フリート
| いいえ
| 非対応

| Fleet → CAPI
| いいえ
| 非対応

| 管理 → プロビジョニング
| 条件付き
| 新しいTurtles管理クラスタはプロビジョニングクラスタを生成しません。
|===

== ユーザーは何を編集すべきか？

=== おすすめだ： `+cluster.x-k8s.io.Cluster+`

CAPIクラスターは、{product_name} におけるクラスター定義のための権威あるリソースである。
ラベルはCAPIクラスタオブジェクトに貼らなければならない。これらは自動的にフリートクラスタに伝搬され、ラベルベースのバンドルセレクタが期待通りに動作するようになります。

=== 条件付きだ： `+provisioning.cattle.io.Cluster+`

Rancher ネイティブのプロビジョニング機能（`+spec.rkeConfig+` や UI ドリブンフローなど）を使用する場合にのみ必要です。明示的に要求されない限り、編集は避ける。

=== 避ける： `+fleet.cattle.io.Cluster+`

フリートクラスタは自動的に管理されます。フリート内部で作業しない限り、編集は推奨されない。

=== 避ける： `+management.cattle.io.Cluster+`

これらのリソースは内部リソースであり、Rancherによって完全に管理されている。修正しないでください。

[NOTE]
====
* **CAPIクラスターは**真実の源である。
* **フリートクラスタは**、CAAPFを介してCAPIクラスタから自動的に生成される。
* Rancherのプロビジョニングが含まれない限り、**プロビジョニングと** **管理クラスタは** *必要ない*。
====

[TIP]
====
CAPIクラスタのラベルを使用して、Fleetバンドルのデプロイを行います。CAAPFコントローラは、これらのラベルがフリートクラスターオブジェクトに同期されるようにします。
====
