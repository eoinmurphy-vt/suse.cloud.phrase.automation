== 前提条件

* https://kind.sigs.k8s.io/[種類]
* https://helm.sh/[helm]
* https://tilt.dev/[チルト]

== ローカル開発環境の構築

. ローカルにhttps://github.com/rancher/turtles[{product_name}]リポジトリをローカルにクローンする
. *tilt-settings.yamlを*作成する：
+
[source,yaml]
----
{
    "k8s_context": "k3d-rancher-test",
    "default_registry": "ghcr.io/turtles-dev",
    "debug": {
        "turtles": {
            "continue": true,
            "port": 40000,
            "insecure_skip_verify": "true"
        }
    }
}
----
+
. {product_name} リポジトリのルートでターミナルを開く。
. 以下を実行する：
+
[source,bash]
----
make dev-env

# Or if you want to use a custom hostname for Rancher and your ngrok credentials
NGROK_API_KEY=`+key+` NGROK_AUTHTOKEN=`+token+` RANCHER_HOSTNAME=my.customhost.dev make dev-env
----
+
. ngrokの認証情報を持っておらず、先のステップで提供しなかった場合は、新しいターミナルを開き、チルトが開始した後にngrokまたはインレットを起動することもできる：
+
[source,bash]
----
kubectl port-forward --namespace cattle-system svc/rancher 10000:443
ngrok http https://localhost:10000
----

== `+make dev-env+` 。

. https://kind.sigs.k8s.io/[]、以下のhttps://github.com/rancher/turtles/blob/main/scripts/kind-cluster-with-extramounts.yaml[] 。
. 以下の前提条件となるコンポーネントは、Helmを使用してインストールされます：
 ** 牧場経営者。
 ** サーティ・マネージャー
 ** ngrok Kubernetesオペレーター。
. Dockerイメージはローカルでビルドされ、Helmは以下のコントローラをクラスタにデプロイします：
 ** コアクラスタAPIコントローラ。
 ** RKE2 Bootstrap/Control Plane Provider コントローラ。
 ** Docker Infrastructure Providerコントローラ。
 ** フリートコントローラ用CAPIアドオンプロバイダ。
 ** {product_name} コントローラー
 ** {product_name} クラスタAPIオペレータコントローラ。
 ** {product_name} ClusterClassコントローラ。
 ** {product_name} 2日目のオペレーションコントローラー。
. 開発環境は`+tilt up+` を使って起動します。
