== 概要

トポロジーパースペクティブには、ITランドスケープのコンポーネントとその関係が表示されます。

image::k8s/k8s-topology-perspective.png[トポロジーの視点]

== 伝説

画面右下（ズームコントロールのすぐ下）にある凡例ボタン（？）をクリックすると、トポロジー視覚化で使用されているアイコンと色の説明が表示されます。

image::k8s/k8s-topology-perspective-legend.png[トポロジー・パースペクティブの凡例]

== コンポーネント

トポロジー パースペクティブには、選択したxref:/use/views/k8s-view-structure.adoc[ビューで]フィルタリングされたコンポーネントとリレーションが表示されます。1つ以上のxref:/use/alerting/k8s-monitors.adoc[モニターが]設定されているコンポーネントは、計算された健康状態を報告する。

* コンポーネントを選択すると、右パネルの詳細タブ -*コンポーネントの*詳細にコンポーネントの詳細情報が表示されます。
* コンポーネントにカーソルを合わせると、<<_component_context_menu,コンポーネントのコンテキストメニューが>>開きます。

=== コンポーネントのコンテキストメニュー

マウスポインタをコンポーネントに合わせると、コンポーネントのコンテキストメニューが表示されます。これは、そのコンポーネントに関する情報を提供する：

* コンポーネント名とタイプ
* 健康状態
* コンポーネント固有の<<_actions,アクション>>
* コンポーネント固有の<<_shortcuts,ショートカット>>

image::k8s/k8s-component-contex-menu.png[コンポーネントのコンテキストメニュー]

=== 行動

アクションを使用してトポロジーの選択を拡張し、選択したコンポーネントのすべての依存関係を表示することができます。その他のアクションは、StackPackの一部としてインストールされるコンポーネントアクションのように、特定のコンポーネントで利用できる場合があります。

使用可能な*アクションの*リストは、コンポーネント -*コンポーネントの詳細を*選択すると、右パネルの詳細タブに表示されます。アクションは、コンポーネントの上にマウスポインタを置くと表示される、コンポーネントのコンテキストメニューにも表示されます。

=== ショートカット

ショートカットを使えば、特定のコンポーネントの詳細情報に直接アクセスできます：

* *コンポーネント・ビューを開く*- このxref:use/views/k8s-metrics-perspective.adoc[コンポーネントのコンポーネント・メトリクスを]開きます。コンポーネント・ビューでは、表示されているコンポーネントのタイプに応じて、このコンポーネントとその直接の隣接コンポーネントに関するすべての重要事項を俯瞰することができます。
* *エクスプローラコンポーネント*- このコンポーネントのみを含むxref:/use/views/k8s-explore-views.adoc[エクスプローラビューを]開きます。エクスプローラービューでは、ビューフィルタを調整することなく、1つのコンポーネントをあらゆる視点から調査することができます。コンポーネントをダブルクリックしても同じ結果になる。
* *Show properties*- コンポーネントのプロパティポップアップを開きます。これは、*コンポーネントの*詳細情報が表示されているときに、右パネルの詳細タブで*「すべてのプロパティを表示*」をクリックするのと同じです。

== 関係

関係は、トポロジーのコンポーネントがどのように接続されているかを示す。破線または実線で表され、リンクするコンポーネント間の依存関係の方向を示す矢印がある。

* *直接関係*- ネットワークによって接続されたコンポーネント、およびその他のあらゆる形態の直接関係
* *間接関係*- ネットワークトラフィックやその他の依存関係に起因するコンポーネント間の関係。
* *グループ化された関係*- 直接関係と間接関係の組み合わせであり、コンポーネントグループ内の1つ以上のコンポーネントを接続する。

リレーションを選択すると、右パネルの詳細タブにそのリレーションの詳細情報が表示されます。

== ビジュアライゼーションの設定

トポロジー・パースペクティブでのコンポーネントとリレーションの表示方法は、ビジュアライザーの右上にあるビジュアライゼーション設定メニューでカスタマイズできます。 視覚化設定はビューと共に保存されます。

=== グリッドオプション

トポロジーパースペクティブのコンポーネントは、レイヤーを行、ドメインを列としてグループ化し、グリッドで構成することもできます。グリッドはデフォルトでオンになっている。グリッドオプションが選択されていない場合、グリッドは表示されません。

* *レイヤー別に整理*- 同じレイヤーのコンポーネントは、トポロジーの可視化で同じグリッド行に配置されます。グリッドから行を削除する場合は無効にしてください。
* *ドメイン別に整理*- 同じドメインのコンポーネントは、トポロジーの可視化で同じグリッド列に配置されます。グリッドから列を削除する場合は無効にしてください。

例えば、4つか5つの異なるソースから来るスタックのビジネス・サービスを視覚化する場合、そのソース（ドメイン）は視覚化においてそれほど重要ではない。これは、*ドメインによるオーガナイズを*オフにしてトポロジーを可視化するには良い状況かもしれない。

=== コンポーネントのグループ化

大量のコンポーネントのトポロジー可視化は、読みにくくなる可能性がある。SUSE Observabilityは、同じビュー内のコンポーネントをグループ化できます。グループ化することで、コンポーネントとリレーションの数を視覚的に管理しやすいものに減らすことができる。

グループ化はデフォルトで有効になっており、選択された<<_grid_options,グリッドオプションを>>尊重します - コンポーネントをグループ化するには、同じグリッド行/列にある必要があります。

3種類のグループ化が可能で、コンポーネントをグループ化しないこともできる：

* *グループ化されていない*- コンポーネントとリレーションはグループ化されていない。
* *自動グループ化*- グループ化の設定は、可視化されたコンポーネントまたはコンポーネントグループの数が35以下になるように自動的に調整されます。
* *状態およびタイプによるグループ化*- 同じタイプで同じ健康状態のコンポーネントは、1つのグループにまとめられます。グループ化されたコンポーネントの1つがヘルスステートを変更し、コンポーネントグループのヘルスステートに一致しなくなった場合、そのコンポーネントはグループからポップアウトする。同じタイプの他のコンポーネントが同じ健康状態を持つ場合、新しいグループが作成される。
* *タイプ、状態、関係によるグループ化*- コンポーネントは、*状態やタイプによるグループ*化と同様にグループ化される。グループ内のすべてのコンポーネントは、同じソースおよびターゲット接続を持つ。状態やタイプだけでコンポーネントをグループ化すると、グループ内のコンポーネントの関係に関する情報が失われる可能性があるため、これは有用である。

== ナビゲーション

=== ズームインとズームアウト

トポロジービジュアライザーの右下にズームボタンがあります。*プラス*ボタンはトポロジーを拡大し、*マイナス*ボタンは縮小する。この2つのボタンの間に、トポロジー全体が見えるようにズームアウトする「*画面に合わせる*」ボタンがある。

=== コンポーネントを探す

`+CTRL+` +`+SHIFT+` +`+F+` をクリックし、コンポーネント名の最初の数文字を入力すると、トポロジー内の特定のコンポーネントを見つけることができます。または、トポロジービジュアライザーの右下隅にある*Find component*虫眼鏡アイコンを選択することもできます。

xref:/use/stackstate-ui/k8sTs-keyboard-shortcuts.adoc[SUSE Observabilityキーボードショートカットの]完全なリストを参照してください。

=== 根本原因を示す

ビューの外側にあるが、ビュー内のコンポーネントに影響を与える可能性のあるモニター付きコンポーネントがある場合、トポロジーパースペクティブは、表示されているすべてのコンポーネントの健全性状態を表示します。

* *Don't show root cause（根本原因を表示しない*） - 現在のトポロジーフィルターで表示されるコンポーネントの根本原因を表示しない。
* *Show root cause only*- 現在のトポロジーフィルターによって表示されるコンポーネントのうち、`+CRITICAL+` または`+DEVIATING+` propagated health を持つコンポーネントの根本原因のみを表示する。あるコンポーネントが、根本原因につながる少なくとも1つの不可視のコンポーネントに直接依存している場合、間接的な関係が可視化される。
* *Show full root cause tree（完全な根本原因ツリーを表示する*） - 現在のトポロジーフィルターによって表示されるコンポーネントのうち、`+CRITICAL+` または`+DEVIATING+` 伝播された健全性を持つコンポーネントから、その根本原因までのすべてのパスを表示します。

image::k8s/k8s-show-root-cause.png[根本原因]

== リストモード

トポロジー可視化のコンポーネントは、グラフの代わりにリストで表示することもできる：

image::k8s/k8s-topology-perspective-list-mode.png[フィルタリング(リスト形式)]

=== CSVエクスポート

リストモードでは、コンポーネントリストをCSVファイルとしてエクスポートできます。CSVファイルには、`+name+` 、`+state+` 、`+type+` 、`+updated+` 、ビュー内の各コンポーネントの詳細が含まれています。

. トポロジーのパースペクティブで、右上の*リストモードアイコンを*クリックして、トポロジーをリストモードで開きます。
. ページ上部の*「CSVでダウンロード*」をクリックします。
 ** コンポーネントリストは、`+<view_name>.csv+` という名前の CSV ファイルとしてダウンロードされます。
