== ログに移動する

ログシッピングが設定されているKubernetes環境のコンテナログは、SUSE Observability UIから調べることができます。
これは、左側のKubernetesメニューから、提供されているパラダイム（サービス、デプロイメント、ステートフルセット、デーモンセットなど）のいずれかを通じて、`+pod+` レベルまでドリルダウンすることで行うことができる。

image::k8s/k8s-menu.png[Kubernetesパラダイムメニュー]

簡略化と完全性のため、この例では`+Pods+` パラダイムを直接使用し、ポッドログとコンテナログを区別できるように、複数のコンテナを含むポッドを選択した。 ポッドビューには複数のログ関連のエントリーポイントがある。

== ログの表示

image::k8s/k8s-pod-view-node-agent.png[Kubernetesポッドビュー]

このポッドに関連するすべてのログを表示するには、赤枠で囲んだ「すべて表示」をクリックします。 ログエントリーの種類（エラー、警告、その他）でフィルタするには、青で囲んだ項目のいずれかを選択します。 ログエントリーのタイプ間の比率を示すヒストグラムは、緑色の丸で囲まれている。

=== 丸太の引き出し

ログドロワーは、トラブルシューティングの経験を向上させる方法で、ログファイルを視覚的に楽しくナビゲートします。

簡単のため、この例では、フィルタリングされていない選択のみを扱い、すべてのコンテナ、すべてのログエントリタイプのすべてのログ行を表示する。 `+View all+` を選択すると、フィルタリングが適用されていないログドロワーが開く：

image::k8s/k8s-pod-view-log-drawer-no-filter.png[Kubernetesログビューのフィルタなし]

この時点から、提供されたフィルター（検索文字列の一致、重大度、コンテナ、ログウィンドウの間隔）にオプションを適用することで、問題を掘り下げていくことができる。 ログドロワーのヒストグラムの直下に、並べ替えや表示の優先順位を設定するためのいくつかのオプションが用意されています。

* 検索文字列の一致は、入力された検索語を受け取り、それを含まないログ行をフィルタリングする。
* 深刻度オプションは`+Any+` 、`+Error+` 、`+Warning+` 、`+Other+` 。`+Any+` 以外を選択すると、選択範囲に一致しないすべてのログ行が除外されます。
* コンテナのオプションには、`+Any+` 、Pod内のコンテナのリストが含まれます。 `+Any+` 以外を選択すると、出力は選択したコンテナから出力されるものに限定される。
* ログウィンドウインターバルオプションは、`+from+` 、`+to+` 間隔を指定できる日付-時間ピッカーインターフェースとなる。 これをデフォルトから変更すると、選択した期間外のすべてのログエントリが除外される。

一例として、この4つすべての動きを以下で見ることができる：

image::k8s/k8s-pod-view-log-drawer-with-filters.png[フィルタによるKubernetesログビュー]

上の画像には以下のフィルターが適用されている：

* 検索用語： `+wrong type in json response+`
* 深刻さ： `+Warning+`
* コンテナ： `+node-agent+`
* ログウィンドウの間隔：現在の`+02:00am+` から`+03:00am+` 。
