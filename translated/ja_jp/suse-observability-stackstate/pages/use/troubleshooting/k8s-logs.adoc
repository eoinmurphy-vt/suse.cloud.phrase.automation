== 概要

Kubernetesは、クラスタのアクティビティとアプリケーションの出力に関する包括的なログを保持している。ログは、コンテナ、ノード、Kubernetes自体のパフォーマンスに関する貴重な洞察を提供する。PodからReplicaSetに至るまで、すべてのログを綿密に記録することで、問題をその発生源まで遡ることができます。

== ポッドログへのアクセス

ログのエラーや警告は、アプリケーション、コンテナ、またはポッド自体に潜在的な問題があることを示す指標です。ポッド画面でログを表示するには、*「ログを表示*」ボタンをクリックします。

== ポッドログの使用

検索を使って、ログの中から特定のエラーや警告を探すことができる。ログはコンテナでフィルタリングできる。ログを表示する時間間隔と、ログを更新する頻度の更新間隔を選択できます。

ログのヒストグラムは、選択した時間間隔内のログの行数を視覚化し、クリックしてドラッグすることにより、特定の間隔にズームすることができます。

これらのオプションを使用して、各ログ行の追加情報を表示または非表示にします：

* タイムスタンプを表示する*時間を表示する*
* ログ行が属するコンテナ名を表示する*コンテナ表示*
* 長い*行の*折り返しを有効にする。
* ログのソート順を入れ替える「*新しい順」「古い順*
