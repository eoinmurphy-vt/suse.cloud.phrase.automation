= 通知のトラブルシューティング
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

通知チャネルが通知の送信に失敗すると、通知の概要にエラーステータスが表示されます。エラーを検査する：

. 通知をクリックする
. ページ上部には、エラーの概要を示すバナーが表示される。
. チャンネルまでスクロールダウンし、該当チャンネルの「エラー」リンクをクリックすると、エラーの詳細が表示されます。

== よくあるエラー

=== スラック

Slackのエラーには通常、エラーコードが含まれている：

* `+token_revoked+``+token_expired+` または`+missing_scope+` ：これらのエラーコードはすべて、SUSE ObservabilityがSlackにメッセージを投稿するための認証に問題があることを示しています。これらを解決するには、チャネルを再作成して、必要なすべてのパーミッションが付与されるように、SlackでSUSE Observabilityを再オーソライズします。
* その他のエラーコードについては、https://api.slack.com/automation/cli/errors[Slack API エラーコードを]参照してください。

=== HTTP

ほとんどのチャンネル（特にウェブフック）は、HTTPレスポンスのステータス200 OKを期待している。それ以外の回答はエラーとみなされる。HTTPエラーには、200 OKの代わりに、通常は短いメッセージと受信したステータスコードが含まれる。

設定されたURLとトークンが正しいこと、ウェブフックの場合はPOSTリクエストが適切に処理され、200 OKレスポンスを返すことを確認する。

=== その他

エラーのタイプが "Other "の場合、SUSE Observabilityは外部サービスまたはWebhookに接続できない可能性が高いです。URLが正しく、解決でき、SUSE Observabilityからアクセスできることを確認します。
