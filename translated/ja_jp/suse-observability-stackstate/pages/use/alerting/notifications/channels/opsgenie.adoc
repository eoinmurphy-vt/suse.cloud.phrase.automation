== Opsgenie通知の設定

Opsgenieに通知を送信するには、以下の手順に従ってください：

. <<_create_integration,OpsGenieで統合を作成する>>
. <<_create_and_test_the_channel,チャンネルの作成>>

=== 統合

SUSE ObservabilityとOpsGenieを統合するには、グローバルな「APIキー」を作成する必要があります。 通知のレスポンダは通知ごとに異なり、SUSE Observabilityで設定されます。

Opsgenieの「設定」→「統合」に進みます。

* 統合を追加 "ボタンをクリック
* API "統合タイプを検索する。
* この統合の名前を選択しますが、_担当者チームは選択しないで_ください。

統合が作成されると、"APIキー "が設定されます。

=== チャンネルの作成とテスト

image::k8s/notifications-opsgenie-channel-configuration.png[チャンネルの作成]

OpsGenieアカウントの適切な地域を選択し、「APIキー」を入力します。 _ユーザー_、_チーム_、_スケジュール_、_エスカレーションが_利用できます。

チャンネルを追加」ボタンをクリックします。これでチャンネルが右側のチャンネルリストに追加される。テスト」ボタンが表示されます。これを押すと、OpsGenieでテスト・アラートが生成される。

== OpsGenieアラート通知

通知が開かれると、新しいOpsGenieアラートが作成されます。このメッセージは、通常、健康状態が変化したときのみ、変化に応じて更新される。通知が閉じられると、OpsGenieアラートは再び閉じられます。 アラートには、そのアラートを閉じた理由が記されている。
