= タイムラインとタイムトラベル
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

== 概要

SUSE Observability UIの下部にあるタイムラインでは、過去の特定の時点のトポロジの状態にタイムスリップできます。選択したトポロジースナップショットで利用可能なすべての遠隔測定をナビゲートできます。タイムラインのヘルスチャートとイベントチャートは、選択されたテレメトリーインターバル中のトポロジーの状態の概要を示します。

image::k8s/k8s-timeline.png[タイムライン]

== タイムライン

=== テレメトリ間隔

テレメトリ間隔は、SUSE Observabilityパースペクティブでイベント、メトリクス、トレースを利用できる時間ウィンドウを指定します。タイムラインは左から右へ流れる。

image::v51_telemetry_interval.png[テレメトリ間隔]

選択したテレメトリ間隔は、相対（ライブ・モード）、またはカスタム・テレメトリ間隔（タイム・トラベル・モード）に設定できます。デフォルトでは、遠隔測定間隔は相対遠隔測定間隔に設定されています - ライブモードでは、過去1時間の遠隔測定が表示されます。特定の時点からの遠隔測定を表示するために、ズームイン/ズームアウト、またはカスタム遠隔測定間隔を設定することができます。

==== 遠隔測定間隔の設定

[NOTE]
====

* 遠隔測定間隔は最大6ヶ月。
* テレメトリ間隔にカスタムテレメトリ間隔を設定すると、SUSE Observabilityは<<_topology_time,トポロジ時間を>>一時停止し、<<_time_travel,タイムトラベルモードに>>なります。
====


テレメトリー間隔は以下の方法で設定できる：

* *ズームイン*

image::v51_timeline_click_drag.png[タイムライン上をクリック＆ドラッグして、選択範囲にカスタムの遠隔測定間隔を設定する。]

* *ズームアウト*

image::v51_telemetry_interval_zoom_out.png[虫眼鏡をクリックして、テレメトリーインターバルのサイズを2倍にする。]

* *テレメトリーインターバルのジャンパー矢印を使用します。*

image::v51_telemetry_interval_jumper.png[時間ジャンパの矢印をクリックすると、テレメトリの間隔を時間軸で前後に移動できます。]

* *相対またはカスタム・テレメトリー間隔を設定する*

image::v51_timeline_telemetry_interval.png[ポップアップの "Set telemetry interval "を使用して、テレメトリ間隔を指定します。]

=== トポロジー時間

SUSE Observabilityのトポロジーは、その時点で観測された環境のスナップショットに基づいています。このスナップショットが取得される瞬間は、トポロジー時間によって指定される。デフォルトでは、SUSE Observabilityはライブモードで、トポロジ時刻は現在の時刻に設定されています。カスタムトポロジー時間を選択することで、トポロジーの以前の状態に<<_time_travel,タイムトラベル>>できる。これは、ある時点での問題を調査するのに役立つ。ポッドがなくなって久しい場合でも、そのポッドがどのように接続されていたか、そのログ、イベント、関連リソース、イベントなどを見ることができる。

image::v51_topology_time.png[トポロジー時間]

タイムライン上では、選択されたトポロジーの時間は、再生ヘッドによって示される。また、タイムラインの上部にある*Topology time*ボックスでも指定される。

==== トポロジー時間の設定

トポロジーの時間は以下の方法で設定できる：

* *タイムラインをクリック*

image::v51_topology_time_timeline.png[タイムライン上のどこかをクリックして、トポロジーの時間をその瞬間に設定する。]

* *トポロジータイムジャンパー矢印を使用*

image::v51_topology_time_jumper.png[トポロジータイムジャンパー矢印をクリックして、トポロジータイムを次のイベントセットまで前後に移動する。]

* *カスタム・トポロジーの時間を設定する*

image::v51_topology_time_popup.png[ポップアップ「トポロジー時間の設定」を使用して、トポロジー時間を指定する。]

* *タイムスタンプをクリック*

image::v51_topology_time_timestamp.png[タイムスタンプをクリックすると、そのトポロジーの時間にジャンプします。]

==== テレメトリ間隔外のトポロジ時間

選択されたトポロジー時間が、現在選択されている<<_telemetry_interval,テレメトリーインターバル>>外の時間である場合、「トポロジー時間は現在のテレメトリーインターバル外です」というメッセージが表示され、タイムライン上部の*トポロジー時間*ボックスが黒くハイライトされます。タイムラインはテレメトリ間隔を左から右に表示するため、現在のトポロジー時間を示す再生ヘッドはタイムライン上に表示されない。

image::v51_topology_time_outside_telemetry_interval.png[テレメトリ間隔外のトポロジ時間]

トポロジーやテレメトリーのブラウズは従来通り可能だ：

* xref:/use/views/k8s-topology-perspective.adoc[トポロジー・パースペクティブでは]、選択したトポロジー時間におけるトポロジーの状態が視覚化される。
* すべてのパースペクティブで、選択したテレメトリ間隔で生成され、選択したトポロジ時間に存在したトポロジ要素に関連するテレメトリが表示される。

==== ライブモード

タイムトラベルを止め、トポロジーの時間をライブモードに戻すには、画面上部の*Go live*または*BACK TO LIVEを*クリックする。

image::v51_timeline_go_live.png[ライブ]

=== 健康

選択されたテレメトリーインターバル中のビューの健康状態は、タイムラインの*ヘルス*ラインに色で表示されます。

単一のリソースの場合、健全性は経時的に表示され、概要では灰色の線が表示される。

image::v51_timeline_no_health_state.png[健康状態は不明]

=== イベント

タイムラインの*イベント*ラインには、各時点で発生したイベントの数が棒グラフで表示される。これは、過去に多くの活動があった瞬間を見るのに役立つ。選択した<<_topology_time,トポロジー>>時間に存在したトポロジー要素によって生成されたイベントのみが表示されることに注意。

興味のあるイベント・バーをズームするには、クリックしてドラッグし、タイムライン上でその周りの小さいテレメトリ間隔を選択します。

image::v51_timeline_click_drag.png[クリック＆ドラッグでテレメトリ間隔を選択]

[NOTE]
====
タイムラインをシングルクリックすると、再生ヘッドがこの時点に移動し、選択された<<_topology_time,トポロジーの時間における>>トポロジーの状態にタイムトラベルする。新しく選択されたトポロジー時間に存在したトポロジー要素によって生成されたイベントのみが表示されるようになりました。
====


== タイムトラベル

SUSE Observabilityの各視点では、ライブモードまたは過去のモードのいずれかになります。ライブモードでは、SUSE Observabilityは常に新しいデータをポーリングします。トポロジーやテレメトリでタイムトラベルする場合、事実上、インフラのスナップショットで作業することになる。利用可能なデータは2つの選択に基づいている：

* <<_topology_time,トポロジー時間>>- Kubernetesリソースのスナップショットを取得したい特定の瞬間。
* <<_telemetry_interval,テレメトリ間隔>>- テレメトリとトレースを表示する時間範囲。

タイムトラベルを止めてライブモードに戻るには、画面上部の*Go live*または*BACK TO LIVEを*クリックする。

image::v51_timeline_go_live.png[ライブ]
