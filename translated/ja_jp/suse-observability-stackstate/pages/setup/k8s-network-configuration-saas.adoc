= ネットワーク構成
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

SUSE Observabilityは、クラウドでホストされるSaaS型サービスだ。構内/クラウドからSUSE Observability SaaSに通信するには、SUSE Observability AgentがSUSE Observability SaaS Receiver APIに接続できる必要があります。
クラスタがプライベートネットワークで実行されている場合、ネットワーク設定がインターネットへのイグジットトラフィックを許可しない可能性があるため、SUSE Observability AgentがSUSE Observability Receiver APIに接続できるようにネットワークを設定する必要がある場合があります。このページでは、SUSE Observability Agentをインストールできるようにネットワークを設定する方法と、SUSE Observability AgentがSUSE Observability Receiver APIと通信できるようにする方法について説明します。

[NOTE]
====
SUSE Observability AgentとSUSE Observability Receiver API間のトラフィックは、常にSUSE Observability Agentによって開始されます。SUSE Observability Receiver APIは、SUSE Observability Agentへのトラフィックを開始しません。
====


== SUSE Observability Agentのインストール

SUSE Observability Agentのインストールは、Helmを介して行います。デフォルトでは、Helm Chart は Quay.io docker レジストリから SUSE Observability Agent コンテナイメージを取得するように設定されています。ネットワーク設定でインターネットへのイグジットトラフィックを禁止している場合、SUSE Observability Agentをインストールするためのオプションがいくつかあります：

. KubernetesクラスタからQuay.ioコンテナレジストリへのイグジットトラフィックを許可するようにネットワークを設定します。
. Quay.ioのコンテナレジストリを自分のコンテナレジストリにプロキシする。
. Dockerイメージを自分のコンテナレジストリに取り込む。

オプション2と3では、独自のコンテナレジストリからSUSE Observability AgentコンテナイメージをプルするようにHelmチャートを設定する必要があります。独自のコンテナレジストリからイメージをプルするためのSUSE Observability Agent Helm Chartの設定ガイドは、xref:/k8s-suse-rancher-prime-agent-air-gapped.adoc[こちらを]参照してください。

== SUSE Observabilityエージェントの通信

SUSE Observability Agentは、SUSE Observability Receiver APIとHTTPSで通信します。SUSE Observability Agentの各部は、テナント内でホストされているSUSE Observability Receiver APIに接続します：

image::k8s/k8s-agent-communication.png[SUSE Observabilityエージェントの通信]

通信はすべてHTTPSで行われ、標準的なHTTPSポート443を使用します。SUSE Observability Agentは、以下のエンドポイントを使用してSUSE Observability Receiver APIと通信します：

* https://*<tenant>.app.stackstate.io/receiver/stsAgent*- SUSE Observability Agentは、メトリクス、イベント、トポロジデータをSUSE Observability Receiver APIに送信します。

SUSE Observability AgentがSUSE Observability Receiver APIと通信できるようにするには、SUSE Observability Receiver APIへのイグジットトラフィックを許可するようにネットワークを設定する必要があります。SUSE Observability Receiver APIはクラウドでホストされ、テナント固有のIPを持ちます。インターネットへのイグレス・トラフィックを許可する必要がある。egressトラフィックを許可する正しいIPアドレスを取得するには、以下のコマンドを使用します：

[,bash]
----
$ dig +short <tenant>.app.stackstate.io
----

または、ブラウザで以下のURLにアクセスしてください： `+https://www.nslookup.io/domains/<tenant>.app.stackstate.io/dns-records/+`
