= ファイルベース
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability セルフホスト

== 概要

外部認証プロバイダを使用できない場合は、ファイルベース認証を使用できます。これにより、すべてのSUSE Observabilityユーザが設定ファイルで事前に設定されている必要があります。設定内のユーザーに変更が加えられるたびに、SUSE ObservabilityはHelmで変更を適用した後に自動的に再起動します。

SUSE Observabilityにはデフォルトのロールが多数含まれています。以下の設定例を参照してください。各デフォルトのロールに割り当てられた権限と、他のロールを作成する方法については、xref:/setup/security/rbac/README.adoc[ロールベースのアクセス制御（RBAC）のドキュメントに]記載されています。

== ファイルベース認証の設定

=== Kubernetes

Kubernetesでファイルベース認証を設定するには、SUSE Observabilityユーザを`+authentication.yaml+` ファイルに追加する必要があります。例えば、こうだ：

[tabs]
====
authentication.yaml::
+
--

[,yaml]
----

stackstate:
  authentication:
    file:
      logins:
        - username: admin
          passwordHash: 5f4dcc3b5aa765d61d8327deb882cf99
          roles: [ stackstate-admin ]
        - username: guest
          passwordHash: 5f4dcc3b5aa765d61d8327deb882cf99
          roles: [ stackstate-guest ]
        - username: poweruser
          passwordHash: 5f4dcc3b5aa765d61d8327deb882cf99
          roles: [ stackstate-power-user ]
        - username: troubleshooter
          passwordHash: 5f4dcc3b5aa765d61d8327deb882cf99
          roles: [ stackstate-k8s-troubleshooter ]
----

--
====

以下の手順に従って、ユーザーを設定し、変更を適用してください：

. `+authentication.yaml+` - ユーザーを追加する。各ユーザーに以下のコンフィギュレーションを追加する必要がある（上の例を参照）：
 ** *username*- SUSE Observabilityへのログインに使用するユーザ名。使用できる文字は英数字と _ のみです。
 ** *passwordHash*- SUSE Observabilityへのログインに使用するパスワード。パスワードはbcryptハッシュとして保存される。
 ** *roles*- ユーザが所属するロールのリスト。xref:/setup/security/rbac/rbac_permissions.adoc#_predefined_roles[デフォルトのSUSE Observabilityロールは]、`+stackstate-admin+` 、`+stackstate-power-user+` 、`+stackstate-guest+` 。その他のロールの作成方法の詳細については、xref:/setup/security/rbac/rbac_roles.adoc[RBACロールを]参照してください。
. ファイル`+authentication.yaml+` は、SUSE Observabilityのインストール手順にあるファイル`+values.yaml+` と一緒に保存してください。
. Helmアップグレードを実行して変更を適用します：
+
[,text]
----
 helm upgrade \
   --install \
   --namespace suse-observability \
   --values values.yaml \
   --values authentication.yaml \
 suse-observability \
 suse-observability/suse-observability
----

[NOTE]
====
*注：*

* bcryptパスワード・ハッシュは、以下のコマンドライン`+htpasswd -bnBC 10 "" <password> | tr -d ':\n'+` 、またはオンライン・ツールを使って生成できる。
* helm upgrade コマンドを最初に実行するとポッドが再起動し、可用性が短時間中断する可能性があります。
* `+helm upgrade+` 実行するたびに`+authentication.yaml+` を含める。
* 認証設定はKubernetesシークレットとして保存される。
====


以下の手順に従って、ユーザーを設定し、変更を適用してください：

. `+authentication.yaml+` - ユーザーを追加する。各ユーザーに以下のコンフィギュレーションを追加する必要がある（上の例を参照）：
 ** *username*- SUSE Observabilityへのログインに使用するユーザ名。使用できる文字は英数字と _ のみです。
 ** *password*- SUSE Observabilityへのログインに使用するパスワード。パスワードはMD5ハッシュまたはbcryptハッシュとして保存される。
 ** *roles*- ユーザが所属するロールのリスト。xref:/setup/security/rbac/rbac_permissions.adoc#_predefined_roles[デフォルトのSUSE Observabilityロールは]、`+stackstate-admin+` 、`+stackstate-power-user+` 、`+stackstate-k8s-troubleshooter+` 、`+stackstate-guest+` 。その他のロールの作成方法の詳細については、xref:/setup/security/rbac/rbac_roles.adoc[RBACロールを]参照してください。
. SUSE Observabilityを再起動して、変更を適用します。

[NOTE]
====
*注：*

* MD5パスワードハッシュは、LinuxやMacの`+md5sum+` または`+md5+` コマンドラインアプリケーションを使って生成することができる。
* bcryptパスワード・ハッシュは、以下のコマンドライン`+htpasswd -bnBC 10 "" <password> | tr -d ':\n'+` 、またはオンライン・ツールを使って生成できる。
====


=== 外部秘密の使用

ユーザー・パスワードが外部のシークレットに由来するものである場合は、xref:/setup/security/external-secrets.adoc#_getting_authentication_data_from_an_external_secret[以下の手順に]従いますが、以下のデータを記入してください：

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
  file_<username1>_password: <base64 of bcrypt of password>
  file_<username2>_password: <base64 of bcrypt of password>
----

ログインセクションのすべてのユーザーについて、テンプレートに記入されたレコードをシークレットに追加する必要があります。例えば、こうだ：

[,yaml]
----

stackstate:
  authentication:
    file:
      logins:
        - username: admin_user
          roles: [ stackstate-admin ]


kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
   file_admin_user_password: "base64EncryptedPass"
----
