= 認証と認可のトラブルシューティング
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability セルフホスト

認証または認可に失敗する場合、通常はプロバイダとSUSE Observabilityの設定の不一致が原因です。トラブルシューティングを容易にするために、SUSE Observabilityで認証と認可に特化したデバッグロギングを有効にすることができます。

[CAUTION]
====
トラブルシューティングが終わり次第、デバッグログを再度無効にしてください。デバッグログに機密情報や個人情報が含まれている可能性が非常に高いからです。
====


デバッグ・ロギングを有効にするには、以下のyamlスニペットを`+debug-auth.yaml+` 。

[,yaml]
----
stackstate:
  components:
    server:
      additionalLogging: |
        <logger name="org.pac4j.core.engine" level="DEBUG"/>
        <logger name="org.pac4j.oidc.profile.creator" level="DEBUG"/>
        <logger name="org.pac4j.oidc.credentials.authenticator" level="DEBUG"/>
    api:
      additionalLogging: |
        <logger name="org.pac4j.core.engine" level="DEBUG"/>
        <logger name="org.pac4j.oidc.profile.creator" level="DEBUG"/>
        <logger name="org.pac4j.oidc.credentials.authenticator" level="DEBUG"/>
----

次に、先ほど使用した`+helm upgrade+` コマンドを実行するが、デバッグ・ロギングを有効にするために、追加のyamlファイル（`+helm upgrade .... --values debug-auth.yaml+` ）を1つ追加する。ポッドは再起動せず、ロギング設定の変更は約30秒後に自動的にロードされます。

デバッグ・ロギングを無効にするには、`+helm upgrade ....+` コマンドを再度実行するが、`+--values debug-auth.yaml+` は省略する。30秒後、更新されたロギング設定がロードされ、デバッグロギングが停止する。
