= 電子メール通知のためのSUSE Observabilityの設定
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability セルフホスト

[CAUTION]
====
SUSE ObservabilityのSaaSユーザーは、追加設定なしで電子メール通知を使用できます。このガイドは、電子メール通知チャネルを使用するセルフホストSUSE Observabilityインストールにのみ適用されます。
====


SUSE Observabilityで電子メール通知チャネルを使用する前に、まず次の手順に従う必要があります：

== SMTP設定でSUSE Observabilityを設定する

SUSE Observabilityは、SMTPサーバに接続するための認証情報を設定する必要があります。これを行うには、SUSE Observabilityインストールの`+values.yaml+` ：

[,yaml]
----
stackstate:
  email:
    enabled: true
    sender: "<stackstate@example.com>"
    server:
      host: "<smtp.example.com>"
      auth:
        username: "<user name>"
        password: "<user password>"
----

これはSMTPサーバーのポート`+587+` を使用し、`+STARTTLS+` コマンドを使って安全な接続を確立する。これらはすべて、カスタマイズ可能な他のオプションである：

[,yaml]
----
stackstate:
  email:
    additionalProperties:
      # Add needed Java email properties for your mail server (use string values), defaults are:
      "mail.smtp.auth": "true"
      "mail.smtp.starttls.enable": "true"
    server:
      protocol: smtp
      port: 587
----

=== 外部秘密の使用

ユーザー名とパスワードが値で指定できず、外部の秘密から取得する必要がある場合は、xref:/setup/security/external-secrets.adoc#_getting_username_and_password_for_email_sending_from_an_external_secret[以下の]手順に従ってください。
