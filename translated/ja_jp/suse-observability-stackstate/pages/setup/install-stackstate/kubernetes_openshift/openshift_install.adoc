=== プロジェクトの作成

{stackstate-product-name} をインストールするプロジェクトを作成することから始めます。このチュートリアルでは、名前空間`+suse-observability+` を使用します：

[,text]
----
oc new-project suse-observability
----

[NOTE]
====
プロジェクト名は、`+helm+` と`+kubectl+` コマンドで、`+--namespace+` フラグのネームスペース名として使用される。
====

=== `+values.yaml+` 設定ファイルの作成

`+values.yaml+` ファイルには、ライセンスキー、ベース URL、管理者パスワード、サイジングプロファイルなど、{stackstate-product-name} の展開に必要なすべての構成が含まれています。

[NOTE]
====
*続ける前に*最新版のHelmチャート（`+helm repo update+` ）があることを確認してください。

====
推奨方法::
+
--
[NOTE]
=====
`+global.suseObservability+` 設定方法は、バージョン`+2.8.0+` 以降で利用可能です。それ以前のバージョンの場合は、レガシー・メソッドを使用する。
=====

`+global.suseObservability+` のコンフィギュレーション構造で`+values.yaml+` ファイルを作成する：

[source,yaml]
----
global:
  # Optional: Override image registry (defaults to registry.rancher.com)
  # Only needed for air-gapped environments or custom registries
  # imageRegistry: "your-private-registry.example.com"

  # Optional: Override storage class (uses cluster default if not set)
  # storageClass: "your-storage-class"

  suseObservability:
    # Required: Your {stackstate-product-name} license key
    license: "YOUR-LICENSE-KEY"

    # Required: Base URL for {stackstate-product-name}
    # Use http://localhost:8080 if ingress is not configured yet
    baseUrl: "https://observability.example.com"

    # Required: Sizing profile
    # Available: trial, 10-nonha, 20-nonha, 50-nonha, 100-nonha,
    #            150-ha, 250-ha, 500-ha, 4000-ha
    sizing:
      profile: "150-ha"

    # Required: Plain text Admin password
    adminPassword: "your-password"
    # Instead of adminPassword you can provide a bcrypt hashed password with adminPasswordBcrypt
    # Generate with: htpasswd -bnBC 10 "" "your-password" | tr -d ':\n'
    # adminPasswordBcrypt: "$2a$10$..."

    # Optional: Receiver API key (auto-generated if not provided)
    # receiverApiKey: "your-receiver-api-key"

    # Optional: Pull secret for private registry
    # pullSecret:
    #   username: "registry-user"
    #   password: "registry-pass"
----

TIP: bcryptパスワード・ハッシュを生成するには、次のようにする： `+htpasswd -bnBC 10 "" "your-password" | tr -d ':\n'+`

アフィニティ設定を含むその他の設定オプションについては、xref:/setup/install-stackstate/kubernetes_openshift/affinity.adoc[Kubernetesアフィニティの設定を]参照してください。

別のHelmチャートを実行することで、{stackstate-product-name} `+baseConfig_values.yaml+` 、`+sizing_values.yaml+` 、`+affinity_values.yaml+` ファイルを生成できます。`+suse-observability/suse-observability-values+` チャート。コマンドラインのサンプルは以下の通り：

[,text]
----
export VALUES_DIR=.
helm template \
  --set license='<your license>' \
  --set baseUrl='<suse-observability-base-url>' \
  --set sizing.profile='<sizing.profile>' \
  suse-observability-values \
  suse-observability/suse-observability-values --output-dir $VALUES_DIR
----

このコマンドは、{stackstate-product-name} Helmチャートをインストールするために必要なコンフィギュレーションを含むファイル`+$VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml+` 、`+$VALUES_DIR/suse-observability-values/templates/sizing_values.yaml+` 、`+$VALUES_DIR/suse-observability-values/templates/affinity_values.yaml+` を生成します。

[NOTE]
=====
{stackstate-product-name} 管理者パスワードは上記のコマンドで自動生成され、生成された`+basicConfig.yaml+` ファイルにコメントとして出力される。実際の値には、これらのパスワードの`+bcrypt+` ハッシュが含まれており、クラスタ内のHelmリリースに安全に保存されます。
=====


このチャートに渡すことのできる値は以下の通りである：

[cols="1,1,2"]
|===
| 構成 | 価値 | 説明

| 基本設定の生成
| `+basicConfig.generate+`
| `+basicConfig_values.yaml+` ファイルの生成のオン・オフ。

| レシーバーAPIキー
| `+receiverApiKey+`
| {stackstate-product-name} がエージェントからデータを受信するために使用する API キー。これは秘密にしておくべき鍵である。これを省略すると、ランダムな鍵が生成される。

| ベースURL
| `+baseUrl+`
| `+<STACKSTATE_BASE_URL>+`.ユーザーやエージェントが接続に使用する{stackstate-product-name} の外部 URL。例えば`+https://suse-observability.internal+` 。まだIngressのコンフィギュレーションを決めていない場合は、`+http://localhost:8080+` 。これは後で生成されたファイルで更新することができる。

| ユーザー名とパスワード
| `+-u+` `+-p+`
| {stackstate-product-name} が画像を取得する際に使用するユーザ名とパスワード。エアギャップ環境の場合、これらはローカルの docker レジストリのユーザー名とパスワードである必要がある。

| ライセンスキー
| `+license+`
| {stackstate-product-name} ライセンスキー。

| デフォルトパスワード
| `+adminPassword+`
| {stackstate-product-name} の UI にアクセスするためのデフォルト・ユーザー (`+admin+`) のパスワード。あるいは、`+adminPasswordBcrypt+` を使用して、bcrypt ハッシュ化されたパスワードを指定することもできます。同時に使用できるのは、2つのオプションのうち1つだけである。

| 画像登録
| `+imageRegistry+`
| {stackstate-product-name} イメージがホストされているレジストリ。提供されない場合、デフォルト値は'quay.io'になります。

| 秘密のユーザー名を引く
| `+pullSecret.username+`
| {stackstate-product-name} イメージがホストされている Docker レジストリからイメージを取得する際に使用するユーザー名。

| 秘密のパスワードを引き出す
| `+pullSecret.password+`
| {stackstate-product-name} イメージがホストされている Docker レジストリからイメージを取得する際に使用するパスワード。

| サイズ生成
| `+sizing.generate+`
| `+sizing_values.yaml+` ファイルの生成のオン・オフ。

| サイズプロファイル
| `+sizing.profile+`
| OneOfトライアル、10-nonha、20-nonha、50-nonha、100-nonha、150-ha、250-ha、500-ha。これらのプロファイルに基づいて、{stackstate-product-name} リソースのデフォルトサイズと、HA または Non-HA モードでデプロイする構成を含む`+sizing_values.yaml+` ファイルを生成できます。例えば、10-nonhaは、Non-HA{stackstate-product-name} インスタンスをデプロイして、10ノード・クラスターをNon-High Availabilityモードで監視するための`+sizing_values.yaml+` 。現在、Non-HAからHA環境への移行は不可能である。もし、あなたの環境が150ノード程度を監視する必要があると予想されるのであれば、すぐにでもHAを導入した方がいいだろう。

| ポッド・アンチアフィニティ
| `+affinity.podAntiAffinity.requiredDuringSchedulingIgnoredDuringExecution+`
| 同じデータサービスのレプリカが異なるKubernetesノードにスケジュールされることを強制する。`+HA+` プロフィールにのみ適用されます。`+true+` もしくは`+false+` 。デフォルトは`+true+` - 異なるノードへのスケジューリングが強制される。
|===

[NOTE]
=====
生成された`+basicConfig.yaml+` と`+sizing_values.yaml+` ファイルを安全に保管する。これらのファイルをアップグレード時に再利用することで、時間を節約し、{stackstate-product-name} 、同じAPIキーを使い続けることができます。これは、エイジェントやその他のデータプロバイダー（{stackstate-product-name} ）が更新の必要がないことを意味するので、望ましいことである。
`+basicConfig.generate=false+` 、`+sizing.generate=false+` のスイッチを使えば、`+output-dir+` に前回生成したバージョンのファイルを残したまま、いずれかのファイルを無効にすることができる。
=====

[NOTE]
=====
{stackstate-product-name} Values チャートは、ポッドのスケジューリング動作を制御するために、メインの{stackstate-product-name} チャートで使用できるアフィニティ構成を生成します。詳細については、「xref:/setup/install-stackstate/kubernetes_openshift/affinity.adoc[Configure Kubernetes Affinities]」のドキュメントを参照してください。
=====

=== 作成 `+openshift-values.yaml+`

OpenShiftは通常のKubernetesよりも厳しいセキュリティモデルを持っているため、デプロイメント内の標準的なセキュリティコンテキストをすべて無効にする必要がある。

以下の内容でHelm values file`+openshift-values.yaml+` を作成し、生成された`+values.yaml+` ファイルの隣に保存します。OpenShiftのデプロイに必要な値が含まれています。

[,yaml]
----
elasticsearch:
  prometheus-elasticsearch-exporter:
    podSecurityContext: ""
  sysctlInitContainer:
    enabled: false
scc:
  enabled: true
----

== 手動で`+SecurityContextConfiguration+` オブジェクトを作成する

管理者アカウントで{stackstate-product-name} を OpenShift にインストールできない場合は、管理者に以下の`+SecurityContextConfiguration+` オブジェクトを適用するよう依頼してください。

[,yaml]
----
apiVersion: security.openshift.io/v1
kind: SecurityContextConstraints
metadata:
  name: {{ template "common.fullname.short" . }}-{{ .Release.Namespace }}
  labels:
    {{- include "common.labels.standard" . | nindent 4 }}
  annotations:
    helm.sh/hook: pre-install
    suse-observability.io/note: "Ignored by helm uninstall, has to be deleted manually"
fsGroup:
  type: RunAsAny
groups:
- system:serviceaccounts:{{ .Release.Namespace }}
runAsUser:
  type: RunAsAny
seLinuxContext:
  type: MustRunAs
supplementalGroups:
  type: RunAsAny
volumes:
- configMap
- downwardAPI
- emptyDir
- ephemeral
- persistentVolumeClaim
- projected
- secret
allowHostDirVolumePlugin: false
allowHostIPC: false
allowHostNetwork: false
allowHostPID: false
allowHostPorts: false
allowPrivilegeEscalation: true
allowPrivilegedContainer: false
readOnlyRootFilesystem: false
seccompProfiles:
  - runtime/default
----
