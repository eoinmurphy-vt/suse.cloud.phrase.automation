= エクステンションのトラブルシューティング
:revdate: 2025-11-03
:page-revdate: {revdate}
:description: SUSE Observability セルフホスト

このセクションでは、{stackstate-product-name} と Rancher UI エクステンション間の相互運用性に問題がある場合のトラブルシューティング手順を説明します。

== HARの収集（Http ARchive）

Http ARchiveは、ブラウザからRancherと{stackstate-product-name} に送信されたリクエストの詳細を収集します。 また、受け取った回答のログも記録する。
このアーカイブは、エラーが発生したページ/ブラウザタブに固有のものです。 バックとなるRancherや{stackstate-product-name} インスタンスにアクセスする必要はない（ただし、それらのログは後の段階で役に立つかもしれない）。

* 開発者ツール」パネルの*「ネットワーク」*タブを開く。 `+F12+` からアクセスできます。 ネットワークタブには、リクエストされたすべてのログが表示されます。
* ログを消去し、問題のある動作を表示しているページを（ハード的に）更新してください。
* これで、RancherとSUSE Observabilityの両方への大量のリクエストが表示されるはずです。
* HARをエクスポート/ダウンロード/保存する - ブラウザによってここでは異なる名前を使用します。

== クラスターは未観測

Rancherでクラスタの健全性ステータスを表示するには、これらの構成間の整合が必要です：

* Rancher UIでのクラスタの(表示)名前。
* クラスタにデプロイされたエージェントに設定された`+stackstate.cluster.name+` 
* SUSE ObservabilityのKubernetesスタックパックの設定におけるクラスタ名

不一致は、Rancherのクラスタ概要で "no data received "と表示される傾向がある。

== 接続に失敗

SUSE Observabilityの設定では、以下の理由によりRancher UIで有効な接続が得られない場合があります：

* RancherホストがSUSE Observabilityの許可オリジンとして設定されていません。 helm値の`+stackstate.allowedOrigins+` リストに追加するか、デプロイコマンドに`+--set stackstate.allowedOrigins={"https://<rancher-url>"}+` 。
* SUSE Observabilityはブラウザからアクセスできません。例えば、ルート証明書がインストールされていない。
