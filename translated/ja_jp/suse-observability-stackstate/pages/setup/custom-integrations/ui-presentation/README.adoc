== 概要

ここで文書化されたコンポーネント・プレゼンテーション・モデルは、コンポーネント（ポッド、VM、サービスなど）が{stackstate-product-name} 、ユーザー・インターフェースでどのように表示されるかを宣言する。
ネーミング、アイコン、テーブル、概要ページやハイライトページ、フィルター、メニュー項目の配置など、*プレゼンテーションに関することだけに*焦点を当てている。

このモデルのゴールは、既存の断片的なアプローチと比較して、コンポーネントのUIビヘイビアを定義するための、よりコンポーザブルで拡張可能な方法を提供することである。
