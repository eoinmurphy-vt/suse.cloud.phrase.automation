== ComponentPresentation

UIプレゼンテーションモデルの中心概念は*ComponentPresentation*です。

`+ComponentPresentation+` は、アクティブなコンテキストに応じて、1つまたは複数の一致するコンポーネントがUIでどのように表示されるかを定義します。
ネーミング、アイコン、テーブル、概要ページやハイライトページ、フィルター、メニュー項目の配置など、*プレゼンテーションに関することだけに*焦点を当てている。

高レベルでは、ComponentPresentation：

* バインディング（単純なSTQLクエリー）を使用してコンポーネントを選択します。
* 特定のモード内で適用される（例えば、観測可能性）。
* 1つまたは複数のプレゼンテーションのアスペクト（概要表やハイライトなど）を定義する。
* プレゼンテーション・プロパティを部分的に定義し、他の一致する定義と合成することができる。

複数のComponentPresentationを同じコンポーネントに適用することができる。このような場合、それぞれの定義を組み合わせて最終的なプレゼンテーションを決定する。
