=== モニターの結果がコンポーネントに表示されない

まず、モニターがxref:use/alerting/k8s-add-monitors-cli.adoc#_verify_the_execution_of_the_monitor[結果を出して]いるかどうかを確認する。もしそうであっても、モニター結果がコンポーネントに表示されない場合は、バインディングに問題がある可能性がある。まず、以下のコマンドで確認する：

[,bash]
----
sts monitor status --id <id>
----

出力が`+Monitor health states with identifier which has no matching topology element (<nr>): ....+` 、これは`+urnTemplate+` 、トポロジーに一致する識別子が生成されない可能性があることを示している。この問題を解決するには、xref:use/alerting/k8s-add-monitors-cli.adoc#_bind_the_results_of_the_monitor_to_the_correct_components[urnTemplateをもう一度見直して]ください。

=== モニターのステータス概要にエラーが表示される

CLIでモニターのステータスを取得する

[,bash]
----
sts monitor status --id <id>
----

セクション`+Monitor Stream errors:+` 、モニター上で発生しているエラーを表示し、さらなるヘルプを提供します。
