= OpenTelemetryのトポロジー
:revdate: 2025-12-23
:page-revdate: {revdate}
:description: OpenTelemetryのトポロジー

{stackstate-product-name} は、OpenTelemetry (OTel) の遠隔測定データから、アプリケーションとインフラのトポロジーを自動的に導き出すことができます。
これは、OpenTelemetryのトポロジーマッピングによって実現されます。このマッピングは、トレースとメトリックシグナルをトポロジーコンポーネント（頂点）とリレーション（エッジ）に宣言的に変換します。

== 概要

ハイレベルでは、OpenTelemetryからのトポロジー生成は次のように動作する：

* OpenTelemetry テレメトリ（メトリクスおよび/またはトレース）をコレクタが受信します。
* OTel コレクタは、構成されたマッピングをテレメトリに対して評価するトポロジ ーコネクタを使用する。
* マッピング条件がテレメトリと一致すると、コンポーネントやリレーションが生成される。
* これらのトポロジーオブジェクトは`+sync+` サービスによって消費され、トポロジーグラフにマージされる。
* マッピングは宣言的で、YAMLベースであり、テレメトリがシステムを流れるときに継続的に評価される。

現在サポートされている信号：

* 指標
* 痕跡

[NOTE]
====
ログベースのトポロジー生成はまだサポートされていない。
====

コンポーネントとリレーションのマッピングの詳細については、xref:/setup/custom-integrations/otelmappings/concepts.adoc[コンセプトの]ページを参照してください。
