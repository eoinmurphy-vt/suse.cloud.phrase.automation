= SUSE Observability Open Telemetry Protocolのサポート
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

SUSE Observabilityは、`+grpc+` (OTLPとも呼ばれます)と`+http/protobuf+` (OTLP over HTTPとも呼ばれます)の2つのバージョンのOTLPプロトコルをサポートしています。コレクタの設定では、使用するエクスポータを選択できますが、SUSE Observability用に正しいURLを設定してください。より大きなペイロードと高いスループットを可能にする`+grpc+` プロトコルのバージョンが望ましい。しかし、あなたのインフラで`+grpc+` のサポートが不十分な場合は、HTTPバージョンに切り替えることができる。xref:/setup/otel/troubleshooting.adoc#_some_proxies_and_firewalls_dont_work_well_with_grpc[トラブルシューティング]

== SUSE Cloud Observability

SUSE Cloud Observabilityのエンドポイントは次のとおりです：

* OTLP:`+otlp-<your-suse-observability>.app.stackstate.io:443+` (URLスキームなし)
* OTLP over HTTP:`+https://otlp-http-<your-suse-observability>.app.stackstate.io+` (URL スキーム:`+https+`)

== セルフホストSUSE Observability

セルフホストインストールの場合は、xref:/setup/install-stackstate/kubernetes_openshift/ingress.adoc#_configure_ingress_rule_for_open_telemetry[ここで説明する]ようにSUSE Observability用のイングレスを設定して、どちらか一方のエンドポイント、または両方を有効にする必要があります。

SUSE Observabilityがコレクタと同じクラスタで実行されている場合、サービスエンドポイントを使用してイングレスなしで使用することもできますが、これはHTTPプロトコルを使用する場合にのみ可能です：

* HTTP 上の OTLP： `+http://suse-observability-otel-collector.<namespace>.svc.cluster.local:4318+`

コレクターの設定（次のセクションを参照）で、`+insecure: true+` を設定し、https ではなく、プレーンな http エンドポイントの使用を許可するようにしてください。

GRPC プロトコルは、安全でない接続での認証情報の送信をサポートしていないため、安全なイングレス設定と組み合わせてのみ使用できる。

== コレクター構成

=== OTLPプロトコル

コレクタ設定の例では、以下のように OTLP プロトコルを使用する：

----
extensions:
  bearertokenauth:
    scheme: SUSEObservability
    token: "${env:API_KEY}"
exporters:
  otlp/suse-observability:
    auth:
      authenticator: bearertokenauth
    # Put in your own otlp endpoint, for example otlp-suse-observability.my.company.com:443
    endpoint: <otlp-suse-observability-endpoint>
    # Optional TLS configurations:
    #tls:
    # To disable certificate verification (but still use TLS):
    #  insecure_skip_verify: true
----

=== OTLP HTTPプロトコル

代わりにOTLP over HTTPプロトコルを使うには、代わりに`+otlphttp+` 。また、パイプライン内のすべてのエクスポーター参照を`+otlp/suse-observability+` から`+otlphttp/suse-observability+` に更新してください！検索/置換を使用して、すべての出現箇所を確実に変更する。

----
extensions:
  bearertokenauth:
    scheme: SUSEObservability
    token: "${env:API_KEY}"
exporters:
  otlphttp/stackstate:
    auth:
      authenticator: bearertokenauth
    # Put in your own otlp-http endpoint, for example https://otlp-http-suse-observability.my.company.com:443
    endpoint: <otlp-http-suse-observability-endpoint>
    # Optional TLS configurations:
    #tls:
    # To disable TLS entirely:
    #  insecure: true
    # To disalbe certificate verification (but still use TLS):
    #  insecure_skip_verify: true
----

エクスポーターの正確な要件と動作を制御するために、より多くの設定が可能です。例えば、カスタムCAルート証明書を使用したり、クライアント証明書を有効にすることも可能である。詳しくは、https://github.com/open-telemetry/opentelemetry-collector/blob/main/exporter/otlpexporter/README.md[]OTLPご覧ください。
