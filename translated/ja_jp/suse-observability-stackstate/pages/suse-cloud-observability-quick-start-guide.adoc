== 概要

Cloud Provider MarketplaceからSUSE Cloud Observabilityを購入すると、SUSE Cloud Observability環境がプロビジョニングされます。

SUSE Cloud Observabilityから、必要なログイン情報と環境へのリンクが記載されたメールが届きます。このクイックスタートガイドは、SUSE Cloud Observabilityの導入を開始し、独自のデータを取得するのに役立ちます。

== はじめの一歩

. パスワードの設定
. SUSE Cloud Observabilityインスタンスにログインします。
. クラスタにSUSE Observabilityエージェントをインストールします。
. 個人アカウント詳細の変更

=== パスワードの設定

SUSE Cloud Observabilityからのメールには、アカウントに初期パスワードを設定できる固有のリンクが含まれています。 これは、ログインして観測可能な環境を設定する前に行う必要があります。

=== SUSE Observability環境へのアクセス

SUSE Cloud Observabilityから受け取ったメールに記載されている固有のリンクをクリックして、SUSE Cloud Observability環境にログインします。 パスワードを入力すると、クラスタを追加するための設定画面が表示されます。

ifdef::feature-agent-install-cluster[]
=== クラスタにSUSE Observabilityエージェントをインストールします。

SUSE Cloud Observabilityは、ダウンストリームクラスタの設定とobservability環境へのデータ取り込みを容易にするためにStackPacksを使用しています。

SUSE Observability UIで画面左上をクリックしてメインメニューを開き、`+StackPacks+` >`+Kubernetes+` に進みます。

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

Kubernetes StackPackの画面に入ったら、クラスタをobservability環境に追加するのはとても簡単だ。

kubeconfig'で使用するクラスタ名と一致する必要はありません。

Kubernetesクラスタ名は、小文字または数字で始まり、小文字、数字、ドット、ダッシュ（.）

名前を入力し、`+Install+` ボタンをクリックします。 クラスターが「Waiting for data（データ待ち）」とフラグが立った状態で、以下の画面が表示されるはずだ。

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - クラスタの追加]

クラスタ名をクリックすると、このセクションが展開され、既存のクラスタに使用できる前提条件やコマンドを含む一連の情報が表示されます。

情報を確認し、Kubernetes環境に正しいパーミッションがあることを確認し、サポートされているバージョンのKubernetesを実行していることを確認します。

これらのコマンドは、あなたのobservabilityデプロイメントに固有であり、必要なAPIキーとURLを含んでいます。 EKS、RKE、一般的なKubnernetesクラスタなどのセクションがあります。

これらのコマンドにより、SUSE Cloud Obervabilityエージェントがインストールされ、クラスタがSUSE Cloud Observability環境に接続されます。

UIから直接コマンドを使用するか、xref:/k8s-quick-start-guide.adoc[クイックスタートガイドの]「StackStateエージェントとクラスタエージェントのデプロイ」セクションに記載されているコマンドに従って、ダウンストリームクラスタにデプロイできます。

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - インストール後]

クラスタが接続されると、SUSE Cloud Observability UIに緑色のチェックマークが表示されます。

この時点で、データの探索を始めることができる。
endif::feature-agent-install-cluster[]

=== 個人アカウント詳細の変更

Eメールに記載されたステップ4は、個人情報の更新です。 Eメールに記載されている固有のリンクをクリックし、必要に応じて基本的な個人情報を追加してください。 必要に応じて、このセクションから2FA認証を設定することもできます。

=== データの探索

データの探索を開始するには、画面左上をクリックしてメインメニューを開き、`+Kubernetes+` 、観測可能な項目のリストを表示します。

image::accessing_views_1.png[SUSE Cloud Observability - データの探索]

インフラストラクチャー・セクションから'Clusters'を選択すると、監視しているクラスタのリストが表示される。

image::accessing_views_2.png[SUSE Cloud Observability - クラスタビュー]

この時点で、自分のクラスタのデータの調査を開始したり、データを収集するクラスタを追加したりすることができる。

カスタムビューの作成など、SUSE Cloud Observabilityの使用方法の詳細については、標準ドキュメントを参照してください。

=== SUSE Cloud Observabilityの制限

SUSE Cloud Observabilityは、REDシグナル（Rate、Errors、Duration）をすぐに提供せず、Rateシグナルのみを表示することに注意してください。 この機能はSUSE Rancher Primeで利用できます。
完全なObservabilityソリューションを提供するためにこれらのシグナルが必要なお客様は、SUSE Rancher PrimeについてSUSEにお問い合わせください。また、このデータはOpenTelemetryコレクタを使用して収集することもできます。
