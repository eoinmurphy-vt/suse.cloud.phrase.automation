プロジェクトはこの図書館を選んだ：

* WebAssemblyバイナリで動作します。
他の一般的なロギング・ソリューションは、WebAssemblyをターゲットにコンパイルすることができない。
* https://github.com/francoispqt/onelog#benchmarks[] 。
* 構造化ロギングをサポートする。

== ロガーの初期化

ロガー構造体を初期化する必要がある。
この初期化をグローバル変数で実行することで、2つの主要なポリシーのエントリーポイントからログを記録することができます:`+validate+` と`+validate_settings+` 。

メインパッケージの`+main.go+` 、ロガーの初期化がある：

[,go]
----
var (
    logWriter = kubewarden.KubewardenLogWriter{}
    logger    = onelog.New(
        &logWriter,
        onelog.ALL, // shortcut for onelog.DEBUG|onelog.INFO|onelog.WARN|onelog.ERROR|onelog.FATAL
    )
)
----

== ロガーの消費

これで、`+logger+` オブジェクトを使って、ポリシーに必要な場所からログを取ることができる：

[,go]
----
func validate(payload []byte) ([]byte, error) {
    // ...
    logger.Info("validating request")
    // ...
}
----

構造化ロギングを追加できる：

[,go]
----
func validate(payload []byte) ([]byte, error) {
    // ...
    logger.WarnWithFields("logging something important", func(e onelog.Entry) {
        e.String("one_field", "a value")
        e.String("another_field", "another value")
    })
    // ...
}
----

詳しくはhttps://pkg.go.dev/github.com/francoispqt/onelog?utm_source=godoc[`+onelog+` の]参照してください。

ポリシーのログは、ポリシー評価者（たとえば、`+kwctl+` または`+policy-server+` ）に送られ、彼らはポリシーの代わりにログを記録する。
https://www.jaegertracing.io/[] Jaegerのような分散トレースを可能にする他のコンポーネントと相互運用可能なメカニズムを使用しています。
