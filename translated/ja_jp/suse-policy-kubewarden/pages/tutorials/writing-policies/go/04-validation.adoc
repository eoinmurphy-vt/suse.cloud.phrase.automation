検証ロジックは`+validate.go+` 。

バリデーション・ロジックが必要だ：

* 受信した`+payload+` オブジェクトから関連情報を抽出する。
* 入力とポリシー設定に基づいた応答を返す。

受信ペイロードは、xref:reference/spec/03-validating-policies.adoc[このドキュメントで]説明されているJSONオブジェクトであり、そこから2つの方法でデータを取得することができる：

. JSONデータをGo型にアンマーシャルする。
. JSONクエリーを実行する(以下のようなもの)https://stedolan.github.io/jq/[`+jq+`]).

このセクションでは、Go型を使った最初のアプローチに焦点を当てる。
2つ目のアプローチについては、後のxref:tutorials/writing-policies/go/09-validation-with-queries.adoc[クエリーによる検証の]セクションで説明する。

[NOTE]
====

Kubernetesのオブジェクトに依存することで、jqのような検索を行う代わりに、より大きなWebAssemblyモジュールを生成することになる。
Kubernetesのオブジェクトを使ったポリシーは1.5&nbsp;MB程度だが、`+gjson+` を使ったポリシーは300&nbsp;KB程度だ。

サイズとは別に、Kubernetesオブジェクトを使用するポリシーは、最初の実行時に多くの時間がかかる。
{project-name} 、Wasmtimeのキャッシュ機能を使用しているため、以下の呼び出しは高速である。
最初の実行は`+kwctl+` 、約20秒かかるが、その後の実行は1～2秒である。
そのため、{short-project-name} Policy Serverの起動時間は遅くなるが、Kubernetesオブジェクトの使用によってポリシーの評価時間が影響を受けることは通常ない。
====


== `+validate+` 関数

`+validate.go+` 、足場テンプレートによって提供されるポリシーは、すでに`+validate+` 関数を持っている。
このチュートリアルでは、これに少し変更を加える必要がある。

これが、完成したときの機能のあるべき姿である：

[,go]
----
func validate(payload []byte) ([]byte, error) {
    // NOTE 1
    // Create a ValidationRequest instance from the incoming payload
    validationRequest := kubewarden_protocol.ValidationRequest{}
    err := json.Unmarshal(payload, &validationRequest)
    if err != nil {
        return kubewarden.RejectRequest(
            kubewarden.Message(err.Error()),
            kubewarden.Code(400))
    }

    // NOTE 2
    // Create a Settings instance from the ValidationRequest object
    settings, err := NewSettingsFromValidationReq(&validationRequest)
    if err != nil {
        return kubewarden.RejectRequest(
            kubewarden.Message(err.Error()),
            kubewarden.Code(400))
    }

    // NOTE 3
    // Access the **raw** JSON that describes the object
    podJSON := validationRequest.Request.Object

    // NOTE 4
    // Try to create a Pod instance using the RAW JSON we got from the
    // ValidationRequest.
    pod := &corev1.Pod{}
    if err := json.Unmarshal([]byte(podJSON), pod); err != nil {
        return kubewarden.RejectRequest(
            kubewarden.Message(
                fmt.Sprintf("Cannot decode Pod object: %s", err.Error())),
            kubewarden.Code(400))
    }

    logger.DebugWithFields("validating pod object", func(e onelog.Entry) {
        e.String("name", pod.Metadata.Name)
        e.String("namespace", pod.Metadata.Namespace)
    })

    // NOTE 5
    for label, value := range pod.Metadata.Labels {
        if err := validateLabel(label, value, &settings); err != nil {
            return kubewarden.RejectRequest(
                kubewarden.Message(err.Error()),
                kubewarden.NoCode)
        }
    }

    return kubewarden.AcceptRequest()
}
----

コードには`+NOTE+` ：

. JSONペイロードをアンマーシャリングして`+kubewarden_protocol.ValidationRequest+` 。
. 先に`+settings.go+` ファイルで定義した関数を使用して、`+Settings+` オブジェクトを作成する。
. `+ValidationRequest+` の一部であるPodの生のJSON表現にアクセスする。
. Podオブジェクトのunmarshal。
. Podのラベルを反復処理する。
`+validateLabel+` 」という新しい関数を使い、ポリシー違反のラベルを特定する。

また、`+validate.go+` ファイルで`+validateLabel+` 関数を定義する必要がある：

[,go]
----
func validateLabel(label, value string, settings *Settings) error {
    if settings.DeniedLabels.Contains(label) {
        return fmt.Errorf("Label %s is on the deny list", label)
    }

    regExp, found := settings.ConstrainedLabels[label]
    if found {
        // This is a constrained label
        if !regExp.Match([]byte(value)) {
            return fmt.Errorf("The value of %s doesn't pass user-defined constraint", label)
        }
    }

    return nil
}
----

== 検証コードのテスト

これで、検証コードの動作をチェックするためのユニットテストを書くことができる。
`+validate_test.go+` ファイルからテストを探す。

これに合わせて足場ファイルの内容を置き換える必要がある：

.`+validate_test.go+`
[%collapsible]
======

[,go]
----
package main

import (
    "regexp"
    "testing"

    "encoding/json"

    mapset "github.com/deckarep/golang-set/v2"
    corev1 "github.com/kubewarden/k8s-objects/api/core/v1"
    metav1 "github.com/kubewarden/k8s-objects/apimachinery/pkg/apis/meta/v1"
    kubewarden_protocol "github.com/kubewarden/policy-sdk-go/protocol"
    kubewarden_testing "github.com/kubewarden/policy-sdk-go/testing"
)

func TestValidateLabel(t *testing.T) {
    // NOTE 1
    cases := []struct {
        podLabels         map[string]string
        deniedLabels      mapset.Set[string]
        constrainedLabels map[string]*RegularExpression
        expectedIsValid   bool
    }{
        {
            // ➀
            // Pod has no labels -> should be accepted
            podLabels:         map[string]string{},
            deniedLabels:      mapset.NewThreadUnsafeSet[string]("owner"),
            constrainedLabels: map[string]*RegularExpression{},
            expectedIsValid:   true,
        },
        {
            // ➁
            // Pod has labels, none is denied -> should be accepted
            podLabels: map[string]string{
                "hello": "world",
            },
            deniedLabels:      mapset.NewThreadUnsafeSet[string]("owner"),
            constrainedLabels: map[string]*RegularExpression{},
            expectedIsValid:   true,
        },
        {
            // ➂
            // Pod has labels, one is denied -> should be rejected
            podLabels: map[string]string{
                "hello": "world",
            },
            deniedLabels:      mapset.NewThreadUnsafeSet[string]("hello"),
            constrainedLabels: map[string]*RegularExpression{},
            expectedIsValid:   false,
        },
        {
            // ➃
            // Pod has labels, one has constraint that is respected -> should be accepted
            podLabels: map[string]string{
                "cc-center": "team-123",
            },
            deniedLabels: mapset.NewThreadUnsafeSet[string]("hello"),
            constrainedLabels: map[string]*RegularExpression{
                "cc-center": {
                    Regexp: regexp.MustCompile(`+team-\d++`),
                },
            },
            expectedIsValid: true,
        },
        {
            // ➄
            // Pod has labels, one has constraint that are not respected -> should be rejected
            podLabels: map[string]string{
                "cc-center": "team-kubewarden",
            },
            deniedLabels: mapset.NewThreadUnsafeSet[string]("hello"),
            constrainedLabels: map[string]*RegularExpression{
                "cc-center": {
                    Regexp: regexp.MustCompile(`+team-\d++`),
                },
            },
            expectedIsValid: false,
        },
        {
            // ➅
            // Settings have a constraint, pod doesn't have this label -> should be rejected
            podLabels: map[string]string{
                "owner": "team-kubewarden",
            },
            deniedLabels: mapset.NewThreadUnsafeSet[string]("hello"),
            constrainedLabels: map[string]*RegularExpression{
                "cc-center": {
                    Regexp: regexp.MustCompile(`+team-\d++`),
                },
            },
            expectedIsValid: false,
        },
    }

    // NOTE 2
    for _, testCase := range cases {
        settings := Settings{
            DeniedLabels:      testCase.deniedLabels,
            ConstrainedLabels: testCase.constrainedLabels,
        }

        pod := corev1.Pod{
            Metadata: &metav1.ObjectMeta{
                Name:      "test-pod",
                Namespace: "default",
                Labels:    testCase.podLabels,
            },
        }

        payload, err := kubewarden_testing.BuildValidationRequest(&pod, &settings)
        if err != nil {
            t.Errorf("Unexpected error: %+v", err)
        }

        responsePayload, err := validate(payload)
        if err != nil {
            t.Errorf("Unexpected error: %+v", err)
        }

        var response kubewarden_protocol.ValidationResponse
        if err := json.Unmarshal(responsePayload, &response); err != nil {
            t.Errorf("Unexpected error: %+v", err)
        }

        if testCase.expectedIsValid && !response.Accepted {
            t.Errorf("Unexpected rejection: msg %s - code %d with pod labels: %v, denied labels: %v, constrained labels: %v",
                *response.Message, *response.Code, testCase.podLabels, testCase.deniedLabels, testCase.constrainedLabels)
        }

        if !testCase.expectedIsValid && response.Accepted {
            t.Errorf("Unexpected acceptance with pod labels: %v, denied labels: %v, constrained labels: %v",
                testCase.podLabels, testCase.deniedLabels, testCase.constrainedLabels)
        }
    }
}
----

======

このテストは「テストケース駆動型」のアプローチを採用している。
テストケースに必要なデータを保持する`+struct+` を定義することから始めます。`+NOTE 1+` を参照してください：

[,go]
----
struct {
        podLabels         map[string]string
        deniedLabels      mapset.Set[string]
        constrainedLabels map[string]*RegularExpression
        expectedIsValid   bool
}
----

そして、いくつかのテストケースを宣言する。
上の大きな折りたたみ可能なコードブロックの➀から➅にスタートラインが記されている。

例えば、ラベルのないポッドは有効であると考えるべきである。
これらの入力値でテストすることができる：

[,go]
----
{
  podLabels:         map[string]string{},
  deniedLabels:      mapset.NewThreadUnsafeSet[string]("owner"),
  constrainedLabels: map[string]*RegularExpression{},
  expectedIsValid:   true,
}
----

`+NOTE 2+` まで、テストはこの方法で新しいシナリオを定義する。
ここでは、以下のコードを使用して、さまざまなテストケースを繰り返し実行する：

. `+testCase+` から提供されるデータを使用して、`+BasicSettings+` オブジェクトを作成する。
. `+Pod+` オブジェクトを作成し、`+testCase+` で定義したラベルを割り当てる。
. `+payload+` オブジェクトを作成する。これには{short-project-name} SDK のヘルパー関数`+kubewarden_testing.BuildValidationRequest+` を使用します。
この関数は、リクエストの対象であるオブジェクト、`+Pod+` 、および設定を記述するオブジェクト、`+BasicSettings+` インスタンスを入力として受け取る。
. 最後に、`+validate+` 関数を呼び出し、その結果をチェックする。

=== 壊れたユニットテストを修正する

あなたが発見した壊れたテストを修正するには、`+validate.go+` の`+validate+` バリデーション関数を変更する必要があります。

現在、バリデーション・ロジックの中核は以下の行だ：

[,go]
----
for label, value := range pod.Metadata.Labels {
    if err := validateLabel(label, value, &settings); err != nil {
        return kubewarden.RejectRequest(
            kubewarden.Message(err.Error()),
            kubewarden.NoCode)
    }
}
----

ここでは、各ラベルを繰り返しチェックして、拒否されていないことと、ユーザーによって指定された制約のいずれかに違反していないことを確認する。
しかし、ポッドが`+Settings.ConstrainedLabels+` で指定されたラベルをすべて持っていることを確認していない。

`+for+` ループの直後に新しいコードを追加する：

[,go]
----
for requiredLabel := range settings.ConstrainedLabels {
    _, found := pod.Metadata.Labels[requiredLabel]
    if !found {
        return kubewarden.RejectRequest(
            kubewarden.Message(fmt.Sprintf(
                "Constrained label %s not found inside of Pod",
                requiredLabel),
            ),
            kubewarden.NoCode)
    }
}
----
