== {project-name} 。

{short-project-name} は、Kubernetes環境におけるポリシーのアドミッションとエンフォースメントに柔軟性を提供する。

=== メリットと価値

- https://webassembly.org[WebAssembly]バイナリを生成する任意のプログラミング言語を使用して、ポリシーを記述します。
- WebAssemblyは、プロセッサやオペレーティング・システム間のポリシーの互換性を可能にします。
- 他のポリシー・エンジンからポリシーを書き換えることなく再利用。
- https://opencontainers.org[]OCI準拠のレジストリなど、標準的で安全なメカニズムを使用してポリシーを配布する。
- 入場時のポリシー実施により、コンプライアンスに準拠したワークロードのみが実行されます。
- {short-project-name} 監査スキャナーは、時間の経過とともにポリシーの実施を能動的かつ継続的にチェックする。
- https://slsa.dev[]SLSA（Supply Chain Levels for Software Artifacts）ツールとプラクティスを使用してポリシーを検証する。
- {project-name} は、アドミッションポリシー管理への包括的なアプローチを提供する。
- CNCFのメンバーシップと成長するオープンソースコミュニティとエコシステムは、{short-project-name} 、透明性、コラボレーション、改善を支援する。

=== 使用例

- セキュリティの強化。例えば、コンテナ特権を制限するポリシーの適用、ネットワーク・ポリシーの適用、安全でないイメージ登録のブロックなどである。
- コンプライアンス監査ワークロードが組織または規制の基準およびベストプラクティスに準拠していることを確認する。
- リソースの最適化。リソース制限とクォータを実施する。

ユースケースの詳細については、xref:use-cases.adoc[ユースケースの]ページを参照されたい。

== WebAssemblyとは？

https://webassembly.org/[WebAssemblyの公式]サイトに記載されている：

____
WebAssembly（略称Wasm）は、スタックベースの仮想マシン用のバイナリ命令フォーマットです。Wasmはプログラミング言語の移植可能なコンパイル・ターゲットとして設計されており、クライアントおよびサーバー・アプリケーションのウェブ上での展開を可能にする。
____

Wasmはもともとブラウザの「拡張機能」として考案された。しかし、WebAssemblyコミュニティは、ブラウザ以外でのWasmコードの実行を可能にする取り組みを行っている。

== なぜWebAssemblyを使うのか？

ツールチェーンがWasmバイナリを生成できれば、ユーザーは好きなプログラミング言語を使ってKubernetesポリシーを書くことができる。

Wasmモジュールはポータブルであり、一度ビルドすれば、どのようなプロセッサアーキテクチャやオペレーティングシステムでも動作する。例えば、アップル・シリコン上で開発・構築されたポリシーは、変換することなくAMD64/Intel64 Linux上で実行できる。

政策立案者は、そのスキル、ツール、ベストプラクティスを再利用することができる。ポリシーは、再利用可能なブロック（通常のライブラリ）を持つことができる「伝統的な」プログラムである。Lintやテストもできるし、現在のCIやCDのワークフローに組み込むこともできる。
