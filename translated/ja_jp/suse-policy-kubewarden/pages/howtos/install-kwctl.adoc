= kwctl のインストール ({project-name} CLI)
:revdate: 2026-03-18
:page-revdate: {revdate}
:title: kwctlのインストール
:description: kwctl ({project-name} CLI)を、様々なオペレーティングシステム用のインストール手順に従ってインストールし、シェル補完を有効にする。
:keywords: ["{project-name}", kubewarden, kubernetes, install kwctl, install, kwctl]。
:doc-persona: kubewarden-all
:doc-type: ハウツー
:doc-topic: install-kwctl

`+kwctl+` は、{project-name} のコマンドラインインターフェイス（CLI）ツールである。以下は、いくつかのオペレーティングシステムのインストール手順です。

[tabs]
====
リナックス::
+
--
Linux用インストール

. ホームブリュー
+
LinuxでHomebrewパッケージ・マネージャーを使っているのなら：
+
[source,console]
----
brew install kwctl
----
+
インストールを確認する：
+
[source,console]
----
kwctl --version
----

. ザイパー（openSUSE）
+
[source,console]
----
zypper install kwctl
----

. AUR
+
Arch Linux や Arch ベースのディストリビューションを使っているなら、AUR から kwctl をインストールできます。
+
.. AURヘルパーを使う（やった）：
+
[source,console]
----
yay -S kwctl
----
+
.. makepkgを使用：
+
[source,console]
----
# Clone the AUR package
git clone https://aur.archlinux.org/kwctl.git
cd kwctl

# Build and install the package
makepkg -si
----
+
.. インストールを確認する：
+
[source,console]
----
kwctl --version
----

. 手動インストール
+
.. `+kwctl+` for Linuxの最新リリースをダウンロードする：
+
[source,console]
----
curl -LO https://github.com/kubewarden/kwctl/releases/latest/download/kwctl-linux-x86_64.zip
----
+
ARM64システム（例：Raspberry Pi）の場合は、これを使用する：
+
[source,console]
----
curl -LO https://github.com/kubewarden/kwctl/releases/latest/download/kwctl-linux-aarch64.zip
----
+
.. ダウンロードした`+.zip+` ファイルからファイルを解凍します：
+
[source,console]
----
unzip kwctl-linux-x86_64.zip
----
+
これは以下のファイルを抽出する：
+
* `+kwctl-linux-x86_64+`:`+kwctl+` バイナリー
* `+kwctl-linux-x86_64.sig+`:バイナリを検証するための署名ファイル
* `+kwctl-linux-x86_64.pem+`:署名を検証するための証明書ファイル
+
.. バイナリをPATH内のディレクトリに移動し、`+kwctl+` にリネームして実行可能にする。
+
.. インストールを確認する：
+
[source,console]
----
kwctl --version
----

macOS用にインストールする。

ウィンドウズ::
+
--
Windows用インストール

. ダウンロード`+kwctl+` ：
    .. ブラウザを開き、https://github.com/kubewarden/kwctl/releases/latest[{short-project-name} リリースページに]アクセスする。
    .. `+kwctl-windows-x86_64.zip+` ファイルをダウンロードしてください。

. ダウンロードしたzipファイルからファイルを解凍します。その中には
+
* `+kwctl-windows-x86_64.exe+``+kwctl+` 。
* `+kwctl-windows-x86_64.sig+`バイナリを検証するための署名ファイル。
* `+kwctl-windows-x86_64.pem+`署名を検証するための証明書ファイル。

. 使いやすいように、バイナリファイルの名前を`+kwctl-windows-x86_64.exe+` から`+kwctl.exe+` に変更する。

. バイナリを`+PATH+` 環境変数がカバーする場所に移動する。

. インストールを確認する。新しいコマンドプロンプトまたはPowerShellウィンドウを開き、`+kwctl+` のインストールを確認する：
+
[source,console]
----
kwctl --version
----

== シェルコンプリートのインストール

`+kwctl+` CLIには、シェルの補完コマンドを生成する`+--shell+` オプションがあります。このコマンドの出力を使って、補完をシェルに組み込むことができる。

[source,console]
----
kwctl completions --shell [bash|elvish|fish|powershell|zsh]
----
