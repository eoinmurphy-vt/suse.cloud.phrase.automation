= Relações de recursos de cluster em {product_name}
:revdate: 2026-01-06
:page-revdate: {revdate}

== Visão geral

O Rancher integra vários componentes que definem seus próprios `+Cluster+` recursos personalizados. O{product_name} se baseia nessa fundação usando a API do Cluster e o Fleet, o que pode gerar confusão sobre como esses `+Cluster+` recursos interagem e o que os usuários devem editar.

Este documento explica a finalidade, o comportamento e os relacionamentos entre os seguintes recursos do cluster:

* `+clusters.cluster.x-k8s.io+` (Grupo CAPI)
* `+clusters.fleet.cattle.io+` (Grupo de Frotas)
* `+clusters.provisioning.cattle.io+` (Cluster de provisionamento)
* `+clusters.management.cattle.io+` (Grupo de gerenciamento)

== Resumo dos recursos

|===
| Grupo API | Recurso de cluster | Função

| `+cluster.x-k8s.io+`
| Cluster CAPI
| Define o ciclo de vida e a topologia do cluster. Atua como a fonte da verdade em Turtles.

| `+fleet.cattle.io+`
| Grupo de frotas
| Usado para direcionamento de pacotes GitOps e sincronização de aplicativos via Fleet.

| `+provisioning.cattle.io+`
| Cluster de provisionamento
| A abstração de cluster de uso geral anterior do Rancher. Encapsula a configuração independente de infraestrutura e específica do provedor e é necessário para gerar tokens e manifestos de registro de cluster. Em {product_name}, esse recurso foi usado para envolver clusters CAPI importados, mas está sendo eliminado gradualmente.

| `+management.cattle.io+`
| Cluster de gerenciamento
| Representação atual do cluster do Rancher. `+management.cattle.io/v3+` clusters são agora o recurso padrão criado e gerenciado pelo Turtles. Isso não deve ser confundido com clusters gerenciados por CAPI, em que "gerenciado" se refere ao gerenciamento do ciclo de vida da API do cluster, e não ao grupo de APIs `+management.cattle.io+` do Rancher.
|===

== Relacionamentos

=== Cluster CAPI → Cluster Fleet

Quando o Cluster API Add-on Provider for Fleet (CAAPF) está ativado, ele propaga rótulos e anotações do CAPI `+Cluster+` para o Fleet correspondente `+Cluster+`. Isso permite direcionar os pacotes GitOps por meio de seletores de rótulos. O CAAPF cria o recurso correspondente do Fleet `+Cluster+` depois que o CAPI `+Cluster+` atinge a condição `+ControlPlaneInitialized+`.

=== Cluster CAPI → Cluster de provisionamento (legado)

Um `+provisioning.cattle.io.Cluster+` pode ser criado para envolver o cluster CAPI e expor campos específicos do Rancher (por exemplo, nome de exibição, configuração do RKE). No entanto, na maioria dos fluxos de trabalho do site {product_name}, ele está sendo eliminado gradualmente e não é necessário que o usuário interaja com esse recurso.

=== Cluster de gerenciamento → Cluster de provisionamento

Para clusters antigos, um `+provisioning.cattle.io/v1.Cluster+` pode ser gerado a partir de um `+management.cattle.io/v3.Cluster+`. Esse objeto é usado pela interface do usuário e pelas APIs do Rancher e não se destina à modificação direta pelos usuários.
Os novos clusters gerenciados exclusivamente por Turtles não criam automaticamente um cluster `+provisioning.cattle.io+`.

== Comportamento de propagação

|===
| De → Para | Propagação | Notas

| CAPI → Frota
| Sim
| Propagado pela CAAPF

| CAPI → Provisionamento
| Não
| Não é criado automaticamente; só é gerado via Turtles para clusters legados

| CAPI → Gerenciamento
| Não
| Somente via Provisioning Cluster para clusters legados

| Provisionamento → Frota
| Não
| Não suportado

| Frota → CAPI
| Não
| Não suportado

| Gerenciamento → Provisionamento
| Condicional
| Somente para clusters legados; os novos clusters gerenciados pelo Turtles não geram clusters de provisionamento
|===

== O que os usuários devem editar?

=== Recomendado: `+cluster.x-k8s.io.Cluster+`

O cluster CAPI é o recurso oficial para a definição de clusters em {product_name}.
As etiquetas devem ser colocadas no objeto do cluster CAPI. Eles serão propagados automaticamente para o Fleet Cluster, permitindo que os seletores de pacotes baseados em rótulos funcionem conforme o esperado.

=== Condicional: `+provisioning.cattle.io.Cluster+`

Necessário somente ao usar os recursos de provisionamento nativos do Rancher (por exemplo, `+spec.rkeConfig+`, fluxos orientados pela interface do usuário). Evite editar, a menos que seja explicitamente necessário.

=== Evitar: `+fleet.cattle.io.Cluster+`

Os Fleet Clusters são gerenciados automaticamente. As edições são desencorajadas, a menos que se esteja trabalhando nos componentes internos do Fleet.

=== Evitar: `+management.cattle.io.Cluster+`

Esses recursos são internos e totalmente gerenciados pelo Rancher. Não os modifique.

[NOTE]
====
* O **Cluster CAPI** é a fonte da verdade.
* **Os Fleet Clusters** são gerados automaticamente a partir dos Clusters CAPI por meio do CAAPF.
* **O provisionamento** e **os clusters de gerenciamento** *não* são *necessários*, a menos que o provisionamento do Rancher esteja envolvido.
====

[TIP]
====
Use os rótulos do cluster CAPI para conduzir a implementação do Fleet bundle. O controlador CAAPF garante que esses rótulos sejam sincronizados com o objeto Fleet Cluster.
====
