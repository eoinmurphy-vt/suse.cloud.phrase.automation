= Guia de migração
:revdate: 2025-11-14
:page-revdate: {revdate}

== Contexto

A partir do Rancher v2.13, o {product_name} é instalado como parte do Rancher, eliminando a necessidade de uma implantação separada. Antes de fazer o upgrade para o Rancher v2.13, os usuários devem seguir este guia para garantir um processo de upgrade tranquilo.

== Migração

Siga as etapas abaixo, em ordem, _antes de_ fazer upgrade para o Rancher v2.13:

. Certifique-se de atualizar o site {product_name} para a versão 0.24.3. Essa etapa é importante porque todas as versões de patch da v0.24 preparam o cluster para a atualização para o Rancher v2.13.
. Desinstalar o site {product_name}. Isso é necessário porque, a partir do Rancher v2.13, o {product_name} não é mais um complemento opcional, mas um componente do sistema Rancher. Dessa forma, o Rancher o instala em um namespace diferente do anterior e, portanto, a versão instalada anteriormente precisa ser desinstalada manualmente primeiro. A desinstalação da versão 0.24.3 do site {product_name} remove o provedor principal do CAPI, mas mantém os CRDs principais do CAPI. Durante a migração, o controlador CAPI principal não estará em execução no cluster de gerenciamento, mas todos os clusters CAPI downstream devem continuar a ser executados normalmente e não serão afetados.
. Execute os seguintes comandos do kubectl:
+
[source,bash]
----
kubectl patch crd capiproviders.turtles-capi.cattle.io --type=json -p='[{"op": "add", "path": "/metadata/annotations/meta.helm.sh~1release-namespace", "value": "cattle-turtles-system"}]'
kubectl patch crd clusterctlconfigs.turtles-capi.cattle.io --type=json -p='[{"op": "add", "path": "/metadata/annotations/meta.helm.sh~1release-namespace", "value": "cattle-turtles-system"}]'
----

Esses comandos corrigirão os CRDs `+capiprovider+` e `+clusterctlconfig+` para que sejam adotados por {product_name} quando instalados como parte do Rancher v2.13. 

Observe que a migração descrita acima é necessária para os usuários que executam as versões {product_name} &le; v0.24.3. Os novos usuários que estão instalando o Rancher v2.13 não precisam realizar nenhuma migração.
