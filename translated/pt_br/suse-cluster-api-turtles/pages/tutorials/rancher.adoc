== Instalação do Rancher

_Se você já estiver executando o Rancher v2.13 ou posterior, pode pular esta seção e ir para xref:tutorials/first-cluster.adoc[Your First Cluster (Seu primeiro cluster])._

[NOTE]
====
A partir do Rancher v2.13, o {product_name} é pré-instalado como um gráfico do sistema Rancher. Não há mais necessidade de instalar o {product_name} separadamente. A simples instalação do Rancher v2.13 ou posterior é tudo o que é necessário para começar a usar a API do cluster (CAPI).

====

O Helm é a maneira recomendada de instalar o Rancher em um cluster Kubernetes novo ou existente.

ifeval::["{build-type}" == "product"]
[TIP]
====
Certifique-se de seguir um dos oficiais do https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/installation-and-upgrade.html[guias de instalação] para o Rancher.
====
endif::[]
ifeval::["{build-type}" == "community"]
[TIP]
====
Certifique-se de seguir um dos oficiais do https://ranchermanager.docs.rancher.com/getting-started/installation-and-upgrade[guias de instalação] para o Rancher.
====
endif::[]

Aqui está um exemplo mínimo de instalação do Rancher com o Helm:

[source,bash]
----
helm install rancher rancher-stable/rancher \
    --namespace cattle-system \
    --create-namespace \
    --set hostname=<rancher-hostname> \
    --version <rancher-version> \
    --wait
----

Substitua `+<rancher-hostname>+` pelo nome real do host do seu servidor Rancher e use `+--version+` para especificar uma versão do Rancher de `+v2.13.0+` ou posterior.

== O que é instalado

Quando o Rancher v2.13 ou posterior é instalado, o {product_name} é implantado automaticamente como um componente do sistema no namespace `+cattle-turtles-system+`. Isso inclui:

* O pod `+rancher-turtles-controller-manager+`, que lida com todos os recursos relacionados ao CAPI
* CRDs principais do CAPI e o controlador principal do CAPI
* O recurso personalizado `+CAPIProvider+`, que permite a definição declarativa de provedores CAPI

[NOTE]
====
O manifesto do controlador CAPI principal está incorporado em um ConfigMap, o que simplifica a operação em ambientes com isolamento acústico.
====

== Instalação de provedores CAPI

Com o {product_name} incorporado ao Rancher, os provedores de CAPI não são mais empacotados por padrão. Os provedores são gerenciados declarativamente usando o recurso personalizado `+CAPIProvider+`, e um gráfico de provedores separado está disponível para a instalação de provedores certificados. Para obter mais informações sobre a tabela de provedores, consulte xref:user/providers.adoc[Instalar provedores certificados].

Por exemplo, para definir um provedor:

[source,yaml]
----
apiVersion: turtles-capi.cattle.io/v1alpha1
kind: CAPIProvider
metadata:
  name: aws
  namespace: cattle-turtles-system
spec:
  type: infrastructure
  name: aws
----

Consulte a xref:reference/capiprovider.adoc[referência do CAPIProvider] para obter a lista completa de opções de configuração.

Agora você está pronto para usar o {product_name}! 🎉
