= ClusterctlConfig
:revdate: 2025-11-14
:page-revdate: {revdate}

O recurso `+ClusterctlConfig+` permite gerenciar substituições de configurações do clusterctl (Operador CAPI) de forma declarativa. É usado para configurar provedores de clusterctl e seus URLs, bem como restrições de versão.

`+ClusterctlConfig+` segue um modelo GitOps - os campos de especificação são entradas declarativas do usuário. O Turtles não cria nem atualiza o recurso; cabe ao usuário especificar as substituições de url do provedor e manter seu estado. Ele tem precedência sobre os padrões incorporados ou sobre o conjunto padrão de definições de provedor do site `+clusterctl+`.

https://github.com/rancher/turtles/blob/main/docs/adr/0012-clusterctl-provider.md[ADR]

== Uso

Para usar o recurso `+ClusterctlConfig+`:

. Crie um recurso `+ClusterctlConfig+` com o nome `+clusterctl-config+` no espaço de nomes `+turtles+`.
. O controlador `+ClusterctlConfig+` tratará das atualizações para o `+ConfigMap+` montado no `+cluster-api-operator+` com a configuração de clusterctl necessária com base na especificação `+ClusterctlConfig+`.
. Gerencie o objeto `+ClusterctlConfig+` de forma declarativa para aplicar alterações às configurações de provedor geradas. Pode levar algum tempo para que as alterações entrem em vigor, pois o site `+kubelet+` se encarrega de atualizar o ponto montado com base no estado do site `+ConfigMap+`.

Aqui está um exemplo do manifesto `+ClusterctlConfig+`:

[,yaml]
----
apiVersion: turtles-capi.cattle.io/v1alpha1
kind: ClusterctlConfig
metadata:
  name: clusterctl-config
  namespace: cattle-turtles-system
spec:
  providers:
  - name: metal3
    url: https://github.com/metal3-io/cluster-api-provider-metal3/releases/v1.10.1/infrastructure-components.yaml
    type: InfrastructureProvider
  images:
  - name: metal3
    repository: "my.registry.mirror"
    tag: customTag
----

Este exemplo gerará uma configuração de clusterctl para o provedor `+metal3+` com o URL e o tipo especificados.

== Especificação

Os principais campos da especificação `+ClusterctlConfig+` são:

* `+providers[].name+` - Nome do provedor (por exemplo, metal3)
* `+providers[].url+` - URL da configuração do provedor (por exemplo
https://github.com/metal3-io/cluster-api-provider-metal3/releases/v1.10.1/infrastructure-components.yaml). Isso pode usar a versão `+latest+`, se compatível, ou fixar a versão máxima em `+v1.10.1+`, por exemplo.
* `+providers[].type+` - Tipo do provedor (por exemplo, InfrastructureProvider)

* `+images[].name+` - O nome da imagem a ser substituída (por exemplo, infrastructure-metal3, cluster-api, bootstrap-rke2, control-plane-rke2)
* `+images[].repository+` - Define a substituição do registro de contêiner para extrair imagens de
* `+images[].tag+` - Permite especificar uma tag para as imagens

== Substituir uma versão de provedor certificado

{product_name} incorpora uma configuração validada em que todas as versões de provedores certificados são fixadas. +
Normalmente, as versões são atualizadas com qualquer lançamento do site {product_name}.

[WARNING]
====
- {product_name} valida apenas a configuração incorporada para provedores certificados. Qualquer personalização da configuração incorporada pode levar a incompatibilidades e cenários sem suporte. 
====

Para substituir a versão de um provedor ou usar um registro diferente para extrair imagens, é possível definir um `+ClusterctlConfig+`. +
Para obter mais informações sobre a substituição de imagens, consulte a link:https://cluster-api.sigs.k8s.io/clusterctl/configuration#image-overrides[documentação do site].

[source,yaml]
----
apiVersion: turtles-capi.cattle.io/v1alpha1
kind: ClusterctlConfig
metadata:
  name: clusterctl-config
  namespace: cattle-turtles-system
spec:
  providers:
  - name: rke2
    url: "https://github.com/rancher/cluster-api-provider-rke2/releases/v0.20.1/control-plane-components.yaml"
    type: ControlPlaneProvider
  - name: rke2
    url: "https://github.com/rancher/cluster-api-provider-rke2/releases/v0.20.1/bootstrap-components.yaml"
    type: BootstrapProvider
  images:
  - name: control-plane-rke2
    repository: "ghcr.io/rancher"
  - name: bootstrap-rke2
    repository: "ghcr.io/rancher"
----

== Exclusão

Quando um recurso `+ClusterctlConfig+` é excluído, o mapa de configuração é revertido para seu estado original, gerenciado por tartarugas.
