== Componentes

Abaixo está uma representação visual dos componentes da arquitetura do Rancher Turtles. Este diagrama ilustra os principais elementos e suas relações no sistema {product_name}. Compreender esses componentes é essencial para obter insights sobre como o Rancher utiliza a API de cluster (CAPI) para o gerenciamento de clusters.

image::30000ft_view.png[visão geral,640,480]

=== Gerente de Rancheiro

Esse é o componente principal do Rancher e os usuários podem aproveitar o recurso Explorer existente no painel para acessar os detalhes da carga de trabalho do cluster.

=== Agente de cluster do Rancher

O agente é implantado em clusters secundários, permitindo que o Rancher importe e estabeleça uma conexão com esses clusters. Essa conexão permite que o Rancher gerencie os clusters filhos de forma eficaz a partir de sua plataforma.

=== {product_name} - Extensão CAPI do Rancher

Ele fornece integração entre o CAPI e o Rancher e, no momento, oferece suporte às seguintes funcionalidades:

* *Importação de clusters CAPI para o Rancher:* instalação do Rancher Cluster Agent em clusters provisionados CAPI.
* *Configuração do operador CAPI:* Suporte de configuração para o Operador CAPI.

== Topologia de implantação

=== Rancher Manager e CAPI Management combinados

Essa é a única topologia suportada atualmente, em que o Rancher Manager e o {product_name} são implantados no mesmo cluster do Kubernetes e atuam como um cluster de gerenciamento centralizado.

Essa arquitetura oferece uma implementação simplificada de componentes e fornece uma visão única de todos os clusters. Por outro lado, é importante considerar que o número de clusters que podem ser gerenciados com eficiência pela API de cluster (CAPI) é limitado pelos recursos disponíveis no único cluster de gerenciamento.
