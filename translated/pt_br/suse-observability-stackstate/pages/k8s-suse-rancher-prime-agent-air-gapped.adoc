= Instalação do SUSE Observability Agent no modo Air-Gapped
:revdate: 2025-07-18
:page-revdate: {revdate}
:description: Observabilidade do SUSE

Este documento fornece um guia passo a passo para a instalação do SUSE Observability Agent usando os gráficos Helm em um ambiente air-gapped. O processo de instalação envolve a preparação das imagens do Docker e do Helm chart necessários em um host com acesso à Internet, sua transferência para um host em uma rede privada, o upload das imagens do Docker para um registro privado e a implantação dos Helm charts. Os privilégios necessários para instalar o agente podem ser encontrados xref:/k8s-suse-rancher-prime.adoc#_required_privileges[aqui]

== Pré-requisitos

=== No host local (acesso à Internet)

* *Sistema operacional*: Linux ou MacOS
* *Ferramentas instaladas*:
 ** https://www.docker.com/products/docker-desktop/[Docker]
 ** https://helm.sh/docs/intro/install/[Clipe do leme]
 ** Scripts para download de imagens do Docker do registro de origem (os links serão fornecidos posteriormente no guia).
* *Acesso à Internet*: Necessário para extrair imagens do Docker do Quay.io e gráficos do Helm do ChartMuseum.

=== No host da rede privada

* *Acesso*: Acesso SSH ao host.
* *Ferramentas instaladas*:
 ** https://www.docker.com/products/docker-desktop/[Docker]
 ** https://helm.sh/docs/intro/install/[Clipe do leme]
 ** Scripts para download de imagens do Docker do registro de origem (os links serão fornecidos posteriormente no guia).
 ** Acesso à rede e credenciais para carregar imagens em um registro privado do Docker.
 ** Um Kubeconfig configurado para instalar os gráficos do Helm nos clusters de destino.

== Preparação das imagens do Docker e do Helm Chart

Execute os seguintes comandos no host local para obter as imagens do Docker e os gráficos do Helm necessários:

*Adição de repositórios Helm ao cache local do Helm:*

[,bash]
----
# Adding the Helm repository for the SUSE Observability Agent
helm repo add suse-observability https://charts.rancher.com/server-charts/prime/suse-observability
helm repo update
----

*Obtenção da versão mais recente do gráfico*

O comando a seguir fará o download de um arquivo TGZ do gráfico do repositório do Helm:

[,bash]
----
# Downloading the chart for the SUSE Observability Agent
# The file will be named stackstate-agent-X.Y.Z.tgz
helm fetch suse-observability/suse-observability-agent
----

*Obter os scripts do Bash para salvar imagens do Docker.*

[,bash]
----
# o11y-agent-get-images.sh
curl -LO https://raw.githubusercontent.com/StackVista/helm-charts/master/stable/suse-observability-agent/installation/o11y-agent-get-images.sh
# o11y-agent-save-images.sh
curl -LO https://raw.githubusercontent.com/StackVista/helm-charts/master/stable/suse-observability-agent/installation/o11y-agent-save-images.sh

# Make the scripts executable
chmod a+x o11y-agent-get-images.sh o11y-agent-save-images.sh
----

*Extração e salvamento de imagens do Docker*

[,bash]
----
# Extract the list of images from the Helm chart and save it to a file.
./o11y-agent-get-images.sh -f suse-observability-agent-X.Y.Z.tgz > o11y-agent-images.txt
----

[NOTE]
====
Substitua `+suse-observability-agent-X.Y.Z.tgz+` pelo nome de arquivo real do arquivo de gráfico baixado anteriormente.*
====


[,bash]
----
# Save Docker images to an archive.
# The script expects the file o11y-agent-images.txt to contain the list of images used by the SUSE Observability Agent.
# The Docker images will be saved to o11y-agent-images.tar.gz.
./o11y-agent-save-images.sh -i o11y-agent-images.txt -f o11y-agent-images.tar.gz
----

== Cópia dos arquivos necessários para o host remoto

Os arquivos a seguir devem ser copiados do host local para o host na rede privada:

* o11y-agent-images.txt (Lista de imagens exigidas pelo gráfico do SUSE Observability Agent)
* o11y-agent-images.tar.gz (Um arquivo com as imagens do Docker do SUSE Observability Agent)
* https://raw.githubusercontent.com/StackVista/helm-charts/master/stable/suse-observability-agent/installation/o11y-agent-load-images.sh[o11y-agent-load-images.sh] (script Bash para carregar imagens do Docker em um registro)
* Gráficos do leme baixados anteriormente:
 ** suse-observability-agent-X.Y.Z.tgz

== Restauração de imagens do Docker do arquivo para o registro privado

*Carregamento das imagens para o registro privado:*

[,bash]
----
# Load Docker images from the archive and push them to the private registry.
# Replace <private-registry> with your private registry's URL.
export DST_REGISTRY_USERNAME="..."
export DST_REGISTRY_PASSWORD="..."
./o11y-agent-load-images.sh -d registry.example.com:5043 -i o11y-agent-images.txt -f o11y-agent-images.tar.gz
----

[NOTE]
====
_Observação: se o registro de destino não usar autenticação, as variáveis de ambiente `+DST_REGISTRY_USERNAME+` e `+DST_REGISTRY_PASSWORD+` não devem ser configuradas ou devem ser definidas com valores vazios._
====


== Instalação do agente de observabilidade do SUSE

O comando para instalar o SUSE Observability Agent deve ser recebido da SUSE Observability UI.
Faça login na sua instância e, no menu do lado esquerdo, escolha `+Stackpacks+`. Pressione `+ADD NEW INSTANCE+` e preencha o nome do cluster. Para a integração correta com o Rancher, ele precisa ser igual ao nome do cluster na interface do usuário do Rancher.

image::rancher-prime-agent-airgap-01.png[Adição de novo agente]

Quando uma instância for adicionada, a interface do usuário fornecerá as instruções sobre como o gráfico do Helm pode ser implementado. Role para baixo até a seção `+Self-hosted+` e copie o comando `+helm upgrade ...+` 

image::rancher-prime-agent-airgap-02.png[Obtendo o comando de instalação do Helm]

O comando deve ser atualizado para a instalação com porta de ar:

* Substituir o registro de imagem pelo valor `+all.image.registry+`.
* Usando o arquivo com o gráfico do Helm em vez do repositório do Helm. `+suse-observability/suse-observability-agent+` \-> `+./suse-observability-agent-X.Y.Z.tgz+`

Execute o comando para instalar o SUSE Observability Agent

[,bash]
----
helm upgrade --install \
--namespace suse-observability \
--create-namespace \
--set-string 'stackstate.apiKey'='<api-key>' \
--set-string 'stackstate.cluster.name'='<cluster-name>' \
--set-string 'stackstate.url'='https://...' \
--set 'nodeAgent.skipKubeletTLSVerify'=true \
--set-string 'all.image.registry'='registry.acme.com:5000' \
--set-string 'global.imageRegistry'='registry.acme.com:5000' \
--set-string 'global.skipSslValidation'=true \
suse-observability-agent ./suse-observability-agent-X.Y.Z.tgz
----

*Validação da implantação*

[,bash]
----
kubectl get pod -n suse-observability
----
