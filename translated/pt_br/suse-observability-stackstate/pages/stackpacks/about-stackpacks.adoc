== Visão geral

Os StackPacks ampliam a funcionalidade do SUSE Observability e oferecem integração automatizada com sistemas externos. Eles podem ser facilmente instalados e desinstalados na página StackPacks do SUSE Observability.

Há dois tipos de StackPack: complementos e integrações. Os complementos e as integrações disponíveis do StackPack podem ser encontrados na página *StackPacks* na interface do usuário do SUSE Observability.

== Instalar ou desinstalar um StackPack

Os StackPacks podem ser instalados e desinstalados na página *StackPacks* no SUSE Observability. São fornecidas instruções completas para instalar e desinstalar o StackPack.

[NOTE]
====
*Observe que:*

* Um StackPack pode exigir etapas de instalação *manual* ou configuração do sistema externo. Leia atentamente as instruções fornecidas.
* Quando um StackPack ou uma instância do StackPack for desinstalado, *todos os dados recebidos por meio do StackPack (instância) serão removidos do SUSE Observability.*
* Qualquer configuração (manual) ou instalação de componentes do StackPack em um sistema externo pode precisar ser desinstalada separadamente.

====


=== Instâncias do StackPack

Alguns StackPacks permitem que você se conecte a várias contas em um sistema externo. Cada conta é configurada em uma instância separada do StackPack.

Por exemplo, o AWS StackPack pode se conectar a várias contas do AWS e combinar informações de todas as contas no SUSE Observability. Para cada conta, uma instância separada do SUSE Observability é configurada com as informações necessárias para receber dados dessa conta do AWS.

== Fazer upgrade de um StackPack

[CAUTION]
====
Quando um StackPack é atualizado, *todas as alterações feitas nos itens de configuração desse StackPack serão substituídas*. Para obter detalhes, consulte xref:/stackpacks/about-stackpacks.adoc#_locked_configuration_items[itens de configuração bloqueados].
====


Se uma nova versão do StackPack estiver disponível, um alerta será exibido na página do SUSE Observability UI StackPack e você terá a opção de visualizar os detalhes da atualização e ler as notas de versão. Se a nova versão for uma pequena atualização da versão atualmente instalada, você também poderá atualizar o StackPack a partir daqui.

=== Nova versão secundária do StackPack

Para atualizar para uma nova versão secundária de um StackPack, clique em *UPGRADE NOW* na página do StackPack na interface do usuário do SUSE Observability.

Observe que todos os itens de configuração do StackPack serão sobrescritos quando você fizer o upgrade. Para continuar usando as alterações feitas, opte por *MANTER* a configuração existente ao atualizar o StackPack. Para obter detalhes, consulte xref:/stackpacks/about-stackpacks.adoc#_locked_configuration_items[itens de configuração bloqueados].

=== Nova versão principal do StackPack

[CAUTION]
====
Quando um StackPack é atualizado, *todas as alterações feitas nos itens de configuração desse StackPack serão substituídas*. Para obter detalhes, consulte xref:/stackpacks/about-stackpacks.adoc#_locked_configuration_items[itens de configuração bloqueados].
====


Para atualizar para uma nova versão principal de um StackPack, xref:/stackpacks/about-stackpacks.adoc#_install_or_uninstall_a_stackpack[desinstale e reinstale] o StackPack.

[TIP]
====

Continuar usando as alterações feitas nos itens de configuração personalizados do StackPack após a atualização:

. Antes de fazer o upgrade, exporte cada item personalizado:
 ** Vá para a página *Settings (Configurações* ) na interface do usuário do SUSE Observability.
 ** Clique em *Exportar* no menu *...* para cada item personalizado.
. Atualize o StackPack (desinstale e reinstale).
. Altere os endereços `+name+` e `+identifier+` para cada item exportado:
 ** Abra o arquivo de exportação em um editor de texto.
 ** Edite os campos de nível superior `+name+` e `+identifier+`.
 ** Salve a exportação.
. xref:/setup/data-management/backup_restore/configuration_backup.adoc#_import_configuration[Importe] o(s) arquivo(s) de exportação atualizado(s).
. Os itens de configuração personalizados agora estarão disponíveis no SUSE Observability e poderão ser copiados para os itens de configuração do StackPack recém-instalados.

====


== Itens de configuração bloqueados

Os StackPacks contêm informações de configuração para o SUSE Observability que são instaladas quando o StackPack (instância) é instalado. Entre outras coisas, isso pode ser modelos de componentes, funções, ações de componentes e visualizações. Quando um StackPack é atualizado, *os itens de configuração instalados pela versão anterior do StackPack serão substituídos pelos itens do StackPack mais recente.* Isso significa que qualquer alteração manual feita nesses itens de configuração será substituída quando o StackPack for atualizado.

Para evitar que um usuário faça alterações nos itens de configuração instalados por um StackPack que serão substituídos na atualização, esses itens de configuração são *bloqueados* por padrão. Isso significa que eles estão protegidos contra alterações pelo usuário e devem ser explicitamente *desbloqueados* para que possam ser alterados.

[TIP]
====

Você pode xref:/setup/data-management/backup_restore/configuration_backup.adoc[fazer um backup dos itens de configuração]. Observe que o status de bloqueio dos itens de configuração não será exportado como parte de um backup de configuração.
====
