= Variáveis do painel
:revdate: 2025-11-24
:page-revdate: {revdate}
:description: Use variáveis para criar painéis dinâmicos.

Ao configurar variáveis para um painel, você pode criar painéis com widgets dinâmicos. As variáveis permitem a reutilização de uma única definição de painel para diferentes casos de uso. 

== Visualização das variáveis do painel
Na barra de navegação superior, use o menu suspenso ao lado do botão "Save..." (Salvar ) e clique em "Edit dashboard variables" (Editar variáveis do painel).

image::dashboards/open-variables-drawer.png[Adicionar todos os itens fixados]

=== Variáveis incorporadas
Há variáveis que são incorporadas ao {stackstate-product-name}. Os nomes dessas variáveis são reservados, e você não pode dar os mesmos nomes às suas próprias variáveis.

Para visualizar todas as variáveis incorporadas, clique no acordeão intitulado "SUSE Observability PromQL variables".

== Adição de uma nova variável

. Abra a gaveta "Edit dashboard variables" (Editar variáveis do painel). 
. Clique no botão "Add variable" (Adicionar variável) para abrir uma segunda gaveta onde você pode configurar a nova variável.

image::dashboards/variable-drawer-example.png[Um exemplo de variável]

Os campos de configuração de uma variável são os seguintes:

|===
| Nome do campo | Descrição

| Nome
| O identificador de nome da variável. Esse é o nome que você pode <<using-variables, usar em campos compatíveis>>.

| Exibir etiqueta
| Um nome amigável para a variável que é mostrado no painel ao <<selecting-variables, selecionar a variável>>. Isso ajuda a identificar a variável.

| Descrição
| Uma descrição para entender a finalidade da variável. Essa descrição é mostrada na gaveta "Editar variáveis do painel".

| Tipo
| O tipo da variável.
|===

== Tipos de variáveis
=== Variável de texto

Uma variável de texto contém texto simples. Você pode definir um valor inicial e tornar a variável somente leitura.

=== Variável de lista

Uma variável de lista tem valores possíveis que são definidos pelo campo "Source" (Fonte). Cada tipo de fonte fornece campos adicionais que permitem restringir os valores possíveis para a variável.

|===
| Tipo de fonte | Campos de origem

| Nomes de etiquetas métricas
| Insira zero ou mais seletores de série para refinar quais nomes de rótulos de métricas estão presentes.

| Valores de rótulos métricos
| Insira um nome de rótulo métrico e zero ou mais seletores de série para refinar quais valores de rótulo métrico estão presentes.

| Métrica PromQL
| Insira uma consulta PromQL personalizada e um nome de rótulo. Os valores possíveis resultantes são os valores para o nome do rótulo fornecido.

| Lista estática
| Insira um ou mais pares de rótulos e valores para criar uma lista estática de valores possíveis.
|===

À medida que você altera o tipo de fonte e seus campos relacionados, a visualização no lado direito do gaveteiro será atualizada com os novos valores possíveis.


==== Configurações comuns de variáveis de lista

Todas as fontes de variáveis de lista também têm as seguintes configurações comuns:

|===
| Nome do campo | Descrição

| Classificar
| Classifique os valores na lista. Por exemplo, em ordem alfabética ou numérica.

| Permitir vários valores
| Permita que os usuários selecionem mais de um valor por vez.

| Ativar "Incluir tudo"
| Adiciona um item para selecionar todos os valores à lista que o usuário vê. Opcionalmente, você pode especificar um valor personalizado para esse item ou permitir que ele seja gerado automaticamente. 
|===

== Uso de variáveis [[using-variables]]
Depois que uma variável é definida, ela pode ser usada para preencher vários campos no painel:

* Outras variáveis
* Nome e descrição do widget
* Campos de links de widgets: URL, nome e dica de ferramenta
* Campos de consulta do widget: Expressão PromQL, alias e etapa mínima
* Editor do widget Markdown

Para usar uma variável em um campo compatível, use o nome da variável com `+${}+` (por exemplo, `+$\{my_variable}+`). O valor da variável é interpolado.

=== Como as variáveis são interpoladas
As variáveis de texto são interpoladas como a cadeia de caracteres que está escrita na caixa de texto da variável.

Da mesma forma, as variáveis de lista que não permitem vários valores nem "Incluir tudo" são interpoladas como o valor do item selecionado da lista.

As variáveis de lista que permitem vários valores são interpoladas como um grupo de captura de expressão regular dos valores selecionados (por exemplo, `+(value_1|value_2)+`), independentemente de uma ou várias opções serem selecionadas.

As variáveis de lista que têm a opção "Include Everything" ativada e selecionada para essa variável serão interpoladas como:* O valor personalizado, se definido, ou Uma expressão regular gerada que captura o grupo de todos os valores possíveis (por exemplo, `+(value_1|value_2|value_3)+`)

Você pode usar o matcher de expressão regular do PromQL `+=~+` ao consumir variáveis que são interpoladas como grupos de captura de expressão regular.

== Seleção de variáveis [[selecting-variables]]
As variáveis são listadas na parte superior do painel, ao lado do seletor de intervalo de tempo. Somente as primeiras variáveis são mostradas. Para ver o restante das variáveis, use o menu suspenso "More" (Mais).

Clique em uma variável para abrir seu seletor e selecionar os valores que deseja mostrar no painel.

À medida que você selecionar valores diferentes, os widgets que usam a variável reagirão às alterações.
