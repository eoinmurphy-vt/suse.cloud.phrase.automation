= Solução de problemas de mapeamentos de telemetria aberta
:revdate: 2025-11-26

:description:  Solução de problemas de mapeamentos de telemetria abertos

== Visão geral

A xref:/setup/cli/cli-sts.adoc[CLI do SUSE Observability] pode ser usada para solucionar problemas de sincronização de topologia do Open Telemetry e corrigir problemas que possam impedir que os dados de topologia sejam corretamente ingeridos e exibidos em {stackstate-product-name}. Esta página descreve as etapas gerais de solução de problemas a serem seguidas ao depurar esse tipo de problema.

== Etapas gerais de solução de problemas

. xref:/setup/custom-integrations/otelmappings/troubleshooting.adoc#_list_open_telemetry_component_mappings[Verifique se o mapeamento Open Telemetry existe].
** Se nenhum mapeamento estiver presente, verifique a instalação do stackpack do Open Telemetry.
. Quando faltarem componentes ou relações de um determinado tipo, xref:/setup/custom-integrations/otelmappings/troubleshooting.adoc#_show_an_open_telemetry_component_or_relation_mappings_status[obtenha o status do mapeamento].
** Verifique as métricas de latência para determinar se os dados estão fluindo ou se há uma falha de comunicação entre o coletor do Open Telemetry e a plataforma.
** Corrija os erros de mapeamento relatados, pois eles estão impedindo a sincronização de alguns elementos da topologia.
** Uma possível fonte de mapeamentos que "silenciosamente" não estão sendo processados como esperado pode ser as condições/filtragem aplicadas no campo de entrada. Por exemplo, se uma condição for falsa, não haverá erro, mas o mapeamento será "pulado".

=== Mostrar um componente Open Telemetry ou status de mapeamento de relação

O comando Open Telemetry component mapping status retorna a latência agregada, as métricas de throughput e o número de elementos de topologia criados. Isso é útil ao depurar o motivo pelo qual uma parte específica da topologia leva muito tempo para ser sincronizada. A saída inclui uma seção `+Mapping errors+` que sinalizará qualquer problema que ocorra ao aplicar as regras de mapeamento aos dados do Open Telemetry.

[,sh]
----
$ sts otel-component-mapping status --identifier urn:stackpack:open-telemetry-2:shared:otel-component-mapping:service


Otel Component Mapping:
NAME                  | IDENTIFIER                                                           | COMPONENTS | RELATIONS
OTel Service Instance | urn:stackpack:open-telemetry-2:shared:otel-component-mapping:service | 9          | 0


Otel Component Mapping Metrics:
METRIC          | 500S AGO | 500-1000S AGO | 1000-1500S AGO
latency seconds | 44.45    | 44.45         | 41.19
----

Da mesma forma, há um status de mapeamento da relação Open Telemetry
[,sh]
----
$ sts otel-relation-mapping status --identifier urn:stackpack:open-telemetry-2:shared:otel-relation-mapping:database


Otel Relation Mapping:
NAME              | IDENTIFIER                                                           | COMPONENTS | RELATIONS
Database Relation | urn:stackpack:open-telemetry-2:shared:otel-relation-mapping:database | 0          | 2


Otel Relation Mapping Metrics:
METRIC          | 500S AGO | 500-1000S AGO | 1000-1500S AGO
latency seconds | 43.404   | 43.404        | 39.978


Otel Relation Mapping Errors:
No otel relation mapping errors found.
----

=== Obtenha o componente ou mapeamento completo do Open Telemetry

Para visualizar rapidamente a definição completa de mapeamento de um componente ou mapeamento de relação, use o comando `+get+`:

[,sh]
----
$ sts otel-component-mapping get --identifier urn:stackpack:open-telemetry-2:shared:otel-component-mapping:service
# Output omitted for brevity
----

Ou para mapeamentos de relações:

[,sh]
----
$ sts otel-relation-mapping get --identifier urn:stackpack:open-telemetry-2:shared:otel-relation-mapping:database
# Output omitted for brevity
----

=== Editar um componente Open Telemetry ou mapeamento de relação

Se você notar problemas na definição de mapeamento, edite-a diretamente usando o comando `+edit+`. Isso abrirá o mapeamento em seu editor de texto padrão.

[,sh]
----
$ sts otel-component-mapping edit --identifier urn:stackpack:open-telemetry-2:shared:otel-component-mapping:service
# Opens the mapping in your default text editor
----

Ou para mapeamentos de relações:

[,sh]
----
$ sts otel-relation-mapping edit --identifier urn:stackpack:open-telemetry-2:shared:otel-relation-mapping:database
# Opens the mapping in your default text editor
----

Se o mapeamento tiver sido modificado, as alterações serão aplicadas quando você salvar e sair do editor.

[NOTE]
====
Lembre-se de atualizar as definições de mapeamento em seus arquivos de origem do StackPack para mantê-las sincronizadas com as alterações feitas por meio da CLI.
====
