== Visão geral

Esta página descreve os esquemas de um `+Monitor+`, juntamente com explicações detalhadas de construções, sintaxe de expressão e semântica.

== Esquema para o Monitor

Um monitor especifica a frequência com que uma função de monitor deve ser executada e com quais argumentos.

[,yaml]
----
_type: "Monitor"
name: string
description?: string
function: string
arguments:
  <dependent on monitor function>
intervalSeconds: integer
remediationHint?: string
status?: "ENABLED" | "DISABLED"       # defaults to "DISABLED"
tags:
  <key>: <value>
identifier?: string
----

* `+_type+`: {stackstate-product-name} precisa saber que se trata de um monitor, portanto, o valor sempre precisa ser `+Monitor+`
* `+name+`: O nome do monitor
* `+description+`: Uma descrição do monitor.
* `+function+`: Uma referência à função de monitor que executará o monitor.
* `+intervalSeconds+`: O intervalo em que o monitor é executado. Para métricas regulares em tempo real, recomenda-se 30 segundos. Para consultas de métricas analíticas de execução mais longa, recomenda-se um intervalo maior.
* `+remediationHint+`: Uma descrição do que o usuário pode fazer quando o monitor falha. O formato é markdown, com o uso opcional de variáveis de guidão para personalizar a dica com base em séries temporais ou outros dadosxref:use/alerting/k8s-add-monitors-cli.adoc#_write_the_remediation_hint[(mais explicações abaixo]).
* `+status+`: `+"DISABLED"+` ou `+"ENABLED"+`. Determina se o monitor será executado ou não.
* `+tags+`: Adicione tags ao monitor para ajudar a organizá-las na visão geral dos monitores de sua instância do {stackstate-product-name}, http://your-instance/#/monitors
* `+identifier+`: Um identificador do formato `+urn:stackpack:<stackpack-name>:monitor:....+` que identifica exclusivamente o monitor ao atualizar sua configuração.

== Funções do monitor

=== Limite

Aciona um estado de integridade quando um determinado limite é excedido para uma consulta de métrica especificada. Limites diferentes podem ser definidos em recursos específicos com xref:/use/alerting/k8s-override-monitor-arguments.adoc[a ajuda de anotações].

[,yaml]
----
function: {{ get "urn:stackpack:common:monitor-function:threshold" }}
arguments:
  metric:
    query: string
    unit: string
    aliasTemplate: string
  comparator: GTE | GT | LTE | LT     # how to compare metric value to threshold
  threshold: double
  failureState: CRITICAL | DEVIATING | UNKNOWN
  urnTemplate: string
  titleTemplate: string
----

* `+query+`: Uma consulta PromQL. Use o xref:/use/metrics/k8sTs-explore-metrics.adoc[explorador de métricas] de sua instância {stackstate-product-name}, http://your-instance/#/metrics, e use-o para construir uma consulta para a métrica de interesse.
* `+unit+`: A unidade dos valores na série temporal retornada pela consulta ou consultas, usada para renderizar o eixo Y do gráfico. Consulte a referência de xref:/develop/reference/k8sTs-chart-units.adoc[unidades suportadas] para todas as unidades.
* `+aliasTemplate+`: Um alias para séries temporais no gráfico de métricas. Esse é um modelo que pode substituir rótulos da série temporal usando o espaço reservado `+${my_label}+`.
* `+comparator+`: Escolha uma das opções LTE/LT/GTE/GT para comparar o limite com a métrica. As séries temporais para as quais `+<metric> <comparator> <threshold>+` é verdadeiro produzirão o estado de falha.
* `+threshold+`: Um limite numérico para comparação.
* `+failureState+`: Ou "CRÍTICO" ou "DESVIADO". "CRÍTICO" será exibido em {stackstate-product-name} e "DESVIADO" em laranja, para indicar diferentes graus de gravidade.
* `+urnTemplate+`: Um modelo para construir a urna do componente xref:use/alerting/k8s-add-monitors-cli.adoc#_bind_the_results_of_the_monitor_to_the_correct_components[ao qual] o resultado do monitor será xref:use/alerting/k8s-add-monitors-cli.adoc#_bind_the_results_of_the_monitor_to_the_correct_components[vinculado].
* `+titleTemplate+`: Um título para o resultado de um monitor. Como vários resultados do monitor podem ser vinculados ao mesmo componente, é possível substituir rótulos de séries temporais usando o espaço reservado `+${my_label}+`.

=== Estado derivado

Deriva seu estado das dependências dos componentes cujo estado de saúde é baseado em observações. Ele produz o estado mais crítico das dependências mais importantes.
Para obter detalhes, consulte a página xref:/setup/custom-integrations/monitors/derived-state-monitors.adoc[Monitores de estado derivados].

[,yaml]
----
function: {{ get "urn:stackpack:common:monitor-function:derived-state-monitor" }}
arguments:
  componentTypes: string
----

* `+componentTypes+`: Os tipos de componentes que contribuem para os estados derivados.  Especificado como uma única sequência de valores separados por `+,+` (vírgula)

=== Limite topológico

Aciona um estado de integridade quando um determinado limite é excedido para uma consulta de métrica especificada. A consulta de métrica pode fazer referência ao nome, às tags e às propriedades dos componentes retornados pela consulta de topologia. Limites diferentes podem ser definidos em recursos específicos com xref:/use/alerting/k8s-override-monitor-arguments.adoc[a ajuda de anotações].

[,yaml]
----
function: {{ get "urn:stackpack:common:monitor-function:topological-threshold"  }}
arguments:
  queries:
    topologyQuery: string
    promqlQuery: string
    aliasTemplate: string
    unit: string
  comparator: GTE | GT | LTE | LT     # how to compare metric value to threshold
  threshold: double
  failureState: CRITICAL | DEVIATING | UNKNOWN
  titleTemplate: string
----

* `+queries+`: As consultas a serem executadas
** `+topologyQuery+`: Consulta STQL para selecionar componentes
** `+promqlQuery+`: Consulta PromQL que pode usar rótulos e propriedades de componentes para selecionar séries temporais
** `+unit+`: A unidade dos valores na série temporal retornada pela consulta ou consultas, usada para renderizar o eixo Y do gráfico. Consulte a referência de xref:/develop/reference/k8sTs-chart-units.adoc[unidades suportadas] para todas as unidades.
** `+aliasTemplate+`: Um alias para séries temporais no gráfico de métricas. Esse é um modelo que pode substituir rótulos da série temporal usando o espaço reservado `+${my_label}+`.
* `+comparator+`: Escolha uma das opções LTE/LT/GTE/GT para comparar o limite com a métrica. As séries temporais para as quais `+<metric> <comparator> <threshold>+` é verdadeiro produzirão o estado de falha.
* `+threshold+`: Um limite numérico para comparação.
* `+failureState+`: Ou "CRÍTICO" ou "DESVIADO". "CRÍTICO" será exibido em {stackstate-product-name} e "DESVIADO" em laranja, para indicar diferentes graus de gravidade.
* `+titleTemplate+`: Um título para o resultado de um monitor. Como vários resultados do monitor podem ser vinculados ao mesmo componente, é possível substituir rótulos de séries temporais usando o espaço reservado `+${my_label}+`.

=== Limite dinâmico

Alerta quando o valor atual está fora do intervalo de linha de base previsto, que é calculado dinamicamente com base em dados históricos, levando em conta padrões sazonais semanais e diários.
Essa função de monitor só está disponível quando o stackpack do Detector de anomalias autônomo está instalado.

Para obter detalhes, consulte a página xref:/setup/custom-integrations/monitors/dynamic-threshold-monitors.adoc[Monitores de limite dinâmico].

[,yaml]
----
function: {{ get "urn:stackpack:aad-v2:shared:monitor-function:dt" }}
arguments:
  telemetryQuery:
    query: string
    unit: string
    aliasTemplate: string
  topologyQuery: string
  falsePositiveRate: float
  checkWindowMinutes: integer
  historicWindowMinutes: integer
  historySizeWeeks: 1 | 2 | 3 (integer)
  includePreviousDay: boolean
  removeTrend: boolean
----

* `+telemetryQuery+`: telemetria para avaliar
** `+query+`: Consulta PromQL que é usada para detecção de anomalias e linha de base
** `+unit+`: A unidade dos valores na série temporal retornada pela consulta ou consultas, usada para renderizar o eixo Y do gráfico. Consulte a referência de xref:/develop/reference/k8sTs-chart-units.adoc[unidades suportadas] para todas as unidades.
** `+aliasTemplate+`: Um alias para séries temporais no gráfico de métricas. Esse é um modelo que pode substituir rótulos da série temporal usando o espaço reservado `+${my_label}+`.
* `+topologyQuery+`: Consulta STQL para selecionar componentes
* `+falsePositiveRate+`: digamos `+!!float 1e-8+` - a sensibilidade do monitor a desvios de comportamento.  Um valor mais baixo suprime mais (falsos) positivos, mas também pode levar a falsos negativos (anomalias não percebidas).
* `+checkWindowMinutes+`: digamos `+10+` minutos - a janela de verificação precisa ser equilibrada entre alertas rápidos (valores pequenos) e anomalias corretamente identificadas (valores altos).  Um punhado de pontos de dados funciona bem na prática.
* `+historicWindowMinutes+`: digamos `+120+` (2 horas) - entre parênteses em torno do horário atual, mas uma ou mais semanas atrás - portanto, de 1 hora antes do horário atual a 1 hora depois.  Também são usadas as duas horas anteriores à janela de verificação.  O monitor de limite dinâmico compara a distribuição desses dados históricos com os pontos de dados na janela de verificação.
* `+historySizeWeeks+`: diga `+2+` - o número de semanas das quais os dados são extraídos para o contexto histórico.  Pode ser `+1+`, `+2+` ou `+3+`.
* `+removeTrend+`Para métricas que têm comportamento de tendência (por exemplo, número de solicitações), de modo que o valor absoluto difere de uma semana para outra, essa tendência (o valor médio) pode ser levada em conta.
* `+includePreviousDay+`: normalmente `+false+` - para métricas que não têm um padrão semanal, mas apenas diário, isso permite o uso de dados mais recentes
