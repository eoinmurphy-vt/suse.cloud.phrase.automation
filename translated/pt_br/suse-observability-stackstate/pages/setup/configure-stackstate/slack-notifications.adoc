= Configuração do SUSE Observability para notificações do Slack
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability Self-hosted

[CAUTION]
====
Os usuários SaaS do SUSE Observability podem usar as notificações do Slack sem configuração adicional. Este guia só se aplica a instalações do SUSE Observability auto-hospedadas que planejam usar o canal de notificação do Slack.
====


Antes de usar o canal de notificação do Slack no SUSE Observability, primeiro você precisa seguir as seguintes etapas para configurar o Slack e o SUSE Observability:

. Crie um aplicativo do Slack para o SUSE Observability em seu espaço de trabalho
. Configure o SUSE Observability com as credenciais para esse aplicativo Slack.

== Criação de um aplicativo do Slack para o SUSE Observability

[NOTE]
====
Você precisa ter as permissões no Slack para gerenciar os aplicativos do Slack para seu workspace.
====


Acesse a https://api.slack.com/apps[página] e clique no botão *Create New App (Criar novo aplicativo* ).

* Selecione a opção "From an app manifest" (Do manifesto de um aplicativo) na caixa de diálogo que se abre.
* Selecione o espaço de trabalho para o qual você deseja enviar notificações e clique em next.
* Copie o conteúdo do manifesto do aplicativo Slack abaixo e cole-o na área de texto. Certifique-se de substituir os valores em `+redirect_urls+` pelo(s) URL(s) de sua instância do SUSE Observability. Clique em next.
* Verifique se o URL está correto e se os "escopos de bot" listados são `+channels:join, channels:read, chat:write, groups:read+` e clique no botão create (criar) para criar o aplicativo.
* Na página "Basic information" (Informações básicas) do aplicativo, é possível alterar o ícone (na seção Display information (Informações de exibição)), substituindo-o, por exemplo, pelo logotipo do SUSE Observability image:logo/stackstate-logo.png[logotipo do SUSE Observability,25,25].

.Manifesto do aplicativo Slack para o SUSE Observability
[,json]
----
{
    "display_information": {
        "name": "SUSE Observability",
        "description": "Receive notification messages from SUSE Observability",
        "background_color": "#000000"
    },
    "features": {
        "bot_user": {
            "display_name": "SUSE Observability",
            "always_online": true
        }
    },
    "oauth_config": {
        "redirect_urls": [
            "https://the.url.of.your.stackstate.installation"
        ],
        "scopes": {
            "bot": [
                "channels:join",
                "channels:read",
                "chat:write",
                "groups:read"
            ]
        }
    },
    "settings": {
        "org_deploy_enabled": false,
        "socket_mode_enabled": false,
        "token_rotation_enabled": false
    }
}
----


== Configure o SUSE Observability com as credenciais para esse aplicativo Slack

O SUSE Observability precisa ser configurado com as credenciais do aplicativo Slack que você criou. Você pode fazer isso adicionando o seguinte ao arquivo `+values.yaml+` da instalação do SUSE Observability:

[,yaml]
----
stackstate:
  components:
    all:
      extraEnv:
        open:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientId: "<app client id>"
        secret:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientSecret: "<app client secret>"
----

Os valores `+<app client id>+` e `+<app client secret>+` podem ser encontrados na seção "App credentials" (Credenciais do aplicativo) na página "Basic Information" (Informações básicas) do aplicativo Slack que você criou. Aplique essas alterações de configuração executando o mesmo comando Helm usado durante a instalação do SUSE Observabilityxref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[(para Kubernetes] ou xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_deploy_suse_observability_with_helm[OpenShift]).

Agora você está pronto para usar o canal de notificação do Slack!
