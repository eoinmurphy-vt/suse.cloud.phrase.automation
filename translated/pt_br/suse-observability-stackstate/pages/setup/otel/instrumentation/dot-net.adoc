== Instrumentação automática

A instrumentação automática para .NET pode capturar automaticamente rastreamentos e métricas para uma variedade de https://github.com/open-telemetry/opentelemetry-dotnet-instrumentation/blob/main/docs/internal/instrumentation-libraries.md[bibliotecas e estruturas].

A instrumentação automática não exige nenhuma modificação no aplicativo. Para configurá-lo, siga estas etapas:

. Faça o download da versão https://github.com/open-telemetry/opentelemetry-dotnet-instrumentation/releases/latest/download/opentelemetry-dotnet-instrumentation-linux-glibc.zip[glibc] ou https://github.com/open-telemetry/opentelemetry-dotnet-instrumentation/releases/latest/download/opentelemetry-dotnet-instrumentation-linux-musl.zip[musl] das bibliotecas de instrumentação (musl para o Alpine, glibc para a maioria das outras imagens do docker) das https://github.com/open-telemetry/opentelemetry-java-instrumentation/releases[versões] do repositório opentelemetry-dotnet-instrumentation. Descompacte os arquivos e inclua-os na imagem da janela de encaixe do seu aplicativo em um diretório, aqui usamos `+/autoinstrumentation+`.
. Defina as seguintes variáveis de ambiente, aqui fazemos isso por meio do `+env+` do contêiner na especificação do pod do Kubernetes:
+
[,yaml]
----
env:
- name: CORECLR_ENABLE_PROFILING
  value: "1"
- name: CORECLR_PROFILER
  value: "{918728DD-259F-4A6A-AC2B-B85E1B658318}"
- name: CORECLR_PROFILER_PATH
  # for glibc:
  value: "/autoinstrumentation/linux-x64/OpenTelemetry.AutoInstrumentation.Native.so"
  # For musl use instead:
  # value: "/autoinstrumentation/linux-musl-x64/OpenTelemetry.AutoInstrumentation.Native.so"
- name: DOTNET_ADDITIONAL_DEPS
  value: "/autoinstrumentation/AdditionalDeps"
- name: DOTNET_SHARED_STORE
  value: "/autoinstrumentation/store"
- name: DOTNET_STARTUP_HOOKS
  value: "/autoinstrumentation/net/OpenTelemetry.AutoInstrumentation.StartupHook.dll"
- name: OTEL_DOTNET_AUTO_HOME
  value: "/autoinstrumentation"
----

. Adicione também as variáveis de ambiente adicionais xref:/setup/otel/instrumentation/sdk-exporter-config.adoc[para configurar o nome do serviço e o ponto de extremidade do exportador] no pod; os protocolos compatíveis são gRPC e protobuf sobre HTTP.
. Implante seu aplicativo com as alterações
. Verifique se o SUSE Observability está recebendo rastreamentos e/ou métricas, procurando as métricas/rastreamentos no explorador de métricas/rastreamentos para o nome do seu serviço

Para obter mais detalhes, consulte a https://opentelemetry.io/docs/languages/java/automatic/[documentação do].

== Instrumentação manual

A instrumentação manual pode ser usada quando você precisa de métricas, rastreamentos ou registros de partes do código que não são compatíveis com a instrumentação automática. Por exemplo, bibliotecas sem suporte, código interno ou métricas de nível comercial.

Para capturar esses dados, você precisa modificar seu aplicativo.

. Incluir o Open Telemetry SDK como uma dependência
. Adicione código ao seu aplicativo para capturar métricas, períodos ou registros quando necessário

Há uma documentação detalhada sobre isso nas https://opentelemetry.io/docs/languages/net/instrumentation/[páginas de documentos do].

Certifique-se de usar o exportador OTLP (esse é o padrão) e https://opentelemetry.io/docs/languages/java/instrumentation/#_autoconfiguration[a configuração automática do site]. Ao implantar o aplicativo, o nome do serviço e o exportador são xref:/setup/otel/instrumentation/sdk-exporter-config.adoc[configurados por meio de variáveis de ambiente].

== Métricas no SUSE Observability

ifdef::ss-ff-stackpacks2_enabled[]
Para algumas métricas do .NET, por exemplo, métricas do coletor de lixo, o SUSE Observability definiu gráficos sobre os componentes relacionados. No Kubernetes, os gráficos estão disponíveis nos pods. É possível xref:/setup/custom-integrations/metric-bindings/index.adoc[adicionar gráficos para mais métricas], o que funciona para métricas de instrumentação automática, mas também para métricas específicas de aplicativos de instrumentação manual.
endif::[]
