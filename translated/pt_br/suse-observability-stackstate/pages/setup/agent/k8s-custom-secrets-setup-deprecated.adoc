= Configuração de segredos personalizados do k8s (obsoleto)
:revdate: 2025-10-27
:page-revdate: {revdate}

== Visão geral

O agente do SUSE Observability suporta a especificação do nome de um segredo personalizado que contém a chave da API e o token de autorização do cluster. Esse recurso é útil para usuários que desejam gerenciar seus próprios segredos e evitar a criação automática de segredos pelo Helm chart.

[CAUTION]
====
*Esse método de personalização de segredos está obsoleto e é incompatível com o recurso RBAC.*

Use xref:/setup/agent/k8s-custom-secrets-setup.adoc[o novo método] para usar segredos externos.
====


== O gráfico do leme

=== Opções de configuração

* `+stackstate.manageOwnSecrets+`: Um sinalizador booleano que determina se o usuário deseja gerenciar seus próprios segredos. O valor padrão é `+false+`.
* `+stackstate.customSecretName+`: (Opcional) Nome do segredo personalizado a ser criado pelo usuário. Necessário se `+stackstate.manageOwnSecrets+` estiver definido como `+true+`.
* `+stackstate.customApiKeySecretKey+`: (Opcional) Nome da chave para a chave de API dentro do segredo personalizado. Necessário se `+stackstate.manageOwnSecrets+` estiver definido como `+true+`.
* `+stackstate.customClusterAuthTokenSecretKey+`: (Opcional) Nome da chave para o token de autorização do cluster no segredo personalizado. Necessário se `+stackstate.manageOwnSecrets+` estiver definido como `+true+`.

=== Descrição do comportamento

* *Criação automática de segredos*: Por padrão, o gráfico continua a criar segredos automaticamente, como antes, se `+stackstate.manageOwnSecrets+` estiver definido como `+false+`.
* *Gerenciamento de segredos personalizado*: Se `+stackstate.manageOwnSecrets+` estiver definido como `+true+`, o gráfico espera que o usuário forneça o nome do segredo personalizado (`+stackstate.customSecretName+`) juntamente com as chaves para a chave de API e o token de autorização (`+stackstate.customApiKeySecretKey+` e `+stackstate.customClusterAuthTokenSecretKey+`, respectivamente).
* *Omissão implícita*: Ao especificar que você gostaria de gerenciar seus próprios segredos, o gráfico ignorará os valores de `+stackstate.apiKey+` e `+stackstate.cluster.authToken+`.
+
== Como usar em values.yaml

. *Usando a criação automática de segredos (padrão)*:
+
[,yaml]
----
 stackstate:
   manageOwnSecrets: false
   apiKey: "<your api key>"
----

. *Gerenciando seus próprios segredos*:
+
[,yaml]
----
 stackstate:
   manageOwnSecrets: true
   customSecretName: my-custom-secret
   customApiKeySecretKey: api-key
   customClusterAuthTokenSecretKey: auth-token
----
