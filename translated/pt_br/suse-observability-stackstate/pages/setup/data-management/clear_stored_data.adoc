= Limpar dados armazenados
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability Self-hosted

Os dados no SUSE Observability são divididos em quatro conjuntos diferentes:

* Dados do Elasticsearch
* Dados do tópico do Kafka
* Dados do StackGraph
* Dados métricos

Com tantos dados para armazenar, é importante ter os meios para gerenciá-los. Há um período padrão de retenção de dados de 30 dias definido no SUSE Observability. Isso pode ser configurado de acordo com suas necessidades usando a CLI do SUSE Observability. Saiba mais sobre a xref:/setup/data-management/data_retention.adoc[retenção de dados do SUSE Observability].

== Limpar dados manualmente

Para limpar os dados armazenados no SUSE Observability em execução no Kubernetes, é recomendável executar uma instalação limpa:

. xref:/setup/install-stackstate/kubernetes_openshift/uninstall.adoc#_un_install_the_helm_chart[Desinstalação do SUSE Observability]
. xref:/setup/install-stackstate/kubernetes_openshift/uninstall.adoc#_remove_remaining_resources[Remova todos os PVCs]
. Instale o SUSE Observability novamente usando a mesma configuração de antes, no xref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[Kubernetes] ou no xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_deploy_suse_observability_with_helm[OpenShift].
