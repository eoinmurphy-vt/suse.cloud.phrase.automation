== Visão geral

O SUSE Observability pode sincronizar informações de integridade de suas próprias fontes de dados por meio de HTTP ou da xref:/setup/cli/cli-sts.adoc[CLI do SUSE Observability].

== API do receptor de observabilidade do SUSE

A API do SUSE Observability Receiver aceita dados de topologia, métricas, eventos e integridade em um objeto JSON comum. O local padrão da API do receptor é `+<STACKSTATE_RECEIVER_API_ADDRESS>+`, construído com o uso de `+<STACKSTATE_BASE_URL>+` e ``<+STACKSTATE_RECEIVER_API_KEY>+``.

O endereço `+<STACKSTATE_RECEIVER_API_ADDRESS>+` para o SUSE Observability implantado no Kubernetes ou no OpenShift é:

[,text]
----
https://<STACKSTATE_BASE_URL>/receiver/stsAgent/intake?api_key=<STACKSTATE_RECEIVER_API_KEY>
----

Os endereços `+<STACKSTATE_BASE_URL>+` e `+<STACKSTATE_RECEIVER_API_KEY>+` são definidos durante a instalação do SUSE Observability; para obter detalhes, consulte xref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_generate_values_yaml[Instalação do Kubernetes - parâmetros de configuração].

== JSON

=== Objeto JSON comum

Os dados de topologia, telemetria e integridade são enviados para a API do receptor via HTTP POST. Há um objeto JSON comum usado para todas as mensagens.

[,javascript]
----
{
  "collection_timestamp": 1548855554, // the epoch timestamp for the collection
  "events": {}, // used to send events data
  "internalHostname": "localdocker.test", // the host sending this data
  "metrics": [], // used to send metrics data
  "service_checks": [],
  "topologies": [], // used to send topology data
  "health": // used for sending health data
}
----

=== Carga de saúde JSON

O SUSE Observability aceita dados de integridade com base em um xref:/configure/health/health-synchronization.adoc#_consistency_models[modelo de consistência] escolhido. As mensagens que podem ser enviadas para cada modelo estão descritas nas páginas abaixo:

* xref:/configure/health/send-health-data/repeat_snapshots.adoc[Repetir instantâneos JSON]
* xref:/configure/health/send-health-data/transactional_increments.adoc[Incrementos transacionais JSON]
