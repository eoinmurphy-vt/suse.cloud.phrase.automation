= Guia de início rápido
:page-languages: [en, ja, ko, zh]
:revdate: 2026-03-03
:page-revdate: {revdate}

Este guia o ajudará a iniciar rapidamente um cluster com opções padrão. Certifique-se de que seus nós atendam aos xref:installation/requirements.adoc[requisitos] antes de continuar.

* Consulte a página de xref:installation/installation.adoc[instalação] para obter mais detalhes sobre a instalação e a configuração do K3s.
* Para obter informações sobre como os componentes do K3s funcionam juntos, consulte a página xref:architecture.adoc[Arquitetura].
* Se você for novo no Kubernetes, https://kubernetes.io/docs/tutorials/kubernetes-basics/[os documentos oficiais do Kubernetes em] têm ótimos tutoriais que abrangem os conceitos básicos com os quais todos os administradores de cluster devem estar familiarizados.

[IMPORTANT]
====
Você pode usar qualquer versão do K3s Prime listada no URL do Prime Artifacts para os ativos mencionados nestas etapas. Para saber mais sobre o URL do Prime Artifacts, consulte https://scc.suse.com/rancher-docs/rancherprime/latest/en/reference-guide.html#prime-artifacts-url[a documentação exclusiva do Prime em]. A autenticação é necessária. Use suas credenciais https://scc.suse.com/home[do] para fazer login.
====


== Instalar o script

{k3s-product-name} fornece um script de instalação que é uma maneira conveniente de instalá-lo como um serviço em sistemas baseados em systemd ou openrc. Esse script está disponível em https://get.k3s.io. Para instalar o {k3s-product-name} usando esse método, faça o seguinte:

Execute o instalador, onde `+INSTALL_K3S_ARTIFACT_URL+` é o https://scc.suse.com/rancher-docs/rancherprime/latest/en/reference-guide.html#prime-artifacts-url[URL do] e `+INSTALL_K3S_CHANNEL+` é um canal de lançamento que você pode assinar e tem como padrão `+stable+`. Neste exemplo, `+INSTALL_K3S_CHANNEL="latest"+` lhe fornece a versão mais recente de {k3s-product-name}.

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_CHANNEL="latest" sh -
----
Se você quiser especificar uma versão, defina a variável de ambiente `+INSTALL_RKE2_VERSION+`:

[,bash]
----
curl -sfL https://get.k3s.io/ | sudo INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_VERSION="<VERSION>" sh -
----

Depois de executar essa instalação:

* O serviço K3s será configurado para reiniciar automaticamente após a reinicialização do nó ou se o processo for interrompido ou eliminado
* Outros utilitários serão instalados, incluindo `+kubectl+`, `+crictl+`, `+ctr+`, `+k3s-killall.sh+`, e `+k3s-uninstall.sh+`
* Um arquivo https://kubernetes.io/docs/concepts/configuration/organize-cluster-access-kubeconfig/[] kubeconfig será gravado em `+/etc/rancher/k3s/k3s.yaml+` e o kubectl instalado por {k3s-product-name} o usará automaticamente

Uma instalação de servidor de nó único é um cluster do Kubernetes totalmente funcional, incluindo todos os componentes de datastore, plano de controle, kubelet e tempo de execução do contêiner necessários para hospedar pods de carga de trabalho. Não é necessário adicionar nós de servidor ou agentes adicionais, mas talvez você queira fazê-lo para adicionar mais capacidade ou redundância ao cluster.

Para instalar nós de agente adicionais e adicioná-los ao cluster, execute o script de instalação com as variáveis de ambiente `+K3S_URL+` e `+K3S_TOKEN+`. Aqui está um exemplo que mostra como ingressar em um agente:

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s K3S_URL=https://myserver:6443 K3S_TOKEN=mynodetoken sh -
----

A definição do parâmetro `+K3S_URL+` faz com que o instalador configure o K3s como um agente, em vez de um servidor. O agente do K3s se registrará no servidor do K3s que está escutando no URL fornecido. O valor a ser usado para `+K3S_TOKEN+` é armazenado em `+/var/lib/rancher/k3s/server/node-token+` no nó do servidor.

[NOTE]
====
Cada máquina deve ter um nome de host exclusivo. Se suas máquinas não tiverem nomes de host exclusivos, passe a variável de ambiente `+K3S_NODE_NAME+` e forneça um valor com um nome de host válido e exclusivo para cada nó.
====


Se você estiver interessado em ter mais nós de servidor, consulte as páginas xref:datastore/ha-embedded.adoc[High Availability Embedded etcd] e xref:datastore/ha.adoc[High Availability External DB] para obter mais informações.
