= Configuração da criptografia de segredos
:page-languages: [en, ja, ko, zh]
:revdate: 2026-03-03
:page-revdate: {revdate}

O K3s suporta a ativação da criptografia de segredos em repouso. Ao iniciar o servidor pela primeira vez, passar o sinalizador `+--secrets-encryption+` fará o seguinte automaticamente:

* Gerar uma chave AES-CBC
* Gerar um arquivo de configuração de criptografia com a chave gerada
* Passe a configuração para a KubeAPI como encryption-provider-config

[TIP]
====
A criptografia de segredos não pode ser ativada em um servidor existente sem que ele seja reiniciado. +
Use `+curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s sh -s - server --secrets-encryption+` se estiver instalando a partir de um script ou de outros métodos descritos em xref:installation/configuration.adoc#_configuration_with_install_script[Opções de configuração].
====

== Escolha do provedor de criptografia

[IMPORTANT]
.Versão Gate
====
Disponível a partir das versões de abril de 2025: v1.30.12+k3s1, v1.31.8+k3s1, v1.32.4+k3s1, v1.33.0+k3s1.
====

Usando o sinalizador `+--secrets-encryption-provider+`, você pode escolher entre os seguintes provedores de criptografia compatíveis com o K3:

* `+aescbc+`: AES-CBC com preenchimento PKCS#7. Esse é o provedor padrão.
* `+secretbox+`: XSalsa20 e Poly1305

### Migração de provedores

Você pode migrar do provedor `+aescbc+` para o provedor `+secretbox+` seguindo estas etapas:

. Certifique-se de que o provedor `+secretbox+` seja compatível com a versão do K3s.
. Atualize/adicione o sinalizador `+secrets-encryption-provider+` em seu arquivo de configuração do K3s para `+secretbox+`.
. Faça a rotação das chaves de criptografia, seguindo a seção xref:cli/secrets-encrypt.adoc#_encryption_key_rotation[Rotação da chave de criptografia] abaixo.

== Arquivo de configuração de criptografia gerado

Quando você iniciar o servidor com `+--secrets-encryption+`, o K3s gerará um arquivo de configuração de criptografia em `+/var/lib/rancher/k3s/server/cred/encryption-config.json+`.

Abaixo está um exemplo do arquivo de configuração de criptografia gerado com o provedor `+aescbc+` padrão:

[,json]
----
{
  "kind": "EncryptionConfiguration",
  "apiVersion": "apiserver.config.k8s.io/v1",
  "resources": [
    {
      "resources": [
        "secrets"
      ],
      "providers": [
        {
          "aescbc": {
            "keys": [
              {
                "name": "aescbckey",
                "secret": "xxxxxxxxxxxxxxxxxxx"
              }
            ]
          }
        },
        {
          "identity": {}
        }
      ]
    }
  ]
}
----

== Ferramenta de criptografia de segredos

O K3s contém uma ferramenta utilitária `+secrets-encrypt+`, que permite o controle automático do seguinte:

* Desativar/ativar a criptografia de segredos
* Adição de novas chaves de criptografia
* Rotação e exclusão de chaves de criptografia
* Recriptografar segredos

Para obter mais informações, consulte a xref:cli/secrets-encrypt.adoc[documentação do comando`+k3s secrets-encrypt+` ].
