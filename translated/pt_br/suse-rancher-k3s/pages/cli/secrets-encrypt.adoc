= k3s secrets-encrypt
:page-languages: [en, ja, ko, zh]
:revdate: 2026-01-05
:page-revdate: {revdate}

O K3s suporta a ativação da criptografia de segredos em repouso. Para obter mais informações, consulte xref:security/secrets-encryption.adoc[Criptografia de segredos].

== Ferramenta de criptografia de segredos

[IMPORTANT]
.Versão Gate
====
Disponível em https://github.com/k3s-io/k3s/releases/tag/v1.21.8%2Bk3s1[.21.8+k3s1] v1
====


O K3s contém uma ferramenta CLI `+secrets-encrypt+`, que permite o controle automático do seguinte:

* Desativar/ativar a criptografia de segredos
* Adição de novas chaves de criptografia
* Rotação e exclusão de chaves de criptografia
* Recriptografar segredos

[CAUTION]
====
O não cumprimento do procedimento adequado de rotação das chaves de criptografia pode deixar seu cluster permanentemente corrompido. Proceda com cautela.
====

[#_encryption_key_rotation]
=== Rotação de chaves de criptografia

[IMPORTANT]
.Versão Gate
====
Disponível a partir das versões de setembro de 2024: v1.30.5+k3s1, v1.31.1+k3s1.
====

[tabs,sync-group-id=se]
======
Servidor único::
+
--
Para alternar as chaves de criptografia de segredos em um cluster de servidor único:

. Inicie o servidor K3s com a bandeira `+--secrets-encryption+`
+
[NOTE]
====
No momento, _não há_ suporte para iniciar o K3s sem criptografia e ativá-la posteriormente.
====

. Girar chaves de criptografia de segredos
+
----
k3s secrets-encrypt rotate-keys
----

. Aguarde a conclusão da reencriptação. Observe os registros do servidor ou aguarde:
+
[,bash]
----
$ k3s secrets-encrypt status
Encryption Status: Enabled
Current Rotation Stage: reencrypt_finished
----
--

Alta disponibilidade::
+
--
Para alternar chaves de criptografia de segredos em configurações de HA:

. Inicie os três servidores K3s com o sinalizador `+--secrets-encryption+`. Para fins de brevidade, os servidores serão chamados de S1, S2 e S3.
+
[NOTE]
====
No momento, _não há_ suporte para iniciar o K3s sem criptografia e ativá-la posteriormente.
====

. Girar chaves de criptografia de segredos em S1
+
[,bash]
----
k3s secrets-encrypt rotate-keys
----

. Aguarde a conclusão da reencriptação. Observe os registros do servidor ou aguarde:
+
[,bash]
----
$ k3s secrets-encrypt status
Encryption Status: Enabled
Current Rotation Stage: reencrypt_finished
----
+
[NOTE]
====
Os K3s recodificarão cerca de 5 segredos por segundo. Clusters com um grande número de segredos podem levar vários minutos para serem criptografados novamente. Você pode acompanhar o progresso nos logs do servidor.
====

. Reinicie o K3s em S1 com os mesmos argumentos. Se estiver executando o K3s como um serviço:
+
[,bash]
----
# If using systemd
systemctl restart k3s
# If using openrc
rc-service k3s restart
----

. Quando S1 estiver funcionando, reinicie os K3s em S2 e S3
--
======

=== Desativar/reativar a criptografia de segredos

[tabs,sync-group-id=se]
======
Servidor único::
+
--
Depois de iniciar um servidor com o sinalizador `+--secrets-encryption+`, a criptografia de segredos pode ser desativada.

Para desativar a criptografia de segredos em um cluster de nó único:

. Desativar
+
[,bash]
----
k3s secrets-encrypt disable
----

. Encerre e reinicie o servidor K3s com os mesmos argumentos. Se estiver executando o K3s como um serviço:
+
[,bash]
----
# If using systemd
systemctl restart k3s
# If using openrc
rc-service k3s restart
----

. Criptografar novamente com sinalizadores
+
[,bash]
----
k3s secrets-encrypt reencrypt --force --skip
----

Para reativar a criptografia de segredos em um cluster de nó único:

. Ativar
+
[,bash]
----
k3s secrets-encrypt enable
----

. Encerrar e reiniciar o servidor K3s com os mesmos argumentos
. Criptografar novamente com sinalizadores
+
[,bash]
----
k3s secrets-encrypt reencrypt --force --skip
----
--

Alta disponibilidade::
+
--
Depois de iniciar um cluster HA com sinalizadores `+--secrets-encryption+`, a criptografia de segredos pode ser desativada.

[NOTE]
====
Embora não seja obrigatório, é recomendável que você escolha um nó do servidor para executar os comandos do `+secrets-encrypt+`.
====

Para fins de brevidade, os três servidores usados neste guia serão chamados de S1, S2 e S3.

Para desativar a criptografia de segredos em um cluster HA:

. Desativar em S1
+
[,bash]
----
k3s secrets-encrypt disable
----

. Encerrar e reiniciar o S1 com os mesmos argumentos. Se estiver executando o K3s como um serviço:
+
[,bash]
----
# If using systemd
systemctl restart k3s
# If using openrc
rc-service k3s restart
----

. Quando o S1 estiver funcionando, elimine e reinicie o S2 e o S3
. Criptografar novamente com sinalizadores em S1
+
[,bash]
----
k3s secrets-encrypt reencrypt --force --skip
----

Para reativar a criptografia de segredos em um cluster HA:

. Ativar em S1
+
[,bash]
----
k3s secrets-encrypt enable
----

. Encerrar e reiniciar o S1 com os mesmos argumentos
. Quando o S1 estiver funcionando, elimine e reinicie o S2 e o S3
. Criptografar novamente com sinalizadores em S1
+
[,bash]
----
k3s secrets-encrypt reencrypt --force --skip
----

=== Status da criptografia de segredos

A ferramenta secrets-encrypt inclui um comando `+status+` que exibe informações sobre o status atual da criptografia de segredos no nó.

Um exemplo do comando em um nó de servidor único:

[,bash]
----
$ k3s secrets-encrypt status
Encryption Status: Enabled
Current Rotation Stage: start
Server Encryption Hashes: All hashes match

Active  Key Type  Name
------  --------  ----
 *      AES-CBC   aescbckey
----

Outro exemplo em um cluster HA, após a rotação das chaves, mas antes de reiniciar os servidores:

[,bash]
----
$ k3s secrets-encrypt status
Encryption Status: Enabled
Current Rotation Stage: rotate
Server Encryption Hashes: hash does not match between node-1 and node-2

Active  Key Type  Name
------  --------  ----
 *      AES-CBC   aescbckey-2021-12-10T22:54:38Z
        AES-CBC   aescbckey
----

Os detalhes de cada seção são os seguintes:

* *Status da criptografia*: Exibe se a criptografia de segredos está desativada ou ativada no nó
* *Estágio de rotação atual*: Indica o estágio de rotação atual no nó. +
Os estágios são: `+start+`, `+prepare+`, `+rotate+`, `+reencrypt_request+`, `+reencrypt_active+`, `+reencrypt_finished+`
* *Hashes de criptografia do servidor*: Útil para clusters HA, indica se todos os servidores estão no mesmo estágio com seus arquivos locais. Isso pode ser usado para identificar se é necessário reiniciar os servidores antes de prosseguir para a próxima etapa. No exemplo de HA acima, o nó 1 e o nó 2 têm hashes diferentes, o que indica que eles não têm a mesma configuração de criptografia no momento. A reinicialização dos servidores sincronizará suas configurações.
* *Tabela principal*: Resume as informações sobre as chaves de criptografia de segredos encontradas no nó.
 ** *Ativo*: O "*" indica qual das chaves, se houver, está sendo usada atualmente para criptografia de segredos. Uma chave ativa é usada pelo Kubernetes para criptografar quaisquer novos segredos.
 ** *Tipo de chave*: Todas as chaves que usam essa ferramenta são do tipo `+AES-CBC+`. Veja mais informações https://kubernetes.io/docs/tasks/administer-cluster/encrypt-data/#providers[em]
 ** *Nome*: Nome da chave de criptografia.
