= Variáveis de ambiente
:page-languages: [en, ja, ko, zh]
:revdate: 2024-12-05
:page-revdate: {revdate}

Conforme mencionado no xref:quick-start.adoc[Guia de início rápido], você pode usar o script de instalação disponível em https://get.k3s.io para instalar o K3s como um serviço em sistemas baseados em systemd e openrc.

A forma mais simples desse comando é a seguinte:

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s sh -
----

Ao usar esse método para instalar o K3s, as seguintes variáveis de ambiente podem ser usadas para configurar a instalação:

|===
| Variável de ambiente | Descrição

| `+INSTALL_K3S_SKIP_DOWNLOAD+`
| Se definido como true, o download do hash ou do binário do K3s não será feito.

| `+INSTALL_K3S_SYMLINK+`
| Por padrão, criará links simbólicos para os binários kubectl, crictl e ctr se os comandos ainda não existirem no caminho. Se definido como "skip", não criará links simbólicos e "force" substituirá.

| `+INSTALL_K3S_SKIP_ENABLE+`
| Se definido como true, não ativará nem iniciará o serviço K3s.

| `+INSTALL_K3S_SKIP_START+`
| Se definido como true, não iniciará o serviço K3s.

| `+INSTALL_K3S_VERSION+`
| Versão do K3s para download no Github. Tentará fazer o download do canal estável se não for especificado.

| `+INSTALL_K3S_BIN_DIR+`
| Diretório para instalar o binário, os links e o script de desinstalação do K3s ou usar `+/usr/local/bin+` como padrão.

| `+INSTALL_K3S_BIN_DIR_READ_ONLY+`
| Se definido como true, não gravará arquivos em `+INSTALL_K3S_BIN_DIR+`, forçando a configuração de `+INSTALL_K3S_SKIP_DOWNLOAD=true+`.

| `+INSTALL_K3S_SYSTEMD_DIR+`
| Diretório para instalar o serviço systemd e os arquivos de ambiente ou usar `+/etc/systemd/system+` como padrão.

| `+INSTALL_K3S_EXEC+`
| Comando com sinalizadores a serem usados para lançar K3s no serviço. Se o comando não for especificado e o `+K3S_URL+` estiver definido, o padrão será "agent". Se `+K3S_URL+` não for definido, o padrão será "server". Para obter ajuda, consulte xref:installation/configuration.adoc#_configuration_with_install_script[este exemplo.]

| `+INSTALL_K3S_NAME+`
| Nome do serviço systemd a ser criado; o padrão será "k3s" se o k3s estiver sendo executado como servidor e "k3s-agent" se o k3s estiver sendo executado como agente. Se especificado, o nome será prefixado com "k3s-".

| `+INSTALL_K3S_TYPE+`
| O tipo de serviço systemd a ser criado será o padrão do comando K3s exec se não for especificado.

| `+INSTALL_K3S_SELINUX_WARN+`
| Se definido como true, continuará se a política k3s-selinux não for encontrada.

| `+INSTALL_K3S_SKIP_SELINUX_RPM+`
| Se definido como true, ignorará a instalação automática do RPM do k3s.

| `+INSTALL_K3S_CHANNEL_URL+`
| URL do canal para obter a URL de download do K3s. O padrão é https://update.k3s.io/v1-release/channels.

| `+INSTALL_K3S_CHANNEL+`
| Canal a ser usado para obter o URL de download do K3s. O padrão é "estável". As opções incluem: `+stable+`, `+latest+`, `+testing+`.
|===

Este exemplo mostra onde colocar as variáveis de ambiente mencionadas anteriormente como opções (após o pipe):

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_CHANNEL=latest sh -
----

As variáveis de ambiente que começam com `+K3S_+` serão preservadas para serem usadas pelos serviços systemd e openrc.

A configuração de `+K3S_URL+` sem definir explicitamente um comando exec terá como padrão o comando "agent".

Ao executar o agente, o endereço `+K3S_TOKEN+` também deve ser definido.

[NOTE]
.Versão Gate
====
Disponível a partir das versões de outubro de 2024: v1.28.15+k3s1, v1.29.10+k3s1, v1.30.6+k3s1, v1.31.2+k3s1.
====

O K3s agora usará o site `+PATH+` para encontrar runtimes de contêineres alternativos, além de verificar os caminhos padrão usados pelos pacotes de runtime de contêineres. Para usar esse recurso, é necessário modificar a variável de ambiente PATH do serviço K3s para adicionar os diretórios que contêm os binários de tempo de execução do contêiner.

Recomenda-se que você modifique um desses dois arquivos de ambiente:

* /etc/default/k3s # ou k3s-agent
* /etc/sysconfig/k3s # ou k3s-agent

Este exemplo adicionará o endereço `+PATH+` em `+/etc/default/k3s+`:

[,bash]
----
echo PATH=$PATH >> /etc/default/k3s
----

[WARNING]
====
`+PATH+` As alterações devem ser feitas com cuidado para evitar colocar binários não confiáveis no caminho dos serviços executados como raiz.
====
