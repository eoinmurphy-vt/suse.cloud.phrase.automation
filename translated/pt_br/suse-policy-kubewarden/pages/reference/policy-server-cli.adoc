== Ajuda da linha de comando para `+policy-server+`

Este documento contém o conteúdo da ajuda do programa de linha de comando `+policy-server+`.

*Visão geral do comando:*

* link:#policy-server[`+policy-server+`]
* link:#policy-server-docs[`+policy-server docs+`]

=== `+policy-server+`

*Uso:* `+policy-server ++[++OPTIONS++]++ ++[++COMMAND++]+++`

==== *Subcomandos:*

* `+docs+` - Gera a documentação markdown para os comandos do servidor de políticas

==== *Opções:*

* `+--addr ++<++BIND++_++ADDRESS++>+++` - Vincular ao ADDRESS
+
Valor padrão: `+0.0.0.0+`
* `+--always-accept-admission-reviews-on-namespace ++<++NAMESPACE++>+++` - Sempre aceite AdmissionReviews que tenham como alvo o namespace fornecido
* `+--cert-file ++<++CERT++_++FILE++>+++` - Caminho para um arquivo de certificado X.509 para HTTPS
* `+--client-ca-file ++<++CLIENT++_++CA++_++FILE++>+++` - Caminho para um arquivo de certificado de CA que emitiu o certificado do cliente. Necessário para ativar o mTLS
* `+--daemon+` - Se definido, executa o policy-server no modo desconectado como um daemon
* `+--daemon-pid-file ++<++DAEMON-PID-FILE++>+++` - Caminho para o arquivo PID, usado somente durante a execução no modo daemon
+
Valor padrão: `+policy-server.pid+`
* `+--daemon-stderr-file ++<++DAEMON-STDERR-FILE++>+++` - Caminho para o arquivo que contém o stderr, usado somente quando executado no modo daemon
* `+--daemon-stdout-file ++<++DAEMON-STDOUT-FILE++>+++` - Caminho para o arquivo que contém o stdout, usado somente quando executado no modo daemon
* `+--disable-timeout-protection+` - Desativar a proteção de tempo limite da política
* `+--docker-config-json-path ++<++DOCKER++_++CONFIG++>+++` - Caminho para um caminho semelhante ao config.json do Docker. Pode ser usado para indicar detalhes de autenticação do registro
* `+--enable-metrics+` - Ativar métricas
* `+--enable-pprof+` - Ativar a criação de perfil pprof
* `+--ignore-kubernetes-connection-failure+` - Não saia com um erro se a conexão com o Kubernetes falhar. Isso fará com que as políticas com reconhecimento de contexto sejam interrompidas quando não houver conexão com o Kubernetes.
* `+--key-file ++<++KEY++_++FILE++>+++` - Caminho para um arquivo de chave privada X.509 para HTTPS
* `+--log-fmt ++<++LOG++_++FMT++>+++` - Formato de saída de registro
+
Valor padrão: `+text+`
+
Valores possíveis: `+text+`, `+json+`, `+otlp+`
* `+--log-level ++<++LOG++_++LEVEL++>+++` - Nível de registro
+
Valor padrão: `+info+`
+
Valores possíveis: `+trace+`, `+debug+`, `+info+`, `+warn+`, `+error+`
* `+--log-no-color+` - Desativar a saída colorida para os registros
* `+--policies ++<++POLICIES++_++FILE++>+++` - Arquivo YAML que contém as políticas a serem carregadas e suas configurações
+
Valor padrão: `+policies.yml+`
* `+--policies-download-dir ++<++POLICIES++_++DOWNLOAD++_++DIR++>+++` - Caminho de download das políticas
+
Valor padrão: `+.+`
* `+--policy-timeout ++<++MAXIMUM++_++EXECUTION++_++TIME++_++SECONDS++>+++`
- Interromper a avaliação da política após o tempo determinado
+
Valor padrão: `+2+`
* `+--port ++<++PORT++>+++` - Ouça no PORT
+
Valor padrão: `+3000+`
* `+--readiness-probe-port ++<++READINESS++_++PROBE++_++PORT++>+++` —
Expose readiness endpoint on READINESS++_++PROBE++_++PORT
+
Valor padrão: `+8081+`
* `+--sigstore-cache-dir ++<++SIGSTORE++_++CACHE++_++DIR++>+++` - Diretório usado para armazenar em cache os dados do sigstore
+
Valor padrão: `+sigstore-data+`
* `+--sources-path ++<++SOURCES++_++PATH++>+++` - Arquivo YAML que contém informações de origem (https, hosts inseguros de registro, CAs personalizadas\...)
* `+--verification-path ++<++VERIFICATION++_++CONFIG++_++PATH++>+++` - Arquivo YAML que contém informações de verificação (URIs, chaves, anotações\...)
* `+--workers ++<++WORKERS++_++NUMBER++>+++` - Número de threads de trabalho a serem criados

=== `+policy-server docs+`

Gera a documentação markdown para os comandos do servidor de políticas

*Uso:* `+policy-server docs --output ++<++FILE++>+++`

==== *Opções:*

* `+-o+`, `+--output ++<++FILE++>+++` - caminho onde o arquivo de documentação será armazenado
