= Distribuição de uma política do Gatekeeper com {project-name}
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: Distribua políticas do Gatekeeper em clusters do Kubernetes anotando arquivos de política com metadados e implantando-os por meio de ClusterAdmissionPolicy.
:doc-persona: ["kubewarden-policy-developer"]
:doc-topic: ["writing-policies", "rego", "gatekeeper", "distribute"]
:doc-type: ["tutorial"]
:keywords: ["kubewarden", "kubernetes", "gatekeeper", "rego"]
:current-version: {page-origin-branch}

As políticas precisam ser anotadas para que sejam enviadas e, por fim, executadas pelo {project-name} `+policy-server+` em um cluster do Kubernetes.

Anotar e distribuir uma política do Gatekeeper é semelhante à distribuição de uma política do Open Policy Agent.

== Anotação da política

Você escreverá um arquivo `+metadata.yaml+` em seu diretório de políticas com o conteúdo:

[subs="+attributes",yaml]
----
rules:
- apiGroups: [""]
  apiVersions: ["*"]
  resources: ["*"]
  operations: ["CREATE"]
mutating: false
contextAware: false
executionMode: gatekeeper
annotations:
  io.kubewarden.policy.title: no-default-namespace
  io.kubewarden.policy.version: 0.1.0 # should match the OCI tag
  io.kubewarden.policy.description: This policy will reject any resource created inside the default namespace
  io.kubewarden.policy.author: The {short-project-name} Authors
  io.kubewarden.policy.url: https://github.com/kubewarden/some-policy
  io.kubewarden.policy.source: https://github.com/kubewarden/some-policy
  io.kubewarden.policy.license: Apache-2.0
  io.kubewarden.policy.usage: |
      This policy is just an example.

      You can write interesting descriptions about the policy here.
----

você pode ver que tudo é igual aos metadados da versão do Open Policy Agent, exceto pelo bit `+executionMode: gatekeeper+`.

Faça anotações sobre a política:

[subs="+attributes",console]
----
$ kwctl annotate policy.wasm --metadata-path metadata.yaml --output-path annotated-policy.wasm
----

== Impulsionando a política

Envie sua apólice para um registro de OCI:

[subs="+attributes",console]
----
$ kwctl push annotated-policy.wasm registry.my-company.com/kubewarden/no-default-namespace-gatekeeper:v0.0.1
Policy successfully pushed
----

== Implementação no Kubernetes

Primeiro, você deve encaminhar sua apólice para a loja local `+kwctl+`:

[subs="+attributes",console]
----
$ kwctl pull registry://registry.my-company.com/kubewarden/no-default-namespace-gatekeeper:v0.0.1
pulling policy...
----

Agora você pode criar um recurso do scaffold `+ClusterAdmissionPolicy+`:

[subs="+attributes",console]
----
$ kwctl scaffold manifest registry://registry.my-company.com/kubewarden/no-default-namespace-gatekeeper:v0.0.1 --type ClusterAdmissionPolicy
---
apiVersion: policies.kubewarden.io/v1alpha2
kind: ClusterAdmissionPolicy
metadata:
  name: generated-policy
spec:
  module: "registry://registry.my-company.com/kubewarden/no-default-namespace-gatekeeper:v0.0.1"
  settings: {}
  rules:
    - apiGroups:
        - ""
      apiVersions:
        - "*"
      resources:
        - "*"
      operations:
        - CREATE
  mutating: false
----

Agora, use esse recurso `+ClusterAdmissionPolicy+` para implantar sua política em um cluster do Kubernetes.
