= Verificar imagens da coleção de aplicativos Rancher
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: Configure os PolicyServers para autenticar e extrair imagens do Rancher Application Collection usando um segredo de configuração do Docker.
:doc-persona: ["kubewarden-operator", "kubewarden-integrator"]
:doc-topic: ["operator-manual", "rancher", "installation"]
:doc-type: ["howto"]
:keywords: ["{project-name}", "rancher", "application collection", "appco", "signature", "verification", "verify"]
:sidebar_label: Verificar imagens
:current-version: {page-origin-branch}

Esta seção descreve como verificar aplicativos e imagens dahttps://apps.rancher.io/[coleção de aplicativos do] Rancher com {project-name}.

Você realiza a verificação da imagem implantando a políticahttps://artifacthub.io/packages/kubewarden/verify-image-signatures/verify-image-signatures[Verify Image Signatures] e configurando os PolicyServers para extrair as camadas de assinatura das imagens da Coleção de Aplicativos.

[IMPORTANT]
====

Você precisa de um token de acesso ou de autenticação de conta de serviço para a Coleção de aplicativos.

====


== Autenticação na coleção de aplicativos Rancher

Seguindo o sitehttps://docs.apps.rancher.io/get-started/authentication/[Documentação da Coleção de Aplicativos], crie um token de acesso e configure o cluster para extrair do registro da Coleção de Aplicativos, com um segredo de configuração do Docker:

[subs="+attributes",console]
----
$ kubectl create secret docker-registry application-collection \
  --docker-server=dp.apps.rancher.io \
  --docker-username=<mymail> \
  --docker-password=<mytoken>
----

== Criar segredo para o PolicyServer para que ele possa extrair da coleção de aplicativos Rancher

O PolicyServer em uso precisa de configuração para que possa extrair as camadas de imagem que contêm as assinaturas.

Siga o xref:/howtos/policy-servers/02-private-registry.adoc[tutorial sobre registros privados] para Policy Servers para criar um Docker Config Secret *no namespace PolicyServer*. Isso é feito instanciando um Secret com o mesmo conteúdo do `+application-collection+` anterior.

Para o PolicyServer `+default+`, instalado com o `+kubewarden-defaults+` Helm chart no namespace `+kubewarden+`, seria:

[subs="+attributes",console]
----
$ kubectl create secret docker-registry application-collection-kw -n kubewarden \
  --docker-server=dp.apps.rancher.io \
  --docker-username=<mymail> \
  --docker-password=<mytoken>
----

== Configure os PolicyServers para usar o novo segredo

Se estiver usando o PolicyServer `+default+` do gráfico do Helm `+kubewarden-defaults+`, configure-o com o imagePullSecret `+application-collection-kw+`. Isso ocorre no mesmo namespace e é feito com a definição dos seguintes valores para o gráfico:

[subs="+attributes",console]
----
$ helm upgrade -i --wait --namespace kubewarden \
  --create-namespace kubewarden-defaults kubewarden/kubewarden-defaults \
  --reuse-values \
  --set policyServer.imagePullSecret=application-collection-kw
----

Se você estiver usando outros PolicyServers, defina os respectivosxref:reference/CRDs.adoc#_policyserverspec[`+spec.imagePullSecret+`]. O Secret deve estar no mesmo namespace que o PolicyServer.

== Aplicar a política

Você pode aplicar uma ClusterAdmissionPolicy usando a políticahttps://artifacthub.io/packages/kubewarden/verify-image-signatures/verify-image-signatures[Verify Image Signatures]. Essa política verifica todas as imagens de contêiner de`+dp.apps.rancher.io/containers/*+`. A política é compatível com os registros e artefatos da OCI.

Você pode definir as configurações de política com ahttps://docs.apps.rancher.io/howto-guides/verify-signatures-with-kubewarden/[chave pública] da Coleção de Aplicativos (no momento em que este artigo foi escrito) na matriz `+pubKeys+`.

[subs="+attributes",console]
----
$ kubectl apply -f - <<EOF
apiVersion: policies.kubewarden.io/v1
kind: ClusterAdmissionPolicy
metadata:
  annotations:
    artifacthub/pkg: verify-image-signatures/verify-image-signatures/0.2.9
  name: check-appcollection-signatures
spec:
  backgroundAudit: true
  mode: protect
  module: ghcr.io/kubewarden/policies/verify-image-signatures:v0.2.9
  mutating: true
  policyServer: default
  # On first policy call, the policy-server downloads the image layers to
  # verify the signatures. Later on it's cached. It may be useful to increase
  # this timeout:
  timeoutSeconds: 30 # default 10 seconds.
  rules:
    - apiGroups: [""]
      apiVersions: ["v1"]
      resources: ["pods"]
      operations: ["CREATE", "UPDATE"]
    - apiGroups: [""]
      apiVersions: ["v1"]
      resources: ["replicationcontrollers"]
      operations: ["CREATE", "UPDATE"]
    - apiGroups: ["apps"]
      apiVersions: ["v1"]
      resources: ["deployments", "replicasets", "statefulsets", "daemonsets"]
      operations: ["CREATE", "UPDATE"]
    - apiGroups: ["batch"]
      apiVersions: ["v1"]
      resources: ["jobs", "cronjobs"]
      operations: ["CREATE", "UPDATE"]
  settings:
    modifyImagesWithDigest: true
    rule: PublicKey
    signatures:
      - image: dp.apps.rancher.io/containers/*
        pubKeys:
          # Note: this array constitutes an AND in validation, not an OR.
          - |-
            -----BEGIN PUBLIC KEY-----
            MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA02FtEt5gBywiyxbmkVsb
            CujcBg5lur0kpEbfDk10gCcs9shVEqEO3ZsOXHursgoaDAWqdPtsYhsgczGeJz9w
            Aw+r6BuRV8YOkE37A8s/7IOQUW0tlqtnt11OKhIiZ9+e5l3ed2H1ymKQO3dgreSy
            rShqYdA3hrItswyp41ApF6zhjSPlR6lAmq3X4wMYLAPptmzfxigTnR4hxB5UNPhs
            i2qA4vLrUM/i+NohECuLr1EAymvupH26HLEdM+eZnlQn+WbhIP5Grc4ba7XrBv7K
            kywgTC7CxkiJZR0mUcUD2wTX/Je8Ewj6oPSalx09e2jtzvmU5Kr9XUyMF7Zsj5CA
            IwIDAQAB
            -----END PUBLIC KEY-----
EOF
----

[subs="+attributes",console]
----
$ kubectl apply -f mypolicy.yml
$ kubectl get admissionpolicies -n default # wait for status active
----

Para testá-lo, implemente um Pod com uma imagem assinada da Coleção de aplicativos:

[subs="+attributes",console]
----
$ kubectl run nginx --image [dp.apps.rancher.io/containers/nginx:1.24.0](http://dp.apps.rancher.io/containers/nginx:1.24.0) --overrides='{"spec": {"imagePullSecrets":[{"name": "application-collection"}]}}'
 pod/nginx created
----

Você pode inspecionar os logs do seu Pod do servidor de políticas para ver se a verificação ocorreu.
