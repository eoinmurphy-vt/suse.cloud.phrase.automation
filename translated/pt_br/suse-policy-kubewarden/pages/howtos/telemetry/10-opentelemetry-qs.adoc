https://opentelemetry.io/[O OpenTelemetry] é uma estrutura da Cloud Native Computing Foundation (CNCF) para observabilidade. Ele permite que seus microsserviços forneçam métricas, registros e rastreamentos.

{project-name}usando o OpenTelemetry SDK, relatam dados a um coletor OpenTelemetry, chamado de agente.

Este guia explica como implantar o coletor OpenTelemetry no modo `+sidecar+` usando o gráfico oficial do Kubernetes Helm.

Esse é um padrão de implantação simples que usa o OpenTelemetry. Sua configuração final é semelhante a esta:

* Cada pod da pilha {short-project-name} (servidor de políticas, controlador) tem um sidecar OpenTelemetry.
* O sidecar recebe informações de rastreamento e monitoramento do componente {short-project-name} por meio do OpenTelemetry Protocol (OTLP)
* O coletor OpenTelemetry:
 ** Envia os eventos de rastreamento para uma instância central do Jaeger
 ** Expõe as métricas do Prometheus em uma porta específica

O gráfico do {short-project-name} Helm não abrange todos os cenários possíveis de implantação do coletor OpenTelemetry. Também é possível configurar o site{short-project-name} para enviar dados a um coletor OpenTelemetry. A documentação para esse cenário está no xref:./40-custom-otel-collector.adoc[guia personalizado do OpenTelemetry].

Primeiro, implante o OpenTelemetry em um cluster do Kubernetes, para que possa usá-lo nas seções a seguir, abordando especificamente o rastreamento e as métricas.

== Configuração de um cluster do Kubernetes

[NOTE]
====
Esta seção tem instruções passo a passo para criar um cluster do Kubernetes para as partidas rápidas de telemetria.

Sinta-se à vontade para pular esta seção se já tiver um cluster do Kubernetes disponível.
====

Você pode criar um cluster do Kubernetes para testes usando https://minikube.sigs.k8s.io/docs/[minikube].

O Minikube tem muitos backends; nesse caso, você pode usar o driverhttps://minikube.sigs.k8s.io/docs/drivers/kvm2/[] kvm2 que depende da libvirt.

Supondo que o site `+libvirtd+` esteja sendo executado corretamente em seu computador, emita o seguinte comando:

[subs="+attributes",console]
----
minikube start --driver=kvm2
----

O comando produz uma saída semelhante à seguinte:

=== Instalar o Traefik

Os guias de telemetria a seguir usam o https://traefik.io/traefik/[Traefik] como o controlador Ingress ao expor a interface do usuário do Jaeger com os recursos do Kubernetes `+Ingress+`.

Instale o Traefik usando seu gráfico Helm:

[,console]
----
helm repo add traefik https://traefik.github.io/charts
helm repo update

helm install --wait \
  --namespace traefik \
  --create-namespace \
  traefik traefik/traefik
----

Os guias a seguir criam um Kubernetes padrão `+Ingress+` com`+ingressClassName: traefik+` e roteiam a interface do usuário do Jaeger Query por meio do Traefik Service.

Se você preferir não instalar o Traefik, os guias de rastreamento também mantêm o site`+kubectl port-forward+` como um recurso para acessar o Jaeger diretamente.

== Instalar o OpenTelemetry

[NOTE]
====
Consulte a xref:/reference/dependency-matrix.adoc[matriz de dependência] para obter as versões mais recentes dos componentes compatíveis.
====

Você usa ohttps://github.com/open-telemetry/opentelemetry-operator[OpenTelemetry Operator] para gerenciar a injeção automática do sidecar do OpenTelemetry Collector no pod do PolicyServer.

O OpenTelemetry Operator requer a instalação dohttps://cert-manager.io/docs/installation/[cert-manager] no cluster.


[subs="+attributes",console]
----
helm repo add jetstack https://charts.jetstack.io

helm install --wait \
    --namespace cert-manager \
    --create-namespace \
    --set crds.enabled=true \
    --version 1.18.2 \
    cert-manager jetstack/cert-manager
----

Quando o cert-manager estiver em execução, você poderá instalar o gráfico Helm do operador OpenTelemetry:

[NOTE]
====

No momento em que este artigo foi escrito (2024-11-11), a versão mais recente do gráfico da operadora OpenTelemetry é `+0.65.0+`

====

[subs="+attributes",console]
----
helm repo add open-telemetry https://open-telemetry.github.io/opentelemetry-helm-charts

helm install --wait \
  --namespace open-telemetry \
  --create-namespace \
  --version 0.97.1 \
  --set "manager.collectorImage.repository=otel/opentelemetry-collector-contrib" \
  my-opentelemetry-operator open-telemetry/opentelemetry-operator
----
