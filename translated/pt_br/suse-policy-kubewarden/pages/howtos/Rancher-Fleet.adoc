Você pode gerenciar os gráficos do {project-name} Helm, como outros gráficos do Helm, com o https://fleet.rancher.io/[Fleet] Rancher.
O Rancher Fleet usa CRDs do Kubernetes para definir uma abordagem GitOps para gerenciar clusters do Kubernetes.
Ele faz isso definindo https://fleet.rancher.io/concepts[Fleet Bundles].

== Instalação

Os gráficos {short-project-name} são gráficos padrão e dependem transitivamente uns dos outros:

`+kubewarden-crds+` ← `+kubewarden-controller+` ← `+kubewarden-defaults+`

Consulte xref:/quick-start.adoc[os documentos de início rápido]para obter mais informações.

Usando o Rancher Fleet, você pode codificar as dependências do gráfico usando`+dependsOn+` no arquivo https://fleet.rancher.io/ref-fleet-yaml[`+fleet.yaml+`] arquivo.

Você poderá ver erros transitórios até que os gráficos estejam prontos, como:

[subs="+attributes",console]
----
ErrApplied(1) [Cluster fleet-local/local: dependent bundle(s) are not ready:
[kubewarden-example-helm-kubewarden-controller]]
----

Esses erros não significam um problema e, quando os gráficos terminam a implantação, eles não aparecem mais.

== Remoção

[CAUTION]
====

Ao remover o GitRepo, todos os 3 gráficos {short-project-name} são removidos de uma vez.
Isso significa que o gráfico `+kubewarden-crds+` é removido.

{short-project-name} usa um trabalho de gancho de helm pré-exclusão no gráfico `+kubewarden-controller+` que exclui o servidor de políticas padrão.
Esse gancho de pré-exclusão é necessário para desocupar os webhooks das políticas antes de excluir o PolicyServer.
Isso se aplica a qualquer mecanismo de política.
Caso contrário, o cluster pode ter webhooks para políticas que não existem mais, rejeitando tudo e ficando em um estado de falha.

Remover o GitRepo e, portanto, o gráfico `+kubewarden-crds+`, ao mesmo tempo que o gráfico `+kubewarden-controller+` faz com que o trabalho do gancho de pré-exclusão falhe.
Isso significa que a remoção está incompleta, deixando "detritos" no cluster.
====


A desinstalação automática de CRDs normalmente não é suportada por nenhuma ferramenta, e o Rancher Fleet não é exceção.

Para realizar uma remoção correta, certifique-se de remover primeiro o Bundle for `+kubewarden-defaults+` do cluster.
Para isso, faça o commit dessas alterações no repositório que contém a configuração do Fleet e aguarde até que elas sejam aplicadas.
Em seguida, remova `+kubewarden-controller+` da mesma forma e, por último, remova `+kubewarden-crds+`.

Outra opção é adicionar 2 GitRepos, um somente para os CRDs e outro para o restante dos gráficos {short-project-name}.
Isso permite que você remova os gráficos do {short-project-name} primeiro e os CRDs do {short-project-name} por último.
