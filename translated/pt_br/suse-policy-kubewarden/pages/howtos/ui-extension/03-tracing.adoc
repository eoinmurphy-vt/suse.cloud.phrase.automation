O rastreamento permite a coleta de detalhes minuciosos sobre avaliações de políticas.
É uma ferramenta útil para depurar problemas em sua implementação e políticas do {project-name}.

{short-project-name} usa https://www.jaegertracing.io/[o] para receber, armazenar e visualizar eventos de rastreamento.

*_Registros de rastreamento de políticas_*image:ui_policy_tracing.png[Registros de rastreamento de políticas] da interface do usuário

== Pré-requisitos

Você precisa do Cert-Manager e do OpenTelemetry. Sigaxref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[estas instruções] para instalar o Cert Manager e o OpenTelemetry Operator.

== Instalar o Jaeger

Aplique as etapas de instalação doxref:howtos/telemetry/20-tracing-qs.adoc#_install_jaeger[início rápido do rastreamento].

Depois que o operador do Jaeger cria todos os recursos, há um serviço em`+my-open-telemetry-collector.jaeger.svc.cluster.local+`.

A IU de consulta da Jaeger pode ser acessada no seguinte endereço:

[subs="+attributes",console]
----
https://<CLUSTER_IP>/api/v1/namespaces/jaeger/services/http:my-open-telemetry-query:16686/proxy/search
----

Esse endpoint pode ser exclusivo, dependendo de sua configuração. Você pode encontrar o endpoint listado para seu recurso Jaeger na página Serviços.

=== Atualize `+rancher-kubewarden-controller+` com o ponto de extremidade Jaeger

Você precisará editar o recurso `+rancher-kubewarden-controller+` para adicionar o ponto de extremidade do Jaeger "my-open-telemetry-collector.jaeger.svc.cluster.local:4317".

Por exemplo:

[subs="+attributes",yaml]
----
telemetry:
  tracing: true
  mode: sidecar
  sidecar:
    tracing:
      jaeger:
        endpoint: my-open-telemetry-collector.jaeger.svc.cluster.local:4317
        tls:
          insecure: true
----

A instalação do serviço, `+my-open-telemetry-collector+`, está sob o namespace`+jaeger+`.

[CAUTION]
====

Para simplificar, não estamos criptografando a comunicação entre o coletor OpenTelemetry e o ponto de extremidade Jaeger.

Isso não é *adequado para a implementação na produção*. Você deve ler do Jaegerhttps://www.jaegertracing.io/docs/latest/kubernetes/[a documentação].

====


Agora você deve conseguir visualizar todas as solicitações com falha de políticas vinculadas a um servidor de políticas específico. Você também pode ver a exibição de detalhes de uma determinada política.
Você pode obter uma visão mais detalhada dos rastros usando a interface do usuário do Jaeger.
