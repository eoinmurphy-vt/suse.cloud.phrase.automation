== Instalar a extensão {short-project-name} UI

Você instala a interface do usuário {short-project-name} como uma extensão global, mas instala o controlador {short-project-name} por meio da interface do usuário do Rancher como um recurso com escopo de cluster.

[NOTE]
====

Para instalações com passagem de ar, siga xref:howtos/airgap/02-install.adoc[estas etapas].

====


Na página Extensões, selecione o botão "Ativar" e escolha a opção para adicionar o Repositório de Extensões do Rancher. Quando ativado, o item de extensão "{short-project-name}" é exibido automaticamente. Selecione esse item para instalar a extensão. Depois de instalado, você pode instalar o {short-project-name} no cluster necessário.

=== Instalar {short-project-name}

Seguindo as etapas anteriores, em seu cluster, um novo item aparece no menu lateral para {short-project-name}. Esta página do painel o orienta no processo de instalação, completando os pré-requisitos.

[NOTE]
====

Durante a etapa "App Install" (Instalação do aplicativo) do assistente de instalação, o botão "Install{short-project-name}" (Instalar ) pode permanecer acinzentado. Se isso acontecer, atualize a página e volte para esta etapa.

====


== Pós-instalação

Após a conclusão da instalação, a página do painel e o menu lateral agora contêm novos itens, como Servidores de políticas, Políticas de admissão de cluster e Políticas de admissão. A partir daí, você pode criar Policy Servers e Policies para controlar o comportamento dentro do cluster.

*_Visualização do painel_*image:ui_dashboard.png[] UI

=== Ativação do servidor de políticas e das políticas padrão

Na página do painel, você pode selecionar o botão "Install Chart" (Instalar gráfico) para instalar o gráfico dohttps://github.com/kubewarden/helm-charts/tree/main/charts/kubewarden-defaults[`+kubewarden-defaults+`]gráfico do Helm. Esse gráfico inclui o Policy Server padrão e algumas políticas selecionadas.

Depois de instalar o gráfico, você pode visualizar os detalhes do Policy Server padrão com as políticas relacionadas em uma tabela classificável.

*_Visualização de detalhes do servidor de políticas_*image:ui_policyserver_detail.png[] UI

=== Criação de políticas

Ao criar políticas, você recebe inicialmente uma opção de "Política personalizada" na grade de políticas. Forneça as informações necessárias para o Nome, o Módulo e as Regras de sua apólice. Recomenda-se adicionar o repositório Policy Catalog para acessar as políticas oficiais do {short-project-name}.

*_Criação de uma política personalizada_*image:ui_policy_custom.png[Lista de permissões de políticas] da interface do usuário

== Instalação com passagem de ar

Como o {short-project-name} é uma extensão oficial do Rancher, a equipe do Rancher fornece um mecanismo para gerar automaticamente uma imagem de catálogo de extensão. Isso é adicionado ao arquivo `+rancher-images.txt+` quandohttps://ranchermanager.docs.rancher.com/getting-started/installation-and-upgrade/other-installation-methods/air-gapped-helm-cli-install/publish-images#1-find-the-required-assets-for-your-rancher-version[instalando o Rancher Manager] para instâncias com air-gap.

Depois que essa imagem for espelhada em um registro acessível ao seu cluster com air-gap, você poderá importar a imagem na interface do usuário do Rancher. Isso cria um repositório local do Helm com o gráfico da interface do usuário {short-project-name} para instalação.

=== Etapas de instalação

. https://ranchermanager.docs.rancher.com/how-to-guides/new-user-guides/kubernetes-resources-setup/secrets[Crie]um segredo de registro no espaço de nomes `+cattle-ui-plugin-system+`. Digite o domínio do endereço da imagem no campo *Nome do domínio do registro*.
. Navegue de volta para a página *Extensões* (por exemplo, `+https://cluster-ip/dashboard/c/local/uiplugins+`).
. No canto superior direito, selecione *Manage Extension Catalogs (Gerenciar catálogos de extensão*). image:ui_airgap_01.png[Manage Catalogs]
. Selecione o botão *Importar catálogo de extensão*. \image:ui_airgap_02.png[Import Catalogs].
. Digite o endereço da imagem no campo *Catalog Image Reference (Referência da imagem do catálogo* ).
. Selecione o segredo que você acabou de criar no menu suspenso *Pull Secrets*. image:ui_airgap_03.png[Inserir informações do catálogo]
. Clique em *Load*. A extensão agora será *Pending*.
. Retorne à página *Extensões*.
. Selecione a guia *Available (Disponível* ) e clique no botão *Reload (Recarregar* ) para garantir que a lista de extensões esteja atualizada. image:ui_airgap_04.png[Install {short-project-name}]
. Localize a extensão {short-project-name} que você acabou de adicionar e selecione o botão *Install (Instalar* ).
