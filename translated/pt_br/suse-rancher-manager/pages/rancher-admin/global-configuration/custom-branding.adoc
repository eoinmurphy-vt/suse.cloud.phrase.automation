[#_prerequisite]
== Pré-requisitos

Você precisará ter pelo menos permissões de membro do cluster.

[#_brand_configuration]
== Configuração da Marca

Para configurar as configurações da marca,

. Clique em *☰ > Configurações Globais*.
. Clique em *Branding*.

[#_private_label_company_name]
=== Nome da Empresa de Marca Privada

Esta opção substitui "Rancher" pelo valor que você fornecer na maioria dos lugares. Arquivos que precisam ter Rancher no nome, como `+rancher-compose.yml+`, não serão alterados.

[#_support_links]
=== Links de Suporte

Use um endereço de url para enviar novos relatórios de "Registrar um Problema" em vez de enviar os usuários para a página de problemas do GitHub. Opcionalmente, mostre links de suporte da comunidade Rancher.

[#_logo]
=== Logotipo

Carregue logotipos claros e escuros para substituir o logotipo do Rancher no cabeçalho de navegação de nível superior.

[#_primary_color]
=== Cor Primária

Você pode substituir a cor primária usada em toda a interface do usuário por uma cor personalizada de sua escolha.

[#_fixed_banners]
=== Banners Fixos

Exiba um banner fixo personalizado no cabeçalho, rodapé ou em ambos.

Para configurar as configurações do banner,

. Clique em *☰ > Configurações Globais*.
. Clique em *Banners*.

[#_custom_navigation_links]
== Links de Navegação Personalizados

Nesta seção, você aprenderá como configurar os links na barra de navegação à esquerda do *Cluster Dashboard*. Para acessar o painel do cluster,

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Na página *Clusters*, vá para o cluster onde você deseja links de navegação personalizados e clique em *Explorar*.

Pode ser útil adicionar um link para acesso rápido aos serviços instalados em um cluster. Por exemplo, você pode adicionar um link para a interface do Kiali para clusters com Istio instalado, ou pode adicionar um link para a interface do Grafana para clusters com monitoramento do Rancher instalado.

Os links personalizados não afetam quem tem acesso a cada serviço.

Links podem ser criados no nível superior e múltiplos links podem ser agrupados juntos.

[#_adding_a_custom_navigation_link]
=== Adicionando um Link de Navegação Personalizado

[NOTE]
.Pré-requisitos:
====

Você precisará ter pelo menos permissões de membro do cluster ou membro do projeto.
====


. Clique em *☰ > Gerenciamento de Cluster*.
. Na página *Clusters*, vá para o cluster onde você gostaria de adicionar links de navegação personalizados e clique em *Explorar*.
. No menu de navegação superior, clique em *🔍 (Pesquisa de Recursos)*.
. Digite *Nav* e clique em *Links de Nav*.
. Clique em *Criar a partir do YAML*.
. A maneira mais simples de criar um link de navegação é adicionar estes campos:

  name: linkname
  toURL: https://example.com
+
For more details on setting up links, including optional fields, see <<_link_configuration,Link Configuration.>>

. Clique em *Criar*.

[#_link_configuration]
== Configuração de Link

[#__name]
=== `+name+`

Nome exibido para o link. Obrigatório.

[#__group]
=== `+group+`

Nome de um grupo de links que expande quando clicado.

Opcional. Se não fornecido, o link aparece de forma independente.

Grupos são exibidos separadamente de links independentes, como mostrado abaixo:

=== `+iconSrc+`

[#__annotation]
=== `+annotation+`

Uma anotação do Kubernetes para o recurso personalizado NavLink.

[#__label]
=== `+label+`

Um rótulo do Kubernetes para o recurso personalizado NavLink.

[#__sidelabel]
=== `+sideLabel+`

Rótulo que aparece na barra de navegação à esquerda

[#__target]
=== `+target+`

Define a propriedade target da tag âncora do link (`+<a>+`), que (dependendo dos navegadores) determina se ele abre em uma nova janela ou em uma guia existente.

O valor padrão é `+_self+`, que abre o link na guia atual. Para abrir o link em uma nova janela ou guia, defina o target como `+_blank+`.

Para mais informações sobre a propriedade target, veja https://www.w3schools.com/tags/att_a_target.asp[esta página.]

[#__toservice]
=== `+toService+`

Possui cinco campos que são construídos para criar uma URL como a seguinte: `+\https://<rancher_server_url>/k8s/clusters/<CLUSTER>/k8s/namespace/<NAMESPACE>/service/<SCHEME>:<NAME>:<PORT>/proxy/<PATH>+`

Por exemplo, um link para um serviço de monitoramento pode ser configurado da seguinte forma:

* nome: `+rancher-monitoring-grafana+`
* namespace: `+cattle-monitoring-system+`
* caminho: `+proxy/?orgId=1+`
* porta: `+"80"+`
* esquema: `+http+`

É necessário fornecer a diretiva `+toService+` ou a diretiva `+toURL+`.

[#__tourl]
=== `+toUrl+`

Pode ser qualquer link, até mesmo links fora do cluster.

É necessário fornecer a diretiva `+toService+` ou a diretiva `+toURL+`.

[#_link_examples]
== Exemplos de links

[#_example_of_link_with_tourl]
=== Exemplo de link com `+toUrl+`

Este exemplo de YAML do NavLink mostra um exemplo de configuração de um NavLink para um painel do Grafana:

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  name: grafana
spec:
  group: "Monitoring Dashboards"
  toURL: https://<RANCHER_SERVER_URL>/api/v1/namespaces/cattle-monitoring-system/services/http:rancher-monitoring-grafana:80/proxy/?orgId=1
----

Adicionar o YAML acima resulta na criação de um link para o Grafana, como mostrado na seguinte captura de tela:

image::example-grafana-link.png[Captura de tela do Link do Grafana]

[#_example_of_link_with_toservice]
=== Exemplo de link com `+toService+`

Este exemplo de YAML mostra um exemplo de `+toService+` usado como destino do link:

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  annotations:
    key: annotation
    labels:
      key: label
  name: navlinkname
spec:
  description: This is a description field # Optional.
  group: "group1" # Optional. If not provided, the links appear standalone.
  iconSrc: data:image/jpeg;base64,[icon source string is clipped for brevity]
  label: This is a label # Optional.
  sideLabel: A side label. # Optional.
  target: _blank #Optional. _blank opens the link in a new tab or window.
  toService: # toService or #toUrl needs to be provided.
    name: rancher-monitoring-grafana
    namespace: cattle-monitoring-system
    path: proxy/?orgId=1
    port: "80"
    scheme: http
----

Adicionar os parâmetros `+toService+` acima resulta na criação de um link para o Grafana, como mostrado na seguinte captura de tela:

image::example-service-link.png[Captura de tela do Link do Grafana]
