[NOTE]
====
As instruções a seguir precisam do Rancher 2.9.x, no mínimo.
====

Este início rápido mostrará como implantar o plug-in {elemental-product-name} e o operador em uma instância existente do Rancher Manager.

Depois de instalado, você poderá provisionar um novo cluster {elemental-product-name} com base no RKE2 ou K3s.

No entanto, se você quiser instalar o operador de teste ou de desenvolvimento, só poderá fazer isso no xref:quickstarts/quickstart-cli.adoc#_non_stable_installations[modo CLI].

== Adicionar o repositório oficial de extensões do Rancher

Se a extensão Elemental não estiver disponível, você precisará adicionar o `+Official Rancher Extensions Repository+`:

image::quickstart-ui-extension-repository.png[Adicionar o repositório de extensões do Rancher Manager]

Se esse repositório não puder ser adicionado por meio das configurações da interface do usuário de extensões, ele poderá ser gerenciado manualmente pelo https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-admin/helm-charts-in-rancher/helm-charts-in-rancher.html#_manage_repositories[site] o repositório `+https://github.com/rancher/ui-plugin-charts+` `+git+` ou, alternativamente, aplicando o seguinte recurso:

[source,yaml]
----
apiVersion: catalog.cattle.io/v1
kind: ClusterRepo
metadata:
  name: rancher-ui-charts
spec:
  gitBranch: main
  gitRepo: https://github.com/rancher/ui-plugin-charts
----

== Instale o plug-in elementar

Depois que o suporte a extensões do Rancher Manager estiver ativado, você poderá instalar o plug-in `+elemental+` da seguinte forma:

* Na guia `+Available+`, você verá o plug-in `+elemental+` disponível

image::quickstart-ui-extensions-available.png[Plug-ins disponíveis do Rancher Manager]

[NOTE]
====
Se a guia `+Available+` não mostrar nenhuma entrada, atualize a página. O plug-in `+elemental+` será exibido.
====

* Clique no botão `+Install+`, uma janela pop-up será exibida e clique em `+Install+` novamente para continuar.

image::quickstart-ui-elemental-plugin-install.png[{elemental-product-name} instalação do plugin]

* Na guia `+Installed+`, o plug-in `+elemental+` agora está listado.

[NOTE]
====
Se o plug-in `+elemental+` estiver listado e o status permanecer em `+Installing...+`, atualize a página. O plug-in `+elemental+` será exibido corretamente.
====

Depois que o plug-in `+elemental+` for instalado, você poderá ver a opção `+OS Management+` no menu do Rancher Manager; atualize a página se não a vir.

image::quickstart-ui-elemental-plugin-menu.png[Menu de gerenciamento do sistema operacional do Rancher Manager]

== Instalar o operador elementar

[NOTE]

image::quickstart-ui-extension-operator-button.png[Botão para acionar o operador elementar]

Ele o redirecionará para o Rancher Marketplace para instalar o operador.

Clique no botão `+Next+`:

image::quickstart-ui-extension-operator-install-1.png[Instalar o {elemental-product-name} operator captura de tela 1]

Nessa tela, você pode personalizar ou usar os valores padrão. Clique em `+Install+` para continuar:

image::quickstart-ui-extension-operator-install-2.png[Instalar o operador {elemental-product-name} captura de tela 2]

Você deverá ver ``+elemental-operator-crds++ e`` +`elemental-operator`+ implantados no namespace `+cattle-elemental-system`:

image::quickstart-ui-extension-operator-install-3.png[Instalar o operador {elemental-product-name} captura de tela 3]

====
Se você não os vir, certifique-se de selecionar o namespace correto na parte superior da página.

== Adicionar um ponto de extremidade de registro de máquina

No painel do OS Management, clique no botão `+Create Registration Endpoint+`.

image::quickstart-ui-registration-endpoint-create.png[Pontos de extremidade de registro do gerenciamento do sistema operacional]

Agora, aqui você pode inserir cada detalhe em seus respectivos lugares ou pode editar isso como YAML e criar o endpoint de uma só vez. Aqui, editaremos todos os campos.

image::quickstart-ui-registration-endpoint-create-details.png[Criar um ponto de extremidade de registro com a interface do usuário]

[IMPORTANT]
.Opções principais
====
`+name: elemental-cluster1+`: altere isso de acordo com sua necessidade

`+device-selector+`: O campo xref:references/machineregistration-reference.adoc#_config_elemental_install_device_selector[seletor de dispositivo] pode ser usado para escolher dinamicamente o dispositivo durante a instalação. O campo contém uma lista de regras para selecionar o dispositivo desejado.

`+snapshotter+`: Tipo de dispositivo usado para gerenciar instantâneos em imagens do sistema operacional.
====

Depois de criar o ponto final de registro da máquina, ele deverá aparecer como ativo.

image::quickstart-ui-registration-endpoint-complete.png[Máquina registrada nos pontos de extremidade de registro]

== Preparando a imagem de instalação (seed)

Agora, esta é a última etapa: você precisa preparar uma imagem semente que inclua a configuração de registro inicial, para que ela possa ser registrada automaticamente, instalada e totalmente implantada como parte do seu cluster. O conteúdo do arquivo nada mais é do que o URL de registro que o nó precisa registrar e o certificado de servidor adequado, para que ele possa se conectar com segurança.

Essa imagem semente pode então ser usada para provisionar um número infinito de máquinas.

A imagem semente é criada como um recurso do Kubernetes acima e pode ser criada usando o botão `+Build Media+`, mas, primeiro, você precisa selecionar a imagem ISO ou RAW.

Ao contrário da ISO, em que são necessários dois dispositivos (dispositivo com ISO e outro disco para instalar {elemental-product-name}), a imagem RAW permite inicializar a partir de um único dispositivo e instalar diretamente o sistema operacional no dispositivo.
A imagem RAW contém apenas uma partição de inicialização e uma de recuperação e é inicializada primeiro no modo de recuperação para instalar o {elemental-product-name} (para fins informativos, o processo é semelhante ao de xref:node-operational-tasks/reset.adoc#_reset_workflow[reinicialização] ).

image::quickstart-ui-registration-endpoint-build-media.png[Criar mídia nos pontos de extremidade de registro]

Quando a compilação estiver concluída, o download da mídia poderá ser feito usando o botão `+Download Media+`:

image::quickstart-ui-registration-endpoint-download-media.png[Download de mídia em pontos de extremidade de registro]

Agora você pode inicializar seus nós com essa imagem e eles o farão:

* Registre-se com o registrationURL fornecido e crie um registro por máquina `+MachineInventory+`
* Instalar o SLE Micro no dispositivo fornecido
* Reinicialização

== Inventário de máquinas

Quando os nós estão sendo inicializados pela primeira vez, eles se conectam ao Rancher Manager e um xref:references/machineinventory-reference.adoc[`+Machine Inventory+`] é criado para cada nó.

image::quickstart-ui-machine-inventory-menu.png[Menu Inventário de máquinas]

As colunas personalizadas são baseadas em `+Machine Inventory Labels+`, que você pode adicionar ao criar seu `+Machine Registration Endpoint+`:

image::quickstart-ui-registration-endpoint-hardware-labels.png[Etiquetas de hardware do endpoint de registro da máquina]

Na captura de tela a seguir, `+Hardware Labels+` é usado como colunas personalizadas:

Você também pode adicionar colunas personalizadas clicando no menu de três pontos.

image::quickstart-ui-machine-inventory-custom-columns.png[Colunas personalizadas do inventário de máquinas]

Por fim, você também pode filtrar seu `+Machine Inventory+` usando esses rótulos.

Por exemplo, se quiser ver apenas as máquinas AMD, você pode filtrar em `+CPUModel+`, como abaixo:

image::quickstart-ui-machine-inventory-filtering.png[Filtragem de inventário de máquinas]

== Crie seu primeiro cluster {elemental-product-name} 

Agora vamos usar esses `+Machine Inventory+` para criar um cluster clicando em `+Create {elemental-product-name} Cluster+`:

image::quickstart-ui-create-cluster-button.png[Botão Create {elemental-product-name} Cluster (Criar cluster )]

Para seu cluster {elemental-product-name}, você pode escolher K3s ou RKE2 para Kubernetes.

image::quickstart-ui-create-cluster-standard-screen-.png[{elemental-product-name} Tela de criação de cluster]

A maioria das opções vem do Rancher, por isso não detalharemos todas as possibilidades.
Sinta-se à vontade para consultar a https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/k3s.html[documentação de configuração da distribuição do] Rancher ou a https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/rke2.html[] Rancher se quiser saber mais.

No entanto, é importante destacar a seção `+Inventory of Machines Selector Template+`.

Ele permite que você escolha qual `+Machine Inventory+` deseja usar para criar o cluster {elemental-product-name} usando o `+Machine Inventory Labels+` definido anteriormente:

image::quickstart-ui-create-cluster-machine-selector-template.png[Usar o modelo de seletor de inventário de máquina]

Como nossos três inventários de máquinas contêm o rótulo `+CPUVendor+` com a chave `+AuthenticAMD+`, as três máquinas serão usadas para criar o cluster {elemental-product-name}.
