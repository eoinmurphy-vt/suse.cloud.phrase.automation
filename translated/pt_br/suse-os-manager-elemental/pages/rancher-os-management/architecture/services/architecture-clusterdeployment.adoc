= Provisionamento de cluster do Kubernetes
:revdate: 2025-11-06
:page-revdate: {revdate}

O objetivo da fase de implantação do cluster do Kubernetes é criar um novo cluster RKE2 ou K3s usando os xref:references/machineinventory-reference.adoc[MachineInventories] disponíveis, ou seja, os hosts que concluíram com êxito a fase xref:rancher-os-management/architecture/services/architecture-machineonboarding.adoc[de integração de máquinas].

A implantação do cluster do Kubernetes em {elemental-product-name} envolve as seguintes etapas:

* O usuário cria um recurso xref:references/machineinventoryselectortemplate-reference.adoc[MachineInventorySelectorTemplate]: ele permite definir um _seletor_ para identificar um subconjunto dos xref:references/machineinventory-reference.adoc[MachineInventories] disponíveis com base no valor de seus rótulos.
* O usuário define um xref:references/cluster-reference.adoc[cluster Rancher] e adiciona à definição `+machinePools+` uma referência ao xref:references/machineinventoryselectortemplate-reference.adoc[MachineInventorySelectorTemplate] criado na etapa anterior.
* O https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/launch-kubernetes-with-rancher.html[provisionamento do cluster] Rancher reage à criação de recursos do cluster Rancher gerando um número de recursos xref:references/machineinventoryselector-reference.adoc[MachineInventorySelectors] igual à _quantidade_ especificada nos _machinePools_.
* O operador {elemental-product-name} emparelha cada recurso xref:references/machineinventoryselector-reference.adoc[MachineInventorySelector] gerado com um xref:references/machineinventory-reference.adoc[MachineInventory] disponível e instala o daemon https://github.com/rancher/system-agent[] rancher-system-agent no host monitorado pelo xref:references/machineinventory-reference.adoc[MachineInventory].
O https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/launch-kubernetes-with-rancher.html[provisionamento do] Rancher assume o provisionamento do K3s/RKE2 usando _os planos_ do https://github.com/rancher/system-agent[] rancher-system-agent: ele instala os componentes necessários (por exemplo, containerd, K3s, ...) e cria os arquivos de configuração até a implantação bem-sucedida do novo cluster do Kubernetes.
