= Integração de máquinas
:revdate: 2025-11-06
:page-revdate: {revdate}

O objetivo do serviço Machine Onboarding é registrar uma nova máquina no catálogo do operador {elemental-product-name} xref:references/machineinventory-reference.adoc[(] recursosxref:references/machineinventory-reference.adoc[MachineInventory] ) e provisionar o host com o sistema operacional, aplicando a configuração desejada.

O Machine Onboarding envolve as seguintes etapas:

* O usuário cria um recurso xref:references/machineregistration-reference.adoc[MachineRegistration]: o MachineRegistration inclui a configuração necessária para a instalação do sistema operacional.
Assim que o MachineRegistration é criado, o operador do {elemental-product-name} expõe um _ponto de extremidade de registro_ HTTP: é o ponto de entrada exigido pelos hosts de integração para se registrar no operador.
* O usuário solicita uma imagem de autoinstalação por meio de um recurso xref:references/seedimage-reference.adoc[SeedImage]: ele requer apenas uma referência a um recurso MachineRegistration e a uma imagem de contêiner de sistema operacional compatível com o {elemental-product-name} Toolkit.
Assim que a SeedImage é criada, o operador do site {elemental-product-name} aciona o processo de compilação do recurso de image: once completed, the URL is exposed in the `+status.downloadURL+` field of the xref:references/seedimage-reference.adoc[SeedImage do site], que é auto-instalável.
* O usuário faz download da imagem de autoinstalação e a utiliza para inicializar um host não provisionado: o host https://documentation.suse.com/cloudnative/os-manager/latest/en/rancher-os-management/authentication.html[se autentica] no _endpoint de registro_ do Operador {elemental-product-name}, obtém a configuração completa armazenada no MachineRegistration e instala o sistema operacional no dispositivo de armazenamento local. Assim que o host concluir o processo de registro, o operador do {elemental-product-name} criará um recurso xref:references/machineinventory-reference.adoc[MachineInventory] exclusivo para rastrear o host.
A imagem de autoinstalação pode ser usada para integrar qualquer número de hosts.
