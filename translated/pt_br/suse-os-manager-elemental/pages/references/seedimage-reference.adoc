= Referência de SeedImage
:revdate: 2025-09-08
:page-revdate: {revdate}


Um recurso `+SeedImage+` permite criar uma mídia de instalação que pode ser usada para instalar o {elemental-product-name} em um nó.

Ele requer um `+baseImage+`, ou seja, um URL para uma imagem ISO de instalação ou de contêiner de nó {elemental-product-name} e uma referência `+registrationRef+` para um recurso `+MachineRegistration+`, do qual a parte de registro da configuração {elemental-product-name} é extraída e injetada na mídia para produzir a _imagem_ final da _semente_.

Também é possível injetar personalizações no campo `+cloud-config+`. Há suporte para as sintaxes yip e cloud-init. Consulte a xref:references/cloud-config-reference.adoc[Referência de configuração da nuvem] para obter informações completas.

Quando a imagem inicial estiver pronta, o URL de download será compartilhado no campo `+.status.downloadURL+`.
Ele permanece disponível para download por `+cleanupAfterMinutes+` minutos (o padrão é `+60+`, 1 hora), após o que é excluído.

A configuração de `+retriggerBuild+` para `+true+` reinicia o processo de criação da imagem semente, enquanto a configuração de `+cleanupAfterMinutes+` para `+0+` mantém a imagem semente até que o recurso `+SeedImage+` seja excluído.

O recurso `+SeedImage+` também tem um campo `+type+` que pode ser definido como `+iso+`, para criar uma ISO, ou `+raw+`, para criar uma imagem de disco bruta. As imagens de disco brutas podem ser copiadas diretamente para a unidade de destino e, na primeira inicialização, serão automaticamente inicializadas em uma partição de recuperação para expandir a unidade e usar o espaço em disco disponível e registrar o nó, após o que serão reinicializadas da mesma forma que na instalação da ISO.

Se nenhum `+BuildContainer+` for especificado para a imagem-semente, ele será preenchido automaticamente com base nos valores padrão e `+type+`.

A criação de uma SeedImage para uma plataforma diferente é realizada usando o campo `+targetPlatform+`. A plataforma é especificada usando `+os/arch+`, por exemplo (`+linux/x86_64+` ou `+linux/aarch64+`). Por padrão, a imagem será criada para a mesma plataforma em que o operador está hospedado.

[CAUTION]
.as imagens semente podem ocupar o armazenamento local
====
As imagens-semente são mantidas no armazenamento local do nó: preste atenção ao número de recursos do `+SeedImage+` que você inicia simultaneamente e aos que você pode deixar com o recurso de limpeza automática desativado (`+cleanupAfterMinutes+` = `+0+`), pois você pode esgotar o armazenamento nos nós do cluster.
====

== Referência do SeedImageSpec

|===
| Chave | Tipo | Valor padrão | Descrição

| baseImage
| string
| vazio
| A imagem elementar de base usada para criar a imagem-semente.

| registrationRef
| referência do objeto.
| nulo
| Uma referência a um MachineRegistration que será usado para o registro de todas as máquinas instaladas.

| buildContainer
| object
| nulo
| Configurações de um contêiner personalizado usado para gerar a imagem para download. (Consulte <<_buildcontainer,a documentação>>).

| cleanupAfterMinutes
| int
| 60
| O tempo após o qual a imagem-semente construída será limpa. Os downloads ativos serão concluídos antes que a imagem seja removida.

| retriggerBuild
| bool
| falso
| Acionador para criar novamente uma imagem-semente limpa.

| tamanho
| string
| 6Gi
| Especifica o tamanho do volume usado para armazenar a imagem.

| tipo
| string
| iso
| Especifica o tipo de imagem semente a ser criada. `+iso+` ou `+raw+`. (Consulte <<_iso_and_raw_images,a documentação>>)

| targetPlatform
| string
| vazio
| Especifica a plataforma de destino para a imagem construída. Exemplo: `+linux/x86_64+` ou `+linux/aarch64+`. (Consulte <<_multi_platform_support,a documentação>>).

| cloud-config
| object
| nulo
| Contém dados de configuração de nuvem a serem incluídos na imagem gerada. (Consulte xref:references/cloud-config-reference.adoc[a documentação]).

|===

=== BuildContainer

As configurações do `+buildContainer+` podem ser usadas para personalizar o contêiner de inicialização do `+build+` no pod do `+SeedImage+`.
Esse pode ser o caso, por exemplo, ao criar imagens Elemental personalizadas.

[,yaml]
----
buildContainer:
  name: "custom-build"
  image: my.registry.com/elemental-custom-builder:1.2.3
  command:
    - build-image
  args:
    - foo
    - bar
  imagePullPolicy: Always
----

Observe que o contêiner também terá dois volumes montados em `+/iso+` e `+/overlay+`.
O processo de compilação do SeedImage espera que o contêiner de compilação coloque o artefato de compilação em `+/iso/$(ELEMENTAL_OUTPUT_NAME)+`.

Os arquivos de configuração estão disponíveis em:

- `+/overlay/reg/livecd-cloud-config.yaml+`: Um arquivo de configuração que pode ser usado por `+elemental-register+` para registrar a máquina.

- `+/overlay/iso-config/cloud-config.yaml+`: A configuração de nuvem definida em `+SeedImage.spec.cloud-config+`

A seguinte lista de variáveis de ambiente também pode ser usada no contêiner de compilação personalizado:

- `+ELEMENTAL_DEVICE+`: O valor `+MachineRegistration.spec.config.elemental.install.device+`.
- `+ELEMENTAL_REGISTRATION_URL+`: O URL exclusivo do MachineRegistration.
- `+ELEMENTAL_BASE_IMAGE+`: A imagem de base definida no site `+SeedImage+`.
- `+ELEMENTAL_OUTPUT_NAME+`: O nome de arquivo esperado do artefato de compilação.

=== Imagens ISO e Raw

O `+SeedImage+` é capaz de criar tipos de imagem `+iso+` ou `+raw+`.
Observe que o Elemental envia dois tipos diferentes de imagens, `+iso+` ou `+container+`. Consulte o xref:references/managedosversion-reference.adoc#_managedosversionspec_reference[tipo de ManagedOSversion].

Ao criar um `+iso+` `+SeedImage+`, você pode usar uma imagem `+iso+` Elemental. As imagens`+iso+` contêm um artefato `+.iso+` pré-construído. Essa é a maneira padrão da Elemental de enviar ISOs oficiais, para que não precisem ser reconstruídas toda vez que você definir um `+SeedImage+`.

.Clique aqui para ver um exemplo de ISO SeedImage
[%collapsible]
====
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: fire-iso
  namespace: fleet-default
spec:
  type: iso
  baseImage: registry.suse.com/suse/sl-micro/6.0/baremetal-iso-image:2.1.1-3.36
  registrationRef:
    apiVersion: elemental.cattle.io/v1beta1
    kind: MachineRegistration
    name: fire-nodes
    namespace: fleet-default
----
====

Como alternativa, ao criar um `+raw+` `+SeedImage+`, você deve usar as imagens do `+container+` Elemental. Essas imagens também são usadas durante o processo de atualização (consulte: xref:references/managedosimage-reference.adoc[ManagedOSImage]), mas também pode ser usado para criar o site `+raw+` `+SeedImages+`.

.Clique aqui para ver um exemplo de Raw SeedImage
[%collapsible]
====
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: fire-raw
  namespace: fleet-default
spec:
  type: raw
  baseImage: registry.suse.com/suse/sl-micro/6.0/baremetal-os-container:2.1.1-3.29
  registrationRef:
    apiVersion: elemental.cattle.io/v1beta1
    kind: MachineRegistration
    name: fire-nodes
    namespace: fleet-default
----
====

=== Suporte a várias plataformas

A Elemental envia as imagens `+linux/x86_64+` e `+linux/aarch64+` para a maioria dos sabores.
Para determinar se uma imagem `+ManagedOSVersion+` é compatível com ambas as plataformas, você pode verificar os valores `+ManagedOSVersion.spec.metadata.platform+`. (Consulte xref:references/managedosversion-reference.adoc#_metadata[a documentação]).

Ao definir um `+SeedImage+`, você pode usar esse valor para o `+targetPlatform+` da imagem.
Deixando o `+targetPlatform+` vazio, o padrão será a plataforma em que o `+elemental-operator+` está sendo executado.

.Clique aqui para ver um exemplo de Raw aarch64 SeedImage
[%collapsible]
====
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: fire-raw-aarch64
  namespace: fleet-default
spec:
  targetPlatform: linux/aarch64
  type: raw
  baseImage: registry.suse.com/suse/sl-micro/6.0/baremetal-os-container:2.1.1-3.29
  registrationRef:
    apiVersion: elemental.cattle.io/v1beta1
    kind: MachineRegistration
    name: fire-nodes
    namespace: fleet-default
----
====

== URLs para download

O recurso `+SeedImage+` rastreia o processo de criação da imagem-semente por meio de duas condições de status:

- **Pronto**: rastreia a criação de todos os recursos secundários necessários que executam o processo de compilação real.
- **SeedImageReady**: rastreia o status do processo de compilação nos recursos secundários.

Como alternativa, também é possível esperar que o pod `+SeedImage+` fique pronto:

[,bash]
----
kubectl wait --for=condition=ready pod -n fleet-default fire-img
----

Aguardar as condições de Pronto é uma prática recomendada antes de fazer o download de qualquer artefato.

Quando o `+SeedImage+` estiver pronto, o `+.status.downloadURL+` conterá o URL para download.
Observe que a URL usará o mesmo endpoint do Rancher, portanto, tenha cuidado com a validação de HTTPS ao usar certificados autoassinados.

[,bash]
----
kubectl get seedimage -n fleet-default fire-img -o jsonpath="{.status.downloadURL}"
----

A soma de verificação da imagem também está disponível para verificar se o download foi correto:

[,bash]
----
kubectl get seedimage -n fleet-default fire-img -o jsonpath="{.status.checksumURL}"
----
