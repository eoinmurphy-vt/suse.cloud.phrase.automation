= Como usar o {elemental-product-name} com o Rancher e a VMware
:revdate: 2025-09-08
:page-revdate: {revdate}

== Excerto

Neste documento, veremos como configurar uma máquina virtual na estação de trabalho VMware para inicializar os nós do Elemental.

== Pré-requisitos

Rancher 2.8 ou superior instalado e em execução. Consulte os guias de início rápido.

== Etapa 1: Criar o ponto final de registro

Precisamos criar uma imagem ISO para inicializar os nós e registrar na instância do Rancher. Para isso, é necessário um ponto final de registroxref:references/machineregistration-reference.adoc[(] recursoxref:references/machineregistration-reference.adoc[MachineRegistration] ).

Veja este exemplo de `+MachineRegistration+`:

[,yaml]
.registration.yaml
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: fire-nodes
  namespace: fleet-default
spec:
  config:
    cloud-config:
      users:
        - name: root
          passwd: root
    elemental:
      install:
        reboot: true
        device: /dev/sda
        debug: true
  machineInventoryLabels:
    element: fire
    manufacturer: "${Product/Vendor}"
    productName: "${Product/Name}"
    serialNumber: "${Product/Serial Number}"
    machineUUID: "${Product/UUID}"
----

O MachineRegistration acima pressupõe que os nós incluem o TPM 2.0. Caso a máquina de destino virtualizada não inclua um dispositivo TPM virtual, uma emulação de software poderá ser configurada na seção `+config.elemental.registration+`.

Considere o exemplo a seguir:

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  ...
spec:
  config:
    cloud-config:
      ...
    elemental:
      install:
        ...
      registration:
        emulate-tpm: true
        emulated-tpm-seed: -1
  machineInventoryLabels:
    ...
----

`+emulated-tpm-seed: -1+` define o cliente para usar uma semente aleatória para calcular o hash do TPM, o que é útil para poder reutilizar a mesma definição de ponto final de registro para várias máquinas. Consulte xref:operator-operational-tasks/tpm.adoc[a documentação] adicional xref:operator-operational-tasks/tpm.adoc[do TPM].

== Etapa 2: Criar a ISO de instalação

A mídia de instalação precisa estar vinculada a um ponto final de registro específico. Ele é criado e tratado com o recurso xref:references/seedimage-reference.adoc[SeedImage].

Considere o exemplo a seguir:

[,yaml]
.seedimage.yaml
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: fire-img
  namespace: fleet-default
spec:
  baseImage: registry.suse.com/suse/sle-micro-iso/5.5:2.0.2
  registrationRef:
    apiVersion: elemental.cattle.io/v1beta1
    kind: MachineRegistration
    name: fire-nodes
    namespace: fleet-default
----

Depois que o recurso SeedImage é criado, ele começa a criar uma ISO com a imagem do sistema operacional fornecida e a vinculá-la ao ponto final de registro fornecido. Uma vez feito isso, um URL de download estará disponível no status do recurso SeedImage.

Você pode baixá-lo com:

[,shell]
----
wget --no-check-certificate --content-disposition $(kubectl get seedimages.elemental.cattle.io -n fleet-default fire-img -o jsonpath="{.status.downloadURL}")
----

== Etapa 3: Inicializar o dispositivo de destino

Agora, o ideal seria gravar a ISO em uma unidade USB e inicializar o dispositivo de borda usando o dispositivo USB e, depois que ele inicializar e se tornar ativo no Rancher, no inventário de máquinas, você poderá selecionar e criar um cluster a partir dele.

=== 3.1 Prepare a VM para emular o TPM

Na estação de trabalho VMware, crie uma vm da maneira que você faria normalmente, certifique-se de que o tamanho do HDD seja de pelo menos 40 GB.

Agora, edite as configurações da máquina e vá para a guia "Options" (Opções). A última opção seria "Advanced".

Clique em "advanced" (avançado) e, no painel direito da janela, altere o tipo de firmware de "BIOS" para "UEFI" e marque a opção "Enable secure boot" (ativar inicialização segura), como segue:

* Configurações padrão com o BIOS selecionado

image::rancher-vmware-vm-boot-bios.png[Opções de inicialização da VM com o BIOS]

* Configurações atualizadas com UEFI selecionado e inicialização segura ativada

image::rancher-vmware-vm-boot-uefi.png[Opções de inicialização da VM com UEFI]

Agora, na mesma guia "Options" (Opções), clique na opção "Access Control" (Controle de acesso) e clique em "Encrypt" (Criptografar) no lado direito.

image::rancher-vmware-access-control-menu.png[Menu de controle de acesso]

Isso solicitará que você digite uma senha para criptografar a máquina. Digite uma senha e clique em "Encrypt" (Criptografar)

image::rancher-vmware-access-control-encrypt.png[Credenciais de criptografia de controle de acesso]

Isso é importante para adicionar o hardware TPM. Em seguida, volte para as opções de hardware e clique em "Add" (Adicionar)

Adicione o hardware TPM (Trusted Platform Module) e clique em "Finish" (Concluir)

Agora, com a conclusão dessa etapa, nossa VM está pronta.

=== 3.2 Inicialize a VM com o ISO elementar

Em seguida, adicione a ISO que criamos anteriormente na VM e inicialize-a.

Ele deve ser inicializado com a ISO e começar a instalar o Elemental:

image::rancher-vmware-elemental-install-grub.png[{elemental-product-name} Menu do grub de instalação do sistema operacional]

image::rancher-vmware-elemental-install-logs.png[{elemental-product-name} Registros de instalação do sistema operacional]

Depois de concluída, a VM será reinicializada e aparecerá como ativa no inventário de máquinas do Rancher.
