= Atrás de um proxy
:revdate: 2025-11-06
:page-revdate: {revdate}

:tabs-sync-option:

Em muitos ambientes corporativos, os servidores ou as VMs em execução no local não têm acesso direto à Internet. Em vez disso, a conexão com serviços externos é feita por meio de um proxy HTTP(S) por motivos de segurança. Este tutorial mostra como configurar uma implementação do {elemental-product-name} em um ambiente como esse.

[IMPORTANT]
====
Este guia não abordará a instalação do Rancher por trás de um proxy. É um caso de uso diferente e você pode encontrar a documentação detalhada na página https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/http-proxy/http-proxy.html[Installing SUSE Rancher Prime behind an HTTP Proxy].
====


[NOTE]
====
Para esta documentação, supomos que você esteja usando um sistema da família SUSE (como o SLE Micro), portanto, as configurações de proxy devem ser gravadas em `+/etc/sysconfig/proxy+`.
====


As configurações de proxy devem ser definidas nos seguintes locais:

* Ponto de extremidade de registro de máquina
* Recurso SeedImage
* {elemental-product-name} configuração do cluster

O site `+elemental-system-agent+` precisa de configurações de proxy para acessar o Rancher Manager.
Para isso, é necessário preencher a seção cloud-init do Machine Registration Endpoint.

Você pode fazer isso com a xref:quickstarts/quickstart-cli.adoc#_prepare_your_kubernetes_resources[interface] xref:quickstarts/quickstart-ui.adoc#_add_a_machine_registration_endpoint[do] usuário ou com xref:quickstarts/quickstart-cli.adoc#_prepare_your_kubernetes_resources[a CLI].

== {elemental-product-name}-Registro

xref:rancher-os-management/architecture/architecture-components.adoc#_elemental_register_command_line_tool[{elemental-product-name}-O registro] é o primeiro ponto de extremidade de comunicação entre o novo host e o Rancher Manager; esse é o primeiro local em que as configurações de proxy precisam ser definidas.

[CAUTION]
====
No momento da redação deste documento, só é possível definir as configurações de proxy para o ISO com a CLI. As configurações de proxy não são implementadas na interface do usuário.
====

O processo acontece quando você inicializa a ISO do {elemental-product-name} pela primeira vez. Para definir as configurações de proxy, é necessário incluir uma definição de `+cloud-init+` na ISO.
Para isso, você precisa criar uma definição de xref:references/seedimage-reference.adoc[SeedImage].

[,yaml]
----
include::example$proxy/seedimage-proxy.yaml[]
----

Aplique o YAML com `+kubectl+` e, em seguida, imprima sua definição de SeedImage para obter o URL para download:

[,bash]
----
kubectl apply -f <my_seedimage_yaml_file>
kubectl get seedimage <seed_image_name> -n <namespace> -o yaml
----

Inicialize a ISO e você verá seu novo sistema aparecer no xref:references/machineinventory-reference.adoc[inventário de máquinas].
