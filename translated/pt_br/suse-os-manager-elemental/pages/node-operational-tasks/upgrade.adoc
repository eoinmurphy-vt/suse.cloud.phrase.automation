Todos os componentes em {elemental-product-name} são gerenciados usando o Kubernetes. Veja a seguir como usar as abordagens do Kubernetes para atualizar os componentes.

== {elemental-product-name} atualização de nós

{elemental-product-name} Os nós são atualizados com o {elemental_operator_name}. Consulte axref:references/operatorchart-reference.adoc[{elemental_operator_name}] documentação para obter informações completas.

A atualização pode ser feita com a CLI ou com a interface do usuário:

* <<_upgrade_via_command_line_interface,Interface de linha de comando>>
* <<_upgrade_via_user_interface,Interface do usuário>>

== Upgrade via interface de linha de comando

Há duas maneiras de selecionar nós para upgrade. Por meio de um alvo de cluster, que corresponderá a TODOS os nós em um cluster que corresponda ao nosso seletor, ou por meio de um seletor de nó, que corresponderá aos nós com base nos rótulos dos nós. O seletor de nós nos permite ser mais direcionados com a atualização, enquanto o seletor de cluster apenas seleciona todos os nós em um cluster correspondente.

A atualização de um site existente `+ManagedOSImage+` acionará um novo ciclo de atualização para reconciliar a imagem configurada (ou a versão da imagem) com todos os nós visados.

=== Gerenciamento de versões disponíveis

É possível criar xref:references/managedosversion-reference.adoc[ManagedOSVersions] diretamente ou assinar xref:references/managedosversionchannel-reference.adoc[ManagedOSVersionChannels] para sincronizar automaticamente `+ManagedOSVersions+` a partir deles.

Para obter mais detalhes e uma lista de canais disponíveis, ou mesmo para criar o seu próprio, leia a xref:operator-operational-tasks/channels.adoc[documentação].

== Atualização por meio da interface do usuário

Para fazer a atualização por meio da interface do usuário, você deve acessar o menu {elemental-product-name} Advanced e, em seguida, clicar em `+Update Groups+`.

Escolha um nome, selecione os clusters a serem direcionados e escolha entre as duas formas de upgrade:

image::upgrade-ui-menu.png[{elemental-product-name} Menu de atualização]

[tabs]
======
Via versão do sistema operacional gerenciado::
+
====
Nesse caso, um `+OS Version Channels+` é usado para preencher automaticamente os recursos do `+OS Versions+`. 

O canal abaixo é fornecido por nós, mas você também pode trazer seu próprio canal. 

Consulte a seção <<_managing_available_versions,Gerenciamento de versões disponíveis>> para entender como o `+ManagedOSVersion+` é gerenciado. 

image:upgrade-ui-version-channel.png[Criar canal de versão do sistema operacional] 

Após um curto período de sincronização, você verá que seu `+OS Versions+` aparece no menu `+OS Versions+`. 

image:upgrade-ui-os-version.png[{elemental-product-name} Menu Versão do sistema operacional] 

Por fim, você pode selecionar `+OS Versions+` ao criar seu `+Upgrade Group+`, conforme a captura de tela a seguir: 

image:upgrade-ui-upgrade-group-channel.png[Selecione a versão do sistema operacional no grupo de upgrade]
====

Guia Via Image from registry (Imagem do registro)::
+
====
Basta especificar uma imagem OCI no campo `+Image path+` para fazer o upgrade: image:upgrade-ui-image-registry.png[Upgrade via Image Registry]
====
======

Clique no botão `+Create+` para iniciar o processo de upgrade. Se você tiver vários nós, o upgrade será feito sequencialmente, nó por nó.
