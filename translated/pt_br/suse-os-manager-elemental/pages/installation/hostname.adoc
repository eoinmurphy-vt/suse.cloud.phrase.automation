== {elemental-product-name} Visão geral do gerenciamento de nomes de host

Quando um host é inicializado a partir do ISO {elemental-product-name}, o nome do host é temporariamente definido_(nome de host transitório_) como aquele fornecido pelo servidor DHCP.
Se o servidor DHCP não fornecer um nome de host, o {elemental-product-name} ISO fornecerá um nome de host temporário no formato: `+rancher-$\{RANDOM}+`.

Assim que o processo de inicialização é concluído, a fase de registro entra em ação: o host se conecta ao operador {elemental-product-name}, que cria um `+MachineInventory+` para o host.
Cada host registrado no operador {elemental-product-name} é rastreado por um recurso `+MachineInventory+`.

*O `+name+` do recurso `+MachineInventory+` associado ao nó é o nome de host permanente (estático)**eventualmente definido para o host*.
Esse nome de host permanente *é definido no nó somente durante a fase de provisionamento do cluster do K8s*.
Antes da fase de provisionamento do K8s, o nome do host do nó é o atribuído pelo DHCP ou `+rancher-$\{RANDOM}+`.

No restante desta seção, vamos nos referir a `+hostname+` como o nome de host _permanente_, ou seja, o nome de host que é definido depois que o host foi provisionado como parte de um cluster do K8s.

== Nome de host padrão

O nome padrão atribuído a cada MachineInventory recém-criado está no formato `+m-{$UUID}+`.
Quando o host é provisionado como parte de um cluster, esse nome `+m-\{UUID}+` é definido como o nome de host do host correspondente, substituindo o nome de host atribuído anteriormente (`+rancher-{$RANDOM}+` ou o nome atribuído pelo DHCP).

== Definir um nome de host personalizado

O nome do host pode ser especificado definindo o campo xref:references/machineregistration-reference.adoc#_machinename[`+machineName+`] no recursoxref:references/machineregistration-reference.adoc["MachineRegistration"].

Espera-se que o nome do host definido no campo `+machineName+` esteja em um formato de modelo, a fim de ser gerado exclusivamente para cada nó de registro, usando dados do SMBIOS e de etiquetas de hardware.

[IMPORTANT]
====
O campo `+machineName+` no recurso `+MachineRegistration+` é usado como modelo não apenas para o nome de host do host de registro, mas também para o nome do recurso `+MachineInventory+` criado para rastrear o host.

Isso significa que, se você não usar um modelo `+machineName+` tal para gerar um nome exclusivo para cada host que inicializará usando os mesmos dados `+MachineRegistration+` (ou seja, a mesma ISO), apenas o primeiro host de registro será bem-sucedido, enquanto os outros falharão: o nome `+MachineInventory+` deve ser exclusivo.
====

== Manter o nome do host atribuído pelo DHCP

Para manter o nome do host atribuído pelo servidor DHCP antes de o host se registrar no operador, o `+MachineRegistration+` xref:references/machineregistration-reference.adoc#_machinename[`+machineName field+`] deve ser definido como `+${System Data/Runtime/Hostname}+` Hardware Label.

Dessa forma, o {elemental-product-name} usará o nome de host atual como o nome `+MachineInventory+` durante a fase de registro, que será definido posteriormente como o nome de host estático do host durante a fase de provisionamento.
