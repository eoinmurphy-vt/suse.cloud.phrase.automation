[#_prerequisite]
== Conditions préalables

Vous devez avoir au moins les autorisations de membre du cluster.

[#_brand_configuration]
== Configuration de la marque

Pour configurer les paramètres de la marque,

. Cliquez sur *☰ > Paramètres globaux*.
. Cliquez sur *Personnalisation*.

[#_private_label_company_name]
=== Nom de l'entreprise en marque blanche

Cette option remplace "Rancher" par la valeur que vous fournissez dans la plupart des endroits. Les fichiers qui doivent contenir Rancher dans le nom, comme `+rancher-compose.yml+`, ne seront pas modifiés.

[#_support_links]
=== Liens de support

Utilisez une adresse URL pour envoyer de nouveaux rapports "Signaler un problème" au lieu d'envoyer les utilisateurs vers la page des problèmes GitHub. Afficher éventuellement les liens de support de la communauté Rancher.

[#_logo]
=== Logo

Téléchargez des logos clairs et foncés pour remplacer le logo Rancher dans l'en-tête de navigation de niveau supérieur.

[#_primary_color]
=== Couleur principale

Vous pouvez remplacer la couleur principale utilisée dans toute l'interface utilisateur par une couleur personnalisée de votre choix.

[#_fixed_banners]
=== Bannières fixes

Affichez une bannière fixe personnalisée dans l'en-tête, le pied de page, ou les deux.

Pour configurer les paramètres de la bannière,

. Cliquez sur *☰ > Paramètres globaux*.
. Cliquez sur *Bannières*.

[#_custom_navigation_links]
== Liens de navigation personnalisés

Dans cette section, vous apprendrez à configurer les liens dans la barre de navigation à gauche du *Tableau de bord du cluster*. Pour accéder au tableau de bord du cluster,

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Sur la page *Clusters*, allez au cluster où vous souhaitez des liens de navigation personnalisés et cliquez sur *Explorer*.

Il peut être utile d'ajouter un lien pour un accès rapide aux services installés sur un cluster. Par exemple, vous pourriez ajouter un lien vers l'interface utilisateur de Kiali pour les clusters avec Istio installé, ou vous pourriez ajouter un lien vers l'interface utilisateur de Grafana pour les clusters avec la surveillance Rancher installée.

Les liens personnalisés n'affectent pas qui a accès à chaque service.

Les liens peuvent être créés au niveau supérieur et plusieurs liens peuvent être regroupés ensemble.

[#_adding_a_custom_navigation_link]
=== Ajouter un lien de navigation personnalisé

[NOTE]
.Condition préalable :
====

Vous devez avoir au moins les autorisations de membre du cluster ou du projet.
====


. Cliquez sur *☰ > Gestion des clusters*.
. Sur la page *Clusters*, allez au cluster où vous souhaitez ajouter des liens de navigation personnalisés et cliquez sur *Explorer*.
. Dans le menu de navigation supérieur, cliquez sur *🔍 (Recherche de ressources)*.
. Tapez *Nav* et cliquez sur *Liens Nav*.
. Cliquez sur *Créer à partir de YAML*.
. La manière la plus simple de créer un lien de navigation est d'ajouter ces champs :

  name: linkname
  toURL: https://example.com
+
For more details on setting up links, including optional fields, see <<_link_configuration,Link Configuration.>>

. Cliquez sur *Create*.

[#_link_configuration]
== Configuration des liaisons

[#__name]
=== `+name+`

Nom d'affichage pour le lien. Requis.

[#__group]
=== `+group+`

Nom d'un groupe de liens qui se développe lorsqu'il est cliqué.

Facultatif. S'il n'est pas fourni, le lien apparaît de manière autonome.

Les groupes sont affichés séparément des liens autonomes, comme indiqué ci-dessous :

image::grouped-vs-standalone-links.png[Capture d'écran du groupe et du lien autonome]

[#__iconsrc]
=== `+iconSrc+`

Source d'icône au format base64.

Voici un exemple du logo Grafana au format base64 :



[#__annotation]
=== `+annotation+`

Une annotation Kubernetes pour la ressource personnalisée NavLink.

[#__label]
=== `+label+`

Une étiquette Kubernetes pour la ressource personnalisée NavLink.

[#__sidelabel]
=== `+sideLabel+`

Étiquette qui apparaît dans la barre de navigation à gauche

[#__target]
=== `+target+`

Définit la propriété target de la balise `+<a>+` (ancre) du lien, qui (selon les navigateurs) détermine si le lien s'ouvre dans une nouvelle fenêtre ou dans un onglet existant.

La valeur par défaut est `+_self+`, qui ouvre le lien dans l'onglet actuel. Pour ouvrir le lien dans une nouvelle fenêtre ou un nouvel onglet, définissez la cible sur `+_blank+`.

Pour plus d'informations sur la propriété target, voir https://www.w3schools.com/tags/att_a_target.asp[cette page.]

[#__toservice]
=== `+toService+`

Contient cinq champs qui sont construits pour créer une URL comme suit : `+\https://<rancher_server_url>/k8s/clusters/<CLUSTER>/k8s/namespace/<NAMESPACE>/service/<SCHEME>:<NAME>:<PORT>/proxy/<PATH>+`

Par exemple, un lien vers un service de surveillance peut être configuré comme suit :

* nom : `+rancher-monitoring-grafana+`
* espace de noms : `+cattle-monitoring-system+`
* chemin : `+proxy/?orgId=1+`
* port : `+"80"+`
* schéma : `+http+`

Il est nécessaire de fournir soit la directive `+toService+`, soit la directive `+toURL+`.

[#__tourl]
=== `+toUrl+`

Peut être n'importe quel lien, même vers des liens en dehors du cluster.

Il est nécessaire de fournir soit la directive `+toService+`, soit la directive `+toURL+`.

[#_link_examples]
== Exemples de liens

[#_example_of_link_with_tourl]
=== Exemple de lien avec `+toUrl+`

Cet exemple de YAML NavLink montre un exemple de configuration d'un NavLink vers un tableau de bord Grafana :

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  name: grafana
spec:
  group: "Monitoring Dashboards"
  toURL: https://<RANCHER_SERVER_URL>/api/v1/namespaces/cattle-monitoring-system/services/http:rancher-monitoring-grafana:80/proxy/?orgId=1
----

L'ajout du YAML ci-dessus entraîne la création d'un lien vers Grafana, comme le montre la capture d'écran suivante :

image::example-grafana-link.png[Capture d'écran du lien Grafana]

[#_example_of_link_with_toservice]
=== Exemple de lien avec `+toService+`

Cet exemple de YAML montre un exemple de `+toService+` utilisé pour la cible du lien :

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  annotations:
    key: annotation
    labels:
      key: label
  name: navlinkname
spec:
  description: This is a description field # Optional.
  group: "group1" # Optional. If not provided, the links appear standalone.
  iconSrc: data:image/jpeg;base64,[icon source string is clipped for brevity]
  label: This is a label # Optional.
  sideLabel: A side label. # Optional.
  target: _blank #Optional. _blank opens the link in a new tab or window.
  toService: # toService or #toUrl needs to be provided.
    name: rancher-monitoring-grafana
    namespace: cattle-monitoring-system
    path: proxy/?orgId=1
    port: "80"
    scheme: http
----

L'ajout des paramètres `+toService+` ci-dessus entraîne la création d'un lien vers Grafana, comme le montre la capture d'écran suivante :

image::example-service-link.png[Capture d'écran du lien Grafana]
