En cas d'urgence, les équipes opérationnelles peuvent être amenées à effectuer des actions que {project-name} bloquerait.

Ce document explique comment désactiver {short-project-name} afin que les équipes d'exploitation puissent remédier aux défaillances des clusters. Après avoir résolu la situation, les équipes opérationnelles peuvent redémarrer {short-project-name}.

== Désactiver {short-project-name}

Tout d'abord, arrêtez le contrôleur {short-project-name}. Pour ce faire, vous devez ramener son déploiement à zéro :

[source,console]
----
kubectl scale deployment kubewarden-controller \
  --replicas=0 \
  -n kubewarden
----

[tip]
====
Veillez à utiliser le nom de l'espace de noms dans lequel la pile {short-project-name} est déployée.
Ici, la pile est déployée dans l'espace de noms `+kubewarden+`.
====

Ensuite, supprimez tous les `+ValidatingWebhookConfigurations+` et `+MutatingWebhookConfigurations+`créés par {short-project-name}:

[source,console]
----
# Delete all the ValidatingWebhookConfiguration created by {short-project-name}
kubectl delete validatingwebhookconfigurations \
  -l app.kubernetes.io/part-of=kubewarden

# Delete all the MutatingWebhookConfiguration created by {short-project-name}
kubectl delete mutatingwebhookconfigurations \
  -l app.kubernetes.io/part-of=kubewarden
----

== Restaurer {short-project-name}

Lorsque la situation d'urgence est terminée, rétablissez la pile {short-project-name} en ramenant le contrôleur {short-project-name}:

[source,console]
----
kubectl scale deployment kubewarden-controller \
  --replicas=1 \
  -n kubewarden
----

[tip]
====
- Veillez à utiliser le nom de l'espace de noms dans lequel la pile {short-project-name} est déployée.
  Ici, la pile est déployée dans l'espace de noms `+kubewarden+`.
- Veillez à ramener le contrôleur à sa valeur d'origine. Cet exemple suppose qu'il n'y a qu'une seule réplique du contrôleur en cours d'exécution.
====
Une fois que le contrôleur fonctionne, il réconcilie les politiques déployées. Les ressources `+ValidatingWebhookConfiguration+` et `+MutatingWebhookConfiguration+`, précédemment supprimées, sont recréées, ce qui permet d'appliquer les politiques.
