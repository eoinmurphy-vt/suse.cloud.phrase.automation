Le traçage permet de recueillir des informations détaillées sur l'évaluation des politiques.
Il s'agit d'un outil utile pour déboguer les problèmes liés au déploiement et aux politiques de {project-name}.

{short-project-name} utilise https://www.jaegertracing.io/[] Jaeger pour recevoir, stocker et visualiser les traces d'événements.

== Conditions préalables

Vous avez besoin de Cert-Manager et d'OpenTelemetry. Suivezxref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[ces instructions] pour installer Cert Manager et l'opérateur OpenTelemetry.

== Installer Jaeger

Appliquer les étapes d'installation de l'xref:howtos/telemetry/20-tracing-qs.adoc#_install_jaeger[aide-mémoire sur le traçage].

Une fois que l'opérateur Jaeger a créé toutes les ressources, un service est disponible à l'adresse`+my-open-telemetry-collector.jaeger.svc.cluster.local+`.

Le Jaeger Query UI est joignable à l'adresse suivante :

[subs="+attributes",console]
----
https://<CLUSTER_IP>/api/v1/namespaces/jaeger/services/http:my-open-telemetry-query:16686/proxy/search
----

Ce point d'arrivée peut être unique en fonction de votre configuration. Vous trouverez le point de terminaison de votre ressource Jaeger sur la page Services.

=== Mise à jour de `+rancher-kubewarden-controller+` avec le point de terminaison Jaeger

Vous devrez modifier la ressource `+rancher-kubewarden-controller+` pour ajouter le point de terminaison Jaeger "my-open-telemetry-collector.jaeger.svc.cluster.local:4317".

Par exemple :

[subs="+attributes",yaml]
----
telemetry:
  tracing: true
  mode: sidecar
  sidecar:
    tracing:
      jaeger:
        endpoint: my-open-telemetry-collector.jaeger.svc.cluster.local:4317
        tls:
          insecure: true
----

L'installation du service, `+my-open-telemetry-collector+`, se trouve sous l'espace de noms`+jaeger+`.

[CAUTION]
====

Pour des raisons de simplicité, nous ne chiffrons pas la communication entre le collecteur OpenTelemetry et le point de terminaison Jaeger.

Cela *ne convient pas à un déploiement en production*. Vous devriez lire la Jaegerhttps://www.jaegertracing.io/docs/latest/kubernetes/[documentation de].

====


Vous devriez maintenant être en mesure d'afficher toutes les demandes qui ont échoué pour les politiques liées à un serveur de politiques spécifique. Vous pouvez également consulter la vue détaillée d'une police donnée.
Vous pouvez obtenir une vue plus approfondie des traces en utilisant l'interface utilisateur de Jaeger.
