{project-name} ne peuvent pas faire de requête réseau à partir du contexte d'exécution de WebAssembly.

Les opérations sur le réseau peuvent être effectuées en exploitant une série de capacités exposées par l'hôte.

== Recherche d'hôtes DNS

Cette fonction effectue une recherche DNS à partir du FQDN indiqué dans la politique.

=== Mise en cache

Les résultats de la recherche sont mis en cache pendant 1 minute.

=== Protocole de communication

Il s'agit de la description du protocole waPC utilisé pour exposer cette capacité :

==== Fonction waPC - `+v1/dns_lookup_host+` input

[,hcl]
----
# hostname - JSON encoded string
string
----

==== Fonction waPC - `+v1/dns_lookup_host+` sortie

[,hcl]
----

{
  # list of IPs
  "ips": [string]
}
----

Toutes les adresses IP associées au FQDN donné seront renvoyées sous forme de chaînes dans la réponse.
Les entrées IPv4 et IPv6 sont renvoyées dans le cadre de la même réponse.

Par exemple, si l'on demande le résumé du manifeste de l'image`+busybox:latest+`, les données utiles seront les suivantes :

* Charge utile d'entrée : `+"google.com"+`
* Charge utile de sortie : `+{ "ips": ["127.0.0.1"]}+`
