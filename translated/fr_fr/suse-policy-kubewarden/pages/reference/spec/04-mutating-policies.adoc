= Mutation des politiques
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: Configurer les politiques de mutation en implémentant des fonctions de validation (validate et validate_settings) dans un waPC (whatever-after Policy Configurator) pour proposer des mutations d'objets dans les clusters Kubernetes.
:doc-persona: ["kubewarden-policy-developer"]
:doc-topic: ["politiques d'écriture", "spécification", "politiques de mutation"].
:doc-type: ["reference"]
:keywords: ["kubewarden", "kubernetes", "policy specification", "mutating policies"]
:sidebar_label: Mutation des politiques
:current-version: {page-origin-branch}

La structure des politiques de mutation est la même que celle des politiques de validation :

* Ils doivent enregistrer les fonctions `+validate+` et `+validate_settings+` waPC.
* L'API de communication utilisée entre l'hôte et la politique est la même que celle utilisée pour la validation des politiques.

Les politiques de mutation acceptent une demande et peuvent proposer une mutation de l'objet entrant en renvoyant un objet `+ValidationResponse+` qui ressemble à ceci :

[subs="+attributes",json]
----
{
  "accepted": true,
  "mutated_object": <object to be created>
}
----

Le champ `+mutated_object+` contient l'objet que la politique veut créer dans le cluster Kubernetes, sérialisé en JSON.

== Un exemple concret

Supposons que la police ait reçu cette adresse `+ValidationRequest+`:

[subs="+attributes",json]
----
{
  "settings": {},
  "request": {
    "operation": "CREATE",
    "object": {
      "apiVersion": "v1",
      "kind": "Pod",
      "metadata": {
        "name": "security-context-demo-4"
      },
      "spec": {
        "containers": [
        {
          "name": "sec-ctx-4",
          "image": "gcr.io/google-samples/node-hello:1.0",
          "securityContext": {
            "capabilities": {
              "add": ["NET_ADMIN", "SYS_TIME"]
            }
          }
        }
        ]
      }
    }
  }
}
----

[NOTE]
====

Dans cet exemple, seuls les champs importants figurent dans l'objet `+request+`.

====


Cette génération de demandes est due au fait que quelqu'un a essayé de créer un pod qui ressemblerait à ceci :

[subs="+attributes",yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: security-context-demo-4
spec:
  containers:
  - name: sec-ctx-4
    image: gcr.io/google-samples/node-hello:1.0
    securityContext:
      capabilities:
        add:
        - NET_ADMIN
        - SYS_TIME
----

Supposons que la politique réponde comme suit : `+ValidationResponse+`:

[subs="+attributes",json]
----
{
  "accepted": true,
  "mutated_object": {
    "apiVersion": "v1",
    "kind": "Pod",
    "metadata": {
      "name": "security-context-demo-4"
    },
    "spec": {
      "containers": [
        {
          "name": "sec-ctx-4",
          "image": "gcr.io/google-samples/node-hello:1.0",
          "securityContext": {
            "capabilities": {
              "add": [
                "NET_ADMIN",
                "SYS_TIME"
              ],
              "drop": [
                "BPF"
              ]
            }
          }
        }
      ]
    }
  }
}
----

Cela conduirait à l'acceptation de la demande, mais le Pod final ressemblerait à ceci :

[subs="+attributes",yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: security-context-demo-4
spec:
  containers:
  - name: sec-ctx-4
    image: gcr.io/google-samples/node-hello:1.0
    securityContext:
      capabilities:
        add:
        - NET_ADMIN
        - SYS_TIME
        drop:
        - BPF
----

Comme vous pouvez le voir, la politique a modifié la section `+securityContext.capabilities.drop+`du seul conteneur déclaré dans le Pod.

Le conteneur abandonne maintenant la capacité `+BPF+` en raison de la politique.

== Récapitulation

Ce sont les fonctions qu'une politique de mutation doit mettre en œuvre :

|===
| *Nom de la fonction waPC* | *Charge utile d'entrée* | *Charge utile en sortie*

| `+validate+`
| `+\{  "request": \{    // AdmissionReview.request data  \},  "settings": \{    // your policy configuration  \}\}+`
| `+\{  **// mandatory**  "accepted": boolean,  // optional, ignored if accepted  // recommended for rejections  "message": string,  // optional, ignored if accepted  "code": integer,   // JSON Object to be created  // Can be used only when the  // request is accepted  "mutated_object": object\}+`

| `+validate_settings+`
| `+\{  // your policy configuration\}+`
| `+\{  **// mandatory**  "validate": boolean,  // optional, ignored if accepted  // recommended for rejections  "message": string,\}+`
|===
