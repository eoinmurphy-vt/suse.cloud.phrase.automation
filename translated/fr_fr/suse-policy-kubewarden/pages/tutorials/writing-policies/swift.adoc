Comme indiqué sur le https://swift.org/[site officiel]:

____
Swift est un langage de programmation polyvalent, accessible aux nouveaux venus et puissant pour les experts.
Il est rapide, moderne, sûr et agréable à écrire.
____

Le compilateur swift ne supporte pas encore WebAssembly, mais le projethttps://swiftwasm.org/[SwiftWasm] fournit un compilateur patché avec cette capacité.

L'équipe SwiftWasm travaille à la fusion de ces changements dans le projet Swift.
En attendant, vous pouvez utiliser la chaîne d'outils fournie par le projet SwiftWasm pour élaborer des politiques {project-name}.

[NOTE]
====

Vous n'avez pas besoin d'un système Apple pour écrire ou exécuter du code Swift. Tout peut être fait également sur une machine Linux ou sur Windows (en utilisant Docker pour Windows).
====


== État actuel

Les auteurs de politiques peuvent utiliser les ressources suivantes :

* https://github.com/kubewarden/policy-sdk-swift[{short-project-name} Swift SDK]: il fournit un ensemble de ``+struct+s``et de fonctions qui simplifient le processus d'écriture des politiques.
* https://github.com/kubewarden/swift-policy-template[{short-project-name} Modèle de projet Swift]: utilisez ce modèle pour élaborer une politique basée sur Swift.
Le modèle est accompagné d'une politique opérationnelle et d'un ensemble d'actions GitHub pour automatiser son cycle de vie.

Il n'y a pas de limitations sévères avec Swift, seulement quelques problèmes mineurs :

* Il est essentiel d'effectuer quelques optimisations post-construction avant d'utiliser la politique _"en production" :_
 .. Déshabillez le module Wasm via `+wasm-strip+` pour en réduire la taille.
 .. Optimiser le module Wasm via `+wasm-opt+`

L'action GitHub fournie par le dépôt de modèles s'en charge déjà.
