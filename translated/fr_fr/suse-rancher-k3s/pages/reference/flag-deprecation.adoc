= Dépréciation du drapeau
:page-languages: [en, ja, ko, zh]
:revdate: 2024-09-09
:page-revdate: {revdate}

K3s est un projet qui évolue rapidement, et en tant que tel, nous avons besoin d'un moyen de déprécier les drapeaux et les options de configuration. Cette page décrit le processus de dépréciation des drapeaux et des options de configuration. Afin de s'assurer que les utilisateurs ne soient pas surpris par la suppression des drapeaux, le processus est similaire à la https://kubernetes.io/docs/reference/using-api/deprecation-policy/[politique de dépréciation de] Kubernetes.

== Processus

. Les drapeaux peuvent être déclarés "à déprécier" à tout moment.
. Les drapeaux "à supprimer" doivent être étiquetés comme tels lors de la prochaine mise à jour de toutes les versions actuellement prises en charge. En outre, l'indicateur commencera à avertir les utilisateurs qu'il sera supprimé dans la prochaine version mineure.
. Lors de la prochaine version mineure, un indicateur sera marqué comme déprécié dans la documentation et converti en indicateur caché dans le code. Le drapeau continuera à fonctionner et à donner des avertissements aux utilisateurs.
. Dans la branche mineure suivante, les drapeaux dépréciés deviendront "non opérationnels" et provoqueront une erreur fatale s'ils sont utilisés. Cette erreur doit expliquer à l'utilisateur tout nouveau drapeau ou toute nouvelle configuration qui remplace ce drapeau.
. Dans la prochaine version mineure, les drapeaux non opérationnels seront supprimés de la documentation et du code.

== Exemple

Un exemple de processus :

* `+--foo+` existe dans les versions 1.22.14, 1.23.10 et 1.24.2.
* Après la sortie de la version 1.24.2, il a été décidé de supprimer `+--foo+` au profit de `+--new-foo+`.
* Dans les versions 1.22.15, 1.23.11 et 1.24.3, `+--foo+` continue d'exister, mais avertit les utilisateurs :
+
----
[Warning] --foo will be deprecated in v1.25.0, use `+--new-foo+` instead
----
+
`+--foo+` continuera d'exister en tant qu'indicateur opérationnel pendant la durée de vie des versions 1.22, 1.23 et 1.24.

* Dans la version 1.25.0, `+--foo+` est indiqué comme obsolète dans la documentation et sera caché dans le code. Il continuera à fonctionner et à avertir les utilisateurs qu'ils doivent se rendre sur `+--new-foo+`.
* Dans la version 1.26.0, `+--foo+` provoquera une erreur fatale s'il est utilisé. Le message d'erreur est le suivant :
+
----
[Fatal] exit 1: --foo is no longer supported, use --new-foo instead
----

* Dans la version 1.27.0, `+--foo+` sera complètement supprimé du code et de la documentation.
