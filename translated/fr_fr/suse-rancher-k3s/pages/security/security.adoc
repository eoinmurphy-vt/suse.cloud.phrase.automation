= Sécurité
:page-languages: [en, ja, ko, zh]
:revdate: 2025-12-31
:page-revdate: {revdate}

Cette section décrit la méthodologie et les moyens de sécuriser un cluster K3s. Il est divisé en deux sections. Ces guides supposent que k3s fonctionne avec etcd intégré.

Tout d'abord, le guide d'endurcissement fournit une liste des meilleures pratiques de sécurité pour sécuriser un cluster K3s.

* xref:security/hardening-guide.adoc[Guide de durcissement]

Deuxièmement, il y a l'auto-évaluation qui permet de valider un cluster durci. Nous disposons actuellement de deux évaluations différentes :

* xref:security/self-assessment-1.9.adoc[CIS 1.9 Benchmark Self-Assessment Guide], pour K3s version v1.27-v1.29
* xref:security/self-assessment-1.10.adoc[CIS 1.10 Benchmark Self-Assessment Guide], pour K3s version v1.28-v1.31
* xref:security/self-assessment-1.11.adoc[CIS 1.11 Benchmark Self-Assessment Guide], pour K3s version v1.29-v1.34
