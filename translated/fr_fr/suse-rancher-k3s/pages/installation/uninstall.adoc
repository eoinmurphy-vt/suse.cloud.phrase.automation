= Désinstallation de K3s
:page-languages: [en, ja, ko, zh]
:revdate: 2026-01-05
:page-revdate: {revdate}

[CAUTION]
====
La désinstallation de K3s peut entraîner une perte de données !
====

Si vous avez installé K3s à l'aide du script d'installation, un script de désinstallation de K3s a été généré lors de l'installation.

L'exécution du script de désinstallation arrête K3s et tous les pods en cours d'exécution, et supprime le datastore local du cluster, les données du volume persistant de xref:add-ons/storage.adoc#_setting_up_the_local_storage_provider[stockage local], la configuration des nœuds et tous les scripts et outils CLI.

Il ne supprime aucune donnée des datastores externes, ou créée par des pods utilisant des volumes persistants Kubernetes externes.

Si vous prévoyez de réintégrer un nœud dans un cluster existant après l'avoir désinstallé et réinstallé, veillez à supprimer le nœud du cluster afin de vous assurer que le secret du mot de passe du nœud est supprimé. Voir la documentation sur l'xref:architecture.adoc#_how_agent_node_registration_works[enregistrement des nœuds] pour plus d'informations.

== Désinstallation des serveurs

Pour désinstaller K3s d'un nœud de serveur, exécutez :

[,bash]
----
/usr/local/bin/k3s-uninstall.sh
----

== Désinstallation des agents

Pour désinstaller les K3 d'un nœud d'agent, exécutez :

[,bash]
----
/usr/local/bin/k3s-agent-uninstall.sh
----
