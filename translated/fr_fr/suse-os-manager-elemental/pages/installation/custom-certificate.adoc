= Comment ajouter un certificat personnalisé
:revdate: 2024-09-24
:page-revdate: {revdate}

Prérequis : Un certificat au format `+.pem+` 

Objectif : Mettre un certificat personnalisé à la disposition de l'ensemble du système

[NOTE]
.Il s'agit des certificats utilisés par les services au niveau du système.
====
Les charges de travail Kubernetes devraient plutôt apporter leurs certificats dans l'image du conteneur.
====

Pour installer un certificat personnalisé, nous devons

* copier le fichier `+.pem+` sur `+/etc/pki/trust/anchors/+`
* courir `+update-ca-certificates+`

L'extrait de `+cloud-config+` correspondant ressemble à ceci :

[,yaml]
----
write_files:
  - path: /etc/pki/trust/anchors/my-custom-certificate.pem
    permission: 0444
    content: |-
      -----BEGIN CERTIFICATE-----
      ...
      -----END CERTIFICATE-----
runcmd:
  - update-ca-certificates
----

(le contenu réel du certificat est omis pour des raisons de brièveté)
