= Personnaliser l'installation de {elemental-product-name} 
:revdate: 2025-09-08
:page-revdate: {revdate}

{elemental-product-name} (Les images installées du système d'exploitation peuvent être personnalisées de différentes manières.

Une option consiste à remasteriser les images des systèmes d'exploitation en conteneur en utilisant simplement une compilation docker. Les images SL Micro sont des images de conteneurs ordinaires, il est donc tout à fait possible de créer une nouvelle image à l'aide d'un fichier Docker basé sur SLE Micro. Voir la section xref:operator-operational-tasks/custom-images.adoc[Construire des images de système d'exploitation personnalisées] pour plus de détails, c'est l'option préférée.

Il est également possible de fournir des ressources et une configuration supplémentaires dans le support d'installation de sorte que pendant l'installation, ou éventuellement au moment du démarrage, des binaires supplémentaires tels que des pilotes ou des fichiers de configuration supplémentaires peuvent être inclus.

Cette section explique comment personnaliser le processus d'installation à partir d'une image de système d'exploitation donnée.

== Options de personnalisation

L'installation Elemental peut être personnalisée de trois manières différentes et non exclusives. Premièrement, en incluant des fichiers cloud-config supplémentaires dans le système installé, deuxièmement, en incluant des fichiers cloud-config supplémentaires dans le média d'installation, et enfin, en ajoutant un fichier de configuration client Elemental personnalisé (`+/etc/elemental/config.yaml+`).

. Les fichiers cloud-config supplémentaires inclus dans le système installé sont utiles pour exécuter des opérations personnalisées au moment du démarrage. Voir la xref:references/cloud-config-reference.adoc[référence de la configuration du nuage].

. Les fichiers cloud-config supplémentaires contenus dans le support d'installation sont utiles pour exécuter des opérations personnalisées au moment de l'installation ou pour adapter l'environnement d'installation à certains besoins spécifiques.

. Un fichier de configuration du client Elemental est, par défaut, situé à l'adresse `+/etc/elemental/config.yaml+` et peut également être divisé en plusieurs fichiers yaml dans le répertoire `+/etc/elemental/config.d+`. Voir la https://rancher.github.io/elemental-toolkit/docs/customizing/general_configuration/[référence du fichier de configuration].

Il est courant de combiner les trois méthodes décrites ci-dessus en fonction des besoins spécifiques.

=== Ajout de fichiers de configuration supplémentaires dans le système d'exploitation installé

Afin d'inclure des fichiers de configuration supplémentaires pendant l'installation, ils doivent être ajoutés aux données d'installation dans la ressource MachineRegistration. La façon la plus simple d'y parvenir est d'ajouter la configuration cloud souhaitée directement dans la xref:references/machineregistration-reference.adoc#_config_cloud_config[section consacrée à] l'enregistrement de la machine. Voir l'exemple ci-dessous


[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: my-nodes
  namespace: fleet-default
spec:
  ...
  config:
    ...
    cloud-config:
      stages:
        boot:
        - name: "Adding 'admin' user"
          users:
            admin:
              passwd: mysecretpasswd
----

Par ailleurs, le chemin des fichiers cloud-config peut également être référencé explicitement, de sorte que la configuration n'est pas strictement nécessaire dans la ressource MachineRegistration elle-même. La section `+config-urls+` du MachineRegistration est utilisée à cette fin précise. Voir la page de xref:references/machineregistration-reference.adoc[référence MachineRegistration].

`+config-urls+` est une liste de chaînes de caractères dont chaque élément est une URL HTTP ou un chemin local pointant vers un fichier de configuration du nuage. Le chemin d'accès local est évalué lors de l'installation, il doit donc exister dans le média d'installation, généralement une image ISO.

Par défaut, les systèmes Elemental live montent la racine ISO à l'adresse `+/run/initramfs/live+`, qui est également le chemin d'accès par défaut défini pour `+config-url+` dans `+MachineRegistrations+`. Voir l'exemple ci-dessous :

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: my-nodes
  namespace: fleet-default
spec:
  ...
  config:
    ...
    elemental:
      ...
      install:
        ...
        config-urls:
        - "/run/initramfs/live/oem/custom_config.yaml"
----

Lors du démarrage, la racine de l'ISO est montée à l'adresse `+/run/initramfs/live+`. D'après cela, l'ISO de l'exemple ci-dessus devrait inclure le fichier `+/oem/custom_config.yaml+`.

[NOTE]
====
`+/run/initramfs/live+` est un point de montage en lecture seule et ce n'est pas un chemin approprié pour un contenu généré dynamiquement au démarrage de l'ISO.
====

=== Ajout de fichiers cloud-config supplémentaires dans le média d'installation

L'ajout de fichiers cloud-config supplémentaires dans le média d'installation peut être nécessaire pour configurer l'environnement d'installation (par exemple, définir la connectivité réseau pour l'enregistrement) ou pour fournir des xref:references/cloud-config-reference.adoc#_elemental_client_cloud_config_hooks[crochets d'installation] afin d'exécuter une logique personnalisée au cours de l'installation elle-même.

Dans Elemental, les ressources xref:references/seedimage-reference.adoc[SeedImage] sont chargées de gérer les supports d'installation. L'endroit le plus simple pour inclure des données cloud-config supplémentaires est donc le média d'installation, en les ajoutant à la xref:references/seedimage-reference.adoc#_seedimagespec_reference[section`+cloud-config+` ]. Ce faisant, la configuration du nuage donnée sera déjà évaluée lors du tout premier démarrage du média d'installation et sera également respectée pendant la phase d'installation dans le cas où un crochet d'installation est fourni. Prenons l'exemple suivant :

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: custom-seed
  namespace: fleet-default
spec:
  ...
  cloud-config:
    stages:
      post-install:
      - name: "Run custom script after installation"
        commands:
        - |
          echo "This is a custom script"
          echo "For instance, this could be used to handle extra drives for an LVM group"
      boot:
      - name: "Add proxy setup for the installation media"
        files:
        - path: /etc/sysconfig/proxy
          permissions: 0664
          content: |
            PROXY_ENABLED="yes"
            HTTP_PROXY=http://<MY_PROXY>:<MY_PORT>
            HTTPS_PROXY=https://<MY_PROXY>:<MY_PORT>
            NO_PROXY="localhost, 127.0.0.1"
----

=== Fichier de configuration personnalisé du client {elemental-product-name} 

https://github.com/rancher/elemental-toolkit/blob/main/docs/elemental.md[clients élémentaires]Les commandes des  `+install+`, `+upgrade+` et `+reset+` peuvent être configurées à l'aide d'un https://rancher.github.io/elemental-toolkit/docs/customizing/general_configuration/[fichier de configuration personnalisé], situé par défaut dans `+/etc/elemental/config.yaml+`. Si vous avez plusieurs fichiers yaml, vous devez les ajouter dans le répertoire `+/etc/elemental/config.d+`.

L'exemple suivant définit une partition supplémentaire au cours de l'installation :

[,yaml]
----
install:
  extra-partitions:
  - size: 10240
    fs: ext4
    label: EXTRA_PARTITION
----

Pour le rendre disponible au moment de l'installation, il suffit d'ajouter le fichier supplémentaire dans la configuration du nuage de ressources SeedImage, comme décrit dans la section précédente de cette page. Prenons un exemple :

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: custom-seed
  namespace: fleet-default
spec:
  ...
  cloud-config:
    stages:
      boot:
      - name: "Add Elemental client configuration file"
        files:
        - path: /etc/elemental/config.d/extra-partition.yaml
          permissions: 0664
          content: |
            install:
              extra-partitions:
              - size: 10240
                fs: ext4
                label: EXTRA_PARTITION
----
