= Installer {elemental-product-name} dans un environnement sans air
:revdate: 2025-11-06
:page-revdate: {revdate}

== Hypothèses

Une installation Rancher à air comprimé doit être déjà configurée conformément à la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/air-gapped/air-gapped.html[documentation officielle de Rancher (]
En particulier, un registre privé devrait être disponible dans l'infrastructure à couverture aérienne.

== Vue d'ensemble

Afin d'exécuter {elemental-product-name} dans un environnement à air comprimé, les éléments suivants sont nécessaires :

* les tableaux de l'opérateur {elemental-product-name} 
* les images de conteneurs référencées dans les tableaux (les images de l _'opérateur élémentaire_ et du _constructeur de semences)_ 
* les images du système d'exploitation conteneurisé

De plus, il pourrait être pratique de créer une _image de canal_ référençant les images de systèmes d'exploitation conteneurisés disponibles.
L'image officielle du canal (celle du _canal élémentaire_ ) fait référence aux URL absolues des images du système d'exploitation sur le registre officiel de SUSE, elle ne peut donc pas être utilisée telle quelle dans un scénario d'air-gapped.

== {elemental-product-name} Installation d'Air-Gapped à partir de la ligne de commande

Toutes les étapes requises peuvent être accomplies en exécutant lehttps://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[script]

Les graphiques {elemental-product-name} sont un paramètre obligatoire pour le script et peuvent être fournis sous forme d'archives téléchargées, d'URL ou comme l'un des mots-clés `+stable+`, `+staging+` et `+dev+`, afin de permettre au script de récupérer la version correcte du graphique pour vous.

`+elemental-airgap.sh+` inspecte le tableau {elemental-product-name} Operator, identifie toutes les images de conteneurs nécessaires, les télécharge et les enregistre dans une archive docker unique.
Il construit également une nouvelle image du canal OS avec les URLs de l'image OS pointant vers le registre privé passé en argument (qui est également un argument obligatoire).

La dernière version du script elemental peut être facilement téléchargée depuis le repo officiel de github :

[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh
chmod 755 elemental-airgap.sh
----

Nous allons maintenant télécharger tous les artefacts et construire un canal personnalisé à partir de la dernière version stable de {elemental-product-name}:

[tabs]
====
Créer une archive Docker::
+
======
[,shell]
----
./elemental-airgap.sh stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

une fois terminé (le script peut prendre un certain temps), les fichiers suivants seront disponibles dans le répertoire actuel :

* `+elemental-operator-crds-chart-<*VERSION*>.tgz+`
* `+elemental-operator-chart-<*VERSION*>.tgz+`
* `+elemental-images.txt+`
* `+elemental-images.tar.gz+`
======

Créer une archive pour les transporteurs::
+
======
[,shell]
----
./elemental-airgap.sh -ha stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

une fois terminé (le script peut prendre un certain temps), les cartes et les images des conteneurs seront placées dans l'archive du transporteur nommée `+elemental-haul.tar.zst+`.
======
====

=== {elemental-product-name} installation 

Les fichiers et les archives créés par le script doivent être copiés sur un hôte qui : 

* A accès au registre privé. 
* Le binaire kubectl est installé et configuré pour accéder au cluster Rancher air-gapped. 
* Le binaire de barre est installé. 

[tabs]
========
Installation à partir d'une archive Docker::
+
======
Si le registre privé nécessite une authentification, vous devez vous y connecter avec docker : 

[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

L'installation de {elemental-product-name} se fait en deux étapes : 

* charger l'archive avec toutes les images de conteneurs nécessaires sur le registre privé : cela peut être fait en utilisant le script `+rancher-load-images.sh+` distribué avec la version de Rancher et déjà utilisé pour le déploiement air-gapped de Rancher : 

[,shell]
----
rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

* installer les graphiques élémentaires téléchargés en configurant le registre local et le canal nouvellement créé : 

[,shell]
----
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ 
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM>
----
======

Installation à partir d'une archive de transport::
+
======
Pour installer à partir d'une archive https://rancherfederal.github.io/hauler-docs/[Hauler] (option`+-ha+` dans `+elemental-airgap.sh+`), l'installation Hauler est également requise sur l'hôte à partir duquel l'installation est effectuée. 

Si le registre privé nécessite une authentification, vous devez vous y connecter avec Hauler : 

[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD
----

L'installation de {elemental-product-name} se fait en trois étapes : 


* Charger l'archive Haul 'elemental-haul.tar.zst' dans l'instance Hauler de l'infrastructure airgapped : 

[,shell]
----
hauler store load 'elemental-haul.tar.zst' 
----

* Si le registre local de l'environnement à air comprimé n'est pas le serveur de Hauler, chargez l'archive Haul dans le registre local : 

[,shell]
----
hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

[NOTE]
.Le transporteur peut également servir de registre
====
Si le registre local est desservi par une instance de Hauler, il suffit de charger l'archive Haul directement à cet endroit (comme indiqué à l'étape (1)) et de sauter l'étape (2).
====

* Extraire les cartes élémentaires du magasin Hauler et les installer : 

[,shell]
----
hauler store extract elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
hauler store extract elemental-operator-chart-<ELEMENTAL-VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<ELEMENTAL-VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ -
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM:PORT>
----
======
========

[NOTE]
.Le script airgap élémentaire génère les commandes nécessaires 
====
Le script `+elemental-airgap.sh+` imprime les commandes requises indiquées ci-dessus, mais en utilisant la version actuelle du graphique et l'URL du registre fournie pour permettre de copier et de coller facilement les commandes exactes.
====

== {elemental-product-name} Air-Gapped installation from the Rancher Marketplace (en anglais) 

Une installation Rancher à air comprimé comprend également les tableaux de l'opérateur {elemental-product-name} et les images des conteneurs de l'opérateur et de l'image de semence. 

Pour collecter les images d'OS manquantes et créer une image de canal d'OS pour votre registre privé, exécutez le https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[script] `+-co+`. 

A titre d'exemple, ciblons l'image `+elemental-channel+` de la dernière version stable de {elemental-product-name}. Le script se chargera de télécharger la carte de l'opérateur {elemental-product-name} (si nécessaire), d'extraire l'URL de l'image du canal OS, de la télécharger, d'inspecter toutes les images OS référencées, de les télécharger toutes et de créer un nouveau canal OS avec des liens vers le registre privé du scénario air-gapped. 

[tabs]
========
Créer une archive Docker::
+
======
[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh 
chmod 755 elemental-airgap.sh 
./elemental-airgap.sh stable -co -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

une fois terminé (le script peut prendre un certain temps), les fichiers suivants seront disponibles dans le répertoire actuel : 

- `+elemental-operator-crds-chart-<*VERSION*>.tgz+` 
- `+elemental-operator-chart-<*VERSION*>.tgz+` 
- `+elemental-images.txt+` 
- `+elemental-images.tar.gz+`
======

Créer une archive pour les transporteurs::
+
======
[,shell]
----
./elemental-airgap.sh -ha -co stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

une fois terminé (le script peut prendre un certain temps), les images des conteneurs seront placées dans l'archive du transporteur nommée `+elemental-haul.tar.zst+`.
======
========

=== {elemental-product-name} installation 

L'archive générée doit être chargée dans le registre privé à air comprimé. 

[tabs]
========
Installation à partir d'une archive Docker::
+
======
Si le registre privé nécessite une authentification, vous devez vous y connecter avec docker : 
[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

Le script imprimera les commandes nécessaires pour charger les images via l'outil Rancher `+rancher-load-images.sh+`, utilisé pour les installations Rancher à air comprimé. Il devrait être quelque chose comme : 

[,shell]
----
NEXT STEPS: 

1) Load the 'elemental-images.tar.gz' to the local registry (<REGISTRY.YOURDOMAIN.COM:PORT>) available in the airgapped infrastructure: 

./rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

Une fois que les images du système d'exploitation et des canaux sont chargées, vous devez ignorer le point (2) de la sortie du script (qui installera les cartes {elemental-product-name} à partir des archives téléchargées) et effectuer l'installation de l'opérateur {elemental-product-name} à partir de l'interface utilisateur Rancher.
======

Installation à partir d'une archive de transport::
+
======
Si le registre privé nécessite une authentification, vous devez vous y connecter avec Hauler : 
[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD 
----

Le script imprimera les commandes nécessaires au chargement des images. Il devrait être quelque chose comme : 

[,shell]
----
NEXT STEPS: 

* Load the 'elemental-haul.tar.zst' Haul archive in the Hauler instance in the airgapped infrastructure: 

hauler store load 'elemental-haul.tar.zst' 

* If the local registry in the air-gapped environment is not server by Hauler, load the Haul archive in the local registry: 

hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

Une fois que les images du système d'exploitation et des canaux sont chargées, vous devez ignorer le point (3) de la sortie du script (qui installera les cartes {elemental-product-name} à partir des archives téléchargées) et effectuer l'installation de l'opérateur {elemental-product-name} à partir de l'interface utilisateur Rancher.
======
========

== {elemental-product-name} Extension de l'interface utilisateur 

Rancher 2.7.x ne supporte pas le plugin d'extension d'interface utilisateur dans les environnements air-gapped, et donc l'interface utilisateur {elemental-product-name} n'est pas disponible dans Rancher 2.7.x. 

Le plugin {elemental-product-name} UI sera présent dans les extensions UI disponibles dans Rancher 2.8.0.
