= Réinitialisation de la machine
:revdate: 2025-09-08
:page-revdate: {revdate}

Il existe deux façons de remettre les machines {elemental-product-name} dans leur état d'origine ou de les mettre hors service :

. Lors de la suppression d'un cluster, toutes les machines associées seront réinitialisées.
+
image::reset-cluster-deletion.png[Supprimer un cluster pour réinitialiser toutes les machines]

. Lors de la gestion d'un cluster, il suffit de supprimer le nœud qui doit être réinitialisé.
+
image::reset-single-node-deletion.png[Supprimer un seul nœud pour le réinitialiser]

== Réinitialiser le flux de travail

Une fois que le site `+MachineInventory+` concerné est signalé pour suppression, un plan de réinitialisation est exécuté par le site `+elemental-system-agent+` qui fonctionne sur la machine.

Si la machine fonctionne encore, ce plan le fera :

. Redémarrez l'ordinateur en mode de récupération.
. Exécuter `+systemctl start elemental-register-reset+`. +
Cela permettra de récupérer le site `+MachineRegistration+` et d'appliquer les options de `+spec.config.elemental.reset+` pour réinitialiser la machine. +
Un nouveau site `+MachineInventory+` sera créé et le site `+spec.config.cloud-config+` défini dans le site `+MachineRegistration+` sera à nouveau appliqué.

Notez que la référence `+MachineRegistration+` *ne* changera pas, la machine *ne sera pas* réinstallée, les partitions `+COS_PERSISTENT+` et `+COS_OEM+` seront effacées par défaut si la réinitialisation est `+enabled+`. Pour plus d'informations, vous pouvez consulter le xref:installation/installation.adoc#_deployed_partition_table[tableau des partitions].

Étant donné que le site `+cloud-config+` est réappliqué au cours du processus de réinitialisation, vous pouvez réinitialiser une machine pour appliquer les mises à jour de la définition `+MachineRegistration+`, par exemple pour faire pivoter les informations d'identification `+users+` et les clés autorisées. Il est fortement recommandé d'activer l'option `+reset-oem+`, afin d'éviter les conflits avec les cloud-configs précédemment configurées.

Si vous devez lier une machine à un autre site `+MachineRegistration+` et déclencher une nouvelle installation complète, vous devez la reprovisionner à nouveau en utilisant une nouvelle image.

== Activer la réinitialisation de la machine

Pour permettre la réinitialisation automatique des machines, l'indicateur `+spec.config.elemental.reset.enabled+` du site `+MachineRegistration+` doit être activé. +
Ce paramètre est désactivé par défaut, mais une fois activé, tous les nouveaux sites `+MachineInventory+` hériteront automatiquement de ce paramètre. +
Par exemple :

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: fire-nodes
  namespace: fleet-default
spec:
  config:
    elemental:
      reset:
        enabled: true
        reset-persistent: true
        reset-oem: true
        # These cloud-init configs will be created during reset and will persist on the system after
        config-urls:
          - "https://my.cloud.init/reset-plan-1.yaml"
          - "https://my.cloud.init/reset-plan-2.yaml"
        # You can select a different image to run the reset.
        # Note that this image will not be installed on the system.
        system-uri: "my.oci.registry/reset-image:latest"
        power-off: false
        reboot: true
----

Il est également possible d'activer la réinitialisation au niveau `+MachineInventory+`, par exemple dans des scénarios où certaines machines sont physiques et bénéficieront d'une réinitialisation automatique, tandis que d'autres sont virtuelles et peuvent simplement être détruites et reprovisionnées si nécessaire. +
Afin de marquer un seul `+MachineInventory+` pour autoriser la réinitialisation, vous pouvez utiliser l'annotation `+elemental.cattle.io/resettable: true+`. +
Par exemple :

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineInventory
metadata:
  annotations:
    elemental.cattle.io/resettable: "true"
----
