= ManagedOSVersionChannel reference
:revdate: 2025-09-08
:page-revdate: {revdate}

La ressource `+ManagedOSVersionChannel+` est chargée de définir le canal de l'image OS.

Plusieurs clés peuvent être configurées sous une spécification de ressource `+ManagedOSVersionChannel+`.

.managedosversionchannel-example.yaml
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: ManagedOSVersionChannel
metadata:
  name: elemental-channel
  namespace: fleet-default
spec:
  options:
    image: registry.suse.com/rancher/elemental-channel:1.4.2
  syncInterval: 1h
  type: custom
----

== `+ManagedOSVersionChannelSpec+` référence

|===
| Clé | Type | Valeur par défaut | Description

| options
| objet
| nul
| Définit les informations facultatives qui peuvent être ajoutées dans un canal OS.

| syncInterval
| chaîne de caractères
| 1h
| Définit quand synchroniser le canal OS

| type
| chaîne de caractères
| vide
| Définit le type de canal, seul `+custom+` est pris en charge actuellement.

| upgradeContainer
| objet
| nul
| Un conteneur de mise à niveau qui peut être défini. Voir la <<_upgradecontainer,référence>>

| deleteNoLongerInSyncVersions
| bool
| faux
| Supprimer automatiquement les versions obsolètes du système d'exploitation qui ne sont plus incluses dans le canal.

| activée
| bool
| vrai
| Active ce canal. Permettre la synchronisation des versions du système d'exploitation.
|===

== `+upgradeContainer+`

Cela permet d'écraser le champ par défaut `+upgrade+` des plans du contrôleur de mise à niveau du système (voir xref:rancher-os-management/upgrade-lifecycle.adoc#_components[composants de mise à niveau]) sur la base de cette ManagedOSVersion.
Ces clés sont traduites par le contrôleur de mise à niveau du système en une spécification de Kubernetes https://kubernetes.io/docs/reference/kubernetes-api/workload-resources/pod-v1/#Container[conteneur].
Il s'agit du conteneur responsable de l'exécution d'une mise à jour du système d'exploitation.
