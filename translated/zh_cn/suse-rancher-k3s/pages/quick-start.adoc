== 安装脚本

{k3s-product-name} 提供了一个安装脚本，可以方便地将其作为服务安装到基于 systemd 或 openrc 的系统上。该脚本可在https://get.k3s.io. 上获取 要使用此方法安装{k3s-product-name} ，请执行以下操作：

运行安装程序，其中`+INSTALL_K3S_ARTIFACT_URL+` 是https://scc.suse.com/rancher-docs/rancherprime/latest/en/reference-guide.html#prime-artifacts-url[Prime Artifacts URL]，`+INSTALL_K3S_CHANNEL+` 是可以订阅的发布频道，默认为`+stable+` 。在本例中，`+INSTALL_K3S_CHANNEL="latest"+` 将为您提供{k3s-product-name} 的最新版本。

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_CHANNEL="latest" sh -
----
如果要指定版本，请设置`+INSTALL_RKE2_VERSION+` 环境变量：

[,bash]
----
curl -sfL https://get.k3s.io/ | sudo INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_VERSION="<VERSION>" sh -
----

运行此安装后：

* K3s 服务将被配置为在节点重启后或进程崩溃或被杀死时自动重启
* 还将安装其他实用程序，包括`+kubectl+`,`+crictl+`,`+ctr+`,`+k3s-killall.sh+`, 和 `+k3s-uninstall.sh+`
* https://kubernetes.io/docs/concepts/configuration/organize-cluster-access-kubeconfig/[]的 kubeconfig 文件将被写入`+/etc/rancher/k3s/k3s.yaml+` ，由{k3s-product-name} 安装的 kubectl 将自动使用该文件

单节点服务器安装是一个功能完备的 Kubernetes 集群，包括托管工作负载 pod 所需的所有数据存储、控制平面、kubelet 和容器运行时组件。没有必要添加额外的服务器或代理节点，但您可能希望这样做来增加群集的容量或冗余。

要安装其他代理节点并将其添加到群集，请使用`+K3S_URL+` 和`+K3S_TOKEN+` 环境变量运行安装脚本。下面是一个如何加入代理的示例：

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s K3S_URL=https://myserver:6443 K3S_TOKEN=mynodetoken sh -
----

设置`+K3S_URL+` 参数会使安装程序将 K3s 配置为代理，而不是服务器。K3s 代理将通过提供的 URL 向监听的 K3s 服务器注册。`+K3S_TOKEN+` 使用的值存储在服务器节点上的`+/var/lib/rancher/k3s/server/node-token+` 中。

[NOTE]
====
每台机器都必须有一个唯一的主机名。如果您的机器没有唯一的主机名，请通过`+K3S_NODE_NAME+` 环境变量，并为每个节点提供一个有效且唯一的主机名值。
====


如果您想拥有更多服务器节点，请参阅xref:datastore/ha-embedded.adoc[高可用性嵌入式 etcd]和xref:datastore/ha.adoc[高可用性外部 DB]页面了解更多信息。
