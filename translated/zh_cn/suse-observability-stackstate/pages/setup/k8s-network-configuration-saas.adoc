= 网络配置
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE 可观察性

SUSE Observability 是一款托管在云中的 SaaS 产品。SUSE Observability Agent 需要能够连接到 SUSE Observability SaaS Receiver API，才能从您的办公场所/云与 SUSE Observability SaaS 进行通信。
当集群在专用网络中运行时，您可能需要对网络进行配置，以允许 SUSE Observability Agent 连接到 SUSE Observability Receiver API，因为您的网络配置可能不允许出口流量到互联网。本页介绍如何配置网络以允许安装 SUSE Observability Agent，并允许 SUSE Observability Agent 与 SUSE Observability Receiver API 通信。

[NOTE]
====
SUSE Observability Agent 与 SUSE Observability Receiver API 之间的流量始终由 SUSE Observability Agent 发起。SUSE Observability Receiver API 不会向 SUSE Observability Agent 发起任何流量。
====


== 安装 SUSE Observability Agent

SUSE Observability Agent 的安装是通过 Helm 完成的。默认情况下，Helm Chart 被配置为从 Quay.io docker 注册表中提取 SUSE Observability Agent 容器映像。如果您的网络配置不允许出口流量到互联网，您可以选择安装 SUSE Observability Agent：

. 配置网络，允许从 Kubernetes 集群向 Quay.io 容器注册中心发送出口流量。
. 通过自己的容器注册表代理 Quay.io 容器注册表。
. 将 Docker 映像拉入自己的容器注册表。

对于选项 2 和 3，您需要配置 Helm Chart，以便从自己的容器注册表中提取 SUSE Observability Agent 容器映像。有关配置 SUSE Observability Agent Helm Chart 以从自己的容器注册表中提取映像的指南，请xref:/k8s-suse-rancher-prime-agent-air-gapped.adoc[点击此处]。

== SUSE Observability Agent 通信

SUSE Observability Agent 通过 HTTPS 与 SUSE Observability Receiver API 通信。SUSE Observability Agent 的不同部分连接到托管在租户中的 SUSE Observability Receiver API，请参见下图：

image::k8s/k8s-agent-communication.png[SUSE Observability Agent 通信]

所有通信都通过 HTTPS 进行，使用标准 HTTPS 端口 443。SUSE Observability Agent 使用以下端点与 SUSE Observability Receiver API 通信：

* https://*<tenant>.app.stackstate.io/receiver/stsAgent*- SUSE Observability Agent 向 SUSE Observability Receiver API 发送度量、事件和拓扑数据。

为了允许 SUSE Observability Agent 与 SUSE Observability Receiver API 通信，您需要配置网络以允许出口流量到 SUSE Observability Receiver API。SUSE Observability Receiver API 托管在云中，并为您的租户提供特定的 IP。您需要允许出口流量进入互联网。为了获取允许出口流量的正确 IP 地址，可以使用以下命令：

[,bash]
----
$ dig +short <tenant>.app.stackstate.io
----

或者，您也可以在浏览器中访问以下 URL： `+https://www.nslookup.io/domains/<tenant>.app.stackstate.io/dns-records/+`
