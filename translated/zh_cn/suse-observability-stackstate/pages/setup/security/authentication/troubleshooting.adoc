= 对身份验证和授权进行故障排除
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability 自助托管

当身份验证或授权失败时，通常是由于提供程序和 SUSE Observability 的配置不匹配造成的。为了更方便地排除故障，可以在 SUSE Observability 上专门针对身份验证和授权启用调试日志。

[CAUTION]
====
一旦完成故障排除，请立即再次禁用调试日志，因为调试日志很可能包含机密和/或个人信息。
====


要启用调试日志，请将以下 yaml 代码段复制/粘贴到`+debug-auth.yaml+` 文件中。

[,yaml]
----
stackstate:
  components:
    server:
      additionalLogging: |
        <logger name="org.pac4j.core.engine" level="DEBUG"/>
        <logger name="org.pac4j.oidc.profile.creator" level="DEBUG"/>
        <logger name="org.pac4j.oidc.credentials.authenticator" level="DEBUG"/>
    api:
      additionalLogging: |
        <logger name="org.pac4j.core.engine" level="DEBUG"/>
        <logger name="org.pac4j.oidc.profile.creator" level="DEBUG"/>
        <logger name="org.pac4j.oidc.credentials.authenticator" level="DEBUG"/>
----

现在运行之前使用的`+helm upgrade+` 命令，但要包含一个额外的 yaml 文件（所以是`+helm upgrade .... --values debug-auth.yaml+` ），以启用调试日志。不会重新启动 pod，日志配置更改将在约 30 秒后自动加载。

要禁用调试记录，请再次运行`+helm upgrade ....+` 命令，但省略`+--values debug-auth.yaml+` 。30 秒后，更新的日志记录配置被加载，调试日志记录停止。
