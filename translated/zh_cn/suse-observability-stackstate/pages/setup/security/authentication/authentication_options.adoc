:revdate: 2025-07-10

:description: SUSE Observability 自助托管

开箱即用，SUSE Observability 采用xref:/setup/security/authentication/file.adoc[单密码身份验证]，管理员用户和随机密码xref:/setup/install-stackstate/initial_run_guide.adoc#_default_username_and_password[在安装过程中]配置。这样，用户就可以通过服务器上的一个简单秘密进行身份验证。不过，这并不是一个生产就绪的设置。

为提高安全性，可将 SUSE Observability 配置为完全使用以下一种身份验证机制（取代标准管理员用户）：

* xref:/setup/security/authentication/single_password.adoc[单密码]
* xref:/setup/security/authentication/file.adoc[基于文件]
* xref:/setup/security/authentication/ldap.adoc[LDAP]
* xref:/setup/security/authentication/oidc.adoc[开放 ID 连接（OIDC）]
* xref:/setup/security/authentication/keycloak.adoc[KeyCloak（OIDC 的专用版本）]

[NOTE]
====
身份验证配置是 Helm 图表的一部分，任何更改都会自动触发需要该配置的 pod 的重启。
====


== 用户角色

当用户通过身份验证后，通常会根据用户的角色为其分配权限。特定身份验证机制的文档还包含如何将外部系统的角色或组映射到 SUSE Observability 的 4 个标准角色的示例：

* *访客*- 可以查看信息，但不能做任何更改。
* *Kubernetes 故障排除器*- 能够查看所有信息，查看并更改监控器和度量配置。
* *高级用户*- 可以查看和更改所有配置并安装 StackPack。
* *管理员*- 能够查看和更改 SUSE Observability 的内容。例如，查看所有配置、安装堆栈包、授予和撤销用户权限以及上传（新版本的）堆栈包。

在决定为用户分配角色时，强烈建议只分配一小部分平台管理员和管理员。例如，只有工程师负责安装 SUSE Observability 和进行初始配置。管理员用户可以管理对 SUSE Observability 的访问，并决定可以使用哪些 StackPack。您可以将 StackPack 的安装和其他配置微调工作委托给更多具有 "高级用户 "角色的用户。平台管理员用户可以清除数据库、更改数据保留设置、查看日志并执行其他平台管理任务。

还可以添加更多角色，请参阅xref:/setup/security/rbac/rbac_roles.adoc[角色 (RBAC)]页面和其他xref:/setup/security/rbac/README.adoc[RBAC 文档页面]
