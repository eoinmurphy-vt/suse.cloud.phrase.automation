= 外部秘密
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability 外部机密

== 概述

SUSE Observability 可以通过舵机安装获取许可证密钥、API 密钥和身份验证密钥等机密信息，也可以从已配置的机密信息中获取这些信息。

下面介绍了如何进行配置。

== 从外部秘密获取许可证密钥

在 SUSE Observability 所安装的命名空间中创建一个秘密，格式如下，请填写空格：

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
  LICENSE_KEY: "<base64 of the license key>"
----

在 helm 安装命令中添加以下内容，以使用该秘密：

[,bash]
----
  --set 'stackstate.license.fromExternalSecret'='<custom-secret-name>'
----

== 从外部秘密获取电子邮件通知的用户名和密码

在 SUSE Observability 所安装的命名空间中创建一个秘密，格式如下，请填写空格：

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
  SMTP_USER_NAME: "<base64 of the smtp username>"
  SMTP_PASSWORD: "<base64 of the smtp password>"
----

在 helm 安装命令中添加以下内容，以使用该秘密：

[,bash]
----
  --set 'stackstate.email.server.auth.fromExternalSecret'='<custom-secret-name>'
----

== 从外部密钥获取 api 密钥

== 从外部秘密获取身份验证数据

在安装 SUSE Observability 的命名空间中创建一个秘密，格式如下。

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
   default_password: <base64 of bcrypted password>
----

根据所选身份验证方法的不同，`+default_password+` 字段会被替换为不同的数据。更多信息请参阅xref:/setup/security/authentication/authentication_options.adoc[身份验证选项]。本例中使用的是xref:/setup/security/authentication/single_password.adoc[单密码]设置。

在 helm 安装命令中添加以下内容，以使用该秘密：

[,bash]
----
  --set 'stackstate.authentication.fromExternalSecret'='<custom-secret-name>'
----
