= 来自 OpenTelemetry 的拓扑
:revdate: 2025-12-23
:page-revdate: {revdate}
:description: 来自 OpenTelemetry 的拓扑

{stackstate-product-name} 可从 OpenTelemetry (OTel) 遥测数据中自动推导出应用程序和基础设施拓扑结构。
这是通过 OpenTelemetry 拓扑映射实现的，该映射可声明性地将轨迹和度量信号转换为拓扑组件（顶点）和关系（边）。

== 概述

在高层次上，OpenTelemetry 的拓扑生成工作原理如下：

* 采集器接收 OpenTelemetry 遥测（度量和/或跟踪）。
* OTel 收集器使用拓扑连接器，根据遥测数据评估配置的映射。
* 当映射条件与遥测匹配时，就会生成组件和/或关系。
* `+sync+` 服务会使用这些拓扑对象，并将其合并到拓扑图中。
* 映射是声明性的，基于 YAML，并随着遥测数据在系统中的流动而不断进行评估。

目前支持的信号

* 衡量标准
* 痕迹

[NOTE]
====
目前还不支持基于日志的拓扑生成。
====

有关组件和关系映射的更多信息，请参阅xref:/setup/custom-integrations/otelmappings/concepts.adoc[概念]页面。
