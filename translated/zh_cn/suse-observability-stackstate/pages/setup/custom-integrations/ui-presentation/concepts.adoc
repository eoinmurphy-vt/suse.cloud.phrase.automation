== ComponentPresentation

用户界面呈现模型的核心概念是*组件呈现（ComponentPresentation*）。

`+ComponentPresentation+` 定义了一个或多个匹配组件在用户界面中的显示方式，具体取决于活动上下文。
它只关注*演示方面的问题*，如命名、图标、表格、概览和高亮页面、过滤器以及菜单项的位置。

在高层次上，一个 ComponentPresentation：

* 使用绑定（简单的 STQL 查询）选择组件。
* 适用于特定模式（如可观察性）。
* 定义一个或多个演示方面（如概览表或要点）。
* 可部分定义呈现属性，并与其他匹配定义组合。

同一组件可使用多个 ComponentPresentations。在这种情况下，它们的定义会合并起来，以确定最终的列报方式。
