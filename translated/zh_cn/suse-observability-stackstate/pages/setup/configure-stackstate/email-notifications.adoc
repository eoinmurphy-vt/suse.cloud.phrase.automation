= 为电子邮件通知配置 SUSE Observability
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability 自助托管

[CAUTION]
====
SUSE Observability 的 SaaS 用户无需额外配置即可使用电子邮件通知。本指南仅适用于计划使用电子邮件通知渠道的自托管 SUSE Observability 安装。
====


在 SUSE Observability 中使用电子邮件通知通道之前，首先需要执行以下步骤：

== 使用 SMTP 配置配置 SUSE Observability

SUSE Observability 需要配置连接到 SMTP 服务器的凭据。您可以在 SUSE Observability 安装的`+values.yaml+` 文件中添加以下内容：

[,yaml]
----
stackstate:
  email:
    enabled: true
    sender: "<stackstate@example.com>"
    server:
      host: "<smtp.example.com>"
      auth:
        username: "<user name>"
        password: "<user password>"
----

这将使用 SMTP 服务器上的`+587+` 端口，并使用`+STARTTLS+` 命令建立安全连接。这些都是可以定制的其他选项：

[,yaml]
----
stackstate:
  email:
    additionalProperties:
      # Add needed Java email properties for your mail server (use string values), defaults are:
      "mail.smtp.auth": "true"
      "mail.smtp.starttls.enable": "true"
    server:
      protocol: smtp
      port: 587
----

=== 使用外部秘密

如果用户名和密码不能在值中提供，而应来自外部秘密，请按xref:/setup/security/external-secrets.adoc#_getting_username_and_password_for_email_sending_from_an_external_secret[以下步骤]操作
