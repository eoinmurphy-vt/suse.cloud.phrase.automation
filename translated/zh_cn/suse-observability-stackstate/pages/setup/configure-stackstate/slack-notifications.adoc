= 为 Slack 通知配置 SUSE Observability
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability 自助托管

[CAUTION]
====
SUSE Observability 的 SaaS 用户无需额外配置即可使用 Slack 通知。本指南仅适用于计划使用 Slack 通知通道的自托管 SUSE Observability 安装。
====


在 SUSE Observability 中使用 Slack 通知通道之前，您首先需要按照以下步骤设置 Slack 和 SUSE Observability：

. 在工作区为 SUSE Observability 创建 Slack 应用程序
. 使用 Slack 应用程序的凭据配置 SUSE Observability。

== 为 SUSE Observability 创建 Slack 应用程序

[NOTE]
====
您需要在 Slack 中拥有管理工作区 Slack 应用程序的权限。
====


访问https://api.slack.com/apps[Slack API 页面]，单击 "*创建新应用程序*"按钮。

* 在打开的对话框中选择 "从应用程序清单 "选项。
* 选择要发送通知的工作区，然后单击下一步。
* 复制下面的 Slack 应用程序清单内容并粘贴到文本区域。确保将`+redirect_urls+` 中的值替换为 SUSE Observability 实例的 URL。点击下一步。
* 验证 URL 是否正确，列出的 "机器人作用域 "是否为`+channels:join, channels:read, chat:write, groups:read+` ，然后单击创建按钮创建应用程序。
* 在应用程序的 "基本信息 "页面，您可以更改图标（在 "显示信息 "部分），例如用 SUSE Observability 徽标代替image:logo/stackstate-logo.png[Observability 徽标,25,25] SUSE。

.用于 SUSE Observability 的 Slack 应用程序清单
[,json]
----
{
    "display_information": {
        "name": "SUSE Observability",
        "description": "Receive notification messages from SUSE Observability",
        "background_color": "#000000"
    },
    "features": {
        "bot_user": {
            "display_name": "SUSE Observability",
            "always_online": true
        }
    },
    "oauth_config": {
        "redirect_urls": [
            "https://the.url.of.your.stackstate.installation"
        ],
        "scopes": {
            "bot": [
                "channels:join",
                "channels:read",
                "chat:write",
                "groups:read"
            ]
        }
    },
    "settings": {
        "org_deploy_enabled": false,
        "socket_mode_enabled": false,
        "token_rotation_enabled": false
    }
}
----


== 使用 Slack 应用程序的凭据配置 SUSE Observability

SUSE Observability 需要使用您创建的 Slack 应用程序的凭据进行配置。您可以在 SUSE Observability 安装的`+values.yaml+` 文件中添加以下内容：

[,yaml]
----
stackstate:
  components:
    all:
      extraEnv:
        open:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientId: "<app client id>"
        secret:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientSecret: "<app client secret>"
----

`+<app client id>+` 和`+<app client secret>+` 值可在您创建的 Slack 应用程序的 "基本信息 "页面的 "应用程序凭据 "部分找到。运行安装 SUSE Observabilityxref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[（用于 Kubernetes]或xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_deploy_suse_observability_with_helm[OpenShift]）时使用的相同 Helm 命令来应用这些配置更改。

现在就可以使用 Slack 通知频道了！
