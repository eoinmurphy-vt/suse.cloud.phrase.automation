=== 创建命名空间

首先创建要安装{stackstate-product-name} 的命名空间，并在该命名空间中部署秘密。在演练中，我们将使用命名空间`+suse-observability+` ：

[,text]
----
kubectl create namespace suse-observability
----

[#create-values-yaml]
=== 创建`+values.yaml+` 配置文件

`+values.yaml+` 文件包含部署{stackstate-product-name} 所需的所有配置，包括许可证密钥、基本 URL、管理员密码和大小配置文件。

[NOTE]
====
*在您继续之前*请确保您拥有最新版本的舵手图表`+helm repo update+` 。
====

[tabs]
====
建议方法::
+
--
[NOTE]
=====
`+global.suseObservability+` 配置方法从`+2.8.0+` 版本开始提供。对于早期版本，请使用传统方法。
=====

使用`+global.suseObservability+` 配置结构创建`+values.yaml+` 文件：

[source,yaml]
----
global:
  # Optional: Override image registry (defaults to registry.rancher.com)
  # Only needed for air-gapped environments or custom registries
  # imageRegistry: "your-private-registry.example.com"

  # Optional: Override storage class (uses cluster default if not set)
  # storageClass: "your-storage-class"

  suseObservability:
    # Required: Your {stackstate-product-name} license key
    license: "YOUR-LICENSE-KEY"

    # Required: Base URL for {stackstate-product-name}
    # Use http://localhost:8080 if ingress is not configured yet
    baseUrl: "https://observability.example.com"

    # Required: Sizing profile
    # Available: trial, 10-nonha, 20-nonha, 50-nonha, 100-nonha,
    #            150-ha, 250-ha, 500-ha, 4000-ha
    sizing:
      profile: "150-ha"

    # Required: Plain text admin password
    adminPassword: "your-password"
    # Instead of adminPassword you can provide a bcrypt hashed password with adminPasswordBcrypt
    # Generate with: htpasswd -bnBC 10 "" "your-password" | tr -d ':\n'
    # adminPasswordBcrypt: "$2a$10$..."

    # Optional: Receiver API key (auto-generated if not provided)
    # receiverApiKey: "your-receiver-api-key"

    # Optional: Pull secret for private registry
    # pullSecret:
    #   username: "registry-user"
    #   password: "registry-pass"
----

有关包括亲缘设置在内的更多配置选项，请参阅xref:/setup/install-stackstate/kubernetes_openshift/affinity.adoc[配置 Kubernetes 亲缘]。

该方法已被弃用。对于新安装的设备，请使用上述推荐方法。对于使用此方法的现有安装，请参阅xref:/setup/upgrade-stackstate/migrate-to-global-mode.adoc[迁移指南]以过渡到新的配置格式。

您可以通过运行单独的 Helm Chart 生成{stackstate-product-name} 配置文件。这是`+suse-observability/suse-observability-values+` 图表：

[,text]
----
export VALUES_DIR=.
helm template \
  --set license='<your license>' \
  --set baseUrl='<suse-observability-base-url>' \
  --set sizing.profile='<sizing.profile>' \
  suse-observability-values \
  suse-observability/suse-observability-values --output-dir $VALUES_DIR
----

该命令生成文件`+$VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml+` 、`+$VALUES_DIR/suse-observability-values/templates/sizing_values.yaml+` 和`+$VALUES_DIR/suse-observability-values/templates/affinity_values.yaml+` 。

[NOTE]
=====
{stackstate-product-name} 管理员密码将由上述命令自动生成，并在生成的`+basicConfig.yaml+` 文件中以注释形式输出。实际值包含这些密码的`+bcrypt+` 哈希值，以便安全地存储在集群中的 Helm 版本中。
=====

可传递给该图表的值有

[cols="1,1,2"]
|===
| 配置 | 价值 | 说明

| 基本配置生成
| `+basicConfig.generate+`
| 打开或关闭`+basicConfig_values.yaml+` 文件的生成。

| 接收器 API 密钥
| `+receiverApiKey+`
| {stackstate-product-name} 从代理接收数据时使用的 API 密钥。这是一个秘密密钥，应予以保密。如果省略此项，将为您生成一个随机密钥。

| 基本 URL
| `+baseUrl+`
| `+<STACKSTATE_BASE_URL>+`.{stackstate-product-name} 的外部 URL，用户和代理将使用该 URL 进行连接。例如`+https://suse-observability.internal+` 。如果您还没有决定入口配置，请使用`+http://localhost:8080+` 。稍后可在生成的文件中进行更新。

| 用户名和密码**
| `+-u+` `+-p+`
| {stackstate-product-name} 提取图像时使用的用户名和密码。对于空气屏蔽环境，这些信息必须是本地控制器注册表的用户名和密码。

| 许可证密钥
| `+license+`
| {stackstate-product-name} 许可证密钥。

| 默认密码
| `+adminPassword+`
| 默认用户（`+admin+` ）访问{stackstate-product-name} 用户界面的密码。或者，您也可以使用`+adminPasswordBcrypt+` 提供 bcrypt 已散列密码。两个选项只能同时使用一个。

| 图像登记处
| `+imageRegistry+`
| {stackstate-product-name} 镜像所在的注册表。如果未提供，默认值为 "registry.rancher.com"。仅适用于气隙环境或自定义注册表。

| 拉动秘密用户名
| `+pullSecret.username+`
| 用于从{stackstate-product-name} 镜像所在的 Docker 注册表中提取镜像的用户名。仅适用于自定义注册表。

| 调出秘密密码
| `+pullSecret.password+`
| 用于从{stackstate-product-name} 镜像所在的 Docker 注册表中提取镜像的密码。仅适用于自定义注册表。

| 确定尺寸
| `+sizing.generate+`
| 打开或关闭`+sizing_values.yaml+` 文件的生成。

| 尺寸规格
| `+sizing.profile+`
| 一次试验、10 公顷、20 公顷、50 公顷、100 公顷、150 公顷、250 公顷、500 公顷。根据这些配置文件，您可以生成`+sizing_values.yaml+` 文件，其中包含{stackstate-product-name} 资源和配置的默认大小，以便在 HA 或非 HA 模式下进行部署。例如，10-nonha 生成一个`+sizing_values.yaml+` ，用于部署一个 NonHa{stackstate-product-name} 实例，在非高可用性模式下观察一个 10 节点集群。目前，从非 HA 环境转移到 HA 环境是不可能的。如果您预计您的环境需要观察 150 个左右的节点，那么最好立即使用 HA。

| Pod AntiAffinity
| `+affinity.podAntiAffinity.requiredDuringSchedulingIgnoredDuringExecution+`
| 确保相同数据服务的副本调度到不同的 Kubernetes 节点。仅适用于`+HA+` 配置文件。`+true+` 或`+false+` 。默认值为`+true+` - 执行调度到不同节点。
|===

[NOTE]
=====
安全存储生成的`+basicConfig.yaml+` 和`+sizing_values.yaml+` 文件。您可以重复使用这些文件进行升级，从而节省时间，并确保{stackstate-product-name} 继续使用相同的 API 密钥。这样做是可取的，因为这意味着{stackstate-product-name} 的代理和其他数据提供者无需更新。
您可以使用`+basicConfig.generate=false+` 和`+sizing.generate=false+` 开关单独重新生成文件，禁用其中任何一个，同时仍将先前生成的文件版本保留在`+output-dir+` 中。
=====

[NOTE]
=====
{stackstate-product-name} 值图表会生成亲和配置，您可以将其与{stackstate-product-name} 主图表一起使用，以控制 pod 的调度行为。更多信息，请参阅xref:/setup/install-stackstate/kubernetes_openshift/affinity.adoc[配置 Kubernetes Affinities]文档。

=== 配置存储类别（可选）

{stackstate-product-name} 如果未指定，则使用群集的默认存储类。您可以选择在`+values.yaml+` 文件中设置`+global.storageClass+` 来覆盖此功能。

[tabs]
====
建议方法::
+
--
如果需要使用特定的存储类，请将其添加到`+global.storageClass+` 下的`+values.yaml+` 文件中：

[source,yaml]
----
global:
  storageClass: "your-storage-class"  # e.g., "local-path" for Rancher local-path provisioner
  suseObservability:
    # ... other configuration
----
--
旧方法（已废弃）::
+
--
编辑文件`+$VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml+` ，添加 "storageClass "配置：

[,text]
----
# Source suse-observability-values/templates/baseConfig_values.yaml
global:
  storageClass: "your-storage-class" # Optional: override default storage class
----

=== 使用 Helm 部署{stackstate-product-name} 

{stackstate-product-name} 的推荐部署方式是生产就绪、高可用性设置，许多服务冗余运行。如果需要，也可以在非冗余设置中运行{stackstate-product-name} ，即每个服务只有一个副本。仅建议在测试环境中使用此设置。

对于空气密封环境，请遵循空气密封安装说明。

[tabs]
====
建议方法::
+
--

在 Kubernetes 上部署{stackstate-product-name} ：

. 部署前
 ** xref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_create_namespace[创建安装{stackstate-product-name} 的命名空间]
 ** <<create-values-yaml,创建`+values.yaml+` 配置文件>>
. 使用以下命令将最新的{stackstate-product-name} 版本部署到`+suse-observability+` 命名空间：

[,text]
----
helm upgrade \
  --install \
  --namespace suse-observability \
  --values values.yaml \
suse-observability \
suse-observability/suse-observability
----

或者，不使用值文件，直接使用`+--set+` flags 进行部署：

[,bash]
----
helm upgrade \
  --install \
  --namespace suse-observability \
  --set global.suseObservability.license="YOUR-LICENSE-KEY" \
  --set global.suseObservability.baseUrl="https://observability.example.com" \
  --set global.suseObservability.sizing.profile="150-ha" \
  --set global.suseObservability.adminPassword='$2a$10$...' \
suse-observability \
suse-observability/suse-observability
----

=== 访问{stackstate-product-name} 用户界面

{stackstate-product-name} 部署完成后，您可以检查所有 pod 是否都已启动并运行：

[,bash]
----
kubectl get pods --namespace suse-observability
----

所有 pod 启动后，即可启用端口转发：

[,bash]
----
kubectl port-forward service/<helm-release-name>-suse-observability-router 8080:8080 --namespace suse-observability
----

对于入口配置，请创建`+ingress_values.yaml+` 文件并添加以下代码。

[NOTE]
====
记住为 TLS 配置设置一个密文。有关如何操作的详细信息，请查阅 kubectl 文档。
====

[,yaml]
----
ingress:
  enabled: true
  class: ""
  annotations:
    nginx.ingress.kubernetes.io/proxy-body-size: "50m"
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    #if external dns (cloudflare and similars) configure:
    #external-dns.alpha.kubernetes.io/target: <INGRESS_FQDN>
    #external-dns.alpha.kubernetes.io/ttl: "60"
    #end if
  hosts:
  - host: <SUSE_OBSERVABILITY_HOST>
  tls:
  - hosts:
    - <SUSE_OBSERVABILITY_HOST>
    secretName: suse-private-ai-tls
  existingSecret: suse-private-ai-tls
  # Set up your secrets in whichever way feels fit. Check k8s docs for details: https://kubernetes.io/docs/reference/kubectl/generated/kubectl_create/kubectl_create_secret_tls/
----
