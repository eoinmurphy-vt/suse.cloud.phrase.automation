= k8s 自定义秘密管理
:revdate: 2025-10-27
:page-revdate: {revdate}

== 概述

SUSE Observability Agent 支持指定包含 API 密钥和群集授权令牌的自定义密文名称。该功能对于希望管理自己的秘密并避免 Helm 图表自动创建秘密的用户非常有用。

NOTE: *以前有一种指定方法现已废弃，请参见xref:/setup/agent/k8s-custom-secrets-setup-deprecated.adoc[废弃方法]。*

== 舵手图

=== 配置选项

* `+global.apiKey.fromSecret+`:在同一命名空间中指定一个预先存在的秘密名称，该名称包含一个包含 api 密钥的`+STS_API_KEY+` 字段。
* `+global.clusterAgentAuthToken.fromSecret+`:指定驻留在同一命名空间中的已有密名，该密名包含一个`+STS_CLUSTER_AGENT_AUTH_TOKEN+` 字段，其中包含用于确保群集和节点代理之间连接安全的令牌。

=== 行为描述

* *自动创建秘密*：默认情况下，图表需要指定`+stackstate.apiKey+` ，并会自行创建一个秘密。`+STS_CLUSTER_AGENT_AUTH_TOKEN+` 是自动生成的。
* *自定义秘密管理*：覆盖`+fromSecret+` 字段时，将从这些秘密中获取 api 密钥和群集认证令牌。
* *默示遗漏*：当指定要管理自己的机密时，图表将忽略`+stackstate.apiKey+` 和`+stackstate.cluster.authToken+` 的值。

== 如何在 values.yaml 中使用

. *使用自动创建密文（默认）*：
+
[,yaml]
----
 stackstate:
   apiKey: "<your api key>"
----

. *管理自己的秘密*
+
[,yaml]
----
 global:
   apiKey:
     fromSecret: "name-of-my-api-key-secret"
   clusterAgentAuthToken:
     fromSecret: "name-of-my-cluster-agent-auth-token-secret"
----
