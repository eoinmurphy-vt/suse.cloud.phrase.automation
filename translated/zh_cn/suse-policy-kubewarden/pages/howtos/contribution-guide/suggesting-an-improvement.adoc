== 打开一个问题

要对现有文档内容提出改进建议或报告错误，请打开一个问题。

* 点击顶部的 GitHub Octocat 图标。这会将您重定向到
ifeval::["{build-type}" == "community"]
  https://github.com/kubewarden/docs[ {short-project-name} 的文档库]。
endif::[]
ifeval::["{build-type}" == "product"]
  https://github.com/rancher/kubewarden-product-docs[ {short-project-name} 的文档库]。
endif::[]
* 导航至 "问题 "选项卡并单击 "*新问题*"。
* 描述问题或改进建议。细节越多越好。
* 单击*提交新问题*。
* 提交问题后，您可以将其分配给自己或等待社区成员来处理。文档团队和社区成员在对您的问题采取行动之前，可能会要求您做出澄清。您应积极检查您的问题或https://docs.github.com/en/account-and-profile/managing-subscriptions-and-notifications-on-github/setting-up-notifications/configuring-notifications[打开 GitHub 通知]。

== 新内容建议

如需建议新内容，请使用这些机制之一提交问题：

* 在您认为内容所属的部分中选择一个现有页面，然后单击*创建问题*。或
ifeval::["{build-type}" == "community"]
* 导航至https://github.com/kubewarden/docs/issues/new/choose[] GitHub 并直接提交问题。
endif::[]
ifeval::["{build-type}" == "product"]
* 导航至https://github.com/rancher/kubewarden-product-docs/issues/new/choose[] GitHub 并直接提交问题。
endif::[]

== 如何计算捐款？

贡献不分大小。但是，为了让社区获得最大价值，我们要求您在报告问题时遵循以下规定：

* 重点是清晰描述问题。需要考虑的一些要点是描述哪些内容缺失、过时、错误，或需要在质量和技术上加以改进。
* 描述问题对用户的具体影响。
* 限制问题的范围。如果范围较大，则将其分解为一个问题中的多个小任务。例如，"创建供稿指南 "的范围很广，因为该问题涉及多项任务。
  然而，"修正快速启动页面上的语法错误 "是一个范围更窄的问题，只需要一个拉取请求。
* 交叉检查现有问题，避免重复工作。
* 检查是否存在拉取请求或问题。检查您是否在打开的问题中引用了现有问题或拉动请求，以便为可能想参与其中的贡献者提供上下文。
