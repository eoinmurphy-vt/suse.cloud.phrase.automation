= 使用 Rancher 备份操作员进行备份和还原
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: 在任何 Kubernetes 集群上使用 rancher-backup 操作员配置和管理备份与还原。
:keywords: "{project-name}", kubernetes, kubewarden, rancher 备份操作员, 备份, 恢复
:doc-persona: kubewarden-operator, kubewarden-integrator
:doc-type: 如何
:doc-topic: 操作员手册，牧场主备份操作员
:current-version: {page-origin-branch}

`+rancher-backup+` 操作员可用于在任何 Kubernetes 集群上备份和还原 Rancher。

自`+v9.0.0+` 版起，`+rancher-backup+` 支持{project-name} 。这包括

- 默认的 Rancher 命名空间`+cattle-kubewarden-system+` （或`+cattle-kubewarden-*+` ），以及默认的{short-project-name} 命名空间`+kubewarden+` 。
- {short-project-name} 所需资源通过 Helm 图表安装。
- {short-project-name} 这些 CRD 在{short-project-name} 控制器还原后进行对账。
- `+policy-reporter+` 子图表`+kubewarden-controller+` 图表的默认值。这不包括 Grafana 集成或其他插件。

== 用户秘密

备份过程不包括用户创建的 "秘密"，例如用于xref:/howtos/policy-servers/02-private-registry.adoc#_creating_the_secret[为私有注册表配置策略服务器]的 "秘密"，除非这些 "秘密 "被正确标注。为此，请用以下方式之一为秘密贴上标签。要么是

[source,shell]
----
kubectl label secret secret-ghcr-docker app.kubernetes.io/part-of=kubewarden
----

或者

[source,shell]
----
kubectl label secret secret-ghcr-docker resources.cattle.io/backup=true
----

== 安装 Rancher 备份操作器

请点击链接：https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/back-up-restore-and-disaster-recovery.html[Rancher Manager] 文档。对于使用 Minikube 开箱即支持的名为`+standard+` 、类型为`+hostPath+` 的 PersistentVolumes 安装 Minikube。

需要将值`+optionalResources.kubewarden.enabled+` 设置为`+true+` ，指示备份操作员也备份{short-project-name}用户定义的 CR。

安装过程如下：

[source,console]
----
helm repo add rancher-charts https://charts.rancher.io
helm repo update
helm install --wait --create-namespace -n cattle-resources-system \
    rancher-backup-crd rancher-charts/rancher-backup-crd
helm install --wait -n cattle-resources-system \
    rancher-backup rancher-charts/rancher-backup \
    --set persistence.enabled=true --set persistence.storageClass=standard \
    --set optionalResources.kubewarden.enabled=true
----

== 备份

使用`+rancher-resource-set-full+` 备份{short-project-name} 秘密。
其中包括 Helm 安装时创建的 TLS Secrets。

如果您喜欢使用`+rancher-resource-basic+` ，请记得备份或手动创建所需的 TLS Secrets。

下面以`+rancher-resource-set-full+`.NET 为例，说明如何向默认位置执行未加密备份：

[source,shell]
----
kubectl apply -f - <<EOF
apiVersion: resources.cattle.io/v1
kind: Backup
metadata:
  name: default-location-backup
spec:
  resourceSetName: rancher-resource-set-full
EOF
backup.resources.cattle.io/default-location-backup created
----

`+rancher-backup+` 日志或备份显示备份文件的创建：

[source,shell]
----
kubectl logs -n cattle-resources-system -l app.kubernetes.io/name=rancher-backup -f
...
INFO[2025/06/26 10:07:48] Processing backup default-location-backup
INFO[2025/06/26 10:07:48] For backup CR default-location-backup, filename: default-location-backup-32d64f39-d3c7-4331-9101-8ca493bd9d2e-2025-06-26T10-07-48Z
...
INFO[2025/06/26 10:07:49] Done with backup
----

您还可以通过描述资源来查看其状态：

[source,shell]
----
kubectl get backups
NAME                      LOCATION   TYPE       LATEST-BACKUP                                                                              RESOURCESET                 AGE    STATUS
default-location-backup   PV         One-time   default-location-backup-43f3ccb7-5624-4eed-9c3b-1c15d287080e-2025-06-26T15-53-27Z.tar.gz   rancher-resource-set-full   111s   Completed
----

有关更多link:https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/configuration/examples.html#_backup[备份示例]，请参阅 Rancher 文档。

== 恢复

要从默认位置还原未加密的备份，请在显示备份资源时从 LATEST-BACKUP 列中获取文件名，以创建还原资源：

[source,shell]
----
kubectl apply -f - <<EOF
apiVersion: resources.cattle.io/v1
kind: Restore
metadata:
  name: restore-default
spec:
  backupFilename: default-location-backup-32d64f39-d3c7-4331-9101-8ca493bd9d2e-2025-06-26T10-07-48Z.tar.gz
EOF
restore.resources.cattle.io/restore-default created

kubectl get restores
NAME              BACKUP-SOURCE   BACKUP-FILE                                                                                AGE   STATUS
restore-default   PV              default-location-backup-43f3ccb7-5624-4eed-9c3b-1c15d287080e-2025-06-26T15-53-27Z.tar.gz   6s    Completed
----

有关更多link:https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/configuration/examples.html#_restore[还原示例]，请参阅 Rancher 文档。
