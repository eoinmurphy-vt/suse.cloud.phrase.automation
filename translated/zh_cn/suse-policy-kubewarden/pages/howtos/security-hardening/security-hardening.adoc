== `+kubewarden-defaults+` 舵手图

操作员可通过安装所有{short-project-name} Helm 图表获得安全部署。建议安装`+kubewarden-defaults+` Helm 图表并启用其推荐的策略：

[source,console]
----
helm install --wait -n kubewarden kubewarden-defaults kubewarden/kubewarden-defaults \
  --set recommendedPolicies.enabled=True \
  --set recommendedPolicies.defaultPolicyMode=protect
----

这提供了一个默认 PolicyServer 和默认策略，以保护模式确保{short-project-name} 堆栈不受其他工作负载的影响。

== RBAC

{short-project-name} 中描述了 RBAC 配置。用户可以对xref:/explanations/audit-scanner/audit-scanner.adoc#_permissions_and_serviceaccounts[审计扫描仪]功能所需的权限进行微调，也可以对上下文感知功能的xref:/explanations/context-aware-policies.adoc[每个策略服务器]服务账户进行微调。

查看所有角色：

[source,console]
----
kubectl get clusterroles,roles -A | grep kubewarden
----

== 每个政策的权限

对于上下文感知策略，操作员可在`+spec.contextAwareResources+` 下为每个策略指定细粒度权限，这些权限与为策略运行所在的策略服务器配置的服务账户共同发挥作用。

== 工作量覆盖范围

默认情况下，{short-project-name} 将特定命名空间排除在{short-project-name} 覆盖范围之外。这样做是为了简化首次使用和与其他工作负载的互操作性。

具有安全意识的操作员可以通过`+.global.skipNamespaces+` 值对`+kubewarden-controller+` 和`+kubewarden-defaults+` 舵图的命名空间列表进行调整。

== 舱位安全接纳

从 1.23 版起，{short-project-name} 的堆栈可在已实施https://kubernetes.io/docs/concepts/security/pod-security-standards/#restricted[`+restricted+`` Pod 安全标准]的命名空间中运行，并采用当前的 Pod 加固最佳实践。

为此，您需要在{short-project-name} 部署命名空间中添加`pod-security.kubernetes.io/enforce: restricted`标签。

```console
kubectl label namespace kubewarden pod-security.kubernetes.io/enforce=restricted --overwrite
```

更多详情，请参阅 Kubernetes 的 Pod Security Admissionhttps://kubernetes.io/docs/concepts/security/pod-security-admission/[官方文档]。

=== SecurityContexts

`+kubewarden-controller+` Helm 图表配置 SecurityContexts 并在其`+values.yaml+` 中公开它们。

`+kubewarden-defaults+` Helm 图表允许在`+.Values.policyServer.securityContexts+` 下配置默认策略服务器`+.spec.securityContexts+` 。

对于由操作员管理的策略服务器，您可以通过它们的xref:/reference/CRDs.adoc#_policyserversecurity[`+spec.securityContexts+`].
