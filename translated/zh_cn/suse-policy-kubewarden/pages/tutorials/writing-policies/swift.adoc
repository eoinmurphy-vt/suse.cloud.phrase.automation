正如https://swift.org/[官方网站]所述：

____
Swift 是一种通用编程语言，对新手来说平易近人，对专家来说功能强大。
它快速、现代、安全，是一种写作乐趣。
____

swift 编译器还不支持 WebAssembly，不过https://swiftwasm.org/[SwiftWasm]项目提供了一个具有 WebAssembly 功能的补丁编译器。

SwiftWasm 团队正在努力将这些变更合并到 Swift 项目中。
同时，您可以使用 SwiftWasm 项目提供的工具链来构建{project-name} 策略。

[NOTE]
====

编写或运行 Swift 代码并不需要苹果系统。所有操作都可以在 Linux 或 Windows 机器上完成（使用 Windows 版 Docker）。
====


== 现状

政策作者可以使用以下资源：

* https://github.com/kubewarden/policy-sdk-swift[{short-project-name} Swift SDK]：它提供了一系列``结构``和函数，简化了编写策略的过程。
* https://github.com/kubewarden/swift-policy-template[{short-project-name} Swift 模板项目]：使用此模板构建基于 Swift 的策略。
该模板附带一个工作策略和一套 GitHub 操作，可自动执行其生命周期。

Swift 没有严重的局限性，只有一些小问题：

* 在_"生产 "中_使用策略之前，执行一些构建后优化至关重要_：_
 .. 通过`+wasm-strip+` 剥离 Wasm 模块以缩小其大小
 .. 通过以下方式优化 Wasm 模块 `+wasm-opt+`

模板库提供的 GitHub 操作已经解决了这个问题。
