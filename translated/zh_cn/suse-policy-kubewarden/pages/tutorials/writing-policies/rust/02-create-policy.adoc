例如，您创建了一个简单的验证策略来处理 Pod 创建请求。

该策略会查看 Pod 的`+metadata.name+` 属性，并拒绝名称无效的 Pod。
政策的最终用户应可配置无效名称列表。

政策设置看起来像这样

[subs="+attributes",yaml]
----
invalid_names:
- bad_name1
- bad_name2
----

该政策应允许创建类似下面这样的 Pod：

[subs="+attributes",yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  containers:
    - name: nginx
      image: nginx:latest
----

它应该拒绝创建类似的花苞：

[subs="+attributes",yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: bad_name1
spec:
  containers:
    - name: nginx
      image: nginx:latest
----

== 为新政策项目搭建脚手架

通过`+cargo generate+` 和https://github.com/kubewarden/rust-policy-template[模板]项目，可以创建一个新的策略项目。

首先，安装`+cargo-generate+` 。这需要https://pkgs.org/download/openssl-devel[openssl-devel]。

[subs="+attributes",shell]
----
cargo install cargo-generate
----

现在，请按照以下步骤为项目搭脚手架：

[subs="+attributes",shell]
----
cargo generate --git https://github.com/kubewarden/rust-policy-template \
               --branch main \
               --name demo
----

该命令的输出结果如下

[subs="+attributes",console]
----
🔧   Creating project called `+demo+`...
✨   Done! New project created /<some-path-name>/demo
----

这将在`+demo+` 子目录下创建新的策略项目。

[NOTE]
====

如果计划在演示中使用 GitHub 容器注册表功能，则需要https://docs.github.com/en/packages/working-with-a-github-packages-registry/enabling-improved-container-support-with-the-container-registry#enabling-the-container-registry-for-your-personal-account[启用改进的容器支持]。
====
