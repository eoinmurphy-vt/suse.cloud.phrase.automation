开放政策代理（OPA）是一种通用政策框架，使用 Rego 语言编写政策。

== 导言

雷戈策略的工作原理是接收输入进行评估，并产生输出作为响应。
从这个意义上说，OPA 没有专门针对 Kubernetes 编写策略的工具。

具体来说，OPA 中的策略接收一个 JSON 输入并产生一个 JSON 输出。
OPA 服务器被配置为接收来自 Kubernetes 的入场审查请求。
策略会收到一个 JSON 格式的 Kubernetes`+AdmissionReview+` 对象。
它们必须返回一个有效的`+AdmissionReview+` 对象作为评估结果。

== 与现有政策的兼容性

所有策略均可通过`+opa+` CLI 官方工具编译至`+wasm+` 目标（WebAssembly）。

关于策略执行，您可以阅读{short-project-name} 中实现的xref:tutorials/writing-policies/rego/02-builtin-support.adoc[OPA 内置支持]的更多信息。
