= 创建新的验证策略
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: 创建新的验证策略，通过配置策略设置拒绝 Pod 对象中的无效主机名，并使用提供的工具进行测试。

:doc-type: 教程

本教程将介绍如何创建验证 Pod 对象主机名的策略。

策略是拒绝所有使用拒绝列表中一个或多个主机名的 Pod。
您可以使用运行时设置提供策略配置。

概括地说，策略设置应如下所示：

[source,yaml]
----
denied_hostnames:
  - bad-host
  - forbidden-host
----

该政策拒绝创建该 Pod：

[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  hostname: bad-host
  containers:
    - name: nginx
      image: nginx:latest
----

不过，它接受创建这个花苞：

[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  hostname: allowed-host
  containers:
    - name: nginx
      image: nginx:latest
----

== 为新政策项目搭建脚手架

您可以使用https://github.com/kubewarden/js-policy-template[模板库]创建一个新的策略项目。选择页面顶部的 "使用此模板 "绿色按钮，然后按照 GitHub 的向导操作。
在本地克隆版本库，并更新`+package.json+` 文件，以反映你的策略细节：

[source,json]
----
{
  "name": "your-policy-name",
  "version": "1.0.0",
  "description": "Your policy description",
  "repository": {
    "type": "git",
    "url": "https://github.com/your-username/your-policy-name"
  }
}
----

确保使用与实际 GitHub 仓库相匹配的仓库路径。
