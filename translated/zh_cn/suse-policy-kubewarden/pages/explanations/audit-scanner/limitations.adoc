== 支持的事件类型

策略可以检查`+CREATE+` 、`+UPDATE+` 和`+DELETE+` 事件。

审计扫描器无法模拟`+UPDATE+` 事件，因为它不知道需要更改资源的哪个部分。

因此，审计扫描器会忽略只与`+UPDATE+` 事件相关的策略。

[NOTE]
====

audit-scanner v1.7.0 版本仅支持`+CREATE+` 事件。`+DELETE+` 事件处理即将推出。

====


== 依赖用户和用户组信息的政策

每个 Kubernetes 准入请求对象都有关于发起事件的用户（或服务账户）及其所属组的信息。

审计扫描仪模拟的所有事件都来自同一个硬编码用户和组。正因为如此，依靠这些价值观来决策的政策不会产生有意义的结果。

对于这些情况，请将策略配置为不可审计。

== 依赖外部数据的政策

在进行评估时，政策可以要求并使用外部数据。您可以通过审计检查来评估这些政策，但其结果可能会因外部数据的变化而改变。

== `+*+`

`+AdmissionPolicy+` 和`+ClusterAdmissionPolicy+` 自定义资源都有以下字段：

[subs="+attributes",yaml]
----
spec:
  rules:
    - apiGroups: [""]
      apiVersions: ["v1"]
      resources: ["pods"]
      operations:
        - CREATE
        - UPDATE
----

`+apiGroups+` 、`+apiVersions+` 和`+resources+` 属性可以使用通配符`+*+` 。该通配符号会导致策略匹配字段中使用的所有值。审计扫描仪会忽略使用`+*+` 符号的策略。
