[#_prerequisite]
== 先决条件

您需要至少具有集群成员权限。

[#_brand_configuration]
== 品牌配置

要配置品牌设置，

. 点击*☰ > 全局设置*。
. 点击*品牌*。

[#_private_label_company_name]
=== 私有标签公司名称

此选项将在大多数地方将“Rancher”替换为您提供的值。需要在名称中包含Rancher的文件，例如`+rancher-compose.yml+`，将不会被更改。

[#_support_links]
=== 支持链接

使用网址地址发送新的“提交问题”报告，而不是将用户发送到GitHub问题页面。可选地显示Rancher社区支持链接。

[#_logo]
=== 徽标

上传浅色和深色徽标，以替换顶级导航头中的Rancher徽标。

[#_primary_color]
=== 主色

您可以使用自定义颜色覆盖整个用户界面中使用的主色。

[#_fixed_banners]
=== 固定横幅

在头部、底部或两者中显示自定义固定横幅。

要配置横幅设置，

. 点击*☰ > 全局设置*。
. 点击*横幅*。

[#_custom_navigation_links]
== 自定义导航链接

在本节中，您将学习如何配置*集群仪表板*左侧导航栏中的链接。要访问集群仪表板，

. 在左上角，单击*☰ > 集群管理*。
. 在*集群*页面上，转到您想要自定义导航链接的集群并点击*探索*。

添加链接以快速访问安装在集群上的服务是很有用的。例如，您可以为安装了Istio的集群添加指向Kiali UI的链接，或者为安装了Rancher监控的集群添加指向Grafana UI的链接。

自定义链接不会影响谁可以访问每个服务。

链接可以在顶层创建，并且可以将多个链接组合在一起。

[#_adding_a_custom_navigation_link]
=== 添加自定义导航链接

[NOTE]
.前提条件：
====

您需要至少具有集群成员或项目成员权限。
====


. 单击 *☰ > 集群管理*。
. 在*集群*页面上，转到您想要添加自定义导航链接的集群并点击*探索*。
. 在顶部导航菜单中，点击*🔍 (资源搜索)*。
. 输入*Nav*并点击*导航链接*。
. 点击*从YAML创建*。
. 创建导航链接的最简单方法是添加以下字段：

  name: linkname
  toURL: https://example.com
+
For more details on setting up links, including optional fields, see <<_link_configuration,Link Configuration.>>

. 单击*创建*。

[#_link_configuration]
== 链接配置

[#__name]
=== `+name+`

链接的显示名称。必需。

[#__group]
=== `+group+`

点击时展开的链接组名称。

可选。如果未提供，链接将独立显示。

组与独立链接分开显示，如下所示：

image::grouped-vs-standalone-links.png[群组和独立链接的截图]

[#__iconsrc]
=== `+iconSrc+`

图标源为base64格式。

以下是Grafana徽标的base64格式示例：



[#__annotation]
=== `+annotation+`

NavLink自定义资源的Kubernetes注释。

[#__label]
=== `+label+`

NavLink自定义资源的Kubernetes标签。

[#__sidelabel]
=== `+sideLabel+`

在左侧导航栏中显示的标签

[#__target]
=== `+target+`

设置链接锚标签的目标属性（`+<a>+`），这决定了链接是在新窗口还是现有标签页中打开（取决于浏览器）。

默认值为`+_self+`，在当前标签中打开链接。要在新窗口或标签中打开链接，请将目标设置为`+_blank+`。

有关目标属性的更多信息，请参见https://www.w3schools.com/tags/att_a_target.asp[此页面。]

[#__toservice]
=== `+toService+`

有五个字段构建成如下URL：`+\https://<rancher_server_url>/k8s/clusters/<CLUSTER>/k8s/namespace/<NAMESPACE>/service/<SCHEME>:<NAME>:<PORT>/proxy/<PATH>+`

例如，可以按如下方式设置监控服务的链接：

* 名称：`+rancher-monitoring-grafana+`
* 名称空间：`+cattle-monitoring-system+`
* 路径：`+proxy/?orgId=1+`
* 端口：`+"80"+`
* 协议：`+http+`

必须提供`+toService+`指令或`+toURL+`指令。

[#__tourl]
=== `+toUrl+`

可以是任何链接，甚至是指向集群外部的链接。

必须提供`+toService+`指令或`+toURL+`指令。

[#_link_examples]
== 链接示例

[#_example_of_link_with_tourl]
=== 带有`+toUrl+`的链接示例

此示例 NavLink YAML 显示了如何配置 NavLink 以链接到 Grafana 仪表板的示例：

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  name: grafana
spec:
  group: "Monitoring Dashboards"
  toURL: https://<RANCHER_SERVER_URL>/api/v1/namespaces/cattle-monitoring-system/services/http:rancher-monitoring-grafana:80/proxy/?orgId=1
----

添加上述 YAML 会创建一个指向 Grafana 的链接，如下图所示：

image::example-grafana-link.png[Grafana 链接的截图]

[#_example_of_link_with_toservice]
=== 带有`+toService+`的链接示例

此示例 YAML 展示了`+toService+`作为链接目标的用法：

[,yaml]
----
apiVersion: ui.cattle.io/v1
kind: NavLink
metadata:
  annotations:
    key: annotation
    labels:
      key: label
  name: navlinkname
spec:
  description: This is a description field # Optional.
  group: "group1" # Optional. If not provided, the links appear standalone.
  iconSrc: data:image/jpeg;base64,[icon source string is clipped for brevity]
  label: This is a label # Optional.
  sideLabel: A side label. # Optional.
  target: _blank #Optional. _blank opens the link in a new tab or window.
  toService: # toService or #toUrl needs to be provided.
    name: rancher-monitoring-grafana
    namespace: cattle-monitoring-system
    path: proxy/?orgId=1
    port: "80"
    scheme: http
----

添加上述 `+toService+` 参数会创建一个指向 Grafana 的链接，如下图所示：

image::example-service-link.png[Grafana 链接的截图]
