== 先决条件

* https://kind.sigs.k8s.io/[一种]
* https://helm.sh/[helm]
* https://tilt.dev/[倾斜度]

== 创建本地开发环境

. 克隆https://github.com/rancher/turtles[{product_name}]版本库
. 创建*tilt-settings.yaml*：
+
[source,yaml]
----
{
    "k8s_context": "k3d-rancher-test",
    "default_registry": "ghcr.io/turtles-dev",
    "debug": {
        "turtles": {
            "continue": true,
            "port": 40000,
            "insecure_skip_verify": "true"
        }
    }
}
----
+
. 在{product_name} 版本库的根目录下打开终端
. 运行以下程序
+
[source,bash]
----
make dev-env

# Or if you want to use a custom hostname for Rancher and your ngrok credentials
NGROK_API_KEY=`+key+` NGROK_AUTHTOKEN=`+token+` RANCHER_HOSTNAME=my.customhost.dev make dev-env
----
+
. 如果没有 ngrok 凭据，也没有在前面的步骤中提供，也可以打开一个新终端，在 tilt 启动后再启动 ngrok 或 inlets：
+
[source,bash]
----
kubectl port-forward --namespace cattle-system svc/rancher 10000:443
ngrok http https://localhost:10000
----

== 运行`+make dev-env+` 时会发生什么？

. 将使用以下https://github.com/rancher/turtles/blob/main/scripts/kind-cluster-with-extramounts.yaml[配置]创建https://kind.sigs.k8s.io/[kind]集群。
. 将使用 Helm 安装以下先决组件：
 ** 牧场主经理。
 ** 认证经理。
 ** ngrok Kubernetes 操作员。
. Docker 映像将在本地构建，Helm 将在集群中部署以下控制器：
 ** 核心集群 API 控制器。
 ** RKE2 引导/控制平面提供程序控制器。
 ** Docker 基础架构提供商控制器。
 ** 车队控制器的 CAPI 附加提供程序。
 ** {product_name} 控制器。
 ** {product_name} 集群 API 操作员控制器。
 ** {product_name} ClusterClass 控制器。
 ** {product_name} 第 2 天运行控制员。
. 将使用`+tilt up+` 启动开发环境。
