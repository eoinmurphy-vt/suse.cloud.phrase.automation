= 迁移指南
:revdate: 2025-11-14
:page-revdate: {revdate}

== 背景

从 Rancher v2.13 版开始，{product_name} 将作为 Rancher 的一部分进行安装，无需单独部署。在升级到 Rancher v2.13 之前，用户需要遵循本指南，以确保升级过程顺利进行。

== 迁移

_在_升级到 Rancher v2.13_之前_，请依次按照以下步骤操作：

. 确保将{product_name} 升级到 v0.24.3。这一步很重要，因为 v0.24 的所有补丁发布都是为群集升级到 Rancher v2.13 做准备。
. 卸载{product_name} 。这是必要的，因为从 Rancher v2.13 开始，{product_name} 不再是一个选择插件，而是一个 Rancher 系统组件。因此，Rancher 会将其安装在与之前不同的命名空间中，因此需要先手动卸载之前安装的版本。卸载{product_name} 的 v0.24.3 版后，核心 CAPI 提供程序本身将被删除，但核心 CAPI CRD 仍会保留。在迁移期间，核心 CAPI 控制器将不会在管理群集上运行，但任何下游 CAPI 群集都应照常运行，不受影响。
. 运行以下 kubectl 命令：
+
[source,bash]
----
kubectl patch crd capiproviders.turtles-capi.cattle.io --type=json -p='[{"op": "add", "path": "/metadata/annotations/meta.helm.sh~1release-namespace", "value": "cattle-turtles-system"}]'
kubectl patch crd clusterctlconfigs.turtles-capi.cattle.io --type=json -p='[{"op": "add", "path": "/metadata/annotations/meta.helm.sh~1release-namespace", "value": "cattle-turtles-system"}]'
----

这些命令将修补`+capiprovider+` 和`+clusterctlconfig+` CRD，使其在作为 Rancher v2.13 的一部分安装时被{product_name} 采用。

请注意，运行{product_name} &le; v0.24.3 版本的用户必须进行上述迁移。安装 Rancher v2.13 的新用户无需执行任何迁移。
