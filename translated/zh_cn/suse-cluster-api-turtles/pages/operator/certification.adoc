= 提供商认证 
:revdate: 2025-06-30
:page-revdate: {revdate}

== 什么是认证提供方？

由于大多数 CAPI 提供商都是由开放社区维护的上游项目，因此没有安全的方法来保证使用特定提供商配置的任何集群都能通过 Turtles 导入 Rancher。不过，我们确实对那些提供者实施了认证程序：

* 作为我们 E2E 测试套件的一部分进行了积极测试。
* 不断更新 E2E 测试套件，以验证提供商的最新版本。

== 认证您的定制供应商

此外，如果您是提供商开发人员或只是想使用未被列为认证的其他提供商，您有两种选择：

* 重新使用现有的 Turtles E2E 套件，开始申请认证状态。更多详细信息，请访问 "xref:./certification.adoc#_provider_certification_using_turtles_test_suite[使用 Turtles 测试套件的提供商认证]"部分。
* 使用自己的集成套件对提供商进行认证。更多详细信息，请点击此处的 "xref:./certification.adoc#_certify_provider_with_custom_integration_suite[使用自定义集成套件认证提供商]"部分。

[discrete]
==== *我为什么要验证我的提供商？*

CAPI 提供商的数量不断增加，社区也为不同的基础设施主机提出了新的项目。这种开放性有助于丰富 CAPI 生态系统，但无法控制和测试 Turtles 将在整个 CAPI 项目中合作的所有供应商。海龟的开发人员将维护一份通过 CI 机制积极测试和验证的xref:../overview/certified.adoc[认证提供商名单]，但该名单将仅限于主要的知名提供商。因此，我们鼓励未列出的提供商的用户重用海龟测试套件，并验证提供商与海龟之间的集成。

[discrete]
==== *我可以与未经认证的提供商一起使用海龟吗？*

Turtles 是一个旨在与整个 CAPI 生态系统保持一致和集成的项目，您可以在不验证任何提供商的情况下自由使用它。你可能会发现大多数上游项目都没有问题。

[discrete]
==== *认证和支持有什么区别？*

由于 CAPI 提供商是由 Kubernetes 社区维护的项目，我们无法保证对特定提供商的支持。因此，我们选择将重点放在对xref:../overview/certified.adoc[认证提供商的]持续验证上，并为用户提供与现有测试套件集成的可能性，以验证他们所选择的提供商。

== 使用 Turtles 测试套件进行提供商认证

认证过程以核实{product_name} 与 CAPI 提供商的整合情况为基础。为了简化这项工作，我们准备了一个通用测试，用于验证下游 CAPI 集群的配置和导入。

[TIP]
====
我们建议您参考https://github.com/rancher-sandbox/turtles-integration-suite-example[示例]，了解如何使用 Turtles 测试套件。
====

=== 测试和认证您的提供商

要验证您的提供商是否与 Turtles 兼容，以及您是否可以提供 CAPI 集群并通过 Turtles 将其导入 Rancher，第一步就是与我们的测试套件集成。我们提供了一个包含https://github.com/rancher-sandbox/turtles-integration-suite-example[集成示例]的资源库，您可以将其作为集成的参考。

Turtles 作为一个项目包含https://github.com/rancher/turtles/tree/main/test/e2e/suites[数量众多的套件]，用于验证不同的功能和流程，但对于提供商认证，我们只要求您运行一个使用 GitOps 流程的测试。Turtles 是一个能与 GitOps 集群配置方法很好集成的项目，这也是我们验证提供商与 Rancher 集成的主要方式。为特定的 CAPI 提供商运行全套程序将：

* 在所需环境中创建管理群集。
* 安装 Rancher 和 Turtles 以及所有先决条件。
* 安装 Gitea。
* 运行该套件，创建 git repo，使用 Fleet 应用集群模板，并验证集群是否已创建并成功导入 Rancher。

==== 测试配置

要成功运行测试套件，您必须提供一些环境变量。其中有些是不可知的，这意味着您要测试的任何提供商都需要这些内容，但有些则是您要验证的提供商所特有的。请注意所测试的提供商的特定规格，如凭据、端点等。

[TIP]
====
接下来，我们建议您阅读xref:../operator/certificationsuite.adoc[测试套件指南]
====

== 利用定制集成套件认证提供商

要使用自定义集成套件对提供商进行认证，请确保其满足以下关键要求：

=== 要求

* 必须使用`+CAPIProvider+` API 部署提供程序。
* 使用提供程序创建的集群必须通过 Turtles 成功导入 Rancher。

=== 认证程序

1. **开发集成套件**
   * 安装并配置 Rancher 和 Turtles。
   * 部署 CAPI 提供程序。
   * 配置集群并将其导入 Rancher。

2. **运行和验证测试**
   * 确保成功创建和导入群集。

3. **提交认证申请**
   * 提供测试结果、日志和步骤。
   * 使用以下https://github.com/rancher/turtles/issues/new?template=certification_request.yaml[模板]提交 GitHub 问题。

== 认证申请

与 Turtles 的测试套件集成，并对您感兴趣的提供商进行检查，就足以验证其是否与{product_name} 兼容。由于我们不可能持续测试每个 CAPI 提供商，因此该认证工作流程将使您作为用户能够验证预期功能。但是，由于我们并不积极测试该提供商的更新迭代（与海龟的更新版本），因此对给定提供商的支持和保证是有限的，您将负责验证未来的版本。

如果在对您的提供商成功运行检查后，您希望将其添加到xref:../overview/certified.adoc[认证提供商]列表中并纳入项目的定期 E2E 套件，您可以使用https://github.com/rancher/turtles/issues/new?template=certification_request.yaml[认证]申请提交 GitHub 问题，申请认证。社区将对提案进行审查，并确定将提供方加入认证矩阵的可行性。
