= 建筑组件
:revdate: 2025-09-08
:page-revdate: {revdate}

提供xref:rancher-os-management/architecture/services/architecture-services.adoc[{elemental-product-name} 服务]所需的组件包括

* 命令行工具 <<_elemental_command_line_tool,`+elemental+`>>命令行工具
* 守护进程 <<_elemental_operator_daemon,`+elemental-operator+`>>守护进程
* 命令行工具 <<_elemental_register_command_line_tool,`+elemental-register+`>>命令行工具
* 守护进程 <<_elemental_system_agent_daemon,`+elemental-system-agent+`>>守护进程
* 一个或多个<<_suse_rancher_prime_os_manager_os_container_image,{elemental-product-name} OS 容器映像>>

== `+elemental+` 命令行工具

`+elemental+` 工具是link:{elemental_toolkit_url}[{elemental_toolkit_name}]项目的一部分。
它在主机上执行实际的操作系统安装和升级操作，并用于执行xref:rancher-os-management/architecture/custom-resources.adoc[{elemental-product-name} CRD] 中添加的xref:references/cloud-config-reference.adoc[云配置]指令。

`+elemental+` 二进制文件包含在随{elemental-product-name} 一起分发的所有基本操作系统镜像中。

== `+elemental-operator+` 守护进程

*元素操作器*守护进程主要执行两项任务：

. 嵌入{elemental-product-name} Kubernetes 控制器，以管理所有xref:rancher-os-management/architecture/custom-resources.adoc[{elemental-product-name} CRD]
. 暴露_注册端点_，允许主机在xref:rancher-os-management/architecture/services/architecture-machineonboarding.adoc[上机]过程中注册并下载操作系统安装配置

`+elemental-operator+` 守护进程通过xref:references/operatorchart-reference.adoc[{elemental-product-name} Operator Helm Chart] 作为`+Deployment+` 部署在 Rancher 集群上。

== `+elemental-register+` 命令行工具

*元素注册（elemental-register*）二进制文件是客户端，用于根据<<_elemental_operator_daemon,元素操作器>>开放的_注册端点_注册主机。它收集并转发主机数据，以便<<_elemental_operator_daemon,元素操作员>>填写 SMBIOS 和硬件标签模板。

如果注册阶段执行成功，*元素注册器*就会从*元素操作器*获取存储在xref:references/machineregistration-reference.adoc[机器注册]中的完整配置。
最后一步，*elemental-register*客户端会调用<<_elemental_command_line_tool,元素>>二进制文件，通过获取的配置启动操作系统安装。

== `+elemental-system-agent+` 守护进程

`+elemental-system-agent+` 由https://github.com/rancher/system-agent[System Agent 项目] Rancher构建，允许{elemental-product-name} 部署_计划_以协助主机调配。
值得注意的是，xref:rancher-os-management/architecture/services/architecture-clusterdeployment.adoc[Kubernetes 集群供应服务]所需的`+rancher-system-agent+` 安装和配置是通过`+elemental-system-agent+` _计划_执行的。

== {elemental-product-name} 操作系统容器映像

操作系统镜像link:{elemental_toolkit_url}[{elemental_toolkit_name}]操作系统映像是一个 OCI 容器映像，其中包含构成目标主机操作系统的所有文件。它不仅包含所有二进制文件和库，还包含 Linux 系统所需的内核和启动文件。

{elemental-product-name} 操作系统镜像是一个基于 SLE Microlink:{elemental_toolkit_url}[{elemental_toolkit_name}]操作系统镜像，它基于https://www.suse.com/products/micro/[Micro] SLE，包含特定的{elemental-product-name} 配置和二进制文件（<<_elemental_command_line_tool,元素>>和<<_elemental_register_command_line_tool,元素寄存器文件>>）。

{elemental-product-name} OS 映像在xref:references/managedosversion-reference.adoc[ManagedOSVersions]资源中进行跟踪。xref:references/managedosversion-reference.adoc[ManagedOSVersions]资源是从xref:references/managedosversionchannel-reference.adoc[ManagedOSVersionChannel]资源动态创建的。每次安装{elemental-product-name} Operator 时都会部署一个默认的`+ManagedOSVersionChannel+` 资源。
