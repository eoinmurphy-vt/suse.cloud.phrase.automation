= 代理背后
:revdate: 2025-11-06
:page-revdate: {revdate}

:tabs-sync-option:

在许多企业环境中，在企业内部运行的服务器或虚拟机无法直接访问互联网。出于安全考虑，与外部服务的连接是通过 HTTP(S) 代理完成的。本教程将向您展示如何在这样的环境中设置{elemental-product-name} 部署。

[IMPORTANT]
====
本指南不包括代理服务器后面的 Rancher 安装。这是一种不同的用例，您可以在https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/http-proxy/http-proxy.html[在 HTTP 代理服务器后面安装 SUSE Rancher Prime]页面找到详细文档。
====


[NOTE]
====
在本文档中，我们假设您使用的是 SUSE 系列系统（如 SLE Micro），因此代理设置必须写入`+/etc/sysconfig/proxy+` 。
====


必须在以下位置配置代理设置：

* 机器注册终端
* 种子图像资源
* {elemental-product-name} 集群配置

`+elemental-system-agent+` 需要代理设置才能访问 Rancher 管理器。
为此，您需要填写机器注册端点的 cloud-init 部分。

您可以使用xref:quickstarts/quickstart-ui.adoc#_add_a_machine_registration_endpoint[用户界面]或xref:quickstarts/quickstart-cli.adoc#_prepare_your_kubernetes_resources[CLI] 进行操作。

== {elemental-product-name}-注册

xref:rancher-os-management/architecture/architecture-components.adoc#_elemental_register_command_line_tool[{elemental-product-name}-注册]是新主机与 Rancher Manager 之间的第一个通信端点，这里是需要设置代理设置的第一个地方。

[CAUTION]
====
在撰写本文时，只能通过 CLI 为 ISO 配置代理设置。用户界面中没有代理设置。
====

当您首次启动{elemental-product-name} ISO 时，为了配置代理设置，您必须在 ISO 中包含`+cloud-init+` 定义。
为此，您必须创建一个xref:references/seedimage-reference.adoc[SeedImage]定义。

[,yaml]
----
include::example$proxy/seedimage-proxy.yaml[]
----

使用`+kubectl+` 应用 YAML，然后打印 SeedImage 定义，获取下载 URL：

[,bash]
----
kubectl apply -f <my_seedimage_yaml_file>
kubectl get seedimage <seed_image_name> -n <namespace> -o yaml
----

启动 ISO，新系统就会出现在xref:references/machineinventory-reference.adoc[机器清单]中。
