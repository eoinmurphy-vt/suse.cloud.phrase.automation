= ManagedOSVersionChannel 引用
:revdate: 2025-09-08
:page-revdate: {revdate}

`+ManagedOSVersionChannel+` 资源负责定义操作系统图像通道。

`+ManagedOSVersionChannel+` 资源规范下可配置多个密钥。

.managedosversionchannel-example.yaml
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: ManagedOSVersionChannel
metadata:
  name: elemental-channel
  namespace: fleet-default
spec:
  options:
    image: registry.suse.com/rancher/elemental-channel:1.4.2
  syncInterval: 1h
  type: custom
----

== `+ManagedOSVersionChannelSpec+` 参照

|===
| 钥匙 | 类型 | 默认值 | 说明

| 选项
| 反对
| 无效
| 定义操作系统通道中可添加的可选信息

| syncInterval
| 字符串
| 1h
| 定义何时同步操作系统通道

| 类型
| 字符串
| 空的
| 定义通道类型，目前只支持`+custom+` 

| upgradeContainer
| 反对
| 无效
| 可定义的升级容器。参见<<_upgradecontainer,参考资料>>

| deleteNoLongerInSyncVersions
| bool
| 错误
| 自动删除通道中不再包含的过时操作系统版本

| 启用
| bool
| true
| 启用此通道。允许同步操作系统版本
|===

== `+upgradeContainer+`

这样就可以根据此 ManagedOSVersion 覆盖系统升级控制器计划（请参阅xref:rancher-os-management/upgrade-lifecycle.adoc#_components[升级组件]）的默认`+upgrade+` 字段。
系统升级控制器会将这些密钥转化为 Kuberneteshttps://kubernetes.io/docs/reference/kubernetes-api/workload-resources/pod-v1/#Container[容器]规范。
这是负责运行操作系统升级的容器。
