== 为 Rancher 安装 rancher-backup 操作员

请遵循https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/back-up.html[备份指南] Rancher，了解如何安装和配置 Rancher 备份操作器。


== 使用 rancher-backup 操作员备份{elemental-product-name} 

创建`+backup object+` （根据需要调整），以备份在 Kubernetes 集群上运行的 Rancher。

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Backup
metadata:
  name: rancher-backup
spec:
  resourceSetName: rancher-resource-set
  schedule: "10 3 * * *"
  retentionCount: 10
----

rancher 备份操作员可提供多种计划、加密和存储类别选项。
您可以阅读https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/configuration/backup.html[官方文档]，了解所有选项。

检查来自 rancher-backup 操作员的日志。

[,shell]
----
kubectl logs -n cattle-resources-system -l app.kubernetes.io/name=rancher-backup -f
----

验证是否在持久卷上创建了备份文件。
