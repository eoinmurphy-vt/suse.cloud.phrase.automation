= 故障排除升级
:revdate: 2025-09-08
:page-revdate: {revdate}

有关升级生命周期和组件的高级概述，请参阅xref:rancher-os-management/upgrade-lifecycle.adoc[相关文件]。

== 牧场主侧

在本例中，我们使用以下 ManagedOSImage 定义升级了群集节点：

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: ManagedOSImage
metadata:
  name: my-upgrade
  namespace: fleet-default
spec:
  # Set to the new {elemental-product-name} version you would like to upgrade to or track the latest tag
  osImage: "registry.suse.com/rancher/sle-micro/5.5:latest"
  clusterTargets:
    - clusterName: my-cluster
----

`+ManagedOSImage+` 应用后，`+elemental-operator+` 将对其进行验证，并生成相关的`+Bundle+` 。
`+Bundle+` 名称的前缀是`+mos+` ，然后是`+ManagedOSImage+` 名称。在这种情况下`+mos-my-upgrade+` 。

在`+Bundle+` 的定义中，您可以找到有关升级计划和预期目标的详细信息。
例如

[,shell]
----

kubectl -n fleet-default get bundle mos-my-upgrade -o yaml
----

== {elemental-product-name} 簇边

任何正确注册并属于目标群集的{elemental-product-name} 节点都会获取捆绑包并开始应用。 +
该操作由{elemental-product-name} 集群上运行的 Rancher's`+system-upgrade-controller+` 执行。 +
要监控控制器的正常运行，可以读取其日志：

[,shell]
----
kubectl -n cattle-system logs deployment/system-upgrade-controller
----

如果一切正常，`+system-upgrade-controller+` 将在群集上创建升级计划：

[,shell]
----
kubectl -n cattle-system get plans
----

对于每个 "计划"，控制器将协调在每个目标节点上应用该计划的作业。 +
任务名称将使用计划名称 (`+os-upgrader-my-upgrade+`) 和目标计算机主机名称 (`+my-host+`) ，以便于发现。 +
例如`+apply-os-upgrader-my-upgrade-on-my-host-7a25e+` +
您可以使用

[,shell]
----
kubectl -n cattle-system get jobs
----

每个作业都将使用`+privileged: true+` 容器，其中包含`+ManagedOSImage+` 定义中指定的 SLE Micro 映像。该容器将尝试升级系统并执行重启。

如果任务失败，可以通过检查日志来查看其状态：

[,shell]
----
kubectl -n cattle-system logs job.batch/apply-os-upgrader-my-upgrade-on-my-host-7a25e
----

[IMPORTANT]
.两个阶段的工作流程
====

请注意，升级过程分两个阶段进行。 +
您会发现，同一个任务运行了两次，第一次以`+Unknown+` Status 结束，无法完成。 +
*这是意料之中的*事，因为{elemental-product-name} 依赖于在机器重启后再次运行作业，以便验证新版本是否正确安装。 +
您会发现工作第二次运行，这次正确完成。

[,shell]
----
kubectl -n cattle-system get jobs
NAMESPACE       NAME                                            COMPLETIONS   DURATION   AGE
cattle-system   apply-os-upgrader-my-upgrade-on-my-host-0b392   1/1           2m34s      6m23s
cattle-system   apply-os-upgrader-my-upgrade-on-my-host-7a25e   0/1           6m23s      6m23s

----

[,shell]
----
kubectl -n cattle-system get pods
NAME                                            READY   STATUS      RESTARTS      AGE
apply-os-upgrader-my-upgrade-on-my-host-zbkrh   0/1     Completed   0             9m40s
apply-os-upgrader-my-upgrade-on-my-host-zvrff   0/1     Unknown     0             12m
----
====

== 从失败中恢复

ManagedOSImage 升级过程有可能失败，导致一个或多个节点处于故障状态。例如，如果要升级的映像在注册表中找不到，或因其他原因损坏，在下游群集上运行的升级任务将不会成功。

在这种情况下，运行失败升级任务的节点将保持封锁。您可以使用正常运行的`+osImage+` 更新 ManagedOSImage，或者删除它以阻止任何进一步的升级尝试。无论如何，为了恢复这些节点并安排后续升级，受影响的节点需要手动解绑。
