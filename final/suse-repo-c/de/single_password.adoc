= Einzelnes Passwort
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: {stackstate-product-name} Selbst gehostet

{stackstate-product-name} wird standardmäßig mit einem einzigen Passwort konfiguriert, das xref:/k8s-suse-rancher-prime.adoc#_installation[während der Installation festgelegt wird]. Dadurch werden die Benutzer mit einem einzigen Kennwort authentifiziert, das bei der Installation angegeben werden muss. 

Dies ist der einfachste Weg, um mit {stackstate-product-name} zu beginnen, aber für die Produktion wird eine der anderen xref:/setup/security/authentication/authentication_options.adoc[Authentifizierungsoptionen] empfohlen.

== Das Passwort von einem externen Geheimnis abrufen

Wenn das Standardkennwort von einem externen Geheimnis stammen soll, befolgen Sie xref:/setup/security/external-secrets.adoc#_getting_authentication_data_from_an_external_secret[diese Schritte], geben aber die folgenden Daten ein:

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
  default_password: <base64 of the encoded password>
----

Die Kodierung des Passworts kann eine der folgenden sein:* Für ein Klartext-Passwort geben Sie an: `$plain$your-password` (`$plain$` ist ein Präfix, das Klartext anzeigt) * Für ein verschlüsseltes Passwort geben Sie an: `bcrypt-hashed-password`. Die Erstellung eines verschlüsselten Hashes des Passworts kann mit `htpasswd -bnBC 10 "" <password> | tr -d ':\n'` durchgeführt werden.
