[IMPORTANT]
====
Bei der benutzerdefinierten Integration handelt es sich um eine noch nicht freigegebene Vorproduktionsfunktion, so dass sich im Laufe der Entwicklung der Funktion noch Änderungen ergeben können. Es ist nicht für den Einsatz in der Produktion bestimmt. Sie können jedoch in Nicht-Produktionsumgebungen getestet werden.
====

== Übersicht

Benutzerdefinierte Integrationen nutzen Open Telemetry, um Telemetriedaten (Metriken, Traces, Logs) aus verschiedenen Quellen zu sammeln und an {stackstate-product-name} zu senden. Mithilfe von Open Telemetry können Benutzer ihre Anwendungen und Dienste instrumentieren, um Einblicke in die Leistung, Zuverlässigkeit und den Betriebszustand zu erhalten.
Auf der Seite {stackstate-product-name} verarbeitet xref:setup/custom-integrations/otelmappings/README.adoc[Open Telemetry Mappings] die eingehenden Telemetriedaten, wendet die notwendigen Transformationen an und ordnet sie den entsprechenden Entitäten auf {stackstate-product-name} zu. xref:setup/custom-integrations/monitors/index.adoc[Monitore], xref:setup/custom-integrations/metric-bindings/index.adoc[Metrik-Bindungen] und xref:setup/custom-integrations/ui-presentation/README.adoc[UI-Präsentation] können definiert werden, um die aufgenommenen Telemetriedaten in der UI darzustellen und Warnmeldungen und Beobachtungsmöglichkeiten hinzuzufügen. Alles wird zur Verteilung in einen xref:setup/custom-integrations/develop.adoc[Stapel] eingepackt.

== Einrichtung der Entwicklung

Benutzerdefinierte Integrationen sollten mit einer Vorproduktionsversion der Plattform {stackstate-product-name} entwickelt werden. Es gibt zwei Möglichkeiten, eine Vorproduktionseinrichtung zu erhalten, um mit der Entwicklung Ihrer individuellen Integration zu beginnen:

=== SaaS-Instanz (empfohlen):
Internen SUSE-Teams empfehlen wir, eine SaaS-Instanz beim Team von {stackstate-product-name} anzufordern. Diese Instanzen verfügen über aktivierte benutzerdefinierte Integrationen und werden mindestens mit jeder neuen Version automatisch aktualisiert, in einigen Fällen erhalten sie Vorabversionen mit wichtigen Fehlerbehebungen oder Erweiterungen experimenteller Funktionen.
Kontaktieren Sie uns über den entsprechenden Kanal (wird noch bekannt gegeben).

=== Selbstgehostet:
Sie können benutzerdefinierte Integrationen in Ihrer eigenen, selbst gehosteten {stackstate-product-name} -Installation aktivieren, indem Sie bei der Installation über helm einige Funktionsflags setzen:

    . Erzeugen Sie die `custom_integrations_values.yaml`. In diesem Leitfaden wird davon ausgegangen, dass Sie die Datei im Verzeichnis `$VALUES_DIR`

+
.$VALUES_DIR/custom_integrations_values.yaml
[,yaml]
----
global:
  features:
    experimentalStackpacks: true
----
+

    . Verteilen Sie die {stackstate-product-name} Steuerkarte mit den generierten Werten sowie die `.$VALUES_DIR/custom_integrations_values.yaml`

+
.helm_deploy.sh
[,text]
----
helm upgrade --install \
    --namespace suse-observability \
    --create-namespace \
    --values $VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml \
    --values $VALUES_DIR/suse-observability-values/templates/sizing_values.yaml \
    --values $VALUES_DIR/suse-observability-values/templates/affinity_values.yaml \
    --values $VALUES_DIR/custom_integrations_values.yaml \
    suse-observability \
    suse-observability/suse-observability
----
+
