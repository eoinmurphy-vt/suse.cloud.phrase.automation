= Secretos Cifrado Config
:page-languages: [en, ja, ko, zh]
:revdate: 2026-03-03
:page-revdate: {revdate}

K3s admite la activación del cifrado de secretos en reposo. Cuando se inicia el servidor por primera vez, al pasar la bandera `--secrets-encryption` se hará lo siguiente automáticamente:

* Generar una clave AES-CBC
* Generar un archivo de configuración de cifrado con la clave generada
* Pasar la configuración a la KubeAPI como encryption-provider-config

[TIP]
====
El cifrado de secretos no puede activarse en un servidor existente sin reiniciarlo. +
Utilice `curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s sh -s - server --secrets-encryption` si instala desde script, u otros métodos descritos en xref:installation/configuration.adoc#_configuration_with_install_script[Opciones de configuración].
====

== Elección del proveedor de cifrado

[IMPORTANT]
.Versión Puerta
====
Disponible a partir de las versiones de abril de 2025: v1.30.12+k3s1, v1.31.8+k3s1, v1.32.4+k3s1, v1.33.0+k3s1.
====

Usando la bandera `--secrets-encryption-provider`, puedes elegir entre los siguientes proveedores de encriptación soportados por K3s:

* `aescbc`: AES-CBC con relleno PKCS#7. Es el proveedor por defecto.
* `secretbox`: XSalsa20 and Poly1305

### Migración de proveedores

Puede migrar del proveedor `aescbc` al proveedor `secretbox` siguiendo estos pasos:

. Asegúrese de que el proveedor `secretbox` es compatible con su versión de K3s.
. Actualice/Añada la bandera `secrets-encryption-provider` en su archivo de configuración K3s a `secretbox`.
. Rote las claves de encriptación, siguiendo la sección xref:cli/secrets-encrypt.adoc#_encryption_key_rotation[Rotación de claves de encriptación] más abajo.

== Archivo de configuración de cifrado generado

Al iniciar el servidor con `--secrets-encryption`, K3s generará un archivo de configuración de cifrado en `/var/lib/rancher/k3s/server/cred/encryption-config.json`.

A continuación se muestra un ejemplo del archivo de configuración de cifrado generado con el proveedor predeterminado `aescbc`:

[,json]
----
{
  "kind": "EncryptionConfiguration",
  "apiVersion": "apiserver.config.k8s.io/v1",
  "resources": [
    {
      "resources": [
        "secrets"
      ],
      "providers": [
        {
          "aescbc": {
            "keys": [
              {
                "name": "aescbckey",
                "secret": "xxxxxxxxxxxxxxxxxxx"
              }
            ]
          }
        },
        {
          "identity": {}
        }
      ]
    }
  ]
}
----

== Herramienta de cifrado de secretos

K3s contiene una herramienta de utilidad `secrets-encrypt`, que permite el control automático de lo siguiente:

* Desactivar/activar la encriptación de secretos
* Añadir nuevas claves de cifrado
* Rotación y eliminación de claves de cifrado
* Reencriptar secretos

Para más información, consulte la xref:cli/secrets-encrypt.adoc[documentación del comando`k3s secrets-encrypt` ].
