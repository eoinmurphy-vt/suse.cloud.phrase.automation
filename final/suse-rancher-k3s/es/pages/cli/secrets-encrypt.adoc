= k3s secrets-encrypt
:page-languages: [en, ja, ko, zh]
:revdate: 2026-01-05
:page-revdate: {revdate}

K3s admite la activación del cifrado de secretos en reposo. Para más información, consulte xref:security/secrets-encryption.adoc[Cifrado de secretos].

== Herramienta de cifrado de secretos

[IMPORTANT]
.Versión Puerta
====
Disponible a partir de https://github.com/k3s-io/k3s/releases/tag/v1.21.8%2Bk3s1[.21.8+k3s1] v1
====


K3s contiene una herramienta CLI `secrets-encrypt`, que permite el control automático de lo siguiente:

* Desactivar/activar la encriptación de secretos
* Añadir nuevas claves de cifrado
* Rotación y eliminación de claves de cifrado
* Reencriptar secretos

[CAUTION]
====
Si no se sigue el procedimiento adecuado para rotar las claves de cifrado, el clúster puede quedar dañado permanentemente. Proceda con precaución.
====

[#_encryption_key_rotation]
=== Rotación de claves de cifrado

[IMPORTANT]
.Versión Puerta
====
Disponible a partir de las versiones de septiembre de 2024: v1.30.5+k3s1, v1.31.1+k3s1.
====

[tabs,sync-group-id=se]
======
Servidor único::
+
--
Para rotar claves de encriptación secretas en un cluster de un solo servidor:

. Inicie el servidor K3s con la bandera `--secrets-encryption`
+
[NOTE]
====
Actualmente _no_ es posible iniciar K3s sin cifrado y activarlo posteriormente.
====

. Rotación de claves secretas de cifrado
+
----
k3s secrets-encrypt rotate-keys
----

. Espere a que termine la recodificación. Mira los registros del servidor, o espera:
+
[,bash]
----
$ k3s secrets-encrypt status
Encryption Status: Enabled
Current Rotation Stage: reencrypt_finished
----
--

Alta disponibilidad::
+
--
Para rotar claves de encriptación secretas en configuraciones HA:

. Inicie los tres servidores K3s con la bandera `--secrets-encryption`. Para abreviar, nos referiremos a los servidores como S1, S2, S3.
+
[NOTE]
====
Actualmente _no_ es posible iniciar K3s sin cifrado y activarlo posteriormente.
====

. Rotar claves secretas de encriptación en S1
+
[,bash]
----
k3s secrets-encrypt rotate-keys
----

. Espere a que termine la recodificación. Mira los registros del servidor, o espera:
+
[,bash]
----
$ k3s secrets-encrypt status
Encryption Status: Enabled
Current Rotation Stage: reencrypt_finished
----
+
[NOTE]
====
Los K3 volverán a cifrar ~5 secretos por segundo. Las agrupaciones con un gran número de secretos pueden tardar varios minutos en volver a cifrarse. Puedes seguir el progreso en los registros del servidor.
====

. Reinicie K3s en S1 con los mismos argumentos. Si ejecuta K3s como servicio:
+
[,bash]
----
# If using systemd
systemctl restart k3s
# If using openrc
rc-service k3s restart
----

. Una vez que S1 está arriba, reinicie K3s en S2 y S3
--
======

=== Secretos Cifrado Desactivar/Reactivar

[tabs,sync-group-id=se]
======
Servidor único::
+
--
Después de lanzar un servidor con la bandera `--secrets-encryption`, se puede desactivar la encriptación de secretos.

Para desactivar el cifrado de secretos en un clúster de un solo nodo:

. Desactivar
+
[,bash]
----
k3s secrets-encrypt disable
----

. Kill y reiniciar el servidor K3s con los mismos argumentos. Si ejecuta K3s como servicio:
+
[,bash]
----
# If using systemd
systemctl restart k3s
# If using openrc
rc-service k3s restart
----

. Reencriptar con banderas
+
[,bash]
----
k3s secrets-encrypt reencrypt --force --skip
----

Para volver a activar el cifrado de secretos en un clúster de un solo nodo:

. Activar
+
[,bash]
----
k3s secrets-encrypt enable
----

. Apagar y reiniciar el servidor K3s con los mismos argumentos
. Reencriptar con banderas
+
[,bash]
----
k3s secrets-encrypt reencrypt --force --skip
----
--

Alta disponibilidad::
+
--
Tras lanzar un clúster de HA con banderas `--secrets-encryption`, se puede desactivar el cifrado de secretos.

[NOTE]
====
Aunque no es obligatorio, se recomienda elegir un nodo servidor desde el que ejecutar los comandos de `secrets-encrypt`.
====

En aras de la brevedad, los tres servidores utilizados en esta guía se denominarán S1, S2, S3.

Para desactivar el cifrado de secretos en un clúster de HA:

. Desactivar en S1
+
[,bash]
----
k3s secrets-encrypt disable
----

. Matar y reiniciar S1 con los mismos argumentos. Si ejecuta K3s como servicio:
+
[,bash]
----
# If using systemd
systemctl restart k3s
# If using openrc
rc-service k3s restart
----

. Una vez que S1 está arriba, matar y reiniciar el S2 y S3
. Reencriptar con banderas en S1
+
[,bash]
----
k3s secrets-encrypt reencrypt --force --skip
----

Para volver a activar el cifrado de secretos en un clúster de HA:

. Activar en S1
+
[,bash]
----
k3s secrets-encrypt enable
----

. Matar y reiniciar S1 con los mismos argumentos
. Una vez que S1 está arriba, matar y reiniciar el S2 y S3
. Reencriptar con banderas en S1
+
[,bash]
----
k3s secrets-encrypt reencrypt --force --skip
----
--

=== Secretos Estado de cifrado

La herramienta secrets-encrypt incluye un comando `status` que muestra información sobre el estado actual de la encriptación de secretos en el nodo.

Un ejemplo del comando en un nodo de un solo servidor:

[,bash]
----
$ k3s secrets-encrypt status
Encryption Status: Enabled
Current Rotation Stage: start
Server Encryption Hashes: All hashes match

Active  Key Type  Name
------  --------  ----
 *      AES-CBC   aescbckey
----

Otro ejemplo en cluster HA, después de rotar las claves, pero antes de reiniciar los servidores:

[,bash]
----
$ k3s secrets-encrypt status
Encryption Status: Enabled
Current Rotation Stage: rotate
Server Encryption Hashes: hash does not match between node-1 and node-2

Active  Key Type  Name
------  --------  ----
 *      AES-CBC   aescbckey-2021-12-10T22:54:38Z
        AES-CBC   aescbckey
----

Los detalles de cada sección son los siguientes:

* *Estado de encriptación*: Muestra si la encriptación de secretos está desactivada o activada en el nodo.
* *Etapa de rotación actual*: Indica la fase de rotación actual del nodo. +
Las etapas son: `start`, `prepare`, `rotate`, `reencrypt_request`, `reencrypt_active`, `reencrypt_finished`
* *Hashes de cifrado del servidor*: Útil para clusters HA, indica si todos los servidores están en el mismo escenario con sus archivos locales. Esto puede utilizarse para identificar si es necesario reiniciar los servidores antes de pasar a la siguiente fase. En el ejemplo de HA anterior, el nodo-1 y el nodo-2 tienen hashes diferentes, lo que indica que actualmente no tienen la misma configuración de cifrado. Al reiniciar los servidores se sincronizará su configuración.
* *Tabla clave*: Resume la información sobre las claves de cifrado secretas encontradas en el nodo.
 ** *Activa*: El "*" indica cuál de las claves, si es que hay alguna, se utiliza actualmente para el cifrado de secretos. Una clave activa es utilizada por Kubernetes para cifrar cualquier nuevo secreto.
 ** *Tipo de llave*: Todas las teclas que utilizan esta herramienta son del tipo `AES-CBC`. Más información https://kubernetes.io/docs/tasks/administer-cluster/encrypt-data/#providers[en]
 ** *Nombre*: Nombre de la clave de encriptación.
