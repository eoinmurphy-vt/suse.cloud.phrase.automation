= Guía de inicio rápido
:page-languages: [en, ja, ko, zh]
:revdate: 2026-03-03
:page-revdate: {revdate}

Esta guía le ayudará a poner en marcha rápidamente un clúster con las opciones predeterminadas. Asegúrese de que sus nodos cumplen los xref:installation/requirements.adoc[requisitos] antes de continuar.

* Consulte la página de xref:installation/installation.adoc[instalación] para obtener más detalles sobre la instalación y configuración de los K3.
* Para más información sobre el funcionamiento conjunto de los componentes del K3, consulte la página xref:architecture.adoc[Arquitectura].
* Si eres nuevo en Kubernetes, https://kubernetes.io/docs/tutorials/kubernetes-basics/[los documentos oficiales de Kubernetes en] tienen excelentes tutoriales que cubren aspectos básicos con los que todos los administradores de clústeres deberían estar familiarizados.

[IMPORTANT]
====
Puede utilizar cualquier versión de K3s Prime que aparezca en la URL de Prime Artifacts para los activos mencionados en estos pasos. Para obtener más información sobre la URL de Prime Artifacts, consulte nuestra https://scc.suse.com/rancher-docs/rancherprime/latest/en/reference-guide.html#prime-artifacts-url[documentación exclusiva de Prime en]. Se requiere autenticación. Utilice sus credenciales SUSE https://scc.suse.com/home[del Centro de servicios al cliente de] para iniciar sesión.
====


== Instalar script

{k3s-product-name} proporciona un script de instalación que es una forma conveniente de instalarlo como servicio en sistemas basados en systemd u openrc. Este script está disponible en https://get.k3s.io.. Para instalar {k3s-product-name} utilizando este método, haga lo siguiente:

Ejecute el instalador, donde `INSTALL_K3S_ARTIFACT_URL` es la https://scc.suse.com/rancher-docs/rancherprime/latest/en/reference-guide.html#prime-artifacts-url[URL de] y `INSTALL_K3S_CHANNEL` es un canal de lanzamiento al que puede suscribirse y que por defecto es `stable`. En este ejemplo, `INSTALL_K3S_CHANNEL="latest"` le proporciona la última versión de {k3s-product-name}.

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_CHANNEL="latest" sh -
----
Si desea especificar una versión, establezca la variable de entorno `INSTALL_RKE2_VERSION`:

[,bash]
----
curl -sfL https://get.k3s.io/ | sudo INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_VERSION="<VERSION>" sh -
----

Después de ejecutar esta instalación:

* El servicio K3s se configurará para que se reinicie automáticamente tras el reinicio del nodo o si el proceso se bloquea o muere.
* Se instalarán utilidades adicionales, como `kubectl`, `crictl`, `ctr`, `k3s-killall.sh`, y `k3s-uninstall.sh`
* Se escribirá un archivo kubeconfig https://kubernetes.io/docs/concepts/configuration/organize-cluster-access-kubeconfig/[] en `/etc/rancher/k3s/k3s.yaml` y el kubectl instalado por {k3s-product-name} lo utilizará automáticamente

Una instalación de servidor de nodo único es un clúster Kubernetes totalmente funcional, que incluye todos los componentes de datastore, plano de control, kubelet y tiempo de ejecución de contenedores necesarios para alojar pods de carga de trabajo. No es necesario añadir nodos de servidor o agentes adicionales, pero es posible que desee hacerlo para añadir capacidad adicional o redundancia a su clúster.

Para instalar nodos de agente adicionales y añadirlos al clúster, ejecute el script de instalación con las variables de entorno `K3S_URL` y `K3S_TOKEN`. A continuación se muestra un ejemplo de cómo unirse a un agente:

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s K3S_URL=https://myserver:6443 K3S_TOKEN=mynodetoken sh -
----

La configuración del parámetro `K3S_URL` hace que el instalador configure el K3s como agente, en lugar de servidor. El agente K3s se registrará en el servidor K3s que escucha en la URL suministrada. El valor a utilizar para `K3S_TOKEN` se almacena en `/var/lib/rancher/k3s/server/node-token` en su nodo servidor.

[NOTE]
====
Cada máquina debe tener un nombre de host único. Si sus máquinas no tienen nombres de host únicos, pase la variable de entorno `K3S_NODE_NAME` y proporcione un valor con un nombre de host válido y único para cada nodo.
====


Si está interesado en disponer de más nodos de servidor, consulte las páginas de xref:datastore/ha-embedded.adoc[etcd incrustado] de xref:datastore/ha-embedded.adoc[alta disponibilidad] y xref:datastore/ha.adoc[BD externa de alta] disponibilidad para obtener más información.
