= Importar imágenes
:page-languages: [en, ja, ko, zh]
:revdate: 2026-01-05
:page-revdate: {revdate}
:page-aliases: import-images.adoc

Las imágenes de los contenedores son almacenadas localmente en cada nodo por el almacén de imágenes containerd. Las imágenes pueden extraerse del registro según las necesiten los pods, precargarse mediante image pull o importarse desde un tarball de imágenes.

== Extracción de imágenes a la carta

Kubernetes, por defecto, extrae automáticamente las imágenes cuando un Pod las requiere si la imagen no está ya presente en el nodo. Este comportamiento puede modificarse mediante el campo https://kubernetes.io/docs/concepts/containers/images/#image-pull-policy[image pull policy] del Pod. Cuando se utiliza la política por defecto `IfNotPresent`, containerd extraerá la imagen de upstream (por defecto) o de su xref:installation/private-registry.adoc[registro privado] y la almacenará en su almacén de imágenes. Los usuarios no necesitan aplicar ninguna configuración adicional para que funcione la extracción de imágenes a petición.


== Imágenes previas a la importación

[IMPORTANT]
.Versión Puerta
====
La función de preimportación de imágenes mientras se ejecuta K3s está disponible a partir de las versiones de enero de 2025: v1.32.0+k3s1, v1.31.5+k3s1, v1.30.9+k3s1, v1.29.13+k3s1. Antes, el K3s preimportaba las imágenes sólo al arrancar.
====

La importación previa de imágenes en el nodo es esencial si se configura `imagePullPolicy` de Kubernetes como `Never`. Puede hacerlo por motivos de seguridad o para reducir el tiempo que tardan en girar los nodos K3s.

K3s incluye dos mecanismos para pre-importar imágenes al almacén de imágenes containerd:

[tabs,sync-group-id=import-images]
======
Importación de imágenes en línea::
+
--

Los usuarios pueden activar una extracción de imágenes en el almacén de imágenes containerd colocando un archivo de texto que contenga los nombres de las imágenes, uno por línea, en el directorio `/var/lib/rancher/k3s/agent/images`. El archivo de texto puede colocarse antes de iniciar K3s, así como crearse o modificarse mientras K3s está funcionando. K3s extrae secuencialmente las imágenes a través de la API CRI, utilizando opcionalmente la configuración xref:installation/private-registry.adoc[registries.yaml].

Por ejemplo:

[,bash]
----
mkdir /var/lib/rancher/k3s/agent/images
cp example.txt /var/lib/rancher/k3s/agent/images
----

Donde `example.txt` contiene:

----
docker.io/library/redis:latest
docker.io/library/mysql:latest
----

Después de unos segundos, las imágenes `redis` y `mysql` están disponibles en el almacén de imágenes containerd del nodo. 

Utilice `sudo k3s ctr images list` para consultar el almacén de imágenes containerd.
--

Importación de imágenes sin conexión::
+
--
Los usuarios pueden importar imágenes directamente al almacén de imágenes containerd colocando los tarballs de imágenes en el directorio `/var/lib/rancher/k3s/agent/images`. El tarball puede colocarse antes de iniciar K3s, así como crearse o modificarse mientras K3s se está ejecutando. K3s descomprime el tarball de imágenes si es necesario, extrae las imágenes y las carga en el almacén de imágenes containerd.

Por ejemplo:

[,bash]
----
mkdir /var/lib/rancher/k3s/agent/images
curl  https://github.com/k3s-io/k3s/releases/download/v1.33.1%2Bk3s1/k3s-airgap-images-amd64.tar.zst -O  /var/lib/rancher/k3s/agent/images/k3s-airgap-images-amd64.tar.zst
----

Tras unos segundos, las imágenes incluidas en el tarball de imágenes están disponibles en el almacén de imágenes containerd del nodo. 

Utilice `sudo k3s ctr images list` para consultar el almacén de imágenes containerd.

Este es el método que se utiliza en los entornos con cámara de aire. Siga la xref:installation/airgap.adoc[documentación de instalación de la cámara de] aire para obtener información detallada.
--
======

== Crear un registro de imágenes

K3s admite dos alternativas para los registros de imágenes:

* xref:installation/private-registry.adoc[Configuración del Registro Privado] cubre el uso de `registries.yaml` para configurar la autenticación y réplica del registro de imágenes de contenedor.
* xref:installation/registry-mirror.adoc[Embedded Registry Mirror] muestra cómo habilitar la réplica del registro de imágenes distribuido embebido, para compartir imágenes entre nodos de igual a igual.
