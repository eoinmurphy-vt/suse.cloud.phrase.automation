= Acceso al clúster
:page-languages: [en, ja, ko, zh]
:revdate: 2025-09-15
:page-revdate: {revdate}

El archivo admin kubeconfig almacenado en `/etc/rancher/k3s/k3s.yaml` puede utilizarse para proporcionar acceso al clúster Kubernetes. Este archivo otorga acceso como el usuario `system:admin` y el grupo `system:masters`, que están codificados por Kubernetes para tener acceso sin restricciones a todos los recursos dentro del clúster.

El comando `kubectl` que viene con el K3s está configurado para cargar configuraciones desde esta ruta por defecto. Si ha instalado herramientas de línea de comandos de Kubernetes como kubectl o Helm, debe configurarlas con la ruta kubeconfig correcta. Esto puede hacerse exportando la variable de entorno `KUBECONFIG` o invocando la opción de línea de comandos `--kubeconfig`. Consulte los ejemplos siguientes para obtener más información.

* Exporte la variable de entorno KUBECONFIG:
+
[,bash]
----
export KUBECONFIG=/etc/rancher/k3s/k3s.yaml
kubectl get pods --all-namespaces
helm ls --all-namespaces
----

* Especifique la ubicación del archivo kubeconfig en el comando:
[,bash]
----
kubectl --kubeconfig /etc/rancher/k3s/k3s.yaml get pods --all-namespaces
helm --kubeconfig /etc/rancher/k3s/k3s.yaml ls --all-namespaces
----

== Acceso al clúster desde el exterior con kubectl

Copie `/etc/rancher/k3s/k3s.yaml` en su máquina situada fuera del cluster como `~/.kube/config`. A continuación, sustituya el valor del campo `server` por la IP o el nombre de su servidor K3s. `kubectl` ya puede gestionar su clúster K3s.

[NOTE]
====
K3s actualiza automáticamente los certificados dentro del kubeconfig del administrador cada vez que se inicia. Si realiza una copia de este archivo, deberá actualizar manualmente esas copias para asegurarse de que los certificados en línea no caducan.
====
