= Métriques
:page-languages: [en, ja, ko, zh]
:revdate: 2025-09-16
:page-revdate: {revdate}

K3s fournit des mesures pour surveiller l'état et les performances du cluster.

La plupart des mesures sont fournies par des composants individuels. Pour plus d'informations, voir la documentation suivante spécifique au composant :

* https://coredns.io/plugins/metrics/[métriques coredns]
* https://etcd.io/docs/v3.5/metrics/[métriques etcd]

D'autres mesures peuvent être fournies par d'autres composants. Consultez la documentation du projet en amont pour tout élément non mentionné ci-dessus.

== Mesures pour le superviseur

Lorsque K3s est lancé avec `supervisor-metrics: true`, les métriques sont exposées par le processus K3s et sont accessibles via le point de terminaison `/metrics` sur chaque nœud au port `6443`:

[,sh]
----
kubectl get --server https://NODENAME:6443 --raw /metrics
----

Le processus de supervision K3s expose les mesures suivantes :

* K3s Cluster Management Metrics (métriques de gestion des clusters)
* https://github.com/rancher/lasso/blob/main/README.md#lasso-controller[Métriques du contrôleur Lasso]
* https://github.com/kubernetes/client-go/blob/master/README.md[Mesures des clients et des files d'attente de Kubernetes]
* https://kubernetes.io/docs/reference/instrumentation/node-metrics/[Mesures des nœuds Kubernetes]
* https://kubernetes.io/docs/reference/instrumentation/metrics/[Mesures des composants de Kubernetes]
* https://pkg.go.dev/runtime/metrics#hdr-Supported_metrics[Mesures de la durée d'exécution de Go]

Si le registre intégré de K3s est activé, le processus de supervision de K3s expose également les mesures suivantes :

* https://spegel.dev/docs/metrics/[Métriques de Spegel]
* https://github.com/libp2p/go-libp2p/blob/master/README.md[métriques libp2p]

K3s exécute tous les composants Kubernetes dans le processus principal de K3s. Étant donné que Kubernetes utilise un seul registre de métriques Prometheus par processus, les métriques de tous les composants sont disponibles via tous les points d'extrémité de métriques exposés. Si vous scrapez tous les points d'extrémité des métriques individuelles, vous pouvez constater que vous collectez des métriques en double. Il suffit de récupérer un seul point d'extrémité de métrique K3s pour obtenir des métriques pour tous les composants Kubernetes intégrés.

== K3s Cluster Management Metrics (métriques de gestion des clusters)

=== k3s_certificate_expiration_seconds

Durée de vie restante en secondes du certificat, étiquetée par le sujet du certificat et les usages.

* Type : Gauge
* Étiquettes : `*subject*`, `*usage*`

=== k3s_loadbalancer_server_connections

Nombre de connexions actuelles au serveur d'équilibrage de charge, identifiées par le nom et l'adresse du serveur d'équilibrage de charge.

* Type : Gauge
* Étiquettes : `*name*`, `*server*`

=== k3s_loadbalancer_server_health

État de santé actuel des serveurs backend de l'équilibreur de charge, identifié par le nom de l'équilibreur de charge et l'adresse du serveur.

State est une énumération de 0=INVALID, 1=FAILED, 2=STANDBY, 3=UNCHECKED, 4=RECOVERING, 5=HEALTHY, 6=PREFERRED, 7=ACTIVE.

* Type : Gauge
* Étiquettes : `*name*`, `*server*`

=== k3s_loadbalancer_dial_duration_seconds

Temps en secondes nécessaire pour établir une connexion avec un serveur backend, identifié par le nom de l'équilibreur de charge et l'état de réussite ou d'échec.

* Type : Histogramme
* Étiquettes : `*name*`, `*status*`

=== k3s_etcd_snapshot_save_duration_seconds

Temps total en secondes pris pour compléter le processus d'instantané etcd, étiqueté par le statut succès/échec.

* Type : Histrogramme
* Étiquettes : `*status*`

=== k3s_etcd_snapshot_save_local_duration_seconds

Temps total en secondes pris pour enregistrer un fichier d'instantané local, étiqueté par l'état de succès/échec.

* Type : Histrogramme
* Étiquettes : `*status*`

=== k3s_etcd_snapshot_save_s3_duration_seconds

Temps total en secondes pris pour télécharger un fichier instantané sur S3, étiqueté par le statut succès/échec.

* Type : Histrogramme
* Étiquettes : `*status*`

=== k3s_etcd_snapshot_reconcile_duration_seconds

Temps total en secondes pris pour synchroniser la liste des instantanés etcd, étiquetés par état de succès/échec.

* Type : Histrogramme
* Étiquettes : `*status*`

=== k3s_etcd_snapshot_reconcile_local_duration_seconds

Temps total (en secondes) nécessaire pour lister les fichiers d'instantanés locaux, étiquetés selon l'état de réussite ou d'échec.

* Type : Histrogramme
* Étiquettes : `*status*`

=== k3s_etcd_snapshot_reconcile_s3_duration_seconds

Temps total en secondes nécessaire pour lister les fichiers d'instantanés S3, étiquetés par état de réussite/échec.

* Type : Histrogramme
* Étiquettes : `*status*`
