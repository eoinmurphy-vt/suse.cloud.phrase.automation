= Variables d'environnement
:page-languages: [en, ja, ko, zh]
:revdate: 2024-12-05
:page-revdate: {revdate}

Comme indiqué dans le xref:quick-start.adoc[Guide de démarrage rapide], vous pouvez utiliser le script d'installation disponible à l'adresse https://get.k3s.io pour installer K3s en tant que service sur les systèmes basés sur systemd et openrc.

La forme la plus simple de cette commande est la suivante :

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s sh -
----

Lorsque vous utilisez cette méthode pour installer K3s, les variables d'environnement suivantes peuvent être utilisées pour configurer l'installation :

|===
| Variable d'environnement | Description

| `INSTALL_K3S_SKIP_DOWNLOAD`
| S'il est fixé à true, il ne téléchargera pas le hash ou le binaire de K3s.

| `INSTALL_K3S_SYMLINK`
| Par défaut, il crée des liens symboliques pour les binaires kubectl, crictl et ctr si les commandes n'existent pas déjà dans le chemin d'accès. Si elle est définie sur "skip", elle ne créera pas de liens symboliques et "force" l'écrasera.

| `INSTALL_K3S_SKIP_ENABLE`
| S'il est fixé à true, il n'activera ni ne démarrera le service K3s.

| `INSTALL_K3S_SKIP_START`
| Si cette valeur est fixée à true, le service K3s ne démarrera pas.

| `INSTALL_K3S_VERSION`
| Version de K3s à télécharger depuis Github. Tentera de télécharger à partir du canal stable s'il n'est pas spécifié.

| `INSTALL_K3S_BIN_DIR`
| Répertoire dans lequel installer le binaire, les liens et le script de désinstallation de K3s, ou utiliser `/usr/local/bin` comme répertoire par défaut.

| `INSTALL_K3S_BIN_DIR_READ_ONLY`
| Si la valeur est fixée à true, les fichiers ne seront pas écrits sur `INSTALL_K3S_BIN_DIR`, ce qui force le réglage sur `INSTALL_K3S_SKIP_DOWNLOAD=true`.

| `INSTALL_K3S_SYSTEMD_DIR`
| Répertoire dans lequel installer les services et les fichiers d'environnement de systemd, ou utiliser `/etc/systemd/system` comme valeur par défaut.

| `INSTALL_K3S_EXEC`
| Commandement avec drapeaux à utiliser pour lancer les K3 dans le service. Si la commande n'est pas spécifiée et que l'adresse `K3S_URL` est définie, la valeur par défaut est "agent". Si `K3S_URL` n'est pas défini, la valeur par défaut est "server". Pour obtenir de l'aide, reportez-vous à xref:installation/configuration.adoc#_configuration_with_install_script[cet exemple.]

| `INSTALL_K3S_NAME`
| Nom du service systemd à créer, par défaut 'k3s' si k3s est exécuté en tant que serveur et 'k3s-agent' si k3s est exécuté en tant qu'agent. S'il est spécifié, le nom sera préfixé par "k3s-".

| `INSTALL_K3S_TYPE`
| Type de service systemd à créer, par défaut à partir de la commande K3s exec s'il n'est pas spécifié.

| `INSTALL_K3S_SELINUX_WARN`
| S'il vaut true, il continuera si la politique k3s-selinux n'est pas trouvée.

| `INSTALL_K3S_SKIP_SELINUX_RPM`
| S'il est fixé à true, l'installation automatique du RPM k3s sera ignorée.

| `INSTALL_K3S_CHANNEL_URL`
| URL du canal pour récupérer l'URL de téléchargement de K3s. La valeur par défaut est https://update.k3s.io/v1-release/channels.

| `INSTALL_K3S_CHANNEL`
| Canal à utiliser pour récupérer l'URL de téléchargement de K3s. La valeur par défaut est "stable". Les options sont les suivantes : `stable`, `latest`, `testing`.
|===

Cet exemple montre où placer les variables d'environnement susmentionnées en tant qu'options (après le tuyau) :

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_CHANNEL=latest sh -
----

Les variables d'environnement commençant par `K3S_` seront préservées pour que les services systemd et openrc puissent les utiliser.

Le fait de définir `K3S_URL` sans définir explicitement une commande exec aura pour valeur par défaut la commande "agent".

Lors de l'exécution de l'agent, `K3S_TOKEN` doit également être défini.

[NOTE]
.Version Gate
====
Disponibles à partir des versions d'octobre 2024 : v1.28.15+k3s1, v1.29.10+k3s1, v1.30.6+k3s1, v1.31.2+k3s1.
====

K3s utilisera désormais `PATH` pour trouver d'autres runtimes de conteneurs, en plus de vérifier les chemins par défaut utilisés par les paquets de runtimes de conteneurs. Pour utiliser cette fonctionnalité, vous devez modifier la variable d'environnement PATH du service K3s afin d'ajouter les répertoires contenant les binaires d'exécution du conteneur.

Il est recommandé de modifier l'un de ces deux fichiers d'environnement :

* /etc/default/k3s # ou k3s-agent
* /etc/sysconfig/k3s # ou k3s-agent

Cet exemple ajoutera le site `PATH` à `/etc/default/k3s`:

[,bash]
----
echo PATH=$PATH >> /etc/default/k3s
----

[WARNING]
====
`PATH` Les modifications doivent être effectuées avec précaution afin d'éviter de placer des binaires non fiables sur le chemin des services qui s'exécutent en tant que root.
====
