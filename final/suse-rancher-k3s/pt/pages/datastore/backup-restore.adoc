= Backup e restauração
:page-languages: [en, ja, ko, zh]
:revdate: 2024-09-11
:page-revdate: {revdate}

A forma como é feito o backup e a restauração do K3s depende do tipo de armazenamento de dados utilizado.

[CAUTION]
====
Além de fazer o backup do próprio datastore, você também deve fazer o backup do arquivo de token do servidor em `/var/lib/rancher/k3s/server/token`.
Você deve restaurar esse arquivo ou passar seu valor na opção `--token` ao restaurar a partir do backup.
Se você não usar o mesmo valor de token ao restaurar, o instantâneo será inutilizável, pois o token é usado para criptografar dados confidenciais dentro do próprio armazenamento de dados.
====


== Backup e restauração com o SQLite

Não são necessários comandos especiais para fazer backup ou restaurar o armazenamento de dados do SQLite.

* Para fazer o backup do repositório de dados do SQLite, faça uma cópia de `/var/lib/rancher/k3s/server/db/`.
* Para restaurar o repositório de dados do SQLite, restaure o conteúdo de `/var/lib/rancher/k3s/server/db` (e o token, conforme discutido acima).

== Backup e restauração com armazenamento de dados externo

Quando um armazenamento de dados externo é usado, as operações de backup e restauração são tratadas fora do K3s. O administrador do banco de dados precisará fazer backup do banco de dados externo ou restaurá-lo a partir de um snapshot ou dump.

Recomendamos configurar o banco de dados para tirar instantâneos recorrentes.

Para obter detalhes sobre como tirar instantâneos do banco de dados e restaurar o banco de dados a partir deles, consulte a documentação oficial do banco de dados:

* https://dev.mysql.com/doc/refman/8.0/en/replication-snapshot-method.html[Documentação oficial do MySQL]
* https://www.postgresql.org/docs/8.3/backup-dump.html[Documentação oficial do PostgreSQL]
* https://etcd.io/docs/latest/op-guide/recovery/[Documentação oficial do etcd]
