= 备份和恢复
:page-languages: [EN, JA, KO, ZH]
:revdate: 2024-09-11
:page-revdate: {revdate}

备份和恢复 K3s 的方式取决于使用的数据存储类型。

[CAUTION]
====
除了备份数据存储本身，还必须备份服务器令牌文件`/var/lib/rancher/k3s/server/token` 。
从备份还原时，必须还原该文件，或在`--token` 选项中传递其值。
如果还原时不使用相同的令牌值，快照将无法使用，因为令牌用于加密数据存储本身的机密数据。
====


== 使用 SQLite 进行备份和还原

备份或还原 SQLite 数据存储不需要特殊命令。

* 要备份 SQLite 数据存储，请复制`/var/lib/rancher/k3s/server/db/` 。
* 要还原 SQLite 数据存储，请还原`/var/lib/rancher/k3s/server/db` 的内容（以及令牌，如上所述）。

== 使用外部数据存储进行备份和还原

使用外部数据存储时，备份和还原操作在 K3s 外部处理。数据库管理员需要备份外部数据库，或从快照或转储中还原。

我们建议将数据库配置为定期快照。

有关获取数据库快照和从中恢复数据库的详细信息，请参阅官方数据库文档：

* https://dev.mysql.com/doc/refman/8.0/en/replication-snapshot-method.html[MySQL 官方文档]
* https://www.postgresql.org/docs/8.3/backup-dump.html[PostgreSQL 官方文档]
* https://etcd.io/docs/latest/op-guide/recovery/[官方 etcd 文档]

== 使用嵌入式 etcd 数据存储进行备份和还原

有关在嵌入式 etcd 数据存储上执行备份和还原操作的信息，请参阅xref:cli/etcd-snapshot.adoc[`k3s etcd-snapshot` 命令文档]。
