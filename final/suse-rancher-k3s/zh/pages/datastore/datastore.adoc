= 群集数据存储
:page-languages: [EN, JA, KO, ZH]
:revdate: 2025-06-16
:page-revdate: {revdate}

使用 etcd 以外的数据存储运行 Kubernetes 的能力让 K3s 与其他 Kubernetes 发行版区别开来。该功能为 Kubernetes 操作员提供了灵活性。通过可用的数据存储选项，您可以选择最适合您使用情况的数据存储。例如

* 如果您的团队不具备操作 etcd 的专业知识，您可以选择企业级 SQL 数据库，如 MySQL 或 PostgreSQL
* 如果需要在 CI/CD 环境中运行一个简单、短暂的集群，可以使用嵌入式 SQLite 数据库
* 如果您希望在边缘部署 Kubernetes 并需要高可用解决方案，但又无法承担在边缘管理数据库的运营开销，您可以使用 K3s 基于嵌入式 etcd 构建的嵌入式 HA 数据存储。

K3s 支持以下数据存储选项：

* *嵌入式https://www.sqlite.org/index.html[] SQLite* +
SQLite 不能在有多个服务器的集群中使用。 +
SQLite 是默认数据存储，如果没有其他数据存储配置，且磁盘上没有嵌入的 etcd 数据库文件，则会使用 SQLite。
* *嵌入式 etcd* +
有关在多个服务器上使用嵌入式xref:datastore/ha-embedded.adoc[etcd 的]更多信息，请参阅xref:datastore/ha-embedded.adoc[高可用性]嵌入式 etcd 文档。
如果 K3s 被配置为初始化一个新的 etcd 集群、加入一个现有的 etcd 集群，或者在启动过程中磁盘上存在 etcd 数据库文件，则会自动选择嵌入式 etcd。
* *外部数据库* +
有关在多个服务器上使用外部数据存储的更多信息，请参阅xref:datastore/ha.adoc[高可用性外部 DB]文档。 +
支持以下外部数据存储：
 ** https://etcd.io/[etcd]（已通过 3.5.21 版认证）
 ** https://www.mysql.com/[MySQL]（已通过 5.7 和 8.0 版本认证）
 ** https://mariadb.org/[MariaDB]（已通过 10.11 和 11.4 版认证）
 ** https://www.postgresql.org/[PostgreSQL]（已通过 15.12、16.7 和 17.3 版本的认证）

[CAUTION]
.准备声明支持
====
K3s 要求数据库提供准备好的报表支持。这意味着连接池（如https://www.pgbouncer.org/faq.html#how-to-use-prepared-statements-with-transaction-pooling[] PgBouncer）可能需要额外配置才能与 K3 一起使用。
====

[WARNING]
.多主机设置
====
不支持将`auto_increment_increment` 或`auto_increment_offset` 设置为大于 1 的多主数据库。Kine 希望修订版从 0 开始，当成功插入一个按键时，修订版总是向前精确移动 1。这会影响 Galera for MySQL/MariaDB 等产品。
====

[#_external_datastore_configuration_parameters]
== 外部数据存储配置参数

如果希望使用 PostgreSQL、MySQL 或 etcd 等外部数据存储，则必须设置`datastore-endpoint` 参数，以便 K3s 知道如何与其连接。您还可以指定参数来配置连接的身份验证和加密。下表总结了这些参数，它们可以作为 CLI 标志或环境变量传递。

|===
| CLI 标志 | 环境变量 | 说明

| `--datastore-endpoint`
| `K3S_DATASTORE_ENDPOINT`
| 指定 PostgreSQL、MySQL 或 etcd 连接字符串。这是一个字符串，用于描述与数据存储的连接。该字符串的结构因后台而异，详情如下。

| `--datastore-cafile`
| `K3S_DATASTORE_CAFILE`
| TLS 证书颁发机构 (CA) 文件，用于确保与数据存储的通信安全。如果数据存储使用由自定义证书颁发机构签署的证书通过 TLS 发送请求，则可使用此参数指定 CA，以便 K3s 客户端能正确验证证书。

| `--datastore-certfile`
| `K3S_DATASTORE_CERTFILE`
| TLS 证书文件，用于对数据存储进行基于客户端证书的身份验证。要使用此功能，您的数据存储必须配置为支持基于客户证书的身份验证。如果指定了该参数，还必须指定`datastore-keyfile` 参数。

| `--datastore-keyfile`
| `K3S_DATASTORE_KEYFILE`
| TLS 密钥文件，用于对数据存储进行基于客户端证书的身份验证。更多详情，请参阅前面的`datastore-certfile` 参数。
|===

作为最佳实践，我们建议将这些参数设置为环境变量，而不是命令行参数，这样你的数据库凭据或其他敏感信息就不会作为进程信息的一部分暴露出来。

[#_datastore_endpoint_format_and_functionality]
== 数据存储端点格式和功能

如前所述，传给`datastore-endpoint` 参数的值的格式取决于数据存储后端。下面将详细介绍每个受支持的外部数据存储的格式和功能。

[tabs,sync-group-id=ext-db]
======
PostgreSQL::
+
--
PostgreSQL 的数据存储端点参数最常见的格式如下：

`postgres://username:password@hostname:port/database-name`

还提供更多高级配置参数。如需了解更多信息，请参阅 https://godoc.org/github.com/lib/pq.

如果指定的数据库名称不存在，服务器将尝试创建它。

如果您只提供`postgres://` 作为端点，K3s 将尝试执行以下操作：

* 使用`postgres` 作为用户名和密码连接到 localhost
* 创建一个名为 `kubernetes`
--

MySQL / MariaDB::
+
--
MySQL 和 MariaDB 的`datastore-endpoint` 参数最常见的格式如下：

`mysql://username:password@tcp(hostname:3306)/database-name`

还提供更多高级配置参数。如需了解更多信息，请参阅 https://github.com/go-sql-driver/mysql#dsn-data-source-name

请注意，由于https://github.com/k3s-io/k3s/issues/1093[已知的]K3s，您无法设置`tls` 参数。支持 TLS 通信，但不能将此参数设置为 "skip-verify"（跳过验证），从而使 K3s 跳过证书验证。

如果指定的数据库名称不存在，服务器将尝试创建它。

如果您只提供`mysql://` 作为端点，K3s 将尝试执行以下操作：

* 使用`root` 用户（无密码）连接到`/var/run/mysqld/mysqld.sock` 的 MySQL 套接字
* 创建一个数据库，名称为 `kubernetes`
--

etcd::
+
--
在最常见的形式中，etcd 的`datastore-endpoint` 参数格式如下：

`\https://etcd-host-1:2379,https://etcd-host-2:2379,https://etcd-host-3:2379\`

以上假设是一个典型的三节点 etcd 集群。该参数可接受一个或多个逗号分隔的 etcd URL。
--
======
