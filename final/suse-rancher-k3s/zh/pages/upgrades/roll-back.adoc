= 滚回 K3
:page-languages: [EN, JA, KO, ZH]
:revdate: 2026-03-03
:page-revdate: {revdate}

您可以在升级后回滚 K3s Kubernetes 版本，结合使用 K3s 二进制降级和数据存储恢复。回滚可在各种类型的集群上执行，包括单节点 SQLite、外部数据存储或嵌入式 etcd。在回滚到之前的 Kubernetes 次版本时，必须在要回滚到的 Kubernetes 次版本上有数据存储快照。

[WARNING]
====
如果无法还原数据库，就无法回滚到以前的次版本。
====

== 重要考虑因素

* *备份：*升级前，请确保您拥有运行旧版 K3s 集群的有效数据库或 etcd 快照。没有备份，就不可能回滚。
* *潜在的数据丢失：*`k3s-killall.sh` 脚本会强制终止 K3s 进程，如果应用程序未正确关闭，可能会导致数据丢失。
* *版本说明：*在回滚前后始终验证 K3 和组件版本。

== 回滚 K3s 集群

[tabs]
======

SQLite::
+
--
要在使用 SQLite 数据库时回滚 K3s 集群，请将`.db` 文件替换为备份数据库时创建的`.db` 文件副本。
--

嵌入式 etcd::
+
--
要在使用嵌入式 etcd 时回滚 K3s 群集，请按照以下步骤操作：

. 如果集群正在运行且 Kubernetes API 可用，则通过耗尽所有节点来优雅地停止工作负载：
+
[,bash]
----
kubectl drain --ignore-daemonsets --delete-emptydir-data <NODE-ONE-NAME> <NODE-TWO-NAME> <NODE-THREE-NAME> ...
----

. 在每个节点上，停止 K3s 服务和所有正在运行的 pod 进程：
+
[,bash]
----
k3s-killall.sh
----

. 在每个节点上，将 K3s 二进制文件回滚到以前的版本，但*不要*启动 K3s。

* 可接入互联网的集群：

** 服务器节点：
+
[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_VERSION=vX.Y.Zk3s1 INSTALL_K3S_EXEC="server" INSTALL_K3S_SKIP_START="true" sh -
----

** 代理节点：
+
[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_VERSION=vX.Y.Zk3s1 INSTALL_K3S_EXEC="agent" INSTALL_K3S_SKIP_START="true" sh -
----

* 气隙式集群：

** 下载工件并在本地运行xref:installation/airgap.adoc#_install_k3s[安装脚本]。在运行安装脚本时添加环境变量`INSTALL_K3S_SKIP_START="true"` ，以防止 K3s 启动。

. 在第一个服务器节点或其xref:installation/configuration.adoc[K3s 配置文件]中没有`server:` 条目的节点上，启动群集还原。有关详细信息，请参阅xref:cli/etcd-snapshot.adoc#_snapshot_restore_steps[快照还原步骤]：
+
[,bash]
----
k3s server --cluster-reset --cluster-reset-restore-path=<PATH-TO-SNAPSHOT>
----
+
[WARNING]
====
这会覆盖 etcd 数据存储中的所有数据。在还原之前验证快照的完整性。请注意，恢复大型快照可能需要很长时间。
====

. 在第一个服务器节点上启动 K3s 服务：
+
[,bash]
----
systemctl start k3s
----
 
. 在其他服务器节点上，删除 K3s 数据库目录：
+
[,bash]
----
rm -rf /var/lib/rancher/k3s/server/db
----
 
. 在其他服务器节点上启动 K3s 服务：
+
[,bash]
----
systemctl start k3s
----

. 在所有代理节点上启动 K3s 服务：
+
[,bash]
----
systemctl start k3s
----

. 通过`systemctl status k3s` 验证 K3 的服务状态。
--

外部数据库:: 
+
--
使用外部数据库（如 PostgreSQL、MySQL）时，要回滚 K3s 集群，请按以下步骤操作：

. 如果集群正在运行且 Kubernetes API 可用，则通过耗尽所有节点来优雅地停止工作负载：
+
[,bash]
----
kubectl drain --ignore-daemonsets --delete-emptydir-data <NODE-ONE-NAME> <NODE-TWO-NAME> <NODE-THREE-NAME> ...
----
+
[NOTE]
====
此过程可能会中断正在运行的应用程序。
====

. 在每个节点上，停止 K3s 服务和所有正在运行的 pod 进程：
+
[,bash]
----
k3s-killall.sh
----

. 还原 K3s 升级前的数据库快照并验证数据库的完整性。例如，如果使用 PostgreSQL，请运行以下命令：
+
[,bash]
----
pg_restore -U <DB-USER> -d <DB-NAME> <BACKUP-FILE>
----

. 在每个节点上，将 K3s 二进制文件回滚到之前的版本。

* 可接入互联网的集群：

** 服务器节点：
+
[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_VERSION=vX.Y.Zk3s1 INSTALL_K3S_EXEC="server" sh -
----

** 代理节点：
+
[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_VERSION=vX.Y.Zk3s1 INSTALL_K3S_EXEC="agent" sh -
----

* 气隙式集群：

** 下载工件并在本地运行xref:installation/airgap.adoc#_install_k3s[安装脚本]。使用`k3s --version` 验证安装后的 K3s 版本，并重新应用升级前使用的任何自定义配置。

. 在每个节点上启动 K3s 服务：
+
[,bash]
----
systemctl start k3s
----

. 通过`systemctl status k3s` 验证 K3 的服务状态。
--
======

== 验证 

回滚后，验证以下内容：

* K3s 版本： `k3s --version`
* Kubernetes 集群健康状况 `kubectl get nodes`
* 应用功能。
* 检查 K3s 日志是否有错误。
