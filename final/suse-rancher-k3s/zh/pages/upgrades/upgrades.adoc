== 升级 K3s 集群

xref:upgrades/manual.adoc[手动]升级》介绍了几种手动升级群集的技术。它还可以作为通过第三方基础设施即代码工具（如https://www.terraform.io/[Terraform]）进行升级的基础。

xref:upgrades/automated.adoc[自动升级]》介绍了如何使用 Rancher 的https://github.com/rancher/system-upgrade-controller[system-upgrade-controller] 执行 Kubernetes 本机自动升级。

== 针对具体版本的注意事项

* *特拉菲克*如果没有禁用特拉菲克，K3s 1.20 及更早版本将安装特拉菲克 v1，而 K3s 1.21 及更高版本将安装特拉菲克 v2（如果尚未安装 v1）。要从旧版 Traefik v1 升级到 Traefik v2，请参考https://doc.traefik.io/traefik/migrate/v1-to-v2/[Traefik 文档]并使用https://github.com/traefik/traefik-migration-tool[迁移工具]。
* *K3s 引导数据：*如果您在带有外部 SQL 数据存储的 HA 配置中使用 K3s，且服务器（控制平面）节点未使用`--token` CLI 标志启动，那么在未指定令牌的情况下，您将无法向群集中添加其他 K3s 服务器。确保保留此令牌的副本，因为从备份恢复时需要它。以前，K3s 在使用外部 SQL 数据存储时不会强制使用令牌。
 ** 受影响的版本是<= v1.19.12+k3s1、v1.20.8+k3s1、v1.21.2+k3s1；已修补的版本是 v1.19.13+k3s1、v1.20.9+k3s1、v1.21.3+k3s1。
 ** 您可以从已加入群集的任何服务器中获取令牌值，方法如下：
+
[,bash]
----
cat /var/lib/rancher/k3s/server/token
----
