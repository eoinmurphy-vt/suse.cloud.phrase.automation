= 集群访问
:page-languages: [EN, JA, KO, ZH]
:revdate: 2025-09-15
:page-revdate: {revdate}

存储在`/etc/rancher/k3s/k3s.yaml` 的管理员 kubeconfig 文件可用于提供对 Kubernetes 集群的访问。该文件授予`system:admin` 用户和`system:masters` 组的访问权限，Kubernetes 对这两个用户和组进行了硬编码，使其可以不受限制地访问集群内的所有资源。

K3s 自带的`kubectl` 命令默认从该路径加载配置。如果安装了 kubectl 或 Helm 等上游 Kubernetes 命令行工具，则需要用正确的 kubeconfig 路径对其进行配置。这可以通过导出`KUBECONFIG` 环境变量或调用`--kubeconfig` 命令行标志来实现。详情请参阅下面的示例。

* 导出 KUBECONFIG 环境变量：
+
[,bash]
----
export KUBECONFIG=/etc/rancher/k3s/k3s.yaml
kubectl get pods --all-namespaces
helm ls --all-namespaces
----

* 在命令中指定 kubeconfig 文件的位置：
[,bash]
----
kubectl --kubeconfig /etc/rancher/k3s/k3s.yaml get pods --all-namespaces
helm --kubeconfig /etc/rancher/k3s/k3s.yaml ls --all-namespaces
----

== 使用 kubectl 从外部访问集群

将`/etc/rancher/k3s/k3s.yaml` 复制到位于群集之外的计算机上，即`~/.kube/config` 。然后将`server` 字段的值替换为 K3s 服务器的 IP 或名称。`kubectl` 现在就可以管理 K3s 群集了。

[NOTE]
====
K3s 每次启动时都会自动更新管理程序 kubeconfig 中的证书。如果您复制了该文件，则需要手动更新这些副本，以确保内嵌证书不会过期。
====
