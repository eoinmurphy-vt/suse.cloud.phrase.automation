= 环境变量
:page-languages: [EN, JA, KO, ZH]
:revdate: 2024-12-05
:page-revdate: {revdate}

如《xref:quick-start.adoc[快速入门指南》]所述，您可以使用https://get.k3s.io 上提供的安装脚本，将 K3s 作为服务安装到基于 systemd 和 openrc 的系统上。

该命令的最简单形式如下：

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s sh -
----

使用此方法安装 K3s 时，可使用以下环境变量配置安装：

|===
| 环境变量 | 说明

| `INSTALL_K3S_SKIP_DOWNLOAD`
| 如果设置为 "true"，则不会下载 K3 的哈希值或二进制文件。

| `INSTALL_K3S_SYMLINK`
| 默认情况下，如果路径中尚未存在 kubectl、ccrictl 和 ctr 二进制命令，则将为其创建符号链接。如果设置为 "skip"，将不会创建符号链接，而设置为 "force "则会覆盖。

| `INSTALL_K3S_SKIP_ENABLE`
| 如果设置为 "true"，则不会启用或启动 K3s 服务。

| `INSTALL_K3S_SKIP_START`
| 如果设置为 "true"，则不会启动 K3 服务。

| `INSTALL_K3S_VERSION`
| 可从 Github 下载的 K3s 版本。如果未指定，将尝试从稳定频道下载。

| `INSTALL_K3S_BIN_DIR`
| 安装 K3s 二进制文件、链接和卸载脚本的目录，或使用`/usr/local/bin` 作为默认设置。

| `INSTALL_K3S_BIN_DIR_READ_ONLY`
| 如果设置为 true 将不会把文件写入`INSTALL_K3S_BIN_DIR` ，强制设置`INSTALL_K3S_SKIP_DOWNLOAD=true` 。

| `INSTALL_K3S_SYSTEMD_DIR`
| 安装 systemd 服务和环境文件的目录，或使用`/etc/systemd/system` 作为默认设置。

| `INSTALL_K3S_EXEC`
| 带有标志的命令，用于在服务中启动 K3。如果未指定命令，且`K3S_URL` 已设置，则默认为 "代理"。如果`K3S_URL` 未设置，则默认为 "服务器"。如需帮助，请参阅xref:installation/configuration.adoc#_configuration_with_install_script[此示例。]

| `INSTALL_K3S_NAME`
| 要创建的 systemd 服务的名称，如果将 k3s 作为服务器运行，则默认为 "k3s"；如果将 k3s 作为代理运行，则默认为 "k3s-agent"。如果指定，则名称前缀为 "k3s-"。

| `INSTALL_K3S_TYPE`
| 要创建的 systemd 服务类型，如果未指定，将默认来自 K3s exec 命令。

| `INSTALL_K3S_SELINUX_WARN`
| 如果设置为 true，在找不到 k3s-selinux 策略时将继续执行。

| `INSTALL_K3S_SKIP_SELINUX_RPM`
| 如果设置为 true，将跳过 k3s RPM 的自动安装。

| `INSTALL_K3S_CHANNEL_URL`
| 获取 K3s 下载 URL 的通道 URL。默认为 https://update.k3s.io/v1-release/channels.

| `INSTALL_K3S_CHANNEL`
| 用于获取 K3 下载 URL 的通道。默认为 "稳定"。选项包括`stable`,`latest`,`testing` 。
|===

本例显示了将上述环境变量作为选项（在管道之后）的位置：

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_CHANNEL=latest sh -
----

以`K3S_` 开头的环境变量将被保留，供 systemd 和 openrc 服务使用。

在没有明确设置执行命令的情况下设置`K3S_URL` ，会将命令默认为 "agent"。

运行代理时，还必须设置`K3S_TOKEN` 。

[NOTE]
.版本闸门
====
2024 年 10 月发布的版本：v1.28.15+k3s1、v1.29.10+k3s1、v1.30.6+k3s1、v1.31.2+k3s1。
====

除了检查容器运行时软件包使用的默认路径外，K3s 现在还将使用`PATH` 查找其他容器运行时。要使用此功能，必须修改 K3s 服务的 PATH 环境变量，以添加包含容器运行时二进制文件的目录。

建议修改这两个环境文件中的一个：

* /etc/default/k3s # 或 k3s-agent
* /etc/sysconfig/k3s # 或 k3s-agent

本例将在`/etc/default/k3s` 中添加`PATH` ：

[,bash]
----
echo PATH=$PATH >> /etc/default/k3s
----

[WARNING]
====
`PATH` 更改时应小心谨慎，避免在以根用户身份运行的服务路径中放置不受信任的二进制文件。
====
