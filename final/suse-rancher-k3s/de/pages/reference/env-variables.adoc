= Umgebungsvariablen
:page-languages: [en, ja, ko, zh]
:revdate: 2024-12-05
:page-revdate: {revdate}

Wie in der xref:quick-start.adoc[Schnellstartanleitung] erwähnt, können Sie das unter https://get.k3s.io verfügbare Installationsskript verwenden, um K3s als Dienst auf systemd- und openrc-basierten Systemen zu installieren.

Die einfachste Form dieses Befehls lautet wie folgt:

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s sh -
----

Wenn Sie diese Methode zur Installation von K3s verwenden, können die folgenden Umgebungsvariablen zur Konfiguration der Installation verwendet werden:

|===
| Umgebungsvariable | Beschreibung

| `INSTALL_K3S_SKIP_DOWNLOAD`
| Wenn diese Option auf true gesetzt ist, wird K3s Hash oder Binary nicht heruntergeladen.

| `INSTALL_K3S_SYMLINK`
| Standardmäßig werden Symlinks für die Binärdateien kubectl, crictl und ctr erstellt, wenn die Befehle nicht bereits im Pfad vorhanden sind. Bei der Einstellung 'skip' werden keine Symlinks erstellt und bei 'force' wird überschrieben.

| `INSTALL_K3S_SKIP_ENABLE`
| Wenn diese Option auf true gesetzt ist, wird der K3-Dienst nicht aktiviert oder gestartet.

| `INSTALL_K3S_SKIP_START`
| Wenn diese Option auf true gesetzt ist, wird der K3-Dienst nicht gestartet.

| `INSTALL_K3S_VERSION`
| Version von K3s zum Download von Github. Wenn nicht angegeben, wird versucht, vom stabilen Kanal herunterzuladen.

| `INSTALL_K3S_BIN_DIR`
| Verzeichnis für die Installation von K3s Binärdateien, Links und Deinstallationsskript, oder verwenden Sie `/usr/local/bin` als Standard.

| `INSTALL_K3S_BIN_DIR_READ_ONLY`
| Wenn auf true gesetzt, werden keine Dateien auf `INSTALL_K3S_BIN_DIR` geschrieben, was das Setzen von `INSTALL_K3S_SKIP_DOWNLOAD=true` erzwingt.

| `INSTALL_K3S_SYSTEMD_DIR`
| Verzeichnis für die Installation des systemd-Dienstes und der Umgebungsdateien, oder verwenden Sie `/etc/systemd/system` als Standard.

| `INSTALL_K3S_EXEC`
| Befehl mit Flaggen, die für den Start von K3s im Dienst zu verwenden sind. Wenn der Befehl nicht angegeben wird und die `K3S_URL` gesetzt ist, wird er standardmäßig auf "agent" gesetzt. Wenn `K3S_URL` nicht eingestellt ist, wird standardmäßig "Server" verwendet. Hilfe finden Sie in xref:installation/configuration.adoc#_configuration_with_install_script[diesem Beispiel.]

| `INSTALL_K3S_NAME`
| Name des zu erstellenden systemd-Dienstes, standardmäßig 'k3s', wenn k3s als Server läuft und 'k3s-agent', wenn k3s als Agent läuft. Falls angegeben, wird der Name mit dem Präfix 'k3s-' versehen.

| `INSTALL_K3S_TYPE`
| Typ des zu erstellenden systemd-Dienstes, wird vom K3s exec-Befehl vorgegeben, wenn nicht angegeben.

| `INSTALL_K3S_SELINUX_WARN`
| Wenn auf true gesetzt, wird fortgefahren, wenn die k3s-selinux-Richtlinie nicht gefunden wird.

| `INSTALL_K3S_SKIP_SELINUX_RPM`
| Wenn diese Option auf true gesetzt ist, wird die automatische Installation des k3s-RPM übersprungen.

| `INSTALL_K3S_CHANNEL_URL`
| Kanal-URL für den Abruf von K3s Download-URL. Standardmäßig wird https://update.k3s.io/v1-release/channels.

| `INSTALL_K3S_CHANNEL`
| Kanal, über den die Download-URL von K3 abgerufen wird. Der Standardwert ist "stabil". Zu den Optionen gehören: `stable`, `latest`, `testing`.
|===

Dieses Beispiel zeigt, wo die oben genannten Umgebungsvariablen als Optionen (nach der Pipe) zu platzieren sind:

[,bash]
----
curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s INSTALL_K3S_CHANNEL=latest sh -
----

Umgebungsvariablen, die mit `K3S_` beginnen, werden für die systemd- und openrc-Dienste erhalten.

Wenn Sie `K3S_URL` einstellen, ohne explizit einen exec-Befehl festzulegen, wird der Befehl standardmäßig auf "agent" gesetzt.

Wenn der Agent ausgeführt wird, muss auch `K3S_TOKEN` eingestellt werden.

[NOTE]
.Version Tor
====
Verfügbar ab Oktober 2024: v1.28.15+k3s1, v1.29.10+k3s1, v1.30.6+k3s1, v1.31.2+k3s1.
====

K3s verwendet nun `PATH`, um alternative Container-Laufzeiten zu finden, zusätzlich zur Überprüfung der von den Container-Laufzeitpaketen verwendeten Standardpfade. Um diese Funktion nutzen zu können, müssen Sie die Umgebungsvariable PATH des K3s-Dienstes ändern, um die Verzeichnisse mit den Container-Laufzeit-Binärdateien hinzuzufügen.

Es wird empfohlen, dass Sie eine dieser beiden Umgebungsdateien ändern:

* /etc/default/k3s # oder k3s-agent
* /etc/sysconfig/k3s # oder k3s-agent

In diesem Beispiel wird die Seite `PATH` unter `/etc/default/k3s` hinzugefügt:

[,bash]
----
echo PATH=$PATH >> /etc/default/k3s
----

[WARNING]
====
`PATH` Änderungen sollten mit Vorsicht vorgenommen werden, um zu vermeiden, dass nicht vertrauenswürdige Binärdateien in den Pfad von Diensten platziert werden, die als Root laufen.
====
