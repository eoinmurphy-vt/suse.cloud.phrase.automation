= Deinstallation von K3s
:page-languages: [en, ja, ko, zh]
:revdate: 2026-01-05
:page-revdate: {revdate}

[CAUTION]
====
Die Deinstallation von K3s kann zu Datenverlust führen!
====

Wenn Sie K3s mithilfe des Installationsskripts installiert haben, wurde während der Installation ein Skript zur Deinstallation von K3s erstellt.

Die Ausführung des Deinstallationsskripts stoppt K3s und alle laufenden Pods und löscht den lokalen Cluster-Datenspeicher, die Daten des persistenten xref:add-ons/storage.adoc#_setting_up_the_local_storage_provider[lokalen Speichervolumens], die Knotenkonfiguration sowie alle Skripte und CLI-Tools.

Es werden keine Daten von externen Datenspeichern oder von Pods, die externe Kubernetes Persistent Volumes verwenden, entfernt.

Wenn Sie vorhaben, einen Knoten nach der Deinstallation und Neuinstallation wieder in einen bestehenden Cluster einzubinden, müssen Sie den Knoten aus dem Cluster löschen, um sicherzustellen, dass das geheime Knotenkennwort entfernt wird. Weitere Informationen finden Sie in der Dokumentation zur xref:architecture.adoc#_how_agent_node_registration_works[Knotenregistrierung].

== Deinstallation von Servern

Um K3s von einem Serverknoten zu deinstallieren, führen Sie aus:

[,bash]
----
/usr/local/bin/k3s-uninstall.sh
----

== Agenten deinstallieren

Um K3s von einem Agentenknoten zu deinstallieren, führen Sie aus:

[,bash]
----
/usr/local/bin/k3s-agent-uninstall.sh
----
