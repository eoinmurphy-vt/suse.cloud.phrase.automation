= バックアップと復元
:page-languages: [en、ja、ko、zh]。
:revdate: 2024-09-11
:page-revdate: {revdate}

K3sのバックアップとリストアの方法は、どのタイプのデータストアを使用するかによって異なります。

[CAUTION]
====
データストア自体のバックアップに加えて、`/var/lib/rancher/k3s/server/token` にあるサーバートークンファイルもバックアップする必要があります。
バックアップからリストアする場合は、このファイルをリストアするか、`--token` オプションにその値を渡す必要があります。
データストア内の機密データを暗号化するためにトークンが使用されるため、リストア時に同じトークン値を使用しないと、スナップショットは使用できなくなります。
====


== SQLiteによるバックアップとリストア

SQLiteデータストアのバックアップやリストアには特別なコマンドは必要ありません。

* SQLiteデータストアをバックアップするには、`/var/lib/rancher/k3s/server/db/` 。
* SQLiteデータストアをリストアするには、`/var/lib/rancher/k3s/server/db` （および前述のトークン）の内容をリストアします。

== 外部データストアによるバックアップとリストア

外部データストアを使用する場合、バックアップとリストア操作はK3sの外部で処理される。データベース管理者は、外部データベースをバックアップするか、スナップショットまたはダンプからリストアする必要があります。

定期的にスナップショットを取るようにデータベースを構成することをお勧めします。

データベース・スナップショットの取得と、そこからデータベースをリストアする方法の詳細については、データベースの公式文書を参照してください：

* https://dev.mysql.com/doc/refman/8.0/en/replication-snapshot-method.html[MySQL 公式ドキュメント]
* https://www.postgresql.org/docs/8.3/backup-dump.html[PostgreSQL公式ドキュメント]
* https://etcd.io/docs/latest/op-guide/recovery/[etcd公式ドキュメント]

== 組み込みetcdデータストアによるバックアップとリストア

組み込みetcdデータストアのバックアップとリストア操作の実行については、xref:cli/etcd-snapshot.adoc[`k3s etcd-snapshot` コマンドのドキュメントを]参照してください。
