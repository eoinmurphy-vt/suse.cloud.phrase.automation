= 秘密の暗号化設定
:page-languages: [en、ja、ko、zh]。
:revdate: 2026-03-03
:page-revdate: {revdate}

K3sは、静止時に秘密の暗号化を有効にすることをサポートしています。サーバーの初回起動時にフラグ`--secrets-encryption` を渡すと、自動的に以下のことが行われる：

* AES-CBCキーの生成
* 生成されたキーで暗号化設定ファイルを生成する。
* 設定をencryption-provider-configとしてKubeAPIに渡す。

[TIP]
====
Secrets-encryptionは、既存のサーバーを再起動せずに有効にすることはできません。 +
スクリプトからインストールする場合は`curl -sfL https://get.k3s.io | INSTALL_K3S_ARTIFACT_URL=<PRIME-ARTIFACTS-URL>/k3s sh -s - server --secrets-encryption` を使用するか、xref:installation/configuration.adoc#_configuration_with_install_script[設定オプションに]記載されているその他の方法を使用します。
====

== 暗号化プロバイダーの選択

[IMPORTANT]
.バージョン・ゲート
====
v1.30.12+k3s1、v1.31.8+k3s1、v1.32.4+k3s1、v1.33.0+k3s1。
====

`--secrets-encryption-provider` フラグを使用すると、以下のK3sがサポートする暗号化プロバイダから選択できます：

* `aescbc`:PKCS#7パディング付きAES-CBC。これはデフォルトのプロバイダーである。
* `secretbox`:XSalsa20とPoly1305

### プロバイダーの移行

以下の手順で、`aescbc` プロバイダーから`secretbox` プロバイダーに移行できます：

. `secretbox` プロバイダーがK3sのバージョンでサポートされていることを確認してください。
. K3sコンフィギュレーションファイルの`secrets-encryption-provider` フラグを`secretbox` に更新/追加してください。
. 以下の「xref:cli/secrets-encrypt.adoc#_encryption_key_rotation[暗号化キーのローテーション]」に従って、暗号化キーをローテーションする。

== 暗号化設定ファイルの生成

`--secrets-encryption` でサーバーを起動すると、K3s は`/var/lib/rancher/k3s/server/cred/encryption-config.json` に暗号化設定ファイルを生成します。

以下は、デフォルトの`aescbc` プロバイダーで生成された暗号化設定ファイルの例である：

[,json]
----
{
  "kind": "EncryptionConfiguration",
  "apiVersion": "apiserver.config.k8s.io/v1",
  "resources": [
    {
      "resources": [
        "secrets"
      ],
      "providers": [
        {
          "aescbc": {
            "keys": [
              {
                "name": "aescbckey",
                "secret": "xxxxxxxxxxxxxxxxxxx"
              }
            ]
          }
        },
        {
          "identity": {}
        }
      ]
    }
  ]
}
----

== 秘密の暗号化ツール

K3sには、以下の自動制御を可能にするユーティリティツール`secrets-encrypt` が含まれています：

* 秘密の暗号化の無効化／有効化
* 新しい暗号化キーの追加
* 暗号化キーのローテーションと削除
* 秘密の再暗号化

詳しくは、xref:cli/secrets-encrypt.adoc[`k3s secrets-encrypt` コマンドのドキュメントを]参照。
