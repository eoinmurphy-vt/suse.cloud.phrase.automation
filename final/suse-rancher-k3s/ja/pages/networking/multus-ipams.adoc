= MultusとIPAMプラグイン
:page-languages: [en、ja、ko、zh]。
:revdate: 2025-09-16
:page-revdate: {revdate}

https://github.com/k8snetworkplumbingwg/multus-cni[Multus CNIは]、ポッドに複数のネットワークインターフェースをアタッチできるCNIプラグインです。MultusはCNIプラグインを置き換えるのではなく、CNIプラグインのマルチプレクサとして機能します。Multusは、特にポッドがネットワークを多用し、SR-IOVのようなデータプレーン・アクセラレーション技術をサポートする余分なネットワーク・インターフェイスが必要な場合など、特定のユースケースで有用である。

Multusの詳細については、https://github.com/k8snetworkplumbingwg/multus-cni/tree/master/docs[multus-cni]ドキュメントを参照してください。

Multusはスタンドアロンでは展開できない。Kubernetesクラスタのネットワーク要件を満たす従来のCNIプラグインが必ず1つ以上必要になる。このCNIプラグインはMultusのデフォルトとなり、すべてのポッドにプライマリインターフェースを提供するために使用されます。デフォルトのオプションでK3をデプロイする場合、CNIプラグインはFlannelになります。

[NOTE]
.バージョン・ゲート
====
K3sは、2024年10月リリースのv1.28.15+k3s1、v1.29.10+k3s1、v1.30.6+k3s1、v1.31.2+k3s1の固定CNIバイナリパスを使用しています。
====

K3sはCNIプラグインバイナリの`$DATA_DIR/data/cni` 。デフォルトは`/var/lib/rancher/k3s/data/cni` 。追加のCNIプラグインはこの場所にインストールする必要があります。

2024年10月のリリース以前は、CNIバイナリはK3sのユーザー空間バンドルの一部で、`$DATA_DIR/data/$HASH/bin` 、ハッシュはK3sの各リリースに固有です。
このため、K3sがアップグレードされるたびにパスが変更されるため、追加のCNIプラグインを導入することが困難でした。

K3sの古いリリースにMultusをデプロイする場合、`/var/lib/rancher/k3s/data/current/bin/` をCNI bin dirとして使用する必要がありますが、K3sがアップグレードされるたびにプラグインを再デプロイする必要があります。

== IPAMプラグインでデプロイする

Multusで作成した余分なインターフェースにIPアドレスを割り当てるには、IP Address Manager (IPAM)プラグインが必要です。以下の例ではそれぞれ1つのIPAMプラグインを使用しているが、必要に応じ て組み合わせることができる。

以下のhelmデプロイの例では、DaemonSetをデプロイしてMultusポッドを作成し、必要なCNIバイナリを`/var/lib/rancher/k3s/data/cni/` 、Multus CNIコンフィグを`/var/lib/rancher/k3s/agent/etc/cni/net.d` 。

[tabs,sync-group-id=MultusIPAMplugins]
======
host-local::
+
--
ホストローカルIPAMプラグインは、一連のアドレス範囲からIPアドレスを割り当てる。状態をホストのファイルシステム上にローカルに保存するので、単一ホスト上のIPアドレスの一意性が保証される。従って、マルチノードクラスターにはお勧めしない。このIPAMプラグインは追加のデプロイメントを必要としない。詳細はこちら： https://www.cni.dev/plugins/current/ipam/host-local/.

ホストローカルプラグインを使用するには、以下の設定でMultusをデプロイします：

[,yaml]
----
apiVersion: helm.cattle.io/v1
kind: HelmChart
metadata:
  name: multus
  namespace: kube-system
spec:
  repo: https://rke2-charts.rancher.io
  chart: rke2-multus
  targetNamespace: kube-system
  valuesContent: |-
    config:
      fullnameOverride: multus
      cni_conf:
        confDir: /var/lib/rancher/k3s/agent/etc/cni/net.d
        binDir: /var/lib/rancher/k3s/data/cni/
        kubeconfig: /var/lib/rancher/k3s/agent/etc/cni/net.d/multus.d/multus.kubeconfig
          # Comment the following line when using rke2-multus < v4.2.202
          multusAutoconfigDir: /var/lib/rancher/k3s/agent/etc/cni/net.d
----
--

居場所::
+
--
https://github.com/k8snetworkplumbingwg/whereabouts[Whereaboutsは]、クラスタ全体にIPアドレスを割り当てるIPアドレス管理（IPAM）CNIプラグインです。

Whereabouts IPAM プラグインを使用するには、以下の構成で Multus をデプロイします：

[,yaml]
----
apiVersion: helm.cattle.io/v1
kind: HelmChart
metadata:
  name: multus
  namespace: kube-system
spec:
  repo: https://rke2-charts.rancher.io
  chart: rke2-multus
  targetNamespace: kube-system
  valuesContent: |-
    config:
      fullnameOverride: multus
      cni_conf:
        confDir: /var/lib/rancher/k3s/agent/etc/cni/net.d
        binDir: /var/lib/rancher/k3s/data/cni/
        kubeconfig: /var/lib/rancher/k3s/agent/etc/cni/net.d/multus.d/multus.kubeconfig
          # Comment the following line when using rke2-multus < v4.2.202
          multusAutoconfigDir: /var/lib/rancher/k3s/agent/etc/cni/net.d
    rke2-whereabouts:
      fullnameOverride: whereabouts
      enabled: true
      cniConf:
        confDir: /var/lib/rancher/k3s/agent/etc/cni/net.d
        binDir: /var/lib/rancher/k3s/data/cni/
----

K3でwhereaboutsを使用する場合、NetworkAttachmentDefinitionの`ipam` コンフィギュレーションで、`configuration_path` を`/var/lib/rancher/k3s/agent/etc/cni/net.d/whereabouts.d/whereabouts.conf` に設定する必要があります。

例えば、macvlanプラグインでIPAMとしてwhereaboutsを使う場合：

[,yaml]
----
apiVersion: k8s.cni.cncf.io/v1
kind: NetworkAttachmentDefinition
metadata:
  name: macvlan-whereabouts
spec:
  config: |-
    {
      "cniVersion": "1.0.0",
      "type": "macvlan",
      "master": "eth0",
      "mode": "bridge",
      "ipam": {
        "type": "whereabouts",
        "range": "172.17.0.0/24",
        "gateway": "172.17.0.1",
        "configuration_path": "/var/lib/rancher/k3s/agent/etc/cni/net.d/whereabouts.d/whereabouts.conf"
      }
    }
----
--

Multus DHCPデーモン::
+
--
dhcp IPAMプラグインは、ネットワーク上ですでにDHCPサーバーが稼動している場合に導入できる。このデーモンセットは、DHCPリースを定期的に更新する。詳しくはhttps://www.cni.dev/plugins/current/ipam/dhcp/[IPAMプラグ] DHCPインの公式ドキュメントをご覧ください。

DHCPプラグインを使用するには、以下の設定でMultusをデプロイします：

[,yaml]
----
apiVersion: helm.cattle.io/v1
kind: HelmChart
metadata:
  name: multus
  namespace: kube-system
spec:
  repo: https://rke2-charts.rancher.io
  chart: rke2-multus
  targetNamespace: kube-system
  valuesContent: |-
    config:
      fullnameOverride: multus
      cni_conf:
        confDir: /var/lib/rancher/k3s/agent/etc/cni/net.d
        binDir: /var/lib/rancher/k3s/data/cni/
        kubeconfig: /var/lib/rancher/k3s/agent/etc/cni/net.d/multus.d/multus.kubeconfig
          # Comment the following line when using rke2-multus < v4.2.202
          multusAutoconfigDir: /var/lib/rancher/k3s/agent/etc/cni/net.d
    manifests:
      dhcpDaemonSet: true
----
--

=== マルチュスの使用

Multusがデプロイされたら、NetworkAttachmentDefinitionリソースを作成し、Pod仕様でこれらを参照して追加のインターフェースをアタッチできます。
例えば、上記のwhereaboutsの例を使うと、`k8s.v1.cni.cncf.io/networks` アノテーションを使ってPod上に`eth1` インタフェースを作成できます：

[,yaml]
----
apiVersion: apps/v1
kind: Deployment
metadata:
  name: multus-demo
  labels:
    app: multus-demo
spec:
  replicas: 1
  selector:
    matchLabels:
      app: multus-demo
  template:
    metadata:
      annotations:
        k8s.v1.cni.cncf.io/networks: macvlan-whereabouts@eth1
      labels:
        app: multus-demo
    spec:
      containers:
      - name: shell
        image: docker.io/rancher/mirrored-library-busybox:1.36.1
        imagePullPolicy: IfNotPresent
        command:
          - sleep
          - "3600"
----

その他の情報や例については、アップストリームのドキュメントを参照してください。
