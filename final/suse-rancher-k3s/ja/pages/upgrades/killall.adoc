= K3を止める
:page-languages: [en、ja、ko、zh]。
:revdate: 2024-09-26
:page-revdate: {revdate}

アップグレード中の高可用性を可能にするため、K3sサービスが停止してもK3sコンテナは実行され続けます。

== K3sサービス

K3sの停止と再起動は、systemdとOpenRCのインストールスクリプトでサポートされています。

[tabs]
======
システムディー::
+
--
サーバーを停止する：

[,sh]
----
sudo systemctl stop k3s
----

サーバーを再起動する：

[,sh]
----
sudo systemctl start k3s
----

エージェントを止めるために

[,sh]
----
sudo systemctl stop k3s-agent
----

エージェントを再起動する

[,sh]
----
sudo systemctl start k3s-agent
----
--

OpenRC::
+
--
サーバーを停止する：

[,sh]
----
sudo rc-service k3s stop
----

サーバーを再起動する：

[,sh]
----
sudo rc-service k3s restart
----

エージェントを止めるために

[,sh]
----
sudo rc-service k3s-agent stop
----

エージェントを再起動する

[,sh]
----
sudo rc-service k3s-agent restart
----
--
======

== キラル・スクリプト

すべてのK3sコンテナを停止し、containerdの状態をリセットするには、`k3s-killall.sh` スクリプトを使用することができます。

killallスクリプトは、コンテナ、K3sディレクトリ、およびネットワークコンポーネントをクリーンアップすると同時に、関連するすべてのルールを含むiptablesチェーンも削除する。クラスタ・データは削除されない。

サーバー・ノードからkillallスクリプトを実行するには、以下を実行します：

[,bash]
----
/usr/local/bin/k3s-killall.sh
----
