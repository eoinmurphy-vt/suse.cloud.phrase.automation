= Vertrauenswürdiges Plattformmodul 2.0 (TPM)
:revdate: 2024-09-24
:page-revdate: {revdate}

Trusted Platform Module (TPM, auch bekannt als ISO/IEC 11889) ist ein internationaler Standard für einen sicheren Kryptoprozessor, einen speziellen Mikrocontroller zur Sicherung von Hardware durch integrierte kryptografische Schlüssel. Der Begriff kann sich auch auf einen Chip beziehen, der der Norm entspricht.

== TPM-Modul zur virtuellen Maschine hinzufügen

Eine einfache Möglichkeit, TPM zu einer virtuellen Maschine hinzuzufügen, ist die Verwendung von Libvirt mit Virt-manager

=== Virtuelle Maschine erstellen

Nach dem Start von virt-manager eine neue virtuelle Maschine erstellen

image::tpm1.png[Neue VM erstellen]

=== Überprüfen und Bearbeiten der Hardware-Modul-Liste

Überprüfen Sie auf dem Bildschirm für die Hardwarekonfiguration die Liste der Module und klicken Sie auf *_Hardware hinzufügen_* Schaltfläche

image::tpm2.png[Liste der Geräte]

=== TPM-Modul zur VM hinzufügen

Wählen Sie aus der Liste der emulierten Geräte das TPM-Modul aus und fügen Sie es der VM hinzu

image::tpm3.png[TPM-Modul hinzufügen]

=== VM-Konfiguration abschließen

Überprüfen Sie auf dem letzten Bildschirm noch einmal, ob das TPM-Modul ordnungsgemäß hinzugefügt wurde

image::tpm4.png[Verify TPM]

== Hinzufügen einer TPM-Emulation zu einem Bare-Metal-Rechner

Fügen Sie bei der Anwendung von `#!yaml MachineRegistration` folgenden Schlüssel in die yaml-Datei ein `config:elemental:registration:emulate-tpm: true`

[IMPORTANT]
====
Wenn Sie planen, mehr als einen Rechner mit TPM-Emulation einzusetzen, stellen Sie sicher, dass Sie `config:elemental:registration:emulated-tpm-seed: -1`so einstellen, dass der für die TPM-Emulation verwendete Seed pro Rechner zufällig ist. Andernfalls erhalten Sie denselben TPM-Hash für alle installierten Rechner, und nur der zuletzt registrierte ist gültig.
