= Installieren Sie {elemental-product-name} in einer luftdurchlässigen Umgebung
:revdate: 2025-11-06
:page-revdate: {revdate}

== Annahmen

Eine Rancher Air-Gapped-Installation sollte bereits gemäß der https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/air-gapped/air-gapped.html[offiziellen Rancher-Dokumentation] konfiguriert sein.
Insbesondere sollte ein privates Register in der luftgestützten Infrastruktur verfügbar sein.

== Übersicht

Um {elemental-product-name} in einer Air-Gapped-Umgebung zu betreiben, sind die folgenden Artefakte erforderlich:

* die {elemental-product-name} Bedienerkarten
* die Container-Bilder, auf die in den Diagrammen verwiesen wird (die Bilder von _elemental-operator_ und _seedimage-builder_ )
* die containerisierten OS-Images

Außerdem könnte es praktisch sein, ein _Kanal-Image_ zu erstellen, das auf die verfügbaren containerisierten Betriebssystem-Images verweist.
Das offizielle Channel-Image (das _Elemental-Channel-Image_ ) verweist auf die absoluten URLs der Betriebssystem-Images in der offiziellen Suse-Registrierung, so dass es in einem Air-Gapped-Szenario nicht als solches verwendet werden kann.

== {elemental-product-name} Air-Gapped-Installation über die Kommandozeile

Alle erforderlichen Schritte können durch Ausführen deshttps://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[Skripts]

Die {elemental-product-name} Charts sind ein erforderlicher Parameter für das Skript und können als heruntergeladene Archive, URLs oder als eines der Schlüsselwörter `stable`, `staging` und `dev` angegeben werden, damit das Skript die richtige Chart-Version für Sie abrufen kann.

`elemental-airgap.sh` prüft die {elemental-product-name} Operator-Karte, identifiziert alle erforderlichen Container-Images, lädt sie herunter und speichert sie in einem einzigen Docker-Archiv.
Es erstellt auch ein neues OS-Channel-Image mit den OS-Image-URLs, die auf die private Registrierung verweisen, die als Argument übergeben wird (was ebenfalls ein obligatorisches Argument ist).

Die neueste Version des Elementar-Skripts kann einfach von der offiziellen Github-Website heruntergeladen werden:

[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh
chmod 755 elemental-airgap.sh
----

Lassen Sie uns nun alle Artefakte herunterladen und einen benutzerdefinierten Kanal aus der letzten stabilen Version von {elemental-product-name} erstellen:

[tabs]
====
Erstellen eines Docker-Archivs::
+
======
[,shell]
----
./elemental-airgap.sh stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

Sobald das Skript abgeschlossen ist (das kann eine Weile dauern), sind die folgenden Dateien im aktuellen Verzeichnis verfügbar:

* `elemental-operator-crds-chart-<*VERSION*>.tgz`
* `elemental-operator-chart-<*VERSION*>.tgz`
* `elemental-images.txt`
* `elemental-images.tar.gz`
======

Ein Hauler-Archiv erstellen::
+
======
[,shell]
----
./elemental-airgap.sh -ha stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

Nach der Fertigstellung (das Skript kann eine Weile dauern) werden sowohl die Karten als auch die Container-Images in das Hauler-Archiv mit dem Namen `elemental-haul.tar.zst` gepackt.
======
====

=== {elemental-product-name} Installation 

Die vom Skript erstellten Dateien und Archive sollten auf einen Host kopiert werden, der: 

* Hat Zugriff auf das private Register. 
* Hat die kubectl-Binärdatei installiert und konfiguriert, um auf den Air-Gapped Rancher-Cluster zuzugreifen. 
* Hat das Steuergerät installiert. 

[tabs]
========
Installation aus einem Docker-Archiv::
+
======
Wenn die private Registrierung eine Authentifizierung erfordert, müssen Sie sich mit Docker bei ihr anmelden: 

[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

Für die Installation von {elemental-product-name} sind zwei Schritte erforderlich: 

* Laden Sie das Archiv mit allen erforderlichen Container-Images in die private Registry: Dies kann mit dem `rancher-load-images.sh` -Skript geschehen, das mit der Rancher-Version verteilt wurde und bereits für die Rancher-Air-Gapped-Bereitstellung verwendet wird: 

[,shell]
----
rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

* Installieren Sie die heruntergeladenen Elementar-Charts und konfigurieren Sie die lokale Registrierung und den neu erstellten Kanal: 

[,shell]
----
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ 
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM>
----
======

Installation aus einem Hauler-Archiv::
+
======
Für die Installation aus einem https://rancherfederal.github.io/hauler-docs/[Hauler-Archiv] (`-ha` Option in `elemental-airgap.sh`) ist eine Hauler-Installation auch auf dem Host erforderlich, von dem aus die Installation durchgeführt wird. 

Wenn die private Registry eine Authentifizierung erfordert, müssen Sie sich mit Hauler dort anmelden: 

[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD
----

Für die Installation von {elemental-product-name} sind drei Schritte erforderlich: 


* Laden Sie das Haul-Archiv "elemental-haul.tar.zst" in die Hauler-Instanz in der Airgapped-Infrastruktur: 

[,shell]
----
hauler store load 'elemental-haul.tar.zst' 
----

* Wenn die lokale Registry in der Air-Gapped-Umgebung kein Server von Hauler ist, laden Sie das Haul-Archiv in die lokale Registry: 

[,shell]
----
hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

[NOTE]
.Hauler kann auch als Registrierstelle dienen
====
Falls die lokale Air-Gapped-Registry von einer Hauler-Instanz bedient wird, laden Sie das Haul-Archiv einfach direkt dorthin (wie in Schritt (1) gezeigt) und überspringen Sie Schritt (2).
====

* Entnehmen Sie die Elementkarten aus dem Hauler-Store und installieren Sie sie: 

[,shell]
----
hauler store extract elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
hauler store extract elemental-operator-chart-<ELEMENTAL-VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<ELEMENTAL-VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ -
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM:PORT>
----
======
========

[NOTE]
.Das Skript elemental airgap gibt die erforderlichen Befehle aus 
====
Das Skript `elemental-airgap.sh` druckt die oben gezeigten erforderlichen Befehle aus, verwendet aber die aktuelle Diagrammversion und die bereitgestellte Registrierungs-URL, damit Sie die genauen Befehle einfach kopieren und einfügen können.
====

== {elemental-product-name} Air-Gapped-Installation aus dem Rancher-Marktplatz 

Eine Rancher-Air-Gapped-Installation umfasst auch die {elemental-product-name} Operator-Karten sowie die Operator- und Seedimage-Container-Images. 

Um die fehlenden Betriebssystem-Images zu sammeln und ein OS-Channel-Image für Ihre private Registry zu erstellen, führen Sie das https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[Skript] `-co`. 

Als Beispiel wollen wir das `elemental-channel` Image aus der letzten stabilen Version von {elemental-product-name} verwenden. Das Skript sorgt für das Herunterladen der {elemental-product-name} Operator-Karte (falls erforderlich), extrahiert die URL des OS-Channel-Images, lädt es herunter, prüft alle referenzierten OS-Images, lädt sie alle herunter und erstellt einen neuen OS-Channel mit Links zur privaten Registry des Air-Gapped-Szenarios. 

[tabs]
========
Erstellen eines Docker-Archivs::
+
======
[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh 
chmod 755 elemental-airgap.sh 
./elemental-airgap.sh stable -co -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

Sobald das Skript abgeschlossen ist (das kann eine Weile dauern), sind die folgenden Dateien im aktuellen Verzeichnis verfügbar: 

- `elemental-operator-crds-chart-<*VERSION*>.tgz` 
- `elemental-operator-chart-<*VERSION*>.tgz` 
- `elemental-images.txt` 
- `elemental-images.tar.gz`
======

Ein Hauler-Archiv erstellen::
+
======
[,shell]
----
./elemental-airgap.sh -ha -co stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

Nach der Fertigstellung (das Skript kann eine Weile dauern) werden die Container-Images in das Hauler-Archiv mit dem Namen `elemental-haul.tar.zst` gepackt.
======
========

=== {elemental-product-name} Installation 

Das erzeugte Archiv sollte in die private Registry geladen werden, die mit einer Luftschleuse versehen ist. 

[tabs]
========
Installation aus einem Docker-Archiv::
+
======
Wenn die private Registrierung eine Authentifizierung erfordert, müssen Sie sich mit Docker bei ihr anmelden: 
[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

Das Skript gibt die Befehle aus, die erforderlich sind, um die Bilder über das Tool Rancher `rancher-load-images.sh` zu laden, das für die Rancher Air-Gapped-Installationen verwendet wird. Sie sollte etwa so lauten: 

[,shell]
----
NEXT STEPS: 

1) Load the 'elemental-images.tar.gz' to the local registry (<REGISTRY.YOURDOMAIN.COM:PORT>) available in the airgapped infrastructure: 

./rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

Sobald das Betriebssystem und die Kanal-Images geladen sind, sollten Sie den Punkt (2) der Skriptausgabe überspringen (der die {elemental-product-name} Charts aus den heruntergeladenen Archiven installiert) und stattdessen die {elemental-product-name} Operator-Installation über die Rancher-Benutzeroberfläche durchführen.
======

Installation aus einem Hauler-Archiv::
+
======
Wenn die private Registry eine Authentifizierung erfordert, müssen Sie sich mit Hauler dort anmelden: 
[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD 
----

Das Skript gibt die zum Laden der Bilder erforderlichen Befehle aus. Sie sollte etwa so lauten: 

[,shell]
----
NEXT STEPS: 

* Load the 'elemental-haul.tar.zst' Haul archive in the Hauler instance in the airgapped infrastructure: 

hauler store load 'elemental-haul.tar.zst' 

* If the local registry in the air-gapped environment is not server by Hauler, load the Haul archive in the local registry: 

hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

Sobald das Betriebssystem und die Kanal-Images geladen sind, sollten Sie den Punkt (3) der Skriptausgabe überspringen (der die {elemental-product-name} Charts aus den heruntergeladenen Archiven installiert) und stattdessen die {elemental-product-name} Operator-Installation über die Rancher-Benutzeroberfläche durchführen.
======
========

== {elemental-product-name} UI-Erweiterung 

Rancher 2.7.x unterstützt keine UI-Erweiterungs-Plugins in Air-Gapped-Umgebungen, und daher ist die {elemental-product-name} UI in Rancher 2.7.x nicht verfügbar. 

Das {elemental-product-name} UI-Plugin wird in den verfügbaren UI-Erweiterungen in Rancher 2.8.0 enthalten sein.
