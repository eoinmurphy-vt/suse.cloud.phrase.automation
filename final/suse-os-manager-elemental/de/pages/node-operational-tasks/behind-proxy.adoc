= Hinter einem Proxy
:revdate: 2025-11-06
:page-revdate: {revdate}

:tabs-sync-option:

In vielen Unternehmensumgebungen haben Server oder VMs, die vor Ort laufen, keinen direkten Zugang zum Internet. Stattdessen wird die Verbindung zu externen Diensten aus Sicherheitsgründen über einen HTTP(S)-Proxy hergestellt. Dieses Tutorial zeigt Ihnen, wie Sie eine {elemental-product-name} Bereitstellung in einer solchen Umgebung einrichten.

[IMPORTANT]
====
Diese Anleitung behandelt nicht die Rancher-Installation hinter einem Proxy. Es handelt sich um einen anderen Anwendungsfall. Eine ausführliche Dokumentation finden Sie auf der Seite https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/http-proxy/http-proxy.html[Installation von SUSE Rancher Prime hinter einem HTTP-Proxy].
====


[NOTE]
====
Für diese Dokumentation gehen wir davon aus, dass Sie ein System der SUSE-Familie (wie SLE Micro) verwenden, daher müssen die Proxy-Einstellungen in `/etc/sysconfig/proxy` geschrieben werden.
====


Die Proxy-Einstellungen müssen an den folgenden Stellen konfiguriert werden:

* Endpunkt der Maschinenregistrierung
* SeedImage-Ressource
* {elemental-product-name} Clusterkonfiguration

Die `elemental-system-agent` benötigt Proxy-Einstellungen, um den Rancher Manager zu erreichen.
Dazu müssen Sie den Abschnitt cloud-init des Endpunkts für die Maschinenregistrierung ausfüllen.

Sie können dies entweder über die xref:quickstarts/quickstart-ui.adoc#_add_a_machine_registration_endpoint[Benutzeroberfläche] oder über xref:quickstarts/quickstart-cli.adoc#_prepare_your_kubernetes_resources[CLI] tun.

== {elemental-product-name}-Register

xref:rancher-os-management/architecture/architecture-components.adoc#_elemental_register_command_line_tool[{elemental-product-name}-Register] ist der erste Kommunikationsendpunkt zwischen dem neuen Host und dem Rancher-Manager, dies ist der erste Ort, an dem Proxy-Einstellungen gesetzt werden müssen.

[CAUTION]
====
Zum Zeitpunkt der Erstellung dieses Dokuments ist es nur möglich, die Proxy-Einstellungen für die ISO mit der CLI zu konfigurieren. Die Proxy-Einstellungen sind nicht in der Benutzeroberfläche implementiert.
====

Der Prozess findet statt, wenn Sie Ihre {elemental-product-name} ISO zum ersten Mal booten. Um die Proxy-Einstellungen zu konfigurieren, müssen Sie eine `cloud-init` Definition in die ISO aufnehmen.
Zu diesem Zweck müssen Sie eine xref:references/seedimage-reference.adoc[SeedImage-Definition] erstellen.

[,yaml]
----
include::example$proxy/seedimage-proxy.yaml[]
----

Wenden Sie die YAML mit `kubectl` an und drucken Sie dann Ihre SeedImage-Definition aus, um die URL zum Herunterladen zu erhalten:

[,bash]
----
kubectl apply -f <my_seedimage_yaml_file>
kubectl get seedimage <seed_image_name> -n <namespace> -o yaml
----

Booten Sie die ISO-Datei und Sie sollten sehen, dass Ihr neues System im xref:references/machineinventory-reference.adoc[Maschineninventar] erscheint.
