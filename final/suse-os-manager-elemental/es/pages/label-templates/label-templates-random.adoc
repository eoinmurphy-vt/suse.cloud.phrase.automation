= Variables de plantilla aleatorias
:revdate: 2025-09-08
:page-revdate: {revdate}

Las variables aleatorias de plantilla están incorporadas en el Operador Elemental.

Permiten incluir valores aleatorios `Int`, `Hex` o `UUID` en xref:label-templates/label-templates.adoc[plantillas de etiquetas] personalizadas.

Los valores se calculan sobre la marcha durante el renderizado de `label template variables`.

[IMPORTANT]
.Las plantillas de etiquetas aleatorias sólo se muestran una vez
====

Una plantilla de etiqueta que contiene una variable aleatoria sólo se muestra si el xref:references/machineinventory-reference.adoc[inventario de máquinas] del host de registro no tiene todavía un valor para esa etiqueta (falta una etiqueta con la misma clave o su valor está vacío).

Así, los tres casos en los que se representa una plantilla de etiqueta con una variable Aleatoria son:

. el host se registra por primera vez y el xref:references/machineinventory-reference.adoc[inventario de máquinas] se crea de nuevo.
. la plantilla de etiqueta se ha añadido al MachineRegistration después de que el host se (re)registrara la última vez.
. la etiqueta xref:references/machineinventory-reference.adoc[MachineInventory] que coincide con la plantilla de etiquetas (misma clave de etiqueta) se ha eliminado manualmente o se ha borrado su valor.
====


|===
| Variable | Descripción | de

| `${Random/UUID}`
| UUID aleatorio (por ejemplo, fd95324a-c26b-4e28-8727-1dcec293a0ec)
| 1.7.0

| `${Random/Hex/[1-32]}`
| cadena hexadecimal aleatoria de la longitud especificada (mín. 1, máx. 32)
| 1.7.0

| `${Random/Int/[MAXINT]`
| entero aleatorio (mín 0, máx MAXINT-1)
| 1.7.0
|===

[NOTE]
.Ejemplos de renderizado
====
|===
| valor de plantilla | ejemplo de valor renderizado

| `${Random/UUID}`
| fd95324a-c26b-4e28-8727-1dcec293a0ec

| `${Random/Hex/12}`
| acd231f222b8

| `${Random/Int/10000}` 
| 9432

|===
====


Las variables de plantilla aleatorias pueden ser útiles para generar nombres de host personalizados que se asignarán al host de registro.

Dado que el nombre de host debe ser único y se asigna a través del campo xref:references/machineregistration-reference.adoc[MachineRegistration] `spec.machineName`, se pueden utilizar variables aleatorias para garantizar la unicidad de un grupo de host que compartan el mismo prefijo y/o sufijo personalizado.

Consulte la sección xref:installation/hostname.adoc[Cómo personalizar el nombre de host] para obtener más información.
