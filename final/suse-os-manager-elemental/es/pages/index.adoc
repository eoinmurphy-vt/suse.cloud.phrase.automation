= Gestión del SO para Rancher
:revdate: 2026-01-16
:page-revdate: {revdate}

{elemental-product-name} es una pila de software que permite la gestión centralizada de sistemas operativos nativos de la nube con Kubernetes.

xref:release-notes.adoc[**Última versión: v1.8.0**]

La pila {elemental-product-name} consta de un puñado de paquetes sobre SLE Micro:

* *elemental-toolkit* - Incluye un conjunto de utilidades de SO para permitir la gestión de SO a través de contenedores. Incluye módulos dracut, configuración del cargador de arranque, servicios de configuración estilo cloud-init, etc.
* *elemental-operator* - Se conecta a Rancher Manager y maneja los CRDs MachineRegistration y MachineInventory.
* *elemental-register* - Registra máquinas a través de machineRegistrations y las instala a través de elemental-cli.
* *elemental-cli* - Instala cualquier derivado basado en elemental-toolkit. Básicamente un instalador basado en nuestro sistema de instalación y actualización A/B.
* *rancher-system-agent* - Se ejecuta en el sistema instalado y obtiene instrucciones ("Planes") de Rancher Manager sobre qué instalar y ejecutar en el sistema.

Los sistemas operativos de los nodos de clúster se construyen y mantienen mediante imágenes de contenedor a través de {elemental_cli_name} y pueden instalarse en nuevos hosts utilizando {elemental_ui_url}[{elemental_ui_name}] para https://www.rancher.com/products/rancher[Rancher Manager] o {elemental_cli_url}[{elemental_cli_name}].

El {elemental_operator_url}[{elemental_operator_name}] y el {ranchersystemagent_url}[{ranchersystemagent_name}] permiten a Rancher Manager controlar completamente los clusters {elemental-product-name}, desde la instalación y gestión del SO en los Nodos hasta el aprovisionamiento de nuevos K3s o clusters RKE2 de forma centralizada.

== {elemental-product-name} en hardware x86-64

{elemental-product-name} está listo para la producción y es totalmente compatible con x86-64 a partir de Rancher v2.7.0.

== {elemental-product-name} en hardware ARM

ARM (aarch64) es funcional en la fase de desarrollo. ARM actualmente sólo se prueba en Raspberry Pi 4 Modelo B con k3s 1.24.8 (o posterior). Los comentarios son bienvenidos.

== {elemental-product-name} en otro hardware

{elemental-product-name} se dirige actualmente a escenarios "periféricos" y, por tanto, no es compatible con otro hardware. Volveremos a evaluarlo a medida que evolucione el mercado.
