= Cómo añadir RPMs de terceros en el momento de la instalación
:revdate: 2025-09-08
:page-revdate: {revdate}

Este ejemplo se refiere al caso en que se incluyen RPMS adicionales (por ejemplo, controladores de hardware específicos) en la ISO y durante la instalación se instalan sobre la imagen del sistema operativo.

Para ese caso de uso se necesitan los siguientes archivos:

* RPMs adicionales para instalar
* archivo adicional cloud-config que define ganchos para copiar e instalar binarios como ganchos de instalación

Podemos manejarlo todo en una única sección `SeedImage.spec.cloud-config` suponiendo que el RPM adicional esté disponible para descargarlo desde un servidor remoto en el momento de la instalación.

Considere los siguientes datos cloud-config que podrían utilizarse como contenido de la sección `cloud-config` en un xref:references/seedimage-reference.adoc#_seedimagespec_reference[recurso SeedImage].

[,yaml]
----
name: "Install extra drivers"
stages:
  before-install:
  # Preload data to the persistent storage
  # During installation persistent partition is mounted at /run/elemental/persistent
  - downloads:
    - path: /tmp/some_package.rpm
      url: "<REMOTE_PACKAGE_URL>"
      permissions: 0777
  - commands:
    - mkdir -o /run/elemental/persistent/extra-pkgs
    - cp -p /tmp/some_package.rpm /run/elemental/persistent/extra-pkgs

  after-install-chroot:
  # Install the package at install time
  - commands:
    - rpm -iv /run/elemental/extra-pkgs/some_package.rpm

  # Include to the install system analog upgrade and reset hooks
  - files:
    - path: /oem/extra-pkg.yaml
      permissions: 0664
      content: |
        name: "Install extra drivers"
        stages:
          after-upgrade-chroot:
          # Install the package after upgrading to a new image
          - commands:
            - rpm -iv /run/elemental/extra-pkgs/some_package.rpm
          after-reset-chroot:
          # Install the package on reset
          - commands:
            - rpm -iv /run/elemental/extra-pkgs/some_package.rpm
----

Tenga en cuenta que los ganchos de instalación sólo cubren los procedimientos de instalación, por lo que los datos adicionales de cloud-config también deben formar parte del sistema instalado para poder seguir instalando el paquete como parte de los procesos de actualización o restablecimiento.

== Reempaquetado de una imagen ISO generada con archivos adicionales

Alternativamente, si la descarga dinámica de contenido en el momento de la instalación no es un comportamiento deseado y el objetivo es una ISO que ya incluya todos los binarios adicionales, esto también es posible, pero requiere volver a empaquetar manualmente la ISO. Usando `xorriso`, la utilidad de linux para crear ISOs, esto resulta ser un proceso relativamente fácil.

Vamos a crear un directorio `overlay` para crear el árbol de directorios que hay que añadir a la raíz ISO. En ese caso, el directorio `overlay` podría contener:

[,yaml]
----
overlay/
  data/
    extra-pkgs/
      some_package.rpm
  iso-config/
    install_hooks.yaml
----

Suponemos que `install_hooks.yaml` es el contenido de la configuración de la nube expuesta en la sección anterior, que se incluye manualmente en la imagen en lugar de incrustarse en un recurso SeedImage. La carpeta `data` incluye los binarios que queremos añadir a la ISO.

Suponiendo que ya descargamos una ISO Elemental vinculada a un MachineRegistration específico con la siguiente llamada `xorriso` todo el contenido de la carpeta `overlay` se incluiría en una nueva ISO:

[,bash]
----
xorriso -indev elemental.x86_64.iso -outdev elemental.custom.x86_64.iso -map overlay / -boot_image any replay
----

Requiere `xorriso` igual o superior a la versión 1.5.

El contenido de `install_hooks.yaml` podría ser eventualmente el mismo que el de la sección anterior pero omitiendo la descarga del paquete RPM y adaptando también la ruta del RPM a `/run/initramfs/live/data/extra-pkgs/some_package.rpm` ya que la carpeta raíz ISO está montada en `/run/initramfs/live`.
