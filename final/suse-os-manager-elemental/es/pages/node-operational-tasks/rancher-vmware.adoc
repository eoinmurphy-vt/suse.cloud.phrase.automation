= Cómo utilizar {elemental-product-name} con Rancher y VMware
:revdate: 2025-09-08
:page-revdate: {revdate}

== Extracto

En este documento veremos cómo configurar una máquina virtual en VMware workstation para arrancar nodos Elemental.

== Requisitos previos

Rancher 2.8 o superior instalado y funcionando. Consulte las guías de inicio rápido.

== Primer paso: Crear el punto final de registro

Necesitamos crear una imagen ISO para arrancar nodos y registrarlos contra la instancia de Rancher. Para ello se necesita un punto final de registroxref:references/machineregistration-reference.adoc[(] recursoxref:references/machineregistration-reference.adoc[MachineRegistration] ).

Véase este ejemplo de `MachineRegistration`:

[,yaml]
.registration.yaml
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: fire-nodes
  namespace: fleet-default
spec:
  config:
    cloud-config:
      users:
        - name: root
          passwd: root
    elemental:
      install:
        reboot: true
        device: /dev/sda
        debug: true
  machineInventoryLabels:
    element: fire
    manufacturer: "${Product/Vendor}"
    productName: "${Product/Name}"
    serialNumber: "${Product/Serial Number}"
    machineUUID: "${Product/UUID}"
----

El MachineRegistration anterior asume que los nodos incluyen TPM 2.0. En caso de que la máquina de destino virtualizada no incluya un dispositivo TPM virtual, se puede configurar una emulación de software en la sección `config.elemental.registration`.

Considere el siguiente ejemplo:

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  ...
spec:
  config:
    cloud-config:
      ...
    elemental:
      install:
        ...
      registration:
        emulate-tpm: true
        emulated-tpm-seed: -1
  machineInventoryLabels:
    ...
----

`emulated-tpm-seed: -1` establece que el cliente utilice una semilla aleatoria para calcular el hash TPM, esto es útil para ser capaz de reutilizar la misma definición de punto final de registro para múltiples máquinas. Consulte más xref:operator-operational-tasks/tpm.adoc[documentación sobre TPM].

== Segundo paso: Crear la ISO de instalación

El soporte de instalación debe estar vinculado a un punto final de registro específico. Se crea y gestiona con el recurso xref:references/seedimage-reference.adoc[SeedImage].

Considere el siguiente ejemplo:

[,yaml]
.seedimage.yaml
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: fire-img
  namespace: fleet-default
spec:
  baseImage: registry.suse.com/suse/sle-micro-iso/5.5:2.0.2
  registrationRef:
    apiVersion: elemental.cattle.io/v1beta1
    kind: MachineRegistration
    name: fire-nodes
    namespace: fleet-default
----

Una vez creado el recurso SeedImage, comienza a crear una ISO con la imagen del SO proporcionada y a vincularla al punto final de registro dado. Una vez hecho esto, aparecerá una URL de descarga en el estado del recurso SeedImage.

Puedes descargarlo con:

[,shell]
----
wget --no-check-certificate --content-disposition $(kubectl get seedimages.elemental.cattle.io -n fleet-default fire-img -o jsonpath="{.status.downloadURL}")
----

== Tercer paso: Arrancar el dispositivo de destino

Lo ideal sería grabar la iso en una unidad USB y arrancar el dispositivo de borde utilizando el dispositivo USB y una vez que arranca y se activa en Rancher en el inventario de la máquina se puede seleccionar y crear un clúster de la misma, sin embargo aquí vamos a utilizar una máquina virtual para imitar un dispositivo de borde para la prueba.

=== 3.1 Preparar la VM para emular el TPM

En VMware workstation cree una vm de la forma en que lo haría normalmente, asegúrese de darle al disco duro un tamaño de al menos 40 GB.

Ahora edite la configuración de la máquina y vaya a la pestaña "Opciones". La última opción sería "Avanzado".

Haga clic en "Avanzado" y en el panel derecho de la ventana cambie el tipo de firmware de "BIOS" a "UEFI" y marque la opción "Activar arranque seguro" como se indica a continuación:

* Ajustes por defecto con la BIOS seleccionada

image::rancher-vmware-vm-boot-bios.png[Opciones de arranque de VM con BIOS]

* Configuración actualizada con UEFI seleccionada y arranque seguro activado.

image::rancher-vmware-vm-boot-uefi.png[Opciones de arranque de máquinas virtuales con UEFI]

Ahora, en la misma pestaña "Opciones", haz clic en la opción "Control de acceso" y, a la derecha, en "Cifrar".

image::rancher-vmware-access-control-menu.png[Menú de control de acceso]

Esto le pedirá que introduzca una contraseña para cifrar la máquina. Introduzca una contraseña y haga clic en "Cifrar".

image::rancher-vmware-access-control-encrypt.png[Credenciales de encriptación del control de acceso]

Esto es importante para añadir el hardware TPM. A continuación, vuelva a las opciones de Hardware y haga clic en "Añadir".

Y añadir el hardware TPM (Trusted Platform Module) y haga clic en "Finalizar".

Ahora con la realización de este paso nuestra VM está lista.

=== 3.2 Arrancar la máquina virtual con la ISO elemental

A continuación, añada la ISO que hemos creado anteriormente en la VM e iníciela.

Debería arrancar con la ISO y empezar a instalar Elemental:

image::rancher-vmware-elemental-install-grub.png[{elemental-product-name} Menú grub de instalación del sistema operativo]

image::rancher-vmware-elemental-install-logs.png[{elemental-product-name} Registros de instalación del SO]

Y una vez completado reiniciará la VM y aparecerá como activa bajo el inventario de máquinas en Rancher.
