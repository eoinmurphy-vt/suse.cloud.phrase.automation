= Detrás de un proxy
:revdate: 2025-11-06
:page-revdate: {revdate}

:tabs-sync-option:

En muchos entornos empresariales, los servidores o máquinas virtuales que se ejecutan en las instalaciones no tienen acceso directo a Internet. En su lugar, la conexión a servicios externos se realiza a través de un proxy HTTP(S) por motivos de seguridad. Este tutorial muestra cómo configurar un despliegue de {elemental-product-name} en un entorno de este tipo.

[IMPORTANT]
====
Esta guía no cubrirá la instalación de Rancher detrás de un proxy. Es un caso de uso diferente y puede encontrar la documentación detallada en la página https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/http-proxy/http-proxy.html[Installing SUSE Rancher Prime behind an HTTP Proxy].
====


[NOTE]
====
Para esta documentación, suponemos que está utilizando un sistema de la familia SUSE (como SLE Micro), por lo que la configuración del proxy debe escribirse en `/etc/sysconfig/proxy`.
====


Los ajustes del proxy deben configurarse en las siguientes ubicaciones:

* Punto final de registro de máquinas
* Recurso SeedImage
* {elemental-product-name} configuración del clúster

El `elemental-system-agent` necesita configuración proxy para llegar al Rancher Manager.
Para ello, debe rellenar la sección cloud-init del punto final de registro de máquinas.

Puedes hacerlo con xref:quickstarts/quickstart-ui.adoc#_add_a_machine_registration_endpoint[UI] o xref:quickstarts/quickstart-cli.adoc#_prepare_your_kubernetes_resources[CLI].

== {elemental-product-name}-registro

xref:rancher-os-management/architecture/architecture-components.adoc#_elemental_register_command_line_tool[{elemental-product-name}-register] es el primer punto final de comunicación entre el nuevo host y Rancher Manager, este es el primer lugar donde se necesita establecer la configuración del proxy.

[CAUTION]
====
En el momento de escribir esto, sólo es posible configurar los ajustes de proxy para la ISO con la CLI. La configuración del proxy no está implementada en la interfaz de usuario.
====

El proceso ocurre cuando arrancas tu ISO de {elemental-product-name} por primera vez, para configurar los ajustes del proxy tienes que incluir una definición de `cloud-init` en la ISO.
Para ello, debe crear una definición de xref:references/seedimage-reference.adoc[SeedImage].

[,yaml]
----
include::example$proxy/seedimage-proxy.yaml[]
----

Aplica el YAML con `kubectl` y luego, imprime tu definición SeedImage para obtener la URL para descargarla:

[,bash]
----
kubectl apply -f <my_seedimage_yaml_file>
kubectl get seedimage <seed_image_name> -n <namespace> -o yaml
----

Arranque la ISO y debería ver que su nuevo sistema aparece en xref:references/machineinventory-reference.adoc[el inventario de máquinas].
