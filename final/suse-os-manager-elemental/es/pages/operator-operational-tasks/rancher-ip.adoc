= Configurar direcciones IP internas y externas de K3s y RKE2
:revdate: 2025-09-08
:page-revdate: {revdate}

K3s y RKE2 permiten especificar las direcciones IP interna y externa de un nodo a través de los parámetros `--node-ip` y `--node-external-ip`.

Rancher Provisioning permite rellenar esos parámetros recogiendo la dirección IP de la interfaz de red de cada nodo.

De todas formas, los métodos para recoger esas opciones ip para los Racimos de Rancheros _Personalizados_ y los Racimos de Rancheros _Elementales_ difieren.

== Configuración de direcciones IP internas y externas de K3s y RKE2 en el aprovisionamiento de Rancher Cluster.

El aprovisionamiento Rancher permite especificar en la sección Agente Rancher las interfaces de red que deben estar vinculadas a las direcciones IP internas y externas de cada nodo aprovisionado.

Esto se realiza añadiendo los nombres de las interfaces de red de las que deben extraerse las direcciones IP en `CATTLE_INTERNAL_ADDRESS` y las Variables de Entorno del Agente `CATTLE_ADDRESS` (https://ranchermanager.docs.rancher.com/reference-guides/cluster-configuration/rancher-server-configuration/use-existing-nodes/rancher-agent-options#ip-address-options[ver documentación de Rancher]).

[NOTE]
====
Las variables de entorno `CATTLE_INTERNAL_ADDRESS` y `CATTLE_ADDRESS` Agent pueden rellenarse directamente con las direcciones IP deseadas. De todas formas, como las direcciones IP internas y externas son _por nodo_, esto no funcionaría sino para clusters de un solo nodo.
====


[CAUTION]
====
Mientras que durante la creación de un Cluster Elemental es posible añadir Variables de Entorno de Agente, las `CATTLE_ADDRESS` y las `CATTLE_INTERNAL_ADDRESS` son ignoradas y no darían lugar a la configuración de las direcciones IP internas y externas de los nodos provisionados.
====


== Configurar direcciones IP internas y externas de K3s y RKE2 en Elemental Clusters

Elemental permite configurar las direcciones IP internas y externas de los Nodos Cluster adjuntando las etiquetas `elemental.cattle.io/InternalIP` y `elemental.cattle.io/ExternalIP` a los recursos de xref:references/machineinventory-reference.adoc[MachineInventory] que rastrean los nodos destino.

Estas etiquetas, cuando se adjuntan a un recurso xref:references/machineinventory-reference.adoc[MachineInventory], se utilizan para rellenar las direcciones IP internas y externas de los nodos asociados.

Las etiquetas pueden añadirse a los campos de xref:references/machineregistration-reference.adoc[MachineRegistration] `machineInventoryLabels`, utilizando como valores las variables de dirección IP de xref:label-templates/label-templates-network.adoc[la plantilla de etiquetas de red], para permitir recoger las direcciones IP de cada nodo.

Ejemplo: xref:references/machineregistration-reference.adoc[MachineRegistration] donde los nodos tendrán la dirección IP interna establecida desde la interfaz eth0 y la dirección IP externa desde eth1.
