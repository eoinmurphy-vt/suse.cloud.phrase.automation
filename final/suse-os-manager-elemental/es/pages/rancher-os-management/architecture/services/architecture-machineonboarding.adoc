= Incorporación de máquinas
:revdate: 2025-11-06
:page-revdate: {revdate}

El objetivo del servicio Machine Onboarding es registrar una nueva máquina en el catálogo {elemental-product-name} Operatorxref:references/machineinventory-reference.adoc[(] recursosxref:references/machineinventory-reference.adoc[MachineInventory] ) y aprovisionar el host con el SO, aplicando la configuración deseada.

La incorporación de la máquina implica los siguientes pasos:

* El usuario crea un recurso xref:references/machineregistration-reference.adoc[MachineRegistration]: el MachineRegistration incluye la configuración necesaria para la instalación del SO.
Tan pronto como se crea el MachineRegistration, el operador {elemental-product-name} expone un _punto final de registro_ HTTP: es el punto de entrada requerido por los hosts onboarding para registrarse en el operador.
* El usuario solicita una imagen autoinstalable a través de un recurso xref:references/seedimage-reference.adoc[SeedImage]: sólo requiere una referencia a un recurso MachineRegistration y a una imagen contenedora del sistema operativo compatible con {elemental-product-name} Toolkit.
En cuanto se crea la, el operador {elemental-product-name} activa el proceso de creación del recurso autoinstalable image: once completed, the URL is exposed in the `status.downloadURL` field of the xref:references/seedimage-reference.adoc[SeedImage].
* El usuario descarga la imagen autoinstalable y la utiliza para arrancar un host no aprovisionado: el host https://documentation.suse.com/cloudnative/os-manager/latest/en/rancher-os-management/authentication.html[se autentica] en el _punto final de registro_ del Operador {elemental-product-name}, obtiene la configuración completa almacenada en el MachineRegistration e instala el SO en el dispositivo de almacenamiento local. Tan pronto como el host ha completado el proceso de registro, el operador de {elemental-product-name} crea un recurso xref:references/machineinventory-reference.adoc[MachineInventory] único que rastrea el host.
La imagen autoinstalable puede utilizarse para integrar cualquier número de hosts.
