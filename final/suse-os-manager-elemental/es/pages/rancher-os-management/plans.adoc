= {elemental-product-name} Planes
:revdate: 2025-10-27
:page-revdate: {revdate}

{elemental-product-name} utiliza el https://github.com/rancher/system-agent[Rancher System Agent], renombrado a {elemental-product-name} System Agent, para arrancar inicialmente el nodo con un plan simple.

El plan aplicará las siguientes configuraciones:

* Establecer algunas etiquetas para el nodo
* Establezca el nombre de host adecuado de acuerdo con el valor `MachineInventory` 
* Instale el agente del sistema Rancher predeterminado desde el servidor Rancher e instale los componentes de Kubernetes adecuados.

El servicio de arranque también acepta planes locales almacenados en `/var/lib/elemental/agent/plans`. Cualquier plan escrito allí también se aplicará durante el arranque inicial del nodo una vez completada la instalación.

[TIP]
====
Los planes locales sólo se ejecutan durante el arranque inicial de {elemental-product-name} *antes de* que Kubernetes se instale en el nodo.
====

== Tipos de planes

Los tipos de planes que puede utilizar {elemental-product-name} son:

* Instrucciones únicas: Sólo se ejecuta una vez
* Instrucciones periódicas: Se ejecutan periódicamente
* Archivos: Crea archivos
* Sondas: sondas http

[TIP]
====
Tanto las instrucciones únicas como las periódicas pueden ejecutar un comando directo o una imagen docker.
====

== Añadir planes locales en {elemental-product-name}

Puede añadir planes locales a {elemental-product-name} como parte del CRD `MachineRegistration`, en la sección `cloud-config` de la siguiente manera:

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: my-nodes
  namespace: fleet-default
spec:
  config:
    cloud-config:
      users:
        - name: root
          passwd: root
      write_files:
        - path: /var/lib/elemental/agent/plans/mycustomplan.plan
          permissions: "0600"
          content: |
            {"instructions":
                [
                  {
                    "name":"set hostname",
                    "command":"hostnamectl",
                    "args": ["set-hostname", "myHostname"]
                  },
                  {
                    "name":"stop sshd service",
                    "command":"systemctl",
                    "args": ["stop", "sshd"]
                  }
                ]
            }
    elemental:
      install:
        reboot: true
        device: /dev/sda
        debug: true
  machineName: my-machine
  machineInventoryLabels:
    element: fire
----

== Ejemplos de planes

Los siguientes planes se ofrecen como referencia rápida y no se garantiza que funcionen en su entorno. Para obtener más información sobre los planes, consulte https://github.com/rancher/system-agent[Rancher System Agent].

[tabs]
======
Ejemplo 1: instrucciones únicas::
+
====
[,json]
----
{"instructions":
    [
        {
            "name":"set hostname",
            "command":"hostnamectl",
            "args": ["set-hostname", "myHostname"]
        },
        {
            "name":"stop sshd service",
            "command":"systemctl",
            "args": ["stop", "sshd"]
        }
    ]
}
----
====

Ejemplo 2: instrucciones periódicas::
+
====
[,json]
----
{"periodicInstructions":
    [
        {
            "name":"set hostname",
            "image":"ghcr.io/rancher-sandbox/elemental-example-plan:main"
            "command": "run.sh"
        }
    ]
}
----
====

Ejemplo 3: ficheros::
+
====
[,json]
----
{"files":
    [
        {
            "content":"Welcome to the system",
            "path":"/etc/motd",
            "permissions": "0644"
        }
    ]
}
----
====

Ejemplo 4: sondas::
+
====
[,json]
----
{"probes":
    "probe1": {
        "name": "Service Up",
        "httpGet": {
            "url": "http://10.0.0.1/healthz",
            "insecure": "false",
            "clientCert": "....",
            "clientKey": "....",
            "caCert": "....."
        }
    }
}
----
====
======
