= Trusted Platform Module 2.0 (TPM)
:revdate: 2024-09-24
:page-revdate: {revdate}

O Trusted Platform Module (TPM, também conhecido como ISO/IEC 11889) é um padrão internacional para um criptoprocessador seguro, um microcontrolador dedicado projetado para proteger o hardware por meio de chaves criptográficas integradas. O termo também pode se referir a um chip em conformidade com o padrão.

== Adicionar o módulo TPM à máquina virtual

A maneira fácil de adicionar o TPM à máquina virtual é usar o Libvirt com o Virt-manager

=== Criar máquina virtual

Depois de iniciar o virt-manager, crie uma nova máquina virtual

image::tpm1.png[Criar nova VM]

=== Verificar e editar a lista de módulos de hardware

Na tela de configuração de hardware, verifique a lista de módulos e clique em *_Adicionar hardware_* botão

image::tpm2.png[Lista de dispositivos]

=== Adicionar o módulo TPM à VM

Na lista de dispositivos emulados, escolha o módulo TPM e adicione-o à VM

image::tpm3.png[Adicionar módulo TPM]

=== Concluir a configuração da VM

Na última tela, verifique mais uma vez se o módulo TPM foi adicionado corretamente

image::tpm4.png[Verificar o TPM]

== Adicionar emulação de TPM à máquina bare metal

Durante a aplicação do `#!yaml MachineRegistration`, adicione a seguinte chave ao yaml `config:elemental:registration:emulate-tpm: true`

[IMPORTANT]
====
Se você planeja implantar mais de uma máquina com emulação de TPM, certifique-se de definir `config:elemental:registration:emulated-tpm-seed: -1`para que a semente usada para a emulação de TPM seja aleatória por máquina. Caso contrário, você obterá o mesmo hash de TPM para todas as máquinas implantadas e somente a última a ser registrada será válida.
