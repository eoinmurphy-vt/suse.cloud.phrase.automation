= Referência do ManagedOSVersionChannel
:revdate: 2025-09-08
:page-revdate: {revdate}

O recurso `ManagedOSVersionChannel` é responsável por definir o canal de imagem do sistema operacional.

Há várias chaves que podem ser configuradas em uma especificação de recurso `ManagedOSVersionChannel`.

.managedosversionchannel-example.yaml
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: ManagedOSVersionChannel
metadata:
  name: elemental-channel
  namespace: fleet-default
spec:
  options:
    image: registry.suse.com/rancher/elemental-channel:1.4.2
  syncInterval: 1h
  type: custom
----

== `ManagedOSVersionChannelSpec` referência

|===
| Chave | Tipo | Valor padrão | Descrição

| opções
| object
| nulo
| Define as informações opcionais que podem ser adicionadas em um canal do sistema operacional

| syncInterval
| string
| 1h
| Define quando sincronizar o canal do sistema operacional

| tipo
| string
| vazio
| Define o tipo de canal, somente `custom` é compatível agora

| upgradeContainer
| object
| nulo
| Um contêiner de upgrade que pode ser definido. Ver <<_upgradecontainer,referência>>

| deleteNoLongerInSyncVersions
| bool
| falso
| Exclua automaticamente as versões obsoletas do sistema operacional que não estão mais incluídas no canal

| habilitado
| bool
| verdadeiro
| Habilita esse canal. Permitir a sincronização de versões do sistema operacional.
|===

== `upgradeContainer`

Isso permite substituir o campo padrão `upgrade` dos planos do System Upgrade Controller (consulte xref:rancher-os-management/upgrade-lifecycle.adoc#_components[os componentes de upgrade]) com base nessa ManagedOSVersion.
Essas chaves são traduzidas pelo Controlador de atualização do sistema para uma especificação do Kubernetes https://kubernetes.io/docs/reference/kubernetes-api/workload-resources/pod-v1/#Container[de contêiner].
Esse é o contêiner responsável pela execução de uma atualização do sistema operacional.
