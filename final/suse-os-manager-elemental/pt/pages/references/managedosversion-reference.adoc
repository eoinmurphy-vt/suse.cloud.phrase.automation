= Referência de ManagedOSVersion
:revdate: 2025-09-08
:page-revdate: {revdate}

O recurso `ManagedOSVersion` é responsável por definir uma versão da imagem do sistema operacional que pode ser usada com os recursos `SeedImage` ou `ManagedOSImage`.

Há várias chaves que podem ser configuradas em uma especificação de recurso `ManagedOSVersion`.

.managedosversion-example.yaml
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: ManagedOSVersion
metadata:
  labels:
    elemental.cattle.io/channel: elemental-channel
  name: v2.0.2
  namespace: fleet-default
spec:
  metadata:
    displayName: SLE Micro
    upgradeImage: registry.suse.com/suse/sle-micro/5.5:2.0.2
  type: container
  version: v2.0.2
----

== `ManagedOSVersionSpec` referência

|===
| Chave | Tipo | Valor padrão | Descrição

| metadados
| object
| nulo
| Isso define alguns dados sobre a imagem do sistema operacional. Consulte [referência](#metadados)

| minVersion
| string
| nulo
| Não utilizado

| tipo
| string
| vazio
| Define o tipo de imagem do sistema operacional, que pode ser `container` ou `iso`

| versão
| string
| vazio
| Versão da imagem do sistema operacional

| upgradeContainer
| object
| nulo
| Um contêiner de upgrade que pode ser definido. Consulte [referência](#upgradecontainer)
|===

== metadados

Esta seção descreve as informações necessárias para definir uma imagem do sistema operacional em {elemental-product-name}.

Se `type` estiver definido como `container`:

|===
| Chave | Tipo | Valor padrão | Descrição

| displayName
| string
| vazio
| Nome da imagem do sistema operacional conforme visto na interface do usuário do Rancher

| upgradeImage
| string
| vazio
| Imagem do contêiner totalmente qualificada (referência OCI ou URI HTTP)
|===

Se `type` estiver definido como `iso`:

|===
| Chave | Tipo | Valor padrão | Descrição

| displayName
| string
| vazio
| Nome da imagem do sistema operacional conforme visto na interface do usuário do Rancher

| uri
| string
| vazio
| Imagem ISO totalmente qualificada
|===

== `upgradeContainer`

Isso permite substituir o campo padrão `upgrade` dos planos do System Upgrade Controller (consulte xref:rancher-os-management/upgrade-lifecycle.adoc#_components[os componentes de upgrade]) com base nessa ManagedOSVersion.
Essas chaves são traduzidas pelo Controlador de atualização do sistema para uma especificação do Kubernetes https://kubernetes.io/docs/reference/kubernetes-api/workload-resources/pod-v1/#Container[de contêiner].
Esse é o contêiner responsável pela execução de uma atualização do sistema operacional.
