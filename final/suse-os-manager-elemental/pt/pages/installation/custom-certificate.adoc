= Como adicionar um certificado personalizado
:revdate: 2024-09-24
:page-revdate: {revdate}

Pré-requisito: Um certificado no formato `.pem` 

Objetivo: Tornar um certificado personalizado disponível em todo o sistema

[NOTE]
.Isso se aplica a certificados usados por serviços em nível de sistema.
====
Em vez disso, as cargas de trabalho do Kubernetes devem trazer seus certificados para a imagem do contêiner.
====

Para instalar um certificado personalizado, precisamos

* Copie o arquivo `.pem` para `/etc/pki/trust/anchors/`
* executar `update-ca-certificates`

O respectivo trecho do site `cloud-config` tem a seguinte aparência:

[,yaml]
----
write_files:
  - path: /etc/pki/trust/anchors/my-custom-certificate.pem
    permission: 0444
    content: |-
      -----BEGIN CERTIFICATE-----
      ...
      -----END CERTIFICATE-----
runcmd:
  - update-ca-certificates
----

(o conteúdo real do certificado foi omitido por motivos de brevidade)
