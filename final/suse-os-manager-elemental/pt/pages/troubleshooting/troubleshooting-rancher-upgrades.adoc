= Solução de problemas de upgrades do Rancher
:revdate: 2025-07-03
:page-revdate: {revdate}

[WARNING]
====
A atualização para o Rancher v2.7.2 falhará se os clusters do {elemental-product-name} estiverem definidos. O pod do rancher fica preso em um loop de falha (consulte https://github.com/rancher/rancher/issues/41145).
====

Observe que o problema só ocorre se pelo menos um cluster {elemental-product-name} estiver definido.

Para contornar o problema, crie um recurso `dynamicschemas.management.cattle.io` vazio chamado `machineinventoryselectortemplate`:

[,shell]
----
kubectl apply -f - <<EOF
kind: DynamicSchema
apiVersion: management.cattle.io/v3
metadata:
  name: machineinventoryselectortemplate
EOF

----

O loop de falhas será interrompido e a atualização do Rancher será concluída com êxito.
