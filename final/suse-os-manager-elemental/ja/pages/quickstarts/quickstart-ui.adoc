= {elemental-product-name} 視覚的方法
:revdate: 2025-11-06
:page-revdate: {revdate}

[NOTE]
====
以下の手順は、少なくともRancher 2.9.xが必要です。
====

このクイックスタートでは、{elemental-product-name} プラグインとオペレータを既存の Rancher Manager インスタンスにデプロイする方法を紹介します。

インストールが完了すると、RKE2またはK3をベースにした新しい{elemental-product-name} クラスタをプロビジョニングできるようになる。

ただし、ステージング・オペレーターや開発オペレーターをインストールしたい場合は、xref:quickstarts/quickstart-cli.adoc#_non_stable_installations[CLIモードでのみ]可能です。

== 公式Rancher拡張機能リポジトリを追加する

Elemental 拡張機能が利用できない場合は、`Official Rancher Extensions Repository` を追加する必要があります：

image::quickstart-ui-extension-repository.png[Rancher Manager Extensionsリポジトリを追加する]

このリポジトリが Extensions UI 設定で追加できない場合は、https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-admin/helm-charts-in-rancher/helm-charts-in-rancher.html#_manage_repositories[] `https://github.com/rancher/ui-plugin-charts` `git` リポジトリをするか、または以下のリソースを適用することで手動で管理できます：

[source,yaml]
----
apiVersion: catalog.cattle.io/v1
kind: ClusterRepo
metadata:
  name: rancher-ui-charts
spec:
  gitBranch: main
  gitRepo: https://github.com/rancher/ui-plugin-charts
----

== elementalプラグインをインストールする

Rancher Manager Extensions Supportを有効にした後、`elemental` プラグインを以下のようにインストールできます：

* `Available` タブの下に`elemental` プラグインがあります。

image::quickstart-ui-extensions-available.png[Rancher Manager 利用可能なプラグイン]

[NOTE]
====
`Available` 」タブにエントリーがない場合は、ページを更新してください。`elemental` プラグインが表示されます。
====

* `Install` ボタンをクリックするとポップアップが表示されるので、`Install` をもう一度クリックして続ける。

image::quickstart-ui-elemental-plugin-install.png[{elemental-product-name} プラグインインストール]

* `Installed` 」タブに、「`elemental` 」プラグインが表示されました。

[NOTE]
====
`elemental` プラグインが表示され、ステータスが`Installing...` のままであれば、ページを更新してください。`elemental` プラグインは正しく表示されます。
====

`elemental` プラグインがインストールされると、Rancher Manager メニューに`OS Management` オプションが表示されます。表示されない場合は、ページを更新してください。

image::quickstart-ui-elemental-plugin-menu.png[Rancher Manager OS管理メニュー]

== マシン登録エンドポイントの追加

OS管理ダッシュボードで、`Create Registration Endpoint` ボタンをクリックする。

image::quickstart-ui-registration-endpoint-create.png[OS管理登録エンドポイント]

ここで、各詳細をそれぞれの場所に入力するか、YAMLとして編集してエンドポイントを一度に作成することができます。ここではすべてのフィールドを編集する。

image::quickstart-ui-registration-endpoint-create-details.png[UIによる登録エンドポイントの作成]

[IMPORTANT]
.主なオプション
====
`name: elemental-cluster1`必要に応じて変更してください。

`device-selector`:xref:references/machineregistration-reference.adoc#_config_elemental_install_device_selector[デバイス選択]フィールドは、インストール中に動的にデバイスを選択するために使用できます。このフィールドには、必要なデバイスを選択するためのルールのリストが含まれています。

`snapshotter`:OSイメージのスナップショットを管理するために使用されるデバイスのタイプ。
====

マシン登録エンドポイントを作成すると、アクティブとして表示されるはずです。

image::quickstart-ui-registration-endpoint-complete.png[登録エンドポイントに登録されたマシン]

== インストール（シード）イメージの準備

これは最後のステップで、初期登録設定を含むシードイメージを準備する必要があります。これにより、自動登録、インストール、クラスタの一部としての完全なデプロイが可能になります。ファイルの中身は、ノードの登録に必要な登録URLと適切なサーバー証明書だけで、安全に接続できる。

このシードイメージを使って、無限のマシンをプロビジョニングすることができる。

シードイメージは上記のKubernetesリソースとして作成され、`Build Media` ボタンを使ってビルドできるが、最初にISOまたはRAWイメージを選択する必要がある。

ISOが2つのデバイス（ISOをインストールしたデバイスと、{elemental-product-name} をインストールするもう1つのディスク）を必要とするのとは反対に、RAWイメージは1つのデバイスから起動し、デバイスに直接オペレーティングシステムをインストールすることができます。
RAWイメージにはブートパーティションとリカバリーパーティションしか含まれておらず、まずリカバリーモードで起動し、{elemental-product-name} （参考までに、プロセスはxref:node-operational-tasks/reset.adoc#_reset_workflow[リセットと]同様です）をインストールします。

image::quickstart-ui-registration-endpoint-build-media.png[登録エンドポイントにメディアを構築する]

ビルドが完了したら、`Download Media` ボタンを使ってメディアをダウンロードできる：

image::quickstart-ui-registration-endpoint-download-media.png[登録エンドポイントでメディアをダウンロードする]

このイメージでノードを起動すれば、ノードは起動する：

* 指定されたregistrationURLで登録し、マシンごとに作成する。 `MachineInventory`
* 指定されたデバイスにSLE Microをインストールする
* リブート

カスタムカラムは、`Machine Registration Endpoint` を作成する際に追加できる`Machine Inventory Labels` に基づいています：

image::quickstart-ui-registration-endpoint-hardware-labels.png[マシン登録エンドポイントハードウェアラベル]

以下のスクリーンショットでは、`Hardware Labels` がカスタム・カラムとして使用されている：

また、3つの点のメニューをクリックして、カスタム列を追加することもできます。

image::quickstart-ui-machine-inventory-custom-columns.png[機械在庫のカスタム列]

== 最初の{elemental-product-name} クラスタを作成する

では、これらの`Machine Inventory` を使って、`Create {elemental-product-name} Cluster` をクリックしてクラスターを作成してみよう：

image::quickstart-ui-create-cluster-button.png[{elemental-product-name} クラスタ作成ボタン]

{elemental-product-name} クラスタには、Kubernetes の K3s または RKE2 を選択できます。

image::quickstart-ui-create-cluster-standard-screen-.png[{elemental-product-name} クラスター作成画面]

ほとんどのオプションはランチャーから提供されている。
https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/k3s.html[]Rancherまたはhttps://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/rke2.html[Manager K3s ディストリビューション設定ドキュメント] Rancher参照してください。

しかし、`Inventory of Machines Selector Template` 。

事前に定義された`Machine Inventory Labels` を使用して{elemental-product-name} クラスターを作成するために、どの`Machine Inventory` を使用するかを選択できます：

image::quickstart-ui-create-cluster-machine-selector-template.png[機械在庫セレクタ・テンプレートを使用する]

3台のマシン・インベントリには、`AuthenticAMD` というキーとともに、`CPUVendor` というラベルが含まれているため、この3台のマシンを使用して、{elemental-product-name} クラスタを作成します。
