== 抜粋

このドキュメントでは、VMware workstationの仮想マシンをElementalノードのブートに設定する方法を説明します。

== 前提条件

Rancher 2.8以上がインストールされ、動作していること。クイックスタートガイドをご覧ください。

== ステップ1：登録エンドポイントの作成

ノードをブートストラップするためのISOイメージを作成し、Rancherインスタンスに対して登録する必要がある。そのためには、登録エンドポイントxref:references/machineregistration-reference.adoc[（MachineRegistration]リソース）が必要である。

`MachineRegistration` の例をご覧いただきたい：

[,yaml]
.registration.yaml
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: fire-nodes
  namespace: fleet-default
spec:
  config:
    cloud-config:
      users:
        - name: root
          passwd: root
    elemental:
      install:
        reboot: true
        device: /dev/sda
        debug: true
  machineInventoryLabels:
    element: fire
    manufacturer: "${Product/Vendor}"
    productName: "${Product/Name}"
    serialNumber: "${Product/Serial Number}"
    machineUUID: "${Product/UUID}"
----

上記のMachineRegistrationは、ノードにTPM 2.0が含まれていることを前提としている。仮想化対象マシンに仮想TPMデバイスが含まれていない場合は、`config.elemental.registration` 。

次の例を考えてみよう：

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  ...
spec:
  config:
    cloud-config:
      ...
    elemental:
      install:
        ...
      registration:
        emulate-tpm: true
        emulated-tpm-seed: -1
  machineInventoryLabels:
    ...
----

`emulated-tpm-seed: -1` これは、複数のマシンで同じ登録エンドポイント定義を再利用するのに便利である。詳しくはxref:operator-operational-tasks/tpm.adoc[TPMのドキュメントを]参照。

== ステップ2：インストールISOの作成

インストールメディアは、特定の登録エンドポイントに紐付けられる必要がある。これはxref:references/seedimage-reference.adoc[SeedImage]リソースで作成され、処理される。

次の例を考えてみよう：

[,yaml]
.seedimage.yaml
----
apiVersion: elemental.cattle.io/v1beta1
kind: SeedImage
metadata:
  name: fire-img
  namespace: fleet-default
spec:
  baseImage: registry.suse.com/suse/sle-micro-iso/5.5:2.0.2
  registrationRef:
    apiVersion: elemental.cattle.io/v1beta1
    kind: MachineRegistration
    name: fire-nodes
    namespace: fleet-default
----

SeedImageリソースが作成されると、提供されたOSイメージでISOの構築を開始し、指定された登録エンドポイントにリンクします。ダウンロードが完了すると、SeedImageリソースのステータスにダウンロードURLが表示されます。

でダウンロードできる：

[,shell]
----
wget --no-check-certificate --content-disposition $(kubectl get seedimages.elemental.cattle.io -n fleet-default fire-img -o jsonpath="{.status.downloadURL}")
----

== ステップ3：ターゲット・デバイスを起動する

理想的には、isoをUSBドライブに書き込み、USBデバイスを使ってエッジデバイスをブートし、ブートしてRancherのマシンインベントリでアクティブになれば、そこからクラスタを選択して作成することができるが、ここではvmを使ってエッジデバイスを模倣してテストする。

=== 3.1 TPMをエミュレートするVMの準備

VMwareワークステーションで通常と同じようにVMを作成し、HDDのサイズを少なくとも40GBにする。

マシンの設定を編集し、"Options"（オプション）タブを開く。最後のオプションは "Advanced "だ。

advanced "をクリックし、右側のウィンドウでファームウェアタイプを "BIOS "から "UEFI "に変更し、以下のように "Enable secure boot "オプションをチェックする：

* BIOS選択時のデフォルト設定

image::rancher-vmware-vm-boot-bios.png[BIOSによるVMブートオプション]

* UEFIを選択し、セキュアブートを有効にして設定を更新

image::rancher-vmware-vm-boot-uefi.png[UEFIによるVMブートオプション]

同じ "Options "タブで "Access Control "オプションをクリックし、右側の "Encrypt "をクリックする。

image::rancher-vmware-access-control-menu.png[アクセス・コントロール・メニュー]

マシンを暗号化するためのパスワードの入力を求められます。パスワードを入力し、"Encrypt "をクリックする。

image::rancher-vmware-access-control-encrypt.png[アクセス制御の暗号化認証情報]

これはTPMハードウェアを追加するために重要である。次にハードウェアのオプションに戻り、"Add "をクリックします。

そして、TPM（Trusted Platform Module）ハードウェアを追加し、"Finish "をクリックします。

このステップの完了で、VMの準備は整った。

=== 3.2 エレメンタルISOでVMをブートする

次に、先ほど作成したISOをVMに追加し、起動する。

ISOが起動し、Elementalのインストールが始まるはずだ：

image::rancher-vmware-elemental-install-grub.png[{elemental-product-name} OSインストールのグラブメニュー]

image::rancher-vmware-elemental-install-logs.png[{elemental-product-name} OSインストールログ]

完了するとVMが再起動し、Rancherのマシンインベントリにアクティブとして表示される。
