= プロキシの背後
:revdate: 2025-11-06
:page-revdate: {revdate}

:tabs-sync-option:

多くの企業環境では、構内で稼働するサーバーやVMはインターネットに直接アクセスできない。その代わり、外部サービスへの接続は、セキュリティ上の理由からHTTP(S)プロキシを介して行われる。このチュートリアルでは、このような環境で{elemental-product-name} のデプロイメントをセットアップする方法を紹介します。

[IMPORTANT]
====
このガイドでは、プロキシの背後にあるRancherのインストールはカバーしません。これは別の使用例で、詳細なドキュメントはhttps://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/http-proxy/http-proxy.html[Installing SUSE Rancher Prime behind an HTTP Proxy]のページにあります。
====


[NOTE]
====
このドキュメントでは、SUSEファミリーシステム(SLE Microなど)を使用していることを想定しているため、プロキシ設定は`/etc/sysconfig/proxy` に記述する必要があります。
====


プロキシ設定は以下の場所で行う必要がある：

* マシン登録エンドポイント
* SeedImageリソース
* {elemental-product-name} クラスタ構成

`elemental-system-agent` が Rancher Manager にアクセスするにはプロキシ設定が必要です。
そのためには、マシン登録エンドポイントのcloud-initセクションを埋める必要がある。

xref:quickstarts/quickstart-ui.adoc#_add_a_machine_registration_endpoint[UIでも] xref:quickstarts/quickstart-cli.adoc#_prepare_your_kubernetes_resources[CLIでも]できる。

== {elemental-product-name}-登録

xref:rancher-os-management/architecture/architecture-components.adoc#_elemental_register_command_line_tool[{elemental-product-name}-registerは]新しいホストとRancher Managerの間の最初の通信エンドポイントで、プロキシ設定が必要な最初の場所です。

[CAUTION]
====
この記事を書いている時点では、ISOのプロキシ設定はCLIでのみ可能です。プロキシ設定はUIに実装されていない。
====

このプロセスは、{elemental-product-name} ISOを初めて起動するときに起こります。プロキシ設定を行うには、ISOに`cloud-init` 定義を含める必要があります。
そのためには、xref:references/seedimage-reference.adoc[SeedImageの]定義を作成する必要がある。

[,yaml]
----
include::example$proxy/seedimage-proxy.yaml[]
----

YAML を`kubectl` で適用し、SeedImage 定義を印刷してダウンロード用の URL を取得します：

[,bash]
----
kubectl apply -f <my_seedimage_yaml_file>
kubectl get seedimage <seed_image_name> -n <namespace> -o yaml
----

ISOをブートすると、xref:references/machineinventory-reference.adoc[マシンのインベントリに]新しいシステムが表示されます。
