このガイドに従って、Rancherでバックアップから{elemental-product-name} 構成をリストアしてください。

== rancher-backupオペレータとリストア用バックアップファイルの準備

https://docs.ranchermanager.rancher.io/how-to-guides/new-user-guides/backup-restore-and-disaster-recovery/restore-rancher[]Rancherの公式参照し、`rancher-backup operator` がインストールされ、バックアップファイルにアクセスできることを確認する。

{elemental-product-name} リソースのバックアップを行うには、まずこのxref:operator-operational-tasks/backup.adoc[ドキュメントに]従わなければならない。

== rancher-backup operator を使って{elemental-product-name} の設定をリストアする。

バックアップtarballをリストアするために、`restore object` （ニーズに合わせて）を作成する。

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Restore
metadata:
  name: elemental-restore
spec:
  backupFilename: rancher-backup-430169aa-edde-4a61-85e8-858f625a755b-2022-10-17T05-15-00Z.tar.gz
----

Kubernetesにマニフェストを適用する。

[,shell]
----
kubectl apply -f elemental-restore.yaml
----

rancher-backupオペレータのログをチェックする。

[,shell]
----
kubectl logs -n cattle-resources-system -l app.kubernetes.io/name=rancher-backup -f
----

バックアップファイルが正常にリストアされたかどうかを確認します。

[,shell]
----
...
INFO[2022/10/31 06:34:50] Processing controllerRef apps/v1/deployments/rancher
INFO[2022/10/31 06:34:50] Done restoring
...
----

https://docs.ranchermanager.rancher.io/how-to-guides/new-user-guides/backup-restore-and-disaster-recovery/migrate-rancher-to-new-cluster[]Rancherのある手順を続ける。
