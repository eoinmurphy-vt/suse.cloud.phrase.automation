= K3sとRKE2の内部および外部IPアドレスの設定
:revdate: 2025-09-08
:page-revdate: {revdate}

K3s と RKE2 では、`--node-ip` と`--node-external-ip` パラメータでノードの内部 IP アドレスと外部 IP アドレスを指定できます。

Rancher Provisioningでは、各ノードのネットワークインターフェースからIPアドレスを収集し、これらのパラメータを埋めることができます。

いずれにせよ、_カスタムランチャークラスターと_ _エレメンタルランチャークラスターでは_、これらのipオプションを集める方法が異なる。

== RancherクラスタのプロビジョニングにおけるK3sとRKE2の内部および外部IPアドレスの設定

Rancher プロビジョニングでは、Rancher Agent セクションで、プロビジョニングされた各ノードの内部および外部 IP アドレスにバインドされるネットワークインターフェースを指定できます。

これは、`CATTLE_INTERNAL_ADDRESS` と`CATTLE_ADDRESS` Agent Environment Variables (https://ranchermanager.docs.rancher.com/reference-guides/cluster-configuration/rancher-server-configuration/use-existing-nodes/rancher-agent-options#ip-address-options[Rancher docsを参照])にIPアドレスを抽出するネットワークインターフェース名を追加することで実行される。

[NOTE]
====
`CATTLE_INTERNAL_ADDRESS` と`CATTLE_ADDRESS` エージェント環境変数には、希望するIPアドレスを直接入力できる。いずれにせよ、内部IPアドレスと外部IPアドレスは_ノード単位_なので、シングルノードクラスター以外では機能しないだろう。
====


[CAUTION]
====
Elementalクラスタの作成中にエージェント環境変数を追加することは可能ですが、`CATTLE_ADDRESS` および`CATTLE_INTERNAL_ADDRESS` のものは無視され、プロビジョニングされたノードの内部および外部IPアドレスの構成にはなりません。
====


== エレメンタルクラスターでK3sとRKE2の内部および外部IPアドレスを設定する

Elemental では、ターゲット・ノードを追跡するxref:references/machineinventory-reference.adoc[MachineInventory]リソースに`elemental.cattle.io/InternalIP` および`elemental.cattle.io/ExternalIP` ラベルを添付するクラスタ・ノードの内部および外部 IP アドレスを構成できます。

これらのラベルは、xref:references/machineinventory-reference.adoc[MachineInventory]リソースにアタッチされると、関連するノードの内部および外部IPアドレスを記入するために使用されます。

各ノードのIPアドレスを収集できるように、xref:label-templates/label-templates-network.adoc[ネットワーク・ラベル・テンプレート]IPアドレス変数を値として使用して、ラベルをxref:references/machineregistration-reference.adoc[MachineRegistration] `machineInventoryLabels` フィールドに追加できます。

例xref:references/machineregistration-reference.adoc[MachineRegistrationでは]、ノードの内部IPアドレスはインターフェースeth0から、外部IPアドレスはeth1から設定される。
