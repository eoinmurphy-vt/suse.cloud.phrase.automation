= バックアップ
:revdate: 2025-11-06

{elemental-product-name} は Rancher の一部として実行されるため、{elemental-product-name} のリソースは Rancher のバックアップにバンドルされる。
Rancherのバックアップ、リストア、ディザスタリカバリオプションの詳細については、https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/back-up-restore-and-disaster-recovery.html[] Rancherの公式ください。

== Rancher用のrancher-backupオペレータをインストールする。

https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/back-up.html[]Rancher、Rancherバックアップオペレーターのインストールと設定方法を学んでください。


== rancher-backup operator を使って{elemental-product-name} をバックアップする。

Kubernetesクラスタ上で動作するRancherをバックアップするために、`backup object` （ニーズに合わせて）を作成する。

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Backup
metadata:
  name: rancher-backup
spec:
  resourceSetName: rancher-resource-set
  schedule: "10 3 * * *"
  retentionCount: 10
----

rancher-backupオペレータは、スケジュール、暗号化、およびストレージクラスのためのいくつかのオプションを提供しています。
https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/configuration/backup.html[の公式ドキュメントを]読めば、すべてのオプションを調べることができる。

rancher-backupオペレータのログをチェックする。

[,shell]
----
kubectl logs -n cattle-resources-system -l app.kubernetes.io/name=rancher-backup -f
----

バックアップファイルが永続ボリュームに作成されたかどうかを確認します。
