= チャンネル
:revdate: 2025-09-30
:page-revdate: {revdate}

{elemental_operator_name} 、1つまたは複数のxref:references/managedosversionchannel-reference.adoc[ManagedOSVersionChannelへの]サブスクリプションを許可し、xref:references/seedimage-reference.adoc[SeedImageを]使用して新しいISOを構築したり、xref:references/managedosimage-reference.adoc[ManagedOSImageを]使用して既存の{elemental-product-name} ノードを新しいOSバージョンにアップグレードしたりするために消費できるxref:references/managedosversion-reference.adoc[ManagedOSVersionsの]リストを自動的に作成します。

チャンネルは通常、OCIコンテナ・イメージとして配布されるが、`ManagedOSVersion` のリストを含むJSONファイルのURIを直接参照することも可能である。ベストプラクティスは、画像を使用してチャンネルを分配することであり、その分配は、{elemental_operator_name} で必要とされる他のすべての画像と一貫性があることに留意されたい。これは例えば、エアギャップ環境で展開する場合などに有効だ。

== チャネルのライフサイクルとベストプラクティス

新しい`ManagedOSVersionChannel` が作成されると、{elemental_operator_name} はチャンネルから提供されたJSONリストを定期的に同期し、新しい`ManagedOSVersions` に変換する。 +
同期された`ManagedOSVersions` はすべて`ManagedOSVersionChannel` が所有する。`ManagedOSVersionChannel` を削除すると、カスケード上のすべての`ManagedOSVersions` が削除される。

`ManagedOSVersionChannel.spec.deleteNoLongerInSyncVersions` オプションが有効な場合、`ManagedOSVersionChannel` は、同期しなくなった`ManagedOSVersions` の自動クリーンアップをサポートしていることに注意。

`ManagedOSVersion` が削除されるとき、ファイナライザーは、`ManagedOSImage` の上にアクティブな参照がないことを確認します。

`ManagedOSVersion` を削除できない場合、どのリソースから参照されているかを調べることができる：

[,shell]
----
kubectl -n fleet-default get managedosimages -l elemental.cattle.io/managed-os-version-name=my-deleted-os-version
----

複数のチャンネルを使用する場合、適切なネーミング戦略を維持することが重要で、所有する`ManagedOSVersions` 。 +
どのチャンネルにも名前を付けることをお勧めします：`{BaseOS}-{BaseOSVersion}-{Flavor}`.

これによって、親チャンネルからベース OS とベース OS バージョンを参照しながら、`ManagedOSVersion` の名前を、特定の{elemental-product-name} ビルド・バージョンのイメージとして使用できるようになるはずです。 +

== 自分のチャンネルを作る

独自のカスタム・シンサーに必要なのは、JSONファイルを`/data/output` に出力し、正しいJSON構造を保持することだけだ。

このファイルは、ISOとコンテナのエントリーを含むJSON配列である。
配列の各エントリは、xref:references/managedosversion-reference.adoc[ManagedOSVersion]オブジェクトと 1 対 1 でマップされる。

`"type": "iso"` エントリーはブート可能な{elemental-product-name} ISOを含んでいなければならず、xref:references/seedimage-reference.adoc[SeedImageによって]使用される。一方、`"type": "container"` エントリーは{elemental-product-name} アップグレードのためにxref:references/managedosimage-reference.adoc[ManagedOSImageによって]使用される。

もし疑問があれば、https://github.com/rancher-sandbox/elemental-channels[elemental-channels]プロジェクトは、あなた自身のチャンネルを構築し、維持する方法についての参照実装として使用することができます。

新しいエントリーを作成する際には、他のチャンネルとの衝突を避けるために、選択したネーミング戦略に注意してください。他のチャンネルは、同じ名前で異なる`ManagedOSVersion` を同期してしまう可能性があるからです。
ベストプラクティスは、慣例を用いることである： `\{Flavor}-\{Version}-\{Type}`
