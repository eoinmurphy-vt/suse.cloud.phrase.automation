= {elemental-product-name} Plans
:revdate: 2025-10-27
:page-revdate: {revdate}

{elemental-product-name} utilise l' https://github.com/rancher/system-agent[agent système] Rancher, rebaptisé {elemental-product-name} System Agent, pour démarrer le nœud avec un plan simple.

Le plan appliquera les configurations suivantes :

* Définir des étiquettes pour le nœud
* Définir le nom d'hôte approprié en fonction de la valeur de `MachineInventory` 
* Installer l'agent système Rancher par défaut à partir du serveur Rancher, et installer les composants Kubernetes appropriés.

Le service bootstrap accepte également les plans locaux stockés sur `/var/lib/elemental/agent/plans`. Tout plan qui y est inscrit sera également appliqué lors du démarrage initial du nœud, une fois l'installation terminée.

[TIP]
====
Les plans locaux ne s'exécutent que pendant le démarrage initial de {elemental-product-name} *, avant que* Kubernetes ne soit installé sur le nœud.
====

== Types de régimes

Les types de plans que {elemental-product-name} peut utiliser sont les suivants :

* Instructions uniques : Ne fonctionne qu'une seule fois
* Instructions périodiques : Elles se déroulent périodiquement
* Fichiers : Création de fichiers
* Sondes : sondes http

[TIP]
====
Les instructions uniques et les instructions périodiques peuvent exécuter soit une commande directe, soit une image docker.
====

== Ajout de plans locaux sur {elemental-product-name}

Vous pouvez ajouter des plans locaux à {elemental-product-name} dans le cadre du CRD `MachineRegistration`, dans la section `cloud-config` comme suit :

[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineRegistration
metadata:
  name: my-nodes
  namespace: fleet-default
spec:
  config:
    cloud-config:
      users:
        - name: root
          passwd: root
      write_files:
        - path: /var/lib/elemental/agent/plans/mycustomplan.plan
          permissions: "0600"
          content: |
            {"instructions":
                [
                  {
                    "name":"set hostname",
                    "command":"hostnamectl",
                    "args": ["set-hostname", "myHostname"]
                  },
                  {
                    "name":"stop sshd service",
                    "command":"systemctl",
                    "args": ["stop", "sshd"]
                  }
                ]
            }
    elemental:
      install:
        reboot: true
        device: /dev/sda
        debug: true
  machineName: my-machine
  machineInventoryLabels:
    element: fire
----

== Exemples de plans

Les plans suivants sont fournis à titre de référence rapide et il n'est pas garanti qu'ils fonctionnent dans votre environnement. Pour en savoir plus sur les plans, veuillez consulter https://github.com/rancher/system-agent[System Agent] Rancher.

[tabs]
======
Exemple 1 : instructions ponctuelles::
+
====
[,json]
----
{"instructions":
    [
        {
            "name":"set hostname",
            "command":"hostnamectl",
            "args": ["set-hostname", "myHostname"]
        },
        {
            "name":"stop sshd service",
            "command":"systemctl",
            "args": ["stop", "sshd"]
        }
    ]
}
----
====

Exemple 2 : instructions périodiques::
+
====
[,json]
----
{"periodicInstructions":
    [
        {
            "name":"set hostname",
            "image":"ghcr.io/rancher-sandbox/elemental-example-plan:main"
            "command": "run.sh"
        }
    ]
}
----
====

Exemple 3 : fichiers::
+
====
[,json]
----
{"files":
    [
        {
            "content":"Welcome to the system",
            "path":"/etc/motd",
            "permissions": "0644"
        }
    ]
}
----
====

Exemple 4 : sondes::
+
====
[,json]
----
{"probes":
    "probe1": {
        "name": "Service Up",
        "httpGet": {
            "url": "http://10.0.0.1/healthz",
            "insecure": "false",
            "clientCert": "....",
            "clientKey": "....",
            "caCert": "....."
        }
    }
}
----
====
