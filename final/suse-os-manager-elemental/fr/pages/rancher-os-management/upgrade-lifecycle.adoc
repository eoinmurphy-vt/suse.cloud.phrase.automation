== Composants

* https://documentation.suse.com/cloudnative/os-manager/[{elemental-product-name} Opérateur]

L'opérateur {elemental-product-name} propose plusieurs façons de configurer la version du système d'exploitation au niveau du cluster. +
Chaque fois qu'un nouveau `ManagedOSIMage` est défini, une flotte connexe `Bundle` est également générée par l'opérateur {elemental-product-name}.

* https://documentation.suse.com/cloudnative/continuous-delivery/[La flotte]

Dans ce contexte, Fleet est utilisé pour appliquer une `upgrade.cattle.io/v1` `Plan` sur chaque grappe ciblée par la `ManagedOSImage`. +
Le site `Plan` est ensuite exécuté par le site `System Upgrade Controller` qui tourne sur le cluster {elemental-product-name} en aval.

* https://github.com/rancher/system-upgrade-controller[Contrôleur de mise à niveau du système]

Le contrôleur de mise à niveau du système est un outil de mise à niveau utilisé par https://documentation.suse.com/cloudnative/k3s/latest/en/upgrades/automated.html[] K3s et https://documentation.suse.com/cloudnative/rke2/latest/en/upgrade/automated_upgrade.html[] RKE2 clusters. Il doit toujours être installé automatiquement sur chaque cluster provisionné {elemental-product-name}. Dans ce contexte, le contrôleur de mise à niveau du système orchestre l'application de `upgrade.cattle.io/v1` `Plan` aux nœuds {elemental-product-name}.

== Version et cycle de vie des composants

Les versions des https://documentation.suse.com/cloudnative/continuous-delivery/[contrôleurs] et https://github.com/rancher/system-upgrade-controller[Fleet] sont contrôlées par Rancher.

Sur le cluster Rancher, vous pouvez exécuter :

[,bash]
----
kubectl get settings fleet-version system-upgrade-controller-chart-version
----

Pour plus d'informations sur les versions des graphiques, vous pouvez visiter le https://github.com/rancher/charts[dépôt] ou lire la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-admin/helm-charts-in-rancher/helm-charts-in-rancher.html[].

La version des graphiques est déterminée par Rancher. +
Selon la façon dont Rancher est installé, les variables d'environnement suivantes peuvent être définies, par exemple lors de l'installation de la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/references/helm-chart-options.html#_setting_extra_environment_variables[carte] Rancher:

[,bash]
----
CATTLE_SYSTEM_UPGRADE_CONTROLLER_CHART_VERSION
CATTLE_FLEET_VERSION
----

La mise à jour manuelle des versions n'est généralement pas recommandée. +
Il faut également savoir qu'une nouvelle adresse `CATTLE_SYSTEM_UPGRADE_CONTROLLER_CHART_VERSION` déclenchera une mise à jour de la carte `system-upgrade-controller` sur tous les clusters provisionnés en aval.

== Vérification de la réussite des mises à niveau

La vérification de l'exécution correcte des tâches de mise à niveau est un élément fondamental pour s'assurer que le processus de mise à niveau de {elemental-product-name} est complet.

Pour résoudre chaque étape du processus de mise à niveau, veuillez consulter le xref:troubleshooting/troubleshooting-upgrade.adoc[document correspondant.] +
Cette section décrit uniquement la dernière étape nécessaire pour vérifier l'application correcte du plan de mise à niveau sur un cluster ciblé.

* Versions supérieures ou égales à la *version 0.13.4* du site `system-upgrade-controller`:
+
[,shell]
----
kubectl -n cattle-system describe plan os-upgrader-my-upgrade
----
+
Une fois le plan appliqué à tous les nœuds, il aura un état `Complete`.

* Versions antérieures :
+
Chaque plan a une valeur d'état `latestHash` qui doit correspondre à l'étiquette du plan sur chaque nœud. +
Un simple script peut être utilisé pour dresser la liste de tous les nœuds où le plan n'a pas encore été appliqué :
+
[,shell]
----
#!/bin/bash

# This script prints a list of all nodes where an upgrade plan was not applied.
# To further determine the cause of failures, you can analyze the Plan status and the related jobs.
# For ex: kubectl -n cattle-system get plans,jobs

# Edit this variable according to your ManagedOSImage name.
MANAGED_OS_IMAGE_NAME=my-upgrade

PLAN_NAME=os-upgrader-$MANAGED_OS_IMAGE_NAME
LATEST_HASH=$(kubectl -n cattle-system get plan $PLAN_NAME -o=jsonpath='{.status.latestHash}')

printf "Plan '$PLAN_NAME' latest hash is: '$LATEST_HASH'\n"

PLAN_LABEL=plan.upgrade.cattle.io/$PLAN_NAME
printf "Listing all nodes with mismatching or missing label '$PLAN_LABEL':\n"
kubectl get nodes -l $PLAN_LABEL!=$LATEST_HASH
----
