Ce quickstart vous montrera comment déployer le plugin {elemental-product-name} et l'opérateur dans une instance existante de Rancher Manager.

Une fois installé, vous pourrez provisionner un nouveau cluster {elemental-product-name} basé sur RKE2 ou K3s.

Cependant, si vous souhaitez installer l'opérateur staging ou dev, vous ne pouvez le faire qu'en xref:quickstarts/quickstart-cli.adoc#_non_stable_installations[mode CLI.]

== Ajouter le dépôt officiel des extensions Rancher

Si l'extension Elemental n'est pas disponible, vous devez ajouter l'extension `Official Rancher Extensions Repository`:

image::quickstart-ui-extension-repository.png[Ajouter le référentiel Rancher Manager Extensions]

Si ce référentiel ne peut pas être ajouté via les paramètres de l'interface utilisateur des extensions, il peut être géré manuellement en référentiel `https://github.com/rancher/ui-plugin-charts` `git`  l'adresse https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-admin/helm-charts-in-rancher/helm-charts-in-rancher.html#_manage_repositories[ajoutant le], ou en appliquant la ressource suivante :

[source,yaml]
----
apiVersion: catalog.cattle.io/v1
kind: ClusterRepo
metadata:
  name: rancher-ui-charts
spec:
  gitBranch: main
  gitRepo: https://github.com/rancher/ui-plugin-charts
----

== Installer le plugin elemental

Après avoir activé le support des extensions du Rancher Manager, vous pouvez installer le plugin `elemental` comme suit :

* Sous l'onglet `Available`, vous verrez que le plugin `elemental` est disponible.

image::quickstart-ui-extensions-available.png[Rancher Manager Plugins disponibles]

[NOTE]
====
Si l'onglet `Available` n'affiche aucune entrée, actualisez la page. Le plugin `elemental` apparaît alors.
====

* Cliquez sur le bouton `Install`, une fenêtre contextuelle apparaît et cliquez à nouveau sur `Install` pour continuer.

image::quickstart-ui-elemental-plugin-install.png[{elemental-product-name} installation du plugin]

* Dans l'onglet `Installed`, le plugin `elemental` est maintenant listé.

[NOTE]
====
Si le plugin `elemental` est listé et que le statut reste à `Installing...`, actualisez la page. Le plugin `elemental` s'affichera correctement.
====

Une fois le plugin `elemental` installé, vous pouvez voir l'option `OS Management` dans le menu Rancher Manager, rafraîchissez la page si vous ne la voyez pas.

image::quickstart-ui-elemental-plugin-menu.png[Menu de gestion du système d'exploitation de Rancher Manager]

== Ajouter un point final d'enregistrement de machine

Dans le tableau de bord OS Management, cliquez sur le bouton `Create Registration Endpoint`.

image::quickstart-ui-registration-endpoint-create.png[Points finaux d'enregistrement de la gestion du système d'exploitation]

Ici, vous pouvez soit saisir chaque détail à son emplacement respectif, soit éditer ce fichier en tant que YAML et créer le point d'accès en une seule fois. Ici, nous allons modifier tous les champs.

image::quickstart-ui-registration-endpoint-create-details.png[Créer un point final d'enregistrement avec l'interface utilisateur]

[IMPORTANT]
.Principales options
====
`name: elemental-cluster1`: à modifier en fonction de vos besoins

`device-selector`: Le champ xref:references/machineregistration-reference.adoc#_config_elemental_install_device_selector["device-selector"] peut être utilisé pour sélectionner dynamiquement un appareil lors de l'installation. Le champ contient une liste de règles permettant de sélectionner l'appareil souhaité.

`snapshotter`: Type de dispositif utilisé pour gérer les instantanés dans les images du système d'exploitation.
====

== Préparation de l'image d'installation (semence)

C'est la dernière étape, vous devez préparer une image de départ qui inclut la configuration d'enregistrement initiale, de sorte qu'elle puisse être enregistrée automatiquement, installée et entièrement déployée dans le cadre de votre cluster. Le contenu du fichier n'est rien d'autre que l'URL d'enregistrement dont le nœud a besoin pour s'enregistrer et le certificat de serveur approprié, afin qu'il puisse se connecter en toute sécurité.

Cette image de départ peut ensuite être utilisée pour approvisionner un nombre infini de machines.

L'image de départ est créée en tant que ressource Kubernetes ci-dessus et peut être construite à l'aide du bouton `Build Media`, mais vous devez d'abord sélectionner une image ISO ou RAW.

Contrairement à l'ISO qui nécessite deux périphériques (un périphérique contenant l'ISO et un autre disque où installer {elemental-product-name}), l'image RAW permet de démarrer à partir d'un seul périphérique et d'y installer directement le système d'exploitation.
L'image RAW ne contient qu'une partition de démarrage et une partition de récupération et elle démarre d'abord en mode de récupération pour installer {elemental-product-name} (pour information, le processus est similaire à celui de la xref:node-operational-tasks/reset.adoc#_reset_workflow[réinitialisation] ).

image::quickstart-ui-registration-endpoint-build-media.png[Créer des médias dans les points finaux d'enregistrement]

Une fois la construction terminée, les médias peuvent être téléchargés à l'aide du bouton `Download Media`:

image::quickstart-ui-registration-endpoint-download-media.png[Télécharger les médias dans les points finaux d'enregistrement]

Vous pouvez maintenant démarrer vos nœuds avec cette image et ils le feront :

* S'enregistrer à l'aide de l'URL d'enregistrement indiquée et créer un fichier `MachineInventory`
* Installer SLE Micro sur l'appareil donné
* Reboot

== Créez votre premier cluster {elemental-product-name} 

Utilisons maintenant ces `Machine Inventory` pour créer une grappe en cliquant sur `Create {elemental-product-name} Cluster`:

image::quickstart-ui-create-cluster-button.png[Bouton Créer un cluster {elemental-product-name} ]

Pour votre cluster {elemental-product-name}, vous pouvez choisir K3s ou RKE2 pour Kubernetes.

image::quickstart-ui-create-cluster-standard-screen-.png[{elemental-product-name} Écran de création de grappes]

La plupart des options proviennent de Rancher, c'est pourquoi nous ne détaillerons pas toutes les possibilités.
N'hésitez pas à consulter la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/k3s.html[documentation de configuration de la distribution] Rancher ou la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/rke2.html[] Rancher si vous souhaitez en savoir plus.

Cependant, il est important de souligner la section `Inventory of Machines Selector Template`.

Il vous permet de choisir le site `Machine Inventory` que vous souhaitez utiliser pour créer votre cluster {elemental-product-name} en utilisant le site `Machine Inventory Labels` précédemment défini :

image::quickstart-ui-create-cluster-machine-selector-template.png[Utiliser le modèle de sélection de l'inventaire des machines]

Comme nos trois inventaires de machines contiennent l'étiquette `CPUVendor` avec la clé `AuthenticAMD`, les trois machines seront utilisées pour créer le cluster {elemental-product-name}.
