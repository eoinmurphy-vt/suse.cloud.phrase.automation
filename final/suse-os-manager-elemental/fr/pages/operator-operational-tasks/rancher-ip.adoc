= Configurer les adresses IP internes et externes de K3s et RKE2
:revdate: 2025-09-08
:page-revdate: {revdate}

K3s et RKE2 permettent de spécifier les adresses IP internes et externes d'un nœud via les paramètres `--node-ip` et `--node-external-ip`.

Rancher Provisioning permet de renseigner ces paramètres en collectant l'adresse IP de l'interface réseau de chaque nœud.

Quoi qu'il en soit, les méthodes de collecte de ces options d'ip pour les _Custom_ Rancher Clusters et les _Elemental_ Rancher Clusters sont différentes.

== Configuration des adresses IP internes et externes des K3s et RKE2 dans le provisionnement du cluster Rancher

Le provisionnement Rancher permet de spécifier dans la section Rancher Agent les interfaces réseau qui doivent être liées aux adresses IP internes et externes de chaque nœud provisionné.

Pour ce faire, il faut ajouter les noms des interfaces réseau à partir desquelles les adresses IP doivent être extraites dans les variables d'environnement de l'agent `CATTLE_INTERNAL_ADDRESS` et `CATTLE_ADDRESS` (https://ranchermanager.docs.rancher.com/reference-guides/cluster-configuration/rancher-server-configuration/use-existing-nodes/rancher-agent-options#ip-address-options [voir la documentation de Rancher]).

[NOTE]
====
Les variables d'environnement `CATTLE_INTERNAL_ADDRESS` et `CATTLE_ADDRESS` Agent peuvent être directement remplies avec les adresses IP souhaitées. Quoi qu'il en soit, étant donné que les adresses IP internes et externes sont attribuées _par nœud_, cela ne fonctionnerait que pour les clusters à nœud unique.
====


[CAUTION]
====
Bien qu'il soit possible d'ajouter des variables d'environnement d'agent lors de la création d'un cluster élémentaire, les variables `CATTLE_ADDRESS` et `CATTLE_INTERNAL_ADDRESS` sont ignorées et n'entraînent pas la configuration des adresses IP internes et externes des nœuds provisionnés.
====


== Configurer les adresses IP internes et externes des K3 et des RKE2 dans les clusters élémentaires

Elemental permet de configurer les adresses IP internes et externes des nœuds de cluster en attachant les étiquettes `elemental.cattle.io/InternalIP` et `elemental.cattle.io/ExternalIP` aux ressources xref:references/machineinventory-reference.adoc[MachineInventory] qui suivent les nœuds cibles.

Ces étiquettes, lorsqu'elles sont attachées à une ressource xref:references/machineinventory-reference.adoc[MachineInventory,] sont utilisées pour remplir les adresses IP internes et externes des nœuds associés.

Les étiquettes peuvent être ajoutées aux champs xref:references/machineregistration-reference.adoc[MachineRegistration] `machineInventoryLabels`, en utilisant les variables d'adresse IP du xref:label-templates/label-templates-network.adoc[modèle d'étiquette de réseau] comme valeurs, afin de permettre la collecte des adresses IP de chaque nœud.
