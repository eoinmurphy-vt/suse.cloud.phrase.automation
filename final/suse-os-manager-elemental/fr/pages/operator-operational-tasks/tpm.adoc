= Module de plateforme de confiance 2.0 (TPM)
:revdate: 2024-09-24
:page-revdate: {revdate}

Trusted Platform Module (TPM, également connu sous le nom d'ISO/IEC 11889) est une norme internationale pour un cryptoprocesseur sécurisé, un microcontrôleur dédié conçu pour sécuriser le matériel grâce à des clés cryptographiques intégrées. Le terme peut également faire référence à une puce conforme à la norme.

== Ajouter un module TPM à la machine virtuelle

La façon la plus simple d'ajouter un TPM à une machine virtuelle est d'utiliser Libvirt avec Virt-manager.

=== Créer une machine virtuelle

Après avoir démarré virt-manager, créer une nouvelle machine virtuelle

image::tpm1.png[Créer une nouvelle VM]

=== Vérifier et modifier la liste des modules matériels

Sur l'écran de configuration du matériel, vérifiez la liste des modules et cliquez sur *_Ajouter du matériel_* et cliquez sur le bouton Ajouter du matériel

image::tpm2.png[Liste des appareils]

=== Ajouter un module TPM à la VM

Dans la liste des périphériques émulés, choisissez le module TPM et ajoutez-le à la VM.

image::tpm3.png[Ajouter un module TPM]

=== Terminer la configuration de la VM

Sur le dernier écran, vérifiez une nouvelle fois que le module TPM a été ajouté correctement

image::tpm4.png[Vérifier le TPM]

== Ajouter l'émulation TPM à une machine bare metal

Lors de l'application de `#!yaml MachineRegistration`, ajoutez la clé suivante au fichier yaml `config:elemental:registration:emulate-tpm: true`

[IMPORTANT]
====
Si vous prévoyez de déployer plus d'une machine avec l'émulation TPM, assurez-vous de définir `config:elemental:registration:emulated-tpm-seed: -1`de sorte que la graine utilisée pour l'émulation TPM soit randomisée pour chaque machine. Sinon, vous obtiendrez le même hachage TPM pour toutes les machines déployées et seul le dernier à être enregistré sera valide.
