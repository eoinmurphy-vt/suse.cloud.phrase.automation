= Sauvegarde
:revdate: 2025-11-06
:page-revdate: {revdate}

Comme {elemental-product-name} fonctionne dans le cadre de Rancher, les ressources de {elemental-product-name} sont regroupées dans la sauvegarde de Rancher.
Pour plus de détails sur les options de sauvegarde, de restauration et de reprise après sinistre de Rancher, veuillez consulter la officielle https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/back-up-restore-and-disaster-recovery.html[documentation] Rancher.

== Installer l'opérateur rancher-backup pour Rancher

Suivez le https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/back-up.html[guide] Rancher pour savoir comment installer et configurer l'opérateur de sauvegarde Rancher.


== Sauvegarde {elemental-product-name} avec l'opérateur rancher-backup

Créez un site `backup object` (adapté à vos besoins) pour sauvegarder Rancher fonctionnant sur un cluster Kubernetes.

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Backup
metadata:
  name: rancher-backup
spec:
  resourceSetName: rancher-resource-set
  schedule: "10 3 * * *"
  retentionCount: 10
----

L'opérateur rancher-backup offre plusieurs options de planification, de cryptage et de classes de stockage.
Vous pouvez explorer toutes les options en lisant la https://documentation.suse.com/cloudnative/rancher-manager/latest/en/rancher-admin/back-up-restore-and-disaster-recovery/configuration/backup.html[documentation officielle de].

Vérifier les journaux de l'opérateur rancher-backup.

[,shell]
----
kubectl logs -n cattle-resources-system -l app.kubernetes.io/name=rancher-backup -f
----

Vérifier si le fichier de sauvegarde a été créé sur le volume persistant.
