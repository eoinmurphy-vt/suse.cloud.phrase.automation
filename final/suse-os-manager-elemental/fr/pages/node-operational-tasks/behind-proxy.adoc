= Derrière un proxy
:revdate: 2025-11-06
:page-revdate: {revdate}

:tabs-sync-option:

Dans de nombreux environnements d'entreprise, les serveurs ou les machines virtuelles fonctionnant dans les locaux n'ont pas d'accès direct à l'internet. Pour des raisons de sécurité, la connexion aux services externes se fait par l'intermédiaire d'un proxy HTTP(S). Ce tutoriel vous montre comment mettre en place un déploiement {elemental-product-name} dans un tel environnement.

[IMPORTANT]
====
Ce guide ne couvre pas l'installation de Rancher derrière un proxy. Il s'agit d'un cas d'utilisation différent et vous pouvez trouver la documentation détaillée sur la page https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/http-proxy/http-proxy.html[Installing SUSE Rancher Prime behind an HTTP Proxy (Installation de SUSE Rancher Prime derrière un proxy HTTP] ).
====


[NOTE]
====
Pour cette documentation, nous supposons que vous utilisez un système de la famille SUSE (comme SLE Micro), les paramètres du proxy doivent donc être écrits dans `/etc/sysconfig/proxy`.
====


Les paramètres du proxy doivent être configurés aux endroits suivants :

* Point final de l'enregistrement de la machine
* Ressource SeedImage
* {elemental-product-name} configuration de la grappe

Le site `elemental-system-agent` a besoin de paramètres de proxy pour accéder au Rancher Manager.
Pour ce faire, vous devez remplir la section cloud-init du point final d'enregistrement de la machine.

Vous pouvez le faire avec l'xref:quickstarts/quickstart-ui.adoc#_add_a_machine_registration_endpoint[interface utilisateur] ou le xref:quickstarts/quickstart-cli.adoc#_prepare_your_kubernetes_resources[CLI].

== {elemental-product-name}-enregistrer

xref:rancher-os-management/architecture/architecture-components.adoc#_elemental_register_command_line_tool[{elemental-product-name}-register] est le premier point de communication entre le nouvel hôte et Rancher Manager, c'est le premier endroit où les paramètres du proxy doivent être définis.

[CAUTION]
====
Au moment de la rédaction de ce document, il n'est possible de configurer les paramètres de proxy pour l'ISO qu'à l'aide de l'interface de gestion. Les paramètres du proxy ne sont pas implémentés dans l'interface utilisateur.
====

Le processus se déroule lorsque vous démarrez votre ISO {elemental-product-name} pour la première fois, afin de configurer les paramètres du proxy, vous devez inclure une définition `cloud-init` dans l'ISO.
Pour ce faire, vous devez créer une définition de xref:references/seedimage-reference.adoc[SeedImage].

[,yaml]
----
include::example$proxy/seedimage-proxy.yaml[]
----

Appliquez le YAML avec `kubectl` et ensuite, imprimez votre définition de SeedImage pour obtenir l'URL pour la télécharger :

[,bash]
----
kubectl apply -f <my_seedimage_yaml_file>
kubectl get seedimage <seed_image_name> -n <namespace> -o yaml
----

Démarrez l'ISO et vous devriez voir votre nouveau système apparaître dans l'xref:references/machineinventory-reference.adoc[inventaire des machines.]
