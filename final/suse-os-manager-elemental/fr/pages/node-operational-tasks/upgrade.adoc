Tous les composants de {elemental-product-name} sont gérés à l'aide de Kubernetes. Voici comment utiliser les approches Kubernetes pour mettre à niveau les composants.

== {elemental-product-name} mise à niveau des nœuds

{elemental-product-name} Les nœuds sont mis à niveau à l'aide du site {elemental_operator_name}. Se référer à laxref:references/operatorchart-reference.adoc[{elemental_operator_name}] pour obtenir des informations complètes.

La mise à jour peut être effectuée soit par le CLI, soit par l'interface utilisateur :

* <<_upgrade_via_command_line_interface,Interface de ligne de commande>>
* <<_upgrade_via_user_interface,Interface utilisateur>>

== Mise à niveau via l'interface de ligne de commande

Il existe deux façons de sélectionner les nœuds à mettre à niveau. Via une cible de grappe, qui correspondra à TOUS les nœuds d'une grappe correspondant à notre sélecteur ou via un sélecteur de nœuds, qui correspondra aux nœuds sur la base des étiquettes de nœuds. Le sélecteur de nœuds nous permet de mieux cibler la mise à niveau, tandis que le sélecteur de grappes sélectionne simplement tous les nœuds d'une grappe donnée.

La mise à jour d'un site `ManagedOSImage` existant déclenchera un nouveau cycle de mise à niveau, afin de rapprocher l'image configurée (ou la version de l'image) de tous les nœuds ciblés.

=== Gestion des versions disponibles

Il est possible de créer directement des xref:references/managedosversion-reference.adoc[ManagedOSVersions] ou de s'abonner à des xref:references/managedosversionchannel-reference.adoc[ManagedOSVersionChannels] pour synchroniser automatiquement `ManagedOSVersions` à partir de ceux-ci.

Pour plus de détails et une liste des canaux disponibles, ou même pour créer votre propre canal, veuillez lire la xref:operator-operational-tasks/channels.adoc[documentation].

== Mise à niveau via l'interface utilisateur

Pour effectuer la mise à jour via l'interface utilisateur, vous devez aller dans le menu {elemental-product-name} Advanced, puis cliquer sur `Update Groups`.

Choisissez un nom, sélectionnez les clusters à cibler et choisissez entre les deux modes de mise à niveau :

image::upgrade-ui-menu.png[{elemental-product-name} Menu de mise à niveau]

[tabs]
======
Via Managed OS Version::
+
====
Dans ce cas, un site `OS Version Channels` est utilisé pour alimenter automatiquement les ressources `OS Versions`. 

La chaîne ci-dessous est fournie par nos soins mais vous pouvez également apporter votre propre chaîne. 

Voir la section <<_managing_available_versions,Gestion des versions disponibles>> pour comprendre comment `ManagedOSVersion` est géré. 

image:upgrade-ui-version-channel.png[Créer un canal de version du système d'exploitation] 

Après une courte période de synchronisation, vous verrez votre `OS Versions` apparaître dans le menu `OS Versions`. 

image:upgrade-ui-os-version.png[{elemental-product-name} Menu Version du système d'exploitation] 

Enfin, vous pouvez sélectionner le site `OS Versions` lorsque vous créez votre site `Upgrade Group`, comme le montre la capture d'écran suivante : 

image:upgrade-ui-upgrade-group-channel.png[Sélectionner la version du système d'exploitation dans le groupe de mise à niveau]
====

Onglet Via Image à partir du registre::
+
====
Il suffit de spécifier une image OCI dans le champ `Image path` pour effectuer la mise à niveau : image:upgrade-ui-image-registry.png[Mise à niveau via le registre d'images]
====
======

Cliquez sur le bouton `Create` pour lancer le processus de mise à niveau. Si vous avez plusieurs nœuds, la mise à niveau sera effectuée de manière séquentielle, nœud par nœud.
