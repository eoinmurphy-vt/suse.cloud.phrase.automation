= 如何在安装时添加第三方 RPM
:revdate: 2025-09-08
:page-revdate: {revdate}

本示例涉及的情况是 ISO 中包含了额外的 RPMS（如特定硬件驱动程序），在安装过程中将它们安装到操作系统映像上。

在这种情况下，需要使用以下文件：

* 需要安装的其他 RPM
* 附加云配置文件，定义复制和安装二进制文件的钩子，作为安装钩子

如果在安装时可以从远程服务器下载额外的 RPM，我们就可以将其全部处理到一个`SeedImage.spec.cloud-config` 部分。

考虑将以下云配置数据用作xref:references/seedimage-reference.adoc#_seedimagespec_reference[种子图像资源] `cloud-config` 部分的内容。

[,yaml]
----
name: "Install extra drivers"
stages:
  before-install:
  # Preload data to the persistent storage
  # During installation persistent partition is mounted at /run/elemental/persistent
  - downloads:
    - path: /tmp/some_package.rpm
      url: "<REMOTE_PACKAGE_URL>"
      permissions: 0777
  - commands:
    - mkdir -o /run/elemental/persistent/extra-pkgs
    - cp -p /tmp/some_package.rpm /run/elemental/persistent/extra-pkgs

  after-install-chroot:
  # Install the package at install time
  - commands:
    - rpm -iv /run/elemental/extra-pkgs/some_package.rpm

  # Include to the install system analog upgrade and reset hooks
  - files:
    - path: /oem/extra-pkg.yaml
      permissions: 0664
      content: |
        name: "Install extra drivers"
        stages:
          after-upgrade-chroot:
          # Install the package after upgrading to a new image
          - commands:
            - rpm -iv /run/elemental/extra-pkgs/some_package.rpm
          after-reset-chroot:
          # Install the package on reset
          - commands:
            - rpm -iv /run/elemental/extra-pkgs/some_package.rpm
----

请注意，安装钩子只涵盖安装程序，因此额外的云配置数据也应是已安装系统的一部分，以便在升级或重置过程中继续安装软件包。

== 用额外文件重新打包生成的 ISO 映像

另外，如果不希望在安装时动态下载内容，而希望 ISO 已经包含所有额外的二进制文件，这也是可行的，但需要手动重新打包 ISO。使用`xorriso` 这个 linux 工具来创建 ISO，过程相对简单。

让我们创建`overlay` 目录，创建需要添加到 ISO 根目录的目录树。在这种情况下，`overlay` 目录可以包含：

[,yaml]
----
overlay/
  data/
    extra-pkgs/
      some_package.rpm
  iso-config/
    install_hooks.yaml
----

我们假定`install_hooks.yaml` 是上一节中公开的实际云配置内容，它是手动包含在图像中的，而不是嵌入在种子图像资源中的。`data` 文件夹最终会包含我们要添加到 ISO 中的二进制文件。

假设我们已经下载了一个与特定机器注册绑定的 Elemental ISO，并调用了以下`xorriso` ，那么所有`overlay` 文件夹的内容都将包含在新的 ISO 中：

[,bash]
----
xorriso -indev elemental.x86_64.iso -outdev elemental.custom.x86_64.iso -map overlay / -boot_image any replay
----

要求`xorriso` 等于或高于 1.5 版。

`install_hooks.yaml` 的内容最终可能与上一节相同，但省略了 RPM 软件包的下载，并将 RPM 路径调整为`/run/initramfs/live/data/extra-pkgs/some_package.rpm` ，因为 ISO 根文件夹已加载到`/run/initramfs/live` 。
