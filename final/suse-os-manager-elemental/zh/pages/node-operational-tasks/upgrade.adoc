== 通过命令行界面升级

选择升级节点有两种方法。通过集群目标，它将匹配集群中与我们的选择器相匹配的所有节点；或者通过节点选择器，它将根据节点标签匹配节点。节点选择器可以让我们更有针对性地进行升级，而集群选择器则只选择匹配集群中的所有节点。

更新现有`ManagedOSImage` 会触发一个新的升级周期，以便将配置的映像（或映像版本）与所有目标节点进行核对。

== 通过用户界面升级

要通过用户界面升级，必须进入{elemental-product-name} 高级菜单，然后点击`Update Groups` 。

选择名称，选择目标群集，并选择两种升级方式：

image::upgrade-ui-menu.png[{elemental-product-name} 升级菜单]

[tabs]
======
通过管理操作系统版本::
+
====
在这种情况下，`OS Version Channels` ，自动填充`OS Versions` 资源。

下面的频道由我们提供，但您也可以自带频道。

请参阅 "<<_managing_available_versions,管理可用版本>>"一节，了解如何管理`ManagedOSVersion` 。

image:upgrade-ui-version-channel.png[创建操作系统版本通道]

经过短暂的同步后，您将看到`OS Versions` 出现在`OS Versions` 菜单中。

image:upgrade-ui-os-version.png[{elemental-product-name} 操作系统版本菜单]

最后，您可以在创建`Upgrade Group` 时根据以下截图选择`OS Versions` ：

image:upgrade-ui-upgrade-group-channel.png[在升级组中选择操作系统版本]
====

选项卡 通过注册表中的图像::
+
====
只需在`Image path` 字段中指定要升级到的 OCI 映像：image:upgrade-ui-image-registry.png[通过图像注册表升级]
====
======

单击`Create` 按钮开始升级过程，如果有多个节点，升级将按节点顺序进行。
