= 机器清单选择器模板参考
:revdate: 2025-09-08
:page-revdate: {revdate}

机器清单选择器模板（MachineInventorySelectorTemplate）是用户定义的资源，将用作创建所需xref:references/machineinventoryselector-reference.adoc[机器清单选择器]的蓝图。

它是负责定义匹配标准的资源，以便将库存机器与群集资源配对。

相关关键字是`selector` ，其中包括标签选择器表达式。

.MachineInventorySelectorTemplate
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: MachineInventorySelectorTemplate
metadata:
  name: my-machine-selector
  namespace: fleet-default
spec:
  template:
    spec:
      selector:
        ...
----

`template.spec.selector` 可包括`matchLabels` 和`matchExpressions` 键。

== `template.spec.selector.matchLabels`

它是`{key,value}` 对`(map[string]string)` 的映射。提供多个标签时，所有标签必须匹配。

.点击此处了解详情
[%collapsible]
====
[yaml]
----
...
spec: template: spec: selector: matchLabels: element: fire manufacturer: somevalue
----
====

使用上述选择器定义的群集将只尝试提供包括这两个标签在内的节点清单。

== `template.spec.selector.matchExpressions`

这是一个标签选择器列表，每个标签选择器可定义为

|===
| 钥匙 | 类型 | 说明

| 密钥
| 字符串
| 这是选择器应用的标签键

| 操作员
| 字符串
| 表示键与一组值之间的关系。有效操作符为 "在"、"不在"、"存在 "和 "不存在"。

| 价值观
| []字符串
| Values 是一个字符串值数组。如果操作符为 "In "或 "NotIn"，值数组必须是非空的。如果操作符为 "存在 "或 "不存在"，则值数组必须为空
|===

.点击此处了解详情
[%collapsible]
====
[,yaml]
----
...
spec:
  template:
    spec:
      selector:
        matchExpressions:
        - key: element
          operator: In
          values: [ 'fire' ]
        - key: manufacturer
          operator: Exists
----
====

使用上述选择器定义的群集将只尝试为`element=fire` 标签（包括使用任何值定义的`manufacturer` 标签）所记录的节点进行配置。
