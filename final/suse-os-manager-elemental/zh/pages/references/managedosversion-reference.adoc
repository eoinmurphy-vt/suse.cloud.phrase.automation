= ManagedOSVersion 参考
:revdate: 2025-09-08
:page-revdate: {revdate}

`ManagedOSVersion` 资源负责定义可与`SeedImage` 或`ManagedOSImage` 资源一起使用的操作系统映像版本。

`ManagedOSVersion` 资源规范下可配置多个密钥。

.managedosversion-example.yaml
[,yaml]
----
apiVersion: elemental.cattle.io/v1beta1
kind: ManagedOSVersion
metadata:
  labels:
    elemental.cattle.io/channel: elemental-channel
  name: v2.0.2
  namespace: fleet-default
spec:
  metadata:
    displayName: SLE Micro
    upgradeImage: registry.suse.com/suse/sle-micro/5.5:2.0.2
  type: container
  version: v2.0.2
----

== `ManagedOSVersionSpec` 参照

|===
| 钥匙 | 类型 | 默认值 | 说明

| 元数据
| 反对
| 无效
| 这定义了操作系统映像的一些数据。参见 [参考文献]（#元数据）

| minVersion
| 字符串
| 无效
| 未使用

| 类型
| 字符串
| 空的
| 定义操作系统映像类型，可以是`container` 或 `iso`

| 版本
| 字符串
| 空的
| 操作系统图像版本

| upgradeContainer
| 反对
| 无效
| 可定义的升级容器。参见 [参考文献]（#upgradecontainer）
|===

== 元数据

本文介绍了在{elemental-product-name} 中定义操作系统映像所需的信息。

如果`type` 设置为`container` ：

|===
| 钥匙 | 类型 | 默认值 | 说明

| displayName
| 字符串
| 空的
| 在 Rancher UI 中看到的操作系统图像名称

| upgradeImage
| 字符串
| 空的
| 完全合格的容器映像（OCI 引用或 HTTP URI）
|===

如果`type` 设置为`iso` ：

|===
| 钥匙 | 类型 | 默认值 | 说明

| displayName
| 字符串
| 空的
| 在 Rancher UI 中看到的操作系统图像名称

| uri
| 字符串
| 空的
| 完全合格的 ISO 映像
|===

== `upgradeContainer`

这样就可以根据此 ManagedOSVersion 覆盖系统升级控制器计划（请参阅xref:rancher-os-management/upgrade-lifecycle.adoc#_components[升级组件]）的默认`upgrade` 字段。
系统升级控制器会将这些密钥转化为 Kuberneteshttps://kubernetes.io/docs/reference/kubernetes-api/workload-resources/pod-v1/#Container[容器]规范。
这是负责运行操作系统升级的容器。
