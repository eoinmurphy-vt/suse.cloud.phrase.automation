= Kubernetes 集群调配
:revdate: 2025-11-06
:page-revdate: {revdate}

Kubernetes 集群部署阶段的目标是使用可用的xref:references/machineinventory-reference.adoc[MachineInventories]（即已成功完成xref:rancher-os-management/architecture/services/architecture-machineonboarding.adoc[机器入驻]阶段的主机）创建一个新的 RKE2 或 K3s 集群。

{elemental-product-name} Kubernetes 集群部署包括以下步骤：

* 用户可创建一个xref:references/machineinventoryselectortemplate-reference.adoc[机器清单] _选择器_ xref:references/machineinventoryselectortemplate-reference.adoc[模板（MachineInventorySelectorTemplate]）资源：它允许定义一个_选择器_，用于根据标签值识别可用xref:references/machineinventory-reference.adoc[机器清单]的子集。
* 用户会定义一个xref:references/cluster-reference.adoc[Rancher 集群]，并在`machinePools` 定义中添加对前一步创建的xref:references/machineinventoryselectortemplate-reference.adoc[MachineInventorySelectorTemplate]的引用。
* https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/launch-kubernetes-with-rancher.html[]Rancher会对 Rancher 集群资源创建做出反应，生成与_machinePools_ 中指定的_数量_相等的xref:references/machineinventoryselector-reference.adoc[MachineInventorySelectors]资源。
* {elemental-product-name} 操作员会将每个生成的xref:references/machineinventory-reference.adoc[MachineInventorySelector]资源与可用的xref:references/machineinventory-reference.adoc[MachineInventory]配对，并在xref:references/machineinventory-reference.adoc[MachineInventory] 跟踪的主机上安装https://github.com/rancher/system-agent[rancher-system-agent]守护进程。
https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/launch-kubernetes-with-rancher.html[]Rancher使用https://github.com/rancher/system-agent[RKE2/K3s 集群供应] _计划_接管 K3s/RKE2 供应：它安装所需的组件（如 containerd、K3s、... ）并创建配置文件，直至成功部署新的 Kubernetes 集群。
