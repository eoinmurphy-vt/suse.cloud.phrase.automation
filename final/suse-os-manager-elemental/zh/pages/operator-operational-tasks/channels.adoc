= 渠道
:revdate: 2025-09-30
:page-revdate: {revdate}

{elemental_operator_name} 允许订阅一个或多个xref:references/managedosversionchannel-reference.adoc[ManagedOSVersionChannels]，自动填充xref:references/managedosversion-reference.adoc[ManagedOSVersions]列表，以便使用xref:references/seedimage-reference.adoc[SeedImage] 构建新的 ISO，或使用xref:references/managedosimage-reference.adoc[ManagedOSImage] 将现有{elemental-product-name} 节点升级到新的操作系统版本。

通道通常作为 OCI 容器映像发布，但也可以直接引用包含`ManagedOSVersion` 列表的 JSON 文件的 URI。请注意，最好的做法是使用图像来分发通道，以便分发的图像与{elemental_operator_name} 所需的所有其他图像保持一致。例如，在 Airgapped 环境中部署时，这一点就非常有用。

== 渠道生命周期和最佳实践

一旦创建了新的`ManagedOSVersionChannel` ，{elemental_operator_name} 就会定期同步频道提供的 JSON 列表，并将其转换为新的`ManagedOSVersions` 。 +
所有同步的`ManagedOSVersions` 都将归`ManagedOSVersionChannel` 所有。删除`ManagedOSVersionChannel` 将导致删除级联上的所有`ManagedOSVersions` 。

请注意，当启用`ManagedOSVersionChannel.spec.deleteNoLongerInSyncVersions` 选项时，`ManagedOSVersionChannel` 支持自动清理不再同步的`ManagedOSVersions` 。

当`ManagedOSVersion` 计划删除时，终结器会确保任何`ManagedOSImage` 上都没有活动引用。

如果`ManagedOSVersion` 无法删除，可以找出它被哪些资源引用：

[,shell]
----
kubectl -n fleet-default get managedosimages -l elemental.cattle.io/managed-os-version-name=my-deleted-os-version
----

在使用多个渠道时，重要的是要保持适当的命名策略，以便在自有`ManagedOSVersions` 上始终有一个快速、人性化的参考。 +
建议将任何频道命名为`{BaseOS}-{BaseOSVersion}-{Flavor}`.

这样，用户就可以使用`ManagedOSVersion` 名称作为映像的特定{elemental-product-name} 构建版本，同时保留对父通道中基础操作系统和基础操作系统版本的参考。 +

== 制作自己的频道

要制作自己的自定义同步器，唯一的要求是让它输出 JSON 文件到`/data/output` ，并保持正确的 JSON 结构。

该文件是一个 JSON 数组，包含 ISO 和容器条目。
数组中的每个条目都与xref:references/managedosversion-reference.adoc[ManagedOSVersion]对象进行 1:1 映射。

`"type": "iso"` 条目必须包含可启动的{elemental-product-name} ISO，并由xref:references/seedimage-reference.adoc[SeedImage] 使用，而`"type": "container"` 条目则由xref:references/managedosimage-reference.adoc[ManagedOSImage]使用，用于{elemental-product-name} 升级。

如有疑问，https://github.com/rancher-sandbox/elemental-channels[elemental-channels]项目可作为如何构建和维护自己的频道的参考实现。

在创建新条目时，请注意您选择的命名策略，以避免与其他频道发生冲突，因为它们最终可能会用相同的名称同步不同的`ManagedOSVersion` 。
最好的做法是使用惯例： `\{Flavor}-\{Version}-\{Type}`
