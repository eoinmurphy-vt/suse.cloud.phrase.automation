= 在空气隔绝的环境中安装{elemental-product-name} 
:revdate: 2025-11-06
:page-revdate: {revdate}

== 假设

Rancher 空气盖安装应该已经按照https://documentation.suse.com/cloudnative/rancher-manager/latest/en/installation-and-upgrade/other-installation-methods/air-gapped/air-gapped.html[Rancher 官方文档]进行了配置。
特别是，应在空气屏蔽基础设施中提供私人登记册。

== 概述

要在空气屏蔽环境中运行{elemental-product-name} ，需要以下工具：

* {elemental-product-name} 运营商图表
* 图表中引用的容器图像（_元素操作器_和_种子生成器_图像）
* 容器化操作系统映像

此外，参考可用的容器化操作系统_镜像_创建_通道_镜像也很方便。
官方渠道镜像（_elemental-channel_镜像）引用了 suse 官方注册表中操作系统镜像的绝对 URL，因此不能在空中加油的情况下直接使用。

== {elemental-product-name} 通过命令行安装 Air-Gapped

所有必要步骤都可通过可上网的主机执行https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[`elemental-airgap.sh` 脚本来]完成。

{elemental-product-name} 图表是脚本的必备参数，可作为下载的存档、URL 或`stable` 、`staging` 和`dev` 关键字之一提供，以便让脚本为您检索正确的图表版本。

`elemental-airgap.sh` 检查{elemental-product-name} Operator 图表，识别所有需要的容器镜像，下载并将它们保存到一个 docker 存档中。
它还会构建一个新的操作系统通道映像，其操作系统映像 URL 指向作为参数传递的私有注册表（这也是一个强制参数）。

最新版本的元素脚本可从官方 github 软件仓库轻松下载：

[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh
chmod 755 elemental-airgap.sh
----

现在，让我们从{elemental-product-name} 的最新稳定版本中下载所有工件并构建一个自定义通道：

[tabs]
====
创建 Docker 存档::
+
======
[,shell]
----
./elemental-airgap.sh stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

完成后（脚本可能需要一段时间），当前目录中将出现以下文件

* `elemental-operator-crds-chart-<*VERSION*>.tgz`
* `elemental-operator-chart-<*VERSION*>.tgz`
* `elemental-images.txt`
* `elemental-images.tar.gz`
======

创建运输工具档案::
+
======
[,shell]
----
./elemental-airgap.sh -ha stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

完成后（脚本可能需要一段时间），图表和容器图像都将被打包到名为`elemental-haul.tar.zst` 的运输工具存档中。
======
====

=== {elemental-product-name} 安装 

应将脚本创建的文件和档案复制到主机上，其中包括

* 可访问私人注册表。
* 已安装并配置 kubectl 二进制程序，以访问空中加油的 Rancher 集群。
* 已安装二进制舵机。

[tabs]
========
从 Docker 压缩包安装::
+
======
如果私有注册表需要身份验证，你就需要用 docker 登录：

[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

安装{elemental-product-name} 需要两个步骤：

* 在私有注册表中加载包含所有所需容器映像的存档：可使用随 Rancher 发行版一起分发的`rancher-load-images.sh` 脚本来完成，该脚本已用于 Rancher air-gapped 部署：

[,shell]
----
rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

* 安装下载的元素图，配置本地注册表和新创建的通道：

[,shell]
----
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ 
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM>
----
======

从拖车存档安装::
+
======
要从https://rancherfederal.github.io/hauler-docs/[Hauler]存档（`elemental-airgap.sh` 中的`-ha` 选项）进行安装，还需要在进行安装的主机上安装 Hauler。

如果私人注册表需要验证，则需要使用 Hauler 登录：

[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD
----

{elemental-product-name} 安装需要三个步骤：


* 将 "elemental-haul.tar.zst "运输档案载入空中加油基础设施中的运输机实例：

[,shell]
----
hauler store load 'elemental-haul.tar.zst' 
----

* 如果空气屏蔽环境中的本地注册表不是由 Hauler 服务器提供，则在本地注册表中加载 Haul 存档：

[,shell]
----
hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

[NOTE]
.拖车还可用作登记册
====
如果本地注册表是由 Hauler 实例提供服务的，只需直接加载 Haul 存档（如步骤 (1) 所示），跳过步骤 (2)。
====

* 从 Hauler 商店提取元素图并安装：

[,shell]
----
hauler store extract elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
hauler store extract elemental-operator-chart-<ELEMENTAL-VERSION>.tgz 

helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator-crds elemental-operator-crds-chart-<ELEMENTAL-VERSION>.tgz 
helm upgrade --create-namespace -n cattle-elemental-system \ 
    --install elemental-operator elemental-operator-chart-<ELEMENTAL-VERSION>.tgz \ 
    --set registryUrl=<REGISTRY.YOURDOMAIN.COM:PORT>\ -
    --set channel.repository=rancher/elemental-channel-<REGISTRY.YOURDOMAIN.COM:PORT>
----
======
========

[NOTE]
.元素气隙脚本会输出所需的命令 
====
`elemental-airgap.sh` 脚本会打印出上面显示的所需命令，但使用的是实际图表版本和提供的注册表 URL，以便轻松复制和粘贴准确的命令。
====

== {elemental-product-name} 来自 "牧场主市场 "的气隙装置 

Rancher 空气盖安装还包括{elemental-product-name} 操作员图表以及操作员和种子图像容器图像。

要收集缺失的操作系统映像，并为专用注册表构建操作系统通道映像，请使用`-co` 选项，在可上网的主机上执行https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh[`elemental-airgap.sh` 脚本]。

例如，让我们以最新稳定版{elemental-product-name} 中的`elemental-channel` 镜像为目标。脚本将负责下载{elemental-product-name} 操作员图表（如需要），提取操作系统通道映像 URL 并下载，检查引用的所有操作系统映像，下载所有映像，并创建一个新的操作系统通道，其中包含指向空气屏蔽场景专用注册表的链接。

[tabs]
========
创建 Docker 存档::
+
======
[,shell]
----
wget https://raw.githubusercontent.com/rancher/elemental-operator/main/scripts/elemental-airgap.sh 
chmod 755 elemental-airgap.sh 
./elemental-airgap.sh stable -co -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

完成后（脚本可能需要一段时间），当前目录中将出现以下文件

- `elemental-operator-crds-chart-<*VERSION*>.tgz` 
- `elemental-operator-chart-<*VERSION*>.tgz` 
- `elemental-images.txt` 
- `elemental-images.tar.gz`
======

创建运输工具档案::
+
======
[,shell]
----
./elemental-airgap.sh -ha -co stable -r <REGISTRY.YOURDOMAIN.COM:PORT>
----

完成后（脚本可能需要一些时间），集装箱图像将被打包到名为`elemental-haul.tar.zst` 的运输工具存档中。
======
========

=== {elemental-product-name} 安装 

生成的存档应加载到空气屏蔽的私人注册表中。

[tabs]
========
从 Docker 压缩包安装::
+
======
如果私有注册表需要身份验证，你就需要用 docker 登录：
[,shell]
----
docker login <REGISTRY.YOURDOMAIN.COM:PORT>
----

脚本将打印出通过 Rancher`rancher-load-images.sh` 工具加载图像所需的命令，该工具用于 Rancher 空气盖安装。应该是这样的

[,shell]
----
NEXT STEPS: 

1) Load the 'elemental-images.tar.gz' to the local registry (<REGISTRY.YOURDOMAIN.COM:PORT>) available in the airgapped infrastructure: 

./rancher-load-images.sh \ 
    --image-list elemental-images.txt \ 
    --images elemental-images.tar.gz \ 
    --registry <REGISTRY.YOURDOMAIN.COM:PORT>
----

加载操作系统和通道映像后，应跳过脚本输出中的第 (2) 点（从下载的存档中安装{elemental-product-name} 图表），而是从 Rancher UI 执行{elemental-product-name} Operator 安装。
======

从拖车存档安装::
+
======
如果私人注册表需要验证，则需要使用 Hauler 登录：
[,shell]
----
hauler login <REGISTRY.YOURDOMAIN.COM:PORT>-u $USERNAME -p $PASSWORD 
----

脚本将打印出加载图像所需的命令。应该是这样的

[,shell]
----
NEXT STEPS: 

* Load the 'elemental-haul.tar.zst' Haul archive in the Hauler instance in the airgapped infrastructure: 

hauler store load 'elemental-haul.tar.zst' 

* If the local registry in the air-gapped environment is not server by Hauler, load the Haul archive in the local registry: 

hauler store copy registry://<REGISTRY.YOURDOMAIN.COM:PORT>
----

加载操作系统和通道映像后，应跳过脚本输出中的第 (3) 点（从下载的存档中安装{elemental-product-name} 图表），而是从 Rancher UI 执行{elemental-product-name} Operator 安装。
======
========

== {elemental-product-name} 用户界面扩展 

Rancher 2.7.x 不支持空气屏蔽环境中的 UI 扩展插件，因此{elemental-product-name} UI 在 Rancher 2.7.x 中不可用。

{elemental-product-name} UI 插件将出现在 Rancher 2.8.0 的可用 UI 扩展中。
