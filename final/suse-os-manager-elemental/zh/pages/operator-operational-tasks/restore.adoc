= 恢复
:revdate: 2025-09-08

请按照本指南使用 Rancher 从备份恢复{elemental-product-name} 配置。

== 准备用于恢复的 rancher-backup 操作员和备份文件

请访问https://docs.ranchermanager.rancher.io/how-to-guides/new-user-guides/backup-restore-and-disaster-recovery/restore-rancher[文档] Rancher 官方，确保已安装`rancher-backup operator` 并能访问备份文件。

首先，您必须按照本xref:operator-operational-tasks/backup.adoc[文档]对{elemental-product-name} 资源进行备份。

== 使用 rancher-backup 操作员恢复{elemental-product-name} 配置

创建`restore object` （根据需要进行调整）以恢复备份压缩包。

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Restore
metadata:
  name: elemental-restore
spec:
  backupFilename: rancher-backup-430169aa-edde-4a61-85e8-858f625a755b-2022-10-17T05-15-00Z.tar.gz
----

在 Kubernetes 上应用清单。

[,shell]
----
kubectl apply -f elemental-restore.yaml
----

检查来自 rancher-backup 操作员的日志。

[,shell]
----
kubectl logs -n cattle-resources-system -l app.kubernetes.io/name=rancher-backup -f
----

验证备份文件是否还原成功。

[,shell]
----
...
INFO[2022/10/31 06:34:50] Processing controllerRef apps/v1/deployments/rancher
INFO[2022/10/31 06:34:50] Done restoring
...
----
