= {elemental-product-name} 视觉方式
:revdate: 2025-11-06
:page-revdate: {revdate}

[NOTE]
====
以下说明至少需要 Rancher 2.9.x。
====

本快速入门将向您展示如何将{elemental-product-name} 插件和操作员部署到现有的 Rancher Manager 实例中。

安装完成后，您就可以基于 RKE2 或 K3 配置一个新的{elemental-product-name} 集群。

不过，如果要安装暂存或开发操作员，只能在xref:quickstarts/quickstart-cli.adoc#_non_stable_installations[CLI 模式]下进行。

== 添加 Rancher 官方扩展库

如果 Elemental 扩展名不可用，则需要添加`Official Rancher Extensions Repository`.NET 扩展名：

image::quickstart-ui-extension-repository.png[添加 Rancher Manager 扩展库]

如果无法通过扩展 UI 设置添加该版本库，可通过https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-admin/helm-charts-in-rancher/helm-charts-in-rancher.html#_manage_repositories[添加] `https://github.com/rancher/ui-plugin-charts` `git` 版本库，或应用以下资源进行手动管理：

[source,yaml]
----
apiVersion: catalog.cattle.io/v1
kind: ClusterRepo
metadata:
  name: rancher-ui-charts
spec:
  gitBranch: main
  gitRepo: https://github.com/rancher/ui-plugin-charts
----

== 安装元素插件

启用 Rancher Manager 扩展支持后，可以按如下步骤安装`elemental` 插件：

* 在`Available` 选项卡下，您将看到`elemental` 插件可用

image::quickstart-ui-extensions-available.png[Rancher 管理器可用插件]

[NOTE]
====
如果`Available` 选项卡没有显示条目，请刷新页面。随后将出现`elemental` 插件。
====

* 点击`Install` 按钮，弹出一个窗口，再次点击`Install` 继续。

image::quickstart-ui-elemental-plugin-install.png[{elemental-product-name} 插件安装]

* 在`Installed` 选项卡上，现已列出`elemental` 插件。

[NOTE]
====
如果`elemental` 插件已列出，且状态仍为`Installing...` ，请刷新页面。`elemental` 插件将正确显示。
====

`elemental` 插件安装后，您可以在 Rancher 管理器菜单中看到`OS Management` 选项，如果没有看到，请刷新页面。

image::quickstart-ui-elemental-plugin-menu.png[Rancher Manager 操作系统管理菜单]

== 添加机器注册端点

在操作系统管理仪表板中，单击`Create Registration Endpoint` 按钮。

image::quickstart-ui-registration-endpoint-create.png[操作系统管理注册端点]

在这里，你可以在各自的位置输入每个细节，也可以将其编辑为 YAML 格式，然后一次性创建端点。在这里，我们将编辑每个字段。

image::quickstart-ui-registration-endpoint-create-details.png[使用用户界面创建注册端点]

[IMPORTANT]
.主要选项
====
`name: elemental-cluster1`：根据需要更改

`device-selector`:xref:references/machineregistration-reference.adoc#_config_elemental_install_device_selector[设备选择器]字段可用于在安装过程中动态选择设备。该字段包含一个规则列表，可用于选择所需的设备。

`snapshotter`:用于管理操作系统映像中快照的设备类型。
====

一旦创建了机器注册终端，它就会显示为激活状态。

image::quickstart-ui-registration-endpoint-complete.png[在注册端点注册的机器]

== 准备安装（种子）映像

现在，这是最后一步，你需要准备一个包含初始注册配置的种子映像，这样它就可以作为集群的一部分进行自动注册、安装和完全部署。该文件的内容无非是节点注册所需的注册 URL 和适当的服务器证书，以便它能安全地连接。

然后，这个种子图像就可以用来配置无限多的机器。

种子镜像已创建为上述 Kubernetes 资源，可使用`Build Media` 按钮构建，但首先必须选择 ISO 或 RAW 镜像。

ISO 需要两个设备（装有 ISO 的设备和另一个安装{elemental-product-name} 的磁盘），而 RAW 映像则与之相反，可以从一个设备启动，并直接在设备上安装操作系统。
RAW 映像只包含一个启动分区和一个恢复分区，它会首先启动到恢复模式以安装{elemental-product-name} （仅供参考，过程与xref:node-operational-tasks/reset.adoc#_reset_workflow[重置]分区类似）。

image::quickstart-ui-registration-endpoint-build-media.png[在注册端点中构建媒体]

构建完成后，可使用`Download Media` 按钮下载媒体：

image::quickstart-ui-registration-endpoint-download-media.png[下载注册终端中的媒体]

现在，您可以使用该镜像启动您的节点，它们就会启动：

* 使用给定的注册 URL 注册，并为每台机器创建一个 `MachineInventory`
* 将 SLE Micro 安装到指定设备
* 重新启动

== 机器库存

当节点第一次启动时，它们会连接到 Rancher 管理器，并为每个节点创建一个 xref:references/machineinventory-reference.adoc[`Machine Inventory`]为每个节点创建。

image::quickstart-ui-machine-inventory-menu.png[机器库存菜单]

自定义列基于`Machine Inventory Labels` ，可在创建`Machine Registration Endpoint` 时添加：

image::quickstart-ui-registration-endpoint-hardware-labels.png[机器注册终端硬件标签]

在下面的截图中，`Hardware Labels` 被用作自定义列：

您还可以点击三点菜单添加自定义列。

image::quickstart-ui-machine-inventory-custom-columns.png[机器库存自定义列]

最后，您还可以使用这些标签过滤`Machine Inventory` 。

例如，如果只想查看 AMD 机器，可以通过`CPUModel` 进行筛选，如下所示：

image::quickstart-ui-machine-inventory-filtering.png[机器库存过滤]

== 创建第一个{elemental-product-name} 集群

现在，让我们点击`Create {elemental-product-name} Cluster` ，使用这些`Machine Inventory` 创建一个群集：

image::quickstart-ui-create-cluster-button.png[创建{elemental-product-name} 群集按钮]

对于{elemental-product-name} 集群，你可以选择 Kubernetes 的 K3s 或 RKE2。

image::quickstart-ui-create-cluster-standard-screen-.png[{elemental-product-name} 群集创建屏幕]

大部分选项都来自 Rancher，因此我们不会详细介绍所有的可能性。
如果您想了解更多信息，请查阅https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/k3s.html[Manager K3s 分发配置文档] Rancher或https://documentation.suse.com/cloudnative/rancher-manager/latest/en/cluster-deployment/configuration/rke2.html[Manager RKE2 分发配置文档] Rancher。

不过，必须强调`Inventory of Machines Selector Template` 部分。

它可让您选择要使用的`Machine Inventory` ，以使用先前定义的`Machine Inventory Labels` 创建{elemental-product-name} 群集：

image::quickstart-ui-create-cluster-machine-selector-template.png[使用机器库存选择器模板]

由于我们的三台机器清单包含带有密钥`AuthenticAMD` 的标签`CPUVendor` ，这三台机器将用于创建{elemental-product-name} 群集。
