== 快速故障排除

AI助手位于本地群集的`cattle-ai-agent-system`名称空间中。可以访问代理日志以了解任何问题。

[source,bash]
----
kubectl get po -n cattle-ai-agent-system
----

要从‘rancher-ai-agent’ pod获取日志：
[source,bash]
----
kubectl logs -n cattle-ai-agent-system -l app=rancher-ai-agent
----

要从‘rancher-ai-mcp’ pod获取日志：
[source,bash]
----
kubectl logs -n cattle-ai-agent-system -l app=rancher-mcp-server
----

== 启用调试日志

您可以在代理的Helm部署期间使用`log.level=debug`来启用调试日志。

[NOTE]
====
如果您通过设置页面配置了模型，请在更新之前在值文件中报告设置。
====

更新values.yaml
[source,yaml]
----
...
log:
  level: debug
...
----

更新图表：
[source,bash,subs="attributes+"]
----
helm upgrade -n cattle-ai-agent-system -f values.yaml rancher-ai-agent {oci-reg}
----

== Rancher 使用不安全的证书

如果您的Rancher管理器使用自签名证书，请使用`insecureSkipTls=true`允许MCP服务器连接到Rancher。

[NOTE]
====
如果您通过设置页面配置了模型，请在更新之前在值文件中报告设置。
====

[WARNING]
====
如果图表已经部署。在升级图表之前，您需要删除rancher-mcp-server部署。
[source,bash,subs="attributes+"]
----
kubectl delete deployments.apps -n cattle-ai-agent-system rancher-mcp-server
----
====

更新values.yaml
[source,yaml]
----
...
insecureSkipTls: true
...
----

更新图表：
[source,bash,subs="attributes+"]
----
helm upgrade -n cattle-ai-agent-system -f values.yaml rancher-ai-agent {oci-reg}
----
