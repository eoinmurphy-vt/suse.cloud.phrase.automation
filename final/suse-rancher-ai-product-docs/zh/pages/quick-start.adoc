= 快速入门
:revdate: 2026-01-23

:description: 开始使用 {project-name}，安装 {project-name} 堆栈并处理先决条件和身份验证。

此快速入门说明如何使用 Ollama 为 {short-project-name} 提供 LLM 功能。您可以在初始安装后通过参考 xref:/how-tos/how-to-admin.adoc[管理员操作指南] 切换提供者。

== 先决条件

当前版本的 AI 助手需要以下组件：

* https://helm.sh/docs/[已安装 Helm]
* Rancher Prime/Community Manager 2.13 或更高版本

确保您在主集群上具有部署 CRD（cluster_admin）的权限。

== 技术要求

以下是支持的 AI 组件及其要求的表格。

[NOTE]
====

您只需满足您打算运行的特定组件的要求。要求将根据您选择部署的特定大语言模型（LLM）而有所不同。

====

[cols="1,1,1", options="header"]
|===
| LLM 模型 | 要求 | GPU 要求

| https://ollama.com/library/gpt-oss[gpt-oss:20b]

| ChatGPT
| OpenAI 账户
| 不适用

| AWS Bedrock
| AWS 账户

| 类似 OpenAI 的提供者
| Azure 账户，Evroc Think，...

[IMPORTANT]
====

经过测试的模型列表可在 xref:/explanations/models.adoc[模型] 文档中找到。

====

== 安装 {project-name}

安装 {short-project-name} 是一个两步过程：

* 通过提供的 Helm 图表部署代理和 MCP。
* 部署 Rancher UI 扩展。

=== 在本地群集上安装代理和 MCP

. 创建 `values.yaml` 设置文件（用于 Ollama）：
+
[source,yaml]
----
ollamaLlmModel: "gpt-oss:20b"
ollamaUrl: "http://ollama:11434"
activeLlm: "ollama"
----
+
[NOTE]
====

您可以稍后在 Rancher 的“全局设置 -> AI 助手选项卡”中更改这些设置和提供者。

====
. 安装 rancher-ai-agent 图表
+
[source,bash,subs="attributes+"]
----
helm install rancher-ai-agent \
  --namespace cattle-ai-agent-system \
  --create-namespace \
  -f values.yaml {oci-reg}
----

=== 安装 UI 扩展

打开 Rancher 管理 UI 并导航到“扩展”页面。

. 在 Rancher 管理器上，点击菜单栏中的扩展。
+
image::menubar-extension.png[菜单栏中的扩展按钮]
+
. 使用右上角的三点菜单，选择“添加 Rancher 储存库”。
+
image::rancher-repositories.png[管理 Rancher 储存库的菜单]
+
. 选择 '添加官方 Rancher 扩展储存库' 以添加该储存库。
+
image::add-extensions-repositories.png[添加官方 Rancher 储存库]
+
. 单击 '添加'。
+
. 找到 AI 助手卡片并单击 '安装'。
. 选择一个版本（或默认使用最新版本）并单击 '安装'。
+
image::config-repo-install.png[选择一个版本或使用最新版本。]
+
. 扩展安装完成后，单击页面顶部出现的 '重新加载' 按钮。
