您可以使用常规的开放容器倡议（OCI）注册表，将{project-name} 策略作为https://opencontainers.org[工件] OCI发布。

{short-project-name} 策略存储与容器镜像并存。
除了常规的容器镜像外，它们不需要额外的设置或维护。

[NOTE]
====
您可以使用页面底部的 "编辑本页 "按钮添加与{short-project-name} 兼容的注册表，或更正任何不准确的注册表。
====


== 实施 OCI 登记册的项目

* https://goharbor.io/[港湾]
* https://github.com/distribution/distribution[分布](https://github.com/distribution/distribution/releases/tag/v2.7.0[≥ 2.7.0])。
* https://access.redhat.com/products/red-hat-quay/[码头]：支持，但https://access.redhat.com/documentation/en-us/red_hat_quay/3/html/use_red_hat_quay/oci-intro#other-oci-artifacts-with-quay[在 v3.6 中默认为不活动]。

== 托管的 OCI 登记册

* https://github.com/container-registry/[GitHub 容器注册中心]。请参见https://docs.github.com/en/packages/working-with-a-github-packages-registry/working-with-the-container-registry[] 。
* https://quay.io[Quay.io]。参见项目清单。
* https://aws.amazon.com/ecr/[亚马逊 ECR]：请参见https://aws.amazon.com/blogs/containers/oci-artifact-support-in-amazon-ecr/[] 。
* https://cloud.google.com/artifact-registry[谷歌艺术品登记处]。请参见https://cloud.google.com/anthos-config-management/docs/how-to/sync-oci-artifacts-from-artifact-registry[] 。
* https://azure.microsoft.com/en-us/products/container-registry/[Azure 容器注册中心]。请参见https://learn.microsoft.com/en-us/azure/container-registry/container-registry-oci-artifacts[] 。
* https://hub.docker.com/[Docker Hub]。请参见https://docs.docker.com/docker-hub/oci-artifacts/[] 。
* https://cloud.ibm.com/docs/Registry[IBM 云容器注册中心]。请参见https://cloud.ibm.com/docs/Registry?topic=Registry-registry_helm_charts[] 。
* https://jfrog.com/artifactory/[JFrog Artifactory].请参见https://jfrog.com/help/r/jfrog-artifactory-documentation/docker-registry[] 。

== 已知问题

=== Docker Hub

目前，Docker Hub 不支持 OCI 工件，因此不能用于存储{short-project-name} 策略。
Docker 公司宣布，Docker Hub 将在https://www.docker.com/blog/announcing-docker-hub-oci-artifacts-support/[未来]支持 OCI 工件。

=== JFrog

虽然 JFrog 支持 OCI 工件，但在遵循其规范时，只能部分地向其推送。https://github.com/kubewarden/kwctl/issues/59[在此阅读更多内容]。
