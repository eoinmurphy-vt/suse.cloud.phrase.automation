== 命令行帮助 `policy-server`

本文档包含`policy-server`命令行程序的帮助内容。

*命令概述：*

* link:#policy-server[`policy-server`]
* link:#policy-server-docs[`policy-server docs`]

=== `policy-server`

*使用方法* `policy-server ++[+OPTIONS+]++ ++[+COMMAND+]++`

==== *子命令：*

* `docs` - 生成策略服务器命令的 markdown 文档

==== *选项：*

* `--addr ++<+BIND_ADDRESS+>++` - 与 ADDRESS 绑定
+
默认值： `0.0.0.0`
* `--always-accept-admission-reviews-on-namespace ++<+NAMESPACE+>++` - 始终接受以给定命名空间为目标的 AdmissionReviews
* `--cert-file ++<+CERT_FILE+>++` - HTTPS X.509 证书文件的路径
* `--client-ca-file ++<+CLIENT_CA_FILE+>++` - 签发客户证书的 CA 证书文件的路径。需要启用 mTLS
* `--daemon` - 如果设置了该选项，则策略服务器将以守护进程的方式在分离模式下运行
* `--daemon-pid-file ++<+DAEMON-PID-FILE+>++` - PID 文件的路径，仅在以守护进程模式运行时使用
+
默认值： `policy-server.pid`
* `--daemon-stderr-file ++<+DAEMON-STDERR-FILE+>++` - 存储 stderr 文件的路径，仅在以守护进程模式运行时使用
* `--daemon-stdout-file ++<+DAEMON-STDOUT-FILE+>++` - 保存 stdout 文件的路径，仅在以守护进程模式运行时使用
* `--disable-timeout-protection` - 停用策略超时保护
* `--docker-config-json-path ++<+DOCKER_CONFIG+>++` - Docker config.json- 类似路径的路径。可用于显示注册表身份验证详细信息
* `--enable-metrics` - 启用度量
* `--enable-pprof` - 启用 pprof 配置文件
* `--ignore-kubernetes-connection-failure` - 如果 Kubernetes 连接失败，不要错误退出。当与 Kubernetes 没有连接时，这会导致上下文感知策略失效。
* `--key-file ++<+KEY_FILE+>++` - HTTPS X.509 私钥文件的路径
* `--log-fmt ++<+LOG_FMT+>++` - 日志输出格式
+
默认值： `text`
+
可能的值：`text`,`json` 、 `otlp`
* `--log-level ++<+LOG_LEVEL+>++` - 日志级别
+
默认值： `info`
+
可能的值：`trace`,`debug`,`info`,`warn` 、 `error`
* `--log-no-color` - 禁用日志彩色输出
* `--policies ++<+POLICIES_FILE+>++` - YAML 文件，其中包含要加载的策略及其设置
+
默认值： `policies.yml`
* `--policies-download-dir ++<+POLICIES_DOWNLOAD_DIR+>++` - 政策的下载路径
+
默认值： `.`
* `--policy-timeout ++<+MAXIMUM_EXECUTION_TIME_SECONDS+>++`
- 在给定时间后中断策略评估
+
默认值： `2`
* `--port ++<+PORT+>++` - 通过端口收听
+
默认值： `3000`
* `--readiness-probe-port ++<+READINESS_PROBE_PORT+>++` —
Expose readiness endpoint on READINESS+_PROBE_+PORT
+
默认值： `8081`
* `--sigstore-cache-dir ++<+SIGSTORE_CACHE_DIR+>++` - 用于缓存 sigstore 数据的目录
+
默认值： `sigstore-data`
* `--sources-path ++<+SOURCES_PATH+>++` - 保存源信息（https、不安全主机注册表、自定义 CA 等）的 YAML 文件
* `--verification-path ++<+VERIFICATION_CONFIG_PATH+>++` - 保存验证信息（URI、键、注释\...\...）的 YAML 文件
* `--workers ++<+WORKERS_NUMBER+>++` - 要创建的工作线程数

=== `policy-server docs`

生成策略服务器命令的 markdown 文档

*使用方法* `policy-server docs --output ++<+FILE+>++`

==== *选项：*

* `-o`,`--output ++<+FILE+>++` - 存放文档文件的路径

本文档由 clap-markdown 自动生成。
