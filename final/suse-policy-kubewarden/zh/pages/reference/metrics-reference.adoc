== 政策服务器

策略服务器组件负责初始化和运行策略。
当接收到来自 Kubernetes API 服务器的请求时，它会将请求转发给策略，并将策略提供的响应返回给 Kubernetes API 服务器。

=== 衡量标准

[NOTE]
====
https://opentelemetry.io/docs/concepts/signals/baggage/[行李]键值属性被添加到指标中，以提供更多信息。
====


|===
| 名称 | 类型 | 行李

| `kubewarden_policy_evaluations_total`
| 计数器
| <<_kubewarden_policy_evaluations_total,行李>>
|===

==== `kubewarden_policy_evaluations_total`

===== 行李

|===
| 标签 | 说明

| `policy_name`
| 政策名称

| `resource_name`
| 被评估资源的名称

| `resource_kind`
| 被评估资源的种类

| `resource_namespace`
| 被评估资源的命名空间。如果资源是群集作用域，则不存在。

| `resource_request_operation`
| 操作类型：`CREATE`,`UPDATE`,`DELETE`,`PATCH` 、 `WATCH`...

| `accepted`
| 请求被接受了吗？

| `mutated`
| 请求是否发生了变异？

| `error_code`
| 政策拒绝时返回的错误代码（如果有）。
  如果保单没有提供，则不存在。
|===
