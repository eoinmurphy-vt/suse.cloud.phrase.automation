在评估过程中，{project-name} 策略可以访问主机环境提供的额外功能。
该机制使用的方法类似于传统的https://en.wikipedia.org/wiki/Remote_procedure_call[] RPC。

这就是{short-project-name} 保单发出请求时的情况：

* {short-project-name} 政策：
 ** 调用主机环境提供的功能。
 ** 调用是一个阻塞操作，因此策略代码将等待主机提供答案。
* 主机环境：
 ** 收到能力调用。
 ** 主机执行操作。
 ** 主机对政策作出答复，答复可能是成功的，也可能是失败的。
* {short-project-name} 政策：
 ** 代码收到主机的回复后，继续执行。
 ** 主机的响应也会得到相应处理。

主机能力功能通过https://wapc.io/[waPC] 实现。
每种功能都使用这些详细信息：

* waPC 功能名称：主机暴露的能力名称。
* 输入有效载荷：政策请求的正文。
这始终以 JSON 格式编码。
* 输出有效载荷：主机回复的正文。这始终以 JSON 格式编码。

当出现问题时，主机会回复一个错误。
这需要使用策略所使用编程语言的错误类型。
错误是一个 UTF-8 字符串，包含一条解释信息。
