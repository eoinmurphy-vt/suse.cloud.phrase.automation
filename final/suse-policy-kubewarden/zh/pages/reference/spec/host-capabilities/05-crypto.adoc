在撰写本文时，由于 Wasm 的限制，一些加密库无法编译到 Wasm 中。与此同时，需要这些库的{project-name} 策略可以执行回调，以评估主机端的加密功能。他们收到结果后，继续执行任务。

== 行为

Wasm 主机依靠`rustls-wepki` 实现 x.509 和 PKI 证书。这意味着

- 主要用途我们选择接受任何密钥的使用，这有助于涵盖多种使用情况（与往常一样，如果您想讨论不同的方法，请联系我们）。
- 证书链：如果中间 CA 过期，证书将被视为不可信任。
- 证书到期日期：如果提供了证书链，我们将始终验证证书的整个有效期。Wepki 不会检查根 CA 的过期情况。
- 我们总是验证证书，如果没有提供证书链，则使用 Mozilla 的 CA。

== WaPC 协议合同

如果您实施的是自己的语言 SDK，那么这些函数将执行主机暴露的加密检查：

=== waPC 功能 -`v1/is_certificate_trusted` 输入

[,hcl]
----
# Certificate:
{
  # **mandatory**: Which encoding is used by the certificate
  # Either the string "Pem" or "Der".
  "encoding": string,
  # Actual certificate
  # The certificate is UTF-8 encoded.
  # It's an array of bytes of the unicode code pointers of a PEM/DER encoded
  # certificate string.
  "data": [byte(int), ..., byte(int)]
}

{
  # **mandatory**: PEM-encoded certificate to verify
  "cert": Certificate,
  # optional:
  "cert_chain": [
      # list of certs, ordered by trust
      # usage (intermediates first, root last)
      # If not provided, Mozilla's CA is used.
      Certificate,
      ...
      Certificate,
    ],
  # RFC 3339 time format string, to check expiration
  # against.
  # If missing, time is now.
  "not_after": string
}
----

=== waPC 功能 -`v1/is_certificate_trusted` 输出

[,hcl]
----
{
   # true if certificate verified:
   "trusted": boolean,
   # empty if trusted == true:
   "reason": string
}
----
