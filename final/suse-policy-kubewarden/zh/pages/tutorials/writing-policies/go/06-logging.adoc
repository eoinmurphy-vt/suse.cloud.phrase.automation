Go SDK 与https://github.com/francoispqt/onelog[`onelog`]项目集成，几乎是开箱即用。
// TODO: Last release 2019. Still good?

该项目选择该图书馆的原因是

* 它可与 WebAssembly 二进制文件一起使用。
其他流行的日志解决方案无法编译为目标 WebAssembly。
* https://github.com/francoispqt/onelog#benchmarks[性能良好]。
* 它支持结构化日志记录。

== 初始化日志记录器

您需要初始化一个日志记录器结构。
通过在全局变量中进行初始化，可以从两个主要策略入口点记录日志：`validate` 和`validate_settings` 。

在主软件包`main.go` 中有日志记录器的初始化：

[,go]
----
var (
    logWriter = kubewarden.KubewardenLogWriter{}
    logger    = onelog.New(
        &logWriter,
        onelog.ALL, // shortcut for onelog.DEBUG|onelog.INFO|onelog.WARN|onelog.ERROR|onelog.FATAL
    )
)
----

== 使用记录仪

现在，您可以使用`logger` 对象从政策要求的任何地方登录：

[,go]
----
func validate(payload []byte) ([]byte, error) {
    // ...
    logger.Info("validating request")
    // ...
}
----

您可以添加结构化日志：

[,go]
----
func validate(payload []byte) ([]byte, error) {
    // ...
    logger.WarnWithFields("logging something important", func(e onelog.Entry) {
        e.String("one_field", "a value")
        e.String("another_field", "another value")
    })
    // ...
}
----

更多信息请参阅https://pkg.go.dev/github.com/francoispqt/onelog?utm_source=godoc[`onelog` 文档]。

策略日志记录由策略评估器（如`kwctl` 或`policy-server` ）负责，它们代表策略记录日志。
它们使用的机制可与其他实现分布式跟踪的组件互操作，如https://www.jaegertracing.io/[] Jaeger。
