== 先决条件

您需要 Cert-Manager 和 OpenTelemetry。按照xref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[以下说明]安装 Cert Manager 和 OpenTelemetry Operator。

== 安装积架

应用xref:howtos/telemetry/20-tracing-qs.adoc#_install_jaeger[跟踪快速入门]中的安装步骤。

一旦积木操作员创建了所有资源，`my-open-telemetry-collector.jaeger.svc.cluster.local` 。

可通过以下地址访问积家查询用户界面：

[subs="+attributes",console]
----
https://<CLUSTER_IP>/api/v1/namespaces/jaeger/services/http:my-open-telemetry-query:16686/proxy/search
----

该端点可能是唯一的，具体取决于您的配置。您可以在 "服务 "页面下找到为您的 Jaeger 资源列出的端点。

=== 使用 Jaeger 端点更新`rancher-kubewarden-controller` 

你需要编辑`rancher-kubewarden-controller` 资源，添加 Jaeger 端点 "my-open-telemetry-collector.jaeger.svc.cluster.local:4317"。

例如

[subs="+attributes",yaml]
----
telemetry:
  tracing: true
  mode: sidecar
  sidecar:
    tracing:
      jaeger:
        endpoint: my-open-telemetry-collector.jaeger.svc.cluster.local:4317
        tls:
          insecure: true
----

服务`my-open-telemetry-collector` 的安装在`jaeger` 命名空间下。

[CAUTION]
====

为简单起见，我们不对 OpenTelemetry 采集器和 Jaeger 端点之间的通信进行加密。

这不*适合用于生产部署*。您应该阅读 Jaeger 的https://www.jaegertracing.io/docs/latest/kubernetes/[文档]。

====


现在，您应该可以查看与特定策略服务器绑定的策略的任何失败请求。您还可以查看任何给定策略的详细视图。
您可以使用 Jaeger UI 更深入地查看轨迹。
