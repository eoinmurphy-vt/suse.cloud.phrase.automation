=== `kwctl` CLI 工具

`kwctl` 是{short-project-name} 的 CLI 工具，供策略作者和集群管理员在将策略应用到 Kubernetes 集群之前进行测试。

该工具的界面与`docker` CLI 工具类似。

==== 使用案例

在这些情况下，您可以使用`kwctl` 来提供帮助：

_作为政策制定者_

* _对政策进行端到端测试_：针对精心制作的 Kubernetes 请求测试策略，确保策略的运行符合预期。您甚至可以测试需要访问运行中的群集的上下文感知策略。
* _在 Wasm 模块中嵌入元数据_：二进制文件包含执行所需的权限注释。您可以使用`kwctl` 查看和修改这些注释。
* _向 OCI 登记处发布政策_：二进制文件是完全符合 OCI 标准的对象，可以存储在 OCI 注册表中。

_作为群集管理员_

* _检查远程策略_：给定 OCI 注册表或 HTTP 服务器中的策略，显示该策略的所有静态信息。
* _在群集中模拟运行策略_：针对精心制作的 Kubernetes 请求测试策略，确保策略的行为符合您提供的输入数据的预期。您还可以测试需要在干运行模式下访问运行中的群集的上下文感知策略。
* _为您的保单生成初始`ClusterAdmissionPolicy` 脚手架_：
  生成包含所有所需设置的`YAML` 文件，可使用`kubectl` 将其应用于 Kubernetes 集群。

==== 安装

`kwctl` 稳定版的二进制文件可从https://github.com/kubewarden/kwctl/releases[库中] GitHub 代码获取。要从 GitHubhttps://github.com/kubewarden/kwctl[代码库中]构建`kwctl` ，需要https://www.rust-lang.org/tools/install[] Rust 开发环境。

== 执行政策

您可以通过定义`ClusterAdmissionPolicy` 来执行策略，然后使用`kubectl` 将其部署到群集。

`kwctl` 有助于从您要执行的策略中生成`ClusterAdmissionPolicy` 。

生成`ClusterAdmissionPolicy` 并将其应用到群集后，您可以按照下面xref:quick-start.adoc#_example_enforce_your_first_policy[快速入门]中所述的步骤进行操作：

* 从策略`manifest` 生成`ClusterAdmissionPolicy` 并保存到文件中
 ** 命令： `kwctl scaffold manifest -t ClusterAdmissionPolicy <policy URI> > <"policy name".yaml>`
+
[subs="+attributes",shell]
----
$ kwctl scaffold manifest -t ClusterAdmissionPolicy registry://ghcr.io/kubewarden/policies/pod-privileged:v0.1.9
---
apiVersion: policies.kubewarden.io/v1alpha2
kind: ClusterAdmissionPolicy
metadata:
  name: privileged-pods
spec:
  module: "registry://ghcr.io/kubewarden/policies/pod-privileged:v0.1.9"
  settings: {}
  rules:
    - apiGroups:
        - ""
      apiVersions:
        - v1
      resources:
        - pods
      operations:
        - CREATE
  mutating: false
----

+

[TIP]
====
默认情况下，`name` 值设置为`generated-policy` 。
您可能需要在部署`ClusterAdmissionPolicy` 之前对其进行编辑。
上例中的名称设置为`privileged-pods` 。
====

* 将`ClusterAdmissionPolicy` 部署到 Kubernetes 集群
 ** 命令： `kubectl apply -f <"policy name".yaml>`
+
[subs="+attributes",shell]
----
$ kubectl apply -f pod-privileged-policy.yaml
clusteradmissionpolicy.policies.kubewarden.io/privileged-pods created
----

在部署`ClusterAdmissionPolicy` 后，发送到群集的所有请求都会通过策略进行评估，看它们是否在策略范围内。
