= 审计扫描仪
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: 使用{project-name} 中的 "审计扫描器 "功能验证群集的合规状态，以确保其配置正确。
:doc-persona: ["kubewarden-operator", "kubewarden-distributor", "kubewarden-integrator"]
:doc-topic: ["howto", "audit-scanner-installation"]
:doc-type: ["howto"]

:sidebar_label: 审计扫描仪
:sidebar_position: 70
:current-version: {page-origin-branch}

从`v1.7.0` 版开始，{project-name} 新增了 "审计扫描器 "功能。一个名为 "审计扫描器 "的新组件会不断检查集群中声明的资源，标记出不符合已部署{project-name} 策略的资源，并将这些结果保存到特定报告 "自定义资源 "中。

政策会随着时间的推移而演变：你需要部署新政策并更新现有政策。版本和配置设置都会改变。这可能导致群集中已有的资源不再符合要求。

审计扫描仪功能为 Kubernetes 管理员提供了一种工具，用于持续验证集群的合规性状态。

审计扫描器可以将结果保存在link:https://openreports.io[openreports.io]的自定义资源定义（CRD）（自 v1.33 起默认设置）或link:https://github.com/kubernetes-sigs/wg-policy-prototypes[OpenReports]PolicyReports（自 v1.30 起标记为过时，自 v1.30 起默认不安装也不创建，计划在未来版本中移除）中。

== 安装

审计扫描仪组件自{short-project-name} `v1.7.0` 起可用。
因此，请确保您安装的 Helm 图表的应用程序版本为`v1.7.0` 或更高。

. 安装`kubewarden-crds` 舵手图。图表默认安装所需的`PolicyReport` CRD。
+
[subs="+attributes",console]
----
helm install kubewarden-crds kubewarden/kubewarden-crds
----
+

[NOTE]
====
在 Kubewarden 1.33 中，审计扫描器功能默认将报告从 [openreports.io](https://openreports.io) 保存到 OpenReports CRD 中。

来自 K8s Policy 工作组的link:https://htmlpreview.github.io/?https://github.com/kubernetes-sigs/wg-policy-prototypes/blob/045372e558b896695b2daae92e8c7a04d4d40282/policy-report/docs/index.html[PolicyReport]CRD 默认未安装，已被标记为过时，并将在未来的版本中移除。

如果用户希望继续使用已废弃的 PolicyReports，即 1.33 之前的行为，则需要- 将`kubewarden-crds` Helm 图表中`.Values.install.installPolicyReportCRDs` 的值更改为`true`. - 将`kubewarden-controller` Helm 图表中`.Values.auditScanner.reportCRDsKind` 的值从`openreports` 更改为`policyreport`.
====
+
[CAUTION]
====
要存储策略报告的结果，需要有可用的 PolicyReport 自定义资源定义 (CRD)。如果集群中已存在必要的 PolicyReport CRD，则无法使用 kubewarden-crds 图表进行安装。在这种情况下，可以通过在图表中将`installOpenReportCRDs` 设置为`false` （如果使用的是过时的 PolicyReports，则将`installPolicyReportCRDs` 设置为`false` ）来禁用 OpenReport CRD 的安装。
这意味着 Kubewarden 堆栈不会管理这些 CRD，而由管理员负责。

有关 CRD 的更多信息，请访问link:https://github.com/kubernetes-sigs/wg-policy-prototypes[政策工作组存储库]。

====


. 安装`kubewarden-controller` 舵手图。
+
[subs="+attributes",console]
----
helm install kubewarden-controller kubewarden/kubewarden-controller
----
+

[NOTE]
====
审计扫描仪默认已启用。要禁用它，请设置`auditScanner.enable=false` 。
====
+
[NOTE]
====
如果希望 Audit Scanner 将报告保存在 PolicyReports CRDs（标记为已废弃）中，而不是默认的 OpenReports CRDs 中，请设置`auditScanner.reportCRDsKind="policyreport"` 。
====

有关{short-project-name} 安装的更多信息，请参阅xref:quick-start.adoc[快速入门指南]

默认情况下，审计扫描程序是作为https://kubernetes.io/docs/concepts/workloads/controllers/cron-jobs[] Cronjob 执行的，每 60 分钟触发一次。你可以通过修改 kubewarden-controller charthttps://github.com/kubewarden/helm-charts/blob/main/charts/kubewarden-controller/values.yaml[values.yaml] 来调整这一设置和其他审计扫描仪设置。

有关审计扫描仪的更多信息，请参见xref:explanations/audit-scanner/audit-scanner.adoc[此处]。

=== 政策报告用户界面（可选）

`kubewarden-controller` 图表附有https://kyverno.github.io/policy-reporter[Policy Reporter] 的子图表。默认情况下禁用，可通过设置`auditScanner.policyReporter=true` 启用。Policy Reporter 子图表值位于`kubewarden-controller` 值的`policyReporter` 键下。

这只安装了 Policy Reporter 上游图表的一部分，即 UI，它提供了群集中 PolicyReports 和 ClusterPolicyReports 的可视化。有关策略报告程序用户界面的更多信息，请参阅xref:explanations/audit-scanner/audit-scanner.adoc[审计扫描仪]。

默认情况下，"策略报告程序用户界面 "仅作为 ClusterIP 服务公开，其名称为`kubewarden-controller-ui` ，安装名称空间为

`kubewarden-controller` 安装了图表。

==== 入侵

用户可以提供自己的入口配置，或通过的子图表配置https://github.com/kyverno/policy-reporter/blob/policy-reporter-2.19.4/charts/policy-reporter/charts/ui/values.yaml#L172-L189[此处] 启用入口。

请参阅该子图的入口配置示例：

[subs="+attributes",yaml]
----
auditScanner:
  policyReporter: true
policy-reporter: # subchart values settings
  ui:
    enabled: true
    ingress:
      enabled: true
      hosts:
        - host: "*.local" # change this to your appropriate domain
          paths:
            - path: /
              pathType: ImplementationSpecific
----

==== 端口转发

为了快速查看或调试，可以设置端口转发服务：

[subs="+attributes",console]
----
kubectl port-forward service/kubewarden-controller-ui 8082:8080 -n kubewarden
----

这样，政策报告用户界面就可以在`\http://localhost:8082` 上使用。

== 触发手动运行

审计扫描器以 Cronjob 的形式运行，默认情况下每 60 分钟运行一次。运行以下命令可以触发手动运行：

[subs="+attributes",bash]
----
kubectl create job \
    --namespace kubewarden \
    --from cronjob/audit-scanner \
    audit-scanner-manual-$(date +%Y-%m-%d-%H-%M-%S)
----

您可以使用以下命令查看任务状态：

[subs="+attributes",console]
----
kubectl get -n kubewarden jobs
----
