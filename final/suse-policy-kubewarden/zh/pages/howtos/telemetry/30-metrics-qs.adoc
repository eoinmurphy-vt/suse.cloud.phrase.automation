本节介绍如何在策略服务器上启用指标报告。

[NOTE]
====

在继续之前，请确保已完成本书前面的xref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[OpenTelemetry]部分。您需要它才能使本部分正常工作。

====

您可以使用https://prometheus.io/[Prometheus]收集策略服务器暴露的指标。

== 安装 Prometheus

您可以使用https://github.com/prometheus-operator/prometheus-operator[普罗米修斯操作器]，让我们定义普罗米修斯的目标。

安装和设置 Prometheus 的方法有很多种。为便于部署，应使用 Prometheus 社区 Helm 图表。

与此 Helm 图表一起部署的 Prometheus Operator 使用https://github.com/prometheus-operator/prometheus-operator/blob/master/Documentation/design.md#servicemonitor[服务监控器]，以定义 Prometheus 监控的服务。

在您的案例中，您正在添加一个 ServiceMonitor，目标是`kubewarden`命名空间中与标签`app=kubewarden-policy-server-default`和`app.kubernetes.io/name: kubewarden-controller` 匹配的服务。这将配置 Prometheus 操作员来检查哪些 Kubernetes 端点属于符合这些条件的服务。

您可以创建名为`kubewarden-controller` 和`kubewarden-policy-server` 的两个 ServiceMonitors，供 Helm 图表安装的默认 Prometheus 实例使用。

为此，您应创建以下值文件：

[subs="+attributes",console]
----
cat <<EOF > kube-prometheus-stack-values.yaml
prometheus:
  additionalServiceMonitors:
    - name: kubewarden
      selector:
        matchLabels:
          app: kubewarden-policy-server-default
      namespaceSelector:
        matchNames:
          - kubewarden
      endpoints:
        - port: metrics
          interval: 10s
    - name: kubewarden-controller
      selector:
        matchLabels:
          app.kubernetes.io/name: kubewarden-controller
      namespaceSelector:
        matchNames:
          - kubewarden
      endpoints:
        - port: metrics
          interval: 10s
EOF
----

您应该安装 Prometheus 堆栈舵手图：

[NOTE]
====
在撰写本报告时（2023-11-17），最新的图表版本是 `51.5.3`
====


[subs="+attributes",console]
----
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts

helm install --wait --create-namespace \
  --namespace prometheus \
  --version 51.5.3 \
  --values kube-prometheus-stack-values.yaml \
  prometheus prometheus-community/kube-prometheus-stack
----

== 安装 {short-project-name}

现在，您可以按照建议的方式，使用 Helm 图表安装{short-project-name} 。

[NOTE]
====

`cert-manager` 是 OpenTelemetry 的一项要求，但您已在本文档的前一部分安装了它。

====


首先，应添加包含{short-project-name} 的 Helm 资源库：

[subs="+attributes",console]
----
helm repo add kubewarden https://charts.kubewarden.io
----

然后安装由{short-project-name} 定义的自定义资源定义（CRD）：

[subs="+attributes",console]
----
helm install --wait \
  --namespace kubewarden --create-namespace \
  kubewarden-crds kubewarden/kubewarden-crds
----

现在，您可以部署{short-project-name} 堆栈的其余部分。Helm 官方图表创建了一个名为`default` 的 PolicyServer。

您应该配置 Helm 图表，以便在{short-project-name} 中启用度量指标。`kubewarden-values.yaml` 文件应包含以下内容：

[subs="+attributes",yaml]
----
telemetry:
  mode: sidecar
  metrics: True
  sidecar:
    metrics:
      port: 8080
----

现在，安装 Helm 图表：

[subs="+attributes",console]
----
helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  --values kubewarden-values.yaml \
  kubewarden-controller kubewarden/kubewarden-controller

helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  kubewarden-defaults kubewarden/kubewarden-defaults \
  --set recommendedPolicies.enabled=True \
  --set recommendedPolicies.defaultPolicyMode=monitor
----

这样就创建了`default` 的`PolicyServer` 实例：

[subs="+attributes",console]
----
kubectl get policyservers.policies.kubewarden.io
NAME      AGE
default   3m7s
----

默认情况下，该策略服务器未启用度量功能。

== 访问普罗米修斯

Prometheus 提供了一个用户界面，您可以用它来检查 Kubernetes 集群中不同组件暴露的指标。

您可以转发 Prometheus 端口，这样就可以访问它了。

[subs="+attributes",console]
----
kubectl port-forward -n prometheus --address 0.0.0.0 svc/prometheus-operated 9090
----

现在，您可以访问`9090` 端口上的 Prometheus 并执行查询，例如：`kubewarden_policy_evaluations_total` 。您可以看到，随着时间的推移，通过该政策提出的申请越来越多，评估的数量也在增加。

== 访问 Grafana

您可以转发 Grafana 服务，以便访问它。

[subs="+attributes",console]
----
kubectl port-forward -n prometheus --address 0.0.0.0 svc/prometheus-grafana 8080:80
----

现在可以使用默认用户名`admin` 和密码`prom-operator` 登录。

=== 使用{short-project-name} Grafana 仪表板

{short-project-name} 有一个 Grafana 仪表盘，其中包含基本指标，概述了{short-project-name} 在集群中的表现。该仪表盘以https://github.com/kubewarden/policy-server/releases/latest/download/kubewarden-dashboard.json[JSON 文件] 的形式发布在{short-project-name} policy-server 软件仓库的 GitHub 版本中，或在https://grafana.com/grafana/dashboards/15314[Grafana 网站]上提供。

要将仪表盘导入环境，可以从 Grafana 网站或软件仓库下载 JSON 文件：

[subs="+attributes",console]
----
curl https://github.com/kubewarden/policy-server/releases/latest/download/kubewarden-dashboard.json
----

获得文件后，您应该访问 Grafana 面板，然后https://grafana.com/docs/grafana/latest/dashboards/export-import/#import-dashboard[导入]。访问 Grafana 面板中的`/dashboard/import` ，然后按照以下步骤操作：

. 将 JSON 文件内容复制并粘贴到 Grafana UI 的`Import via panel json` 框中。
. 单击`Load` 按钮。
. 选择`Prometheus` 作为数据源。
. 单击`Import` 按钮。

另一种方法是从 grafana.com 网站直接导入。为此：

. 从https://grafana.com/grafana/dashboards/15314[仪表板页面]复制仪表板 ID、
. 将其粘贴到`Import via grafana.com` 字段
. 单击`load` 按钮。
. 导入仪表板后，定义要使用的 Prometheus 数据源并完成导入过程。

Grafana 面板上的窗格显示了由{short-project-name} 管理的所有策略的状态。它还设有针对具体政策的小组。

通过更改`policy_name` 变量的值以匹配所需的策略名称，可以获得特定策略的详细指标。
