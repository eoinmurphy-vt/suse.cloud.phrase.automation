本节介绍如何为 Policy Server 启用跟踪支持。

[NOTE]
====

在继续之前，请确保您已完成本文档前面的xref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[OpenTelemetry]部分。它是本节正常工作的必要条件。

====


跟踪功能可让您收集有关政策评估的详细资料。它是调试{project-name} 部署和策略问题的有用工具。

您使用https://www.jaegertracing.io/[] Jaeger-- 用于接收、存储和可视化跟踪事件。

== 安装积架

您可以使用https://github.com/jaegertracing/jaeger-operator[机甲操作员]来管理所有不同的机甲组件。您可以使用舵手图表安装积家操作器。

[NOTE]
====

在撰写本文时（2022-06-21），只有特定版本的 Jaeger 与 Cert Manager 兼容，https://github.com/jaegertracing/helm-charts/blob/main/charts/jaeger-operator/COMPATIBILITY.md[参见兼容性图表]。

====

安装舵手图：

[subs="+attributes",console]
----
helm repo add jaegertracing https://jaegertracing.github.io/helm-charts

helm upgrade -i --wait \
  --namespace jaeger \
  --create-namespace \
  --version 2.49.0 \
  jaeger-operator jaegertracing/jaeger-operator \
  --set rbac.clusterRole=true
----

[CAUTION]
====

这不*适合用于生产部署*。您应该阅读 Jaeger 的https://www.jaegertracing.io/docs/latest/kubernetes/[官方文档]。

====

创建机甲战士资源：

[subs="+attributes",console]
----
kubectl apply -f - <<EOF
apiVersion: jaegertracing.io/v1
kind: Jaeger
metadata:
  name: my-open-telemetry
  namespace: jaeger
spec: {}
EOF
----

创建积家操作员资源后，您将在`my-open-telemetry-collector.jaeger.svc.cluster.local` 下拥有一个服务。Jaeger 查询用户界面由`jaeger` 命名空间中的`my-open-telemetry-query` 服务提供。

== 安装 {short-project-name}

现在，您可以按常规方式部署{short-project-name} 。

[NOTE]
====

`cert-manager` 是 OpenTelemetry 的一项要求，但您已在本文档的前一部分安装了它。

====

第一步，添加包含{short-project-name} 的 Helm 资源库：

[subs="+attributes",console]
----
helm repo add kubewarden https://charts.kubewarden.io
----

然后安装由{short-project-name} 定义的自定义资源定义（CRD）：

[subs="+attributes",console]
----
helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  kubewarden-crds kubewarden/kubewarden-crds
----

现在，您可以部署{short-project-name} 堆栈的其余部分。`kubewarden-defaults` 官方舵图创建了一个名为`default` 的 PolicyServer。您希望启用此 PolicyServer 实例的跟踪功能。

为此，您需要为`kubewarden-controller` 图表指定一些额外的值。您应创建一个`values.yaml` 文件，内容如下：

[subs="+attributes",yaml]
----
telemetry:
  mode: sidecar
  tracing: True
  sidecar:
    tracing:
      jaeger:
        endpoint: "my-open-telemetry-collector.jaeger.svc.cluster.local:4317"
        tls:
          insecure: true
----

[CAUTION]
====

为简单起见，OpenTelemetry采集器和积家终端之间的通信没有加密。

同样，这也不*适合生产部署*。请查阅 Jaeger'shttps://www.jaegertracing.io/docs/latest/kubernetes/[官方文档]。

====

然后就可以继续安装舵图了：

[subs="+attributes",console]
----
helm install --wait --namespace kubewarden --create-namespace \
  --values values.yaml \
  kubewarden-controller kubewarden/kubewarden-controller

helm install --wait --namespace kubewarden --create-namespace \
  kubewarden-defaults kubewarden/kubewarden-defaults
----

这样就创建了`default` 的`PolicyServer` 实例：

[subs="+attributes",console]
----
kubectl get policyservers.policies.kubewarden.io
NAME      AGE
default   3m7s
----

仔细观察运行 PolicyServer 实例的 Pod，可以看到其中有两个容器。`policy-server` 和 OpenTelemetry Collector sidecar`otc-container`.

== 执行政策

首先部署https://github.com/kubewarden/safe-labels-policy[安全标签]策略。

您希望仅在标签`environment`值为`production` 的命名空间中执行策略。

到具有此类标签的命名空间：

[subs="+attributes",console]
----
kubectl apply -f - <<EOF
apiVersion: v1
kind: Namespace
metadata:
  name: team-alpha-prod
  labels:
    environment: production
EOF
----

接下来，您可以定义 ClusterAdmissionPolicy：

[subs="+attributes",yaml]
----
kubectl apply -f - <<EOF
apiVersion: policies.kubewarden.io/v1
kind: ClusterAdmissionPolicy
metadata:
  name: safe-labels
spec:
  module: registry://ghcr.io/kubewarden/policies/safe-labels:v0.1.6
  settings:
    mandatory_labels:
    - owner
  rules:
    - apiGroups:
        - apps
      apiVersions:
        - v1
      resources:
        - deployments
      operations:
        - CREATE
        - UPDATE
  namespaceSelector:
    matchExpressions:
    - key: environment
      operator: In
      values: ["production"]
  mutating: false
EOF
----

您需要等待保单生效：

[subs="+attributes",console]
----
kubectl wait --for=condition=PolicyActive clusteradmissionpolicy/safe-labels
----

保单激活后，您就可以试用了：

[subs="+attributes",console]
----
kubectl apply -f - <<EOF
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  namespace: team-alpha-prod
  labels:
    owner: octocat
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 0
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
EOF
----

政策允许创建此部署对象，因为它不违反政策。

策略阻止此部署对象：

[subs="+attributes",console]
----
kubectl apply -f - <<EOF
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment-without-labels
  namespace: team-alpha-prod
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 0
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
EOF
----

该策略不会在另一个命名空间中执行。

该命令将创建一个名为`team-alpha-staging` 的新命名空间：

[subs="+attributes",console]
----
kubectl apply -f - <<EOF
apiVersion: v1
kind: Namespace
metadata:
  name: team-alpha-staging
  labels:
    environment: staging
EOF
----

该策略允许在`team-alpha-staging` 命名空间中创建不带任何标签的部署资源：

[subs="+attributes",console]
----
kubectl apply -f - <<EOF
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment-without-labels
  namespace: team-alpha-staging
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 0
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
EOF
----

不出所料，该资源已成功创建。

== 探索积家用户界面

您可以看到 PolicyServer 实例向 Jaeger 发送的跟踪事件，因为用户界面上列出了一个新服务`kubewarden-policy-server` ：

image::jaeger-ui-home.png[积家仪表盘]

Jaeger 收集器正在接收 PolicyServer 生成的跟踪信息。

如果您在上一指南中安装了特莱菲克，请使用`Ingress` 公开 Jaeger Query UI：

[,yaml]
----
kubectl apply -f - <<EOF
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: my-open-telemetry-query
  namespace: jaeger
spec:
  ingressClassName: traefik
  rules:
    - http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: my-open-telemetry-query
                port:
                  number: 16686
EOF
----

要使用特拉菲克访问积家用户界面，请运行

[,console]
----
kubectl -n traefik port-forward service/traefik 8080:80
----

然后访问`\http://localhost:8080` 。

如果您想绕过特拉菲克，请直接访问 Jaeger：

[,console]
----
kubectl -n jaeger port-forward service/my-open-telemetry-query 16686
----

然后访问`\http://localhost:16686` 。
