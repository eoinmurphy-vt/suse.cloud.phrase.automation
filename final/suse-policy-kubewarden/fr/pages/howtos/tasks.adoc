=== `kwctl` Outil CLI

`kwctl` est l'outil CLI de {short-project-name} qui permet aux auteurs de politiques et aux administrateurs de clusters de tester les politiques avant de les appliquer à un cluster Kubernetes.

Cet outil a une interface similaire à celle de l'outil CLI `docker`.

==== Cas d'utilisation

Vous pouvez utiliser `kwctl` pour vous aider dans ces scénarios :

_En tant qu'auteur de politiques_

* _Test de bout en bout de votre politique_: Testez votre politique contre des requêtes Kubernetes élaborées et assurez-vous que votre politique se comporte comme vous l'attendez. Vous pouvez même tester des politiques contextuelles qui nécessitent l'accès à une grappe en cours d'exécution.
* _Intégrer des métadonnées dans votre module Wasm_: Le binaire contient des annotations sur les autorisations dont il a besoin pour être exécuté. Vous pouvez inspecter et modifier ces annotations à l'aide de `kwctl`.
* _Publier les politiques dans les registres du BCI_: Le binaire est un objet OCI entièrement conforme et peut être stocké dans les registres OCI.

_En tant qu'administrateur de cluster_

* _Inspecter les politiques à distance_: Étant donné une politique dans un registre OCI ou dans un serveur HTTP, afficher toutes les informations statiques sur la politique.
* _Exécution à blanc d'une politique dans votre cluster_: Testez la politique contre des requêtes Kubernetes élaborées et assurez-vous que la politique se comporte comme vous l'attendez compte tenu des données d'entrée que vous fournissez. Vous pouvez également tester les politiques contextuelles qui nécessitent l'accès à un cluster en cours d'exécution dans un mode de fonctionnement à blanc.
* _Générer un échafaudage initial `ClusterAdmissionPolicy` pour votre politique :_
  Générez un fichier `YAML` avec tous les paramètres requis, qui peut être appliqué à votre cluster Kubernetes à l'aide de `kubectl`.

==== Installation

`kwctl` pour les versions stables sont disponibles sur le GitHubhttps://github.com/kubewarden/kwctl/releases[dépôt]. Pour construire`kwctl` à partir du GitHub https://github.com/kubewarden/kwctl[repo], vous avez besoin d'un environnement de développement Rust https://www.rust-lang.org/tools/install[].

== Appliquer les politiques

Vous appliquez une politique en définissant un `ClusterAdmissionPolicy` et en le déployant ensuite dans votre cluster à l'aide de `kubectl`.

`kwctl` permet de générer une adresse `ClusterAdmissionPolicy` à partir de la politique que vous souhaitez appliquer.

Après avoir généré le site `ClusterAdmissionPolicy` et l'avoir appliqué à votre cluster, vous pouvez suivre les étapes décrites dans la sectionxref:quick-start.adoc#_example_enforce_your_first_policy[Démarrage rapide] ci-dessous :

* Générer le site `ClusterAdmissionPolicy` à partir de la politique `manifest` et l'enregistrer dans un fichier.
 ** Commandement : `kwctl scaffold manifest -t ClusterAdmissionPolicy <policy URI> > <"policy name".yaml>`
+
[subs="+attributes",shell]
----
$ kwctl scaffold manifest -t ClusterAdmissionPolicy registry://ghcr.io/kubewarden/policies/pod-privileged:v0.1.9
---
apiVersion: policies.kubewarden.io/v1alpha2
kind: ClusterAdmissionPolicy
metadata:
  name: privileged-pods
spec:
  module: "registry://ghcr.io/kubewarden/policies/pod-privileged:v0.1.9"
  settings: {}
  rules:
    - apiGroups:
        - ""
      apiVersions:
        - v1
      resources:
        - pods
      operations:
        - CREATE
  mutating: false
----

+

[TIP]
====
Par défaut, la valeur `name` est fixée à `generated-policy`.
Vous pouvez l'éditer avant de déployer le site `ClusterAdmissionPolicy`.
Dans l'exemple précédent, le nom est défini comme suit : `privileged-pods`.
====

* Déployez le site `ClusterAdmissionPolicy` sur votre cluster Kubernetes
 ** Commandement : `kubectl apply -f <"policy name".yaml>`
+
[subs="+attributes",shell]
----
$ kubectl apply -f pod-privileged-policy.yaml
clusteradmissionpolicy.policies.kubewarden.io/privileged-pods created
----

Après avoir déployé le site `ClusterAdmissionPolicy`, toutes les requêtes envoyées à votre cluster sont évaluées par la politique si elles entrent dans son champ d'application.
