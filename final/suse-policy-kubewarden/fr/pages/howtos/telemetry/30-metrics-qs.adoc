Cette section explique comment activer les rapports de métriques sur le serveur de stratégie.

[NOTE]
====

Avant de continuer, assurez-vous d'avoir terminé la section précédente surxref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[OpenTelemetry]de ce livre. Vous en avez besoin pour que cette section fonctionne correctement.

====

Vous utilisez https://prometheus.io/[Prometheus] pour collecter les métriques exposées par le serveur de stratégie.

== Installer Prometheus

Vous utilisez https://github.com/prometheus-operator/prometheus-operator[Prometheus Operator], qui nous permet de définir les cibles de Prometheus.

Il existe de nombreuses façons d'installer et de configurer Prometheus. Pour faciliter le déploiement, vous devriez utiliser le diagramme Helm de la communauté Prometheus.

L'opérateur Prometheus déployé avec ce diagramme de Helm utilisehttps://github.com/prometheus-operator/prometheus-operator/blob/master/Documentation/design.md#servicemonitor[Service Monitors], pour définir les services que Prometheus surveille.

Dans votre cas, vous ajoutez un ServiceMonitor ciblant l'espace de noms `kubewarden`pour les services correspondant aux étiquettes `app=kubewarden-policy-server-default`et `app.kubernetes.io/name: kubewarden-controller`. Cela configure l'opérateur Prometheus pour qu'il inspecte les points d'extrémité Kubernetes qui appartiennent à des services correspondant à ces conditions.

Vous pouvez créer deux ServiceMonitors nommés `kubewarden-controller` et`kubewarden-policy-server` qui seront utilisés par l'instance Prometheus par défaut installée par la carte Helm.

Pour ce faire, vous devez créer le fichier de valeurs suivant :

[subs="+attributes",console]
----
cat <<EOF > kube-prometheus-stack-values.yaml
prometheus:
  additionalServiceMonitors:
    - name: kubewarden
      selector:
        matchLabels:
          app: kubewarden-policy-server-default
      namespaceSelector:
        matchNames:
          - kubewarden
      endpoints:
        - port: metrics
          interval: 10s
    - name: kubewarden-controller
      selector:
        matchLabels:
          app.kubernetes.io/name: kubewarden-controller
      namespaceSelector:
        matchNames:
          - kubewarden
      endpoints:
        - port: metrics
          interval: 10s
EOF
----

Vous devez installer la pile Prometheus Helm Chart :

[NOTE]
====
Au moment où nous écrivons ces lignes (2023-11-17), la dernière version du graphique est la suivante `51.5.3`
====


[subs="+attributes",console]
----
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts

helm install --wait --create-namespace \
  --namespace prometheus \
  --version 51.5.3 \
  --values kube-prometheus-stack-values.yaml \
  prometheus prometheus-community/kube-prometheus-stack
----

== Installer {short-project-name}

Vous pouvez maintenant installer {short-project-name}, de la manière recommandée avec les cartes Helm.

[NOTE]
====

Le site `cert-manager` est une exigence d'OpenTelemetry, mais vous l'avez déjà installé dans une section précédente de cette documentation.

====


Tout d'abord, vous devez ajouter le dépôt Helm qui contient {short-project-name}:

[subs="+attributes",console]
----
helm repo add kubewarden https://charts.kubewarden.io
----

Vous installez ensuite les définitions de ressources personnalisées (CRD) définies par{short-project-name}:

[subs="+attributes",console]
----
helm install --wait \
  --namespace kubewarden --create-namespace \
  kubewarden-crds kubewarden/kubewarden-crds
----

Vous pouvez maintenant déployer le reste de la pile {short-project-name}. La charte officielle de Helm crée un PolicyServer nommé `default`.

Vous devez configurer le diagramme Helm de manière à ce que les métriques soient activées dans {short-project-name}. Le fichier `kubewarden-values.yaml` doit contenir les éléments suivants :

[subs="+attributes",yaml]
----
telemetry:
  mode: sidecar
  metrics: True
  sidecar:
    metrics:
      port: 8080
----

Installez maintenant les cartes Helm :

[subs="+attributes",console]
----
helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  --values kubewarden-values.yaml \
  kubewarden-controller kubewarden/kubewarden-controller

helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  kubewarden-defaults kubewarden/kubewarden-defaults \
  --set recommendedPolicies.enabled=True \
  --set recommendedPolicies.defaultPolicyMode=monitor
----

Cela crée l'instance `default` de `PolicyServer`:

[subs="+attributes",console]
----
kubectl get policyservers.policies.kubewarden.io
NAME      AGE
default   3m7s
----

Par défaut, les métriques ne sont pas activées pour ce serveur de politiques.

== Accès à Prometheus

Prometheus expose une interface utilisateur que vous pouvez utiliser pour inspecter les métriques exposées par différents composants au sein de votre cluster Kubernetes.

Vous pouvez transférer le port Prometheus afin de pouvoir y accéder.

[subs="+attributes",console]
----
kubectl port-forward -n prometheus --address 0.0.0.0 svc/prometheus-operated 9090
----

Vous pouvez maintenant visiter Prometheus sur le port `9090` et effectuer une requête, par exemple :`kubewarden_policy_evaluations_total`. Vous pouvez constater que le nombre d'évaluations augmente avec le temps, au fur et à mesure que le nombre de demandes passe par la politique.

== Accéder à Grafana

Vous pouvez transférer le service Grafana afin de pouvoir y accéder.

[subs="+attributes",console]
----
kubectl port-forward -n prometheus --address 0.0.0.0 svc/prometheus-grafana 8080:80
----

Vous pouvez maintenant vous connecter avec le nom d'utilisateur par défaut `admin` et le mot de passe`prom-operator`.

=== Utilisation du tableau de bord {short-project-name} Grafana

{short-project-name} dispose d'un tableau de bord Grafana avec des métriques de base donnant un aperçu de la façon dont {short-project-name} se comporte dans le cluster. Ce tableau de bord est disponible dans les versions GitHub du référentiel {short-project-name} policy-server en tant quehttps://github.com/kubewarden/policy-server/releases/latest/download/kubewarden-dashboard.json[fichier JSON] ou sur le https://grafana.com/grafana/dashboards/15314[site web].

Pour importer le tableau de bord dans votre environnement, vous pouvez télécharger le fichier JSON depuis le site web de Grafana ou le dépôt :

[subs="+attributes",console]
----
curl https://github.com/kubewarden/policy-server/releases/latest/download/kubewarden-dashboard.json
----

Une fois que vous avez le fichier, vous devez accéder au tableau de bord Grafana ethttps://grafana.com/docs/grafana/latest/dashboards/export-import/#import-dashboard[import it]. Visitez `/dashboard/import` dans le tableau de bord Grafana et suivez ces étapes :

. Copiez et collez le contenu du fichier JSON dans la boîte `Import via panel json` de l'interface utilisateur de Grafana.
. Cliquez sur le bouton `Load`.
. Choisissez `Prometheus` comme source.
. Cliquez sur le bouton `Import`.

Une autre option consiste à l'importer directement depuis le site grafana.com. Pour cela :

. Copiez l'identifiant du tableau de bord à partir de la tableau de bord https://grafana.com/grafana/dashboards/15314[page du]
. Collez-la dans le champ `Import via grafana.com` 
. Cliquez sur le bouton `load`.
. Après avoir importé le tableau de bord, définissez la source de données Prometheus à utiliser et terminez le processus d'importation.

Le tableau de bord Grafana comporte des volets montrant l'état de toutes les politiques gérées par{short-project-name}. Il existe également des groupes de travail spécifiques à chaque politique.

Vous pouvez obtenir des mesures détaillées pour une politique spécifique en modifiant la valeur de la variable `policy_name` pour qu'elle corresponde au nom de la politique requise.
