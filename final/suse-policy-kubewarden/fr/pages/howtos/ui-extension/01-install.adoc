Cette section décrit l'installation de l'interface utilisateur {project-name} en tant qu'extension de https://github.com/rancher/rancher[Manager] Rancher.

== Installer l'extension {short-project-name} UI

Vous installez l'interface {short-project-name} en tant qu'extension globale, mais vous installez le contrôleur {short-project-name} via l'interface Rancher en tant que ressource de cluster.

[NOTE]
====

Pour les installations à gaine d'air, suivre les xref:howtos/airgap/02-install.adoc[étapes suivantes.]

====


Dans la page Extensions, sélectionnez le bouton "Enable" et choisissez l'option pour ajouter le Rancher Extensions Repository. Lorsqu'il est activé, l'élément d'extension "{short-project-name}" apparaît automatiquement. Sélectionnez cet élément pour installer l'extension. Une fois installé, vous pouvez installer {short-project-name} dans le cluster requis.

=== Installer {short-project-name}

En suivant les étapes précédentes, dans votre cluster, un nouvel élément apparaît dans le menu latéral pour {short-project-name}. Cette page du tableau de bord vous guide tout au long du processus d'installation, en remplissant les conditions préalables.

[NOTE]
====

Lors de l'étape "App Install" de l'assistant d'installation, le bouton "Installer{short-project-name}" peut rester grisé. Si cela se produit, actualisez la page et revenez à cette étape.

====


== Après l'installation

Une fois l'installation terminée, la page du tableau de bord et le menu latéral contiennent désormais de nouveaux éléments, à savoir les serveurs de politiques, les politiques d'admission des clusters et les politiques d'admission. À partir de là, vous pouvez créer des serveurs de politiques et des politiques pour contrôler le comportement au sein de votre cluster.

*_Vue du tableau de bord_*image:ui_dashboard.png[Tableau de bord de l]'interface utilisateur

=== Activation du serveur de stratégies et des stratégies par défaut

Dans la page du tableau de bord, vous pouvez sélectionner le bouton "Installer la carte" pour installer la carte Helm.https://github.com/kubewarden/helm-charts/tree/main/charts/kubewarden-defaults[`kubewarden-defaults`]Helm. Ce tableau comprend le serveur de stratégies par défaut et quelques stratégies curatives.

Après avoir installé la carte, vous pouvez visualiser les détails du serveur de politiques par défaut avec les politiques associées dans un tableau triable.

*_Vue détaillée du serveur de politiques_*image:ui_policyserver_detail.png[] UI

=== Création de politiques

Lors de la création de politiques, l'option "Politique personnalisée" est proposée dans la grille des politiques. Fournissez les informations requises pour le nom, le module et les règles de votre police. Il est recommandé d'ajouter le référentiel Policy Catalog pour accéder aux politiques officielles de {short-project-name}.

*_Création d'une politique personnalisée_*image:ui_policy_custom.png[Liste blanche des règles de l']interface utilisateur

=== Caractéristiques supplémentaires

Suivez les instructions pour inclure laxref:howtos/ui-extension/02-metrics.adoc[surveillance] ou lexref:howtos/ui-extension/03-tracing.adoc[traçage].

== Installation à gaine d'air

Comme {short-project-name} est une extension officielle de Rancher, l'équipe de Rancher fournit un mécanisme pour générer automatiquement une image de catalogue d'extension. Ceci est ajouté au fichier `rancher-images.txt` lors dehttps://ranchermanager.docs.rancher.com/getting-started/installation-and-upgrade/other-installation-methods/air-gapped-helm-cli-install/publish-images#1-find-the-required-assets-for-your-rancher-version[installation de Rancher Manager] pour les instances air-gapped.

Une fois que cette image est mise en miroir dans un registre accessible à votre cluster air-gapped, vous pouvez importer l'image dans l'interface utilisateur de Rancher. Cela crée un dépôt local Helm avec la charte de l'interface utilisateur {short-project-name} pour l'installation.

=== Étapes de l'installation

. https://ranchermanager.docs.rancher.com/how-to-guides/new-user-guides/kubernetes-resources-setup/secrets[Créer]un secret de registre dans l'espace de noms `cattle-ui-plugin-system`. Saisissez le domaine de l'adresse de l'image dans le champ *Nom de domaine du registre*.
. Retournez à la page des *extensions* (par exemple, `https://cluster-ip/dashboard/c/local/uiplugins`).
. En haut à droite, sélectionnez *Gérer les catalogues d'extensions.* image:ui_airgap_01.png[Gérer les catalogues]
. Sélectionnez le bouton *Importer un catalogue d'extension*. \image:ui_airgap_02.png[Importer des catalogues]
. Saisissez l'adresse de l'image dans le champ *Référence de l'image du catalogue*.
. Sélectionnez le secret que vous venez de créer dans le menu déroulant *Pull Secrets*. image:ui_airgap_03.png[Saisir les informations du catalogue]
. Cliquez sur *Charger*. L'extension est désormais *en attente.*
. Retournez à la page *Extensions*.
. Sélectionnez l'onglet *Disponible* et cliquez sur le bouton *Recharger* pour vous assurer que la liste des extensions est à jour. image:ui_airgap_04.png[Installer {short-project-name}]
. Trouvez l'extension {short-project-name} que vous venez d'ajouter et sélectionnez le bouton *Installer*.
