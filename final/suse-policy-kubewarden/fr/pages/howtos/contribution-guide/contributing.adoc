_{short-project-name} accueille favorablement les suggestions d'amélioration de tous les contributeurs, qu'ils soient nouveaux ou expérimentés._

Vous pouvez contribuer à la documentation de {short-project-name} en

* Améliorer le contenu existant
* Créer un nouveau contenu
* _(à venir_) Aide à la traduction de la documentation

== Pour commencer

ifeval::["{build-type}" == "community"]

Tout le monde est invité à suggérer des changements en soumettant une demande d'extraction. Vous pouvez également signaler un bogue ou une erreur à l'aide d'une question GitHub sur lehttps://github.com/kubewarden/docs[dépôt]

S'impliquer dans la documentation :

* Familiarisez-vous avec le https://github.com/kubewarden/docs[référentiel de documentation] et le https://docusaurus.io/[].
* Docusaurus utilise la syntaxe Markdown et utilise https://mdxjs.com/[MDX]comme analyseur.
* Comprendre comment xref:howtos/contribution-guide/suggesting-an-improvement.adoc[suggérer une amélioration] et ouvrir une pull request.

endif::[]

ifeval::["{build-type}" == "product"]

Tout le monde est invité à suggérer des changements en soumettant une demande d'extraction. Vous pouvez également signaler un bogue ou une erreur à l'aide d'une question GitHub sur lehttps://github.com/rancher/kubewarden-product-docs[dépôt]

S'impliquer dans la documentation :

* Familiarisez-vous avec le https://github.com/rancher/kubewarden-product-docs[référentiel de documentation] et le https://antora.org/[].
* Antora utilise la syntaxe AsciiDoc.
* Comprendre comment xref:howtos/contribution-guide/suggesting-an-improvement.adoc[suggérer une amélioration] et ouvrir une pull request.

endif::[]
