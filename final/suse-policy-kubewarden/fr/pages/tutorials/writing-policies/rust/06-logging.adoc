Vous pouvez demander à votre politique d'effectuer une journalisation.
Le site `policy-server` ou `kwctl` transmet ces entrées de journal avec les informations appropriées.

La bibliothèque de journalisation choisie pour le SDK Rust esthttps://github.com/slog-rs/slog[`slog`].
Il s'agit d'une caisse populaire et bien connue qui s'intègre parfaitement à {project-name}.

== Initialisation de l'enregistreur

Le projet recommande de créer un puits global auquel vous pouvez vous connecter, à partir de n'importe quel endroit de votre politique.
Pour cela, utilisez la caisse `lazy_static`:

[subs="+attributes",rust]
----
use slog::{o, Logger};

lazy_static! {
    static ref LOG_DRAIN: Logger = Logger::root(
        logging::KubewardenDrain::new(),
        o!("policy" => "sample-policy")
    );
}
----

== Consommer l'enregistreur

Désormais, à partir des fonctions `validate` ou `validate_settings`, vous pouvez enregistrer les données à l'aide des macros exportées par `slog` qui correspondent à chaque niveau d'enregistrement pris en charge :

[subs="+attributes",rust]
----
use slog::{info, o, warn, Logger};

fn validate(payload: &[u8]) -> CallResult {
    // ...
    info!(LOG_DRAIN, "starting validation");
    // ...
    warn!(
        LOG_DRAIN, "structured log";
        "some_resource_name" => &some_resource_name
    );
    // ...
}
----

La bibliothèque `slog` envoie tous les journaux au drain initialisé dans la variable globale.
Cette opération est synchronisée avec l'évaluateur qui exécute la politique.
Il s'agit de `kwctl` ou de `policy-server`.
L'évaluateur de politique consigne ensuite ces informations, en y ajoutant d'autres informations contextuelles connues, telles que la requête Kubernetes `uid`.

De plus amples informations sur leshttps://docs.rs/slog/2.7.0/slog/macro.log.html[macros d'enregistrement]proposées par slog sont disponibles dans lahttps://docs.rs/slog/2.7.0/slog/index.html[].
