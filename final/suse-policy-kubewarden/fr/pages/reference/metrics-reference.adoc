== Serveur de politique

Le composant Policy Server initialise et exécute les politiques.
Lorsqu'il reçoit des demandes du serveur API Kubernetes, il transmet la demande à la politique et renvoie la réponse fournie par la politique au serveur API Kubernetes.

===== Bagages

|===
| Étiquette | Description

| `policy_name`
| Nom de la police

| `resource_name`
| Nom de la ressource évaluée

| `resource_kind`
| Type de la ressource évaluée

| `resource_namespace`
| Espace de noms de la ressource évaluée. Non présent si la ressource est en cluster.

| `resource_request_operation`
| Type d'opération : `CREATE`, `UPDATE`, `DELETE`, `PATCH`, `WATCH`...

| `accepted`
| La demande a-t-elle été acceptée ?

| `mutated`
| La demande a-t-elle fait l'objet d'une mutation ?

| `error_code`
| Code d'erreur renvoyé par la politique en cas de rejet, le cas échéant.
  Non présent si la police n'en prévoyait pas.
|===
