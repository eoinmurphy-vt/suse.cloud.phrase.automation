=== `kwctl` CLIツール

`kwctl` は、{short-project-name}'のCLIツールで、ポリシー作成者とクラスタ管理者がKubernetesクラスタに適用する前にポリシーをテストするためのものだ。

このツールは、`docker` CLIツールに似たインターフェースを持っている。

==== 使用例

このような場面では、`kwctl` ：

_政策立案者として_

* _ポリシーのエンド・ツー・エンドのテスト_：細工したKubernetesリクエストに対してポリシーをテストし、ポリシーが期待通りに動作することを確認する。実行中のクラスタへのアクセスを必要とするコンテキストアウェアポリシーをテストすることもできる。
* _Wasmモジュールにメタデータを埋め込む_：バイナリには、実行に必要なパーミッションの注釈が含まれている。これらの注釈は、`kwctl` で確認したり変更したりすることができます。
* _OCI登録に方針を公表する_：バイナリは完全に準拠したOCIオブジェクトであり、OCIレジストリに格納することができる。

_クラスタ管理者として_

* _リモートポリシーを検査_します：OCI レジストリまたは HTTP サーバーにポリシーがある場合、そのポリシーに関するすべての静的情報を表示します。
* _クラスタ内のポリシーのドライラン_：作成したKubernetesリクエストに対してポリシーをテストし、提供した入力データからポリシーが期待通りに動作することを確認する。また、ドライランモードで実行中のクラスタへのアクセスを必要とするコンテキストアウェアポリシーをテストすることもできます。
* _ `ClusterAdmissionPolicy` 、保険契約の足場を作る_：
  `kubectl` を使用してKubernetesクラスタに適用できる、必要な設定をすべて含む`YAML` ファイルを生成します。

==== インストール

`kwctl` 安定版リリースのバイナリは、https://github.com/kubewarden/kwctl/releases[GitHubリポジトリから]入手できる。GitHubhttps://github.com/kubewarden/kwctl[リポジトリから]`kwctl` をビルドするには、https://www.rust-lang.org/tools/install[]Rust 開発環境が必要です。

== ポリシーの実施

`ClusterAdmissionPolicy` を定義してポリシーを適用し、`kubectl` を使用してクラスタにデプロイします。

`kwctl` は、強制したいポリシーから`ClusterAdmissionPolicy` 。

`ClusterAdmissionPolicy` を生成してクラスタに適用したら、以下のxref:quick-start.adoc#_example_enforce_your_first_policy[クイック・スタートで]説明する手順に従います：

* ポリシー`manifest` から`ClusterAdmissionPolicy` を生成し、ファイルに保存する。
 ** 命令だ： `kwctl scaffold manifest -t ClusterAdmissionPolicy <policy URI> > <"policy name".yaml>`
+
[subs="+attributes",shell]
----
$ kwctl scaffold manifest -t ClusterAdmissionPolicy registry://ghcr.io/kubewarden/policies/pod-privileged:v0.1.9
---
apiVersion: policies.kubewarden.io/v1alpha2
kind: ClusterAdmissionPolicy
metadata:
  name: privileged-pods
spec:
  module: "registry://ghcr.io/kubewarden/policies/pod-privileged:v0.1.9"
  settings: {}
  rules:
    - apiGroups:
        - ""
      apiVersions:
        - v1
      resources:
        - pods
      operations:
        - CREATE
  mutating: false
----

+

[TIP]
====
デフォルトでは、`name` の値は`generated-policy` に設定されている。
`ClusterAdmissionPolicy` をデプロイする前に編集しておくとよいだろう。
前の例の名前は`privileged-pods` に設定されている。
====

* `ClusterAdmissionPolicy` を Kubernetes クラスタにデプロイする。
 ** 命令だ： `kubectl apply -f <"policy name".yaml>`
+
[subs="+attributes",shell]
----
$ kubectl apply -f pod-privileged-policy.yaml
clusteradmissionpolicy.policies.kubewarden.io/privileged-pods created
----

`ClusterAdmissionPolicy` をデプロイした後、クラスタに送信されたすべてのリクエストは、ポリシーの範囲内であれば、ポリシーによって評価されます。
