例として、ポッド作成要求を処理する単純な検証ポリシーを作成します。

ポリシーはPodの`metadata.name` 属性を見て、無効な名前を持つPodを拒否します。
無効な名前のリストは、ポリシーのエンドユーザーが設定できるようにすべきである。

ポリシーの設定は次のようなものだ：

[subs="+attributes",yaml]
----
invalid_names:
- bad_name1
- bad_name2
----

ポリシーは以下のようなポッドの作成を受け入れるべきである：

[subs="+attributes",yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  containers:
    - name: nginx
      image: nginx:latest
----

というようなポッドの創設を拒否すべきだ：

[subs="+attributes",yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: bad_name1
spec:
  containers:
    - name: nginx
      image: nginx:latest
----

== 新政策プロジェクトの足場作り

https://github.com/kubewarden/rust-policy-template[テンプレート・プロジェクトで] `cargo generate` を使用して、新しいポリシー・プロジェクトを作成できます。

まず、`cargo-generate` をインストールする。これにはhttps://pkgs.org/download/openssl-devel[openssl-devel] が必要です。

[subs="+attributes",shell]
----
cargo install cargo-generate
----

では、次のように足場を組んでみよう：

[subs="+attributes",shell]
----
cargo generate --git https://github.com/kubewarden/rust-policy-template \
               --branch main \
               --name demo
----

このコマンドは次のような出力を生成する：

[subs="+attributes",console]
----
🔧   Creating project called `demo`...
✨   Done! New project created /<some-path-name>/demo
----

これにより、`demo` サブディレクトリに新しいポリシー・プロジェクトが作成されます。

[NOTE]
====

デモで GitHub コンテナ・レジストリの機能を利用する場合は、https://docs.github.com/en/packages/working-with-a-github-packages-registry/enabling-improved-container-support-with-the-container-registry#enabling-the-container-registry-for-your-personal-account[コンテナ・サポートの改善を有効にする]必要があります。
====


=== テスト

試してみるといい：

[subs="+attributes",console]
----
cargo test
----

これは生成された足場をテストするものだ。すべてが正しく配置されていれば、次のような出力で終わる一連のコンパイル・メッセージが表示される：

[subs="+attributes",console]
----
running 4 tests
test settings::tests::validate_settings ... ok
test tests::accept_request_with_non_pod_resource ... ok
test tests::accept_pod_with_valid_name ... ok
test tests::reject_pod_with_invalid_name ... ok

test result: ok. 4 passed; 0 failed; 0 ignored; 0 measured; 0 filtered out; finished in 0.00s
----
