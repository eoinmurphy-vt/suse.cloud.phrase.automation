このチュートリアルでは、Podオブジェクトのホスト名を検証するポリシーの作成について説明します。

ポリシーは、拒否リストにある1つ以上のホスト名を使用するすべてのPodを拒否します。
ランタイム設定を使用してポリシー・コンフィギュレーションを提供する。

要約すると、ポリシー設定は次のようになる：

[source,yaml]
----
denied_hostnames:
  - bad-host
  - forbidden-host
----

ポリシーはこのPodの作成を拒否する：

[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  hostname: bad-host
  containers:
    - name: nginx
      image: nginx:latest
----

しかし、このポッドの誕生は受け入れている：

[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  hostname: allowed-host
  containers:
    - name: nginx
      image: nginx:latest
----

== 新しい政策プロジェクトの足場作り

https://github.com/kubewarden/js-policy-template[テンプレートリポジトリを]使用して、新しいポリシープロジェクトを作成できます。ページ上部の緑色のボタン「Use this template」を選択し、GitHubのウィザードに従ってください。
リポジトリをローカルにクローンし、`package.json` ファイルを更新して、ポリシーの詳細を反映させます：

[source,json]
----
{
  "name": "your-policy-name",
  "version": "1.0.0",
  "description": "Your policy description",
  "repository": {
    "type": "git",
    "url": "https://github.com/your-username/your-policy-name"
  }
}
----

実際の GitHub リポジトリと同じリポジトリパスを使用してください。
