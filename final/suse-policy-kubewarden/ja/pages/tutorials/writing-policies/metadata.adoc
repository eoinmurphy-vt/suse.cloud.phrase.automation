:page-revdate: {revdate}
:description: metadata.yamlファイルでポリシーのメタデータを構成し、ポリシー設定、作成者情報、リソースパーミッションを定義する。
:doc-persona: ["kubewarden-policy-developer"]
:doc-topic: ["kubewarden", "writing-policies", "policy-metadata"]
:doc-type: ["tutorial"]
:keywords: ["kubewarden"、"kubernetes policy development"、"policy metadata"]。

{project-name} メタデータ・ファイル（`metadata.yaml` ）は、{short-project-name} 内で使用されるポリシーに関連する重要な情報と設定を含むコンフィギュレーション・ファイルである。
このドキュメントは、メタデータ・ファイルの目的と使用法の概要である。

ポリシー`metadata.yaml` ファイルには、ポリシーのデフォルトと、ポリシーの作成者によって設定された作成者や説明などのメタデータがあります。
`kwctl annotate` コマンドは、このファイルを使用して、ポリシーを含む`.wasm` ファイルに注釈を付ける。
そのため、政策の実行に必要な関連情報はすべて入手可能である。
ポリシーに注釈を付ける方法の詳細については、「xref:../../explanations/distributing-policies.adoc[ポリシーの配布]」ガイドを参照してください。

ポリシーユーザーがポリシーを使用したい場合、`kwctl scaffold` を使用して YAML マニフェストを生成します。
このコマンドは出荷されたWasmモジュールに埋め込まれたポリシーメタデータを読み込み、チェックを実行し、作成者がそのまま使用または修正できるYAMLマニフェストを返します。

ポリシーの作成者として、ポリシーの足場作り中に提供された`metadata.yaml` ファイルを適応させることができます。

`metadata.yaml` の例をご覧ください：

[subs="+attributes",yaml]
----
rules:
  - apiGroups: [""]
    apiVersions: ["v1"]
    resources: ["pods"]
    operations: ["CREATE"]
mutating: false
contextAwareResources: []
executionMode: kubewarden-wapc
policyType: kubernetes
backgroundAudit: true
annotations:
  # kubewarden specific:
  io.kubewarden.policy.title: My policy
  io.kubewarden.policy.version: 0.1.0 # should match the OCI tag
  io.kubewarden.policy.description: Short description
  io.kubewarden.policy.author: myself
  io.kubewarden.policy.url: https://github.com/yourorg/my-policy
  io.kubewarden.policy.source: https://github.com/yourorg/my-policy
  io.kubewarden.policy.license: Apache-2.0
  # The next two annotations are used in the policy report generated by the
  # Audit scanner. Severity indicates policy check result criticality and
  # Category indicates policy category. See more here at docs.kubewarden.io
  io.kubewarden.policy.severity: medium
  io.kubewarden.policy.category: Resource validation
  # artifacthub specific: (optional, to release in Artifact Hub)
  io.kubewarden.policy.ociUrl: ghcr.io/myorg/my-policy
  io.artifacthub.displayName: Policy Name
  io.artifacthub.resources: Pod
  io.artifacthub.keywords: pod, cool policy, kubewarden
----

== バックグラウンド監査チェックの有効化

メタデータ・ファイルには、特定のポリシーのバックグラウンド監査チェックを有効にするフラグ、`backgroundAudit` が含まれています。
デフォルトでは、このフラグは`true` に設定されている。

その仕組みやイベントの種類によって、このフィールドを`false` に設定すべきポリシーがある。
詳細は、xref:explanations/audit-scanner/limitations.adoc[監査スキャナのドキュメントの]制限事項のセクションに記載されています。

== ポリシーがアクセスできるKubernetesリソースの定義

メタデータファイル内で、`contextAwareResources` フィールドを使用して、ユーザーはポリシーがアクセスできる Kubernetes リソースを定義できる。
例えば、ポリシーが`Namespace` リソースへのアクセスを必要とする場合。
ポリシー作成者は`contextAwareResources` ：

[subs="+attributes",yaml]
----
[...]
contextAwareResources:
  - apiVersion: v1 kind: Namespace
[...]
----

== ポリシーを変異させるか変異させないかを指定する

メタデータ・ファイルには、`mutating` というフラグがあり、ポリシーを変異させるか変異させないかを設定することができる。
変異するポリシーは、入ってくるリクエストや管理されているリソースを変更する。
変異させないポリシーは、変更を加えることなく、制限を観察し、実行する。
この区別は、ポリシーがKubernetesリソースとどのように相互作用し、クラスターにどのような影響を与えるかを決定する上で極めて重要である。

== ポリシーのタイプをKubernetesまたはRawに指定する

メタデータファイルには`policyType` というフラグがあり、ユーザはポリシーを`kubernetes` または`raw` のどちらかにマークすることができます。
Kubernetesポリシーは、Kubernetesリソースを検証するポリシーである。
Rawポリシーは、任意のJSONドキュメントを検証するポリシーである。
デフォルトでは、ユーザーによって指定されない場合、ポリシーに注釈を付けるとき、このフィールドは`kubernetes` に設定されます。
詳しくは「xref:../../howtos/raw-policies.adoc[生ポリシー]」の項を参照。

== リソースタイプのターゲットの定義

メタデータ・ファイルは、ポリシーが適用されるリソース・タイプを決定する`rules` フィールド内のルールを定義する機能をユーザに提供する。
この機能により、ユーザーはポリシーの適用を正確に制御できるようになり、ポリシーが意図したリソースタイプにのみ適用されることが保証される。
このようにきめ細かく制御することで、ユーザーは、ポリシーが正確にターゲットに設定され、特定の要件に合致し、無関係なリソースタイプにポリシーが意図しない形で適用されることを避けることができる。
