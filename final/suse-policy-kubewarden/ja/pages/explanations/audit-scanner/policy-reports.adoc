{project-name} Audit Scanner を使用する場合、https://openreports.io/[OpenReports]Custom Resources を使用してポリシースキャンの結果を保存します。

[CAUTION]
====

{project-name} 1.33では、監査スキャナ機能はデフォルトで[openreports.io](https://openreports.io)からOpenReports CRDにレポートを保存します。

K8s Policyワーキンググループのlink:https://htmlpreview.github.io/?https://github.com/kubernetes-sigs/wg-policy-prototypes/blob/045372e558b896695b2daae92e8c7a04d4d40282/policy-report/docs/index.html[PolicyReport]CRDは、デフォルトではインストールされず、非推奨とマークされ、将来のリリースで削除される予定です。

非推奨のPolicyReports、つまり、1.33以前の動作を使用し続けたいユーザは、以下を行う必要があります：-`.Values.install.installPolicyReportCRDs` の`kubewarden-crds` Helm chart 値を`true` に変更する。 -`.Values.auditScanner.reportCRDsKind` の`kubewarden-controller` Helm chart 値を`openreports` から`policyreport` に変更する。

====


これらのOpenReport CRDは、監査結果を構造化して保存・管理する方法を提供します。

監査スキャナ・ポリシーのスキャン結果は、`Report` および`ClusterReport` を使用して保存します。
監査スキャナは、リソースのスコープに応じて、スキャンされた各リソースに対して`Report` または`ClusterReport` を作成します。`Report` オブジェクトはリソースのネームスペースで使用でき、`ClusterReport` オブジェクトはクラスタスコープで使用できます。

スキャナーによって生成される監査結果には以下のようなものがある：

* 政策評価
* スキャンされたリソース
* 評価結果（合格、不合格、スキップ）
* 評価がいつ行われたかを示すタイムスタンプ。

また、ポリシーに重大度やカテゴリの注釈を定義することもできます。

オペレータは、`kubectl` を使用してレポートを照会できます。また、https://kyverno.github.io/policy-reporter[policy-reporter]オープンソースプロジェクトによって提供されるオプションのUIを使用して、OpenReports CRDのモニタリングと監視を行うこともできます。

== レポートの照会

kubectl CLIを使えば、スキャン結果を問い合わせることができる：

デフォルト・ネームスペースのレポートを一覧表示します：

[source,console,subs="+attributes"]
----
$ kubectl get report -o wide
NAME                                   KIND   NAME            PASS   FAIL   WARN   ERROR   SKIP   AGE
44da974b-6057-4745-a92c-9dd8a1b01cc4   Pod    nginx-secrets   6      1      0      0       0      33m
----

クラスタ全体のレポートを一覧表示します：

[source,console,subs="+attributes"]
----
$ kubectl get clusterreport -o wide
NAME                                   KIND        NAME                 PASS   FAIL   WARN   ERROR   SKIP   AGE
6c66175f-e15f-47e8-8c33-2c63d5ecde3e   Namespace   namespace1           3      1     0       0       0      1h
d0ffbf4f-7348-4350-af7b-fdbfc079cc49   Namespace   namespace2           0      4     0       0       0      1h
e48d9bbc-a9c4-4c36-8519-bc7390d9ec64   Namespace   namespace3           4      0     0       0       0      15h
----

特定のレポートの詳細を取得します：

[source,console,subs="+attributes"]
----
kubectl get report 44da974b-6057-4745-a92c-9dd8a1b01cc4 -o yaml
----

特定のClusterReportの詳細を取得します：

[source,console,subs="+attributes"]
----
kubectl get clusterreport e48d9bbc-a9c4-4c36-8519-bc7390d9ec64 -o yaml
----

  namespace: default
  ownerReferences:

      kind: Deployment

== ClusterReportの例

== ポリシーレポーターUI

ポリシー・レポーターは、`kubewarden-controller` のサブチャートとして出荷されます。詳細については、xref:howtos/audit-scanner.adoc[監査スキャナーのインストール]ページを参照してください。

Policy Reporter UI は、すべての違反を表示するダッシュボードを提供します。下のスクリーンショットをご覧ください：

image::policy-reporter_dashboard.png[ポリシーレポーターダッシュボードの例]

{short-project-name} は、Policy Reporter UI に、{short-project-name} ポリシーだけを表示する事前設定済みのフィルタと、拡張情報を同梱しています：

image::policy-reporter_policyreports.png[ポリシー・レポーター PolicyReports の例]

Policy Reporterの他の機能には、異なるクライアント（Grafana Loki、Elasticsearch、チャット・アプリケーションなど）への結果の転送、メトリクス・エンドポイントなどがある。詳しくはhttps://kyverno.github.io/policy-reporter[policy-reporterのコミュニティドキュメントを]参照。
