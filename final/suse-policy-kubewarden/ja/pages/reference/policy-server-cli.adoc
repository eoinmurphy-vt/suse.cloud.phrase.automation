== コマンドラインヘルプ `policy-server`

この文書には、`policy-server`コマンドライン・プログラムのヘルプ内容が含まれています。

*コマンドの概要：*

* link:#policy-server[`policy-server`]
* link:#policy-server-docs[`policy-server docs`]

=== `policy-server`

*使い方：* `policy-server ++[+OPTIONS+]++ ++[+COMMAND+]++`

==== *サブコマンド：*

* `docs` - policy-serverコマンドのマークダウン文書を生成します。

==== *オプション*

* `--addr ++<+BIND_ADDRESS+>++` - ADDRESSに対するバインド
+
デフォルト値： `0.0.0.0`
* `--always-accept-admission-reviews-on-namespace ++<+NAMESPACE+>++` - 指定された名前空間を対象とするアドミッションレビューを常に受け付ける。
* `--cert-file ++<+CERT_FILE+>++` - HTTPS用のX.509証明書ファイルへのパス
* `--client-ca-file ++<+CLIENT_CA_FILE+>++` - クライアント証明書を発行した CA 証明書ファイルへのパス。mTLSを有効にするために必要
* `--daemon` - 設定されている場合、policy-server をデーモンとしてデタッチモードで実行します。
* `--daemon-pid-file ++<+DAEMON-PID-FILE+>++` - デーモンモードで実行している場合にのみ使用されるPIDファイルへのパス。
+
デフォルト値： `policy-server.pid`
* `--daemon-stderr-file ++<+DAEMON-STDERR-FILE+>++` - デーモンモードで実行している場合にのみ使用される、標準エラー（stderr）を格納するファイルへのパス。
* `--daemon-stdout-file ++<+DAEMON-STDOUT-FILE+>++` - デーモンモードで実行している場合にのみ使用される、標準出力ファイルへのパス。
* `--disable-timeout-protection` - ポリシーのタイムアウト保護を無効にする
* `--docker-config-json-path ++<+DOCKER_CONFIG+>++` - Dockerのconfig.jsonのようなパス。レジストリ認証の詳細を示すために使用できる
* `--enable-metrics` - メトリクスの有効化
* `--enable-pprof` - pprof プロファイリングを有効にする
* `--ignore-kubernetes-connection-failure` - Kubernetes接続に失敗してもエラーで終了しない。このため、Kubernetesとの接続がない場合、コンテキストを考慮したポリシーが壊れてしまう。
* `--key-file ++<+KEY_FILE+>++` - HTTPS用のX.509秘密鍵ファイルへのパス。
* `--log-fmt ++<+LOG_FMT+>++` - ログ出力フォーマット
+
デフォルト値： `text`
+
可能な値`text` `json` 、 `otlp`
* `--log-level ++<+LOG_LEVEL+>++` - ログレベル
+
デフォルト値： `info`
+
可能な値`trace` `debug`,`info`,`warn` 、 `error`
* `--log-no-color` - ログのカラー出力を無効にする
* `--policies ++<+POLICIES_FILE+>++` - ロードされるポリシーとその設定を保持するYAMLファイル
+
デフォルト値： `policies.yml`
* `--policies-download-dir ++<+POLICIES_DOWNLOAD_DIR+>++` - ポリシーのダウンロードパス
+
デフォルト値： `.`
* `--policy-timeout ++<+MAXIMUM_EXECUTION_TIME_SECONDS+>++`
- 与えられた時間後にポリシー評価を中断する
+
デフォルト値： `2`
* `--port ++<+PORT+>++` - PORTで聴く
+
デフォルト値： `3000`
* `--readiness-probe-port ++<+READINESS_PROBE_PORT+>++` —
Expose readiness endpoint on READINESS+_PROBE_+PORT
+
デフォルト値： `8081`
* `--sigstore-cache-dir ++<+SIGSTORE_CACHE_DIR+>++` - シグストアのデータをキャッシュするためのディレクトリ
+
デフォルト値： `sigstore-data`
* `--sources-path ++<+SOURCES_PATH+>++` - ソース情報（https、レジストリの安全でないホスト、カスタム CA\...）を保持する YAML ファイル。
* `--verification-path ++<+VERIFICATION_CONFIG_PATH+>++` - 検証情報（URI、キー、アノテーション\...）を保持するYAMLファイル。
* `--workers ++<+WORKERS_NUMBER+>++` - 作成するワーカースレッド数

*使い方：* `policy-server docs --output ++<+FILE+>++`
