{short-project-name} は、クラスタ管理者やポリシー開発者がパターンや潜在的な問題を特定できるように、関連するプラットフォーム・メトリクスを公開します。

== ポリシーサーバー

Policy Server コンポーネントはポリシーを初期化し、実行する。
Kubernetes APIサーバーからのリクエストを受信すると、リクエストをポリシーに転送し、ポリシーが提供するレスポンスをKubernetes APIサーバーに返す。

[NOTE]
====
https://opentelemetry.io/docs/concepts/signals/baggage/[手荷物の]キーバリュー属性は、追加情報を提供するためにメトリックに追加される。
====

==== `kubewarden_policy_evaluations_total`

===== 手荷物

|===
| ラベル | 説明

| `policy_name`
| ポリシーの名称

| `resource_name`
| 評価リソースの名前

| `resource_kind`
| 評価リソースの種類

| `resource_namespace`
| 評価されるリソースの名前空間。リソースがクラスタスコープの場合は存在しない。

| `resource_request_operation`
| 操作タイプ`CREATE` `UPDATE`,`DELETE`,`PATCH` 、 `WATCH`...

| `accepted`
| リクエストは受け入れられたのか？

| `mutated`
| リクエストは突然変異だったのか？

| `error_code`
| 拒否された場合にポリシーによって返されるエラーコード（もしあれば）。
  保険証券に記載がない場合は存在しない。
|===
