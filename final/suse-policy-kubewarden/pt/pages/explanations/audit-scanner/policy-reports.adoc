Ao usar o {project-name} Audit Scanner, você armazena os resultados das varreduras de políticas usando o https://openreports.io/[OpenReports] Custom Resources.

[CAUTION]
====

Com o {project-name} 1.33, o recurso Audit Scanner salva os relatórios em CRDs do OpenReports a partir de [openreports.io] (https://openreports.io) por padrão.

Os CRDs dolink:https://htmlpreview.github.io/?https://github.com/kubernetes-sigs/wg-policy-prototypes/blob/045372e558b896695b2daae92e8c7a04d4d40282/policy-report/docs/index.html[PolicyReport]do grupo de trabalho do K8s Policy não são instalados por padrão, estão marcados como obsoletos e serão removidos em uma versão futura.

Os usuários que desejam continuar usando os PolicyReports obsoletos, ou seja, o comportamento anterior à versão 1.33, precisam:- Alterar o valor do gráfico `kubewarden-crds` Helm de `.Values.install.installPolicyReportCRDs` para `true`. - Alterar o valor do gráfico `kubewarden-controller` Helm de `.Values.auditScanner.reportCRDsKind` de `openreports` para `policyreport`.

====


Esses CRDs do OpenReport oferecem uma maneira estruturada de armazenar e gerenciar os resultados da auditoria.

Você armazena os resultados da verificação da política do scanner de auditoria usando `Report` e `ClusterReport`.
O scanner de auditoria cria um `Report` ou um `ClusterReport` para cada recurso verificado, dependendo do escopo do recurso. Os objetos `Report` estão disponíveis no namespace do recurso, enquanto os objetos `ClusterReport` estão disponíveis no escopo do cluster.

Os resultados de auditoria gerados pelo scanner incluem:

* a política avaliada
* o recurso digitalizado
* o resultado da avaliação (aprovado, reprovado ou ignorado)
* um registro de data e hora indicando quando a avaliação foi realizada.

Você também pode definir anotações de gravidade e categoria para suas políticas.

Os operadores podem consultar os relatórios usando o site `kubectl`. Eles também podem usar a interface de usuário opcional fornecida pelo projeto de código abertohttps://kyverno.github.io/policy-reporter[policy-reporter] para monitoramento e observabilidade dos CRDs do OpenReports.

== Consultando os relatórios

Usando a CLI do kubectl, é possível consultar os resultados da verificação:

Lista os relatórios no namespace padrão:

[source,console,subs="+attributes"]
----
$ kubectl get report -o wide
NAME                                   KIND   NAME            PASS   FAIL   WARN   ERROR   SKIP   AGE
44da974b-6057-4745-a92c-9dd8a1b01cc4   Pod    nginx-secrets   6      1      0      0       0      33m
----

Liste os relatórios de todo o cluster:

[source,console,subs="+attributes"]
----
$ kubectl get clusterreport -o wide
NAME                                   KIND        NAME                 PASS   FAIL   WARN   ERROR   SKIP   AGE
6c66175f-e15f-47e8-8c33-2c63d5ecde3e   Namespace   namespace1           3      1     0       0       0      1h
d0ffbf4f-7348-4350-af7b-fdbfc079cc49   Namespace   namespace2           0      4     0       0       0      1h
e48d9bbc-a9c4-4c36-8519-bc7390d9ec64   Namespace   namespace3           4      0     0       0       0      15h
----

Obtenha os detalhes de um relatório específico:

[source,console,subs="+attributes"]
----
kubectl get report 44da974b-6057-4745-a92c-9dd8a1b01cc4 -o yaml
----

Obtenha os detalhes de um ClusterReport específico:

[source,console,subs="+attributes"]
----
kubectl get clusterreport e48d9bbc-a9c4-4c36-8519-bc7390d9ec64 -o yaml
----

== Interface do usuário do Policy Reporter

O Policy Reporter é fornecido como um subgráfico do site `kubewarden-controller`. Consulte a página xref:howtos/audit-scanner.adoc[Instalação do scanner de auditoria] para obter mais informações.

A interface de usuário do Policy Reporter fornece um painel que mostra todas as violações. Veja a captura de tela abaixo:

image::policy-reporter_dashboard.png[Exemplo de painel do Policy Reporter]

{short-project-name} envia a interface de usuário do Policy Reporter com um filtro pré-configurado que mostra apenas as políticas do {short-project-name}, juntamente com informações expandidas:

image::policy-reporter_policyreports.png[Exemplo de Policy Reporter PolicyReports]

Outros recursos do Policy Reporter incluem o encaminhamento de resultados para diferentes clientes (como Grafana Loki, Elasticsearch, aplicativos de bate-papo), pontos de extremidade de métricas e muito mais. Consultehttps://kyverno.github.io/policy-reporter[os documentos da comunidade do] para obter mais informações.
