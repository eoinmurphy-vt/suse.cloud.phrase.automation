https://artifacthub.io/[O Artifact Hub] é um site em que os usuários podem encontrar, instalar e publicar pacotes e configurações para projetos https://cncf.io[do CNCF].

{project-name} As políticas podem ser publicadas no Artifact Hub e descobertas pelo amplo público de usuários do CNCF.

[NOTE]
====
O Artifact Hub é uma plataforma de agregação de conteúdo e não hospeda de fato os artefatos que são publicados nele.

O Artifact Hub exige que você hospede fisicamente repositórios de imagens de contêineres em um registro de contêineres ou em um servidor da Web.
Consulte a seção xref:explanations/distributing-policies.adoc[de distribuição de políticas] para obter mais informações sobre como hospedar suas políticas.
====


Este documento concentra-se nas etapas necessárias para tornar uma política {short-project-name} detectável no Artifact Hub.

== Prepare seu repositório Git

O Artifact Hub rastreia os repositórios Git em busca de arquivos de metadados especiais.

Há diferentes tipos de layouts que o repositório Git pode ter. Todos eles estão documentados em detalhes na https://artifacthub.io/docs/topics/repositories/#kubewarden-policies-repositories[documentação oficial do Artifact Hub do site].

O Artifact Hub é flexível e permite que você organize seu código dessas maneiras:

* Tenha um repositório Git dedicado ao Artifact Hub: esse repositório não conterá nenhum código-fonte de política. Será uma coleção de arquivos YAML exigidos pelo Artifact Hub.

* Adicione um diretório do Artifact Hub no repositório Git que contém a fonte de sua política. Essa é uma iteração da abordagem anterior, concentrando-se em apenas uma política, a que está definida no repositório Git. Essa abordagem permite manter várias versões da política publicadas no Artifact Hub.

* Adicione os arquivos `artifacthub-pkg.yml` e `artifacthub-repo.yml` à raiz do repositório Git que contém o código-fonte da política. Essa abordagem é a mais simples. A única limitação é que somente a versão mais recente da política ficará visível no Artifact Hub.

A última abordagem é usada em nossos modelos de políticas oficiais. O repositório do Git que é estruturado inclui o arquivo `artifacthub-repo.yml`, e nossas GitHub Actions geram e enviam o `artifacthub-pkg.yml` para um branch`artifacthub` para o Artifact Hub consumir.

== Etapas de publicação

Antes de publicar uma política no Artifact Hub, você deve criar uma contahttps://artifacthub.io/[no site].

Antes de publicar a política, certifique-se de que seu repositório Git tenha o layout adequado, com um `metadata.yml` bem formatado com as anotações obrigatórias.

Se quiser fazer isso manualmente, você pode criar o arquivo `artifacthub-pkg.yml` fazendo um `kwctl scaffold artifacthub` (com a versão `>= 1.23`). Esse comando usa o endereço `metadata.yml` no caminho atual e gera um`artifacthub-pkg.yml`. O site `artifacthub-pkg.yml` contém campos como`version:`, `createdAt:`, que precisam corresponder a um formato específico e estar atualizados. O formato do site `artifacthub-pkg.yml` está descritohttps://github.com/artifacthub/hub/blob/master/docs/metadata/artifacthub-pkg.yml[aqui].

Se você usar nosso GitHub Actions >= `v4.0.0`, o fluxo de trabalho de liberação poderá ser configurado com a entrada `artifacthub: true`. Se configurado dessa forma, após uma versão bem-sucedida (depois que a política tiver sido criada, assinada e enviada), nossas GitHub Actions terão um último trabalho que gera o`artifacthub-pkg.yml` para você, faz o commit e envia as alterações para o branch`artifacthub`. Os arquivos canônicos são sempre os que estão na ramificação `main`. Para ver o comportamento das versões anteriores de nossas GitHub Actions, consulte as versões da documentação anteriores à 1.23.

Por fim, certifique-se de que sua política seja publicada dentro de um registro de contêiner ou em um servidor da Web.

[NOTE]
====
No momento, o conteúdo do arquivo `artifacthub-repo.yml` não é relevante.
====


Quando tudo estiver no lugar, faça login no Artifact Hub e acessehttps://artifacthub.io/control-panel/repositories?page=1[o plano de controle].

Decida se deseja publicar a política como um usuário ou em uma organização do Artifact Hub à qual você pertence. Isso é feito escolhendo o _"contexto do painel de controle"_ correto _._

Em seguida, pressione o botão _"Add" (Adicionar)_ e preencha o formulário:

* Escolha _"{short-project-name} policies"_ como tipo.
* Digite um _"Nome"_ e um _"Nome de exibição"_ de sua escolha.
* Digite o URL do seu repositório Git.
* Digite `artifacthub` como o ramo a ser rastreado.

Por fim, pressione o botão _"Add" (Adicionar)_. Isso o levará de volta à página _"Repositories" (Repositórios_), onde você verá o repositório recém-criado.

Cada repositório tem vários campos de informações. Localize a propriedade _"ID"_ do repositório que você acabou de criar e copie-a.

Volte para o repositório Git e edite o endereço `artifacthub-repo.yml`. Certifique-se de que a chave`repositoryID` encontrada dentro do documento tenha o valor que você acabou de copiar da página da Web do Artifact Hub.

[TIP]
====
O formato do arquivo `artifacthub-repo.yml` é definidohttps://github.com/artifacthub/hub/blob/master/docs/metadata/artifacthub-repo.yml[aqui].

Agora é um bom momento para fazer algumas personalizações adicionais nesse arquivo.
====


Quando terminar de fazer as alterações, confirme o arquivo `artifacthub-repo.yml`atualizado e envie-o. Durante a próxima verificação, o Artifact Hub encontrará esse arquivo e adicionará ohttps://artifacthub.io/docs/topics/repositories/#verified-publisher[selo]ao seu repositório do Artifact Hub.

== Mantendo o Artifact Hub em sincronia

Não se esqueça de atualizar o conteúdo do arquivo `artifacthub-pkg.yml` sempre que lançar uma nova versão da sua política. Por exemplo, você deve atualizar o campo`io.kubewarden.policy.version` (que geralmente corresponde à tag OCI da política) e qualquer outra anotação que deseje alterar.

[NOTE]
====
O conteúdo do arquivo `artifacthub-repo.yml` não precisa ser alterado.
====
