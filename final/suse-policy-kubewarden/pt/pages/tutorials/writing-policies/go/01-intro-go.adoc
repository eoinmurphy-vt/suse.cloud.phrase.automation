O compilador Go oficial pode produzir binários do WebAssembly, para execução fora de um navegador, desde a versão 1.21.

Há outro compilador Go que pode criar binários do WebAssembly que podem ser usados pelo site {project-name}.
Esse projeto de compilador é https://tinygo.org/[] TinyGo:

____
O TinyGo traz a linguagem de programação Go para sistemas incorporados e para a Web moderna, criando um novo compilador baseado no LLVM.

Você pode compilar e executar programas TinyGo em mais de 94 placas de microcontroladores diferentes, como o BBC micro:bit e o Arduino Uno.

O TinyGo também pode produzir código WebAssembly (Wasm), que é muito compacto.
Você pode compilar programas para navegadores da Web, bem como para ambientes de servidor e de computação de borda que suportam a família de interfaces WebAssembly System Interface (WASI).
____

O projeto {short-project-name} atualmente sugere o uso do TinyGo por dois motivos:

* os binários são menores
* suporte para https://wapc.io[waPC] pela capacidade de exportar funções para o tempo de execução

== Limitações do TinyGo

O TinyGo ainda não oferece suporte a todos os recursos do Go. Consultesuporte ao idioma TinyGo https://tinygo.org/lang-support/[a página de] para ver o status atual do projeto.
Atualmente, sua maior limitação é a falta de um pacote `reflect` totalmente compatível.
Isso significa que os tipos oficiais de API do Kubernetes Go (por exemplo: `k8s.io/api/core/v1`) não são compilados.

{short-project-name} As políticas precisam processar dados JSON, como configurações de política e a solicitação recebida pelo Kubernetes.

Apesar das limitações atuais do TinyGo, ainda é fácil escrever políticas de validação {short-project-name} com ele.

== Ferramentas

Escrever políticas {short-project-name} requer uma versão do TinyGo superior a `v0.28.1`.
No entanto, use a versão mais recente para obter os melhores resultados.

[CAUTION]
====
O uso de versões mais antigas do TinyGo resulta em erros de tempo de execução devido ao suporte limitado à reflexão em Go.
====


Essas bibliotecas Go são úteis ao escrever uma política {short-project-name}:

* https://github.com/kubewarden/policy-sdk-go[{short-project-name} Go SDK]:
Fornece estruturas e funções que reduzem a quantidade de código necessária.
Ele também fornece ajudantes de teste.
* https://github.com/kubewarden/k8s-objects[Tipos de Kubernetes Go]:
O https://github.com/kubernetes/kubernetes/tree/master/staging/src/k8s.io[oficial Kubernetes Go Types]não pode ser usado com o TinyGo.
Este módulo fornece todos os tipos de Kubernetes de uma forma amigável ao TinyGo.
* https://github.com/tidwall/gjson[gjson]:
Isso fornece uma linguagem de consulta para navegação rápida de documentos JSON e recuperação de dados.
Essa biblioteca não usa o pacote `encoding/json` fornecido pelo `stdlib` do Go e, portanto, pode ser usada com o TinyGo.
* https://github.com/deckarep/golang-set[mapset]:
Fornece uma implementação em Go da estrutura de dadoshttps://en.wikipedia.org/wiki/Set_(abstract_data_type)[Set].
Essa biblioteca reduz a quantidade de código a ser escrito, pois operações como Set `union`, `intersection`, `difference` são operações comuns nas políticas {short-project-name}.

Por fim, o projeto {short-project-name} fornece umhttps://github.com/kubewarden/go-policy-template[projeto de política Go modelo]que você pode usar para criar políticas Go {short-project-name}.

== Obtendo as dependências do TinyGo

A maneira mais fácil de obter o TinyGo é usar as imagens de contêineres upstream.
As versões oficiais sãohttps://hub.docker.com/r/tinygo/tinygo[aqui], enquanto as compilações do ramo de desenvolvimento são automaticamente enviadashttps://hub.docker.com/r/tinygo/tinygo-dev[aqui].

Se necessário, tente a página dehttps://tinygo.org/getting-started/[introdução]do TinyGo para obter mais informações.

== Pré-requisitos do tutorial

Durante este tutorial, você precisará dessas ferramentas em sua máquina de desenvolvimento:

* docker ou outro mecanismo de contêiner: usado para criar a política do WebAssembly.
Você usará o compilador fornecido na imagem oficial do contêiner do TinyGo.
* https://github.com/bats-core/bats-core[bats]: usado para escrever os testes e automatizar sua execução.
* https://github.com/kubewarden/kwctl/releases[`kwctl`]:
Ferramenta CLI fornecida por {short-project-name} para executar suas políticas fora do Kubernetes, entre outras ações.
Isso é abordado xref:tutorials/testing-policies/index.adoc[nesta seção] da documentação.
