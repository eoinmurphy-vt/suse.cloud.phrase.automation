Como exemplo de outro uso de recursos de host no site`cel-policy` do kubewarden, vamos criar uma política que verifique todas as imagens de contêiner em um pod verificando suas assinaturas sem chave Sigstore.

== Exemplo: Política de verificação da Sigstore

Essa política verificará todas as imagens de contêineres no Pod Sigstore e verificará se as imagens são assinadas e confiáveis.

Nesse caso, verificaremos se há uma assinatura sem chave do Sigstore realizada no GitHub Actions. Esse tipo de assinatura sem chave é vinculado ao emissor OIDC do GitHub ao criar os certificados criptográficos, portanto, só precisaremos saber a organização do GitHub sob a qual a imagem do contêiner é publicada. Você pode ler mais sobre os recursos de host do Sigstorexref:reference/spec/host-capabilities/02-signature-verifier-policies.adoc[aqui].

Para conseguir isso no CEL, usamos as bibliotecas de extensão CEL https://github.com/kubewarden/cel-policy?tab=readme-ov-file#host-capabilities[{project-name}'s para recursos de host] na política, especialmente a função https://pkg.go.dev/github.com/kubewarden/cel-policy/internal/cel/library#Sigstore[`githubAction`].

Como sempre, podemos começar com `kwctl`:

[subs="+attributes",console]
----
$ kwctl scaffold manifest -t ClusterAdmissionPolicy \
  registry://ghcr.io/kubewarden/policies/cel-policy:v1.0.0`
----

Que, então, podemos editar para que seja relevante para a nossa política de verificação de contêineres:

[subs="+attributes",yaml]
----
apiVersion: policies.kubewarden.io/v1
kind: ClusterAdmissionPolicy
metadata:
  name: "cel-sigstore-keyless-verification"
spec:
  module: ghcr.io/kubewarden/policies/cel-policy:v1.0.0
  namespaceSelector:
    matchLabels:
      kubernetes.io/metadata.name: default
  rules:
    - apiGroups: [""]
      apiVersions: ["v1"]
      resources: ["pods"]
      operations: ["CREATE", "UPDATE"]
----

Agora, vamos à parte do CEL. Obteremos uma lista de imagens de contêiner no objeto Pod atual e, em seguida, verificaremos se elas são verificadas por uma assinatura que corresponda à organização que escolhemos (neste caso, github.com/opencontainers):

[subs="+attributes",yaml]
----
apiVersion: policies.kubewarden.io/v1
kind: ClusterAdmissionPolicy
metadata:
  name: "cel-sigstore-keyless-verification"
spec:
  module: ghcr.io/kubewarden/policies/cel-policy:v1.0.0
  namespaceSelector:
    matchLabels:
      kubernetes.io/metadata.name: default
  rules:
    - apiGroups: [""]
      apiVersions: ["v1"]
      resources: ["pods"]
      operations: ["CREATE", "UPDATE"]
  settings:
    variables:
      - name: containerImages
        expression: |
          object.spec.containers.map(c, c.image)
      - name: containerImagesNotVerified
        expression: |
          variables.containerImages.filter(image, !kw.sigstore.image(image).githubAction("opencontainers").verify().isTrusted())
    validations:
      - expression: |
          size(variables.containerImagesNotVerified) == 0
        messageExpression: "'These container images are not signed by the kubewarden GitHub organization: ' + variables.containerImagesNotVerified.join(', ')"
----

Mas espere, não podemos nos esquecer de que os InitContainers também podem fazer parte dos Pods. Então, vamos adicionar outra variável e validação:

```yaml title="./cel-policy-sigstore.yaml" apiVersion: policies.kubewarden.io/v1 kind: Metadados de ClusterAdmissionPolicy: name: "cel-sigstore-keyless-verification" spec: module: ghcr.io/kubewarden/policies/cel-policy:v1.0.0 namespaceSelector: matchLabels: kubernetes.io/metadata.name: default rules:- apiGroups: [""] apiVersions: ["v1"] recursos: ["pods"] operações: ["CREATE", "UPDATE"] configurações: variáveis: - nome: containerImages expressão: | object.spec.containers.map(c, c.image) - nome: initContainerImages expressão: | has(object.spec.initContainerImages) ? object.spec.initContainers.map(c, c.image) : [] - nome: containerImagesNotVerified expressão: | variables.containerImages.filter(image, !kw.sigstore.image(image).githubAction("opencontainers").verify().isTrusted()) - nome: initContainerImagesNotVerified expressão: | variables.initContainerImages.filter(image, !kw.sigstore.image(image).githubAction("opencontainers").verify().isTrusted()) validations:- expressão: | size(variables.containerImagesNotVerified) == 0 messageExpression: "'Essas imagens de contêiner não são assinadas pela organização kubewarden GitHub: ' + variables.containerImagesNotVerified.join(', ')" - expression: | size(variables.initContainerImagesNotVerified) == 0 messageExpression: "'These init container images are not signed by the kubewarden GitHub organization: ' + variables.initContainerImagesNotVerified.join(', ')"
```

As usual with CEL, we can add several `validations` under
`settings.validations`, and they are evaluated in parallel, joined with an AND
operation, which is short-circuited.

We can now deploy the policy, and try to deploy a Pod with unsigned images:

```console
$ kubectl apply -f - <<EOF
apiVersion: v1
kind: Pod
metadata:
  name: golanci-lint-example
spec:
  containers:
  - name: nginx
    image: ghcr.io/opencontainers/golangci-lint:v1.52.1
    ports:
    - containerPort: 80
EOF
Error from server: error when creating "STDIN":
  admission webhook "clusterwide-cel-sigstore-keyless-verification.kubewarden.admission" denied the request:
  failed to verify image: Callback evaluation failure: no signatures found for image: ghcr.io/opencontainers/golangci-lint:v1.52.1
```
