Ao compreender o JavaScript, o TypeScript economiza seu tempo ao detectar erros e fornecer correções antes de executar o código.

{project-name} usa o https://github.com/bytecodealliance/javy[] Javy (um projeto da Bytecode Alliance) para criar binários do WebAssembly a partir do JavaScript e do TypeScript.

[quote]
O Javy pega seu código JavaScript e o executa em um contexto do WebAssembly.

Ele apresenta um mecanismo QuickJS incorporado compilado para WebAssembly que pode executar JavaScript.

O projeto fornece uma CLI e um conjunto de APIs para incorporar e personalizar o comportamento ao executar JavaScript no WebAssembly.

O projeto {short-project-name} atualmente usa o Javy por esses motivos:

* Mecanismo JavaScript maduro (QuickJS) compilado para WebAssembly.
* Suporte à xref:/tutorials/writing-policies/wasi/01-intro-wasi.adoc[interface WASI] por meio de funções personalizadas do host.
* Tamanhos menores de binários em comparação com outras soluções JavaScript-to-WebAssembly.
* Desenvolvimento e manutenção ativos pela Bytecode Alliance.

== Limitações de Javy

O Javy executa JavaScript em um ambiente WebAssembly em sandbox com determinadas restrições:

* *Somente ambiente WASI*: Acesso limitado a stdin/stdout/stderr e a recursos de host fornecidos explicitamente.
* *Não há APIs do Node.js*: Módulos padrão do Node.js como `fs`, `http` ou `crypto` não estão disponíveis.
* *Biblioteca padrão limitada*: Somente os principais recursos do JavaScript e as APIs ativadas explicitamente são acessíveis.
* *Execução em um único thread*: Não há suporte para Web Workers ou multi-threading.

Apesar dessas limitações, o Javy oferece recursos suficientes para a criação de políticas de validação eficazes para o site {short-project-name} por meio do sistema de recursos de hosts.

[WARNING]
====
A gravação em STDOUT viola as políticas - em vez disso, use STDERR para registro.
====

== Ferramentas

Escrever políticas para o site {short-project-name} requer:

* *Node.js*: Tempo de execução do JavaScript.
* *npm*: Para o gerenciamento de dependências.
* *TypeScript*: Recomendado para segurança de tipo (opcional).

[WARNING]
====
Verifique se você está usando o Node.js 18 ou superior. As versões mais antigas podem não ser compatíveis com a cadeia de ferramentas de compilação.
====

Essas bibliotecas TypeScript/JavaScript são úteis ao escrever uma política {short-project-name}:

* https://github.com/kubewarden/policy-sdk-js[{short-project-name} JavaScript SDK]: Fornece estruturas e funções que reduzem a quantidade de código necessária. Ele também fornece ajudantes de teste e acesso a todos os recursos do host.
* https://github.com/silverlyra/kubernetes-types[Kubernetes TypeScript types]: Fornece definições TypeScript para todos os recursos do Kubernetes, permitindo o desenvolvimento de políticas com segurança de tipo.

O projeto {short-project-name} fornece um https://github.com/kubewarden/js-policy-template[projeto de política JavaScript/TypeScript modelo] que você pode usar para criar políticas {short-project-name}.

== Obtendo a cadeia de ferramentas

A maneira mais fácil de obter o conjunto de ferramentas é usar o {short-project-name} JavaScript SDK, que inclui o plug-in de compilação Javy:

[source,console]
----
npm install kubewarden-policy-sdk
----

O binário do plug-in Javy é incluído automaticamente e você pode encontrá-lo em:

[source,console]
----
node_modules/kubewarden-policy-sdk/plugin/javy-plugin-kubewarden.wasm
----

== Pré-requisitos do tutorial

Durante este tutorial, você precisará dessas ferramentas em sua máquina de desenvolvimento:

* *Node.js*: Versão 18 ou superior com npm para gerenciamento de dependências.
* https://github.com/bats-core/bats-core[`bats`]: Usado para escrever os testes e automatizar sua execução.
* https://github.com/kubewarden/kwctl/releases[`kwctl` ] ≥ : Ferramenta CLI fornecida por {short-project-name} para executar suas políticas fora do Kubernetes, entre outras ações. Isso é abordado na xref:/tutorials/testing-policies/index.adoc[seção de políticas de teste] da documentação.
* https://github.com/bytecodealliance/javy[`javy` ≥ 6.0.0]: Ferramenta CLI para compilar código JavaScript em módulos do WebAssembly.
