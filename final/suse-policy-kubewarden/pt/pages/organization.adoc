{project-name} organiza a documentação do projeto da seguinte forma:

* xref:/introduction.adoc[Introdução]
* xref:/quick-start.adoc[Início rápido]
* _Tutoriais - aprendizado_por meio de exemplos.
* _Explicações -_conceitos _subjacentes_.
* _Como fazer para realizar_as tarefas do {short-project-name}.
* _Referência_
* Organização - esta página
* xref:/personas.adoc[Personas -]para quem é o {short-project-name}?
* xref:/disclosure.adoc[Segurança]
* xref:/glossary.adoc[Glossário]

A maior parte do material está nas quatro seções, _Tutoriais_, _Explicações_,_Como fazer_ e _Referência_. Pode haver sobreposição, pois as seções não são mutuamente exclusivas. O projeto {short-project-name} tenta seguir a estrutura de documentaçãolink:https://diataxis.fr/[do].

== Navegação

O menu de conteúdo à esquerda sempre indica o local atual do documento.
O menu de seções à direita mostra as seções da página atual. Há também um recurso de "breadcrumbs" na parte superior do painel de conteúdo principal de cada página.

== Pesquisa

Há uma caixa de pesquisa no painel do cabeçalho de cada página.

== Edição

Um link para a fonte da página no GitHub está na parte inferior de cada página.

== Outros recursos

Também são mantidos:

* link:https://github.com/kubewarden/rfc[RFCs] para o projeto.
* Uma série de link:https://www.kubewarden.io/blog/[postagens no blog] relacionadas a lançamentos {community-name}.
* Itens relacionados ao gerenciamento link:https://github.com/kubewarden/community[da comunidade]de {community-name}.
