{project-name} As políticas com reconhecimento de contexto exigem acesso a recursos do cluster do Kubernetes em que estão sendo executadas.
Para isso, os SDKs do {short-project-name} expõem funções que usam o protocolo de comunicação waPC para conversar com o sistema host solicitando dados sobre o cluster.

== Contrato de protocolo waPC

Internamente, os SDKs dependem dessas funções expostas pelo ambiente do host de política:

* `list_resources_by_namespace` : Dado um tipo de recurso e um namespace, liste todos os recursos desse tipo que estão definidos nele.
Isso não pode ser usado para listar recursos de todo o cluster, como `Namespace`.
* `list_resources_all`: Dado um tipo de recurso, liste todos os recursos desse tipo que estão definidos em todo o cluster.
Isso pode ser usado para listar recursos de todo o cluster, como `Namespace`.
* `get_resource`: Localiza o recurso exato identificado pelo tipo de recurso fornecido, pelo nome fornecido e por um identificador de namespace opcional.

Essa comunicação guest-host é realizada usando o mecanismo padrão de chamada de host waPC.
Qualquer convidado que implemente o mecanismo de intercomunicação waPC pode solicitar essas informações ao host.

O waPC tem os seguintes argumentos de função ao realizar uma chamada do convidado para o host:

* Encadernação `kubewarden`
* Espaço de nome - `kubernetes`
* Operação - `list_resources_all`, `list_resources_by_namespace`, ou `get_resource`
* Carga útil - carga útil de entrada - veja abaixo

e retornos:

* Carga útil - carga útil de saída - veja abaixo

Por contrato ou por convenção, as políticas podem recuperar as informações do cluster do Kubernetes chamando o host das seguintes maneiras:

=== Operação - `list_resources_all`

==== Entrada

[,hcl]
----
# API Group version. Use `v1` for the `core` group and
  # `groupName/groupVersion` for all other groups
  "api_version": string,
  # Resource kind
  "kind": string,
  # Label selector to filter the resources
  "label_selector": string,
  # Field selector to filter the resources
  "field_selector": string,
  # Field masks: A list of fields to include in the response.
  #
  # If strictly defined, the host will prune the Kubernetes resource to contain *only*
  # the specified fields, reducing memory usage and serialization overhead.
  "field_masks": [string]
----

==== Saída

Retorna um Kuberneteshttps://github.com/kubernetes/community/blob/master/contributors/devel/sig-architecture/api-conventions.md#types-kinds[`List`]que é uma coleção de objetos Kubernetes do mesmo tipo.

[IMPORTANT]
====
Use essa função da API para buscar recursos em todo o cluster (por exemplo, namespaces)
====


=== Operação - `list_resources_by_namespace`

==== Entrada

[,hcl]
----
{
  # API Group version. Use `v1` for the `core` group and
  # `groupName/groupVersion` for all other groups
  "api_version": string,
  # Resource kind
  "kind": string,
  # Namespace where the requested resource lives in
  "namespace": string,
  # Label selector to filter the resources
  "label_selector": string,
  # Field selector to filter the resources
  "field_selector": string,
  # Field masks: A list of fields to include in the response.
  #
  # If strictly defined, the host will prune the Kubernetes resource to contain *only*
  # the specified fields, reducing memory usage and serialization overhead.
  "field_masks": [string]
}
----

==== Saída

Retorna um Kubernetes https://github.com/kubernetes/community/blob/master/contributors/devel/sig-architecture/api-conventions.md#types-kinds[`List`]que é uma coleção de objetos Kubernetes do mesmo tipo.

[CAUTION]
====
Essa função da API retorna um erro quando usada para buscar recursos em todo o cluster (por exemplo, namespaces).
Use o endereço `list_resources_all` ao lidar com recursos de todo o cluster.
====


=== Operação - `get_resource`

==== Armazenamento em cache

O resultado da operação `get_resource` é armazenado em cache por cinco segundos por padrão para melhorar o desempenho e reduzir a carga desnecessária no servidor da API do Kubernetes.

Use o campo `disable_cache` para ignorar o cache quando forem necessários dados novos.

==== Entrada

[,hcl]
----
{
  # API Group version. Use `v1` for the `core` group
  # and `groupName/groupVersion` for all other groups
  "api_version": string,
  # Singular PascalCase name of the resource
  "kind": string,
  # Namespace scoping the search
  "namespace": string,
  # The name of the resource
  "name": string,
  # Disable caching of results obtained from Kubernetes API Server
  "disable_cache": bool,
  # Field masks: A list of fields to include in the response.
  #
  # If strictly defined, the host will prune the Kubernetes resource to contain *only*
  # the specified fields, reducing memory usage and serialization overhead.
  "field_masks": [string]
}
----

==== Saída

Resultado de `GET /apis/$api_version/namespaces/$namespace/$kind/$name`
