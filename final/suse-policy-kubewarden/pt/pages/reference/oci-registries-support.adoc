Você distribui suas políticas {project-name} comohttps://opencontainers.org[artefatos]OCI usando registros regulares da Open Container Initiative (OCI).

{short-project-name} o armazenamento de políticas está ao lado das imagens de contêineres.
Eles não exigem configuração ou manutenção adicionais além daquelas necessárias para imagens de contêineres comuns.

[NOTE]
====
Você pode adicionar um registro que funcione com o {short-project-name} ou corrigir quaisquer imprecisões de registro usando o botão "Edit this page" na parte inferior da página
====


== Projetos que implementam registros de OCI

* https://goharbor.io/[Porto].
* https://github.com/distribution/distribution[Distribuição] (https://github.com/distribution/distribution/releases/tag/v2.7.0[≥ 2.7.0]).
* https://access.redhat.com/products/red-hat-quay/[Cais]: Suportado, mas https://access.redhat.com/documentation/en-us/red_hat_quay/3/html/use_red_hat_quay/oci-intro#other-oci-artifacts-with-quay[inativo por padrão na v3.6].

* https://github.com/container-registry/[Registro de contêineres do GitHub]. Consulte https://docs.github.com/en/packages/working-with-a-github-packages-registry/working-with-the-container-registry[aqui].
* https://quay.io[Quay.io]. Veja a lista de projetos.
* https://aws.amazon.com/ecr/[Amazon ECR]: Consulte https://aws.amazon.com/blogs/containers/oci-artifact-support-in-amazon-ecr/[aqui].
* https://cloud.google.com/artifact-registry[Registro de artefatos do Google]. Consulte https://cloud.google.com/anthos-config-management/docs/how-to/sync-oci-artifacts-from-artifact-registry[aqui].
* https://azure.microsoft.com/en-us/products/container-registry/[Registro de contêineres do Azure]. Consulte https://learn.microsoft.com/en-us/azure/container-registry/container-registry-oci-artifacts[aqui].
* https://hub.docker.com/[Docker Hub]. Consulte https://docs.docker.com/docker-hub/oci-artifacts/[aqui].
* https://cloud.ibm.com/docs/Registry[Registro de contêineres da IBM Cloud]. Consulte https://cloud.ibm.com/docs/Registry?topic=Registry-registry_helm_charts[aqui].
* https://jfrog.com/artifactory/[JFrog Artifactory]. Consulte https://jfrog.com/help/r/jfrog-artifactory-documentation/docker-registry[aqui].

== Problemas conhecidos

=== Docker Hub

Atualmente, o Docker Hub não oferece suporte a artefatos OCI, portanto não pode ser usado para armazenar políticas do {short-project-name}.
A Docker Inc. anunciou que o Docker Hub oferecerá suporte a artefatos OCI nohttps://www.docker.com/blog/announcing-docker-hub-oci-artifacts-support/[futuro].

=== JFrog

Embora o JFrog ofereça suporte a artefatos OCI, só é possível fazer push para ele parcialmente, quando se segue sua especificação.https://github.com/kubewarden/kwctl/issues/59[Leia mais aqui].
