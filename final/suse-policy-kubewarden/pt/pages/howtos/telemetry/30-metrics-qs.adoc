Esta seção documenta como ativar os relatórios de métricas no Servidor de políticas.

[NOTE]
====

Antes de continuar, certifique-se de ter concluído a seção anterior doxref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[OpenTelemetry]deste livro. Você precisa dela para que esta seção funcione corretamente.

====

Você usa o https://prometheus.io/[Prometheus] para coletar métricas expostas pelo Servidor de políticas.

== Instalar o Prometheus

Você usa https://github.com/prometheus-operator/prometheus-operator[Prometheus Operator], que nos permite definir os alvos do Prometheus.

Há muitas maneiras de instalar e configurar o Prometheus. Para facilitar a implantação, você deve usar o Helm chart da comunidade Prometheus.

O Prometheus Operator implantado com esse Helm Chart usahttps://github.com/prometheus-operator/prometheus-operator/blob/master/Documentation/design.md#servicemonitor[Service Monitors] para definir quais serviços o Prometheus monitora.

No seu caso, você está adicionando um ServiceMonitor que tem como alvo o namespace `kubewarden`para serviços que correspondem aos rótulos `app=kubewarden-policy-server-default`e `app.kubernetes.io/name: kubewarden-controller`. Isso configura o Prometheus Operator para inspecionar quais Kubernetes Endpoints pertencem a serviços que correspondem a essas condições.

Você pode criar os dois ServiceMonitors denominados `kubewarden-controller` e`kubewarden-policy-server` para serem usados pela instância padrão do Prometheus instalada pelo Helm chart.

Para fazer isso, você deve criar o seguinte arquivo de valores:

[subs="+attributes",console]
----
cat <<EOF > kube-prometheus-stack-values.yaml
prometheus:
  additionalServiceMonitors:
    - name: kubewarden
      selector:
        matchLabels:
          app: kubewarden-policy-server-default
      namespaceSelector:
        matchNames:
          - kubewarden
      endpoints:
        - port: metrics
          interval: 10s
    - name: kubewarden-controller
      selector:
        matchLabels:
          app.kubernetes.io/name: kubewarden-controller
      namespaceSelector:
        matchNames:
          - kubewarden
      endpoints:
        - port: metrics
          interval: 10s
EOF
----

Você deve instalar o Helm Chart da pilha do Prometheus:

[NOTE]
====
No momento da redação deste artigo (2023-11-17), a versão mais recente do gráfico é `51.5.3`
====


[subs="+attributes",console]
----
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts

helm install --wait --create-namespace \
  --namespace prometheus \
  --version 51.5.3 \
  --values kube-prometheus-stack-values.yaml \
  prometheus prometheus-community/kube-prometheus-stack
----

== Instalar {short-project-name}

Agora você pode instalar o {short-project-name}, da maneira recomendada, com os gráficos do Helm.

[NOTE]
====

O `cert-manager` é um requisito do OpenTelemetry, mas você já o instalou em uma seção anterior desta documentação.

====


Primeiro, você deve adicionar o repositório do Helm que contém {short-project-name}:

[subs="+attributes",console]
----
helm repo add kubewarden https://charts.kubewarden.io
----

Em seguida, você instala as Custom Resource Definitions (CRDs) definidas por{short-project-name}:

[subs="+attributes",console]
----
helm install --wait \
  --namespace kubewarden --create-namespace \
  kubewarden-crds kubewarden/kubewarden-crds
----

Agora você pode implementar o restante da pilha {short-project-name}. O gráfico oficial do Helm cria um PolicyServer chamado `default`.

Você deve configurar o gráfico do Helm para ter as métricas ativadas em {short-project-name}. O arquivo `kubewarden-values.yaml` deve ter o seguinte conteúdo:

[subs="+attributes",yaml]
----
telemetry:
  mode: sidecar
  metrics: True
  sidecar:
    metrics:
      port: 8080
----

Agora, instale os gráficos do Helm:

[subs="+attributes",console]
----
helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  --values kubewarden-values.yaml \
  kubewarden-controller kubewarden/kubewarden-controller

helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  kubewarden-defaults kubewarden/kubewarden-defaults \
  --set recommendedPolicies.enabled=True \
  --set recommendedPolicies.defaultPolicyMode=monitor
----

Isso cria a instância `default` de `PolicyServer`:

[subs="+attributes",console]
----
kubectl get policyservers.policies.kubewarden.io
NAME      AGE
default   3m7s
----

Por padrão, esse servidor de política não tem métricas ativadas.

== Acessando o Prometheus

O Prometheus expõe uma interface do usuário que você pode usar para inspecionar as métricas expostas por diferentes componentes em seu cluster do Kubernetes.

É possível encaminhar a porta do Prometheus para que você possa acessá-la.

[subs="+attributes",console]
----
kubectl port-forward -n prometheus --address 0.0.0.0 svc/prometheus-operated 9090
----

Agora, você pode visitar o Prometheus na porta `9090` e realizar uma consulta, por exemplo:`kubewarden_policy_evaluations_total`. Você pode ver que o número de avaliações aumenta com o tempo, à medida que mais solicitações passam pela política.

== Acessando o Grafana

É possível encaminhar o serviço Grafana para que você possa acessá-lo.

[subs="+attributes",console]
----
kubectl port-forward -n prometheus --address 0.0.0.0 svc/prometheus-grafana 8080:80
----

Agora você pode fazer login com o nome de usuário padrão `admin` e a senha`prom-operator`.

=== Usando o painel do {short-project-name} Grafana

{short-project-name} tem um painel do Grafana com métricas básicas que fornecem uma visão geral sobre como o {short-project-name} se comporta no cluster. Esse painel está disponível nas versões do GitHub do repositório {short-project-name} policy-server como um arquivohttps://github.com/kubewarden/policy-server/releases/latest/download/kubewarden-dashboard.json[JSON] ou no https://grafana.com/grafana/dashboards/15314[site].

Para importar o painel para seu ambiente, você pode baixar o arquivo JSON do site do Grafana ou do repositório:

[subs="+attributes",console]
----
curl https://github.com/kubewarden/policy-server/releases/latest/download/kubewarden-dashboard.json
----

Depois de obter o arquivo, você deve acessar o painel do Grafana ehttps://grafana.com/docs/grafana/latest/dashboards/export-import/#import-dashboard[import it]. Acesse `/dashboard/import` no painel do Grafana e siga estas etapas:

. Copie e cole o conteúdo do arquivo JSON na caixa `Import via panel json` na UI do Grafana.
. Clique no botão `Load`.
. Escolha `Prometheus` como a fonte.
. Clique no botão `Import`.

Outra opção é importá-lo diretamente do site grafana.com. Para isso:

. Copie o ID do painel da https://grafana.com/grafana/dashboards/15314[página do painel]
. Cole-o no campo `Import via grafana.com` 
. Clique no botão `load`.
. Depois de importar o painel, defina a fonte de dados do Prometheus a ser usada e conclua o processo de importação.

O painel do Grafana tem painéis que mostram o estado de todas as políticas gerenciadas por{short-project-name}. Ele também possui painéis específicos de políticas.

Você pode obter métricas detalhadas de uma política específica alterando o valor da variável `policy_name` para corresponder ao nome da política necessária.
