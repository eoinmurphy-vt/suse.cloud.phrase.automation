A partir da versão `v1.7.0`, o {project-name} tem um novo recurso chamado "Audit Scanner". Um novo componente, chamado "audit-scanner", verifica constantemente os recursos declarados no cluster, sinalizando aqueles que não estão em conformidade com as políticas implementadas do {project-name} e salvando esses resultados em um relatório específico de recursos personalizados.

As políticas evoluem com o tempo: você implementa novas políticas e atualiza as existentes. As definições de versão e configuração são alteradas. Isso pode levar a situações em que os recursos que já estão no cluster não são mais compatíveis.

O recurso de scanner de auditoria fornece aos administradores do Kubernetes uma ferramenta para verificar consistentemente o estado de conformidade de seus clusters.

O Audit Scanner pode salvar seus resultados em Definições de Recursos Personalizados (CRDs) OpenReports delink:https://openreports.io[openreports.io](a configuração padrão desde a versão 1.33) ou PolicyReports dolink:https://github.com/kubernetes-sigs/wg-policy-prototypes[grupo de trabalho de políticas do Kubernetes](marcado como obsoleto desde a versão 1.30, não instalado nem criado por padrão desde a versão 1.30, programado para remoção em uma versão futura).

== Instalação

O componente do scanner de auditoria está disponível desde {short-project-name} `v1.7.0`.
Portanto, certifique-se de que esteja instalando o Helm chart com a versão do aplicativo`v1.7.0` ou superior.

. Instale o gráfico do `kubewarden-crds` Helm. Por padrão, o gráfico instala os CRDs necessários do site`PolicyReport`.
+
[subs="+attributes",console]
----
helm install kubewarden-crds kubewarden/kubewarden-crds
----
+

[NOTE]
====
Com o Kubewarden 1.33, o recurso Audit Scanner salva os relatórios em CRDs do OpenReports a partir de [openreports.io] (https://openreports.io) por padrão.

Os CRDs dolink:https://htmlpreview.github.io/?https://github.com/kubernetes-sigs/wg-policy-prototypes/blob/045372e558b896695b2daae92e8c7a04d4d40282/policy-report/docs/index.html[PolicyReport]do grupo de trabalho do K8s Policy não são instalados por padrão, estão marcados como obsoletos e serão removidos em uma versão futura.

Os usuários que desejam continuar usando os PolicyReports obsoletos, ou seja, o comportamento anterior à versão 1.33, precisam:- Alterar o valor do gráfico `kubewarden-crds` Helm de `.Values.install.installPolicyReportCRDs` para `true`. - Alterar o valor do gráfico `kubewarden-controller` Helm de `.Values.auditScanner.reportCRDsKind` de `openreports` para `policyreport`.
====
+
[CAUTION]
====
Para armazenar os resultados dos relatórios de política, você precisa ter disponíveis as definições de recursos personalizados (CRDs) do PolicyReport. Se os CRDs do PolicyReport necessários já estiverem no cluster, você não poderá instalá-los usando o gráfico kubewarden-crds. Nesse caso, você pode desativar a instalação dos CRDs do OpenReport definindo `installOpenReportCRDs` como`false` no gráfico (ou `installPolicyReportCRDs` como `false` se estiver usando o PolicyReports obsoleto).
Isso significa que a pilha Kubewarden não gerenciará esses CRDs, e a responsabilidade é dos administradores.

Veja mais informações sobre os CRDs nolink:https://github.com/kubernetes-sigs/wg-policy-prototypes[repositório do grupo de trabalho de políticas].

====


. Instale o gráfico do `kubewarden-controller` Helm.
+
[subs="+attributes",console]
----
helm install kubewarden-controller kubewarden/kubewarden-controller
----
+

[NOTE]
====
O scanner de auditoria é ativado por padrão. Para desativá-lo, defina o endereço`auditScanner.enable=false`.
====
+
[NOTE]
====
Se você quiser que o Audit Scanner salve seus relatórios em CRDs do PolicyReports (marcados como obsoletos) em vez dos CRDs padrão do OpenReports, defina `auditScanner.reportCRDsKind="policyreport"`.
====

Para obter mais informações sobre a instalação do {short-project-name}, consulte oxref:quick-start.adoc[Guia de início rápido]

Por padrão, a implementação do Audit Scanner é como umhttps://kubernetes.io/docs/concepts/workloads/controllers/cron-jobs[] Cronjob acionado a cada 60 minutos. Você pode ajustar essa e outras configurações do scanner de auditoria alterando o gráfico kubewarden-controllerhttps://github.com/kubewarden/helm-charts/blob/main/charts/kubewarden-controller/values.yaml[values.yaml].

Veja xref:explanations/audit-scanner/audit-scanner.adoc[aqui] mais informações sobre o Audit Scanner.

=== UI do Policy Reporter (opcional)

O gráfico `kubewarden-controller` vem com um subgráfico dohttps://kyverno.github.io/policy-reporter[Policy Reporter]. Ele é desativado por padrão e ativado pela configuração `auditScanner.policyReporter=true`. Os valores do subgráfico do Policy Reporter estão sob a chave `policyReporter` dos valores`kubewarden-controller`.

Isso instala apenas parte do gráfico upstream do Policy Reporter, a interface do usuário, que fornece uma visualização dos PolicyReports e ClusterPolicyReports em um cluster. Consulte xref:explanations/audit-scanner/audit-scanner.adoc[Audit Scanner]para obter mais informações sobre a interface do usuário do Policy Reporter.

Por padrão, a interface do usuário do Policy Reporter só é exposta como um serviço ClusterIP com o nome `kubewarden-controller-ui` no namespace de instalação de

`kubewarden-controller` gráfico foi instalado.

==== Ingresso

Os usuários podem fornecer sua própria configuração de Ingress ou ativar um Ingress por meio da configuração do subgráficohttps://github.com/kyverno/policy-reporter/blob/policy-reporter-2.19.4/charts/policy-reporter/charts/ui/values.yaml#L172-L189[aqui].

Veja este exemplo de uma configuração Ingress por meio do subgráfico:

[subs="+attributes",yaml]
----
auditScanner:
  policyReporter: true
policy-reporter: # subchart values settings
  ui:
    enabled: true
    ingress:
      enabled: true
      hosts:
        - host: "*.local" # change this to your appropriate domain
          paths:
            - path: /
              pathType: ImplementationSpecific
----

==== Encaminhamento de portas

Para uma rápida olhada ou depuração, é possível configurar um encaminhamento de porta para o serviço com:

[subs="+attributes",console]
----
kubectl port-forward service/kubewarden-controller-ui 8082:8080 -n kubewarden
----

O que torna a interface do usuário do Policy Reporter disponível em `\http://localhost:8082`.

== Acionar a execução manual

A implementação do scanner de auditoria é como um Cronjob que é executado a cada 60 minutos por padrão. É possível acionar uma execução manual executando o seguinte comando:

[subs="+attributes",bash]
----
kubectl create job \
    --namespace kubewarden \
    --from cronjob/audit-scanner \
    audit-scanner-manual-$(date +%Y-%m-%d-%H-%M-%S)
----

Você pode verificar o status do trabalho com:

[subs="+attributes",console]
----
kubectl get -n kubewarden jobs
----
