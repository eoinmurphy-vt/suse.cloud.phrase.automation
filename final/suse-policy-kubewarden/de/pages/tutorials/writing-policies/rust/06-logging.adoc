Sie können Ihre Richtlinie eine Protokollierung durchführen lassen.
Die `policy-server` oder `kwctl` leitet diese Protokolleinträge mit den entsprechenden Informationen weiter.

Die für das Rust-SDK gewählte Logging-Bibliothek isthttps://github.com/slog-rs/slog[`slog`].
Es ist eine beliebte, bekannte Kiste und integriert sich sauber in {project-name}.

== Logger initialisieren

Das Projekt empfiehlt Ihnen, eine globale Senke einzurichten, bei der Sie sich bei Bedarf in Ihrer Richtlinie anmelden können.
Verwenden Sie dazu die Kiste `lazy_static`:

[subs="+attributes",rust]
----
use slog::{o, Logger};

lazy_static! {
    static ref LOG_DRAIN: Logger = Logger::root(
        logging::KubewardenDrain::new(),
        o!("policy" => "sample-policy")
    );
}
----

== Verbrauchen des Loggers

Jetzt können Sie in den Funktionen `validate` oder `validate_settings` die von `slog` exportierten Makros verwenden, die zu jeder unterstützten Protokollierungsebene passen:

[subs="+attributes",rust]
----
use slog::{info, o, warn, Logger};

fn validate(payload: &[u8]) -> CallResult {
    // ...
    info!(LOG_DRAIN, "starting validation");
    // ...
    warn!(
        LOG_DRAIN, "structured log";
        "some_resource_name" => &some_resource_name
    );
    // ...
}
----

Die Bibliothek `slog` sendet alle Protokolle an den in der globalen Variable initialisierten Abfluss.
Dies synchronisiert sich mit dem Richtlinienauswerter, der die Richtlinie ausführt.
Dies ist entweder `kwctl` oder die `policy-server`.
Anschließend protokolliert der Policy Evaluator diese Informationen und fügt weitere bekannte Kontextinformationen hinzu, wie z. B. die Kubernetes-Anfrage `uid`.

Weitere Informationen über diehttps://docs.rs/slog/2.7.0/slog/macro.log.html[Logging-Makros], die von slog angeboten werden, finden Sie in derhttps://docs.rs/slog/2.7.0/slog/index.html[Dokumentation].
