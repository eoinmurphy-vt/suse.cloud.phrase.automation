== Gestaltung der Politik

Bisher haben Sie die Richtlinie mit einem Kompilierungsziel erstellt, das demselben Betriebssystem und derselben Architektur wie Ihr Entwicklungsrechner entspricht.

Nun ist es an der Zeit, die Richtlinie als WebAssembly-Binärdatei zu erstellen, eine `.wasm` -Datei.

Sie verwenden den Befehl:

[subs="+attributes",console]
----
make policy.wasm
----

Mit diesem Befehl wird der Code im Freigabemodus mit einem WebAssembly-Kompilierungsziel erstellt.

Der Build erzeugt eine Datei `policy.wasm`:

[subs="+attributes",console]
----
$ file policy.wasm
policy.wasm: WebAssembly (wasm) binary module version 0x1 (MVP)
----

== Weitere Beispiele

Sie können mehr Rust {short-project-name} Richtlinien in {short-project-name}'s GitHub Raum finden.https://github.com/search?l=Rust&q=topic%3Apolicy-as-code+org%3Akubewarden&type=Repositories[Diese Abfrage] kann Ihnen helfen, sie zu finden.

Die {short-project-name} Richtlinien-Repositories, die durch diese Abfrage angezeigt werden, verfügen über GitHub-Aktionen, die die folgenden Aufgaben automatisieren:

* Führen Sie Unit-Tests und Code-Linting bei Pull-Requests und nach dem Zusammenführen von Code in den Hauptzweig durch.
* Erstellen Sie die Richtlinie im Modus `release` und übergeben Sie sie als Artefakt an eine OCI-Registrierung.
