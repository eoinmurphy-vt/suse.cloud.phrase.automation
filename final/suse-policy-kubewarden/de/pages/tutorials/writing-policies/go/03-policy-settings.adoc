Zunächst müssen Sie die Struktur definieren, die die Richtlinieneinstellungen enthält.

Sie tun dies, indem Sie den Code in der Datei `settings.go` (aus Ihrer lokalen Version der Go-Richtlinienvorlage) ändern.
Sie müssen zwei zusätzliche Zeilen in den Abschnitt `import` einfügen, die Struktur `Settings` ändern und die Struktur `RegularExpression` hinzufügen.

Sie sollte dem folgenden Code entsprechen:

[,go]
----
import (
    "encoding/json"
    "fmt"
    "regexp"

    mapset "github.com/deckarep/golang-set/v2"
    kubewarden "github.com/kubewarden/policy-sdk-go"
    kubewarden_protocol "github.com/kubewarden/policy-sdk-go/protocol"
)

type Settings struct {
    DeniedLabels      mapset.Set[string]            `json:"denied_labels"`
    ConstrainedLabels map[string]*RegularExpression `json:"constrained_labels"`
}

type RegularExpression struct {
    *regexp.Regexp
}
----

[NOTE]
====
`DeniedNames` ist in der Struktur `Settings`, die in `settings.go` definiert ist, nicht mehr erforderlich.

Da `DeniedNames` nicht mehr definiert ist, sollten Sie auch die Funktion `IsNameDefined` in `settings.go` löschen.
Sie sollten auch die Funktion entfernen, die auf `settings_test.go` verweist, `TestIsNameDenied`.
====


Sie verwenden das Paket `regexp`, um Objekte mit regulären Ausdrücken zu behandeln, und denhttps://github.com/deckarep/golang-set[`mapset.Set`] Typ, um die Liste der verweigerten Bezeichnungen zu speichern.

Da `regexp.Regexp` keine Deserialisierung durchführt, müssen Sie benutzerdefinierte Funktionen definieren, um das Marshaling und Unarshaling von regulären Ausdrücken zu behandeln:

[,go]
----
// UnmarshalText satisfies the encoding.TextMarshaler interface,
// also used by json.Unmarshal.
func (r *RegularExpression) UnmarshalText(text []byte) error {
    nativeRegExp, err := regexp.Compile(string(text))
    if err != nil {
        return err
    }
    r.Regexp = nativeRegExp
    return nil
}

// MarshalText satisfies the encoding.TextMarshaler interface,
// also used by json.Marshal.
func (r *RegularExpression) MarshalText() ([]byte, error) {
    if r.Regexp != nil {
        return []byte(r.Regexp.String()), nil
    }

    return nil, nil
}
----

Außerdem benötigen Sie die `UnmarshalJSON` Methode, um die Deserialisierung der `Settings` Struktur zu behandeln:

[,go]
----
func (s *Settings) UnmarshalJSON(data []byte) error {
    rawSettings := struct {
        DeniedLabels      []string          `json:"denied_labels"`
        ConstrainedLabels map[string]*RegularExpression `json:"constrained_labels"`
    }{}

    err := json.Unmarshal(data, &rawSettings)
    if err != nil {
        return err
    }

    s.DeniedLabels = mapset.NewThreadUnsafeSet[string](rawSettings.DeniedLabels...)
    s.ConstrainedLabels = rawSettings.ConstrainedLabels

    return nil
}
----

== Erstellung von `Settings` Instanzen

Eine Richtlinie {short-project-name} stellt zwei verschiedene Funktionen bereit, die die Richtlinieneinstellungen als Eingabe erhalten:

* `validate`: Verwenden Sie diese Funktion, wenn das Kubernetes-Objekt eine Validierung durch die Richtlinie erfordert.
Die Einstellungen sind Teil eineshttps://pkg.go.dev/github.com/kubewarden/policy-sdk-go@v0.2.1/protocol#ValidationRequest[`ValidationRequest`]Objekts.
* `validate_settings`: Rufen Sie diese Funktion auf, wenn die Richtlinie zum ersten Mal von {short-project-name} geladen wird.
Die Funktion erhält die Richtlinieneinstellungen als Eingabe und prüft deren Gültigkeit.

Sie müssen eine Hilfsfunktion erstellen, die aus der JSON-Nutzlast ein `Settings` -Objekt erzeugt:

[,go]
----
func NewSettingsFromValidationReq(validationReq *kubewarden_protocol.ValidationRequest) (Settings, error) {
    settings := Settings{}
    err := json.Unmarshal(validationReq.Settings, &settings)
    if err != nil {
        return Settings{}, err
    }

    return settings, nil
}
----

[NOTE]
====
Alle `mapset.Set` Objekte werden mit derhttps://pkg.go.dev/github.com/deckarep/golang-set?utm_source=godoc#NewThreadUnsafeSet[thread-unsafe Variante] erstellt.
Der WebAssembly-Code wird in einem einzigen Thread ausgeführt, daher gibt es keine Probleme mit der Gleichzeitigkeit.

Der WebAssembly-Standard umfasst noch keine Threads.
Weitere Einzelheiten finden Sie im unter https://github.com/WebAssembly/threads[offiziellen Vorschlag].
====


== Implementierung von `Settings` validation

Alle {short-project-name} Richtlinien müssen die Validierung von Einstellungen implementieren.

Dazu fügen Sie eine `Valid` Methode zu den `Settings` Instanzen hinzu:

[,go]
----
func (s *Settings) Valid() (bool, error) {
    constrainedLabels := mapset.NewThreadUnsafeSet[string]()

    for label := range s.ConstrainedLabels {
        constrainedLabels.Add(label)
    }

    constrainedAndDenied := constrainedLabels.Intersect(s.DeniedLabels)
    if constrainedAndDenied.Cardinality() != 0 {
        return false,
            fmt.Errorf("These labels cannot be constrained and denied at the same time: %v", constrainedAndDenied)
    }

    return true, nil
}
----

Die Methode `Valid` stellt sicher, dass kein "verweigertes" Label auch Teil der "eingeschränkten" Karte ist.
Die Verwendung der Methode `Intersect`, die von `mapset.Set` bereitgestellt wird, vereinfacht die Prüfung.

[NOTE]
====
Der Aufruf der Methode `Valid` bezieht sich auf ein bereits instanziiertes Objekt `Setting`.
Das bedeutet, dass die Validierung des vom Benutzer angegebenen regulären Ausdrucks bereits im `Settings` unmarshaler stattgefunden hat.
====


Schließlich müssen Sie die Funktion `validateSettings`, die vom Gerüst zur Verfügung gestellt wird, so ändern, dass sie wie folgt aussieht:

[,go]
----
func validateSettings(payload []byte) ([]byte, error) {
    settings := Settings{}
    err := json.Unmarshal(payload, &settings)
    if err != nil {
        return kubewarden.RejectSettings(
            kubewarden.Message(fmt.Sprintf("Provided settings are not valid: %v", err)))
    }

    valid, err := settings.Valid()
    if valid {
        return kubewarden.AcceptSettings()
    }

    return kubewarden.RejectSettings(
        kubewarden.Message(fmt.Sprintf("Provided settings are not valid: %v", err)))
}
----

Wie Sie sehen, nutzt die Funktion die Hilfsfunktionen, die dashttps://github.com/kubewarden/policy-sdk-go[SDK von]

== Testen des Einstellungscodes

Es ist wichtig, eine gute Testabdeckung für den von Ihnen geschriebenen Code zu haben.
Der Code, den Sie aus dem Gerüst verwenden, enthält eine Reihe von Unit-Tests, die in der Datei `settings_test.go` definiert sind.

Sie müssen den Inhalt dieser Datei ändern, um das neue Verhalten der Klasse `Settings` widerzuspiegeln.

Geben Sie die von Ihnen verwendeten Go-Pakete an:

[,go]
----
import (
    "testing"

    "encoding/json"

    kubewarden_protocol "github.com/kubewarden/policy-sdk-go/protocol"
)
----

Sie können damit beginnen, einen Unit-Test zu schreiben, der sicherstellt, dass Sie eine `Settings`Instanz von einem `ValidationRequest` Objekt zuweisen können:

[,go]
----
func TestParseValidSettings(t *testing.T) {
    settingsJSON := []byte(`
        {
            "denied_labels": [ "foo", "bar" ],
            "constrained_labels": {
                    "cost-center": "cc-\\d+"
            }
        }`)

    settings := Settings{}
    err := json.Unmarshal(settingsJSON, &settings)
    if err != nil {
        t.Errorf("Unexpected error %+v", err)
    }

    expected_denied_labels := []string{"foo", "bar"}
    for _, exp := range expected_denied_labels {
        if !settings.DeniedLabels.Contains(exp) {
            t.Errorf("Missing value %s", exp)
        }
    }

    re, found := settings.ConstrainedLabels["cost-center"]
    if !found {
        t.Error("Didn't find the expected constrained label")
    }

    expected_regexp := `cc-\d+`
    if re.String() != expected_regexp {
        t.Errorf("Expected regexp to be %v - got %v instead",
            expected_regexp, re.String())
    }
}
----

Als Nächstes benötigen Sie einen Test, der überprüft, ob eine `Settings` -Instanz erzeugt wird, wenn der Benutzer einen fehlerhaften regulären Ausdruck angibt:

[,go]
----
func TestParseSettingsWithInvalidRegexp(t *testing.T) {
    settingsJSON := []byte(`
        {
            "denied_labels": [ "foo", "bar" ],
            "constrained_labels": {
                    "cost-center": "cc-[a+"
            }
        }`)

    err := json.Unmarshal(settingsJSON, &Settings{})
    if err == nil {
        t.Errorf("Didn't get expected error")
    }
}
----

Nun können Sie einen Test definieren, der das Verhalten des Einstiegspunkts`validate_settings` überprüft.

Sie sehen sich das Objekt `SettingsValidationResponse` an, das von Ihrer Funktion `validateSettings` zurückgegeben wird:

[,go]
----
func TestDetectValidSettings(t *testing.T) {
    settingsJSON := []byte(`
    {
        "denied_labels": [ "foo", "bar" ],
        "constrained_labels": {
            "cost-center": "cc-\\d+"
        }
    }`)

    responsePayload, err := validateSettings(settingsJSON)
    if err != nil {
        t.Errorf("Unexpected error %+v", err)
    }

    var response kubewarden_protocol.SettingsValidationResponse
    if err := json.Unmarshal(responsePayload, &response); err != nil {
        t.Errorf("Unexpected error: %+v", err)
    }

    if !response.Valid {
        t.Errorf("Expected settings to be valid: %s", *response.Message)
    }
}
----

Schließlich schreiben Sie zwei weitere Tests, um zu prüfen, ob die Funktion `validateSettings` ungültige Einstellungen mit den richtigen Meldungen zurückweist:

[,go]
----
func TestDetectNotValidSettingsDueToBrokenRegexp(t *testing.T) {
    settingsJSON := []byte(`
    {
        "denied_labels": [ "foo", "bar" ],
        "constrained_labels": {
            "cost-center": "cc-[a+"
        }
    }
    `)

    responsePayload, err := validateSettings(settingsJSON)
    if err != nil {
        t.Errorf("Unexpected error %+v", err)
    }

    var response kubewarden_protocol.SettingsValidationResponse
    if err := json.Unmarshal(responsePayload, &response); err != nil {
        t.Errorf("Unexpected error: %+v", err)
    }

    if response.Valid {
        t.Error("Expected settings to not be valid")
    }

    if *response.Message != "Provided settings are not valid: error parsing regexp: missing closing ]: `[a+`" {
        t.Errorf("Unexpected validation error message: %s", *response.Message)
    }
}

func TestDetectNotValidSettingsDueToConflictingLabels(t *testing.T) {
    settingsJSON := []byte(`
    {
        "denied_labels": [ "foo", "bar", "cost-center" ],
        "constrained_labels": {
            "cost-center": ".*"
        }
    }`)
    responsePayload, err := validateSettings(settingsJSON)
    if err != nil {
        t.Errorf("Unexpected error %+v", err)
    }

    var response kubewarden_protocol.SettingsValidationResponse
    if err := json.Unmarshal(responsePayload, &response); err != nil {
        t.Errorf("Unexpected error: %+v", err)
    }

    if response.Valid {
        t.Error("Expected settings to not be valid")
    }

    if *response.Message != "Provided settings are not valid: These labels cannot be constrained and denied at the same time: Set{cost-center}" {
        t.Errorf("Unexpected validation error message: %s", *response.Message)
    }
}
----
