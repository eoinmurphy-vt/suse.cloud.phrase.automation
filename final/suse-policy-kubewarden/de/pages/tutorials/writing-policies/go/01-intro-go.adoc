Der offizielle Go-Compiler kann seit Version 1.21 WebAssembly-Binärdateien erzeugen, die außerhalb eines Browsers ausgeführt werden können.

Es gibt einen weiteren Go-Compiler, der WebAssembly-Binärdateien erstellen kann, die von {project-name} verwendet werden können.
Dieses Compiler-Projekt ist https://tinygo.org/[] TinyGo:

____
TinyGo bringt die Programmiersprache Go auf eingebettete Systeme und in das moderne Web, indem es einen neuen, auf LLVM basierenden Compiler entwickelt.

Sie können TinyGo-Programme auf über 94 verschiedenen Mikrocontroller-Boards wie dem BBC micro:bit und dem Arduino Uno kompilieren und ausführen.

TinyGo kann auch WebAssembly (Wasm)-Code erzeugen, der sehr kompakt in der Größe ist.
Sie können Programme für Webbrowser sowie für Server- und Edge-Computing-Umgebungen kompilieren, die die WebAssembly System Interface (WASI) Familie von Schnittstellen unterstützen.
____

Das Projekt {short-project-name} empfiehlt derzeit die Verwendung von TinyGo aus zwei Gründen:

* Binärdateien sind kleiner
* Unterstützung für https://wapc.io[waPC] durch die Möglichkeit, Funktionen zur Laufzeit zu exportieren

== TinyGo-Einschränkungen

TinyGo unterstützt noch nicht alle Go-Funktionen, siehe die TinyGo-Sprachunterstützung https://tinygo.org/lang-support/[Seite], um den aktuellen Projektstatus zu sehen.
Die größte Einschränkung ist derzeit das Fehlen eines vollständig unterstützten `reflect` Pakets.
Das bedeutet, dass offizielle Kubernetes-Go-API-Typen (z.B.: `k8s.io/api/core/v1`) nicht kompiliert werden können.

{short-project-name} Richtlinien müssen JSON-Daten wie Richtlinieneinstellungen und die von Kubernetes empfangene Anfrage verarbeiten.

Trotz der derzeitigen Einschränkungen von TinyGo ist es immer noch einfach, {short-project-name} Validierungsrichtlinien damit zu schreiben.

== Werkzeugbau

Das Schreiben von {short-project-name} Richtlinien erfordert eine Version von TinyGo, die größer als `v0.28.1` ist.
Verwenden Sie jedoch die neueste Version, um die besten Ergebnisse zu erzielen.

[CAUTION]
====
Die Verwendung älterer Versionen von TinyGo führt aufgrund der eingeschränkten Unterstützung für Go-Reflexion zu Laufzeitfehlern.
====


Diese Go-Bibliotheken sind beim Schreiben einer {short-project-name} -Policy nützlich:

* https://github.com/kubewarden/policy-sdk-go[{short-project-name} Go SDK]:
Bietet Strukturen und Funktionen, die den Umfang des erforderlichen Codes reduzieren.
Es bietet auch Testhilfsmittel.
* https://github.com/kubewarden/k8s-objects[Kubernetes Go-Typen]:
Die https://github.com/kubernetes/kubernetes/tree/master/staging/src/k8s.io[offiziellen]können nicht mit TinyGo verwendet werden.
Dieses Modul bietet alle Kubernetes-Typen in einer TinyGo-freundlichen Weise.
* https://github.com/tidwall/gjson[gjson]:
Diese bietet eine Abfragesprache für die schnelle Navigation in JSON-Dokumenten und den Abruf von Daten.
Diese Bibliothek verwendet nicht das `encoding/json` -Paket, das von Go's `stdlib` bereitgestellt wird, daher ist sie mit TinyGo verwendbar.
* https://github.com/deckarep/golang-set[mapset]:
Stellt eine Go-Implementierung der Datenstrukturhttps://en.wikipedia.org/wiki/Set_(abstract_data_type)[Set]zur Verfügung.
Diese Bibliothek reduziert den Umfang des zu schreibenden Codes, da Operationen wie Set `union`, `intersection`, `difference` in {short-project-name} Policies üblich sind.

Schließlich bietet das Projekt {short-project-name} einehttps://github.com/kubewarden/go-policy-template[Go-Richtlinienvorlage], mit der Sie {short-project-name} Go-Richtlinien erstellen können.

== TinyGo-Abhängigkeiten erhalten

Der einfachste Weg, TinyGo zu bekommen, ist die Verwendung der Upstream-Container-Images.
Offizielle Veröffentlichungen sindhttps://hub.docker.com/r/tinygo/tinygo[hier], während Builds aus dem Entwicklungszweig automatischhttps://hub.docker.com/r/tinygo/tinygo-dev[] veröffentlicht werden.

Falls erforderlich, finden Sie weitere Informationen auf der TinyGo-Seitehttps://tinygo.org/getting-started/[].

== Voraussetzungen für das Tutorial

Während dieses Tutorials benötigen Sie diese Werkzeuge auf Ihrem Entwicklungsrechner:

* Docker oder eine andere Container-Engine: wird zur Erstellung der WebAssembly-Richtlinie verwendet.
Sie werden den Compiler verwenden, der im offiziellen TinyGo-Container-Image enthalten ist.
* https://github.com/bats-core/bats-core[bats]: für das Schreiben der Tests und die Automatisierung ihrer Ausführung.
* https://github.com/kubewarden/kwctl/releases[`kwctl`]:
CLI-Tool, das von {short-project-name} bereitgestellt wird, um unter anderem seine Richtlinien außerhalb von Kubernetes auszuführen.
Dies wird in xref:tutorials/testing-policies/index.adoc[diesem Abschnitt] der Dokumentation behandelt.
