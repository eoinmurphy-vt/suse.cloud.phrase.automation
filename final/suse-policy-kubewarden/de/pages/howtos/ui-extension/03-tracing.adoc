Die Rückverfolgung ermöglicht die Erfassung detaillierter Informationen über die Bewertung von Maßnahmen.
Es ist ein nützliches Tool zur Fehlersuche in Ihrer {project-name} Bereitstellung und Ihren Richtlinien.

{short-project-name} verwendet https://www.jaegertracing.io/[Jaeger], um Trace-Ereignisse zu empfangen, zu speichern und zu visualisieren.

*_Protokolle der Richtlinienverfolgung_*image:ui_policy_tracing.png[UI-Richtlinienverfolgungsprotokolle]

== Voraussetzungen

Sie benötigen Cert-Manager und OpenTelemetry. Folgen Siexref:howtos/telemetry/10-opentelemetry-qs.adoc#_install_opentelemetry[diesen Anweisungen], um Cert Manager und OpenTelemetry Operator zu installieren.

== Jaeger installieren

Wenden Sie die Installationsschritte aus demxref:howtos/telemetry/20-tracing-qs.adoc#_install_jaeger[Tracing Quickstart] an.

Sobald der Jaeger-Operator alle Ressourcen erstellt hat, gibt es einen Service unter`my-open-telemetry-collector.jaeger.svc.cluster.local`.

Die Jaeger Query UI ist unter der folgenden Adresse zu erreichen:

[subs="+attributes",console]
----
https://<CLUSTER_IP>/api/v1/namespaces/jaeger/services/http:my-open-telemetry-query:16686/proxy/search
----

Dieser Endpunkt kann je nach Ihrer Konfiguration eindeutig sein. Sie können den Endpunkt für Ihre Jaeger-Ressource auf der Seite Dienste finden.

=== `rancher-kubewarden-controller` mit Jaeger-Endpunkt aktualisieren

Sie müssen die Ressource `rancher-kubewarden-controller` bearbeiten, um den Jaeger-Endpunkt "my-open-telemetry-collector.jaeger.svc.cluster.local:4317" hinzuzufügen.

Zum Beispiel:

[subs="+attributes",yaml]
----
telemetry:
  tracing: true
  mode: sidecar
  sidecar:
    tracing:
      jaeger:
        endpoint: my-open-telemetry-collector.jaeger.svc.cluster.local:4317
        tls:
          insecure: true
----

Die Installation des Dienstes `my-open-telemetry-collector` erfolgt unter dem Namensraum`jaeger`.

[CAUTION]
====

Der Einfachheit halber verschlüsseln wir die Kommunikation zwischen dem OpenTelemetry-Kollektor und dem Jaeger-Endpunkt nicht.

Dies ist *für den Produktionseinsatz ungeeignet*. Sie sollten die von Jaegerhttps://www.jaegertracing.io/docs/latest/kubernetes/[Dokumentation] lesen.

====


Sie sollten nun in der Lage sein, alle fehlgeschlagenen Anfragen für Richtlinien anzuzeigen, die an einen bestimmten Policy Server gebunden sind. Sie können auch die Detailansicht für eine bestimmte Richtlinie einsehen.
Mit der Jaeger-Benutzeroberfläche erhalten Sie eine detailliertere Ansicht der Spuren.
