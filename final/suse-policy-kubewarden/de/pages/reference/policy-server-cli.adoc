== Kommandozeilen-Hilfe für `policy-server`

Dieses Dokument enthält den Inhalt der Hilfe für das Befehlszeilenprogramm `policy-server`.

*Befehlsübersicht:*

* link:#policy-server[`policy-server`]
* link:#policy-server-docs[`policy-server docs`]

=== `policy-server`

*Verwendung:* `policy-server ++[+OPTIONS+]++ ++[+COMMAND+]++`

==== *Unterbefehle:*

* `docs` - Erzeugt die Markdown-Dokumentation für die Befehle des Policy-Servers

==== *Optionen:*

* `--addr ++<+BIND_ADDRESS+>++` - Bindung gegen ADDRESS
+
Standardwert: `0.0.0.0`
* `--always-accept-admission-reviews-on-namespace ++<+NAMESPACE+>++` - Akzeptiert immer AdmissionReviews, die auf den angegebenen Namensraum abzielen
* `--cert-file ++<+CERT_FILE+>++` - Pfad zu einer X.509-Zertifikatsdatei für HTTPS
* `--client-ca-file ++<+CLIENT_CA_FILE+>++` - Pfad zu einer CA-Zertifikatsdatei, die das Client-Zertifikat ausgestellt hat. Erforderlich zur Aktivierung von mTLS
* `--daemon` - Wenn gesetzt, läuft policy-server im abgetrennten Modus als Daemon
* `--daemon-pid-file ++<+DAEMON-PID-FILE+>++` - Pfad zur PID-Datei, wird nur im Daemon-Modus verwendet
+
Standardwert: `policy-server.pid`
* `--daemon-stderr-file ++<+DAEMON-STDERR-FILE+>++` - Pfad zur Datei stderr, wird nur im Daemon-Modus verwendet
* `--daemon-stdout-file ++<+DAEMON-STDOUT-FILE+>++` - Pfad zu der Datei, die stdout enthält, wird nur im Daemon-Modus verwendet
* `--disable-timeout-protection` - Deaktivieren des Zeitüberschreitungsschutzes für Richtlinien
* `--docker-config-json-path ++<+DOCKER_CONFIG+>++` - Pfad zu einem Docker config.json-ähnlichen Pfad. Kann zur Angabe von Registrierungsauthentifizierungsdetails verwendet werden
* `--enable-metrics` - Metriken einschalten
* `--enable-pprof` - Profilierung von pprof aktivieren
* `--ignore-kubernetes-connection-failure` - Beenden Sie nicht mit einem Fehler, wenn die Kubernetes-Verbindung fehlschlägt. Dies führt dazu, dass kontextabhängige Richtlinien nicht mehr funktionieren, wenn keine Verbindung zu Kubernetes vorhanden ist.
* `--key-file ++<+KEY_FILE+>++` - Pfad zu einer X.509-Schlüsseldatei für HTTPS
* `--log-fmt ++<+LOG_FMT+>++` - Format der Protokollausgabe
+
Standardwert: `text`
+
Mögliche Werte: `text`, `json`, `otlp`
* `--log-level ++<+LOG_LEVEL+>++` - Log-Stufe
+
Standardwert: `info`
+
Mögliche Werte: `trace`, `debug`, `info`, `warn`, `error`
* `--log-no-color` - Farbige Ausgabe für Protokolle deaktivieren
* `--policies ++<+POLICIES_FILE+>++` - YAML-Datei mit den zu ladenden Richtlinien und deren Einstellungen
+
Standardwert: `policies.yml`
* `--policies-download-dir ++<+POLICIES_DOWNLOAD_DIR+>++` - Download-Pfad für die Richtlinien
+
Standardwert: `.`
* `--policy-timeout ++<+MAXIMUM_EXECUTION_TIME_SECONDS+>++`
- Unterbrechung der Richtlinienauswertung nach einer bestimmten Zeit
+
Standardwert: `2`
* `--port ++<+PORT+>++` - PORT anhören
+
Standardwert: `3000`
* `--readiness-probe-port ++<+READINESS_PROBE_PORT+>++` —
Expose readiness endpoint on READINESS+_PROBE_+PORT
+
Standardwert: `8081`
* `--sigstore-cache-dir ++<+SIGSTORE_CACHE_DIR+>++` - Verzeichnis für die Zwischenspeicherung von Sigstore-Daten
+
Standardwert: `sigstore-data`
* `--sources-path ++<+SOURCES_PATH+>++` - YAML-Datei mit Quellinformationen (https, unsichere Hosts in der Registrierung, benutzerdefinierte CAs\...)
* `--verification-path ++<+VERIFICATION_CONFIG_PATH+>++` - YAML-Datei mit Überprüfungsinformationen (URIs, Schlüssel, Anmerkungen\...)
* `--workers ++<+WORKERS_NUMBER+>++` - Anzahl der zu erstellenden Arbeits-Threads
