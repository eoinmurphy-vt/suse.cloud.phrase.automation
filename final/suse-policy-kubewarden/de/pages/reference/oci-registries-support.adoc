Sie verteilen Ihre {project-name} Richtlinien alshttps://opencontainers.org[OCI-Artefakte]unter Verwendung regulärer Open Container Initiative (OCI)-Registrierungen.

{short-project-name} Die Speicherung der Richtlinien erfolgt neben den Container-Images.
Sie erfordern keine zusätzliche Einrichtung oder Wartung, die über die für normale Container-Images erforderliche hinausgeht.

[NOTE]
====
Sie können eine Registrierung hinzufügen, die mit {short-project-name} funktioniert, oder ungenaue Registrierungen korrigieren, indem Sie die Schaltfläche "Diese Seite bearbeiten" unten auf der Seite verwenden
====

== Bekannte Probleme

=== Docker-Hub

Derzeit unterstützt Docker Hub keine OCI-Artefakte und kann daher nicht zum Speichern von {short-project-name} Richtlinien verwendet werden.
Docker Inc. hat angekündigt, dass Docker Hub in OCI-Artefakte unterstützen wird (https://www.docker.com/blog/announcing-docker-hub-oci-artifacts-support/[Zukunft]).

=== JFrog

Obwohl JFrog OCI-Artefakte unterstützt, ist es nur teilweise möglich, sie zu pushen, wenn man ihrer Spezifikation folgt.https://github.com/kubewarden/kwctl/issues/59[Lesen Sie hier mehr].
