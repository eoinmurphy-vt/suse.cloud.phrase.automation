= Colector OpenTelemetry personalizado
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: Configure el {project-name} para enviar de forma segura los datos de telemetría a un colector OpenTelemetry personalizado desplegado en el clúster.
:keywords: kubewarden, kubernetes, métricas, rastreo, opentelemetry
:doc-persona: kubewarden-operator, kubewarden-integrator
:doc-type: cómo
:doc-topic: manual del operador, telemetría, métricas, inicio rápido

Esta guía explica cómo configurar {short-project-name} para enviar datos de telemetría a un recopilador OpenTelemetry ya desplegado en el clúster.

Sólo debe desplegar una instancia dehttps://opentelemetry.io/docs/collector/[OpenTelemetry Collector] en el clúster.

== Instalar dependencias

En primer lugar, comience por instalar las dependencias de OpenTelemetry Collector.

Necesita una comunicación cifrada entre los componentes de {project-name} y el recopilador. Puede utilizar https://cert-manager.io/[cert-manager] para gestionar todos los certificados necesarios para una comunicación segura.

Las trazas de OpenTelemetry Collector se envían a una instancia dehttps://www.jaegertracing.io/[Jaeger].

La pila {short-project-name} envía métricas al colector OpenTelemetry.
Este expone las métricas como un punto final de Prometheus. A continuación, una instancia de Prometheus recoge las métricas y las almacena en su base de datos. La misma instancia de Prometheus también expone una interfaz de usuario para ver y utilizar las métricas.

Los recursos que se crean se definen en el espacio de nombres `kubewarden`, o se espera su existencia. Debido a ello, debe empezar por crear el Namespace:

[source,console]
----
kubectl create namespace kubewarden
----

=== Instalar cert-manager y OpenTelemetry

Instale cert-manager y el operador OpenTelemetry de esta manera:

[source,console]
----
helm repo add jetstack https://charts.jetstack.io
helm install --wait \
  --namespace cert-manager \
  --create-namespace \
  --set crds.enabled=true \
  --version 1.18.2 \
  cert-manager jetstack/cert-manager

helm repo add open-telemetry https://open-telemetry.github.io/opentelemetry-helm-charts
helm install --wait \
  --namespace open-telemetry \
  --create-namespace \
  --version 0.97.1 \
  --set "manager.collectorImage.repository=otel/opentelemetry-collector-contrib" \
  my-opentelemetry-operator open-telemetry/opentelemetry-operator
----

Usted configura la comunicación entre los componentes de {short-project-name} y el Colector OpenTelemetry utilizando mTLS.

Para ello, es necesario crear toda la Infraestructura de Clave Pública (PKI):

[source,yaml]
----
# pki.yaml file
apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: my-client-certificate
  namespace: kubewarden
spec:
  dnsNames:
  - kubewarden.kubewarden.svc
  - kubewarden.kubewarden.svc.cluster.local
  issuerRef:
  kind: Issuer
  name: my-selfsigned-issuer
  secretName: my-client-cert
---
apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: my-certificate
  namespace: kubewarden
spec:
  dnsNames:
  - my-collector-collector.kubewarden.svc
  - my-collector-collector.kubewarden.svc.cluster.local
  issuerRef:
  kind: Issuer
  name: my-selfsigned-issuer
  secretName: my-server-cert
---
apiVersion: cert-manager.io/v1
kind: Issuer
metadata:
  name: my-selfsigned-issuer
  namespace: kubewarden
spec:
  selfSigned: {}
----
Aplica el manifiesto:
[source,console]
----
kubectl apply -f pki.yaml
----

=== Instalar Jaeger y Prometheus

A continuación, instale https://www.jaegertracing.io/[Jaeger] para almacenar y visualizar los eventos de rastreo.

[source,console]
----
helm repo add jaegertracing https://jaegertracing.github.io/helm-charts
helm upgrade -i --wait \
  --namespace jaeger \
  --create-namespace \
  --version 2.57.0 \
  jaeger-operator jaegertracing/jaeger-operator \
  --set rbac.clusterRole=true

kubectl apply -f - <<EOF
apiVersion: jaegertracing.io/v1
kind: Jaeger
metadata:
  name: my-open-telemetry
  namespace: jaeger
spec: {}
EOF
----

Si ha instalado Traefik en el inicio rápido de OpenTelemetry, exponga la interfaz de usuario de consulta Jaeger con este `Ingress`:

[yaml]
----
kubectl apply -f - <<EOF apiVersion: networking.k8s.io/v1 kind: Metadatos de entrada: name: my-open-telemetry-query namespace: jaeger spec: ingressClassName: traefik rules:- http: paths: - path: / pathType: Prefijo backend: servicio: nombre: my-open-telemetry-query puerto: número: 16686
EOF
----

Ahora instale https://prometheus.io/[Prometheus] para almacenar y visualizar las métricas.

[source,console]
----
cat <<EOF > kube-prometheus-stack-values.yaml
prometheus:
  additionalServiceMonitors:
  - name: kubewarden
    selector:
    matchLabels:
      app.kubernetes.io/instance: kubewarden.my-collector
    namespaceSelector:
    matchNames:
      - kubewarden
    endpoints:
    - port: prometheus
    interval: 10s
EOF

helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm install --wait --create-namespace \
  --namespace prometheus \
  --version 77.13.0 \
  --values kube-prometheus-stack-values.yaml \
  prometheus prometheus-community/kube-prometheus-stack
----

[NOTE]
====

El monitor de servicios Prometheus obtiene las métricas de {short-project-name} extrayendo el recopilador OpenTelemetry que se ejecuta en el espacio de nombres `kubewarden`.

====

== Instalar el colector OpenTelemetry

Ahora puede desplegar un colector personalizado de OpenTelemetry en el espacio de nombres `kubewarden`.

[source,yaml]
----
# otel-collector.yaml file
apiVersion: opentelemetry.io/v1beta1
kind: OpenTelemetryCollector
metadata:
  name: my-collector
  namespace: kubewarden
spec:
  mode: deployment # This configuration is omittable.
  volumes:
  - name: server-certificate
    secret:
    secretName: my-server-cert
  - name: client-certificate
    secret:
    secretName: my-client-cert
  volumeMounts:
  - name: server-certificate
    mountPath: /tmp/etc/ssl/certs/my-server-cert
    readOnly: true
  - name: client-certificate
    mountPath: /tmp/etc/ssl/certs/my-client-cert
    readOnly: true
  config:
  receivers:
    otlp:
    protocols:
      grpc:
      tls:
        cert_file: /tmp/etc/ssl/certs/my-server-cert/tls.crt
        key_file: /tmp/etc/ssl/certs/my-server-cert/tls.key
        client_ca_file: /tmp/etc/ssl/certs/my-client-cert/ca.crt
  processors: {}
  exporters:
    debug:
    verbosity: normal
    prometheus:
    endpoint: ":8080"
    otlp/jaeger:
    endpoint: "my-open-telemetry-collector.jaeger.svc.cluster.local:4317"
    tls:
      insecure: true
  service:
    pipelines:
    metrics:
      receivers: [otlp]
      processors: []
      exporters: [debug, prometheus]
    traces:
      receivers: [otlp]
      processors: []
      exporters: [debug, otlp/jaeger]
----
Aplica el manifiesto:
[source,console]
----
kubectl apply -f otel-collector.yaml
----

Esta configuración utiliza una cadena de procesamiento trivial para recibir eventos de rastreo y reenviarlos a Jaeger. También recibe métricas y las expone para que las recopile Prometheus.

Usted asegura la comunicación entre la pila {short-project-name} y el colector OpenTelemetry utilizando mTLS. Sin embargo, la comunicación entre el OpenTelemetry Collector y Jaeger no está asegurada, para reducir la complejidad del ejemplo.

== Instale la pila {short-project-name} 

Cuando el colector OpenTelemetry se está ejecutando, puede implementar{short-project-name} de la forma habitual.

Es necesario configurar los componentes de {short-project-name} para que envíen eventos y métricas al Colector de OpenTelemetry.

[source,yaml]
----
# values.yaml
telemetry:
  mode: custom
  metrics: True
  tracing: True
  custom:
  endpoint: "https://my-collector-collector.kubewarden.svc:4317"
  insecure: false
  otelCollectorCertificateSecret: "my-server-cert"
  otelCollectorClientCertificateSecret: "my-client-cert"
----

El Secreto al que hace referencia la clave `otelCollectorCertificateSecret` debe tener una entrada denominada `ca.crt`. Que contiene el certificado de la CA que emitió el certificado utilizado por el OpenTelemetry Collector.

El Secreto referenciado por la clave `otelCollectorClientCertificateSecret` debe tener las siguientes entradas: `tls.crt` y `tls.key` claves. Estos son el certificado de cliente y su clave utilizada por la pila {short-project-name} para autenticar contra el colector OpenTelemetry.

Deje estos valores vacíos si no utiliza cifrado o mTLS.

Instale la pila {short-project-name}:

[source,console]
----
helm install --wait \
  --namespace kubewarden --create-namespace \
  kubewarden-crds kubewarden/kubewarden-crds
helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  --values values.yaml \
  kubewarden-controller kubewarden/kubewarden-controller
helm install --wait \
  --namespace kubewarden \
  --create-namespace \
  kubewarden-defaults kubewarden/kubewarden-defaults \
  --set recommendedPolicies.enabled=True \
  --set recommendedPolicies.defaultPolicyMode=monitor
----

Ahora todo está en su sitio.

== Explorar la interfaz de usuario de Jaeger

Puede ver los eventos de rastreo generados por {short-project-name} utilizando la interfaz de usuario web de Jaeger. Están agrupados en el servicio `kubewarden-policy-server`:

image::jaeger-custom-otel-collector.png[El salpicadero de Jaeger]

Para acceder a la interfaz de usuario de Jaeger con Traefik, reenvíe por puerto el servicio Traefik:

[source,console]
----
kubectl -n traefik port-forward service/traefik 8080:80
----

La interfaz de usuario web está disponible en `\http://localhost:8080`.

Si prefiere prescindir de Traefik, reenvíe por puerto directamente el servicio de consulta de Jaeger:

[source,console]
----
kubectl -n jaeger port-forward service/my-open-telemetry-query 16686
----

La interfaz de usuario web está disponible en `\http://localhost:16686`.

== Explorar la interfaz de Prometheus

Puede acceder a la interfaz de usuario de Prometheus reenviando el puerto a su máquina local:

[source,console]
----
kubectl port-forward -n prometheus --address 0.0.0.0 svc/prometheus-operated 9090
----

Ahora se puede acceder a la interfaz de usuario web en `\http://localhost:9090`.
