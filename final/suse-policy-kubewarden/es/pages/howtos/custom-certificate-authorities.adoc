Tanto con `kwctl` como con `policy-server`puede extraer políticas de los registros de la Open Container Initiative (OCI) y de los servidores HTTP.
Sólo se pueden enviar pólizas a registros OCI.
Por defecto, se utiliza HTTPS con verificación TLS del host para esto.

El almacén de autoridades de certificación (CA) del sistema se utiliza para validar la cadena de confianza de certificados del registro OCI.
En una instalación estándar de {project-name}, `policy-server` utiliza el almacén CA incluido en su contenedor Linux.
En el lado del cliente, `kwctl` utiliza el almacén CA de su sistema operativo.

Si utiliza elhttps://github.com/kubewarden/kubewarden-controller[controlador], puede configurar el PolicyServer a través de susxref:/howtos/policy-servers/01-custom-cas.adoc[campos`spec` ].

[NOTE]
====

El comportamiento predeterminado de `kwctl` y `policy-server` impone HTTPS con certificados de confianza que coinciden con el almacén de CA del sistema.
Puede interactuar con los registros utilizando certificados no fiables o incluso sin TLS, utilizando la configuración `insecure_sources`.
Está claro que no es para entornos de producción.
====


== El archivo `sources.yaml` 

Puede ajustar el comportamiento push-pull de `kwctl` y `policy-server` utilizando el archivo `sources.yaml`.

Para más detalles, consulte la xref:../reference/sources_yaml.adoc[`sources.yaml`] referencia.
