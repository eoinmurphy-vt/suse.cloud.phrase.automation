= Creación de una política
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: Aprenda a crear y configurar políticas de validación que controlen las solicitudes de creación de Pod basándose en criterios de rechazo personalizables.
:doc-persona: ["kubewarden-policy-developer"]
:doc-topic: ["kubewarden", "writing-policies", "rust", "creating-policies"]
:doc-type: ["tutorial"]
:keywords: ["kubewarden", "kubernetes", "policy creation"]
:sidebar_label: Creación de una política
:current-version: {page-origin-branch}

Por ejemplo, puede crear una política de validación sencilla que procese las solicitudes de creación de Pod.

La política mira el atributo `metadata.name` del Pod y rechaza los pods que tienen un nombre no válido.
Su lista de nombres no válidos debe ser configurable por los usuarios finales de la política.

La configuración de la política es algo así

[subs="+attributes",yaml]
----
invalid_names:
- bad_name1
- bad_name2
----

La política debe aceptar la creación de un Pod como el siguiente:

[subs="+attributes",yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  containers:
    - name: nginx
      image: nginx:latest
----

Debe rechazar la creación de un Pod como:

[subs="+attributes",yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: bad_name1
spec:
  containers:
    - name: nginx
      image: nginx:latest
----

== Andamiaje del proyecto de nueva política

Puede crear un nuevo proyecto de política utilizando `cargo generate` con elhttps://github.com/kubewarden/rust-policy-template[proyecto de plantilla].

En primer lugar, instale `cargo-generate`. Esto requiere https://pkgs.org/download/openssl-devel[openssl-devel].

[subs="+attributes",shell]
----
cargo install cargo-generate
----

Ahora andamiaje el proyecto de la siguiente manera:

[subs="+attributes",shell]
----
cargo generate --git https://github.com/kubewarden/rust-policy-template \
               --branch main \
               --name demo
----

El comando produce una salida como:

[subs="+attributes",console]
----
🔧   Creating project called `demo`...
✨   Done! New project created /<some-path-name>/demo
----

Esto crea el nuevo proyecto de política en el subdirectorio `demo`.

[NOTE]
====

Si tiene previsto utilizar la funcionalidad de registro de contenedores de GitHub en la demostración, debehttps://docs.github.com/en/packages/working-with-a-github-packages-registry/enabling-improved-container-support-with-the-container-registry#enabling-the-container-registry-for-your-personal-account[activar la compatibilidad mejorada con contenedores en].
