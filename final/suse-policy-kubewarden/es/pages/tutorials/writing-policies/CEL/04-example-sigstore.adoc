Como ejemplo de otro uso de las capacidades de host en`cel-policy` de kubewarden, vamos a crear una política que verifique todas las imágenes de contenedor en un Pod comprobando sus firmas sin clave Sigstore.

== Ejemplo: Política de verificación de Sigstore

Esta política comprobará todas las imágenes de contenedor en el Pod Sigstore, y verificará que las imágenes están firmadas y son de confianza.

En este caso, buscaremos una firma sin clave Sigstore realizada en GitHub Actions. Este tipo de firmas sin clave se vinculan al emisor OIDC de GitHub a la hora de crear los certificados criptográficos, por lo que sólo necesitaremos conocer la organización de GitHub bajo la que se publica la imagen del contenedor. Puede obtener más información sobre las capacidades de acogida de Sigstorexref:reference/spec/host-capabilities/02-signature-verifier-policies.adoc[aquí].

Para conseguirlo en CEL, utilizamos las bibliotecas de extensión de CEL https://github.com/kubewarden/cel-policy?tab=readme-ov-file#host-capabilities[{project-name}'s para capacidades de host] en la política, en particular la función https://pkg.go.dev/github.com/kubewarden/cel-policy/internal/cel/library#Sigstore[`githubAction`].

Como siempre, podemos empezar con `kwctl`:

[subs="+attributes",console]
----
$ kwctl scaffold manifest -t ClusterAdmissionPolicy \
  registry://ghcr.io/kubewarden/policies/cel-policy:v1.0.0`
----

Que luego podemos editar para que sea relevante para nuestra política de verificación de contenedores:

[subs="+attributes",yaml]
----
apiVersion: policies.kubewarden.io/v1
kind: ClusterAdmissionPolicy
metadata:
  name: "cel-sigstore-keyless-verification"
spec:
  module: ghcr.io/kubewarden/policies/cel-policy:v1.0.0
  namespaceSelector:
    matchLabels:
      kubernetes.io/metadata.name: default
  rules:
    - apiGroups: [""]
      apiVersions: ["v1"]
      resources: ["pods"]
      operations: ["CREATE", "UPDATE"]
----

Ahora, vamos a la parte del CEL. Obtendremos una lista de imágenes contenedoras en el objeto Pod actual y, a continuación, comprobaremos que están verificadas por una firma que coincide con nuestra organización de elección (en este caso, github.com/opencontainers):

[subs="+attributes",yaml]
----
apiVersion: policies.kubewarden.io/v1
kind: ClusterAdmissionPolicy
metadata:
  name: "cel-sigstore-keyless-verification"
spec:
  module: ghcr.io/kubewarden/policies/cel-policy:v1.0.0
  namespaceSelector:
    matchLabels:
      kubernetes.io/metadata.name: default
  rules:
    - apiGroups: [""]
      apiVersions: ["v1"]
      resources: ["pods"]
      operations: ["CREATE", "UPDATE"]
  settings:
    variables:
      - name: containerImages
        expression: |
          object.spec.containers.map(c, c.image)
      - name: containerImagesNotVerified
        expression: |
          variables.containerImages.filter(image, !kw.sigstore.image(image).githubAction("opencontainers").verify().isTrusted())
    validations:
      - expression: |
          size(variables.containerImagesNotVerified) == 0
        messageExpression: "'These container images are not signed by the kubewarden GitHub organization: ' + variables.containerImagesNotVerified.join(', ')"
----

Pero espera, no debemos olvidar que los InitContainers también pueden formar parte de los Pods. Así que vamos a añadir otra variable y validación:

```yaml title="./cel-policy-sigstore.yaml"
apiVersion: policies.kubewarden.io/v1
kind: ClusterAdmissionPolicy metadata: name: "cel-sigstore-keyless-verification" spec: module: ghcr.io/kubewarden/policies/cel-policy:v1.0.0 namespaceSelector: matchLabels: kubernetes.io/metadata.name: default rules:- apiGroups: [""] apiVersions: ["v1"] resources: ["pods"] operations: ["CREATE", "UPDATE"] settings: variables: - name: containerImages expression: | Nombre: initContainerImages Expresión: | has(object.spec.initContainerImages) ? object.spec.initContainers.map(c, c.image) : [] - nombre: containerImagesNotVerified expresión: | variables.containerImages.filter(image, !kw.sigstore.image(image).githubAction("opencontainers").verify().isTrusted()) - nombre: initContainerImagesNotVerified expresión: | variables.initContainerImages.filter(image, !kw.sigstore.image(image).githubAction("opencontainers").verify().isTrusted()) validaciones:- expresión: | size(variables.containerImagesNotVerified) == 0 messageExpression: "'Estas imágenes de contenedor no están firmadas por la organización GitHub de kubewarden: ' + variables.containerImagesNotVerified.join(', ')" - expression: | size(variables.initContainerImagesNotVerified) == 0 messageExpression: "'Estas imágenes de contenedor init no están firmadas por la organización GitHub de kubewarden: ' + variables.initContainerImagesNotVerified.join(', ')"
```

As usual with CEL, we can add several `validations` under
`settings.validations`, and they are evaluated in parallel, joined with an AND
operation, which is short-circuited.

We can now deploy the policy, and try to deploy a Pod with unsigned images:

```console
$ kubectl apply -f - <<EOF
apiVersion: v1
kind: Pod
metadata:
  name: golanci-lint-example
spec:
  containers:
  - name: nginx
    image: ghcr.io/opencontainers/golangci-lint:v1.52.1
    ports:
    - containerPort: 80
EOF
Error from server: error when creating "STDIN":
  admission webhook "clusterwide-cel-sigstore-keyless-verification.kubewarden.admission" denied the request:
  failed to verify image: Callback evaluation failure: no signatures found for image: ghcr.io/opencontainers/golangci-lint:v1.52.1
```
