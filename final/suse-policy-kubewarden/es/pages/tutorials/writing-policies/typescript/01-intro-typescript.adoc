Como se indica en el https://www.typescriptlang.org/[sitio web oficial]

[quote]
TypeScript amplía JavaScript añadiendo tipos.

Al entender JavaScript, TypeScript le ahorra tiempo detectando errores y proporcionando correcciones antes de ejecutar el código.

{project-name} utiliza https://github.com/bytecodealliance/javy[Javy] (un proyecto de Bytecode Alliance) para construir binarios WebAssembly a partir de JavaScript y TypeScript.

[quote]
Javy toma tu código JavaScript y lo ejecuta en un contexto WebAssembly.

Incorpora un motor QuickJS compilado en WebAssembly que puede ejecutar JavaScript.

El proyecto proporciona tanto una CLI como un conjunto de API para incrustar y personalizar el comportamiento al ejecutar JavaScript en WebAssembly.

El proyecto {short-project-name} utiliza actualmente Javy por estas razones:

* Motor JavaScript maduro (QuickJS) compilado en WebAssembly.
* Compatibilidad con xref:/tutorials/writing-policies/wasi/01-intro-wasi.adoc[la interfaz WASI] mediante funciones de host personalizadas.
* Menor tamaño de los binarios en comparación con otras soluciones JavaScript-to-WebAssembly.
* Desarrollo y mantenimiento activos por la Bytecode Alliance.

== Limitaciones Javy

Javy ejecuta JavaScript en un entorno sandboxed WebAssembly con ciertas restricciones:

* *Sólo entorno WASI*: Acceso limitado a stdin/stdout/stderr y a las capacidades del host proporcionadas explícitamente.
* *Sin API Node.js*: Los módulos estándar de Node.js como `fs`, `http`, o `crypto` no están disponibles.
* *Biblioteca estándar limitada*: Sólo son accesibles las funciones básicas de JavaScript y las API habilitadas explícitamente.
* *Ejecución monohilo*: No soporta Web Workers ni multi-threading.

A pesar de estas limitaciones, Javy proporciona suficientes capacidades para escribir políticas efectivas de validación de {short-project-name} a través del sistema de capacidades de hosts.

[WARNING]
====
Escribir en STDOUT rompe las políticas - utilice STDERR para el registro en su lugar.
====

== Herramientas

Redactar las políticas de {short-project-name} requiere:

* *Node.js*: Tiempo de ejecución de JavaScript.
* *npm*: Para la gestión de dependencias.
* *TypeScript*: Recomendado para la seguridad del tipo (opcional).

[WARNING]
====
Asegúrese de que está utilizando Node.js 18 o superior. Las versiones anteriores pueden no ser compatibles con la cadena de herramientas de compilación.
====

Estas bibliotecas TypeScript/JavaScript son útiles cuando se escribe una política {short-project-name}:

* https://github.com/kubewarden/policy-sdk-js[{short-project-name} JavaScript SDK]: Proporciona estructuras y funciones que reducen la cantidad de código necesario. También proporciona ayudantes de prueba y acceso a todas las capacidades del host.
* https://github.com/silverlyra/kubernetes-types[Tipos TypeScript de Kubernetes]: Proporciona definiciones de TypeScript para todos los recursos de Kubernetes, lo que permite el desarrollo de políticas a prueba de tipos.

El proyecto {short-project-name} proporciona un https://github.com/kubewarden/js-policy-template[proyecto de política JavaScript/TypeScript de plantilla] que puede utilizar para crear políticas {short-project-name}.

== Obtener la cadena de herramientas

La forma más sencilla de obtener la cadena de herramientas es utilizando el SDK JavaScript de {short-project-name}, que incluye el plug-in de compilación Javy:

[source,console]
----
npm install kubewarden-policy-sdk
----

El binario del plug-in Javy se incluye automáticamente y puedes encontrarlo en:

[source,console]
----
node_modules/kubewarden-policy-sdk/plugin/javy-plugin-kubewarden.wasm
----

== Requisitos previos de la tutoría

Durante este tutorial necesitarás estas herramientas en tu máquina de desarrollo:

* *Node.js*: Versión 18 o superior con npm para la gestión de dependencias.
* https://github.com/bats-core/bats-core[`bats`]: Se utiliza para escribir las pruebas y automatizar su ejecución.
* https://github.com/kubewarden/kwctl/releases[`kwctl` ≥ v1.30]: Herramienta CLI proporcionada por {short-project-name} para ejecutar sus políticas fuera de Kubernetes, entre otras acciones. Se trata en xref:/tutorials/testing-policies/index.adoc[la sección de políticas de pruebas] de la documentación.
* https://github.com/bytecodealliance/javy[`javy` ≥ 6.0.0]: Herramienta CLI para compilar código JavaScript en módulos WebAssembly.
