Puede ajustar el comportamiento push-pull de los binarios `kwctl` y `policy-server` con el argumento `--sources-path`, para especificar la ruta al archivo`sources.yaml`.

Para configurar un CR PolicyServer, establezca sus campos `spec.insecureSources` y`spec.sourceAuthorities`. El formato de estos campos se corresponde con las secciones respectivas que figuran a continuación.

== El archivo `sources.yaml` 

Si omite el argumento `--sources-path` del comando `kwctl`, éste intentará cargar el archivo `sources.yaml` desde estas carpetas:

* Linux: `$HOME/.config/kubewarden/sources.yaml`
* Mac: `$HOME/Library/Application Support/io.kubewarden.kubewarden/sources.yaml`
* Ventanas: `$HOME\AppData\Roaming\kubewarden\config\sources.yaml`

Su estructura es la siguiente:

[subs="+attributes",yaml]
----
insecure_sources:
  - "registry-dev.example.com"
  - "registry-dev2.example.com:5500"
source_authorities:
  "registry-pre.example.com":
    - type: Path
      path: /opt/example.com/pki/ca-pre1-1.pem
    - type: Path
      path: /opt/example.com/pki/ca-pre1-2.der
  "registry-pre2.example.com:5500":
    - type: Data
      data: |
            -----BEGIN CERTIFICATE-----
            ca-pre2 PEM cert
            -----END CERTIFICATE-----
----

Este archivo está en formato YAML o JSON. Todas las claves son opcionales, por lo que el siguiente es un archivo `sources.yaml` válido:

[subs="+attributes",yaml]
----
insecure_sources: ["dev.registry.example.com"]
----

Tal cual:

[subs="+attributes",json]
----
{
    "source_authorities": {
        "host.k3d.internal:5000": [
            {"type": "Data","data":"pem cert 1"},
            {"type": "Data","data":"pem cert 2"}
        ]
    }
}
----

=== Sección de fuentes inseguras

Los hosts de la sección `insecure_sources` se comportan de forma diferente a los que no aparecen en la lista.

* Los hosts no aparecen, inténtalo:
 ** conectarse mediante HTTPS, verificando la identidad del servidor

+
Si la conexión falla, la operación se detiene.* Hosts listados en `insecure_sources`, inténtelo en orden: para conectarse ** usando HTTPS verificando la identidad del servidor ** para conectarse usando HTTPS, saltándose la verificación del host ** para conectarse usando HTTP.

+
La operación se detiene si fallan todas.

[NOTE]
====

Por lo general, está bien utilizar `insecure_sources` cuando se utilizan registros locales o servidores HTTP para el desarrollo. Evita la carga de gestionar certificados.
Claramente, esto no es para uso de producción.

====


=== Sección de autoridades de origen

La sección `source_authorities` contiene URIs y certificados CA. Forma una cadena de certificados para ese URI. Se utiliza para verificar la identidad de los registros de la Iniciativa OpenContainer (OCI) y los servidores HTTPS.

Estos certificados se codifican en los formatos Privacy Enhanced Mail (PEM) o Distinguished Encoding Rule (DER). Para especificar certificados en formato DER se utiliza una ruta a un archivo que contenga el certificado. En el formato PEM se especifica una ruta al archivo del certificado o una cadena con el certificado real. Especifique ambos con una tecla `type`:

[subs="+attributes",yaml]
----
source_authorities:
  "registry-pre.example.com":
    - type: Path
      path: /opt/example.com/pki/ca-pre1-1.pem
    - type: Path
      path: /opt/example.com/pki/ca-pre1-2.der
    - type: Data
      data: |
            -----BEGIN CERTIFICATE-----
            A string with the ca-pre1-3 PEM cert
            -----END CERTIFICATE-----
  "registry-pre2.example.com:5500":
    - type: Path
      path: /opt/example.com/pki/ca-pre2-1.der
----

=== Sección de proxies

La sección `proxies` contiene la configuración del proxy análoga a las variables de entorno habituales `HTTP_PROXY`, `HTTPS_PROXY`, `NO_PROXY` (y sus equivalentes en minúsculas).

Esta configuración tiene prioridad sobre la configuración de esas variables de entorno.

Consulta su xref:/howtos/proxy-configuration.adoc[página de instrucciones] para obtener más información sobre esta función.

Su formato es:

[,yaml]
----
proxies:
  http_proxy: "http://proxy.corp:3128"
  https_proxy: "http://proxy.corp:3129"
  no_proxy: "localhost,.corp"
----
