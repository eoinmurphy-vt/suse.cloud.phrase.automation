== Ayuda en línea de comandos para `policy-server`

Este documento contiene la ayuda del programa de línea de comandos `policy-server`.

*Visión general del mando:*

* link:#policy-server[`policy-server`]
* link:#policy-server-docs[`policy-server docs`]

=== `policy-server`

*Uso:* `policy-server ++[+OPTIONS+]++ ++[+COMMAND+]++`

==== *Subcomandos:*

* `docs` - Genera la documentación markdown para los comandos del servidor de políticas

==== *Opciones:*

* `--addr ++<+BIND_ADDRESS+>++` - Vincular con DIRECCIÓN
+
Valor por defecto: `0.0.0.0`
* `--always-accept-admission-reviews-on-namespace ++<+NAMESPACE+>++` - Aceptar siempre AdmissionReviews que se dirijan al espacio de nombres dado
* `--cert-file ++<+CERT_FILE+>++` - Ruta a un archivo de certificado X.509 para HTTPS
* `--client-ca-file ++<+CLIENT_CA_FILE+>++` - Ruta a un archivo de certificado CA que emitió el certificado de cliente. Necesario para activar mTLS
* `--daemon` - Si se establece, ejecuta policy-server en modo independiente como demonio
* `--daemon-pid-file ++<+DAEMON-PID-FILE+>++` - Ruta al archivo PID, utilizado sólo cuando se ejecuta en modo demonio.
+
Valor por defecto: `policy-server.pid`
* `--daemon-stderr-file ++<+DAEMON-STDERR-FILE+>++` - Ruta al archivo que contiene stderr, sólo se utiliza cuando se ejecuta en modo demonio
* `--daemon-stdout-file ++<+DAEMON-STDOUT-FILE+>++` - Ruta al archivo que contiene stdout, utilizado sólo cuando se ejecuta en modo demonio
* `--disable-timeout-protection` - Desactivar la protección de tiempo de espera de la política
* `--docker-config-json-path ++<+DOCKER_CONFIG+>++` - Ruta a una ruta tipo config.json de Docker. Puede utilizarse para indicar los datos de autenticación del registro
* `--enable-metrics` - Activar métricas
* `--enable-pprof` - Activar la creación de perfiles pprof
* `--ignore-kubernetes-connection-failure` - No salir con un error si la conexión Kubernetes falla. Esto hará que las políticas conscientes del contexto se rompan cuando no haya conexión con Kubernetes.
* `--key-file ++<+KEY_FILE+>++` - Ruta a un archivo de clave privada X.509 para HTTPS
* `--log-fmt ++<+LOG_FMT+>++` - Formato de salida del registro
+
Valor por defecto: `text`
+
Valores posibles: `text`, `json`, `otlp`
* `--log-level ++<+LOG_LEVEL+>++` - Nivel de registro
+
Valor por defecto: `info`
+
Valores posibles: `trace`, `debug`, `info`, `warn`, `error`
* `--log-no-color` - Desactivar la salida de color para los registros
* `--policies ++<+POLICIES_FILE+>++` - Fichero YAML que contiene las políticas a cargar y su configuración
+
Valor por defecto: `policies.yml`
* `--policies-download-dir ++<+POLICIES_DOWNLOAD_DIR+>++` - Ruta de descarga de las políticas
+
Valor por defecto: `.`
* `--policy-timeout ++<+MAXIMUM_EXECUTION_TIME_SECONDS+>++`
- Interrumpir la evaluación de la política después del tiempo dado
+
Valor por defecto: `2`
* `--port ++<+PORT+>++` - Escuchar en PORT
+
Valor por defecto: `3000`
* `--readiness-probe-port ++<+READINESS_PROBE_PORT+>++` —
Expose readiness endpoint on READINESS+_PROBE_+PORT
+
Valor por defecto: `8081`
* `--sigstore-cache-dir ++<+SIGSTORE_CACHE_DIR+>++` - Directorio utilizado para almacenar en caché los datos de sigstore
+
Valor por defecto: `sigstore-data`
* `--sources-path ++<+SOURCES_PATH+>++` - Archivo YAML con información de origen (https, registro de hosts inseguros, CA personalizadas\...)
* `--verification-path ++<+VERIFICATION_CONFIG_PATH+>++` - Archivo YAML con información de verificación (URI, claves, anotaciones\...)
* `--workers ++<+WORKERS_NUMBER+>++` - Número de subprocesos de trabajo a crear

=== `policy-server docs`

Genera la documentación markdown para los comandos del servidor de políticas

*Uso:* `policy-server docs --output ++<+FILE+>++`

==== *Opciones:*

* `-o`, `--output ++<+FILE+>++` - ruta donde se almacenará el archivo de documentación
