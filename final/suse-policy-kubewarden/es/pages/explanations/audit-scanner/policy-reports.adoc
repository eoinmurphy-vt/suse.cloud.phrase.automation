= Escáner de auditoría - Informes sobre políticas
:revdate: 2026-03-18

Cuando se utiliza {project-name} Audit Scanner, los resultados de los análisis de las políticas se almacenan en https://openreports.io/[OpenReports] Custom Resources.

[CAUTION]
====

Con {project-name} 1.33, la función Audit Scanner guarda por defecto los informes en CRDs de OpenReports desde [openreports.io](https://openreports.io).

Los CRDslink:https://htmlpreview.github.io/?https://github.com/kubernetes-sigs/wg-policy-prototypes/blob/045372e558b896695b2daae92e8c7a04d4d40282/policy-report/docs/index.html[PolicyReport]del grupo de trabajo K8s Policy no están instalados por defecto, están marcados como obsoletos y serán eliminados en una futura versión.

Los usuarios que deseen seguir utilizando los PolicyReports obsoletos, es decir, el comportamiento anterior a la versión 1.33, deben:- Cambiar el valor del gráfico Helm `kubewarden-crds` de `.Values.install.installPolicyReportCRDs` a `true`. - Cambiar el valor del gráfico Helm `kubewarden-controller` de `.Values.auditScanner.reportCRDsKind` de `openreports` a `policyreport`.

====


Estos CRD de OpenReport ofrecen una forma estructurada de almacenar y gestionar los resultados de las auditorías.

Los resultados del análisis de la política del escáner de auditoría se almacenan en `Report` y `ClusterReport`.
El escáner de auditoría crea un `Report` o un `ClusterReport` para cada recurso escaneado, dependiendo del ámbito del recurso. Los objetos `Report` están disponibles en el espacio de nombres del recurso, mientras que los objetos `ClusterReport` están disponibles en el ámbito del clúster.

Los resultados de auditoría generados por el escáner incluyen:

* la política evaluada
* el recurso escaneado
* el resultado de la evaluación (aprobado, suspenso u omitido)
* una marca de tiempo que indica cuándo tuvo lugar la evaluación.

También puede definir anotaciones de gravedad y categoría para sus políticas.

Los operadores pueden consultar los informes mediante `kubectl`. También pueden utilizar la interfaz de usuario opcional proporcionada por el proyecto de código abiertohttps://kyverno.github.io/policy-reporter[policy-reporter] para supervisar y observar los CRD de OpenReports.

== Consulta de los informes

Usando la CLI kubectl, es posible consultar los resultados del escaneo:

Lista los informes en el espacio de nombres por defecto:

[source,console,subs="+attributes"]
----
$ kubectl get report -o wide
NAME                                   KIND   NAME            PASS   FAIL   WARN   ERROR   SKIP   AGE
44da974b-6057-4745-a92c-9dd8a1b01cc4   Pod    nginx-secrets   6      1      0      0       0      33m
----

Enumera los informes de todo el clúster:

[source,console,subs="+attributes"]
----
$ kubectl get clusterreport -o wide
NAME                                   KIND        NAME                 PASS   FAIL   WARN   ERROR   SKIP   AGE
6c66175f-e15f-47e8-8c33-2c63d5ecde3e   Namespace   namespace1           3      1     0       0       0      1h
d0ffbf4f-7348-4350-af7b-fdbfc079cc49   Namespace   namespace2           0      4     0       0       0      1h
e48d9bbc-a9c4-4c36-8519-bc7390d9ec64   Namespace   namespace3           4      0     0       0       0      15h
----

Obtener los detalles de un Informe específico:

[source,console,subs="+attributes"]
----
kubectl get report 44da974b-6057-4745-a92c-9dd8a1b01cc4 -o yaml
----

Obtener los detalles de un ClusterReport específico:

[source,console,subs="+attributes"]
----
kubectl get clusterreport e48d9bbc-a9c4-4c36-8519-bc7390d9ec64 -o yaml
----

== Interfaz de usuario de Policy Reporter

El Policy Reporter se distribuye como una subcarpeta de `kubewarden-controller`. Consulte la página xref:howtos/audit-scanner.adoc[Instalación del escáner de auditoría] para obtener más información.

La interfaz de usuario de Policy Reporter proporciona un panel de control que muestra todas las infracciones. Véase la captura de pantalla siguiente:

image::policy-reporter_dashboard.png[Ejemplo de cuadro de mandos de Policy Reporter]

{short-project-name} envía la interfaz de usuario de Policy Reporter con un filtro preconfigurado que sólo muestra las políticas de {short-project-name}, junto con información ampliada:

image::policy-reporter_policyreports.png[Policy Reporter PolicyReports ejemplo]

Otras características de Policy Reporter incluyen el reenvío de resultados a diferentes clientes (como Grafana Loki, Elasticsearch, aplicaciones de chat), puntos finales de métricas, etc. Si desea más información, consultehttps://kyverno.github.io/policy-reporter[la documentación de la comunidad].
