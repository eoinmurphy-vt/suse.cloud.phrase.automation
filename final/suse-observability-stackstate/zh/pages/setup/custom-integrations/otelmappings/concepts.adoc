= 开放式遥测映射概念
:revdate: 2025-12-23
:page-revdate: {revdate}
:description: 开放式遥测映射概念

== 组件映射

`OtelComponentMapping` 描述了如何根据 OpenTelemetry 数据创建（或更新）拓扑组件。

由 OpenTelemetry 创建的组件的典型示例包括（这些都是默认 OpenTelemetry StackPack 的一部分）：

* 服务（源自`service.name` /`service.namespace`)
* 服务实例（源自`service.instance.id`)
* 数据库
* 主机和 Kubernetes 资源（节点、命名空间和 pod）
* 无服务器或批处理环境中的功能和任务

== 关系映射

`OtelRelationMapping` 描述了如何从 OpenTelemetry 数据中创建两个组件之间的关系。

关系的常见示例包括（这些是默认 OpenTelemetry 堆栈包的一部分）：

* 同步服务调用（客户端-> 服务器）
* 异步信息流（生产者-> 消费者、消费者-> 同行、生产者-> 同行）
* 服务到数据库的连接
* 服务提供的服务实例
* 执行关系（服务或功能执行任务、主机或功能）

== 表达式和通用表达式语言 (CEL)

拓扑映射依靠https://cel.dev/[通用表达式语言 (CEL)]来选择遥测数据、提取值以及构建组件和关系字段。

大多数表达式都是简单的布尔检查、属性查找或字符串结构。可以根据需要逐步引入更高级的表达式。

有关如何在拓扑映射中利用 CEL 的更多信息，请参阅xref:/setup/custom-integrations/otelmappings/schemas-ref.adoc[模式]页面。
