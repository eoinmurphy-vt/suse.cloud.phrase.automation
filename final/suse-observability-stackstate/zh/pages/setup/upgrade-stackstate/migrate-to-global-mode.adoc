= 迁移到全局配置模式
:description: {stackstate-product-name} 自托管
:page-toclevels: 3

== 概述

本指南介绍如何从使用`suse-observability-values` 图表的传统两步安装迁移到使用`global.suseObservability` 配置的简化单图表安装。

[NOTE]
====
`global.suseObservability` 配置方法从`2.8.0` 版开始提供，现在是推荐的安装方法。传统的`suse-observability-values` 图表方法已被淘汰，但在现有安装中仍可继续使用。
====

=== 旧版安装（已废弃）

旧的安装方法需要两个步骤：

. 使用`suse-observability-values` 图表生成数值文件
. 使用生成的数值安装`suse-observability` 图表

[,text]
----
# Step 1: Generate values
helm template suse-observability-values suse-observability/suse-observability-values \
  --set license="YOUR-LICENSE-KEY" \
  --set baseUrl="https://observability.example.com" \
  --set sizing.profile="150-ha" \
  --set adminPassword="your-password" \
  --set pullSecret.username="registry-user" \
  --set pullSecret.password="registry-pass" \
  > generated-values.yaml

# Step 2: Install with generated values
helm install suse-observability suse-observability/suse-observability \
  -f generated-values.yaml \
  -n suse-observability
----

=== 新安装（推荐）

新方法使用单个值文件，`global.suseObservability` 配置：

[,text]
----
helm install suse-observability suse-observability/suse-observability \
  -f values.yaml \
  -n suse-observability
----

== 安装前进行转换

如果您已为传统`suse-observability-values` 图表准备好参数，但尚未安装，则可以直接转换为新方法，而无需创建中间值文件。

=== 直接参数转换

通过映射参数，将`helm template` 命令转换为`helm upgrade --install` 命令：

[cols="1,1"]
|===
| 旧参数 | 新参数

| `--set license="..."`
| `--set global.suseObservability.license="..."`

| `--set baseUrl="..."`
| `--set global.suseObservability.baseUrl="..."`

| `--set sizing.profile="..."`
| `--set global.suseObservability.sizing.profile="..."`

| `--set receiverApiKey="..."`
| `--set global.suseObservability.receiverApiKey="..."`

| `--set adminPassword="..."`
| `--set global.suseObservability.adminPassword="..."`

| `--set imageRegistry="..."`
| `--set global.imageRegistry="..."`

| `--set pullSecret.username="..."`
| `--set global.suseObservability.pullSecret.username="..."`

| `--set pullSecret.password="..."`
| `--set global.suseObservability.pullSecret.password="..."`
|===

=== 例如前后对比

[NOTE]
====
在运行 helm 命令之前，请确保您拥有最新的图表版本：

[,bash]
----
helm repo update
----
====

.传统命令（请勿使用）
[,bash]
----
export VALUES_DIR=.
helm template \
  --set license="YOUR-LICENSE-KEY" \
  --set baseUrl="https://observability.example.com" \
  --set sizing.profile="10-nonha" \
  --set receiverApiKey="YOUR-API-KEY" \
  suse-observability-values \
  suse-observability/suse-observability-values --output-dir $VALUES_DIR

# Then: helm install with generated files...
----

.新命令（推荐）
[,bash]
----
helm upgrade --install \
  --namespace suse-observability \
  --create-namespace \
  --set global.suseObservability.license="YOUR-LICENSE-KEY" \
  --set global.suseObservability.baseUrl="https://observability.example.com" \
  --set global.suseObservability.sizing.profile="10-nonha" \
  --set global.suseObservability.receiverApiKey="YOUR-API-KEY" \
  --set global.suseObservability.adminPassword='your-password' \
  suse-observability suse-observability/suse-observability
----

[NOTE]
====
* `adminPassword` 必须是新方法中的 bcrypt 哈希值。如何生成一个，请参见<<bcrypt-hash>> 。
* 对于气隙环境或自定义注册表，请参阅xref:/k8s-suse-rancher-prime-air-gapped.adoc[在气隙模式下安装{stackstate-product-name} ]，了解其他配置，包括`imageRegistry` 和`pullSecret` 设置。
====

=== 使用数值文件

如果您更喜欢使用值文件，请使用<<_step_3_create_new_values_file>> 中的配置创建`values.yaml` ，确保有最新的图表`helm repo update` ，然后运行：

[,bash]
----
helm upgrade --install \
  --namespace suse-observability \
  --create-namespace \
  --values values.yaml \
  suse-observability suse-observability/suse-observability
----

== 迁移之前

在执行从传统安装方法迁移时，请注意以下几点：

[CAUTION]
====
本迁移指南适用于目前使用`suse-observability-values` 图表方法的用户。如果是从 StackState 6.0 升级，请遵循xref:/setup/upgrade-stackstate/migrate-from-6.adoc[从 StackState 迁移指南]。
====

[NOTE]
====
* 迁移使用标准的 Helm 升级命令进行
* 您现有的数据和配置将被保留
* 迁移可在维护窗口期间进行
* 可以使用标准的 Helm 回滚程序进行回滚
====

== 迁移步骤

=== 步骤 1：备份当前配置

在开始迁移之前，请保存现有的 Helm 值和任何自定义设置：

[,text]
----
# Save current Helm values
helm get values suse-observability -n suse-observability > backup-values.yaml

# Save your original suse-observability-values input (if available)
# This is typically your values file used with helm template

# Backup important resources
kubectl get secrets -n suse-observability -o yaml > backup-secrets.yaml
----

=== 步骤 2：确定当前配置

查看现有配置，确定以下设置：

* *尺寸概况*：在 suse-observability-values 输入中检查`sizing.profile` 
* *许可证密钥*：您的{stackstate-product-name} 许可证
* *基本 URL*：可访问{stackstate-product-name} 的 URL
* *管理员密码*：管理员密码（也可以使用密码的 bcrypt 哈希值，其值为`adminPasswordBcrypt`)
* *调出秘密凭证*注册表用户名和密码（如果使用）
* *自定义亲和设置*：任何节点亲和性或 pod 反亲和性定制

您可以从当前安装中提取其中一些：

[,text]
----
# Check current values
helm get values suse-observability -n suse-observability

# The sizing profile is visible in resource configurations
# Look for patterns like replica counts, resource limits, etc.
----

=== 了解自动生成值与自定义值

运行`helm get values` 时，可能会看到数百行配置。不过，其中大部分都是由大小配置文件*自动生成的*，*无需*在迁移值文件中手动保留。

==== 尺寸剖面自动提供的功能

设置`global.suseObservability.sizing.profile` 时，图表会自动配置：

* 所有组件（Elasticsearch、Kafka、ClickHouse、服务器、接收器等）的资源请求和限制
* 有状态服务的副本数量
* 持久卷的存储容量
* HBase 部署模式（Mono 适用于非 HA，Distributed 适用于 HA）
* 服务器和接收器分割配置
* 维多利亚度量衡 HA 设置
* 环境变量 (`extraEnv`) 包括用于性能调整的`CONFIG_FORCE_*` 设置

==== 您需要保存的东西

只保留*真正自定义的*值\--您添加或修改的配置超出了尺寸配置文件提供的范围：

[cols="1,1"]
|===
| 保存（自定义） | 不保存（自动生成）

| 核心设置（许可证、baseUrl、adminPassword、receiverApiKey）
| 资源限制和请求

| 入口配置（主机、TLS、注释）
| 复制品数量

| 您添加的自定义环境变量
| 存储尺寸

| 身份验证设置（LDAP、OIDC）
| `CONFIG_FORCE_*` 环境变量

| 定制 `allowedOrigins`
| Kafka/Zookeeper 复制因子

| OpenTelemetry 采集器自定义入口
| HBase 部署模式
|===

==== 例如最小迁移值文件

迁移值文件应该比`helm get values` 的输出*小得多*：

[,yaml]
----
global:
  suseObservability:
    license: "YOUR-LICENSE"
    baseUrl: "https://your-url.example.com"
    sizing:
      profile: "10-nonha"  # This handles all resource configuration!
    adminPassword: "your-password"
    receiverApiKey: "your-api-key"

# Only include truly custom configurations below
ingress:
  enabled: true
  ingressClassName: your-ingress-class
  annotations:
    # your custom annotations
  hosts:
  - host: your-url.example.com
  tls:
  - hosts:
    - your-url.example.com
    secretName: your-tls-secret
----

[TIP]
====
如果不确定某个值是自定义的还是自动生成的，请将其与尺寸配置文件默认值进行比较。`elasticsearch.resources` 、`kafka.resources` 、`stackstate.components.server.resources` 和`CONFIG_FORCE_*` 等环境变量值均由配置文件生成，可以放心省略。
====

=== 步骤 3：创建新的数值文件

使用`global.suseObservability` 结构创建一个新的`values.yaml` ：

[,yaml]
----
global:
  suseObservability:
    # Required: Your license key
    license: "YOUR-LICENSE-KEY"

    # Required: Base URL for {stackstate-product-name}
    baseUrl: "https://observability.example.com"

    # Required: Sizing profile
    # Available: trial, 10-nonha, 20-nonha, 50-nonha, 100-nonha,
    #            150-ha, 250-ha, 500-ha, 4000-ha
    sizing:
      profile: "150-ha"

    adminPassword: "your-password"
    # Instead of adminPassword you can provide a bcrypt hashed password
    # Generate with: htpasswd -bnBC 10 "" "your-password" | tr -d ':\n'
    # adminPasswordBcrypt: "$2a$10$..."

    # Optional: Receiver API key (auto-generated if not provided)
    receiverApiKey: "your-receiver-api-key"

    # Optional: Affinity configuration
    # These settings apply to all infrastructure components (Elasticsearch,
    # Kafka, ClickHouse, etc.) automatically - no need to configure per-component
    affinity:
      # Node affinity for all components
      nodeAffinity: null

      # Pod anti-affinity for HA profiles (infrastructure components)
      podAntiAffinity:
        requiredDuringSchedulingIgnoredDuringExecution: true
        topologyKey: "kubernetes.io/hostname"

  # For air-gapped or custom registry configurations, see:
  # xref:/k8s-suse-rancher-prime-air-gapped.adoc[Air-Gapped Installation Guide]
----

=== 步骤 5：处理自定义重载

如果在生成的值文件中有自定义重载，则仍可使用。新的全局模式提供了合理的默认设置，可根据每个组件进行重写：

[,yaml]
----
global:
  suseObservability:
    sizing:
      profile: "150-ha"
    license: "YOUR-LICENSE-KEY"
    baseUrl: "https://observability.example.com"
    adminPassword: "your-password"
    # Global affinity - applied to all components by default
    affinity:
      nodeAffinity:
        requiredDuringSchedulingIgnoredDuringExecution:
          nodeSelectorTerms:
          - matchExpressions:
            - key: topology.kubernetes.io/zone
              operator: In
              values:
              - us-west-2a

# Custom per-component overrides still work - they take precedence over global defaults
stackstate:
  components:
    api:
      resources:
        requests:
          memory: 16Gi  # Override default from sizing profile
        limits:
          memory: 20Gi

elasticsearch:
  volumeClaimTemplate:
    resources:
      requests:
        storage: 500Gi  # Override default storage size
  # Override affinity for Elasticsearch specifically
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
        - matchExpressions:
          - key: node-type
            operator: In
            values:
            - storage-optimized
----

=== 步骤 6：执行升级

首先，确保您拥有最新的图表版本：

[,text]
----
helm repo update
----

然后执行升级：

[,text]
----
# Dry-run first to see what will change
helm upgrade suse-observability suse-observability/suse-observability \
  -n suse-observability \
  -f values.yaml \
  --dry-run

# If everything looks good, perform the actual upgrade
helm upgrade suse-observability suse-observability/suse-observability \
  -n suse-observability \
  -f values.yaml
----

=== 步骤 7：验证升级

[,text]
----
# Check all pods are running
kubectl get pods -n suse-observability

# Check for any errors in events
kubectl get events -n suse-observability --sort-by='.lastTimestamp' | tail -20

# Verify the Helm release
helm status suse-observability -n suse-observability

# Check the applied values
helm get values suse-observability -n suse-observability
----

[[bcrypt-hash]]
== 管理员密码

对于传统方法，`adminPassword` 是以纯文本形式提供给 suse-observability-values 图表的，该图表会将其散列为 bcrypt 哈希值。在新方法中，您可以向 suse-observability 图表提供纯文本`adminPassword` ，也可以使用`adminPasswordBcrypt` 直接向 suse-observability 图表提供 bcrypt 哈希值。

如果需要生成新的 bcrypt 哈希值：

[,bash]
----
htpasswd -bnBC 10 "" "your-password" | tr -d ':\n'
----

== 故障排除

=== 豆荚卡在待处理状态

这通常表明反亲和性规则存在调度问题。对于节点有限的集群，可使用软反亲和性：

[,yaml]
----
global:
  suseObservability:
    affinity:
      podAntiAffinity:
        requiredDuringSchedulingIgnoredDuringExecution: false  # Use soft anti-affinity
----

=== 资源与先前安装的不同

规模配置文件可能会更新资源建议。要保留以前的设置，请添加显式覆盖：

[,yaml]
----
global:
  suseObservability:
    sizing:
      profile: "150-ha"

# Override with your previous resource values
stackstate:
  components:
    api:
      resources:
        requests:
          memory: 8Gi  # Your previous value
----

=== 配置风格冲突

新旧配置风格不要混用。使用其中一种：

* `global.suseObservability.*` (新式），或
* `stackstate.license.key`,`stackstate.baseUrl` 等（旧式）

[,text]
----
# Check for conflicting values
helm get values suse-observability -n suse-observability | grep -E "license|baseUrl"
----

=== 必要时回滚

[,text]
----
# List release history
helm history suse-observability -n suse-observability

# Rollback to previous revision
helm rollback suse-observability <revision-number> -n suse-observability
----

== 主要区别

[cols="1,1,1"]
|===
| 方面 | 遗产（苏斯观测值） | 新建（global.suseObservability）

| 安装步骤
| 2 （模板 + 安装）
| 1 (安装)

| 值文件管理
| 已生成，必须存储
| 单一真相来源

| 简介更新
| 需要手动再生
| 图表自动升级

| 拉动秘密
| 单独的分图表
| 内置，自动配置

| 维多利亚度量衡 HA
| 手册 `enabled: true/false`
| 按配置文件自动配置

| 亲和配置
| 按组件生成（每个服务单独设置）
| 集中式全局设置（如有需要，仍可按组件进行重写）
|===

[NOTE]
====
全局亲和配置为所有基础架构组件提供默认设置。必要时，您仍然可以通过设置组件级亲和值来覆盖特定组件（如 Elasticsearch 或 Kafka）的亲和性。组件级设置优先于全局默认设置。
====

== 清理

成功迁移后，您可以

. 删除生成的值文件（不再需要它们）
. 更新部署脚本以使用新的单步安装程序
. 删除 CI/CD 管道中对`suse-observability-values` 图表的任何引用
