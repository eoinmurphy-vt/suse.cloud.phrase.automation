= 分机故障排除
:revdate: 2025-11-03
:page-revdate: {revdate}
:description: SUSE Observability 自助托管

本节提供了{stackstate-product-name} 和 Rancher UI 扩展之间任何互操作性问题的故障排除步骤。

== 收集 HAR（Http ARchive）

Http ARchive 会收集浏览器向 Rancher 和{stackstate-product-name} 发送的请求的详细信息。 它还记录收到的答复。
此存档针对出现错误的页面/浏览器选项卡。 它不需要访问后备 Rancher 或{stackstate-product-name} 实例（尽管它们的日志在稍后阶段可能有用）。

* 打开 "开发工具 "面板中的*"网络*"选项卡。 您可以使用`F12` 进行访问。 网络选项卡显示所有请求的日志。
* 清除日志并（硬）刷新显示有问题行为的页面。
* 现在应该可以看到大量向 Rancher 和 SUSE Observability 提出的请求。
* 导出/下载/保存 HAR - 不同的浏览器在此处使用不同的名称。

== 群组无观测

要在 Rancher 中显示集群健康状况，这些配置之间必须保持一致：

* 集群在 Rancher UI 中的（显示）名称。
* 在群集上部署的代理上设置的`stackstate.cluster.name` 
* SUSE Observability 中 Kubernetes 堆栈包配置中的群集名称

不匹配往往会在 Rancher 的群集概览中显示为 "未收到数据"。

== 连接失败

由于以下原因，SUSE Observability 的配置有时可能无法在 Rancher UI 中生成有效连接：

* Rancher 主机未配置为 SUSE Observability 允许的来源。 您可以将其添加到 helm 值中的`stackstate.allowedOrigins` 列表中，或通过向部署命令传递`--set stackstate.allowedOrigins={"https://<rancher-url>"}` 来解决这个问题。
* 浏览器无法访问 SUSE Observability。例如，根证书尚未安装。
