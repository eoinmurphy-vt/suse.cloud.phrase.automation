= 将 SUSE Observability 暴露于群集之外
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: {stackstate-product-name} 自托管

== 概述

{stackstate-product-name} 可以通过 Kubernetes Ingress 资源公开。本页的示例展示了如何使用xref:/setup/install-stackstate/kubernetes_openshift/ingress.adoc#_configure_ingress_via_the_suse_observability_helm_chart[在 Kubernetes 上运行的 SUSE Observability Helm] 配置 Traefik 或 Nginx Ingress 控制器。本页还记录了在使用不同方法配置入口流量时应暴露的服务/端口组合。

在观察同时托管{stackstate-product-name} 的群集时，可通过在安装代理时xref:/setup/install-stackstate/kubernetes_openshift/ingress.adoc#_agents_in_the_same_cluster[更改代理配置]，将代理流量完全控制在群集内。

== 通过{stackstate-product-name} Helm 图表配置入口

{stackstate-product-name} 舵手图的数值中有一个`ingress` 部分。默认情况下禁用。下面的示例展示了如何使用 Helm 图表配置启用 TLS 加密的入口控制器。请注意，控制器本身和证书的设置超出了本文件的范围。

要为{stackstate-product-name} 配置入口，请创建内容如下的文件`ingress_values.yaml` 。将`MY_DOMAIN` 替换为您自己的域（与您的入口控制器相连），并为`tls-secret` 设置正确的名称。有关要设置的正确注释，请查阅入口控制器的文档。以下所有字段均为可选项，例如，如果不使用 TLS，则省略该部分，但请注意{stackstate-product-name} 也不会对流量进行加密。

[CAUTION]
====
请注意，使用 rancher UI 扩展需要设置 TLS。
====

[tabs]
====
Traefik::
+
--
.已知问题
[WARNING]
=====
请确保使用 Traefik< 3.6.3 or >= 3.6.7（Helm 图表版本 >= 39.0.0），因为 3.6.3 至 3.6.6 版本存在一个问题，会导致某些 API 端点出错。特别是，由于 API 调用被特拉菲克以`400 Bad Request` 错误阻止，组件高亮页面无法加载。
=====

[,text]
----
ingress:
  enabled: true
  ingressClassName: traefik
  annotations:
    traefik.ingress.kubernetes.io/router.entrypoints: websecure
  hosts:
    - host: suse-observability.MY_DOMAIN
  tls:
    - hosts:
        - suse-observability.MY_DOMAIN
      secretName: tls-secret
----
--

Nginx 入口::
+
--
[NOTE]
=====
Ingress Nginx 项目正在 https://kubernetes.io/blog/2025/11/11/ingress-nginx-retirement/[退役] 。建议用户考虑 Traefik 等替代品。
=====

[,text]
----
ingress:
  enabled: true
  ingressClassName: nginx
  annotations:
    nginx.ingress.kubernetes.io/proxy-body-size: "50m"
  hosts:
    - host: suse-observability.MY_DOMAIN
  tls:
    - hosts:
        - suse-observability.MY_DOMAIN
      secretName: tls-secret
----

该文件中最引人注目的是 Nginx 注释将允许的`proxy-body-size` 增加到`50m` （大于任何预期请求）。默认情况下，Nginx 允许的正文大小最大为`1m` 。{stackstate-product-name} 代理和其他数据提供者有时会发送更大的请求。因此，无论使用的是 Nginx 还是其他入口控制器，都应确保允许的主体大小足够大。
--
====

确保更新初始安装时生成的值文件中的`baseUrl` ，{stackstate-product-name} 将使用该值文件为代理生成便捷的安装说明。

示例使用`ingressClassName` 字段来指定https://kubernetes.io/docs/concepts/services-networking/ingress/#_ingress_class[入口]，而不是废弃的`kubernetes.io/ingress.class` 注释。如果集群已定义了默认入口类，则可以省略入口类名称字段。
示例使用`ingressClassName` 字段来指定https://kubernetes.io/docs/concepts/services-networking/ingress/#_ingress_class[入口]，而不是废弃的`kubernetes.io/ingress.class` 注释。如果集群已定义了默认入口类，则可以省略入口类名称字段。

运行`helm upgrade` 命令部署{stackstate-product-name} 时，请包含`ingress_values.yaml` 文件：

[,text]
----
helm upgrade --install \
  --namespace "suse-observability" \
  --values "ingress_values.yaml" \
  --values $VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/sizing_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/affinity_values.yaml \
suse-observability \
suse-observability/suse-observability
----

[NOTE]
====
此步骤假定已执行xref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_generate_baseconfig_values.yaml_and_sizing_values.yaml[生成`baseConfig_values.yaml` 和`sizing_values.yaml`]。

== 为开放式遥测配置输入规则

{stackstate-product-name} Helm 图表在其值中公开了一个`opentelemetry-collector` 服务，可在其中创建一个专用的`ingress` 。默认情况下禁用。`opentelemetry-collector` 所需的入口需要支持 GRPC 协议。请注意，控制器本身和证书的设置超出了本文件的范围。

要为{stackstate-product-name} 配置`opentelemetry-collector` 入口，请创建内容如下的文件`ingress_otel_values.yaml` 。将`MY_DOMAIN` 替换为您自己的域（与您的入口控制器相连），并为`otlp-tls-secret` 设置正确的名称。有关要设置的正确注释，请查阅入口控制器的文档。以下所有字段均为可选项，例如，如果不使用 TLS，则省略该部分，但请注意{stackstate-product-name} 也不会对流量进行加密。

.已知问题

[NOTE]
=====
从{stackstate-product-name} Helm 图表版本 v2.8.0 开始支持通过特拉菲克公开 OpenTelemetry Collector GRPC 端点。

请注意，GRPC 主机条目需要显式`serviceName: suse-observability-otel-collector-grpc` 才能将流量路由到正确的后端服务。
=====

[,text]
----
opentelemetry-collector:
  ingress:
    enabled: true
    ingressClassName: traefik
    annotations:
      traefik.ingress.kubernetes.io/router.entrypoints: websecure
    hosts:
      - host: otlp-suse-observability.MY_DOMAIN
        paths:
          - path: /
            pathType: Prefix
            serviceName: suse-observability-otel-collector-grpc
            port: 4317
    tls:
      - hosts:
          - otlp-suse-observability.MY_DOMAIN
        secretName: otlp-tls-secret
    additionalIngresses:
      - name: otlp-http
        ingressClassName: traefik
        annotations:
          traefik.ingress.kubernetes.io/router.entrypoints: websecure
        hosts:
          - host: otlp-http-suse-observability.MY_DOMAIN
            paths:
              - path: /
                pathType: Prefix
                port: 4318
        tls:
          - hosts:
              - otlp-http-suse-observability.MY_DOMAIN
            secretName: otlp-http-tls-secret
----

== 通过外部工具进行配置

要使{stackstate-product-name} 可在其安装的 Kubernetes 集群之外访问，只需将流量路由到`<namespace>-stackstate-k8s-router` 服务的`8080` 端口即可。{stackstate-product-name} 的用户界面可直接在该服务的根路径下访问（即`http://<namespace>-stackstate-k8s-router:8080` ），而代理将使用`/receiver` 路径 (`http://<namespace>-stackstate-k8s-router:8080/receiver`)。

确保更新初始安装时生成的值文件中的`baseUrl` ，{stackstate-product-name} 将使用该值文件为代理生成便捷的安装说明。

[NOTE]
====
在手动配置 Nginx 或类似 HTTP 服务器作为反向代理时，请确保它也能代理 websockets。对于 Nginx，可通过在`location` 指令中加入以下指令进行配置：

[,text]
----
proxy_set_header Upgrade                 $http_upgrade;
proxy_set_header Connection              "Upgrade";
----

====


[CAUTION]
====
{stackstate-product-name} 本身不使用 TLS 加密流量，TLS 加密预计将由入口控制器或外部负载平衡器处理。
====


== 同一群组中的代理

与{stackstate-product-name} 部署在同一集群上的代理当然可以使用{stackstate-product-name} 暴露的外部 URL，但也可以配置代理只通过 Kubernetes 内部网络直接连接到{stackstate-product-name} 实例。为此，请将xref:/k8s-quick-start-guide.adoc[代理 Kubernetes 安装]中`helm install` 命令中`'stackstate.url'` 的值替换为路由器服务的内部集群 URL（另见上文）： `http://<namespace>-suse-observability-router.<namespace>.svc.cluster.local:8080/receiver/stsAgent` (`<namespace>` 部分需要替换为{stackstate-product-name} 的命名空间）。


== 转发{stackstate-product-name} 路由器服务端口

默认情况下，{stackstate-product-name} Helm 图表会部署路由器 pod 和服务。该服务会暴露`8080` 端口，这是唯一需要通过 Ingress 暴露的入口点。
您也可以通过端口转发访问{stackstate-product-name} ；这样做时，必须允许将 localhost 作为请求源。

[WARNING]
====
* 在`stackstate.allowedOrigins` 中允许`localhost` 仅用于本地开发或在使用端口转发时进行调试。
* 这*不是*生产设置。通过 Ingress 公开{stackstate-product-name} 时，移除此允许的来源。
====

要在不配置入口的情况下访问用户界面，请转发路由器服务端口：

[,text]
----
kubectl port-forward service/<helm-release-name>-suse-observability-router 8080:8080 --namespace suse-observability
----

通过端口转发访问{stackstate-product-name} 时，浏览器将`http://localhost:8080` 作为请求源。
要允许来自该来源的请求，请将其添加到 Helm 值的`stackstate.allowedOrigins` 列表中，或直接将其传递给 Helm 命令：：

[,bash]
----
helm upgrade \
  --install \
  --namespace suse-observability \
  --values $VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/sizing_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/affinity_values.yaml \
  --set stackstate.allowedOrigins={"http://localhost:8080"} \
suse-observability \
suse-observability/suse-observability
----
