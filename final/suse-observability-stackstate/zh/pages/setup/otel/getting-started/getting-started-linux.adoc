= 开始使用开放式遥测技术
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE 可观察性

下面是我们要为需要监控的应用程序创建的设置：

* 在 Linux 主机上运行的受监控应用程序/工作负载
* 在同一 Linux 主机上运行的 Open Telemetry 采集器
* SUSE Observability 或 SUSE Cloud Observability

image::otel/open-telemetry-collector-linux.png[在 linux 主机上使用 Open Telemetry 收集器进行应用程序检测]

== 安装开放式遥测采集器

[NOTE]
====
对于生产设置，强烈建议安装收集器，因为它可以让您的服务快速卸载数据，而且收集器还可以处理重试、批处理、加密甚至敏感数据过滤等额外事务。
====


首先，我们要安装采集器。我们将其配置为

* 接收可能来自许多仪器应用的数据
* 用主机属性丰富收集到的数据
* 生成轨迹指标
* 将数据转发到 SUSE Observability，包括使用 API 密钥进行身份验证

此外，当出现连接问题时，它还会重试发送数据。

=== 创建服务令牌

创建服务令牌有两种方法：

* **SUSE Observability UI**- 点击屏幕左上角打开主菜单，进入`StackPacks` >`Open Telemetry` 。 如果之前没有这样做，请单击`INSTALL` 按钮。 单击`CREATE NEW SERVICE TOKEN` 按钮，将数值复制到剪贴板上。
* **SUSE Observability CLI**- 请参阅xref:/use/security/k8s-service-tokens.adoc#_manage_service_tokens[管理服务令牌]

在以下说明中提到`<SERVICE_TOKEN>` 时，必须使用服务令牌值。

=== 安装和配置采集器

收集器为大多数 Linux 版本和架构提供软件包（apk、deb 和 rpm），并使用`systemd` 进行自动服务配置。要安装它，请查找https://github.com/open-telemetry/opentelemetry-collector-releases/releases[在 Github 上]版本，并更新示例中的 URL 以使用最新版本：

有关其他安装选项，请使用https://opentelemetry.io/docs/collector/installation/#_linux[Open Telemetry 说明]。

安装后，通过编辑`/etc/otelcol-contrib/config.yaml` 修改采集器配置。更改文件，使其与此处的`config.yaml` 示例相似，将`<otlp-suse-observability-endpoint:port>` 替换为您的 OTLP 端点（请参阅xref:/setup/otel/otlp-apis.adoc[OTLP API]了解您的端点），并为`<SERVICE_TOKEN>` 插入您的服务令牌：

.config.yaml
[,yaml]
----
receivers:
  otlp:
    protocols:
      # Only bind to localhost to keep the collector secure, see https://github.com/open-telemetry/opentelemetry-collector/blob/main/docs/security-best-practices.md#safeguards-against-denial-of-service-attacks
      grpc:
        endpoint: 127.0.0.1:4317
      http:
        endpoint: 127.0.0.1:4318
  # Collect own metrics
  prometheus:
    config:
      scrape_configs:
      - job_name: 'otel-collector'
        scrape_interval: 10s
        static_configs:
        - targets: ['0.0.0.0:8888']
extensions:
  health_check: {}
  pprof:
    endpoint: 0.0.0.0:1777
  zpages:
    endpoint: 0.0.0.0:55679
  # Use the API key from the env for authentication
  bearertokenauth:
    scheme: SUSEObservability
    token: "<SERVICE_TOKEN>"
exporters:
  nop: {}
  debug: {}
  otlp/suse-observability:
    compression: snappy
    auth:
      authenticator: bearertokenauth
    # Put in your own otlp endpoint, for example otlp-suse-observability.my.company.com:443
    endpoint: <otlp-suse-observability-endpoint:port>
processors:
  memory_limiter:
    check_interval: 5s
    limit_percentage: 80
    spike_limit_percentage: 25
  batch: {}
  # Optionally include resource information from the system running the collector
  resourcedetection/system:
    detectors: [env, system] # Replace system with gcp, ec2, azure when running in cloud environments
    system:
      hostname_sources: ["os"]
connectors:
  # Generate metrics for spans
  spanmetrics:
    metrics_expiration: 5m
    namespace: otel_span
service:
  extensions: [ bearertokenauth, health_check, pprof, zpages ]
  pipelines:
    traces:
      receivers: [otlp]
      processors: [memory_limiter, resourcedetection/system, batch]
      exporters: [debug, spanmetrics, otlp/suse-observability]
    metrics:
      receivers: [otlp, spanmetrics, prometheus]
      processors: [memory_limiter, batch, resourcedetection/system]
      exporters: [debug, otlp/suse-observability]
    logs:
      receivers: [otlp]
      processors: []
      exporters: [nop]
----


最后重新启动采集器：

[,bash]
----
sudo systemctl restart otelcol-contrib
----

要查看采集器的日志，请使用

[,bash]
----
sudo journalctl -u otelcol-contrib
----

== 从应用程序中收集遥测数据

收集遥测数据的常用方法是使用开放式遥测 SDK 对应用程序进行检测。我们已经记录了一些语言的快速入门指南，但还有更多语言：

* xref:/setup/otel/instrumentation/java.adoc[Java]
* xref:/setup/otel/instrumentation/dot-net.adoc[.NET]
* xref:/setup/otel/instrumentation/node.js.adoc[Node.js]

SDK 无需额外配置，默认情况下通过 OTLP 或 OTLP over HTTP（取决于支持的协议）导出到本地主机。

其他语言请参考https://opentelemetry.io/docs/languages/[opentelemetry.io] 上的文档。

== 查看结果

进入 SUSE Observability，确保已安装 Open Telemetry Stackpack（通过主菜单 \-> Stackpacks）。

过了一会儿，如果您的应用程序正在处理一些流量，您就可以在 Open Telemetry \-> services and service instances overviews（打开遥测 \-> 服务和服务实例概览）中找到它的服务名称。跟踪将出现在服务和服务实例组件的xref:/use/views/k8s-traces-perspective.adoc[跟踪] xref:/use/traces/k8sTs-explore-traces.adoc[资源管理器]和xref:/use/views/k8s-traces-perspective.adoc[跟踪视角]中。跨度指标和特定语言指标（如有）将出现在组件的xref:/use/views/k8s-metrics-perspective.adoc[指标视角]中。
