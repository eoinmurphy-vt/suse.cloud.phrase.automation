= 应用程序接口密钥（已废弃）
:revdate: 2025-07-22
:page-revdate: {revdate}
:description: SUSE 可观察性

使用 API 密钥对{stackstate-product-name} 进行验证，以便向{stackstate-product-name} 发送遥测数据。在安装 Helm 代理时，这是通过设置`stackstate.apiKey` helm 变量来实现的。

API 密钥和输入 API 密钥已过时，将于 2026 年第四季度移除。服务令牌现在是首选的验证机制，也用于发送遥测数据。

在xref:/k8s-suse-rancher-prime.adoc#_installing_the_suse_observability_agent[StackPack 安装流程]中向{stackstate-product-name} 添加新群集的流程包括创建与 Rancher RBAC 的 RBAC 代理兼容的群集特定令牌。

== 从应用程序接口密钥迁移到服务令牌

从使用单一 API 密钥迁移到使用服务令牌获取遥测数据有两种选择。

1. 迁移到每个群集的服务令牌。虽然设置起来比较麻烦，但它支持 Rancher RBAC。
2. 迁移到单一服务令牌，供所有 Kubernetes 集群和 OTel 收集器使用。这不支持 Rancher RBAC。

将所有代理和 Open Telemetry 收集器迁移到服务令牌后，现在可以从 Helm 值中移除`global.receiverApiKey` 或`stackstate.apiKey` 以安装{stackstate-product-name} ，并运行`helm upgrade` 以应用更改。

=== 迁移到每个群集的服务令牌

1. 在用户界面中打开 Kubernetes Stackpack
2. 扩展要迁移的群集
3. 单击 "创建新服务令牌 "按钮，获取群集特定服务令牌并复制令牌
4. 按照软件包中的代理安装/升级说明，使用服务令牌作为 API 密钥将代理重新部署到群集。

您也可以使用 CLI 生成令牌，而不是使用用户界面，xref:setup/security/authentication/service_tokens#_telemetry_data_ingestion[请参阅此处]。确保将`update-scoped-permissions` 的作用域也分配给要迁移的群集。其他步骤与用户界面相同。

=== 迁移到单一服务令牌

使用 CLI 生成服务令牌，xref:setup/security/authentication/service_tokens#_telemetry_data_ingestion[请参阅此处]：

[,sh]
----
> sts rbac create-subject --subject my-agent
✅ Created subject 'my-agent'
> sts rbac grant --subject my-agent --permission update-metrics
✅ Granted permission 'update-metrics' on 'system' to subject 'my-agent'
PERMISSION   | RESOURCE
update-metrics | system
----

请勿添加`update-scoped-permissions` 权限，因为它不适用于从多个群集使用的令牌。

按照 Kubernetes StackPack 用户界面和 Open Telemetry 收集器xref:setup/otel/getting-started/getting-started-k8s.adoc[文档]中的说明，重新部署代理和xref:setup/otel/getting-started/getting-started-k8s.adoc[Open Telemetry 收集器]。使用生成的服务令牌作为 API 密钥，方法如下。

== 从输入应用程序接口密钥迁移

外部工具使用导入 API 密钥向 SUSE Observability 集群导入数据（如指标、事件和跟踪）。
这些工具包括 STS Agent 或/和 OTel Collector。服务令牌具有相同甚至更多的功能。按照 API 密钥的相同步骤（上一组），迁移到<<_migrate_to_a_service_token_per_cluster, 每个群集的服务令牌>>或<<_migrate_to_a_single_service_token, 单个服务令牌>>。
