= TLS検証を無効にする
:revdate: 2025-08-25
:page-revdate: {revdate}
:description: SUSE Observability セルフホスト

== SUSE Observabilityエージェント

xref:/use/security/self-signed-certificates.adoc#_suse_observability_agent[SUSE Observability Agentの]SSL検証を無効にするには、以下のようにHelm文字列値を設定する必要があります：

[,bash]
----
--set-string 'global.skipSslValidation'=true \
----

== SUSE Observability CLI

xref:/use/security/self-signed-certificates.adoc#_suse_observability_cli[SUSE Observability CLIから]SSL検証を無効にするには、`sts CLI` コマンドで`--skip-ssl` フラグを指定する必要があります。これは以下の2つの方法のいずれかを使って行うことができる：

*方法1*：

PEMエンコードされたCA証明書ファイルに`--skip-ssl` フラグを指定し、永続的な設定を行う：

[,bash]
----
sts context save \
 --name staging \
 --url https://staging.internal \
 --api-token YOUR_API_TOKEN \
 --skip-ssl
----

*方法2*：

1 回限りの証明書検証に使用する CLI コマンドでは、`--skip-ssl` フラグを指定する：

[,bash]
----
# Using certificate file path
sts agent list \
 --url https://staging.internal \
 --api-token YOUR_API_TOKEN \
 —-skip-ssl


# Using base64-encoded certificate data
sts settings list \
 --url https://staging.internal \
 --api-token YOUR_API_TOKEN \
 —-skip-ssl
----

[WARNING]
====
`--skip-ssl` フラグを使うと、`STS_CA_CERT_PATH` と`STS_CA_CERT_BASE64_DATA` の設定は無視される。セキュリティが低下するので、注意して使用すること。
====

== SUSE Observability用のRancher UI拡張機能

Rancher UIエクステンションからSSL検証を無効にするには、`agent-tls-mode` の設定を変更する必要があります。

UIから設定値を変更するには、グローバル設定ページに移動し、ページの下部にある`agent-tls-mode` 。UIで設定を変更した後、クラスタはリクエストを許可する前に、条件`AgentTlsStrictCheck` が'True'に設定されているかチェックされます。これにより、証明書の不一致による機能停止を防ぐことができる。
