== 概要

このページでは、SUSE Observabilityが収集するKubernetes関連の一般的なメトリクスを定義し、その計算方法を説明します。

=== container_memory_usage

[horizontal]
名称:: `container_memory_usage`
タイプ:: ゲージ
単位:: バイト
ソース:: コンテナの cgroup 階層から直接読み込む。

[cols="1,3"]
|===
|cgroupバージョン | ソースファイル

|v1 | `memory.usage_in_bytes`
|v2 | `memory.current`
|===

概要:: cgroupとその子孫が現在使用しているメモリの総量。これは、匿名メモリー（RSS）、ページキャッシュ、カーネルメモリーを含む合計である。特定のメモリ領域に関する詳細を得るには、以下のメトリクスのいずれかを使用します。

=== container_memory_kernel

[horizontal]
名称:: `container_memory_kernel`
タイプ:: ゲージ
単位:: バイト
ソース:: コンテナの cgroup 階層から直接読み込む。

[cols="1,3"]
|===
|cgroupバージョン | ソースファイル

|v1 | `memory.kmem.usage_in_bytes`
|v2 | `memory.stat` -> `kernel` ファイル内の
|===
概要:: (kernel_stack、pagetables、percpu、vmalloc、slab)を含む総カーネルメモリ量。

=== container_memory_rss

[horizontal]
名称:: `container_memory_rss`
タイプ:: ゲージ
単位:: バイト
ソース:: コンテナの cgroup 階層から直接読み込む。

[cols="1,3"]
|===
|cgroupバージョン | ソースファイル

|v1 | `memory.stat` -> `total_rss` ファイル内の
|v2 | `memory.stat` -> `anon` ファイル内の
|===
概要:: 匿名メモリーの常駐セットサイズ（RSS）。これは、cgroup内のプロセスによってアクティブに使用される、ファイルバッキングされていないメモリ（ヒープとスタック）である。この値は多くの場合、アプリケーションの真の「ワーキング・セット」を反映している。

=== container_memory_cache

[horizontal]
名称:: `container_memory_cache`
タイプ:: ゲージ
単位:: バイト
ソース:: コンテナの cgroup 階層から直接読み込む。

[cols="1,3"]
|===
|cgroupバージョン | ソースファイル

|v1 | `memory.stat` -> `total_cache` ファイル内の
|v2 | `memory.stat` -> `file` ファイル内の
|===
概要:: ページキャッシュ（tmpfsや共有メモリを含むファイルシステムのデータをキャッシュするためのメモリ）。
