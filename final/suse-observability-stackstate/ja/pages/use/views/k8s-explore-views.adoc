= ビューを探る
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

== 概要

一般に、SUSE Observabilityのxref:/use/views/k8s-view-structure.adoc[ビューの]概念では、以前に保存したITランドスケープの領域を監視できます。しかし、特定のビューのサブセットを調査する必要がある場合、そのビューのスコープを失いたくないことがよくある。ここで、*エクスプローラー・ビューの*出番となる。

特定のビュー（`my view` など）のスコープを維持するため、トポロジー要素または要素の選択（コンポー ネント、リレーション、グループなど）に適用されるすべての調査アクションは、自動的に一時的なエクスプローラービューで開き、開始したビュー（`my view / explore` など）の下に表示されます。

エクスプローラビューで自動的に開かれる*調査アクションの*例：

* xref:/use/views/k8s-topology-perspective.adoc[トポロジーパースペクティブの]コンポーネント上で*+ボタンを*クリックすると、そのコンポーネントの非表示の隣接コンポーネントが表示されます。
* コンポーネントに*クイックアクションが*実行された
* トポロジー要素（コンポーネント、リレーション、グループ）をxref:/use/views/k8s-topology-perspective.adoc[トポロジー・パースペクティブで] *ダブルクリック*すると、その要素が調査される。
* トポロジー要素（コンポーネント、リレーション、グループ）を調査するために、*「Explore... 」*リンクまたはボタンがクリックされた。

[TIP]
====
既存のエクスプローラビューから実行された調査アクションは、新しいエクスプローラビューを開く*ことはありません*。
====


== ハンドリング

エクスプローラー・ビューが自動的に作成されると、その元のスコープも定義される。これは、エクスプローラー・ビューに変更を加えた後、元のスコープに戻したい場合に便利です。

エクスプローラービューは一時的なものですが、後でモニターしたり、見直したりしたい場合は、xref:/use/views/k8s-custom-views.adoc[カスタムビューとして]保存することができます。

エクスプローラービューを保存したくない場合は、単にそのビューから離れるか、上部のナビゲーションバーにあるパンくずを使用して元のビューに戻ります。
