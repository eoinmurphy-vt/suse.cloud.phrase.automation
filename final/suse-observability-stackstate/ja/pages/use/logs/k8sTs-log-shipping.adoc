== エージェントのインストール

=== オープンシフト

Openshiftでは、サードパーティのログシッパーは容易にサポートされていませんが、プラットフォームにはログ転送のオプションがあり、SUSE Observabilityのヘルムチャートに含まれるプロムテールの設定の代わりに使用されます。 この設定方法の詳細については、実行中のSUSE ObservabilityインスタンスのKubernetes stackpackドキュメントを参照してください。

=== Kubernetes

SUSE Observability k8s Agentのhelmチャートのデフォルト設定では、チャートで提供されるhelm値によってログシッピングが有効に設定されます：

[,yaml]
----
logsAgent:
  # logsAgent.enabled -- Enable / disable k8s pod log collection
  enabled: true
----

上記により、promtailコンテナが各ノードにデプロイされ、ログを収集してSUSE Observabilityに送信するようになります。 SUSE Observabilityにログを送信することが望ましくないデプロイメントでは、上記の値を`false` に設定します。

=== プロムテールポッドの追加

SUSE Observabilityは、ログの取り込みに調整された設定を使用しますが、これは通常、補助的な要件に沿ったものではありません。 したがって、他のデスティネーションエンドポイントへのログの取り込みのために別のコンフィギュレーションを実行することはできません。その代わりに、エージェントヘルムチャートによってデプロイされるpromtailとは別の懸念として、これらの要求に対処する第2のpromtailポッドを実行することが推奨されます。
