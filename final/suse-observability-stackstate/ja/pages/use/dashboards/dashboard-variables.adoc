= ダッシュボード変数
:revdate: 2025-11-24
:page-revdate: {revdate}
:description: 変数を使用して動的なダッシュボードを作成します。

ダッシュボードの変数を設定することで、動的なウィジェットを持つダッシュボードを作成できます。変数によって、1つのダッシュボード定義をさまざまなユースケースで再利用することができます。

== ダッシュボード変数の表示
上部のナビゲーションバーで、"Save..."ボタンの隣にあるドロップダウンメニューを使い、"Edit dashboard variables "をクリックする。

image::dashboards/open-variables-drawer.png[すべてのピン留めアイテムを追加する]

=== 組み込み変数
{stackstate-product-name} に組み込まれている変数がある。これらの変数名は予約されており、自分の変数に同じ名前をつけることはできない。

すべての組み込み変数を表示するには、「SUSE Observability PromQL変数」というタイトルのアコーディオンをクリックします。

== 新しい変数の追加

. ダッシュボード変数の編集」ドロワーを開く。
. Add variable "ボタンをクリックすると、新しい変数を設定できる2番目のドロワーが開きます。

image::dashboards/variable-drawer-example.png[変数の例]

変数の設定フィールドは以下の通り：

|===
| フィールド名 | 説明

| 名称
| 変数名の識別子。これは<<using-variables, 互換性のあるフィールドで使用>>できる名前です。

| 表示ラベル
| 変数を<<selecting-variables, 選択する>>ときにダッシュボードに表示される変数のフレンドリーな名前。これは変数の特定に役立つ。

| 説明
| 変数の目的を理解するための説明。この説明は「ダッシュボード変数の編集」ドロワーに表示されます。

| タイプ
| 変数の型。
|===

== 変数の種類
=== テキスト変数

テキスト変数には単純なテキストが格納される。初期値を設定し、その変数を読み取り専用にすることができる。

=== リスト変数

リスト変数は、"ソース "フィールドによって定義される可能な値を持つ。各ソース・タイプには、変数の可能な値を絞り込むための追加フィールドが用意されています。

|===
| ソース・タイプ | ソース・フィールド

| メトリック・ラベル名
| 0個以上のシリーズセレクタを入力して、どのメトリックラベル名が存在するかを絞り込む。

| メトリックラベル値
| メトリック・ラベル名と、どのメトリック・ラベル値が存在するかを絞り込むための0個以上の系列セレクタを入力する。

| メトリックPromQL
| カスタムPromQLクエリとラベル名を入力します。結果として得られる可能な値は、与えられたラベル名に対する値である。

| 静的リスト
| 1つ以上のラベルと値のペアを入力し、可能な値の静的リストを作成する。
|===

ソースタイプとその関連フィールドを変更すると、ドロワーの右側のプレビューが新しい可能な値で更新されます。


==== 共通リスト変数設定

また、すべてのリスト変数ソースは以下の共通設定を持つ：

|===
| フィールド名 | 説明

| ソート
| リストの値を並べ替える。例えば、アルファベット順や数字順など。

| 複数の値を許可する
| 一度に複数の値を選択できるようにする。

| すべてを含める」を有効にする
| ユーザーが見るリストに、すべての値を選択する項目を追加します。この項目には、オプションでカスタム値を指定することも、自動生成させることもできます。
|===

== 変数の使用 [[using-variables]]
変数が定義されると、ダッシュボードの様々なフィールドに入力するために使用することができる：

* その他の変数
* ウィジェット名と説明
* ウィジェットのリンクフィールド：URL、名前、ツールチップ
* ウィジェットのクエリフィールド：PromQL式、エイリアス、最小ステップ
* Markdownウィジェットのエディタ

互換フィールドで変数を使用するには、変数名を`${}` で囲んでください（例：`$\{my_variable}` ）。変数の値が補間される。

=== 変数の補間方法
テキスト変数は、その変数のテキストボックスに書かれた文字列として補間される。

同様に、複数の値を許容しないリスト変数や "すべてを含む "リスト変数は、リストから選択された項目の値として補間される。

複数の値を許容するリスト変数は、1つのオプションが選択されているか、複数のオプションが選択されているかに関係なく、選択された値のグループ（例えば`(value_1|value_2)` ）をキャプチャする正規表現として補間されます。

Include Everything "オプションが有効で、その変数に対して選択されているリスト変数は、以下のように補間されます：* 定義されている場合はカスタム値、または * すべての可能な値のグループをキャプチャする生成された正規表現（例：`(value_1|value_2|value_3)` ）。

正規表現キャプチャ・グループとして補間される変数を消費する場合、PromQL正規表現マッチャー`=~` 。

== 変数の選択 [[selecting-variables]]
変数は、ダッシュボードの上部、時間範囲セレクタの隣に表示されます。最初の数変数のみ表示。その他の変数を見るには、"More "ドロップダウンを使用する。

変数をクリックしてセレクタを開き、ダッシュボードに表示したい値を選択する。

異なる値を選択すると、その変数を使用しているウィジェットがその変化に反応します。
