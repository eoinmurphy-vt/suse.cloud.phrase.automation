= オープン・テレメトリーを始める
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

監視が必要なアプリケーションのセットアップを紹介しよう：

* Linuxホスト上で動作する監視対象のアプリケーション/ワークロード
* 同じ Linux ホスト上で動作する Open Telemetry コレクター
* SUSE ObservabilityまたはSUSE Cloud Observability

image::otel/open-telemetry-collector-linux.png[Linux ホスト上で Open Telemetry コレクターを実行し、アプリケーションの計測を行う。]

== オープン・テレメトリー・コレクターの取り付け

[NOTE]
====
本番セットアップでは、コレクターをインストールすることを強く推奨する。なぜなら、コレクターによってサービスがデータを迅速にオフロードできるようになり、コレクターが再試行、バッチ処理、暗号化、あるいは機密データのフィルタリングなどの追加処理を行うことができるからである。
====


まず、コレクターを取り付ける。私たちはそのように設定している：

* 多くの計測アプリケーションからデータを受信する。
* 収集したデータをホスト属性で充実させる
* トレースのメトリクスを生成する
* APIキーを使用した認証を含め、データをSUSE Observabilityに転送します。

さらに、接続に問題がある場合は、データ送信を再試行する。

=== サービス・トークンの作成

サービス・トークンを作成するには2つの方法がある：

* **SUSE Observability UI**- 画面左上をクリックしてメインメニューを開き、`StackPacks` >`Open Telemetry`. まだの方は、`INSTALL` ボタンをクリックしてください。 `CREATE NEW SERVICE TOKEN` ボタンをクリックし、値をクリップボードにコピーする。
* **SUSE Observability CLI**-xref:/use/security/k8s-service-tokens.adoc#_manage_service_tokens[サービス・トークンの管理を]参照してください。

サービス・トークンの値は、以下の説明で`<SERVICE_TOKEN>` と記述されている場所で使用する必要があります。

=== コレクタのインストールと構成

コレクターは、ほとんどのLinuxバージョンとアーキテクチャ用のパッケージ（apk、deb、rpm）を提供し、自動サービス設定には`systemd` 。インストールするには、 https://github.com/open-telemetry/opentelemetry-collector-releases/releases[Githubで]見つけ、最新バージョンを使用するようにサンプル内のURLを更新する：

その他のインストールオプションについては、https://opentelemetry.io/docs/collector/installation/#_linux[Open Telemetry の説明を]参照してください。

インストール後、`/etc/otelcol-contrib/config.yaml` を編集してコレクタの設定を変更します。ファイルを`config.yaml` の例のように変更し、`<otlp-suse-observability-endpoint:port>` をあなたの OTLP エンドポイント (エンドポイントについてはxref:/setup/otel/otlp-apis.adoc[OTLP API を]参照してください) に置き換えて、`<SERVICE_TOKEN>` にサービス・トークンを挿入します：

.config.yaml
[,yaml]
----
receivers:
  otlp:
    protocols:
      # Only bind to localhost to keep the collector secure, see https://github.com/open-telemetry/opentelemetry-collector/blob/main/docs/security-best-practices.md#safeguards-against-denial-of-service-attacks
      grpc:
        endpoint: 127.0.0.1:4317
      http:
        endpoint: 127.0.0.1:4318
  # Collect own metrics
  prometheus:
    config:
      scrape_configs:
      - job_name: 'otel-collector'
        scrape_interval: 10s
        static_configs:
        - targets: ['0.0.0.0:8888']
extensions:
  health_check: {}
  pprof:
    endpoint: 0.0.0.0:1777
  zpages:
    endpoint: 0.0.0.0:55679
  # Use the API key from the env for authentication
  bearertokenauth:
    scheme: SUSEObservability
    token: "<SERVICE_TOKEN>"
exporters:
  nop: {}
  debug: {}
  otlp/suse-observability:
    compression: snappy
    auth:
      authenticator: bearertokenauth
    # Put in your own otlp endpoint, for example otlp-suse-observability.my.company.com:443
    endpoint: <otlp-suse-observability-endpoint:port>
processors:
  memory_limiter:
    check_interval: 5s
    limit_percentage: 80
    spike_limit_percentage: 25
  batch: {}
  # Optionally include resource information from the system running the collector
  resourcedetection/system:
    detectors: [env, system] # Replace system with gcp, ec2, azure when running in cloud environments
    system:
      hostname_sources: ["os"]
connectors:
  # Generate metrics for spans
  spanmetrics:
    metrics_expiration: 5m
    namespace: otel_span
service:
  extensions: [ bearertokenauth, health_check, pprof, zpages ]
  pipelines:
    traces:
      receivers: [otlp]
      processors: [memory_limiter, resourcedetection/system, batch]
      exporters: [debug, spanmetrics, otlp/suse-observability]
    metrics:
      receivers: [otlp, spanmetrics, prometheus]
      processors: [memory_limiter, batch, resourcedetection/system]
      exporters: [debug, otlp/suse-observability]
    logs:
      receivers: [otlp]
      processors: []
      exporters: [nop]
----


最後にコレクタを再起動します：

[,bash]
----
sudo systemctl restart otelcol-contrib
----

コレクターのログを見るには、以下を使用する：

[,bash]
----
sudo journalctl -u otelcol-contrib
----

== アプリケーションから遠隔測定データを収集する

テレメトリーデータを収集する一般的な方法は、Open Telemetry SDK を使用してアプリケーションを計測することです。いくつかの言語のクイックスタートガイドを文書化したが、他にもたくさんある：

* xref:/setup/otel/instrumentation/java.adoc[ジャワ]
* xref:/setup/otel/instrumentation/dot-net.adoc[.NET]
* xref:/setup/otel/instrumentation/node.js.adoc[Node.js]

SDKには追加の設定は必要なく、デフォルトでOTLPまたはOTLP over HTTP(サポートされているプロトコルによる)を介してlocalhostにエクスポートされます。

その他の言語については、https://opentelemetry.io/docs/languages/[opentelemetry.ioの]ドキュメントに従ってください。

== 結果を見る

SUSE Observabilityにアクセスし、Open Telemetry Stackpackがインストールされていることを確認します（メインメニュー \-> Stackpacks）。

しばらくして、アプリケーションが何らかのトラフィックを処理していれば、Open Telemetry \-> services and service instances overviews にあるサービス名で見つけることができるはずです。トレースは、サービスとサービス インスタンス コンポーネントのxref:/use/traces/k8sTs-explore-traces.adoc[トレース エクスプローラ]およびxref:/use/views/k8s-traces-perspective.adoc[トレース パースペクティブに]表示されます。スパン・メトリクスと言語固有のメトリクス（利用可能な場合）は、コンポーネントのxref:/use/views/k8s-metrics-perspective.adoc[メトリクス・パースペクティブで]利用可能になります。
