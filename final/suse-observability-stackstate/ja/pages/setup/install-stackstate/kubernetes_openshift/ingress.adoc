= SUSE Observabilityをクラスタの外部に公開する
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: {stackstate-product-name} セルフホスト

== 概要

{stackstate-product-name} はKubernetes Ingressリソースで公開できる。このページの例では、xref:/setup/install-stackstate/kubernetes_openshift/ingress.adoc#_configure_ingress_via_the_suse_observability_helm_chart[Kubernetes上で動作するSUSE Observability用のHelmを]使用して、TraefikまたはNginx Ingressコントローラを設定する方法を示します。このページでは、異なる方法でイングレス・トラフィックを設定する場合に、どのサービス/ポートの組み合わせを公開するかについても説明する。

{stackstate-product-name} もホストしているクラスタを観察する場合、エージェントのインストール時にxref:/setup/install-stackstate/kubernetes_openshift/ingress.adoc#_agents_in_the_same_cluster[エージェント設定を変更する]ことで、エージェントのトラフィックを完全にクラスタ内に保つことができます。

== {stackstate-product-name} Helmチャートでイングレスを設定する。

{stackstate-product-name} ヘルムチャートは、その値に`ingress` 。デフォルトでは無効になっている。以下の例は、Helmチャートを使用して、TLS暗号化を有効にしたイングレスコントローラを設定する方法を示している。コントローラ自体および証明書のセットアップは、本文書の範囲外であることに 注意。

{stackstate-product-name} のイングレスを設定するには、以下のような内容のファイル`ingress_values.yaml` を作成する。`MY_DOMAIN` を独自ドメイン（イングレスコントローラーとリンクしている）に置き換え、`tls-secret` に正しい名前を設定する。設定すべき正しい注釈については、お使いのイングレスコントローラのマニュアルを参照してください。例えば、TLSを使用しない場合、そのセクションは省略できますが、{stackstate-product-name} 、トラフィックも暗号化されないことに注意してください。

[CAUTION]
====
rancher UIエクステンションを使用するには、TLSの設定が必要です。
====

== オープン・テレメトリーのイングレス・ルールを設定する

{stackstate-product-name} Helmチャートは、`ingress` を作成することができる`opentelemetry-collector` サービスをその値で公開している。デフォルトでは無効になっている。`opentelemetry-collector` 、イングレスはGRPCプロトコルをサポートする必要がある。コントローラ自体および証明書のセットアップは、本文書の範囲外であることに 注意。

{stackstate-product-name} 用に`opentelemetry-collector` イングレスを設定するには、以下のような内容のファイル`ingress_otel_values.yaml` を作成する。`MY_DOMAIN` を独自ドメイン（イングレスコントローラーとリンクしている）に置き換え、`otlp-tls-secret` に正しい名前を設定する。設定すべき正しい注釈については、お使いのイングレスコントローラのマニュアルを参照してください。例えば、TLSを使用しない場合、そのセクションは省略できますが、{stackstate-product-name} 、トラフィックも暗号化されないことに注意してください。

GRPCホスト・エントリーは、正しいバックエンド・サービスにトラフィックをルーティングするために、明示的な`serviceName: suse-observability-otel-collector-grpc` が必要であることに注意。

[,text]
----
opentelemetry-collector:
  ingress:
    enabled: true
    ingressClassName: traefik
    annotations:
      traefik.ingress.kubernetes.io/router.entrypoints: websecure
    hosts:
      - host: otlp-suse-observability.MY_DOMAIN
        paths:
          - path: /
            pathType: Prefix
            serviceName: suse-observability-otel-collector-grpc
            port: 4317
    tls:
      - hosts:
          - otlp-suse-observability.MY_DOMAIN
        secretName: otlp-tls-secret
    additionalIngresses:
      - name: otlp-http
        ingressClassName: traefik
        annotations:
          traefik.ingress.kubernetes.io/router.entrypoints: websecure
        hosts:
          - host: otlp-http-suse-observability.MY_DOMAIN
            paths:
              - path: /
                pathType: Prefix
                port: 4318
        tls:
          - hosts:
              - otlp-http-suse-observability.MY_DOMAIN
            secretName: otlp-http-tls-secret
----

== 外部ツールによる設定

インストールされているKubernetesクラスタの外から{stackstate-product-name} にアクセスできるようにするには、`<namespace>-stackstate-k8s-router` サービスのポート`8080` にトラフィックをルーティングすれば十分だ。{stackstate-product-name} のUIは、そのサービスのルートパス(すなわち`http://<namespace>-stackstate-k8s-router:8080`)で直接アクセスすることができ、エージェントは`/receiver` パス(`http://<namespace>-stackstate-k8s-router:8080/receiver`)を使用する。

初期インストール中に生成された値ファイルの`baseUrl` を必ず更新してください。この値は、{stackstate-product-name} がエージェントの便利なインストール手順を生成するために使用します。

[NOTE]
====
Nginxまたは同様のHTTPサーバーをリバースプロキシとして手動で設定する場合は、それがウェブソケットもプロキシできることを確認してください。Nginxでは、以下のディレクティブを`location` ：

[,text]
----
proxy_set_header Upgrade                 $http_upgrade;
proxy_set_header Connection              "Upgrade";
----

====


[CAUTION]
====
{stackstate-product-name} 自体はTLS暗号化トラフィックを使用しないので、TLS暗号化はイングレスコントローラーまたは外部のロードバランサーで処理されることが期待される。
====


== 同じクラスタ内のエージェント

{stackstate-product-name} と同じクラスタにデプロイされたエージェントは、もちろん{stackstate-product-name} が公開されている外部 URL を使用できるが、Kubernetes の内部ネットワークのみを介して{stackstate-product-name} インスタンスに直接接続するようにエージェントを設定することも可能だ。そのためには、xref:/k8s-quick-start-guide.adoc[Agent Kubernetesのインストールから] `helm install` コマンドの`'stackstate.url'` の値を、ルーターサービスの内部クラスタURLに置き換えます（上記も参照）： `http://<namespace>-suse-observability-router.<namespace>.svc.cluster.local:8080/receiver/stsAgent` (`<namespace>` の部分は{stackstate-product-name} の名前空間に置き換える必要があります）。


== {stackstate-product-name} ルーターサービスポートの転送

デフォルトでは、{stackstate-product-name} Helmチャートはルーターポッドとサービスをデプロイする。このサービスはポート`8080` を公開する。これはIngress経由で公開する必要がある唯一のエントリーポイントである。
ポートフォワーディングで{stackstate-product-name} にアクセスすることもできる。その場合、リクエストのオリジンとしてlocalhostを許可しなければならない。

[WARNING]
====
* `stackstate.allowedOrigins` で`localhost` を許可するのは、ポートフォワーディングを使用する場合、ローカルでの開発またはデバッグのみを目的としている。
* これは本番用の*セットアップではない*。Ingress経由で{stackstate-product-name} 、この許可されたオリジンを削除する。
====

Ingressを設定せずにUIにアクセスするには、ルーターのサービスポートを転送します：

[,text]
----
kubectl port-forward service/<helm-release-name>-suse-observability-router 8080:8080 --namespace suse-observability
----

ポートフォワードで{stackstate-product-name} にアクセスする場合、ブラウザは`http://localhost:8080` をリクエストオリジンとして使用する。
このオリジンからのリクエストを許可するには、Helmの値の`stackstate.allowedOrigins` リストに追加するか、Helmのコマンド:に直接渡します：

[,bash]
----
helm upgrade \
  --install \
  --namespace suse-observability \
  --values $VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/sizing_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/affinity_values.yaml \
  --set stackstate.allowedOrigins={"http://localhost:8080"} \
suse-observability \
suse-observability/suse-observability
----
