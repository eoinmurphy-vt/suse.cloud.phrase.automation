= SUSE Observabilityサポートパッケージ（ログ）
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability セルフホスト

== 概要

このページでは、SUSE Observabilityクラスタからログを収集するためのツールとガイダンスを提供します。これらのログは、問題を診断し、根本原因を特定し、期待通りに機能しない可能性のあるクラスタの問題をトラブルシューティングするために不可欠です。
{stackstate-product-name} クラスターで予期しない動作、パフォーマンスの低下、または障害が発生した場合、このページを使用して必要なデータを収集し、何が問題であったかを理解して対策を講じることができます。このスクリプトの詳細の隣には、アクティブなスクリプトxref:/setup/install-stackstate/support-package-performance.adoc[Support Package (Performance)]があり、クラスタと{stackstate-product-name} のパフォーマンス測定をアクティブに実行します。

== 備考

このスクリプトは、SUSE Observabilityクラスタにアクセスできるようにkubectlが設定されているホスト、または必要な権限とクラスタへの接続を持つ他のホストから実行する必要があります。
これは、ホスト上でrootユーザーを使って直接行うか、sudo経由で行うことができる。 kubeconfig が設定されていない場合は、コマンド`export KUBECONFIG=$PATH-TO-YOUR/kubeconfig` を使用する。

== 使用方法

このスクリプトは、`root` ユーザーまたは`sudo` を使用して、ホスト上で直接ダウンロードして実行する必要があります。

=== スクリプトをダウンロードして実行する

xref::attachment$suse-observability_logs_collector.sh[suse-observability_logs_collector.sh]スクリプトをダウンロードしてください。

* スクリプトを別名で保存する： `suse-observability_logs_collector.sh`
* 以下のコマンドを使ってスクリプトを実行する：
+
[,bash]
----
bash suse-observability_logs_collector.sh
----
+
これにより、カレント・ディレクトリに`suse-observability_logs_<date>.tar.gz` というファイルが生成される。このファイルをケースにアップロードしてください。

追加の引数を指定することもできる：
[,bash]
----
> bash suse-observability_logs_collector.sh -h
SUSE Observability support package collector.
Fetches details of a SUSE Observability instance for troubleshooting.

Usage: ./suse-observability_logs_collector.sh [options] [<namespace>]

options:
  -h  Print this help
  -e  Fetch logs for suse-observability pods from Elasticsearch
  -r  Time range for logs to fetch from Elasticsearch,
      when not specified "7d" is used

<namespace>:
  The namespace that is running SUSE Observability, or
  "suse-observability" when not specified
----

例えば、{stackstate-product-name} がネームスペース`my-observability` にデプロイされている場合、組み込みの elasticsearch から3日分のログを収集する：
[,bash]
----
bash suse-observability_logs_collector.sh -e -r 3d my-observability
----
