== 概要

{stackstate-product-name} CLIを使用して、メトリックのバインディングを検査し、変更することができます。 これらの設定は、他の設定と同様に`sts settings` 。

=== メートルバインディングについて

既存のメトリック・バインディングの定義は、`describe` ：

[,bash]
----
sts settings describe --ids 190567588459765
_version: 1.0.93
nodes:
- _type: MetricBinding
  chartType: line
  description: Bytes allocated to GC Heap
  enabled: true
  id: -1
  identifier: urn:stackpack:kubernetes-v2:shared:metric-binding:pod:dotnet-gc-allocated
  layout:
    metricPerspective:
      section: GC
      tab: .NET
      weight: 3
  name: .NET GC Allocated
  priority: high
  queries:
  - alias: allocated
    expression: rate(process_runtime_dotnet_gc_allocations_size_bytes_total{k8s_cluster_name="${tags.cluster-name}", k8s_namespace_name="${tags.namespace}", k8s_pod_name="${name}"}[${__rate_interval}])
  scope: (label = "stackpack:kubernetes" and type = "pod")
  unit: bytes(IEC)
timestamp: 2026-01-14T13:11:07.575662922Z[Etc/UTC]
----

== メトリック・バインディングの変更

[NOTE]
====
推奨される作業方法は、メトリックバインディング (および{stackstate-product-name} で作成されたその他のカスタムリソース) を YAML ファイルとしてxref:/setup/custom-integrations/overview.adoc[StackPack] に保存することです。そこから変更を手動で適用することも、GitHubアクションやGitLabパイプラインなどのCI/CDシステムでSUSE Observability CLIを使用して完全に自動化することもできます。
====

=== メトリックバインディングの作成/更新
次のようなファイル`metric-bindings.yaml` ：

[source,bash]
----
nodes:
- _type: MetricBinding
  chartType: line
  enabled: true
  tags: {}
  unit: short
  name: Replica counts
  priority: MEDIUM
  identifier: urn:stackpack:my-stackpack:metric-binding:my-deployment-replica-counts
  queries:
    - expression: max_over_time(kubernetes_state_deployment_replicas{cluster_name="${tags.cluster-name}", namespace="${tags.namespace}", deployment="${name}"}[${__interval}])
      alias: Total replicas
  scope: type = "deployment" and label = "stackpack:kubernetes"
----

xref:/setup/cli/cli-sts.adoc[SUSE Observability CLIを]使用して、メトリックバインディングを作成します：

[,bash]
----
sts settings apply -f metric-bindings.yaml
----

デプロイのメトリクスパースペクティブを開いて、SUSE Observabilityで結果を確認します。結果に満足できない場合は、YAMLファイルのメトリックバインディングを変更し、コマンドを再度実行して更新するだけです。ノードのリストは、多くのメトリック・バインディングの追加をサポートしている。先ほどと同じ手順で、YAML配列に別のメトリックバインディングのエントリを追加するだけです。

[CAUTION]
====
識別子は、メトリックバインディングの一意のキーとして使用される。識別子を変更すると、既存のバインディングが更新されるのではなく、新しいメトリックバインディングが作成される。
====

=== メトリックバインディングの削除

最後に、メトリックバインディングを削除するには

[,bash]
----
sts settings delete --ids <id>
----

このコマンドの`<id>` は識別子ではなく、`sts settings list` 出力の`Id` 列の番号である。
