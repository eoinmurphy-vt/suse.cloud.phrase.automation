== コンポーネントマッピング

`OtelComponentMapping` には、OpenTelemetry データからトポロジー・コンポーネントがどのように作成（または更新）されるかが記述されています。

OpenTelemetryから作成されるコンポーネントの典型的な例は以下の通りです（これらはデフォルトのOpenTelemetry StackPackの一部です）：

* サービス (`service.name` /`service.namespace` より)
* サービスインスタンス（`service.instance.id` から派生）
* データベース
* ホストとKubernetesリソース（ノード、ネームスペース、ポッド）
* サーバーレスまたはバッチ環境での関数とタスク

== 関係マッピング

`OtelRelationMapping` は、OpenTelemetry データから2つのコンポーネント間の関係がどのように作られるかを説明しています。

リレーションシップの一般的な例を以下に挙げる（これらはデフォルトのOpenTelemetry StackPackの一部である）：

* 同期サービスコール (クライアント-> サーバ)
* 非同期メッセージングフロー（プロデューサー-> コンシューマー、コンシューマー-> ピア、プロデューサー-> ピア）
* サービスからデータベースへの接続
* サービス・インスタンス
* 実行関係（サービスまたは関数がタスク、ホスト、または関数を実行する）

== 式と共通表現言語（CEL）

トポロジーマッピングは、https://cel.dev/[Common Expression Language（CEL]）を使用して、テレメトリを選択し、値を抽出し、コンポーネントフィールドとリレーションフィールドを構築する。

ほとんどの式は、単純なブール値のチェック、属性の検索、文字列の構築です。より高度な表現は、必要に応じて段階的に導入することができる。

トポロジー・マッピングでCELを活用する方法の詳細については、xref:/setup/custom-integrations/otelmappings/schemas-ref.adoc[スキーマの]ページを参照のこと。
