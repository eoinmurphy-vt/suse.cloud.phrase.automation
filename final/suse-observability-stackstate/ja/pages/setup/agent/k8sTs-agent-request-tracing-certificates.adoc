= リクエスト・トレース用証明書 サイドカー・インジェクション
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

エージェントのインストール時に`--set httpHeaderInjectorWebhook.enabled=true` を使用すると有効になるxref:/setup/agent/k8sTs-agent-request-tracing.adoc#_enabling_the_trace_header_injection_sidecar[サイドカーインジェクションメカニズムは]、自己署名証明書を作成し、Kubernetesクラスタ内の`Secret` と`MutatingWebhookConfiguration` オブジェクトへの書き込みアクセスを許可する`ClusterRole` を使用する。

セキュリティ上、クラスタ全体に書き込み権限を与える`ClusterRoles` を作成することが望ましくない場合、または証明書を提供する別の方法がある場合：

. 自己署名証明書を<<_generate_a_certificate_locally,ローカルで>>生成する。
. k8shttps://cert-manager.io/[cert-manager](すでにクラスタ上にある場合)を<<_generate_a_certificate_using_the_cert_manager, `ClusterIssuer` で>>使用してください。

== ローカルで証明書を生成する

ローカルで証明書を生成するには、以下の手順を踏む：

. 証明書生成スクリプトをダウンロードして実行し、正しい証明書を含むヘルム値(`tls_values.yaml`)ファイルを生成する：
+
----
wget https://raw.githubusercontent.com/StackVista/http-header-injector/main/scripts/generate_ca_cert.sh
chmod +x generate_ca_cert.sh
./generate_ca_cert.sh <helm-agent-release-name> <helm-agent-namespace>
----
+
helmコマンドとネームスペースで使用するリリース名を必ず使用してください。

. helmの起動コマンドに`--set httpHeaderInjectorWebhook.enabled=true -f tls_values.yaml` を追加して、追加設定を追加したエージェントをインストールします。

== cert-managerを使用して証明書を生成する

クラスタにhttps://cert-manager.io/[cert-managerが]インストールされ、`ClusterIssuer` が設定されている場合、`ClusterIssuer` によって発行された証明書をサイドカーインジェクタのエージェントで使用することができます。これを行うには、以下のコマンドライン引数を追加してエージェントをインストールする：`--set httpHeaderInjectorWebhook.enabled=true --set-string httpHeaderInjectorWebhook.webhook.tls.mode="cert-manager" --set-string httpHeaderInjectorWebhook.webhook.tls.certManager.issuer="<my-cluster-issuer>"`.必ずmy-cluster-issuerをクラスタ内の発行者名に置き換えてください。
