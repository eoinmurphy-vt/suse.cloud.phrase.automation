= 接続をプロキシするSUSE Observability Kubernetesエージェントの設定
:revdate: 2025-07-10
:page-revdate: {revdate}

SUSE Observability Kubernetes Agentでは、開始する接続に対してHTTPまたはHTTPSプロキシ設定を行うことができます。

== SUSE Observabilityとの通信用プロキシ

SUSE Observabilityバックエンドへの接続をプロキシするようにエージェントを設定するには、Helm設定を使用します。

=== ヘルム構成

==== Via`values.yaml` ファイル

. Helmチャート`values.yaml` ファイルを開きます。
. `global.proxy.url` 、プロキシURLを指定します：
+
[,yaml]
----
 global:
   proxy:
     url: "https://proxy.example.com:8080"
----

. オプションとして、プロキシが署名付き証明書を持っていない場合、`global.skipSslValidation` を`true` に設定してSSL検証を無効にする：
+
[,yaml]
----
 global:
   skipSslValidation: true
----

==== コマンドラインフラグ

. Helmチャートのインストール時に、`--set` フラグを使用してプロキシURLを指定します：
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.proxy.url="https://proxy.example.com:8080"
----

. コマンドラインでSSL検証を無効にするには、次のようにする：
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.skipSslValidation=true
----
