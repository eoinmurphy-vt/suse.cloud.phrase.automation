= Slack通知のためのSUSE Observabilityの設定
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability セルフホスト

[CAUTION]
====
SUSE ObservabilityのSaaSユーザーは、追加設定なしでSlack通知を使用できます。このガイドは、Slack通知チャネルの使用を計画している、セルフホスト型のSUSE Observabilityインストールにのみ適用されます。
====


SUSE ObservabilityでSlack通知チャネルを使用する前に、まず次の手順に従ってSlackとSUSE Observabilityの両方を設定する必要があります：

. ワークスペースにSUSE Observability用のSlackアプリを作成する
. Slackアプリの認証情報を使用してSUSE Observabilityを設定します。

== SUSE Observability用のSlackアプリの作成

[NOTE]
====
ワークスペースのSlackアプリを管理するには、Slackの権限が必要です。
====


https://api.slack.com/apps[Slack APIページに]アクセスし、*Create New App*ボタンをクリックします。

* 開いたダイアログで「アプリのマニフェストから」オプションを選択します。
* 通知を送信するワークスペースを選択し、nextをクリックします。
* 以下の Slack アプリ マニフェストの内容をコピーし、テキスト エリアに貼り付けます。`redirect_urls` 、必ずSUSE ObservabilityインスタンスのURLに置き換えてください。次へ」をクリックする。
* URLが正しいこと、リストされた「ボットスコープ」が`channels:join, channels:read, chat:write, groups:read` 、作成ボタンをクリックしてアプリを作成することを確認する。
* アプリの "基本情報 "ページでは、アイコンを変更できます(表示情報セクション)。例えば、SUSE Observabilityロゴimage:logo/stackstate-logo.png[SUSE Observabilityロゴに,25,25]置き換えることができます。

.SUSE Observability用のSlackアプリのマニフェスト
[,json]
----
{
    "display_information": {
        "name": "SUSE Observability",
        "description": "Receive notification messages from SUSE Observability",
        "background_color": "#000000"
    },
    "features": {
        "bot_user": {
            "display_name": "SUSE Observability",
            "always_online": true
        }
    },
    "oauth_config": {
        "redirect_urls": [
            "https://the.url.of.your.stackstate.installation"
        ],
        "scopes": {
            "bot": [
                "channels:join",
                "channels:read",
                "chat:write",
                "groups:read"
            ]
        }
    },
    "settings": {
        "org_deploy_enabled": false,
        "socket_mode_enabled": false,
        "token_rotation_enabled": false
    }
}
----


== Slackアプリの認証情報を使用してSUSE Observabilityを設定します。

SUSE Observabilityには、作成したSlackアプリの認証情報を設定する必要があります。これを行うには、SUSE Observabilityインストールの`values.yaml` ：

[,yaml]
----
stackstate:
  components:
    all:
      extraEnv:
        open:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientId: "<app client id>"
        secret:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientSecret: "<app client secret>"
----

`<app client id>` と`<app client secret>` の値は、作成したSlackアプリの "基本情報 "ページの "App credentials "セクションにあります。SUSE Observabilityxref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[(Kubernetes]またはxref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_deploy_suse_observability_with_helm[OpenShift]xref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[用])のインストール時に使用したのと同じHelmコマンドを実行して、これらの設定変更を適用します。

これでSlackの通知チャンネルを使う準備ができました！
