== Instrumentación automática

La instrumentación automática para Node.js se realiza incluyendo las librerías Javascript de instrumentación automática con tu aplicación.  una amplia gama de https://github.com/open-telemetry/opentelemetry-js-contrib/tree/main/metapackages/auto-instrumentations-node#_supported_instrumentations[Es compatible con].

La instrumentación automática no requiere ninguna modificación de la aplicación. Para configurarlo, sigue estos pasos:

. Añada el SDK de instrumentación de Open Telemetry a su aplicación:
+
[,bash]
----
npm install --save @opentelemetry/api
npm install --save @opentelemetry/auto-instrumentations-node
----

. Actualice el comando que inicia su aplicación para cargar el SDK, ya sea actualizando el punto de entrada o el comando de la imagen Docker o actualizando `command` en el manifiesto de Kubernetes para su aplicación. Añadir `--require @opentelemetry/auto-instrumentations-node/register`:
+
[,bash]
----
node --require @opentelemetry/auto-instrumentations-node/register app.js
----

. Despliegue su aplicación con las variables de entorno adicionales xref:/setup/otel/languages/sdk-exporter-config.adoc[para configurar el nombre del servicio y el punto final del exportador].
. xref:/setup/otel/languages/verify.adoc[Compruebe] que SUSE Observability recibe trazas y/o métricas

Para más detalles, consulte la https://opentelemetry.io/docs/languages/js/automatic/[documentación de].

[CAUTION]
====
La instrumentación automática configurada mediante variables de entorno sólo admite trazas hasta que se resuelva este https://github.com/open-telemetry/opentelemetry-js/issues/4551[problema de]. Para habilitar las métricas de la instrumentación automática se necesitan cambios en el código. Siga las instrucciones de la https://opentelemetry.io/docs/languages/js/exporters/#_usage_with_nodejs[documentación de] para realizar estos cambios.
====


== Instrumentación manual

La instrumentación manual puede utilizarse cuando se necesitan métricas, trazas o registros de partes del código que no son compatibles con la instrumentación automática. Por ejemplo, bibliotecas no compatibles, código interno o métricas a nivel empresarial.

Para capturar esos datos es necesario modificar la aplicación.

. Incluya el SDK de telemetría abierta como dependencia
. Añada código a su aplicación para capturar métricas, intervalos o registros cuando sea necesario

Existe documentación detallada al respecto en https://opentelemetry.io/docs/languages/js/instrumentation/[las páginas de documentación de].

Asegúrese de utilizar el exportador OTLP y configurar correctamente el punto final del exportador desde el código. Consulte también la https://opentelemetry.io/docs/languages/js/exporters/#_usage_with_nodejs[documentación de]. Asumiendo que ha configurado el exportador como xref:/setup/otel/collector.adoc[se documenta], el endpoint que necesita ser configurado es `http://opentelemetry-collector.open-telemetry.svc.cluster.local:4317`, usando gRPC. Véase también xref:/setup/otel/languages/sdk-exporter-config.adoc#_grpc_vs_http[gRPC vs HTTP] en caso de que gRPC sea problemático.
