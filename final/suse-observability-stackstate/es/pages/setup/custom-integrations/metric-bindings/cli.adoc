== Visión general

Puede utilizar la CLI {stackstate-product-name} para inspeccionar y modificar los enlaces métricos.  Estos se manejan como otros ajustes utilizando el comando `sts settings`.

=== Descripción de las fijaciones métricas

Puede obtener la definición de un enlace métrico existente utilizando el comando `describe`:

[,bash]
----
sts settings describe --ids 190567588459765
_version: 1.0.93
nodes:
- _type: MetricBinding
  chartType: line
  description: Bytes allocated to GC Heap
  enabled: true
  id: -1
  identifier: urn:stackpack:kubernetes-v2:shared:metric-binding:pod:dotnet-gc-allocated
  layout:
    metricPerspective:
      section: GC
      tab: .NET
      weight: 3
  name: .NET GC Allocated
  priority: high
  queries:
  - alias: allocated
    expression: rate(process_runtime_dotnet_gc_allocations_size_bytes_total{k8s_cluster_name="${tags.cluster-name}", k8s_namespace_name="${tags.namespace}", k8s_pod_name="${name}"}[${__rate_interval}])
  scope: (label = "stackpack:kubernetes" and type = "pod")
  unit: bytes(IEC)

----

== Modificación de las fijaciones métricas

[NOTE]
====
La forma recomendada de trabajar es almacenar los enlaces métricos (y cualquier otro recurso personalizado creado en {stackstate-product-name}) como archivos YAML en xref:/setup/custom-integrations/overview.adoc[un StackPack]. A partir de ahí, los cambios se pueden aplicar manualmente o se pueden automatizar por completo utilizando la CLI de SUSE Observability en un sistema CI/CD como acciones de GitHub o canalizaciones de GitLab.
====

=== Crear/actualizar un enlace métrico
Cree un archivo `metric-bindings.yaml` con el siguiente aspecto:

[source,bash]
----
nodes:
- _type: MetricBinding
  chartType: line
  enabled: true
  tags: {}
  unit: short
  name: Replica counts
  priority: MEDIUM
  identifier: urn:stackpack:my-stackpack:metric-binding:my-deployment-replica-counts
  queries:
    - expression: max_over_time(kubernetes_state_deployment_replicas{cluster_name="${tags.cluster-name}", namespace="${tags.namespace}", deployment="${name}"}[${__interval}])
      alias: Total replicas
  scope: type = "deployment" and label = "stackpack:kubernetes"
----

Utilice xref:/setup/cli/cli-sts.adoc[la CLI de SUSE Observability] para crear el enlace métrico:

[,bash]
----
sts settings apply -f metric-bindings.yaml
----

Verifique los resultados en SUSE Observability abriendo la perspectiva de métricas para una implantación. Si no está satisfecho con el resultado, simplemente cambie el enlace métrico en el archivo YAML y vuelva a ejecutar el comando para actualizarlo. La lista de nodos permite añadir muchos enlaces métricos. Simplemente añada otra entrada de enlace métrico a la matriz YAML siguiendo los mismos pasos que antes.

[CAUTION]
====
El identificador se utiliza como clave única de un enlace métrico. Al cambiar el identificador se creará un nuevo enlace métrico en lugar de actualizar el existente.
====

=== Eliminar un enlace métrico

Por último, para eliminar un enlace métrico, utilice

[,bash]
----
sts settings delete --ids <id>
----

El `<id>` en este comando no es el identificador sino el número en la columna `Id` de la salida `sts settings list`.
