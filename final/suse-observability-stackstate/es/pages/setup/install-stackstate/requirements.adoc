=== Recursos necesarios

Existen diferentes opciones de instalación disponibles para SUSE Observability. Es posible instalar SUSE Observability en una configuración de alta disponibilidad (HA) o de instancia única (no HA). La configuración noHA se recomienda sólo para fines de prueba. Para entornos de producción, se recomienda instalar SUSE Observability en una configuración de HA. Para una implantación estándar, de producción, el cuadro de SUSE Observability Helm implantará muchos servicios en una configuración redundante con 3 instancias de cada servicio.

En la siguiente tabla encontrará los recursos necesarios para las distintas opciones de instalación. Para la configuración de HA puede encontrar diferentes perfiles de instalación en función del tamaño del entorno observado.
Un nodo observado en esta tabla de dimensionamiento se toma con 4 vCPU y 16 GB de memoria, nuestro `default node size`.
Si los nodos en tu cluster observado son más grandes, pueden contar para múltiples `default nodes`, así que un nodo de 12vCPU y 48GB cuenta como 3 `default nodes` bajo observación cuando se escoge un perfil.

[NOTE]
====
Los requisitos mostrados para el perfil representan la cantidad total de recursos necesarios para ejecutar el servidor Suse Observability.
Para garantizar que se puedan asignar todos los diferentes servicios del servidor Suse Observability:

* Para instalaciones noHA el tamaño de nodo recomendado es 4VCPU, 8GB
* Para instalaciones de HA de hasta 500 nodos, el tamaño mínimo recomendado es de 8 VCPU y 16 GB.
* Para instalaciones HA de 4000 nodos, el tamaño de nodo mínimo recomendado es 16VCPU, 32GB
====


Estos son sólo los límites superior e inferior de los recursos que puede consumir SUSE Observability en las diferentes opciones de instalación. El uso real de recursos dependerá de las funciones utilizadas, los límites de recursos configurados y los patrones de uso dinámicos, como el despliegue o el escalado de DaemonSet. Para nuestros clientes autoalojados, recomendamos empezar con los requisitos predeterminados y supervisar el uso de recursos de los componentes de SUSE Observability.

[NOTE]
====
Los requisitos mínimos no incluyen la capacidad sobrante de CPU/Memoria para garantizar actualizaciones continuas de la aplicación sin problemas.
====


Para instalar SUSE Observability, siga las instrucciones de instalación que se indican a continuación:

* xref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc[Kubernetes]
* xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc[OpenShift]

=== Almacenamiento

SUSE Observability utiliza reclamaciones de volumen persistentes para los servicios que necesitan almacenar datos. La clase de almacenamiento predeterminada para el clúster se utilizará para todos los servicios a menos que se anule mediante valores especificados en la línea de comandos o en un archivo `values.yaml`. Todos los servicios vienen con un tamaño de volumen preconfigurado que debería ser bueno para empezar, pero se puede personalizar más tarde utilizando variables según sea necesario.

[NOTE]
====
Para entornos de producción, NFS no se recomienda ni se admite para la provisión de almacenamiento en SUSE Observability debido al riesgo potencial de corrupción de datos.
====


Para nuestros diferentes perfiles de instalación, los siguientes son los requisitos de almacenamiento por defecto:

=== Límites de recursos del espacio de nombres

No se recomienda establecer una ResourceQuota, ya que puede interferir con las solicitudes de recursos. Los recursos requeridos por SUSE Observability variarán en función de las funciones utilizadas, los límites de recursos configurados y los patrones de uso dinámicos, como la implantación o la ampliación de DaemonSet.

Si es necesario establecer un ResourceQuota para su implementación, el límite de recursos del espacio de nombres debe establecerse para que coincida con los xref:/setup/install-stackstate/requirements.adoc#_resource_requirements[requisitos de tamaño] del nodo.
