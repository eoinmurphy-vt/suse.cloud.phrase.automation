== Visión general

Esta página proporciona herramientas y orientación para recopilar registros de un clúster de SUSE Observability. Estos registros son esenciales para diagnosticar problemas, identificar las causas raíz y solucionar problemas en clusters que pueden no estar funcionando como se esperaba.
Si experimenta un comportamiento inesperado, un rendimiento degradado o un fallo en su clúster {stackstate-product-name}, esta página le ayudará a recopilar los datos necesarios para comprender qué ha fallado y tomar medidas correctivas. Junto al script detallado aquí, hay un script activo xref:/setup/install-stackstate/support-package-performance.adoc[Support Package (Performance)] que ejecutará activamente mediciones de rendimiento en el cluster y {stackstate-product-name}.

== Notas

El script debe ejecutarse desde un host que tenga kubectl configurado con acceso al clúster de SUSE Observability o desde cualquier otro host con los permisos necesarios y conectividad al clúster.
Esto se puede hacer directamente en el host utilizando el usuario root o a través de sudo.  Si kubeconfig no está configurado, utilice el comando `export KUBECONFIG=$PATH-TO-YOUR/kubeconfig`.

== Utilización

El script debe descargarse y ejecutarse directamente en el host, utilizando el usuario `root` o `sudo`.

=== Descargar y ejecutar el script

Descargue el script xref::attachment$suse-observability_logs_collector.sh[suse-observability_logs_collector.sh].

* Guarda el script como: `suse-observability_logs_collector.sh`
* Ejecute el script utilizando los siguientes comandos:
+
[,bash]
----
bash suse-observability_logs_collector.sh
----
+
Esto generará un archivo en el directorio actual llamado `suse-observability_logs_<date>.tar.gz`. Por favor, suba este archivo al caso.

Se pueden especificar argumentos adicionales:
[,bash]
----
> bash suse-observability_logs_collector.sh -h
SUSE Observability support package collector.
Fetches details of a SUSE Observability instance for troubleshooting.

Usage: ./suse-observability_logs_collector.sh [options] [<namespace>]

options:
  -h  Print this help
  -e  Fetch logs for suse-observability pods from Elasticsearch
  -r  Time range for logs to fetch from Elasticsearch,
      when not specified "7d" is used

<namespace>:
  The namespace that is running SUSE Observability, or
  "suse-observability" when not specified
----

Por ejemplo, para recopilar 3 días de registros del elasticsearch integrado cuando {stackstate-product-name} se despliega en el espacio de nombres `my-observability`:
[,bash]
----
bash suse-observability_logs_collector.sh -e -r 3d my-observability
----
