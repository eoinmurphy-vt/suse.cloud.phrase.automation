= Configuración de valores de afinidad
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: {stackstate-product-name} Configuración de afinidad

Las configuraciones de afinidad controlan el comportamiento de la programación de pods en {stackstate-product-name}. Los valores de afinidad ayudan a optimizar la utilización de los recursos y a garantizar una alta disponibilidad controlando dónde se programan los pods.

== Opciones de configuración disponibles

=== Afinidad de nodos

Puede utilizar la afinidad de nodos para programar pods en nodos o grupos de instancias específicos, como nodos EC2 desplegados en la misma zona de disponibilidad.

[,yaml]
----
affinity:
  # Node Affinity settings - applied to all components when configured
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
----

=== Pod Anti-Affinity

Puede utilizar pods antiafinidad para programar réplicas de servicios de datos en diferentes nodos para garantizar una alta disponibilidad. Por defecto, la programación es obligatoria (antiafinidad dura) y `topologyKey` es `kubernetes.io/hostname`.

[,yaml]
----
affinity:
  podAntiAffinity:
    # Enable required pod anti-affinity (true = hard, false = soft)
    requiredDuringSchedulingIgnoredDuringExecution: true
    # Topology key for pod anti-affinity
    topologyKey: "kubernetes.io/hostname"
----

== Comportamiento

=== Afinidad de nodos

* **Cuando está configurado**: Se aplica a todos los componentes

=== Pod Anti-Affinity

* **Cuando está configurado**: Sólo se aplica cuando `sizing.profile` termina con `-ha` (perfiles de alta disponibilidad)
* **Perfiles de HA**: `150-ha`, `250-ha`, `500-ha`, `4000-ha`
* **Componentes afectados**: Todos los servicios de datos con estado, incluidos Clickhouse, Kafka, Zookeeper, VictoriaMetrics, StackGraph y Elasticsearch.

== Ejemplos de configuración

=== Afinidad básica de nodos (misma zona de disponibilidad) + PodAntiAffinity duro para despliegue en HA

Supongamos que configura `sizing.profile` en uno de los perfiles de HA. Los valores de abajo configuran nodeAffinity para programar componentes de {stackstate-product-name} a la zona `us-west-2a` y forzar la programación de las réplicas de los mismos servicios de datos a diferentes hosts (`affinity.podAntiAffinity.requiredDuringSchedulingIgnoredDuringExecution`=`true` por defecto).
[,yaml]
----
affinity:
  # Schedule all pods to nodes in the same AZ
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
----

=== PodAntiAffinity suave para despliegue en HA
Si no desea imponer la programación de servicios de datos en diferentes nodos, puede configurar podAntiAffinity en modo "suave":
[,yaml]
----
affinity:
  podAntiAffinity:
    requiredDuringSchedulingIgnoredDuringExecution: false
----

== Utilización

[tabs]
====
Método recomendado::
+
--
[NOTE]
=====
El método de configuración `global.suseObservability` está disponible a partir de la versión `2.8.0`. Para versiones anteriores, utilice el método heredado.
=====

Añada la configuración de afinidad directamente a su archivo `values.yaml` en `global.suseObservability.affinity`:

[source,yaml]
----
global:
  suseObservability:
    # Your existing configuration
    license: "YOUR-LICENSE-KEY"
    baseUrl: "https://observability.example.com"
    sizing:
      profile: "150-ha"
    adminPassword: "your-password"

    # Affinity configuration
    affinity:
      # Node affinity - schedule all pods to nodes in the same AZ
      nodeAffinity:
        requiredDuringSchedulingIgnoredDuringExecution:
          nodeSelectorTerms:
          - matchExpressions:
            - key: topology.kubernetes.io/zone
              operator: In
              values:
              - us-west-2a

      # Pod anti-affinity for HA profiles (infrastructure components)
      podAntiAffinity:
        # true = hard anti-affinity, false = soft anti-affinity
        requiredDuringSchedulingIgnoredDuringExecution: true
        topologyKey: "kubernetes.io/hostname"
----

A continuación, instale o actualice con un solo comando:

[,bash]
----
helm upgrade --install \
  --namespace suse-observability \
  --create-namespace \
  --values values.yaml \
  suse-observability \
  suse-observability/suse-observability
----

TIP: En los perfiles de HA, la antiafinidad de pods se aplica automáticamente a todos los servicios de datos con estado. Sólo tiene que configurarlo si desea cambiar el comportamiento por defecto.

Para más detalles sobre la migración desde el método heredado, consulte la xref:/setup/upgrade-stackstate/migrate-to-global-mode.adoc[guía de migración].

==== Primer paso: Cree su archivo de valores de afinidad

Cree un archivo de valores independiente con la configuración de afinidad que desee. Por ejemplo, guarde lo siguiente como `suse-observability-values-values.yaml`:

[,yaml]
----
affinity:
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
  podAntiAffinity:
    requiredDuringSchedulingIgnoredDuringExecution: true
----

==== Segundo paso: Generar valores de plantilla de afinidad

Ejecute el siguiente comando para generar la plantilla de valores de afinidad:

[,text]
----
export VALUES_DIR=.
helm template \
  --set license='<your license>' \
  --set baseUrl='<suse-observability-base-url>' \
  --set sizing.profile='<sizing.profile>' \
  --values suse-observability-values-values.yaml \
  suse-observability-values \
  suse-observability/suse-observability-values --output-dir $VALUES_DIR
----

==== Tercer paso: Utilizar valores generados en la instalación de Helm

Incluya los valores de afinidad generados en su instalación de Helm:

[,bash]
----
helm upgrade \
  --install \
  --namespace suse-observability \
  --values $VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/sizing_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/affinity_values.yaml \
  suse-observability \
  suse-observability/suse-observability
----
