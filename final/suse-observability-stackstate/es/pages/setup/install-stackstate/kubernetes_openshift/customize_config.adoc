== Configuración personalizada para {stackstate-product-name} `api`

Para el servicio {stackstate-product-name} `api`, la configuración personalizada puede soltarse directamente en el gráfico de Helm. Esta es la forma recomendada de anular la configuración por defecto de {stackstate-product-name} y es especialmente conveniente para personalizar la autenticación. La configuración establecida de este modo estará disponible en el archivo de configuración {stackstate-product-name} en formato https://github.com/lightbend/config/blob/master/HOCON.md[] HOCON.

Por ejemplo, puede establecer un "enlace de olvido de contraseña" personalizado para la página de inicio de sesión {stackstate-product-name}:

[tabs]
====
values.yaml::
+
--

[,text]
----
stackstate:
  components:
    api:
      config: |
        stackstate.api.authentication.forgotPasswordLink =
        "https://www.stackstate.com/forgotPassword.html"
----

--
====

[NOTE]
====
Tenga en cuenta que la configuración personalizada establecida aquí será anulada por las xref:/setup/install-stackstate/kubernetes_openshift/customize_config.adoc#_environment_variables[variables de entorno].
====


== Variables de entorno

La configuración de todos los servicios de {stackstate-product-name} (`receiver`, `k2es-*`, `correlation` y `api`) puede personalizarse mediante variables de entorno. Las variables de entorno se especifican en el archivo `values.yaml` y pueden ser `secret` (como contraseñas) o `open` (para valores normales). Para convertir un elemento de configuración en un nombre de variable de entorno, sustituya `.` por `_` y añada el prefijo `CONFIG_FORCE_`.

[,text]
----
# configuration item
stackstate.api.authentication.forgotPasswordLink

# environment variable name
CONFIG_FORCE_stackstate_api_authentication_forgotPasswordLink
----

Por ejemplo, puede establecer un "enlace de olvido de contraseña" personalizado para la página de inicio de sesión {stackstate-product-name}:

[tabs]
====
values.yaml::
+
--

[,text]
----
stackstate:
  components:
    api:
      extraEnv:
        # The value for open env vars is defined on the deployment
        open:
          CONFIG_FORCE_stackstate_api_authentication_forgotPasswordLink: "https://www.stackstate.com/forgotPassword.html"
        # The value for secret env vars is defined in a secret and referenced from the deployment
        secret:
          CONFIG_FORCE_some_config_with_sensitive_data: "data-stored-in-secret"
----

--
====

[NOTE]
====
Para el servicio {stackstate-product-name} `api`, las variables de entorno anularán la <<_custom_configuration_for_suse_observability_api,configuración personalizada establecida mediante `config`>>.
====
