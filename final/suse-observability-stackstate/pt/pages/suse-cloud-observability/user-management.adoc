= Gerenciamento de usuários
:description: Gerenciamento de usuários do SUSE Cloud Observability

Os usuários dos locatários do SUSE Cloud Observability (instâncias do SUSE Cloud Observability) são gerenciados com o https://www.keycloak.org/[Keycloak]. Cada cliente (locatário) tem um domínio Keycloak dedicado. Um link para o console do Keycloak é enviado na mensagem de boas-vindas quando um usuário é criado.

O SUSE Cloud Observability redireciona os usuários para o Keycloak para autenticação. Espera-se que os usuários sejam membros de um ou mais grupos do Keycloak.

Os grupos predefinidos do Keycloak:

* *realm-admin*: Os membros desse grupo podem fazer login no console do Keycloak realm e realizar operações de gerenciamento de usuários.
* *stackstate-k8s-troubleshooter*: Os usuários desse grupo recebem a função de cliente `stackstate-k8s-troubleshooter` Keycloak, que é mapeada para a função SUSE Cloud Observability com o mesmo nome. A função concede permissões regulares do SUSE Cloud Observability.
* *stackstate-k8s-admin*: Os usuários desse grupo recebem a função de cliente `stackstate-k8s-admin` Keycloak, que é mapeada para a função SUSE Cloud Observability com o mesmo nome. A função concede permissões privilegiadas para o SUSE Cloud Observability.

== URL de gerenciamento de usuários

Quando um novo usuário é criado, ele recebe uma mensagem de boas-vindas com um link para a página de gerenciamento de usuários do Keycloak. Esse link é *exclusivo para o administrador do locatário*, que é o primeiro usuário por padrão. O formato do URL é: `https://<keycloak_FQDN>/realms/<your_dedicated_Keycloak_realm>/account`.

== Gerenciamento de usuários

* Faça login no console de administração do Keycloak.

image::keycloak_admin_console.png[Console de administração do Keycloak]

=== Gerenciar usuários

* No menu à esquerda, selecione `Users` na seção `Manage`.

==== Adição de um novo usuário

Clique no botão `Add user` e insira as informações do usuário, como "Username" (nome de usuário), "Email" (e-mail), "First Name" (nome) e "Last Name" (sobrenome).

. Deixe o campo `Required users actions` vazio.
. Adicione o usuário aos grupos necessários.
. Clique em `Save`.
+
A mensagem de boas-vindas com o link de inscrição e os links para o locatário do SUSE Cloud Observability e os consoles Keycloak Admin e Account são enviados por e-mail ao usuário.

NOTE: Para ativar a conta, que inclui a confirmação por e-mail e a redefinição de senha, o usuário deve seguir o link de inscrição.

image::keycloak_create_user.png[Keycloak Criar usuário]

==== Atualização dos detalhes do usuário

Para *editar os detalhes do usuário*, selecione o usuário clicando em Username (Nome de usuário).

. Altere os detalhes conforme necessário.
. Defina um ou mais `Required user actions`, por exemplo, para forçar os usuários a atualizar a senha ou configurar senhas únicas.
. Pressione o botão `Save` quando terminar.

image::keycloak_update_user.png[Usuário da atualização do Keycloak]

==== Exclusão de um usuário

. Para *excluir um ou mais usuários*, selecione os usuários necessários e pressione o botão `Delete user`.

=== Participação no grupo

. Faça login no console de administração do Keycloak.
. Na seção `Groups`, pesquise o grupo que deseja gerenciar.
. Clique no nome do grupo para abrir os detalhes do grupo e vá para a guia `Members`.
. Para adicionar um novo membro ao grupo, pressione o botão `Add Member` e selecione os usuários necessários.
. Para excluir usuários do grupo, selecione os usuários na lista e, em seguida, no menu que está na mesma linha do botão `Add member` marcado como "⋮", selecione `Leave group`.
