= Configuração de rede
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: Observabilidade do SUSE

O SUSE Observability é uma oferta de SaaS hospedada na nuvem. Para poder se comunicar de suas instalações/nuvem com o SUSE Observability SaaS, o SUSE Observability Agent precisa ser capaz de se conectar à API do SUSE Observability SaaS Receiver.
Quando o cluster estiver sendo executado em uma rede privada, talvez seja necessário configurar a rede para permitir que o SUSE Observability Agent se conecte à API do SUSE Observability Receiver, pois a configuração da rede pode não permitir o tráfego de saída para a Internet. Esta página descreve como configurar sua rede para permitir a instalação do SUSE Observability Agent, bem como para permitir que o SUSE Observability Agent se comunique com a API do SUSE Observability Receiver.

[NOTE]
====
O tráfego entre o SUSE Observability Agent e a API do SUSE Observability Receiver é sempre iniciado pelo SUSE Observability Agent. A API do SUSE Observability Receiver não inicia nenhum tráfego para o SUSE Observability Agent.
====


== Instalação do SUSE Observability Agent

A instalação do SUSE Observability Agent é feita por meio do Helm. Por padrão, o Helm Chart é configurado para extrair as imagens de contêiner do SUSE Observability Agent do registro de docas do Quay.io. Se a sua configuração de rede não permitir o tráfego de saída para a Internet, você terá várias opções para instalar o SUSE Observability Agent:

. Configure sua rede para permitir o tráfego de saída para o registro de contêineres do Quay.io a partir do seu cluster do Kubernetes.
. Faça proxy do registro de contêineres do Quay.io por meio do seu próprio registro de contêineres.
. Coloque as imagens do Docker em seu próprio registro de contêineres.

Para as opções 2 e 3, é necessário configurar o Helm Chart para extrair as imagens de contêiner do SUSE Observability Agent do seu próprio registro de contêineres. Um guia para configurar o SUSE Observability Agent Helm Chart para extrair imagens do seu próprio registro de contêiner pode ser encontrado xref:/k8s-suse-rancher-prime-agent-air-gapped.adoc[aqui].

== Comunicação do SUSE Observability Agent

O SUSE Observability Agent se comunica com a API do SUSE Observability Receiver por HTTPS. As diferentes partes do SUSE Observability Agent se conectam à API do SUSE Observability Receiver, hospedada em seu locatário, veja o diagrama a seguir:

image::k8s/k8s-agent-communication.png[Comunicação do SUSE Observability Agent]

Toda a comunicação é feita por HTTPS, usando a porta HTTPS padrão 443. O SUSE Observability Agent usa os seguintes pontos de extremidade para se comunicar com a API do SUSE Observability Receiver:

* https:// *<tenant>.app.stackstate.io/receiver/stsAgent* - o SUSE Observability Agent envia métricas, eventos e dados de topologia para a API do SUSE Observability Receiver.

Para permitir que o SUSE Observability Agent se comunique com a API do SUSE Observability Receiver, você precisa configurar sua rede para permitir o tráfego de saída para a API do SUSE Observability Receiver. A API do SUSE Observability Receiver é hospedada na nuvem e tem um IP específico para o seu locatário. Você precisa permitir o tráfego de saída para a Internet. Para obter os endereços IP corretos para permitir o tráfego de saída, você pode usar o seguinte comando:

[,bash]
----
$ dig +short <tenant>.app.stackstate.io
----

Como alternativa, você pode visitar o seguinte URL em seu navegador: `https://www.nslookup.io/domains/<tenant>.app.stackstate.io/dns-records/`
