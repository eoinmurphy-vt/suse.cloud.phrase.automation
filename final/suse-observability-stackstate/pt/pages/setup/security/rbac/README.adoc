= Controle de acesso baseado em função (RBAC)
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability Self-hosted

O Access Management ajuda a gerenciar quem tem acesso a elementos específicos da topologia, dados de métricas, rastreamentos, elementos da interface do usuário e quais APIs podem ser chamadas.

O RBAC é um sistema de autorização que fornece gerenciamento de acesso refinado dos recursos do SUSE Observability, uma maneira simples e fácil de auditar os privilégios do usuário e corrigir problemas identificados com direitos de acesso.

== O que você pode fazer com o RBAC?

Aqui estão alguns exemplos do que você pode fazer com o RBAC:

* Permita que um usuário tenha acesso apenas ao cluster de desenvolvimento, outro ao cluster de produção e ao de desenvolvimento e um terceiro possa acessar o cluster de desenvolvimento e apenas um namespace no cluster de produção.
* Dê a um pequeno grupo de usuários uma função de administrador para instalar e configurar o SUSE Observability. Ao mesmo tempo em que dá a todos os desenvolvedores uma função de solucionador de problemas para visualizar toda a topologia, métricas, rastreamentos, registros e eventos, mas com capacidade de configuração limitada.

== O que é uma função no SUSE Observability?

Uma função no SUSE Observability é uma combinação de um assunto configurado e um conjunto de xref:/setup/security/rbac/rbac_permissions.adoc[permissões]. O processo de configuração de uma função no SUSE Observability é descrito em xref:/setup/security/rbac/rbac_roles.adoc[Como configurar funções].

== Fontes de funções RBAC

Você pode definir funções de várias maneiras.  Todos usam as mesmas xref:/setup/security/rbac/rbac_permissions.adoc[permissões].

* **Configuração** - nos xref:setup/security/rbac/rbac_roles.adoc#_custom_roles_via_the_configuration_file[valores do leme] para o SUSE Observability
* **Dinâmico** - xref:setup/security/rbac/rbac_roles.adoc#_custom_roles_via_the_cli[use a CLI do `sts` ]
* **Mapeado do Kubernetes** - xref:/setup/security/rbac/rbac_rancher.adoc[mapeie o Kubernetes `Roles` e] xref:/setup/security/rbac/rbac_rancher.adoc[ `RoleBindings`]
