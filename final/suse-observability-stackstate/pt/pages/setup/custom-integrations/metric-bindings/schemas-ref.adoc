== Visão geral

Esta página descreve os esquemas de um `MetricBinding`, juntamente com explicações detalhadas de construções, sintaxe de expressão e semântica.

== Esquema para vinculação de métricas

Cada ligação métrica:

* Seleciona componentes onde pode ser aplicado
* Define modelos de consulta PromQL para recuperação de dados
* Especifica como os dados resultantes devem ser renderizados
* Inclui dicas de layout para selecionar um local apropriado na interface do usuário para mostrar o gráfico

[,yaml]
----
_type: "MetricBinding"
name: string
chartType: "line"                     # "line" is the only type for now
unit?: string
scope: string                         # Topology scope - components to bind to
enabled: boolean                      # default: true
description?: string
valuation?: "higher-is-better" | "lower-is-better"
priority?: "HIGH" | "MEDIUM" | "LOW" | "NONE"    # deprecated
queries:
  - _type: "MetricBindingQuery"
    expression: string                # promql query
    alias: string                     # name in legend
    componentIdentifierTemplate?: string    # URN template for linking
    primary?: boolean                 # is this query the primary one
tags:
  <key>: <value>
layout?:                              # where should chart be shown
  metricPerspective?:                 # the metrics perspective for a component
    tab: string
    section: string
    weight?: integer
  componentHighlight?:                # highlight perspective of a component
    section: string
    weight?: integer
  componentSummary?:                  # summary - supporting panel on the right
    weight?: integer
identifier?: string
----

* `_type`{stackstate-product-name} precisa saber que se trata de uma ligação métrica, portanto, o valor sempre precisa ser `MetricBinding`
* `name`: O nome da ligação métrica
* `chartType`{stackstate-product-name} será compatível com diferentes tipos de gráficos (`line`, `bar`, etc.), atualmente apenas `line` é compatível
* `unit`: A unidade dos valores na série temporal retornada pela consulta ou consultas, usada para renderizar o eixo Y do gráfico. Consulte a referência de xref:/develop/reference/k8sTs-chart-units.adoc[unidades suportadas] para todas as unidades
* `scope`: O escopo da topologia da vinculação de métrica, uma consulta de topologia que seleciona os componentes nos quais essa vinculação de métrica será exibida
* `enabled`: Defina como `false` para manter a vinculação da métrica, mas não mostrá-la aos usuários
* `description`: Descrição opcional, exibida ao lado do nome
* `valuation`: Se valores mais altos ou mais baixos são "melhores"
* `priority`: [Depreciado] Um dos sites `HIGH`, `MEDIUM`, ou `LOW`. A ordem de classificação principal para métricas em um componente (na ordem em que são mencionadas aqui), a ordem de classificação secundária é `name`.
* `queries`: Uma lista de consultas a serem exibidas no gráfico para a associação de métricas (consulte xref:/setup/custom-integrations/metric-bindings/writing-promql.adoc[Como escrever consultas PromQL])
** `expression`: A consulta promql (modelada)
** `alias`: Nome da consulta na legenda
** `componentIdentifierTemplate`: Modelo para identificador de componente relacionado, preenchido com rótulos de séries temporais resultantes da consulta
** `primary`: Essa consulta é a principal?
* `tags`: Será usado para organizar as métricas na interface do usuário; pode ser deixado vazio usando `{}`
* `layout`: Como agrupar gráficos em diferentes visualizações de perspectiva, por exemplo, na xref:/use/views/k8s-metrics-perspective.adoc[perspectiva Metrics]
** `metricPerspective` - Define as métricas a serem exibidas em `Metrics Perspective`. As métricas são agrupadas em guias e depois em seções.
*** `tab` - Nome da guia. As guias são classificadas em ordem alfabética
*** `section` - Nome da seção. As seções são classificadas em ordem alfabética
*** `weight` - As métricas em uma seção são classificadas principalmente por peso (ascendente) e, em segundo lugar, por nome (alfabético)
** `componentHighlight` - Define as métricas a serem exibidas em `Component Highlight`. As métricas são agrupadas em seções.
*** `section` - Nome da seção. As seções são classificadas em ordem alfabética
*** `weight` - As métricas em uma seção são classificadas principalmente por peso (ascendente) e, em segundo lugar, por nome (alfabético)
** `componentSummary` - Especifica as métricas a serem exibidas na barra lateral `Components details` na seleção do componente. Os gráficos são exibidos somente quando essa propriedade é definida.
*** `weight` - Isso representa o peso do gráfico. Os gráficos são classificados em ordem crescente por peso e, em seguida, exibem os três primeiros gráficos.
* `identifier`: Um URN (identificador universal de recursos), usado como identificador exclusivo da ligação métrica. Ele deve começar com `urn:stackpack:<stackpack-name>:metric-binding:`, o restante é de formato livre, desde que seja exclusivo entre todas as ligações métricas.
