== ComponentPresentation

O conceito central do modelo de apresentação da interface do usuário é o *ComponentPresentation*.

O site `ComponentPresentation` define como um ou mais componentes correspondentes são apresentados na interface do usuário, dependendo do contexto ativo.
Ele se concentra exclusivamente em *questões de apresentação*, como nomes, ícones, tabelas, páginas de visão geral e de destaque, filtros e posicionamento de itens de menu.

Em um alto nível, um ComponentPresentation:

* Seleciona componentes usando uma ligação (uma consulta STQL simples).
* Aplica-se em um modo específico (por exemplo, observabilidade).
* Define um ou mais aspectos de apresentação (como tabelas de visão geral ou destaques).
* Pode definir parcialmente as propriedades de apresentação e compor com outras definições correspondentes.

Várias ComponentPresentations podem se aplicar ao mesmo componente. Nesses casos, suas definições são combinadas para determinar a apresentação final.
