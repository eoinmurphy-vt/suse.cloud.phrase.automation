== ComponentPresentation

O site `ComponentPresentation` define como os componentes correspondentes são apresentados na interface do usuário em um determinado modo e contexto.

[,yaml]
----
_type: ComponentPresentation
identifier: urn:...:component-presentation:<presentation-name>
mode: ["observability" | "security" | "all"]

binding:
  _type: ComponentPresentationQueryBinding
  query: string                       # Primitive STQL query used to match components

rank:
  specificity: number                 # Higher values indicate more specific definitions

presentation:
  icon: string                        # Optional icon (base64-encoded)
  name:                               # Optional naming configuration
    plural: string
    singular: string
    title: string

  mainMenu:                           # Optional main menu entry
    group: string                     # Name of an existing MainMenuGroup
    rank: number                      # Optional override within the group

  overview:                           # Optional overview page configuration
    columns:
      - _type: string                 # ViewTypeTableCol*

  highlight:                          # Optional highlight page configuration
    summary:
      ...
    fields:
      - _type: string                 # ComponentTypeField
        fieldId: string
        label:
          title: string
          helpBubbleText: string      # Optional
        valueExtractor:
          _type: string
        display:
          _type: string
    about:
      _type: ComponentTypeAbout
      fields:
        - string

  filter:                             # Optional overview filters
    - _type: FilterField
      name: string
      tagField: string
----

== Identificador

Os identificadores devem seguir o formato do identificador {stackstate-product-name} (ou seja, `urn:...`). Consulte a documentação xref:/configure/topology/identifiers.adoc[dos identificadores] para obter mais informações.

== Encadernação

A vinculação determina a quais componentes um `ComponentPresentation` se aplica.

As associações são avaliadas em relação aos dados do componente e devem ser simples e eficientes, pois são usadas durante a avaliação da interface do usuário.

Atualmente compatível com o tipo `binding`:

* `ComponentPresentationQueryBinding`: uma consulta STQL (consulte o xref:/develop/reference/k8sTs-stql_reference.adoc[STQL] para obter mais informações)

== Classificação

A seção `rank` controla como várias definições correspondentes de `ComponentPresentation` são combinadas.

Valores mais altos de especificidade indicam definições mais especializadas e têm precedência sobre valores mais baixos ao compor o comportamento da apresentação.

== Seções de apresentação

Todos os campos em `presentation` são opcionais. O site `ComponentPresentation` pode definir apenas um subconjunto de aspectos de apresentação.

As seções de apresentação são compostas quando várias definições se aplicam ao mesmo componente.

=== Ícone e nome

Define a iconografia e a nomenclatura usadas na interface do usuário.

=== Menu principal

Controla se os componentes correspondentes aparecem no menu principal.

O grupo de menus referenciado deve ser definido separadamente por meio do site `MainMenuGroup`.

=== Visão geral

Define as colunas mostradas nas tabelas de síntese.

As colunas podem substituir ou estender colunas definidas por outras apresentações correspondentes.

=== Destaque

Define os campos e as seções mostrados nas páginas de destaque.

==== Resumo

==== Campos

==== Sobre

Define os campos mostrados no painel sobre na página de destaques do componente.

=== Filtro

Define os filtros exibidos nas páginas de visão geral.

Os filtros são exibidos quando existem dados correspondentes para os componentes vinculados.
