= Restaurar backups
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability Self-hosted

[NOTE]
====
Esta página se aplica a {stackstate-product-name} v2.7.0 ou mais recente.
====

== Visão geral

Esta página descreve como restaurar backups para armazenamentos de dados {stackstate-product-name} usando a CLI de backup.

[WARNING]
====
*A restauração a partir do backup eliminará ou sobrescreverá os dados existentes. Essa operação não pode ser desfeita.*

Sempre verifique o nome do backup e o namespace antes de confirmar uma operação de restauração.

*Fluxo de trabalho do GitOps afetado*Essa ferramenta de backup modifica os recursos do K8s diretamente em seu cluster:

- Dimensiona StatefulSets/Deployments para 0 réplicas
- Adiciona anotações para rastreamento de restauração

Essas alterações entrarão em conflito com seu fluxo de trabalho do GitOps, pois ignoram a reconciliação baseada no Git. Os controladores do GitOps podem tentar reverter essas alterações durante as operações de backup.
*Importante:* A reconciliação automática para a implantação do SUSE Observability Helm chart deve ser desativada durante a restauração do backup para evitar conflitos.
====

[NOTE]
====
*Antes de usar a CLI, certifique-se de que:*

* Você está usando a versão mais recente da CLI de backup.
* A CLI espera um kubeconfig com um contexto atual para o cluster em que o SUSE Observability está instalado.
====

== Faça o download da CLI de backup

Faça o download da versão mais recente da CLI de backup usando os comandos abaixo para sua plataforma. O binário é denominado `sts-backup`.

[tabs]
====

+
--

[,bash]
----

curl -LO "https://github.com/StackVista/stackstate-backup-cli/releases/download/${VERSION}/stackstate-backup-cli-${VERSION#v}.darwin-arm64.tar.gz"
tar -xzf stackstate-backup-cli-${VERSION#v}.darwin-arm64.tar.gz

== Dados de configuração e topologia (StackGraph)

=== Lista de backups do StackGraph

Para listar os backups do StackGraph, execute o seguinte comando:

[,bash]
----
sts-backup stackgraph list --namespace <NAMESPACE>
----

Substitua `<NAMESPACE>` pelo namespace em que {stackstate-product-name} está instalado.

O resultado deve ser semelhante a este:

[,bash]
----
Setting up port-forward to suse-observability-minio:9000 in namespace <NAMESPACE>...
✓ Port-forward established successfully
Listing Stackgraph backups in bucket 'sts-stackgraph-backup'...
NAME                            LAST MODIFIED            SIZE
sts-backup-20251128-0300.graph  2025-11-28 03:08:42 UTC  2GiB
----

O registro de data e hora em que o backup é feito faz parte do nome do backup.


=== Restaurar um backup do StackGraph

Para restaurar um backup do StackGraph, use uma das abordagens a seguir:

==== Restaurar um backup específico

[,bash]
----
sts-backup stackgraph restore --namespace <NAMESPACE> --archive <BACKUP_NAME>
----

==== Restaurar o backup mais recente

[,bash]
----
sts-backup stackgraph restore --namespace <NAMESPACE> --latest
----

==== Bandeiras comuns

* `--yes` ou `-y` - Ignorar prompt de confirmação (útil para automação)
* `--background` - Executar a restauração em segundo plano sem aguardar a conclusão

==== Restauração de histórico

Ao usar `--background`, a restauração é executada de forma assíncrona. Depois de iniciar a restauração, use o seguinte comando para verificar o status e finalizar:

[,bash]
----
sts-backup stackgraph check-and-finalize --job <JOB_NAME> --wait --namespace <NAMESPACE>
----

O comando `check-and-finalize`:

* Verifica o status do trabalho de restauração
* Com o sinalizador `--wait`, aguarda a conclusão do trabalho
* Amplia automaticamente as implantações que foram reduzidas durante a restauração
* Limpa os recursos após a conclusão

[NOTE]
====
Se uma restauração executada sem o site `--background` for interrompida (por exemplo, por Ctrl+C), você deverá executar o site `check-and-finalize` para aumentar a escala das implantações e limpar os recursos.
====

== Configurações

Os backups de configurações incluem StackPacks instalados com suas configurações e outras personalizações criadas pelo usuário, como monitores, exibições personalizadas e tokens de serviço. Os backups de configurações são leves (normalmente apenas alguns megabytes) e de rápida restauração, com tempo mínimo de inatividade.

=== Listar backups de configurações

Para listar os backups das configurações, execute o seguinte comando:

[,bash]
----
sts-backup settings list --namespace <NAMESPACE>
----

Substitua `<NAMESPACE>` pelo namespace em que o SUSE Observability está instalado.

O resultado deve ser semelhante a este:

[,bash]
----
Setting up port-forward to suse-observability-minio:9000 in namespace <NAMESPACE>...
✓ Port-forward established successfully
Listing Settings backups in bucket 'sts-configuration-backup'...
NAME                          LAST MODIFIED            SIZE
sts-backup-20251128-1328.sty  2025-11-28 13:29:12 UTC  2MiB
----

O registro de data e hora em que o backup foi feito faz parte do nome do backup.

=== Restaurar um backup das configurações

[CAUTION]
====
A restauração de um backup de configurações também removerá toda a topologia, inclusive os estados de integridade, os alertas e o histórico da topologia.
====

Para restaurar um backup de configurações, use uma das abordagens a seguir:

==== Restaurar um backup específico

[,bash]
----
sts-backup settings restore --namespace <NAMESPACE> --archive <BACKUP_NAME>
----

==== Restaurar o backup mais recente

[,bash]
----
sts-backup settings restore --namespace <NAMESPACE> --latest
----

==== Bandeiras comuns

* `--yes` ou `-y` - Ignorar prompt de confirmação (útil para automação)

== Métricas (Victoria Metrics)

Dependendo do perfil, `nonha` ou `ha`, o Victoria Metrics é implantado em diferentes modos:

* *perfil nonha* - Modo de nó único com uma instância do Victoria Metrics (`victoria-metrics-0`)
* *perfil ha* - modo HA (espelho) com duas instâncias do Victoria Metrics (`victoria-metrics-0` e `victoria-metrics-1`)

=== Lista de backups do Victoria Metrics

Para listar os backups do Victoria Metrics, execute o seguinte comando:

[,bash]
----
sts-backup victoria-metrics list --namespace <NAMESPACE>
----

Substitua `<NAMESPACE>` pelo namespace em que o SUSE Observability está instalado.

==== Saída de modo de nó único (perfil nonha)

[,bash]
----
Setting up port-forward to suse-observability-minio:9000 in namespace <NAMESPACE>...
✓ Port-forward established successfully
Listing VictoriaMetrics backups in bucket ...
NAME ({bucket}/{instance}-{created})                           UPDATED
sts-victoria-metrics-backup/victoria-metrics-0-20251030152500  2025-11-28 09:25:05 UTC
----

==== Saída do modo HA (perfil ha)

[,bash]
----
Setting up port-forward to suse-observability-minio:9000 in namespace <NAMESPACE>...
✓ Port-forward established successfully
Listing VictoriaMetrics backups in bucket ...
NAME ({bucket}/{instance}-{created})                           UPDATED
sts-victoria-metrics-backup/victoria-metrics-1-20251030152500  2025-11-28 09:35:08 UTC
sts-victoria-metrics-backup/victoria-metrics-0-20251030152500  2025-11-28 09:25:04 UTC

NOTE: In HA mode, backups from both instances (victoria-metrics-0 and victoria-metrics-1) are listed.
      The restore command accepts either backup to restore both instances.
----

No modo HA, os backups são criados para ambas as instâncias com prefixos diferentes (`victoria-metrics-0` e `victoria-metrics-1`). Ao restaurar, você pode especificar qualquer backup - a operação de restauração restaurará o backup selecionado em ambas as instâncias.

=== Restaurar um backup do Victoria Metrics

[NOTE]
====
Todas as novas métricas serão armazenadas em cache pelo site `vmagent` durante o processo de restauração. Certifique-se de que o site `vmagent` tenha memória suficiente para armazenar as métricas em cache.
====

Para restaurar um backup do Victoria Metrics, use uma das abordagens a seguir:

==== Restaurar um backup específico

[,bash]
----

----

==== Restaurar o backup mais recente

[,bash]
----

----

==== Bandeiras comuns

* `--yes` ou `-y` - Ignorar prompt de confirmação (útil para automação)
* `--background` - Executar a restauração em segundo plano sem aguardar a conclusão

==== Restauração de histórico

Ao usar `--background`, a restauração é executada de forma assíncrona. Depois de iniciar a restauração, use o seguinte comando para verificar o status e finalizar:

[,bash]
----
sts-backup victoria-metrics check-and-finalize --job <JOB_NAME> --wait --namespace <NAMESPACE>
----

O comando `check-and-finalize`:

* Verifica o status do trabalho de restauração
* Com o sinalizador `--wait`, aguarda a conclusão do trabalho
* Aumenta automaticamente a escala dos StatefulSets que foram reduzidos durante a restauração
* Limpa os recursos após a conclusão

[NOTE]
====
Se uma restauração em execução sem o site `--background` for interrompida (por exemplo, por Ctrl+C), você deverá executar o site `check-and-finalize` para aumentar a escala dos StatefulSets e limpar os recursos.

== OpenTelemetry (ClickHouse)

=== Listar backups do ClickHouse

Para listar os backups do ClickHouse, execute o seguinte comando:

[,bash]
----
sts-backup clickhouse list --namespace <NAMESPACE>
----

Substitua `<NAMESPACE>` pelo namespace em que o SUSE Observability está instalado.

O resultado deve ser semelhante a este:

[,bash]
----
Setting up port-forward to suse-observability-clickhouse-backup:7171 in namespace <NAMESPACE>...
✓ Port-forward established successfully
Listing Clickhouse backups...
NAME                             CREATED              SIZE
incremental_2025-11-28T09-45-00  2025-11-28 09:45:03  65MiB
incremental_2025-11-28T08-45-00  2025-11-28 08:45:03  223MiB
full_2025-11-28T00-45-00         2025-11-28 00:45:03  3GiB
incremental_2025-11-27T23-45-00  2025-11-27 23:45:03  118MiB
----

Os nomes de backup que começam com `full_` são backups completos, enquanto os nomes que começam com `incremental_` são backups incrementais.

=== Restaurar um backup do ClickHouse

[NOTE]
====
O processo de restauração reduz automaticamente as cargas de trabalho que produzem dados (como os exportadores do OpenTelemetry) para evitar a perda de dados durante a restauração.
====

Para restaurar um backup do ClickHouse, use uma das abordagens a seguir:

==== Restaurar um backup específico

[,bash]
----
sts-backup clickhouse restore --namespace <NAMESPACE> --snapshot <BACKUP_NAME>
----

==== Restaurar o backup mais recente

[,bash]
----
sts-backup clickhouse restore --namespace <NAMESPACE> --latest
----

==== Bandeiras comuns

* `--yes` ou `-y` - Ignorar prompt de confirmação (útil para automação)
* `--background` - Executar a restauração em segundo plano sem aguardar a conclusão

==== Restauração de histórico

Ao usar `--background`, a restauração é executada de forma assíncrona. Depois de iniciar a restauração, use o seguinte comando para verificar o status e finalizar:

[,bash]
----
sts-backup clickhouse check-and-finalize --operation-id <OPERATION_ID> --wait --namespace <NAMESPACE>
----

O comando `check-and-finalize`:

* Verifica o status da operação de restauração
* Com o sinalizador `--wait`, aguarda a conclusão da operação
* Executa comandos SQL pós-restauração
* Aumenta automaticamente a escala dos StatefulSets que foram reduzidos durante a restauração
* Limpa os recursos após a conclusão

[NOTE]
====
Se uma restauração em execução sem o site `--background` for interrompida (por exemplo, por Ctrl+C), você deverá executar o site `check-and-finalize` com o ID da operação para aumentar a escala dos StatefulSets e limpar os recursos.
====

[NOTE]
====
