=== Requisitos de recursos

Há diferentes opções de instalação disponíveis para o SUSE Observability. É possível instalar o SUSE Observability em uma configuração de alta disponibilidade (HA) ou de instância única (não HA). A configuração sem HA é recomendada apenas para fins de teste. Para ambientes de produção, recomenda-se instalar o SUSE Observability em uma configuração de HA. Para uma implantação padrão, de produção, o gráfico do SUSE Observability Helm implantará muitos serviços em uma configuração redundante com 3 instâncias de cada serviço.

Na tabela abaixo, você pode encontrar os requisitos de recursos para as diferentes opções de instalação. Para a configuração do HA, você pode encontrar diferentes perfis de instalação, dependendo do tamanho do ambiente que está sendo observado.
Um nó observado nessa tabela de dimensionamento é considerado como tendo 4 vCPUs e 16 GB de memória, nosso `default node size`.
Se os nós em seu cluster observado forem maiores, eles poderão contar para vários `default nodes`, portanto, um nó de 12vCPU e 48GB conta como 3 `default nodes` em observação ao escolher um perfil.

[NOTE]
====
Os requisitos mostrados para o perfil representam a quantidade total de recursos necessários para executar o servidor Suse Observability.
Para garantir que todos os diferentes serviços do servidor Suse Observability possam ser alocados:

* Para instalações não HA, o tamanho recomendado do nó é 4VCPU, 8GB
* Para instalações de HA de até 500 nós, o tamanho mínimo recomendado do nó é 8VCPU, 16GB
* Para instalações de HA de 4000 nós, o tamanho mínimo recomendado do nó é 16VCPU, 32GB
====


Esses são apenas os limites superior e inferior dos recursos que podem ser consumidos pelo SUSE Observability nas diferentes opções de instalação. O uso real dos recursos dependerá dos recursos usados, dos limites de recursos configurados e dos padrões de uso dinâmico, como o dimensionamento do Deployment ou do DaemonSet. Para nossos clientes auto-hospedados, recomendamos começar com os requisitos padrão e monitorar o uso de recursos dos componentes do SUSE Observability.

[NOTE]
====
Os requisitos mínimos não incluem capacidade sobressalente de CPU/memória para garantir atualizações contínuas do aplicativo.
====

=== Armazenamento

O SUSE Observability usa declarações de volume persistente para os serviços que precisam armazenar dados. A classe de armazenamento padrão do cluster será usada para todos os serviços, a menos que seja substituída por valores especificados na linha de comando ou em um arquivo `values.yaml`. Todos os serviços vêm com um tamanho de volume pré-configurado que deve ser bom para você começar, mas pode ser personalizado posteriormente usando variáveis, conforme necessário.

[NOTE]
====
Para ambientes de produção, o NFS não é recomendado nem suportado para provisionamento de armazenamento no SUSE Observability devido ao risco potencial de corrupção de dados.
====


Para nossos diferentes perfis de instalação, os requisitos de armazenamento padrão são os seguintes:

=== Limites de recursos do namespace

Não é recomendável definir uma ResourceQuota, pois isso pode interferir nas solicitações de recursos. Os recursos exigidos pelo SUSE Observability variam de acordo com os recursos usados, os limites de recursos configurados e os padrões de uso dinâmico, como o dimensionamento do Deployment ou do DaemonSet.

Se for necessário definir uma ResourceQuota para sua implementação, o limite de recursos do namespace deverá ser definido para corresponder aos xref:/setup/install-stackstate/requirements.adoc#_resource_requirements[requisitos de dimensionamento do] nó.
