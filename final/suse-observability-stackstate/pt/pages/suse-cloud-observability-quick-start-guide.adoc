== Visão geral

Após a compra do SUSE Cloud Observability no Cloud Provider Marketplace, seu ambiente do SUSE Cloud Observability é provisionado.

Você receberá um e-mail do SUSE Cloud Observability com os detalhes de login necessários e links para o seu ambiente. Este guia de início rápido o ajudará a começar e a colocar seus próprios dados na implantação do SUSE Cloud Observability.

== Introdução - Primeiros passos

. Configurar uma senha
. Faça login na sua instância do SUSE Cloud Observability
. Instale o agente do SUSE Observability em seu cluster
. Alterar os detalhes de sua conta pessoal

=== Configurar uma senha

O e-mail do SUSE Cloud Observability contém um link exclusivo que permite que você defina sua senha inicial na conta.  Isso deve ser feito antes que você possa fazer login e configurar o ambiente de observabilidade.

=== Acesso ao ambiente do SUSE Observability

Faça login no seu ambiente do SUSE Cloud Observability clicando no link exclusivo no e-mail que você recebeu do SUSE Cloud Observability.  Ao digitar a senha, você será levado à tela de configuração para adicionar clusters.

ifdef::feature-agent-install-cluster[]
=== Instale o agente do SUSE Observability em seu cluster

O SUSE Cloud Observability usa StackPacks para facilitar a configuração de seus clusters downstream e a obtenção de dados no ambiente de observabilidade.

Para chegar até aqui, na interface do usuário do SUSE Observability, abra o menu principal clicando no canto superior esquerdo da tela e acesse `StackPacks` > `Kubernetes`.

image::integrating_first_cluster_stackpacks.png[Observabilidade da nuvem SUSE - StackPacks]

Quando estiver na tela do Kubernetes StackPack, é muito simples adicionar um cluster ao ambiente de observabilidade.

Digite um nome para o cluster que você deseja observar; ele não precisa corresponder ao nome do cluster usado no "kubeconfig".

O nome do cluster do Kubernetes deve começar e terminar com uma letra minúscula ou dígito e pode consistir apenas em letras minúsculas, dígitos, pontos e traços (. -)

Digite o nome e clique no botão `Install`.  Isso deve levá-lo à tela a seguir, com o cluster marcado como "Waiting for data" (Aguardando dados)

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - Adicionando um cluster]

Clique no nome do cluster para expandir essa seção e revelar uma série de informações, incluindo pré-requisitos e comandos que podem ser usados nos clusters existentes para adicioná-los ao seu ambiente de observabilidade.

Revise as informações e verifique se você tem as permissões corretas para o seu ambiente do Kubernetes e se está executando uma versão compatível do Kubernetes.

Esses comandos são exclusivos de sua implantação de observabilidade e incluem as chaves de API e os URLs necessários.  Selecione os comandos apropriados para o seu cluster; há seções para EKS, RKE, cluster genérico do Kubnernetes e muitos outros.

Esses comandos instalarão os agentes do SUSE Cloud Obervability e conectarão o cluster ao seu ambiente do SUSE Cloud Observability.

Você pode usar os comandos diretamente da interface do usuário ou pode implantar nos clusters downstream seguindo os comandos fornecidos na seção "Implantar o StackState Agent e o Cluster Agent" no xref:/k8s-quick-start-guide.adoc[guia de início rápido]

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - Pós-instalação]

Depois que o cluster tiver sido conectado, deverá haver uma marca verde na interface do usuário do SUSE Cloud Observability.

Nesse ponto, você pode começar a explorar seus dados.
endif::feature-agent-install-cluster[]

=== Alterar os detalhes de sua conta pessoal

A etapa 4 do seu e-mail é atualizar suas informações pessoais.  Clique no link exclusivo do e-mail e adicione seus dados pessoais básicos, conforme necessário.  Também é possível configurar a autenticação 2FA nessa seção, se necessário.

=== Explore seus dados

Para começar a explorar seus dados, abra o menu principal clicando no canto superior esquerdo da tela e acesse `Kubernetes` para revelar uma lista de itens observáveis.

image::accessing_views_1.png[SUSE Cloud Observability - Explorando seus dados]

Selecione "Clusters" na seção de infraestrutura, que deve mostrar uma lista de clusters monitorados. Selecione seu cluster para revelar uma das muitas exibições integradas.

image::accessing_views_2.png[SUSE Cloud Observability - Visualização do cluster]

Nesse ponto, você pode começar a explorar os dados do seu cluster ou adicionar mais clusters para coletar dados.

Para obter mais informações sobre como usar o SUSE Cloud Observability, incluindo a criação de exibições personalizadas, consulte a documentação padrão.

=== Limitações de observabilidade do SUSE Cloud

Observe que o SUSE Cloud Observability não fornece sinais RED (taxa, erros e duração) prontos para uso; apenas o sinal de taxa é mostrado.  Esse recurso está disponível com o SUSE Rancher Prime.
Os clientes que precisam desses sinais para ter uma solução completa de Observabilidade devem entrar em contato com a SUSE para discutir o SUSE Rancher Prime. Como alternativa, esses dados podem ser coletados usando os coletores OpenTelemetry.

== Informações adicionais

Para obter mais informações, incluindo pré-requisitos e versões do Kubernetes compatíveis com sua plataforma, consulte o xref:/k8s-quick-start-guide.adoc[guia de início rápido].
