== Visão geral

O Kubernetes mantém registros abrangentes da atividade do cluster e da saída do aplicativo. O registro em log fornece informações valiosas sobre o desempenho de contêineres, nós e do próprio Kubernetes. Registrando meticulosamente tudo, de Pods a ReplicaSets, ele permite rastrear os problemas até a sua origem.

== Acesso a registros de pods

Os erros e avisos nos logs são indicadores de possíveis problemas no aplicativo, nos contêineres ou no próprio pod. Para explorar os logs na tela do pod, clique no botão *Mostrar logs*.

== Uso de registros de pods

Você pode usar a pesquisa para procurar erros ou avisos específicos nos registros. Os registros podem ser filtrados por um contêiner. Você pode selecionar o intervalo de tempo desejado para a exibição dos registros e um intervalo de atualização para a frequência com que deseja que os registros sejam atualizados.

O histograma de registros visualiza o número de linhas de registro em um intervalo de tempo selecionado e permite que você amplie um intervalo específico clicando e arrastando.

Use essas opções para mostrar ou ocultar informações adicionais de cada linha de registro:

* *Mostrar hora* para mostrar registros de data e hora
* *Mostrar contêineres* para mostrar o nome do contêiner ao qual a linha de registro pertence
* *Enrolar linhas* para permitir o enrolamento de linhas longas
* *Newest first/Oldest first* para alternar a ordem de classificação do registro
