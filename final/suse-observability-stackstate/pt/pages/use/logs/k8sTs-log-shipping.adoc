== Instalação do agente

=== Openshift

Os expedidores de logs de terceiros não são prontamente compatíveis com o Openshift; a plataforma tem opções para encaminhamento de logs, que são usadas em vez da configuração de promtail incluída no gráfico de leme do SUSE Observability.  Para obter instruções detalhadas sobre como configurar isso, consulte a documentação do stackpack do Kubernetes em sua instância do SUSE Observability em execução.

=== Kubernetes

A configuração padrão do helm chart do SUSE Observability k8s Agent define o envio de registros como ativado por meio dos valores de helm fornecidos com o chart:

[,yaml]
----
logsAgent:
  # logsAgent.enabled -- Enable / disable k8s pod log collection
  enabled: true
----

O procedimento acima garantirá que um contêiner promtail seja implantado em cada nó para coletar logs e enviá-los ao SUSE Observability.  Para implantações em que não é desejável enviar registros para o SUSE Observability, defina o valor acima como `false`.

=== Execução de pods Promtail adicionais

O SUSE Observability usa uma configuração ajustada para a ingestão de registros, e isso geralmente não está de acordo com os requisitos auxiliares.  Portanto, não é possível executar uma configuração separada para ingestão de logs em outros pontos de extremidade de destino. Em vez disso, recomenda-se executar um segundo pod promtail que lide com esses requisitos como uma preocupação separada do promtail implantado pelo helm chart do agente.
