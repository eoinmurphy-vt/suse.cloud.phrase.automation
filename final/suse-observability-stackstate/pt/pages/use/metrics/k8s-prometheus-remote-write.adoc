= Prometheus remote_write
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: Observabilidade do SUSE

Quando você tem seu próprio Prometheus local ou auto-hospedado em execução, no qual as métricas do seu ambiente são agregadas, é possível espelhar essas métricas no SUSE Observability. Isso permitirá que você use os recursos avançados de topologia e correlação do SUSE Observability para solucionar problemas do seu ambiente Kubernetes sem precisar alternar entre as ferramentas.

Para tornar isso possível, o SUSE Observability expõe o protocolo Prometheus `remote_write` como um ponto de extremidade. Usando esse endpoint, você pode configurar sua instância do Prometheus para enviar métricas para o SUSE Observability. As métricas são então automaticamente ingeridas, e será possível vincular as métricas aos componentes observados pelo SUSE Observability. O diagrama a seguir mostra como isso funciona:

image::k8s/k8s-prometheus-remotewrite.png[Gravação remota do Prometheus]

== Pré-requisitos

Para espelhar suas métricas do Prometheus no SUSE Observability, você precisa procurar a chave de API usada para enviar métricas para o SUSE Observability. A chave de API pode ser encontrada na descrição do Kubernetes StackPack instalado no SUSE Observability. As etapas a seguir mostram como encontrar a chave de API:

. Abra a interface do usuário do SUSE Observability e navegue até a página StackPacks
+
image::k8s/k8s-stackpacks.png[Empilhamentos]

. Localize o Kubernetes StackPack e clique nele.
. Na descrição do StackPack, você encontrará a chave de API que é usada para enviar métricas para o SUSE Observability.
+
image::k8s/k8s-stackpacks-apikey.png[Chave da API]

== Configuração de sua instância do Prometheus

Para configurar sua instância do Prometheus para enviar métricas ao SUSE Observability, você precisa adicionar um novo endpoint de gravação remota à sua configuração do Prometheus. Isso pode ser feito atualizando a seção `remote_write` no seu arquivo `prometheus.yml`. Há duas variantes que você pode usar para a autenticação no endpoint, seja por meio de um cabeçalho ou de autenticação básica.

=== Autenticação de cabeçalho

Você pode se autenticar usando a chave de API adicionando o cabeçalho `sts-api-key` às solicitações remote_write do Prometheus. O exemplo a seguir mostra como configurar isso:

[,yaml]
----
remote_write:
- url: https://<tenant>.app.stackstate.io/receiver/prometheus/api/v1/write
  headers:
    sts-api-key: <API Key>
----

=== Autenticação básica

É possível autenticar usando a autenticação básica adicionando a seção `basic_auth` às solicitações remote_write do Prometheus. Como as solicitações a essa API são feitas de um serviço sem cabeçalho, o campo de nome de usuário é definido com o valor estático `apikey`. O exemplo a seguir mostra como configurar isso:

[,yaml]
----
remote_write:
- url: https://<tenant>.app.stackstate.io/receiver/prometheus/api/v1/write
  basic_auth:
    username: apikey
    password: <API Key>
----

== Finalização

Após as alterações de configuração terem sido aplicadas ao arquivo de configuração do Prometheus, o Prometheus precisa ser reiniciado. Após a reinicialização, o Prometheus começará a enviar métricas para o SUSE Observability. As métricas serão automaticamente correlacionadas com o restante de seu ambiente e ficarão visíveis na interface do usuário do SUSE Observability.
