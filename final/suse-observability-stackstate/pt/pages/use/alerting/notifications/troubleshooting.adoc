= Notificações de solução de problemas
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: Observabilidade do SUSE

Quando um canal de notificação não conseguir enviar notificações, a visão geral da notificação mostrará um status de erro. Para inspecionar os erros:

. Clique na notificação
. A parte superior da página mostra um banner com um resumo dos erros
. Role para baixo até os canais, clique no link "erro" do(s) canal(is) afetado(s) para abrir os detalhes do erro.

== Erros comuns

=== Slack

Os erros do Slack geralmente contêm um código de erro:

* `token_revoked`, `token_expired` ou `missing_scope`: Todos esses códigos de erro indicam um problema com a autorização do SUSE Observability para publicar mensagens no Slack. Para resolver isso, recrie o canal para autorizar novamente o SUSE Observability com o Slack, de modo que todas as permissões necessárias sejam concedidas.
* Para outros códigos de erro, consulte https://api.slack.com/automation/cli/errors[Códigos de erro da API do Slack].

=== HTTP

A maioria dos canais, especificamente o webhook, espera uma resposta HTTP com status 200 OK. Outras respostas são consideradas um erro. Os erros de HTTP contêm uma mensagem, geralmente curta, e o código de status que foi recebido em vez de 200 OK.

Verifique se todos os URLs e tokens configurados estão corretos e, no caso do webhook, se as solicitações POST são tratadas corretamente e retornam uma resposta 200 OK.

=== Outros

Caso o erro seja do tipo "Outro", o SUSE Observability provavelmente não conseguirá nem mesmo se conectar ao serviço externo ou ao webhook. Verifique se o URL está correto, se pode ser resolvido e se pode ser acessado pelo SUSE Observability.
