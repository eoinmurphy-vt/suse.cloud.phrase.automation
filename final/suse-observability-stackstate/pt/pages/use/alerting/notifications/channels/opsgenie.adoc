== Configurar as notificações do Opsgenie

Para enviar notificações ao Opsgenie, siga estas etapas:

. <<_create_integration,Criar uma integração no OpsGenie>>
. <<_create_and_test_the_channel,Criar o canal>>

=== Criar integração

Para integrar o SUSE Observability ao OpsGenie, é necessário criar uma "chave de API" global.  Os respondentes de uma notificação podem ser diferentes para cada notificação e serão configurados no SUSE Observability.

No Opsgenie, vá para "Settings \-> Integrations" (Configurações \-> Integrações).

* Clique no botão "Add integration" (Adicionar integração)
* Procure o tipo de integração "API".
* Escolha um nome para essa integração, mas _não selecione uma equipe de responsáveis_.

Quando a integração tiver sido criada, ela terá uma "chave de API".

=== Criar e testar o canal

image::k8s/notifications-opsgenie-channel-configuration.png[Criar o canal]

Selecione a região apropriada para sua conta do OpsGenie e insira a "API key" (chave de API).  Escolha os respondentes para a notificação; _usuários_, _equipes_, _cronogramas_e _escalonamentos_ estão disponíveis para isso.

Clique no botão "Add channel" (Adicionar canal). Isso adiciona o canal à lista de canais à direita. Ele mostrará um botão "Test" (Teste). Pressionar esse botão gerará um alerta de teste no OpsGenie.

== Alertas do OpsGenie para notificações

Quando uma notificação é aberta, um novo alerta do OpsGenie é criado. Essa mensagem será atualizada para alterações, geralmente apenas quando o estado de saúde for alterado. Quando a notificação é fechada, o alerta do OpsGenie é fechado novamente.  Uma nota sobre o alerta contém o motivo do fechamento.
