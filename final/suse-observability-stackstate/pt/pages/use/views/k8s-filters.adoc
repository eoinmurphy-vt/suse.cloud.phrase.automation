== Visão geral

O menu *Filtros*, no canto superior direito da interface do usuário da visualização, permite filtrar os componentes (topologia) e os eventos exibidos em uma visualização. Uma vez aplicados, os filtros afetarão o conteúdo de todas as perspectivas em uma visualização.

image::k8s/k8s-filters.png[Exibir filtros]

== Topologia de filtro

Os filtros de topologia podem ser usados para selecionar um subconjunto de componentes de topologia a ser mostrado em qualquer uma das perspectivas disponíveis. Embora o filtro de eventos seja o mesmo para todos os tipos de visualização, os filtros de topologia dependem do tipo de visualização em que você está. Leia mais:

* xref:/use/views/k8s-filters.adoc#_topology_filters_on_kubernetes_views[Filtros de topologia nas exibições do Kubernetes]
* xref:/use/views/k8s-filters.adoc#_topology_filters_on_other_view_types[Filtros de topologia em outros tipos de visualização]

=== Filtros de topologia nas exibições do Kubernetes

Nas xref:/use/views/k8s-views.adoc[exibições do Kubernetes], os filtros de topologia são limitados a um pequeno conjunto de filtros básicos que persistem em todas as exibições do Kubernetes: `clusters` e/ou `namespaces`. Os filtros de topologia persistentes para exibições do Kubernetes são colocados fora do menu regular `filters` na interface do usuário e não são interdependentes entre si (por exemplo, a seleção de um cluster *não* atualiza automaticamente o filtro `namespaces` para refletir apenas os namespaces desse cluster).

image::k8s/k8s-filters-kube-topology.png[Filtros de topologia do Kubernetes]

=== Filtros de topologia em outros tipos de visualização

Para outros tipos de visualização, você pode navegar em sua topologia usando filtros de topologia básicos ou avançados. Leia mais sobre:

* xref:/use/views/k8s-filters.adoc#_basic_topology_filters[Filtros básicos de topologia]
* xref:/use/views/k8s-filters.adoc#_advanced_topology_filters[Filtros de topologia avançada]
* xref:/use/views/k8s-filters.adoc#_topology_filtering_limits[Limites de filtragem de topologia]

==== Filtros básicos de topologia

A principal maneira de filtrar a topologia é usar os filtros básicos disponíveis. Quando você define um filtro, a perspectiva aberta é atualizada para mostrar apenas a visualização ou os dados do subconjunto de sua topologia que corresponde ao filtro. A definição de vários filtros restringirá ainda mais sua pesquisa. Você pode definir mais de um valor para cada filtro para expandir sua pesquisa

|===
| Filtro | Descrição

| Camadas, domínios, ambientes e tipos de componentes
| Filtrar pelos detalhes do componente incluídos quando os componentes são importados ou criados.

| Saúde dos componentes
| Inclua apenas componentes com o estado de integridade nomeado, conforme relatado pela verificação de integridade associada.

| Rótulos de componentes
| Inclua apenas componentes com um rótulo específico.

| Incluir componentes
| Os componentes nomeados aqui serão incluídos na topologia *, além dos* componentes retornados de outros filtros.
|===

Para filtrar a topologia usando filtros básicos, clique no menu *Filtros* no canto superior direito da interface do usuário e selecione *Alternar para básico* na guia vertical *Topologia*.

image::k8s/k8s-filters-basic-topology.png[Filtros básicos de topologia]

==== Filtros de topologia avançada

Você pode usar a xref:/develop/reference/k8sTs-stql_reference.adoc[SUSE Observability Query Language (STQL)] incorporada para criar um filtro de topologia avançado que amplie uma área específica da topologia.

Para filtrar a topologia usando uma consulta STQL, clique no menu *Filtros* no canto superior direito da interface do usuário e selecione *Alternar para STQL* na guia vertical *Topologia*.

O exemplo de consulta STQL abaixo retornará componentes que correspondem às seguintes condições:

* No *domínio* `security check`
* E tem um estado de *saúde* de `Clear` OU `Deviating`
* OR é o *Componente* com o nome `ai_engine`

[,yaml]
----
(domain IN ("security check") AND healthstate IN ("CLEAR", "DEVIATING")) OR name IN ("ai_engine")
----

image::k8s/k8s-filters-advanced-topology.png[Filtros de topologia avançada]

==== Compatibilidade de filtros básicos e avançados

Você pode alternar entre a filtragem básica e a avançada selecionando *Alternar para básico* ou *Alternar para STQL* na guia *Topologia* vertical no menu *Filtros*.

É sempre possível alternar da filtragem básica para a avançada. Os filtros básicos selecionados serão convertidos diretamente em uma consulta STQL. Para consultas simples, também é possível alternar da filtragem avançada para a básica; no entanto, algumas consultas avançadas não são compatíveis com os filtros básicos.

➡️ xref:/develop/reference/k8sTs-stql_reference.adoc#_compatibility_basic_and_advanced_filters[Saiba mais sobre a compatibilidade dos filtros de topologia básica e avançada]

==== Outros filtros

Os filtros avançados listados abaixo são compatíveis com a filtragem básica, mas não podem ser definidos ou ajustados como um filtro básico.

* *withNeighborsOf* - quando um filtro avançado inclui a função xref:/develop/reference/k8sTs-stql_reference.adoc#_withneighborsof[withNeighborsOf], o número de componentes cujos vizinhos são consultados é mostrado na caixa *Outros filtros*. Para ser compatível com a filtragem básica, uma função `withNeighborsOf` deve ser unida a outros filtros usando um operador `OR`.
* *identificador* - quando um filtro avançado seleciona componentes por xref:/develop/reference/k8sTs-stql_reference.adoc#_filters[identificador], o número de identificadores de componentes consultados é informado na caixa *Outros filtros*. Para ser compatível com a filtragem básica, um filtro `identifier` deve ser especificado e associado a outros filtros usando o operador `OR identifier IN (...)`.

A caixa *Outros filtros* nos filtros de topologia básica lista todos esses filtros avançados e o número de componentes afetados.

[NOTE]
====
A caixa *Outros filtros* fornece apenas detalhes de filtros avançados que foram definidos e são compatíveis com a filtragem básica.
====


==== Limites de filtragem de topologia

Para otimizar o desempenho, é colocado um limite na quantidade de elementos que podem ser carregados para produzir uma visualização de topologia. O limite de filtragem tem um valor padrão de 10.000 elementos. Se uma xref:/use/views/k8s-filters.adoc#_advanced_topology_filters[consulta de] xref:/use/views/k8s-filters.adoc#_basic_topology_filters[filtro básico] ou xref:/use/views/k8s-filters.adoc#_advanced_topology_filters[avançado] exceder o limite de filtragem, uma mensagem será exibida na tela e nenhuma visualização de topologia será exibida.

Observe que o limite de filtragem é aplicado à quantidade total de elementos que precisam ser *carregados* e não à quantidade de elementos que serão exibidos no final.

No exemplo abaixo, primeiro carregamos todos os vizinhos de cada componente em nossa topologia e, em seguida, exibimos apenas os que pertencem à camada `applications`. Isso provavelmente falharia com um erro de limite de filtragem, pois exige que todos os componentes da topologia sejam carregados.

[,text]
----
withNeighborsOf(direction = "both", components = (name = "*"), levels = "15")
   AND layer = "applications"
----

Para produzir com êxito essa visualização de topologia, precisaríamos reescrever a consulta para manter o número de componentes carregados abaixo do limite de filtragem configurado ou aumentar o limite de filtragem. Ao buscar apenas componentes na camada `applications`, exibiremos os mesmos componentes da consulta acima, sem precisar primeiro carregar todos os componentes. Portanto, é menos provável que essa consulta resulte em um erro de limite de filtragem.

[,yaml]
----
layer = "applications"
----

== Filtrar eventos

O painel *View Filters (Filtros de visualização* ), à esquerda da interface do usuário do SUSE Observability, pode ser usado para filtrar os eventos mostrados na xref:/use/views/k8s-events-perspective.adoc[Perspectiva de eventos]. Eles também estão incluídos na lista de *eventos* no painel direito da guia *Resumo da visualização* e nas guias de detalhes - *Detalhes do componente* e *Detalhes da relação direta*.

Os seguintes filtros de eventos estão disponíveis:

|===
| Filtro | Descrição

| *Categoria*
| Mostrar apenas eventos de uma ou mais xref:/use/views/k8s-events-perspective.adoc#_event_category[categorias].

| *Tipo*
| Clique na caixa de filtro *Tipo* para abrir uma lista de todos os tipos de eventos que foram gerados para os componentes filtrados no momento na janela de tempo atual. Você pode selecionar um ou mais tipos de eventos para refinar os eventos exibidos.

| *Fonte*
| Os eventos podem ser gerados pelo SUSE Observability ou recuperados de um sistema de fonte externa, como o Kubernetes ou o ServiceNow, por meio de uma integração. Clique na caixa de filtro *Source (Origem)* para abrir uma lista de todos os sistemas de origem dos eventos que foram gerados para os componentes atualmente filtrados na janela de tempo atual. Selecione um ou mais sistemas de origem para ver apenas esses eventos.

| *Tags*
| As propriedades relevantes do evento serão adicionadas como tags quando um evento for recuperado de um sistema externo. Por exemplo, `status:open` ou `status:production`. Isso pode ajudar a identificar eventos relevantes para um problema ou ambiente específico.
|===
