:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Cloud Beobachtbarkeit

== Übersicht

Nach dem Erwerb von SUSE Cloud Observability über den Cloud Provider Marketplace wird Ihre SUSE Cloud Observability-Umgebung provisioniert.

Sie erhalten eine E-Mail von SUSE Cloud Observability mit den erforderlichen Anmeldedaten und Links zu Ihrer Umgebung. Diese Schnellstartanleitung hilft Ihnen bei den ersten Schritten und bei der Übernahme Ihrer eigenen Daten in Ihre SUSE Cloud Observability-Implementierung.

== Erste Schritte - Erste Schritte

. Ein Passwort einrichten
. Anmeldung bei Ihrer SUSE Cloud Observability-Instanz
. Installieren Sie den SUSE Observability-Agenten auf Ihrem Cluster
. Ändern Sie Ihre persönlichen Kontodaten

=== Ein Passwort einrichten

Die E-Mail von SUSE Cloud Observability enthält einen eindeutigen Link, über den Sie Ihr Initialpasswort für das Konto festlegen können.  Dies muss geschehen, bevor Sie sich anmelden und die Beobachtungsumgebung konfigurieren können.

=== Zugriff auf Ihre SUSE Observability-Umgebung

Melden Sie sich bei Ihrer SUSE Cloud Observability-Umgebung an, indem Sie auf den eindeutigen Link in der E-Mail klicken, die Sie von SUSE Cloud Observability erhalten haben.  Wenn Sie Ihr Passwort eingeben, gelangen Sie zum Konfigurationsbildschirm für das Hinzufügen von Clustern.

ifdef::feature-agent-install-cluster[]
=== Installieren Sie den SUSE Observability-Agenten auf Ihrem Cluster

SUSE Cloud Observability verwendet StackPacks, um die Konfiguration Ihrer Downstream-Cluster und die Übertragung von Daten in die Observability-Umgebung zu erleichtern.

Um hierher zu gelangen, öffnen Sie in der SUSE Observability-Benutzeroberfläche das Hauptmenü, indem Sie oben links auf dem Bildschirm auf `StackPacks` > `Kubernetes` klicken.

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

Sobald Sie sich im Kubernetes StackPack-Bildschirm befinden, ist es sehr einfach, einen Cluster zur Beobachtungsumgebung hinzuzufügen.

Geben Sie einen Namen für den Cluster ein, den Sie beobachten möchten. Dieser muss nicht mit dem in "kubeconfig" verwendeten Clusternamen übereinstimmen.

Der Name des Kubernetes-Clusters muss mit einem Kleinbuchstaben oder einer Ziffer beginnen und enden und darf nur aus Kleinbuchstaben, Ziffern, Punkten und Bindestrichen (. -) bestehen.

Geben Sie den Namen ein und klicken Sie auf die Schaltfläche `Install`.  Daraufhin wird der folgende Bildschirm angezeigt, auf dem der Cluster als "Warten auf Daten" gekennzeichnet ist

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - Hinzufügen eines Clusters]

Klicken Sie auf den Clusternamen, um diesen Bereich zu erweitern und eine Reihe von Informationen anzuzeigen, darunter Voraussetzungen und Befehle, die für Ihre vorhandenen Cluster verwendet werden können, um sie zu Ihrer Beobachtungsumgebung hinzuzufügen.

Überprüfen Sie die Informationen und stellen Sie sicher, dass Sie die richtigen Berechtigungen für Ihre Kubernetes-Umgebung haben und dass Sie eine unterstützte Version von Kubernetes verwenden.

Diese Befehle sind einzigartig für Ihre Observability-Bereitstellung und enthalten die erforderlichen API-Schlüssel und URLs.  Wählen Sie die entsprechenden Befehle für Ihren Cluster aus. Es gibt Abschnitte für EKS, RKE, generische Kubnernetes-Cluster und viele mehr.

Mit diesen Befehlen werden die SUSE Cloud Obervability-Agenten installiert und der Cluster mit Ihrer SUSE Cloud Observability-Umgebung verbunden.

Sie können die Befehle direkt von der Benutzeroberfläche aus verwenden, oder Sie können die Befehle im Abschnitt "StackState-Agent und Cluster-Agent bereitstellen" in der xref:/k8s-quick-start-guide.adoc[Schnellstartanleitung] befolgen, um sie in Ihren nachgelagerten Clustern einzusetzen

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - Nach der Installation]

Nachdem der Cluster verbunden wurde, sollte in der SUSE Cloud Observability-Benutzeroberfläche ein grünes Häkchen zu sehen sein.

An dieser Stelle können Sie mit der Auswertung Ihrer Daten beginnen.
endif::feature-agent-install-cluster[]

=== Ändern Sie Ihre persönlichen Kontodaten

Schritt 4 Ihrer E-Mail ist die Aktualisierung Ihrer persönlichen Daten.  Klicken Sie auf den Link in der E-Mail und fügen Sie bei Bedarf Ihre grundlegenden persönlichen Daten hinzu.  Bei Bedarf können Sie in diesem Bereich auch eine 2FA-Authentifizierung einrichten.

=== Erkunden Sie Ihre Daten

Um mit der Erkundung Ihrer Daten zu beginnen, öffnen Sie das Hauptmenü, indem Sie oben links auf den Bildschirm klicken, und gehen Sie zu `Kubernetes`, um eine Liste der beobachtbaren Elemente anzuzeigen.

image::accessing_views_1.png[SUSE Cloud Observability - Erkundung Ihrer Daten]

Wählen Sie im Abschnitt "Infrastruktur" die Option "Cluster", die eine Liste der überwachten Cluster anzeigen sollte. Wählen Sie Ihren Cluster aus, um eine der vielen integrierten Ansichten anzuzeigen.

image::accessing_views_2.png[SUSE Cloud Observability - Cluster-Ansicht]

An dieser Stelle können Sie mit der Untersuchung der Daten für Ihren Cluster beginnen oder weitere Cluster hinzufügen, aus denen Sie Daten sammeln möchten.

Weitere Informationen zur Verwendung von SUSE Cloud Observability, einschließlich der Erstellung benutzerdefinierter Ansichten, finden Sie in der Standarddokumentation.

=== Einschränkungen der SUSE Cloud Observability

Beachten Sie, dass SUSE Cloud Observability keine RED-Signale (Rate, Fehler und Dauer) bereitstellt, sondern nur das Raten-Signal.  Diese Funktion ist mit SUSE Rancher Prime verfügbar.
Kunden, die diese Signale für eine vollständige Observability-Lösung benötigen, sollten sich mit SUSE in Verbindung setzen, um SUSE Rancher Prime zu besprechen. Alternativ können diese Daten auch mit den OpenTelemetry-Kollektoren erfasst werden.

== Zusätzliche Informationen

Weitere Informationen, einschließlich der Voraussetzungen und unterstützten Kubernetes-Versionen für Ihre Plattform, finden Sie in der xref:/k8s-quick-start-guide.adoc[Schnellstartanleitung].
