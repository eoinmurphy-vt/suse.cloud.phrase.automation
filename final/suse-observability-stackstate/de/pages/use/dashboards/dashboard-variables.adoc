= Dashboard-Variablen
:revdate: 2025-11-24
:page-revdate: {revdate}
:description: Verwenden Sie Variablen, um dynamische Dashboards zu erstellen.

Durch die Konfiguration von Variablen für ein Dashboard können Sie Dashboards mit dynamischen Widgets erstellen. Variablen ermöglichen die Wiederverwendung einer einzigen Dashboard-Definition für verschiedene Anwendungsfälle. 

== Anzeigen von Dashboard-Variablen
Verwenden Sie in der oberen Navigationsleiste das Dropdown-Menü neben der Schaltfläche "Speichern..." und klicken Sie auf "Dashboard-Variablen bearbeiten".

image::dashboards/open-variables-drawer.png[Hinzufügen aller angehefteten Artikel]

=== Eingebaute Variablen
Es gibt Variablen, die in {stackstate-product-name} eingebaut sind. Die Namen dieser Variablen sind reserviert, und Sie können Ihren eigenen Variablen nicht die gleichen Namen geben.

Um alle integrierten Variablen anzuzeigen, klicken Sie auf das Akkordeon mit dem Titel "SUSE Observability PromQL-Variablen".

== Hinzufügen einer neuen Variablen

. Öffnen Sie die Schublade "Variablen des Dashboards bearbeiten". 
. Klicken Sie auf die Schaltfläche "Variable hinzufügen", um eine zweite Schublade zu öffnen, in der Sie die neue Variable konfigurieren können.

image::dashboards/variable-drawer-example.png[Eine Beispielvariable]

Die Konfigurationsfelder für eine Variable sind wie folgt:

|===
| Name des Feldes | Beschreibung

| Name
| Der Namensbezeichner der Variablen. Dies ist der Name, den Sie <<using-variables, in kompatiblen Feldern verwenden>> können.

| Etikett anzeigen
| Ein freundlicher Name für die Variable, der auf dem Dashboard angezeigt wird, wenn <<selecting-variables, die Variable ausgewählt wird>>. Dies hilft Ihnen, die Variable zu identifizieren.

| Beschreibung
| Eine Beschreibung, um den Zweck der Variablen zu verstehen. Diese Beschreibung wird in der Schublade "Variablen des Dashboards bearbeiten" angezeigt.

| Typ
| Der Typ der Variablen.
|===

== Variable Typen
=== Text-Variable

Eine Textvariable enthält einfachen Text. Sie können einen Anfangswert festlegen und die Variable schreibgeschützt machen.

=== Variable auflisten

Eine Listenvariable hat mögliche Werte, die durch das Feld "Quelle" definiert sind. Jeder Quellentyp bietet zusätzliche Felder, mit denen Sie die möglichen Werte für die Variable eingrenzen können.

|===
| Quelle: Typ | Quelle: Felder

| Metrische Bezeichnungen
| Geben Sie null oder mehr Serienselektoren ein, um die vorhandenen metrischen Bezeichnungen zu verfeinern.

| Metrische Kennzeichnungswerte
| Geben Sie einen Namen für die metrische Bezeichnung und null oder mehr Serienselektoren ein, um die vorhandenen Werte der metrischen Bezeichnung einzugrenzen.

| Metric PromQL
| Geben Sie eine benutzerdefinierte PromQL-Abfrage und einen Etikettennamen ein. Die daraus resultierenden möglichen Werte sind die Werte für den angegebenen Etikettennamen.

| Statische Liste
| Geben Sie ein oder mehrere Paare aus Bezeichnung und Wert ein, um eine statische Liste möglicher Werte zu erstellen.
|===

Wenn Sie den Quellentyp und die zugehörigen Felder ändern, wird die Vorschau auf der rechten Seite der Schublade mit den neuen möglichen Werten aktualisiert.


==== Gemeinsame Einstellungen der Listenvariablen

Alle Listenvariablenquellen haben auch die folgenden gemeinsamen Einstellungen:

|===
| Name des Feldes | Beschreibung

| Sortieren
| Sortieren Sie die Werte in der Liste. Zum Beispiel in alphabetischer oder numerischer Reihenfolge.

| Mehrere Werte zulassen
| Erlauben Sie den Benutzern, mehr als einen Wert auf einmal auszuwählen.

| Aktivieren Sie "Alles einschließen".
| Fügt der Liste, die der Benutzer sieht, ein Element zur Auswahl aller Werte hinzu. Sie können optional einen benutzerdefinierten Wert für dieses Element angeben oder ihn automatisch generieren lassen. 
|===

== Verwendung von Variablen [[using-variables]]
Sobald eine Variable definiert ist, kann sie verwendet werden, um verschiedene Felder im Dashboard auszufüllen:

* Andere Variablen
* Name und Beschreibung des Widgets
* Widget-Link-Felder: URL, Name und Tooltip
* Widget-Abfragefelder: PromQL-Ausdruck, Alias und Min-Schritt
* Editor für das Markdown-Widget

Um eine Variable in einem kompatiblen Feld zu verwenden, verwenden Sie den Variablennamen in `${}` (z. B. `$\{my_variable}`). Der Wert der Variablen wird interpoliert.

=== Wie Variablen interpoliert werden
Textvariablen werden als die Zeichenkette interpoliert, die in das Textfeld der Variablen geschrieben wird.

In ähnlicher Weise werden Listenvariablen, die weder mehrere Werte zulassen noch "Alles einschließen", als Wert des ausgewählten Elements aus der Liste interpoliert.

Listenvariablen, die mehrere Werte zulassen, werden als regulärer Ausdruck interpoliert, der die Gruppe der ausgewählten Werte erfasst (z. B. `(value_1|value_2)`), unabhängig davon, ob eine oder mehrere Optionen ausgewählt sind.

Bei Listenvariablen, bei denen die Option "Alles einbeziehen" aktiviert und für die Variable ausgewählt ist, wird entweder interpoliert als:* den benutzerdefinierten Wert, falls definiert, oder * einen generierten regulären Ausdruck, der eine Gruppe aller möglichen Werte erfasst (zum Beispiel `(value_1|value_2|value_3)`)

Sie können den PromQL-Matcher für reguläre Ausdrücke `=~` verwenden, wenn Sie Variablen konsumieren, die als Erfassungsgruppen für reguläre Ausdrücke interpoliert werden.

== Auswahl der Variablen [[selecting-variables]]
Die Variablen werden im oberen Teil des Dashboards neben der Zeitspannenauswahl aufgeführt. Es werden nur die ersten paar Variablen angezeigt. Um die übrigen Variablen zu sehen, verwenden Sie die Dropdown-Liste "Mehr".

Klicken Sie auf eine Variable, um ihren Selektor zu öffnen und die Werte auszuwählen, die Sie auf dem Dashboard anzeigen möchten.

Wenn Sie verschiedene Werte auswählen, reagieren die Widgets, die die Variable verwenden, auf die Änderungen.
