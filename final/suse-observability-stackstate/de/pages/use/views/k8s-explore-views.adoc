:revdate: 2025-07-10

== Übersicht

Das Konzept einer xref:/use/views/k8s-view-structure.adoc[Ansicht] in SUSE Observability ermöglicht es Ihnen, einen Bereich Ihrer IT-Landschaft zu überwachen, den Sie zuvor gespeichert haben. Wenn Sie jedoch eine Teilmenge einer bestimmten Ansicht untersuchen müssen, möchten Sie oft nicht, dass der Umfang dieser Ansicht verloren geht. Hier kommen die *Erkundungsansichten* ins Spiel.

Um den Geltungsbereich einer bestimmten Ansicht (z. B. `my view`) zu erhalten, werden alle Untersuchungsaktionen, die auf ein Topologieelement oder eine Auswahl von Elementen (z. B. Komponenten, Relationen, Gruppen) angewendet werden, automatisch in einer temporären Erkundungsansicht unter der Ansicht geöffnet, mit der Sie begonnen haben (z. B. `my view / explore`).

Beispiele für *Ermittlungsmaßnahmen*, die automatisch in einer Erkundungsansicht geöffnet werden:

* Die * Schaltfläche +* wurde auf eine Komponente in der xref:/use/views/k8s-topology-perspective.adoc[Topologieperspektive] geklickt, um die verborgenen Nachbarn dieser Komponente anzuzeigen
* Eine *Schnellaktion* wurde an einer Komponente ausgeführt
* Ein Topologieelement (Komponente, Relation, Gruppe) wurde *mit einem Doppelklick* in der xref:/use/views/k8s-topology-perspective.adoc[Topologieperspektive] untersucht
* Der Link oder die Schaltfläche *'Explore...'* wurde auf ein Topologieelement (Komponente, Relation, Gruppe) geklickt, um es zu untersuchen

[TIP]
====
Die von einer bestehenden Erkundungsansicht aus ausgeführten Untersuchungsaktionen öffnen *keine* neue Erkundungsansicht, sondern ändern den Umfang der Erkundungsansicht, in der Sie sich befinden.
====


== Handhabung

Wenn eine Explore-Ansicht automatisch erstellt wird, wird auch ihr ursprünglicher Bereich definiert. Dies ist hilfreich, wenn Sie die Erkundungsansicht auf ihren ursprünglichen Umfang zurücksetzen möchten, nachdem Sie einige Änderungen daran vorgenommen haben.

Obwohl eine Erkundungsansicht nur vorübergehend gedacht ist, kann sie als xref:/use/views/k8s-custom-views.adoc[benutzerdefinierte Ansicht] gespeichert werden, wenn Sie sie später überwachen oder überprüfen möchten.

Wenn Sie eine Erkundungsansicht nicht speichern möchten, bewegen Sie sich einfach von ihr weg oder kehren Sie zu der Ansicht zurück, mit der Sie begonnen haben, indem Sie die Brotkrümel in der oberen Navigationsleiste verwenden.

== Struktur

Die Erkundungsansichten haben eine identische Struktur wie die xref:/use/views/k8s-custom-views.adoc[benutzerdefinierten Ansichten]: Sie haben die gleichen xref:/use/views/k8s-view-structure.adoc#_filters[Filter] und die gleichen xref:/use/views/k8s-view-structure.adoc#_perspectives[Perspektiven].
