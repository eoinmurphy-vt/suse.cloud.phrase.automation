== KomponentePräsentation

Eine `ComponentPresentation` definiert, wie passende Komponenten in der Benutzeroberfläche für einen bestimmten Modus und Kontext dargestellt werden.

[,yaml]
----
_type: ComponentPresentation
identifier: urn:...:component-presentation:<presentation-name>
mode: ["observability" | "security" | "all"]

binding:
  _type: ComponentPresentationQueryBinding
  query: string                       # Primitive STQL query used to match components

rank:
  specificity: number                 # Higher values indicate more specific definitions

presentation:
  icon: string                        # Optional icon (base64-encoded)
  name:                               # Optional naming configuration
    plural: string
    singular: string
    title: string

  mainMenu:                           # Optional main menu entry
    group: string                     # Name of an existing MainMenuGroup
    rank: number                      # Optional override within the group

  overview:                           # Optional overview page configuration
    columns:
      - _type: string                 # ViewTypeTableCol*

  highlight:                          # Optional highlight page configuration
    summary:
      ...
    fields:
      - _type: string                 # ComponentTypeField
        fieldId: string
        label:
          title: string
          helpBubbleText: string      # Optional
        valueExtractor:
          _type: string
        display:
          _type: string
    about:
      _type: ComponentTypeAbout
      fields:
        - string

  filter:                             # Optional overview filters
    - _type: FilterField
      name: string
      tagField: string
----

== Identifier

Die Bezeichner sollten dem Format {stackstate-product-name} (d. h. `urn:...`) entsprechen. Weitere Informationen finden Sie in der Dokumentation zu xref:/configure/topology/identifiers.adoc[Identifikatoren].

== Verbindlich

Der Datenfluss bestimmt, für welche Komponenten ein `ComponentPresentation` gilt.

Bindungen werden anhand von Komponentendaten ausgewertet und müssen einfach und effizient sein, da sie bei der UI-Auswertung verwendet werden.

Derzeit unterstützter `binding` Typ:

* `ComponentPresentationQueryBinding`: eine STQL-Abfrage (siehe xref:/develop/reference/k8sTs-stql_reference.adoc[STQL] für weitere Informationen)

== Rang

Der Abschnitt `rank` steuert, wie mehrere übereinstimmende `ComponentPresentation` Definitionen kombiniert werden.

Höhere Spezifitätswerte weisen auf speziellere Definitionen hin und haben beim Zusammenstellen des Präsentationsverhaltens Vorrang vor niedrigeren Werten.

== Abschnitte der Präsentation

Alle Felder unter `presentation` sind fakultativ. Eine `ComponentPresentation` kann nur eine Teilmenge der Darstellungsaspekte definieren.

Darstellungsabschnitte werden zusammengestellt, wenn sich mehrere Definitionen auf dieselbe Komponente beziehen.

=== Icon und Name

Definiert die in der gesamten Benutzeroberfläche verwendete Ikonographie und Benennung.

=== Hauptmenü

Steuert, ob passende Komponenten im Hauptmenü erscheinen.

Die referenzierte Menügruppe muss separat über `MainMenuGroup` definiert werden.

=== Übersicht

Definiert die in Übersichtstabellen angezeigten Spalten.

Die Spalten können die von anderen passenden Präsentationen definierten Spalten außer Kraft setzen oder erweitern.

=== Highlight

Definiert Felder und Abschnitte, die auf Hervorhebungsseiten angezeigt werden.

==== Felder

==== Über

Definiert die Felder, die auf der Seite mit den Komponentenhighlights im Infofenster angezeigt werden.

=== Filter

Definiert Filter, die für Übersichtsseiten angezeigt werden.

Filter werden angezeigt, wenn für die gebundenen Komponenten übereinstimmende Daten vorhanden sind.
