== Übersicht

Falls kein externer Authentifizierungsanbieter verwendet werden kann, können Sie die dateibasierte Authentifizierung verwenden. Dazu muss jeder SUSE Observability-Benutzer in der Konfigurationsdatei vorkonfiguriert werden. Bei jeder Änderung, die an einem Benutzer in der Konfiguration vorgenommen wird, startet SUSE Observability automatisch neu, nachdem die Änderungen mit Helm übernommen wurden.

SUSE Observability enthält eine Reihe von Standardrollen, siehe die Beispielkonfiguration unten. Die jeder Standardrolle zugewiesenen Berechtigungen und Anweisungen zum Erstellen weiterer Rollen finden Sie in der xref:/setup/security/rbac/README.adoc[Dokumentation zur rollenbasierten Zugriffskontrolle (RBAC)].

== Einrichten der dateibasierten Authentifizierung

=== Kubernetes

Um die dateibasierte Authentifizierung in Kubernetes zu konfigurieren, müssen SUSE Observability-Benutzer zur Datei `authentication.yaml` hinzugefügt werden. Zum Beispiel:

[tabs]
====
authentication.yaml::
+
--

[,yaml]
----

stackstate:
  authentication:
    file:
      logins:
        - username: admin
          passwordHash: 5f4dcc3b5aa765d61d8327deb882cf99
          roles: [ stackstate-admin ]
        - username: guest
          passwordHash: 5f4dcc3b5aa765d61d8327deb882cf99
          roles: [ stackstate-guest ]
        - username: poweruser
          passwordHash: 5f4dcc3b5aa765d61d8327deb882cf99
          roles: [ stackstate-power-user ]
        - username: troubleshooter
          passwordHash: 5f4dcc3b5aa765d61d8327deb882cf99
          roles: [ stackstate-k8s-troubleshooter ]
----

--
====

Führen Sie die folgenden Schritte aus, um Benutzer zu konfigurieren und Änderungen vorzunehmen:

. Unter `authentication.yaml` - Benutzer hinzufügen. Die folgende Konfiguration sollte für jeden Benutzer hinzugefügt werden (siehe das Beispiel oben):
 ** *Benutzername* - der für die Anmeldung bei SUSE Observability verwendete Benutzername. Es sind nur alphanumerische und _-Zeichen erlaubt.
 ** *passwordHash* - das für die Anmeldung bei SUSE Observability verwendete Passwort. Passwörter werden als bcrypt-Hash gespeichert.
 ** *roles* - die Liste der Rollen, in denen der Benutzer Mitglied ist. Die xref:/setup/security/rbac/rbac_permissions.adoc#_predefined_roles[Standardrollen von SUSE Observability] sind `stackstate-admin`, `stackstate-power-user` und `stackstate-guest`, Einzelheiten zum Erstellen anderer Rollen finden Sie unter xref:/setup/security/rbac/rbac_roles.adoc[RBAC-Rollen].
. Speichern Sie die Datei `authentication.yaml` zusammen mit der Datei `values.yaml` aus der Installationsanleitung von SUSE Observability.
. Führen Sie ein Helm-Upgrade durch, um die Änderungen zu übernehmen:
+
[,text]
----
 helm upgrade \
   --install \
   --namespace suse-observability \
   --values values.yaml \
   --values authentication.yaml \
 suse-observability \
 suse-observability/suse-observability
----

[NOTE]
====
*Anmerkung:*

* Ein bcrypt-Passwort-Hash kann mit der folgenden Befehlszeile `htpasswd -bnBC 10 "" <password> | tr -d ':\n'` oder mit einem Online-Tool erstellt werden.
* Die erste Ausführung des Befehls helm upgrade führt zu einem Neustart der Pods, was zu einer kurzen Unterbrechung der Verfügbarkeit führen kann.
* Fügen Sie `authentication.yaml` in jeden `helm upgrade` Lauf ein.
* Die Authentifizierungskonfiguration wird als Kubernetes-Geheimnis gespeichert.
====


Führen Sie die folgenden Schritte aus, um Benutzer zu konfigurieren und Änderungen vorzunehmen:

. Unter `authentication.yaml` - Benutzer hinzufügen. Die folgende Konfiguration sollte für jeden Benutzer hinzugefügt werden (siehe das Beispiel oben):
 ** *Benutzername* - der für die Anmeldung bei SUSE Observability verwendete Benutzername. Es sind nur alphanumerische und _-Zeichen erlaubt.
 ** *Passwort* - das Passwort, das für die Anmeldung bei SUSE Observability verwendet wird. Passwörter werden entweder als MD5-Hash oder als bcrypt-Hash gespeichert.
 ** *roles* - die Liste der Rollen, in denen der Benutzer Mitglied ist. Die xref:/setup/security/rbac/rbac_permissions.adoc#_predefined_roles[Standardrollen von SUSE Observability] sind `stackstate-admin`, `stackstate-power-user`, `stackstate-k8s-troubleshooter` und `stackstate-guest`, Einzelheiten zum Erstellen anderer Rollen finden Sie unter xref:/setup/security/rbac/rbac_roles.adoc[RBAC-Rollen].
. Starten Sie SUSE Observability neu, um die Änderungen zu übernehmen.

[NOTE]
====
*Anmerkung:*

* Ein MD5-Passwort-Hash kann mit den Befehlszeilenprogrammen `md5sum` oder `md5` unter Linux und Mac erstellt werden.
* Ein bcrypt-Passwort-Hash kann mit der folgenden Befehlszeile `htpasswd -bnBC 10 "" <password> | tr -d ':\n'` oder mit einem Online-Tool erstellt werden.
====


=== Verwendung eines externen Geheimnisses

Wenn die Benutzerkennwörter von einem externen Geheimnis stammen sollen, befolgen Sie xref:/setup/security/external-secrets.adoc#_getting_authentication_data_from_an_external_secret[diese Schritte], geben aber die folgenden Daten ein:

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
  file_<username1>_password: <base64 of bcrypt of password>
  file_<username2>_password: <base64 of bcrypt of password>
----

Für jeden Benutzer im Bereich "Logins" sollte ein Datensatz zum Geheimnis hinzugefügt werden, der die Vorlage ausfüllt. Zum Beispiel:

[,yaml]
----

stackstate:
  authentication:
    file:
      logins:
        - username: admin_user
          roles: [ stackstate-admin ]


kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
   file_admin_user_password: "base64EncryptedPass"
----
