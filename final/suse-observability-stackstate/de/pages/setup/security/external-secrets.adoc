= Externe Geheimnisse
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability Externe Geheimnisse

== Übersicht

SUSE Observability kann Geheimnisse wie Lizenzschlüssel, API-Schlüssel und Authentifizierungsschlüssel über die Helm-Installation übernehmen, aber auch aus bereits bereitgestellten Geheimnissen.

Hier wird beschrieben, wie Sie dies konfigurieren.

== Abrufen des Lizenzschlüssels von einem externen Geheimnis

Erstellen Sie in dem Namensraum, in dem SUSE Observability installiert ist, ein Geheimnis der folgenden Form, indem Sie die Leerstellen ausfüllen:

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
  LICENSE_KEY: "<base64 of the license key>"
----

Fügen Sie dem Befehl helm install folgendes hinzu, um das Geheimnis zu verwenden:

[,bash]
----
  --set 'stackstate.license.fromExternalSecret'='<custom-secret-name>'
----

== Abrufen von Benutzername und Passwort für E-Mail-Benachrichtigungen von einem externen Geheimnis

Erstellen Sie in dem Namensraum, in dem SUSE Observability installiert ist, ein Geheimnis der folgenden Form, indem Sie die Leerstellen ausfüllen:

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
  SMTP_USER_NAME: "<base64 of the smtp username>"
  SMTP_PASSWORD: "<base64 of the smtp password>"
----

Fügen Sie dem Befehl helm install folgendes hinzu, um das Geheimnis zu verwenden:

[,bash]
----
  --set 'stackstate.email.server.auth.fromExternalSecret'='<custom-secret-name>'
----

== Abrufen von Authentifizierungsdaten aus einem externen Geheimnis

Erstellen Sie in dem Namensraum, in dem SUSE Observability installiert ist, ein Geheimnis der folgenden Form.

[,yaml]
----
kind: Secret
metadata:
   name: "<custom-secret-name>"
type: Opaque
data:
   default_password: <base64 of bcrypted password>
----

Je nach der gewählten Authentifizierungsmethode wird das Feld `default_password` durch andere Daten ersetzt. Weitere Informationen finden Sie unter den xref:/setup/security/authentication/authentication_options.adoc[Authentifizierungsoptionen]. In diesem Beispiel wird die Einstellung xref:/setup/security/authentication/single_password.adoc[Einzelpasswort] verwendet.

Fügen Sie dem Befehl helm install folgendes hinzu, um das Geheimnis zu verwenden:

[,bash]
----
  --set 'stackstate.authentication.fromExternalSecret'='<custom-secret-name>'
----
