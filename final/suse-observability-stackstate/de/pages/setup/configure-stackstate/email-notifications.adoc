= Konfigurieren von SUSE Observability für E-Mail-Benachrichtigungen
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability Selbstgehostet

[CAUTION]
====
SaaS-Benutzer von SUSE Observability können E-Mail-Benachrichtigungen ohne zusätzliche Konfiguration verwenden. Diese Anleitung gilt nur für selbst gehostete SUSE Observability-Installationen, die den E-Mail-Benachrichtigungskanal verwenden möchten.
====


Bevor Sie den E-Mail-Benachrichtigungskanal in SUSE Observability verwenden können, müssen Sie zunächst die folgenden Schritte ausführen:

== Konfigurieren Sie SUSE Observability mit der SMTP-Konfiguration

SUSE Observability muss mit Anmeldeinformationen für die Verbindung zum SMTP-Server konfiguriert werden. Sie können dies tun, indem Sie der Datei `values.yaml` Ihrer SUSE Observability-Installation Folgendes hinzufügen:

[,yaml]
----
stackstate:
  email:
    enabled: true
    sender: "<stackstate@example.com>"
    server:
      host: "<smtp.example.com>"
      auth:
        username: "<user name>"
        password: "<user password>"
----

Dabei wird der Port `587` des SMTP-Servers verwendet und mit dem Befehl `STARTTLS` eine sichere Verbindung hergestellt. Dies sind alle anderen Optionen, die angepasst werden können:

[,yaml]
----
stackstate:
  email:
    additionalProperties:
      # Add needed Java email properties for your mail server (use string values), defaults are:
      "mail.smtp.auth": "true"
      "mail.smtp.starttls.enable": "true"
    server:
      protocol: smtp
      port: 587
----

=== Verwendung eines externen Geheimnisses

Wenn der Benutzername und das Passwort nicht in den Werten angegeben werden können, sondern von einem externen Geheimnis stammen sollen, gehen Sie wie folgt xref:/setup/security/external-secrets.adoc#_getting_username_and_password_for_email_sending_from_an_external_secret[vor]
