= Konfigurieren von SUSE Observability für Slack-Benachrichtigungen
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability Selbstgehostet

[CAUTION]
====
SaaS-Benutzer von SUSE Observability können Slack-Benachrichtigungen ohne zusätzliche Konfiguration verwenden. Diese Anleitung gilt nur für selbst gehostete SUSE Observability-Installationen, die den Slack-Benachrichtigungskanal nutzen möchten.
====


Bevor Sie den Slack-Benachrichtigungskanal in SUSE Observability verwenden können, müssen Sie zunächst die folgenden Schritte ausführen, um sowohl Slack als auch SUSE Observability einzurichten:

. Erstellen Sie eine Slack-App für SUSE Observability in Ihrem Arbeitsbereich
. Konfigurieren Sie SUSE Observability mit den Anmeldeinformationen für diese Slack-App.

== Erstellen einer Slack-App für SUSE Observability

[NOTE]
====
Sie müssen die Berechtigungen in Slack haben, um Slack-Apps für Ihren Arbeitsbereich zu verwalten.
====


Rufen Sie die https://api.slack.com/apps[Seite] Slack auf und klicken Sie auf die Schaltfläche *Neue App erstellen*.

* Wählen Sie in dem sich öffnenden Dialog die Option "Aus einem App-Manifest".
* Wählen Sie den Arbeitsbereich, an den Sie Benachrichtigungen senden möchten, und klicken Sie auf Weiter.
* Kopieren Sie den Inhalt des unten stehenden Manifests der Slack-App und fügen Sie ihn in den Textbereich ein. Stellen Sie sicher, dass Sie die Werte in `redirect_urls` durch die URL(s) Ihrer SUSE Observability-Instanz ersetzen. Klicken Sie auf Weiter.
* Vergewissern Sie sich, dass die URL korrekt ist und dass die aufgelisteten "Bot-Scopes" `channels:join, channels:read, chat:write, groups:read` sind, und klicken Sie auf die Schaltfläche "Erstellen", um die App zu erstellen.
* Auf der Seite "Grundlegende Informationen" der App ist es möglich, das Symbol zu ändern (im Abschnitt "Anzeigeinformationen"). Sie können es z. B. durch das SUSE Observability-Logo image:logo/stackstate-logo.png[,25,25] ersetzen.

.Slack-App-Manifest für SUSE Observability
[,json]
----
{
    "display_information": {
        "name": "SUSE Observability",
        "description": "Receive notification messages from SUSE Observability",
        "background_color": "#000000"
    },
    "features": {
        "bot_user": {
            "display_name": "SUSE Observability",
            "always_online": true
        }
    },
    "oauth_config": {
        "redirect_urls": [
            "https://the.url.of.your.stackstate.installation"
        ],
        "scopes": {
            "bot": [
                "channels:join",
                "channels:read",
                "chat:write",
                "groups:read"
            ]
        }
    },
    "settings": {
        "org_deploy_enabled": false,
        "socket_mode_enabled": false,
        "token_rotation_enabled": false
    }
}
----


== Konfigurieren Sie SUSE Observability mit den Anmeldeinformationen für diese Slack-App

SUSE Observability muss mit den Anmeldedaten für die von Ihnen erstellte Slack-App konfiguriert werden. Sie können dies tun, indem Sie der Datei `values.yaml` Ihrer SUSE Observability-Installation Folgendes hinzufügen:

[,yaml]
----
stackstate:
  components:
    all:
      extraEnv:
        open:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientId: "<app client id>"
        secret:
          CONFIG_FORCE_stackstate_notifications_channels_slack_authentication_clientSecret: "<app client secret>"
----

Die Werte `<app client id>` und `<app client secret>` finden Sie im Abschnitt "App credentials" auf der Seite "Basic Information" der von Ihnen erstellten Slack-App. Wenden Sie diese Konfigurationsänderungen an, indem Sie denselben Helm-Befehl ausführen, der bei der Installation von SUSE Observabilityxref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[(für Kubernetes] oder xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_deploy_suse_observability_with_helm[OpenShift]) verwendet wurde.

Sie sind jetzt bereit, den Slack-Benachrichtigungskanal zu nutzen!
