== Übersicht

Dieses Dokument beschreibt das Upgrade-Verfahren für SUSE Observability.

== Bevor Sie aktualisieren

Beachten Sie bei der Durchführung eines SUSE Observability-Upgrades die folgenden Punkte:

[CAUTION]
====
*Lesen Sie immer die* xref:/setup/upgrade-stackstate/version-specific-upgrade-instructions.adoc[*Versionsspezifischen Upgrade-Hinweise*] *bevor Sie ein Upgrade von SUSE Observability durchführen.*
====


[CAUTION]
====
Wenn Sie ein StackPack aktualisieren, *werden alle Änderungen, die Sie an Konfigurationselementen dieses StackPacks vorgenommen haben, überschrieben*. Weitere Informationen finden Sie unter xref:/stackpacks/about-stackpacks.adoc#_locked_configuration_items[Konfigurationssperre].
====


[WARNING]
====
Wenn in Ihrer SUSE Observability-Installation *Hotfixes* installiert sind, wenden Sie sich vor dem Upgrade an den technischen Support von SUSE Observability.
====


== Schritte zum Upgrade

=== Minor- oder Wartungsversion von SUSE Observability

Eine Nebenversion von SUSE Observability wird durch eine Änderung in der zweiten Ziffer der Versionsnummer angezeigt, z. B. 4.1.0. Wartungsversionen sind durch eine Änderung an der dritten Stelle der Versionsnummer gekennzeichnet, zum Beispiel 4.1.1.

Wenn Sie ein Upgrade auf eine neue *Nebenversion* von SUSE Observability oder eine *Wartungsversion* durchführen, sind SUSE Observability selbst und die StackPacks mit der aktuellen Installation kompatibel.

Ein Minor Upgrade besteht aus den folgenden Schritten:

. <<_create_a_backup,Erstellen Sie ein Backup>>
. <<_upgrade_suse_observability,Upgrade von SUSE Observability>>
. <<_verify_the_new_installation,Überprüfen Sie die neue Installation>>
. Prüfen Sie, ob installierte StackPacks ein Upgrade benötigen

=== Größere Version von SUSE Observability

Eine Hauptversion von SUSE Observability wird durch eine Änderung der ersten Ziffer der Versionsnummer angezeigt, z. B. 4.0.0.

Wenn Sie ein Upgrade auf eine neue *Hauptversion* von SUSE Observability durchführen, können SUSE Observability und die installierten StackPacks mit der aktuellen Installation inkompatibel sein. Einzelheiten finden Sie in den xref:/setup/upgrade-stackstate/version-specific-upgrade-instructions.adoc[versionsspezifischen Upgrade-Hinweisen].

Ein Major Upgrade besteht aus den folgenden Schritten:

. xref:/setup/upgrade-stackstate/steps-to-upgrade.adoc#_create_a_backup[Erstellen Sie ein Backup]
. Optional: xref:/setup/upgrade-stackstate/steps-to-upgrade.adoc#_uninstall_stackpacks_optional[StackPacks deinstallieren]
. xref:/setup/upgrade-stackstate/steps-to-upgrade.adoc#_upgrade_stackstate[Upgrade von SUSE Observability]
. Optional: xref:/setup/upgrade-stackstate/steps-to-upgrade.adoc#_install_stackpacks_optional[StackPacks installieren]
. xref:/setup/upgrade-stackstate/steps-to-upgrade.adoc#_verify_the_new_installation[Überprüfen Sie die neue Installation]

== Durchgehender Überblick über ein Upgrade

=== Erstellen Sie ein Backup

Vor dem Upgrade von SUSE Observability wird empfohlen, eine Sicherungskopie Ihrer Konfigurations- und Topologiedaten zu erstellen:

* xref:/setup/data-management/backup_restore/kubernetes_backup.adoc[Kubernetes-Sicherung]
* xref:/setup/data-management/backup_restore/configuration_backup.adoc[Sicherung der Konfiguration]

[NOTE]
====
Beachten Sie, dass es nicht möglich ist, die Sicherung in der aktualisierten Version von SUSE Observability wiederherzustellen. Die Sicherung von SUSE Observability kann nur in der SUSE Observability-Version vor dem Upgrade wiederhergestellt werden.
====


=== Upgrade von SUSE Observability

Lesen Sie unbedingt die Versionshinweise und alle optionalen Upgrade-Hinweise, bevor Sie das Upgrade durchführen.

[tabs]
====

Kubernetes::
+
--

. Die aktuelle Steuerkarte erhalten Sie unter `helm repo update`.
. Überprüfen Sie die xref:/setup/upgrade-stackstate/version-specific-upgrade-instructions.adoc[versionsspezifischen Upgrade-Hinweise] auf alle Änderungen zwischen Ihrer aktuellen Version und der Version, auf die Sie aktualisieren werden. Wenn an den in der Datei `values.yaml` angegebenen Konfigurationselementen Änderungen vorgenommen wurden, sollte die Datei aktualisiert werden.
. Verwenden Sie für das Upgrade denselben helm-Befehl wie für die xref:/setup/install-stackstate/kubernetes_openshift/kubernetes_install.adoc#_deploy_suse_observability_with_helm[erstmalige Installation von Kubernetes]. Die neue Steuerkarte zieht neuere Versionen von Docker-Images und führt das Upgrade durch.
--

OpenShift::
+
--

. Die aktuelle Steuerkarte erhalten Sie unter `helm repo update`.
. Überprüfen Sie die xref:/setup/upgrade-stackstate/version-specific-upgrade-instructions.adoc[versionsspezifischen Upgrade-Hinweise] auf alle Änderungen zwischen Ihrer aktuellen Version und der Version, auf die Sie aktualisieren werden. Wenn an den in der Datei `values.yaml` angegebenen Konfigurationselementen Änderungen vorgenommen wurden, sollte die Datei aktualisiert werden.
. xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_additional_openshift_values_file[Aktualisieren Sie die] Datei xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_additional_openshift_values_file[ `openshift-values.yaml`.] 
. Verwenden Sie für das Upgrade denselben helm-Befehl wie für die xref:/setup/install-stackstate/kubernetes_openshift/openshift_install.adoc#_deploy_suse_observability_with_helm[erstmalige Installation von OpenShift]. Die neue Steuerkarte zieht neuere Versionen von Docker-Images und führt das Upgrade durch.
--

====

=== Überprüfen Sie die neue Installation

Nachdem SUSE Observability aktualisiert und gestartet wurde, überprüfen Sie, ob die neue Installation von SUSE Observability erreichbar ist und die Anwendung ausgeführt wird.
