= Erste Schritte mit Open Telemetry
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observabilität

Hier ist das Setup, das wir für eine zu überwachende Anwendung erstellen werden:

* Die überwachte Anwendung/Workload läuft auf einem Linux-Host
* Der Open Telemetry-Kollektor läuft auf demselben Linux-Host
* SUSE Observability oder SUSE Cloud Observability

image::otel/open-telemetry-collector-linux.png[Anwendungsinstrumentierung auf einem Linux-Host mit Open Telemetry Collector auf dem Host]

== Installieren Sie den offenen Telemetriekollektor

[NOTE]
====
Für eine Produktionseinrichtung wird dringend empfohlen, den Collector zu installieren, da er es Ihrem Dienst ermöglicht, Daten schnell zu entladen, und der Collector zusätzliche Aufgaben wie Wiederholungsversuche, Stapelverarbeitung, Verschlüsselung oder sogar die Filterung sensibler Daten übernehmen kann.
====


Zuerst installieren wir den Kollektor. Wir konfigurieren es so:

* Empfang von Daten aus möglicherweise vielen instrumentierten Anwendungen
* Erfasste Daten mit Host-Attributen anreichern
* Erzeugen von Metriken für Traces
* Weiterleitung der Daten an SUSE Observability, einschließlich Authentifizierung mit dem API-Schlüssel

Außerdem wird bei Verbindungsproblemen versucht, die Daten erneut zu senden.

=== Erstellen eines Service-Tokens

Es gibt zwei Möglichkeiten, ein Service-Token zu erstellen:

* **SUSE Observability UI** - öffnen Sie das Hauptmenü, indem Sie oben links auf den Bildschirm klicken und zu `StackPacks` > `Open Telemetry` gehen.  Falls Sie dies noch nicht getan haben, klicken Sie auf die Schaltfläche `INSTALL`.  Klicken Sie auf die Schaltfläche `CREATE NEW SERVICE TOKEN` und kopieren Sie den Wert in Ihre Zwischenablage.
* **SUSE Observability CLI** - siehe xref:/use/security/k8s-service-tokens.adoc#_manage_service_tokens[Verwalten von Service-Tokens]

Der Wert des Service-Tokens muss verwendet werden, wenn in den nachstehenden Anweisungen `<SERVICE_TOKEN>` erwähnt wird.

=== Installieren und Konfigurieren des Kollektors

Der Kollektor stellt Pakete (apk, deb und rpm) für die meisten Linux-Versionen und -Architekturen bereit und verwendet `systemd` für die automatische Dienstkonfiguration. Um es zu installieren, suchen Sie die https://github.com/open-telemetry/opentelemetry-collector-releases/releases[neueste Version von] und aktualisieren Sie die URL im Beispiel, um die neueste Version zu verwenden:

[tabs]

sudo dpkg -i otelcol-contrib_0.123.1_linux_amd64.deb

Für andere Installationsoptionen verwenden Sie die https://opentelemetry.io/docs/collector/installation/#_linux[Anleitung].

Nach der Installation ändern Sie die Kollektorkonfiguration, indem Sie `/etc/otelcol-contrib/config.yaml` bearbeiten. Ändern Sie die Datei so, dass sie wie das Beispiel `config.yaml` aussieht, ersetzen Sie `<otlp-suse-observability-endpoint:port>` durch Ihren OTLP-Endpunkt (siehe xref:/setup/otel/otlp-apis.adoc[OTLP-API] für Ihren Endpunkt) und fügen Sie Ihr Service-Token für `<SERVICE_TOKEN>` ein:

.config.yaml
[,yaml]
----
receivers:
  otlp:
    protocols:
      # Only bind to localhost to keep the collector secure, see https://github.com/open-telemetry/opentelemetry-collector/blob/main/docs/security-best-practices.md#safeguards-against-denial-of-service-attacks
      grpc:
        endpoint: 127.0.0.1:4317
      http:
        endpoint: 127.0.0.1:4318
  # Collect own metrics
  prometheus:
    config:
      scrape_configs:
      - job_name: 'otel-collector'
        scrape_interval: 10s
        static_configs:
        - targets: ['0.0.0.0:8888']
extensions:
  health_check: {}
  pprof:
    endpoint: 0.0.0.0:1777
  zpages:
    endpoint: 0.0.0.0:55679
  # Use the API key from the env for authentication
  bearertokenauth:
    scheme: SUSEObservability
    token: "<SERVICE_TOKEN>"
exporters:
  nop: {}
  debug: {}
  otlp/suse-observability:
    compression: snappy
    auth:
      authenticator: bearertokenauth
    # Put in your own otlp endpoint, for example otlp-suse-observability.my.company.com:443
    endpoint: <otlp-suse-observability-endpoint:port>
processors:
  memory_limiter:
    check_interval: 5s
    limit_percentage: 80
    spike_limit_percentage: 25
  batch: {}
  # Optionally include resource information from the system running the collector
  resourcedetection/system:
    detectors: [env, system] # Replace system with gcp, ec2, azure when running in cloud environments
    system:
      hostname_sources: ["os"]
connectors:
  # Generate metrics for spans
  spanmetrics:
    metrics_expiration: 5m
    namespace: otel_span
service:
  extensions: [ bearertokenauth, health_check, pprof, zpages ]
  pipelines:
    traces:
      receivers: [otlp]
      processors: [memory_limiter, resourcedetection/system, batch]
      exporters: [debug, spanmetrics, otlp/suse-observability]
    metrics:
      receivers: [otlp, spanmetrics, prometheus]
      processors: [memory_limiter, batch, resourcedetection/system]
      exporters: [debug, otlp/suse-observability]
    logs:
      receivers: [otlp]
      processors: []
      exporters: [nop]
----


Starten Sie schließlich den Kollektor neu:

[,bash]
----
sudo systemctl restart otelcol-contrib
----

Um die Protokolle des Kollektors zu sehen, verwenden Sie:

[,bash]
----
sudo journalctl -u otelcol-contrib
----

== Sammeln Sie Telemetriedaten von Ihrer Anwendung

Die übliche Methode zur Erfassung von Telemetriedaten ist die Instrumentierung Ihrer Anwendung mit den Open Telemetry SDKs. Wir haben einige Schnellstartanleitungen für einige Sprachen dokumentiert, aber es gibt noch viele mehr:

* xref:/setup/otel/instrumentation/java.adoc[Java]
* xref:/setup/otel/instrumentation/dot-net.adoc[.NET]
* xref:/setup/otel/instrumentation/node.js.adoc[Node.js]

Für die SDKs ist keine zusätzliche Konfiguration erforderlich, sie exportieren standardmäßig über OTLP oder OTLP over HTTP (je nach den unterstützten Protokollen) nach localhost.

Für andere Sprachen folgen Sie der Dokumentation auf https://opentelemetry.io/docs/languages/[opentelemetry.io].

== Die Ergebnisse anzeigen

Gehen Sie zu SUSE Observability und stellen Sie sicher, dass das Open Telemetry Stackpack installiert ist (über das Hauptmenü \-> Stackpacks).

Nach kurzer Zeit und wenn Ihre Anwendung etwas Datenverkehr verarbeitet, sollten Sie sie unter ihrem Dienstnamen in den Übersichten Open Telemetry \-> Services und Service Instances finden können. Traces werden im xref:/use/traces/k8sTs-explore-traces.adoc[Trace-Explorer] und in der xref:/use/views/k8s-traces-perspective.adoc[Trace-Perspektive] für die Service- und Service-Instanz-Komponenten angezeigt. Bereichsmetriken und sprachspezifische Metriken (falls vorhanden) werden in der xref:/use/views/k8s-metrics-perspective.adoc[Metrikperspektive] für die Komponenten verfügbar.
