== Automatische Instrumentierung

Die automatische Instrumentierung für Node.js erfolgt durch Einbindung der Javascript-Bibliotheken für die automatische Instrumentierung in Ihre Anwendung. Eine breite Palette von https://github.com/open-telemetry/opentelemetry-js-contrib/tree/main/metapackages/auto-instrumentations-node#_supported_instrumentations[Bibliotheken und Frameworks wird unterstützt].

Die automatische Instrumentierung erfordert keine Änderungen an der Anwendung. Führen Sie die folgenden Schritte aus, um sie einzurichten:

. Fügen Sie das Open Telemetry Instrumentation SDK zu Ihrer Anwendung hinzu:
+
[,bash]
----
npm install --save @opentelemetry/api
npm install --save @opentelemetry/auto-instrumentations-node
----

. Aktualisieren Sie den Befehl, mit dem Ihre Anwendung gestartet wird, um das SDK zu laden, entweder durch Aktualisieren des Docker-Image-Einstiegspunkts oder des Befehls oder durch Aktualisieren der `command` im Kubernetes-Manifest für Ihre Anwendung. `--require @opentelemetry/auto-instrumentations-node/register` hinzufügen:
+
[,bash]
----
node --require @opentelemetry/auto-instrumentations-node/register app.js
----

. Stellen Sie Ihre Anwendung mit den zusätzlichen Umgebungsvariablen bereit xref:/setup/otel/instrumentation/sdk-exporter-config.adoc[, um den Dienstnamen und den Exporter-Endpunkt zu konfigurieren]. Unterstützte Protokolle sind gRPC und protobuf über HTTP.
. Vergewissern Sie sich, dass SUSE Observability Abläufe und/oder Metriken empfängt, indem Sie im Metrik-/Ablaufverfolgungsexplorer für Ihren Servicenamen nach den Metriken/Abläufen suchen.

Weitere Einzelheiten entnehmen Sie bitte der https://opentelemetry.io/docs/languages/js/automatic/[Dokumentation].

[CAUTION]
====
Die über Umgebungsvariablen konfigurierte automatische Instrumentierung unterstützt nur Traces, bis dieses https://github.com/open-telemetry/opentelemetry-js/issues/4551[Open Telemetry-Problem] behoben ist. Um die Metriken der automatischen Instrumentierung zu aktivieren, sind Codeänderungen erforderlich. Bitte folgen Sie den Anweisungen in der https://opentelemetry.io/docs/languages/js/exporters/#_usage_with_nodejs[Dokumentation], um diese Änderungen vorzunehmen.
====


== Manuelle Instrumentierung

Manuelle Instrumentierung kann verwendet werden, wenn Sie Metriken, Traces oder Protokolle von Teilen des Codes benötigen, die von der automatischen Instrumentierung nicht unterstützt werden. Zum Beispiel nicht unterstützte Bibliotheken, firmeneigener Code oder Metriken auf Unternehmensebene.

Um diese Daten zu erfassen, müssen Sie Ihre Anwendung ändern.

. Einbindung des Open Telemetry SDK als Abhängigkeit
. Fügen Sie Ihrer Anwendung Code hinzu, um bei Bedarf Metriken, Spans oder Protokolle zu erfassen.

Eine ausführliche Dokumentation hierzu finden Sie auf den https://opentelemetry.io/docs/languages/js/instrumentation/[Dokumentseiten des].

Stellen Sie sicher, dass Sie den OTLP-Exporter verwenden und den Endpunkt des Exporters im Code korrekt konfigurieren. Siehe auch die https://opentelemetry.io/docs/languages/js/exporters/#_usage_with_nodejs[Dokumentation]. Unter der Annahme, dass Sie den Exporter wie xref:/setup/otel/collector.adoc[dokumentiert] eingerichtet haben, ist der Endpunkt, der konfiguriert werden muss, `http://opentelemetry-collector.open-telemetry.svc.cluster.local:4317`, unter Verwendung von gRPC. Siehe auch xref:/setup/otel/instrumentation/sdk-exporter-config.adoc#_grpc_vs_http[gRPC vs. HTTP], falls gRPC problematisch ist.
