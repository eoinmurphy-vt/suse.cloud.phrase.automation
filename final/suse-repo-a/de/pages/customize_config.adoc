= Standardkonfiguration außer Kraft setzen
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: {stackstate-product-name} Selbst gehostet

Eine Reihe von Werten kann in der https://github.com/StackVista/helm-charts/tree/master/stable/suse-observability[{stackstate-product-name} Helmkarte] eingestellt werden. Zum Beispiel ist es möglich, die `tolerations` und `nodeSelectors` für jede der Komponenten anzupassen. Sie können auch eine benutzerdefinierte Konfiguration hinzufügen und Umgebungsvariablen einfügen

== Benutzerdefinierte Konfiguration für {stackstate-product-name} `api`

Für den Dienst {stackstate-product-name} `api` kann die benutzerdefinierte Konfiguration direkt in das Helm-Diagramm eingefügt werden. Dies ist der empfohlene Weg, um die Standardkonfiguration zu überschreiben, mit der {stackstate-product-name} ausgeliefert wird, und ist besonders praktisch für die Anpassung der Authentifizierung. Die auf diese Weise eingestellte Konfiguration steht der Konfigurationsdatei {stackstate-product-name} im Format https://github.com/lightbend/config/blob/master/HOCON.md[HOCON] zur Verfügung.

Sie können zum Beispiel einen benutzerdefinierten Link "Passwort vergessen" für die Anmeldeseite {stackstate-product-name} einrichten:

[tabs]
====
values.yaml::
+
--

[,text]
----
stackstate:
  components:
    api:
      config: |
        stackstate.api.authentication.forgotPasswordLink =
        "https://www.stackstate.com/forgotPassword.html"
----

--
====

[NOTE]
====
Beachten Sie, dass die hier eingestellte benutzerdefinierte Konfiguration durch xref:/setup/install-stackstate/kubernetes_openshift/customize_config.adoc#_environment_variables[Umgebungsvariablen] außer Kraft gesetzt wird.
====


== Umgebungsvariablen

Die Konfiguration für alle {stackstate-product-name} Dienste (`receiver`, `k2es-*`, `correlation` und `api`) kann mit Hilfe von Umgebungsvariablen angepasst werden. Umgebungsvariablen werden in der Datei `values.yaml` angegeben und können entweder `secret` (wie Passwörter) oder `open` (für normale Werte) sein. Um ein Konfigurationselement in den Namen einer Umgebungsvariablen umzuwandeln, ersetzen Sie `.` durch `_` und fügen Sie das Präfix `CONFIG_FORCE_` hinzu.

[,text]
----
# configuration item
stackstate.api.authentication.forgotPasswordLink

# environment variable name
CONFIG_FORCE_stackstate_api_authentication_forgotPasswordLink
----

Sie können zum Beispiel einen benutzerdefinierten Link "Passwort vergessen" für die Anmeldeseite {stackstate-product-name} einrichten:

[tabs]
====
values.yaml::
+
--

[,text]
----
stackstate:
  components:
    api:
      extraEnv:
        # The value for open env vars is defined on the deployment
        open:
          CONFIG_FORCE_stackstate_api_authentication_forgotPasswordLink: "https://www.stackstate.com/forgotPassword.html"
        # The value for secret env vars is defined in a secret and referenced from the deployment
        secret:
          CONFIG_FORCE_some_config_with_sensitive_data: "data-stored-in-secret"
----

--
====

[NOTE]
====
Für den Dienst {stackstate-product-name} `api` überschreiben die Umgebungsvariablen die <<_custom_configuration_for_suse_observability_api,benutzerdefinierte Konfiguration, die mit `config` festgelegt wurde>>.
====


* Alle Einzelheiten zur Benennung der verschiedenen Dienste finden Sie in der https://github.com/StackVista/helm-charts/tree/master/stable/suse-observability[Readme-Datei].
* Hier finden Sie weitere Informationen zur xref:/setup/security/authentication/README.adoc[Anpassung der Authentifizierung].
