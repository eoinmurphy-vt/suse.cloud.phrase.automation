= 亲和值配置
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: {stackstate-product-name} 亲和配置

亲和配置可控制{stackstate-product-name} 中的 pod 调度行为。亲和值有助于优化资源利用率，并通过控制 Pod 的调度位置来确保高可用性。

== 可用配置选项

=== 节点亲和力

您可以使用节点亲和性将 pod 调度到特定节点或实例组，例如部署到同一可用性区域的 EC2 节点。

[,yaml]
----
affinity:
  # Node Affinity settings - applied to all components when configured
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
----

=== Pod 抗亲和力

您可以使用 pod anti-affinity，将数据服务的副本调度到不同的节点，以确保高可用性。默认情况下，调度是必需的（硬反亲和），`topologyKey` 是`kubernetes.io/hostname` 。

[,yaml]
----
affinity:
  podAntiAffinity:
    # Enable required pod anti-affinity (true = hard, false = soft)
    requiredDuringSchedulingIgnoredDuringExecution: true
    # Topology key for pod anti-affinity
    topologyKey: "kubernetes.io/hostname"
----

== 行为

=== 节点亲和力

* **配置时**适用于所有组件

=== Pod 抗亲和力

* **配置时**仅当`sizing.profile` 以`-ha` 结尾时才会应用（高可用性配置文件）
* **医管局简介**：`150-ha`,`250-ha`,`500-ha` 、 `4000-ha`
* **受影响的组件**：所有有状态数据服务，包括 Clickhouse、Kafka、Zookeeper、VictoriaMetrics、StackGraph、Elasticsearch

== 配置示例

=== 基本节点亲和性（同一可用区）+用于 HA 部署的硬 PodAntiAffinity

假设您将`sizing.profile` 设置为其中一个 HA 配置文件。下面的值将配置 nodeAffinity，以便将{stackstate-product-name} 的组件调度到`us-west-2a` 区域，并强制将相同数据服务的副本调度到不同主机（默认为`affinity.podAntiAffinity.requiredDuringSchedulingIgnoredDuringExecution`=`true` ）。
[,yaml]
----
affinity:
  # Schedule all pods to nodes in the same AZ
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
----

=== 用于 HA 部署的软 PodAntiAffinity
如果不希望将数据服务强制调度到不同的节点，可以将 podAntiAffinity 配置为 "软 "模式：
[,yaml]
----
affinity:
  podAntiAffinity:
    requiredDuringSchedulingIgnoredDuringExecution: false
----

== 使用方法

[tabs]
====
建议方法::
+
--
[NOTE]
=====
`global.suseObservability` 配置方法从`2.8.0` 版本开始提供。对于早期版本，请使用传统方法。
=====

将亲和配置直接添加到`global.suseObservability.affinity` 下的`values.yaml` 文件中：

[source,yaml]
----
global:
  suseObservability:
    # Your existing configuration
    license: "YOUR-LICENSE-KEY"
    baseUrl: "https://observability.example.com"
    sizing:
      profile: "150-ha"
    adminPassword: "your-password"

    # Affinity configuration
    affinity:
      # Node affinity - schedule all pods to nodes in the same AZ
      nodeAffinity:
        requiredDuringSchedulingIgnoredDuringExecution:
          nodeSelectorTerms:
          - matchExpressions:
            - key: topology.kubernetes.io/zone
              operator: In
              values:
              - us-west-2a

      # Pod anti-affinity for HA profiles (infrastructure components)
      podAntiAffinity:
        # true = hard anti-affinity, false = soft anti-affinity
        requiredDuringSchedulingIgnoredDuringExecution: true
        topologyKey: "kubernetes.io/hostname"
----

然后只需一条命令即可完成安装或升级：

[,bash]
----
helm upgrade --install \
  --namespace suse-observability \
  --create-namespace \
  --values values.yaml \
  suse-observability \
  suse-observability/suse-observability
----

TIP: 对于 HA 配置文件，pod anti-affinity 会自动应用于所有有状态数据服务。只有当您想更改默认行为时，才需要对其进行配置。

有关从传统方法迁移的详细信息，请参阅xref:/setup/upgrade-stackstate/migrate-to-global-mode.adoc[迁移指南]。
--

==== 步骤 1：创建亲和力价值观文件

创建一个单独的值文件，其中包含所需的亲和配置。例如，将以下内容保存为`suse-observability-values-values.yaml` ：

[,yaml]
----
affinity:
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
  podAntiAffinity:
    requiredDuringSchedulingIgnoredDuringExecution: true
----

==== 步骤 2：生成亲和模板值

运行以下命令生成亲和值模板：

[,text]
----
export VALUES_DIR=.
helm template \
  --set license='<your license>' \
  --set baseUrl='<suse-observability-base-url>' \
  --set sizing.profile='<sizing.profile>' \
  --values suse-observability-values-values.yaml \
  suse-observability-values \
  suse-observability/suse-observability-values --output-dir $VALUES_DIR
----

==== 步骤 3：在 Helm 安装中使用生成的值

在 Helm 安装中包含生成的亲和值：

[,bash]
----
helm upgrade \
  --install \
  --namespace suse-observability \
  --values $VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/sizing_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/affinity_values.yaml \
  suse-observability \
  suse-observability/suse-observability
----
