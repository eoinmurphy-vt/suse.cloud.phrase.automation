== Mapeamentos de componentes

O site `OtelComponentMapping` descreve como um componente de topologia é criado (ou atualizado) a partir dos dados da OpenTelemetry.

Exemplos típicos de componentes criados a partir do OpenTelemetry incluem (eles fazem parte do OpenTelemetry StackPack padrão):

* Serviços (derivado de `service.name` / `service.namespace`)
* Instâncias de serviço (derivadas de `service.instance.id`)
* Bancos de dados
* Hosts e recursos do Kubernetes (nós, namespaces, pods)
* Funções e tarefas em ambientes sem servidor ou em lote

== Mapeamentos de relações

Um `OtelRelationMapping` descreve como uma relação entre dois componentes é criada a partir dos dados da OpenTelemetry.

Exemplos comuns de relações incluem (eles fazem parte do OpenTelemetry StackPack padrão):

* Chamadas de serviço síncronas (cliente -> servidor)
* Fluxos de mensagens assíncronas (produtor -> consumidor, consumidor -> par, produtor -> par)
* Conexões de serviço para banco de dados
* Instância de serviço fornecida pelo serviço
* Relações de execução (serviço ou função executa tarefa, host ou função)

== Expressões e a linguagem de expressão comum (CEL)

Os mapeamentos de topologia dependem da https://cel.dev/[Common Expression Language (CEL)] para selecionar telemetria, extrair valores e construir campos de componentes e relações.

Não é necessário ter experiência prévia em CEL para começar - a maioria das expressões são simples verificações booleanas, pesquisas de atributos ou construções de strings. Expressões mais avançadas podem ser introduzidas de forma incremental, conforme necessário.

Para obter mais informações sobre como aproveitar o CEL nos mapeamentos de topologia, consulte a página de xref:/setup/custom-integrations/otelmappings/schemas-ref.adoc[esquemas].
