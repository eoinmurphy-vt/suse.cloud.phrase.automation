= 親和値の設定
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: {stackstate-product-name} 親和性の構成

アフィニティ設定は、{stackstate-product-name} におけるポッドのスケジューリング動作を制御する。アフィニティ値は、ポッドがスケジューリングされる場所を制御することで、リソースの利用を最適化し、高可用性を確保するのに役立ちます。

== 利用可能な設定オプション

=== ノード・アフィニティ

ノードアフィニティを使用すると、同じアベイラビリティゾーンにデプロイされたEC2ノードなど、特定のノードまたはインスタンスグループにポッドをスケジュールできる。

[,yaml]
----
affinity:
  # Node Affinity settings - applied to all components when configured
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
----

=== ポッド・アンチ・アフィニティ

ポッド・アンチ・アフィニティを使用して、データ・サービスのレプリカを異なるノードにスケジュールし、高可用性を確保することができます。デフォルトでは、スケジューリングは必須（ハード・アンチ・アフィニティ）で、`topologyKey` は`kubernetes.io/hostname` 。

[,yaml]
----
affinity:
  podAntiAffinity:
    # Enable required pod anti-affinity (true = hard, false = soft)
    requiredDuringSchedulingIgnoredDuringExecution: true
    # Topology key for pod anti-affinity
    topologyKey: "kubernetes.io/hostname"
----

== 行動

=== ノード・アフィニティ

* **設定されている場合**：すべてのコンポーネントに適用

=== ポッド・アンチ・アフィニティ

* **設定されている場合**：`sizing.profile` の末尾が`-ha` の場合のみ適用（高可用性プロファイル）
* **HAのプロフィール** `150-ha` `250-ha` `500-ha` 、 `4000-ha`
* **影響を受ける部品**Clickhouse、Kafka、Zookeeper、VictoriaMetrics、StackGraph、Elasticsearchを含むすべてのステートフル・データ・サービス

== 構成例

=== 基本ノードアフィニティ（同一アベイラビリティゾーン）＋ハードPodAntiAffinityによるHA展開

`sizing.profile` 、HAプロファイルの1つに設定したと仮定する。以下の値は、{stackstate-product-name} のコンポーネントを`us-west-2a` ゾーンにスケジューリングし、同じデータサービスのレプリカを異なるホスト（デフォルトでは`affinity.podAntiAffinity.requiredDuringSchedulingIgnoredDuringExecution`=`true` ）にスケジューリングするようにnodeAffinityを設定するものです。
[,yaml]
----
affinity:
  # Schedule all pods to nodes in the same AZ
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
----

=== HA展開のためのソフトPodAntiAffinity
異なるノードにデータサービスを強制的にスケジューリングすることが望ましくない場合、PodAntiAffinityを「ソフト」モードで設定することができます：
[,yaml]
----
affinity:
  podAntiAffinity:
    requiredDuringSchedulingIgnoredDuringExecution: false
----

== 使用方法

[tabs]
====
推奨方法::
+
--
[NOTE]
=====
`global.suseObservability` 設定方法は、バージョン`2.8.0` 以降で利用可能です。それ以前のバージョンの場合は、レガシー・メソッドを使用する。
=====

アフィニティ・コンフィギュレーションを`global.suseObservability.affinity` の`values.yaml` ファイルに直接追加する：

[source,yaml]
----
global:
  suseObservability:
    # Your existing configuration
    license: "YOUR-LICENSE-KEY"
    baseUrl: "https://observability.example.com"
    sizing:
      profile: "150-ha"
    adminPassword: "your-password"

    # Affinity configuration
    affinity:
      # Node affinity - schedule all pods to nodes in the same AZ
      nodeAffinity:
        requiredDuringSchedulingIgnoredDuringExecution:
          nodeSelectorTerms:
          - matchExpressions:
            - key: topology.kubernetes.io/zone
              operator: In
              values:
              - us-west-2a

      # Pod anti-affinity for HA profiles (infrastructure components)
      podAntiAffinity:
        # true = hard anti-affinity, false = soft anti-affinity
        requiredDuringSchedulingIgnoredDuringExecution: true
        topologyKey: "kubernetes.io/hostname"
----

その後、コマンドひとつでインストールやアップグレードができる：

[,bash]
----
helm upgrade --install \
  --namespace suse-observability \
  --create-namespace \
  --values values.yaml \
  suse-observability \
  suse-observability/suse-observability
----

TIP: HAプロファイルの場合、ポッド・アンチ・アフィニティはすべてのステートフル・データ・サービスに自動的に適用される。設定する必要があるのは、デフォルトの動作を変更したい場合だけだ。

レガシー方式からの移行の詳細については、xref:/setup/upgrade-stackstate/migrate-to-global-mode.adoc[移行ガイドを]参照してください。
--
レガシーメソッド（非推奨）::
+
--
[WARNING]
=====
このメソッドは非推奨。新規設置の場合は、上記の推奨方法を使用してください。この方法を使用している既存のインストールについては、新しい構成フォーマットに移行するためのxref:/setup/upgrade-stackstate/migrate-to-global-mode.adoc[移行ガイドを]参照してください。

==== ステップ1：アフィニティ・バリュー・ファイルの作成

希望するアフィニティ構成で別の値ファイルを作成する。例えば、以下を`suse-observability-values-values.yaml` として保存する：

[,yaml]
----
affinity:
  nodeAffinity:
    requiredDuringSchedulingIgnoredDuringExecution:
      nodeSelectorTerms:
      - matchExpressions:
        - key: topology.kubernetes.io/zone
          operator: In
          values:
          - us-west-2a
  podAntiAffinity:
    requiredDuringSchedulingIgnoredDuringExecution: true
----

==== ステップ2：アフィニティ・テンプレート値の生成

以下のコマンドを実行して、親和性値のテンプレートを生成する：

[,text]
----
export VALUES_DIR=.
helm template \
  --set license='<your license>' \
  --set baseUrl='<suse-observability-base-url>' \
  --set sizing.profile='<sizing.profile>' \
  --values suse-observability-values-values.yaml \
  suse-observability-values \
  suse-observability/suse-observability-values --output-dir $VALUES_DIR
----

==== ステップ3：Helmのインストールで生成された値を使用する

生成されたアフィニティ値をHelmのインストールに含めます：

[,bash]
----
helm upgrade \
  --install \
  --namespace suse-observability \
  --values $VALUES_DIR/suse-observability-values/templates/baseConfig_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/sizing_values.yaml \
  --values $VALUES_DIR/suse-observability-values/templates/affinity_values.yaml \
  suse-observability \
  suse-observability/suse-observability
----
