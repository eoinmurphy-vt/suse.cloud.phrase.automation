:revdate: 2025-07-01

== Requisitos previos:

* https://kind.sigs.k8s.io/[amable]
* https://helm.sh/[timón]
* https://tilt.dev/[inclinación]

== Crear un entorno de desarrollo local

. Clonar el https://github.com/rancher/turtles[{product_name}] repositorio localmente
. Crear *tilt-settings.yaml*:
+
[source,yaml]
----
{
    "k8s_context": "k3d-rancher-test",
    "default_registry": "ghcr.io/turtles-dev",
    "debug": {
        "turtles": {
            "continue": true,
            "port": 40000,
            "insecure_skip_verify": "true"
        }
    }
}
----
+
. Abra un terminal en la raíz del repositorio {product_name} 
. Ejecuta lo siguiente:
+
[source,bash]
----
make dev-env

# Or if you want to use a custom hostname for Rancher and your ngrok credentials
NGROK_API_KEY=`key` NGROK_AUTHTOKEN=`token` RANCHER_HOSTNAME=my.customhost.dev make dev-env
----
+
. Si no dispone de credenciales ngrok y no las ha proporcionado en el paso anterior, también puede abrir un nuevo terminal e iniciar ngrok o inlets después de que se haya iniciado tilt:
+
[source,bash]
----
kubectl port-forward --namespace cattle-system svc/rancher 10000:443
ngrok http https://localhost:10000
----

== ¿Qué ocurre cuando se ejecuta `make dev-env`?

. Se creará un clúster https://kind.sigs.k8s.io/[del tipo] utilizando la siguiente https://github.com/rancher/turtles/blob/main/scripts/kind-cluster-with-extramounts.yaml[].
. Los siguientes componentes de prerrequisitos serán instalados usando Helm:
 ** Gerente ranchero.
 ** Gestor de certificados.
 ** ngrok Kubernetes Operator.
. Las imágenes Docker se construirán localmente, y Helm desplegará los siguientes controladores en el clúster:
 ** Controlador API de Core Cluster.
 ** RKE2 Bootstrap/Control Plane Provider controllers.
 ** Controlador del proveedor de infraestructura Docker.
 ** Proveedor de complementos CAPI para el controlador de flotas.
 ** {product_name} controlador.
 ** {product_name} Cluster API Controlador de operador.
 ** {product_name} ClusterClass controller.
 ** {product_name} Controlador de operaciones del día 2.
. El entorno de desarrollo se iniciará mediante `tilt up`.
