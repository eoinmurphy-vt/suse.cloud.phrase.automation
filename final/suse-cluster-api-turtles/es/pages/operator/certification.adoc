= Certificación de proveedores 
:revdate: 2025-06-30
:page-revdate: {revdate}

== ¿Qué es un proveedor certificado?

Como la mayoría de los proveedores CAPI son proyectos upstream mantenidos por la comunidad abierta, no hay forma segura de garantizar que cualquier cluster aprovisionado con un proveedor dado pueda ser importado a Rancher a través de Turtles. No obstante, aplicamos un proceso de certificación para los proveedores que sí lo están:

* Probado activamente como parte de nuestro conjunto de pruebas E2E.
* Mantiene actualizado este conjunto de pruebas E2E para validar las versiones recientes del proveedor.

== Certifique a su proveedor personalizado

Además, si usted es un desarrollador de proveedores o simplemente desea utilizar un proveedor diferente que no figura como certificado, tiene 2 opciones:

* Reutilice la suite Turtles E2E existente para comenzar con la solicitud del estado de certificación. Encontrará información más detallada aquí, en la secciónxref:./certification.adoc#_provider_certification_using_turtles_test_suite[Certificación de proveedores mediante Turtles Test Suite].
* Utilice su propia suite de integración para certificar a su proveedor. Encontrará información más detallada aquí, en la secciónxref:./certification.adoc#_certify_provider_with_custom_integration_suite[Certificar al proveedor con el paquete de integración personalizado].

[discrete]
==== *¿Por qué querría validar a mi proveedor?*

El número de proveedores de CAPI sigue creciendo y la comunidad presenta nuevos proyectos para distintos hosts de infraestructuras. Esta apertura contribuye a enriquecer el ecosistema CAPI, pero hace imposible controlar y probar a todos los proveedores con los que trabajará Turtles en todo el proyecto CAPI. Los desarrolladores de Turtles mantendrán una xref:../overview/certified.adoc[lista de proveedores certificados] que son probados y validados activamente por mecanismos de IC, pero esta lista se limitará a proveedores clave y conocidos. Por ello, animamos a los usuarios de los proveedores no incluidos en la lista a reutilizar el conjunto de pruebas de Turtles y validar la integración entre el proveedor y Turtles.

[discrete]
==== *¿Puedo utilizar Turtles con un proveedor no certificado?*

Turtles es un proyecto que pretende ser agnóstico e integrarse con todo el ecosistema CAPI y eres libre de utilizarlo sin validar a ningún proveedor. Lo más probable es que no encuentres problemas en la mayoría de los proyectos ascendentes.

[discrete]
==== *¿Cuál es la diferencia entre certificado y respaldado?*

Dado que los proveedores CAPI son proyectos mantenidos por la comunidad Kubernetes, no podemos garantizar la compatibilidad con un proveedor determinado. Por eso optamos por centrarnos en la validación constante de los proveedores xref:../overview/certified.adoc[certificados] y ofrecemos a los usuarios la posibilidad de integrarse en el conjunto de pruebas existente para verificar los proveedores de su elección.

== Certificación de proveedores mediante Turtles Test Suite

El proceso de certificación se basa en la verificación de la integración de {product_name} con los proveedores de CAPI. Para simplificar esta tarea, hemos preparado una prueba genérica que valida el aprovisionamiento y la importación de un clúster CAPI descendente.

[TIP]
====
Le recomendamos que consulte este https://github.com/rancher-sandbox/turtles-integration-suite-example[ejemplo de] sobre cómo utilizar el conjunto de pruebas de Turtles.
====

=== Pruebe y certifique a su proveedor

El primer paso para validar que su proveedor es compatible con Turtles y que puede aprovisionar clusters CAPI e importarlos a Rancher a través de Turtles es integrarse con nuestra suite de pruebas. Proporcionamos un repositorio con un https://github.com/rancher-sandbox/turtles-integration-suite-example[ejemplo de integración en] que puede utilizar como referencia para su integración.

Turtles como proyecto contiene un https://github.com/rancher/turtles/tree/main/test/e2e/suites[número] para verificar diferentes características y procesos pero, para la certificación de proveedores, requerimos que ejecutes sólo una prueba que utilice un flujo GitOps. Turtles es un proyecto que se integra bien con un enfoque GitOps para el aprovisionamiento de clústeres y por eso es nuestra principal forma de validar la integración de proveedores con Rancher. Si se ejecuta el paquete completo para un proveedor CAPI determinado:

* Cree un clúster de gestión en el entorno deseado.
* Instale Rancher y Turtles con todos los requisitos previos.
* Instala Gitea.
* Ejecute la suite que creará un repositorio git, aplique la plantilla de clúster utilizando Fleet y verifique que el clúster se ha creado y se ha importado correctamente en Rancher.

==== Configuración de prueba

Para ejecutar correctamente el conjunto de pruebas, tendrá que proporcionar una serie de variables de entorno. Algunos de ellos son agnósticos, lo que significa que son necesarios para cualquier proveedor que desee probar, pero otros serán específicos para el proveedor que está validando. Por favor, tenga en cuenta las especificaciones particulares del proveedor que se está probando, tales como credenciales, puntos finales, etc.

[TIP]
====
A continuación, le recomendamos que lea la xref:../operator/certificationsuite.adoc[Guía del conjunto de pruebas]
====

== Certifique a su proveedor con una suite de integración personalizada

Para certificar a su proveedor mediante una suite de integración personalizada, asegúrese de que cumple los siguientes requisitos clave:

=== Requisitos

* El proveedor debe desplegarse utilizando la API `CAPIProvider`.
* Los clusters creados con el proveedor deben ser importados con éxito a Rancher a través de Turtles.

=== Proceso de certificación

1. **Desarrollar una suite de integración** 
   * Instalar y configurar Rancher y Turtles.
   * Despliegue el proveedor CAPI.
   * Aprovisione un clúster e impórtelo en Rancher.

2. **Ejecutar y validar pruebas**
   * Asegúrese de que los clusters se crean e importan correctamente.

3. **Presentar una solicitud de certificación**
   * Proporcione los resultados de las pruebas, los registros y los pasos seguidos.
   * Envíe una incidencia a GitHub utilizando la siguiente https://github.com/rancher/turtles/issues/new?template=certification_request.yaml[plantilla].

== Solicitud de certificación

La integración con el conjunto de pruebas de Turtles y la ejecución de comprobaciones en el proveedor de su interés bastan para validar que es compatible con {product_name}. Dado que no nos resulta factible probar continuamente todos los proveedores CAPI, este flujo de trabajo de certificación le permitirá a usted, como usuario, verificar la funcionalidad esperada. Sin embargo, como no estamos probando activamente nuevas iteraciones del proveedor (con nuevas versiones de Turtles), el soporte y la garantía para el proveedor dado son limitados, y usted será responsable de validar futuras versiones.

Si, tras haber realizado correctamente las comprobaciones de su proveedor, desea solicitar que se añada a la lista de xref:../overview/certified.adoc[Proveedores Certificados] y se incluya en el conjunto E2E periódico del proyecto, puede solicitar la certificación enviando una incidencia a GitHub utilizando la https://github.com/rancher/turtles/issues/new?template=certification_request.yaml[plantilla]. La comunidad revisará la propuesta y determinará la viabilidad de añadir el proveedor a la matriz de certificación.
