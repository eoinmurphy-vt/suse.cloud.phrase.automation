= Guía de migración
:revdate: 2025-11-14
:page-revdate: {revdate}

== Contexto

A partir de Rancher v2.13, {product_name} se instala como parte de Rancher, eliminando la necesidad de un despliegue separado. Antes de actualizar a Rancher v2.13, los usuarios deben seguir esta guía para garantizar un proceso de actualización sin problemas.

== Migración

Siga los siguientes pasos en orden, _antes de_ actualizar a Rancher v2.13:

. Asegúrese de actualizar {product_name} a la versión 0.24.3. Este paso es importante porque todas las versiones de parches para v0.24 preparan el clúster para la actualización a Rancher v2.13.
. Desinstalar {product_name}. Esto es necesario porque a partir de la versión 2.13 de Rancher, {product_name} ya no es un complemento opcional, sino un componente del sistema Rancher. Como tal, Rancher lo instala en un espacio de nombres diferente al anterior y, por lo tanto, es necesario desinstalar manualmente primero la versión instalada previamente. La desinstalación de la versión 0.24.3 de {product_name} elimina el núcleo del proveedor CAPI, pero conserva los CRD de CAPI. Durante la migración, el controlador CAPI central no se ejecutará en el clúster de gestión, pero los clústeres CAPI posteriores deberían seguir ejecutándose como de costumbre y no verse afectados.
. Ejecute los siguientes comandos kubectl:
+
[source,bash]
----
kubectl patch crd capiproviders.turtles-capi.cattle.io --type=json -p='[{"op": "add", "path": "/metadata/annotations/meta.helm.sh~1release-namespace", "value": "cattle-turtles-system"}]'
kubectl patch crd clusterctlconfigs.turtles-capi.cattle.io --type=json -p='[{"op": "add", "path": "/metadata/annotations/meta.helm.sh~1release-namespace", "value": "cattle-turtles-system"}]'
----

Estos comandos parchearán los CRDs `capiprovider` y `clusterctlconfig` para que sean adoptados por {product_name} cuando se instalen como parte de Rancher v2.13. 

Tenga en cuenta que la migración descrita anteriormente es necesaria para los usuarios que utilicen las versiones de {product_name} &le; v0.24.3. Los nuevos usuarios que instalen Rancher v2.13 no necesitan realizar ninguna migración.
