= 中的集群资源关系 {product_name}
:revdate: 2026-01-06
:page-revdate: {revdate}

== 概述

Rancher 集成了多个组件，这些组件定义了自己的`Cluster` 自定义资源。{product_name} 使用 Cluster API 和 Fleet 在此基础上进行构建，这可能会导致`Cluster` 资源如何交互以及用户应编辑哪些内容方面的混乱。

本文件解释了以下群集资源的目的、行为和相互关系：

* `clusters.cluster.x-k8s.io` (CAPI 小组）
* `clusters.fleet.cattle.io` (舰队群）
* `clusters.provisioning.cattle.io` (供应群组）
* `clusters.management.cattle.io` (管理组）

== 资源概要

|===
| API 集团 | 集群资源 | 角色

| `cluster.x-k8s.io`
| CAPI 小组
| 定义群集生命周期和拓扑结构。在《乌龟》中充当真理的源泉。

| `fleet.cattle.io`
| 舰队集群
| 用于 GitOps 捆绑目标定位和通过 Fleet 同步应用程序。

| `provisioning.cattle.io`
| 供应群集
| Rancher 早期的通用集群抽象。封装了与基础架构无关的配置和特定于提供商的配置，是生成集群注册令牌和清单所必需的。在{product_name} ，这一资源被用于包装进口的 CAPI 集群，但目前正在逐步淘汰。

| `management.cattle.io`
| 管理小组
| `management.cattle.io/v3` 集群现在是由 Turtles 创建和管理的默认资源。这不应与 CAPI 管理集群相混淆，其中的 "管理 "指的是集群 API 生命周期管理，而不是 Rancher 的`management.cattle.io` API 组。
|===

== 关系

=== CAPI 群 → 船队群

启用用于 Fleet 的集群 API 附加提供程序 (CAAPF) 后，它会将标签和注释从 CAPI`Cluster` 传播到相应的 Fleet`Cluster` 。这样就能通过标签选择器定位 GitOps 捆绑程序。在 CAPI`Cluster` 达到`ControlPlaneInitialized` 条件后，CAAPF 将创建相应的 Fleet`Cluster` 资源。

=== CAPI 集群 → Provisioning 集群（传统）

可以创建一个`provisioning.cattle.io.Cluster` 来封装 CAPI 集群，并公开 Rancher 的特定字段（如显示名称、RKE 配置）。不过，在{product_name} 的大多数工作流程中，它正被逐步淘汰，用户无需与这一资源进行交互。

=== 管理群集 → 供应群集

对于传统集群，可通过`management.cattle.io/v3.Cluster` 生成`provisioning.cattle.io/v1.Cluster` 。该对象由 Rancher 的用户界面和 API 使用，用户不得直接修改。
仅通过 Turtles 管理的新群集不会自动创建`provisioning.cattle.io` 群集。

== 传播行为

|===
| 从 → 到 | 传播 | 说明

| CAPI → 船队
| 是
| 由 CAAPF 传播

| CAPI → 供应
| 没有
| 不能自动创建；只能通过 Turtles 为传统集群生成

| CAPI → 管理
| 没有
| 仅通过传统群集的 "供应群集

| 供应 → 船队
| 没有
| 不支持

| 舰队 → CAPI
| 没有
| 不支持

| 管理→调配
| 有条件
| 仅适用于传统集群；新的 Turtles 管理集群不会生成供应集群
|===

== 用户应该编辑什么？

=== 推荐使用： `cluster.x-k8s.io.Cluster`

CAPI 群组是{product_name} 中群组定义的权威资源。
标签必须放置在 CAPI 群集对象上。它们将自动传播到舰队集群，使基于标签的捆绑选择器按预期运行。

=== 有条件： `provisioning.cattle.io.Cluster`

仅在使用 Rancher 本机配置功能（如`spec.rkeConfig` 、UI 驱动的流程）时需要。除非有明确要求，否则避免编辑。

=== 避免： `fleet.cattle.io.Cluster`

自动管理机群。除非对舰队内部结构进行修改，否则不鼓励编辑。

=== 避免： `management.cattle.io.Cluster`

这些资源是内部资源，完全由 Rancher 管理。请勿修改。

[NOTE]
====
* **CAPI 群组**是真相的源泉。
* **舰队集群**是通过 CAAPF 从 CAPI 集群自动生成的。
* 除非涉及 Rancher 供应，否则*不需要* **供应**和**管理集群**。
====

[TIP]
====
使用 CAPI 集群的标签来驱动舰队捆绑部署。CAAPF 控制器会确保这些标签同步到舰队集群对象。
====
