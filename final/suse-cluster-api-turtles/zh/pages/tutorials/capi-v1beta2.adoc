= 升级到 CAPI v1beta2

`v0.26.0` 版的 Rancher Turtles 采用了 CAPI`v1beta2` 合同，该合同附带 CAPI core v1.12.2。如果您正在使用基于 CAPI 的调配，则在升级前需要检查一些事项。

变化分为三个方面：

* xref:#_clusterclass[ `ClusterClass` 资源的结构]。
* xref:#_cluster[ `Cluster` 拓扑如何引用其类别]。
* xref:#_rke2_provider_caprke2[RKE2 提供商的变化]。

在进行任何更改之前，请通读适用于您设置的章节。

== 关于向后兼容性的说明

CAPI v1.12 继续与`v1beta2` 一起为`v1beta1` 资源提供服务。由于 API 服务器会在两个版本之间进行透明转换，因此集群上的现有资源无需立即迁移即可继续运行。也就是说，您编写的任何新清单都应以`v1beta2` 为目标，而且您应该计划随着时间的推移迁移现有清单。`v1beta1` API 将在未来的 CAPI 版本（2026 年 8 月）中删除，届时生态系统将有时间对`v1beta2` 进行调整。

*建议*：首先评估您是否准备好采用`v1beta2` ，并采取相应行动。如果您选择不立即迁移，请制定一个切换到使用新 API 的计划，以避免将来出现中断。

此外，上游社区还提供了官方迁移指南：

* https://cluster-api.sigs.k8s.io/developer/providers/migrations/v1.10-to-v1.11[CAPI v1.10 至 v1.11]：涵盖核心 CAPI 从`v1beta1` 到`v1beta2` 的所有更改。
* https://cluster-api.sigs.k8s.io/developer/providers/migrations/v1.11-to-v1.12[CAPI v1.11 到 v1.12]：CAPI v1.12 中的大部分改动都可以被认为是小改动。

== ClusterClass

=== 模板参考

在`v1beta1` 中，指向`ClusterClass` 模板对象的字段称为`ref` 。在`v1beta2` 中，它被称为`templateRef` 。这适用于`spec.infrastructure`,`spec.controlPlane`, 和`spec.controlPlane.machineInfrastructure` 。

之前

[,yaml]
----
spec:
  infrastructure:
    ref:
      apiVersion: infrastructure.cluster.x-k8s.io/v1beta1
      kind: DockerClusterTemplate
      name: my-cluster-template
----

之后

[,yaml]
----
spec:
  infrastructure:
    templateRef:
      apiVersion: infrastructure.cluster.x-k8s.io/v1beta2
      kind: DockerClusterTemplate
      name: my-cluster-template
----

=== 工人部分

在`v1beta1` 中，机器部署的引导和基础设施引用被嵌套在`template` 字段下。`v1beta2` 中的根密钥已消失。`bootstrap` 和`infrastructure` 现在是机器部署条目的直接子代。

之前

[,yaml]
----
workers:
  machineDeployments:
    - class: default-worker
      template:
        bootstrap:
          ref:
            apiVersion: bootstrap.cluster.x-k8s.io/v1beta1
            kind: KubeadmConfigTemplate
            name: my-bootstrap-template
        infrastructure:
          ref:
            apiVersion: infrastructure.cluster.x-k8s.io/v1beta1
            kind: DockerMachineTemplate
            name: my-worker-template
----

之后

[,yaml]
----
workers:
  machineDeployments:
    - class: default-worker
      bootstrap:
        templateRef:
          apiVersion: bootstrap.cluster.x-k8s.io/v1beta2
          kind: KubeadmConfigTemplate
          name: my-bootstrap-template
      infrastructure:
        templateRef:
          apiVersion: infrastructure.cluster.x-k8s.io/v1beta2
          kind: DockerMachineTemplate
          name: my-worker-template
----

== 群组

`Cluster` 资源现在通过结构化对象而不是内联字符串引用其`ClusterClass` 。在`v1beta1` 中，类名和命名空间被指定为`spec.topology.class` 和`spec.topology.classNamespace` 。在`v1beta2` 中，这两个字段都由一个`spec.topology.classRef` 对象代替。

之前

[,yaml]
----
apiVersion: cluster.x-k8s.io/v1beta1
kind: Cluster
spec:
  topology:
    class: my-clusterclass
    classNamespace: capi-system
----

之后

[,yaml]
----
apiVersion: cluster.x-k8s.io/v1beta2
kind: Cluster
spec:
  topology:
    classRef:
      name: my-clusterclass
      namespace: capi-system
----

`Cluster` 规范的其他部分（网络设置、`controlPlane.replicas`,`workers.machineDeployments`,`variables`,`version` 等）保持不变。

== RKE2 提供者（CAPRKE2）

本版本发布了 CAPRKE2 v0.23.1，高于 v0.22.1。用于 RKE2 引导和控制平面资源的`v1beta2` API 是在 v0.22.0 中引入的，同时还支持 CAPI v1.11，而 v0.23.0 则通过升级到 CAPI v1.12.2 完成了这项工作。v0.22.1 和 v0.23.1 都是在这些次版本基础上发布的补丁。

*注意*：请阅读link:https://caprke2.docs.rancher.com/05_reference/01_api_versions.html#v1beta1-to-v1beta2[Provider RKE2 一书中] CAPI有关 API 更改的提供程序文档。

=== 新版 v1beta2 应用程序接口

CAPRKE2 现在为自己的`v1beta2` API 提供服务，包括`RKE2ControlPlane`,`RKE2ControlPlaneTemplate`, 和`RKE2Config`/`RKE2ConfigTemplate` 。`v1beta1` API 仍然可用，但在`v1beta1` 中被标记为弃用的几个字段现已在`v1beta2` 中删除。

=== 删除了 RKE2C ControlPlane 中的字段

以下字段已从`RKE2ControlPlane.spec` `v1beta2` 中删除：

* `infrastructureRef` - 使用`spec.machineTemplate.spec.infrastructureRef` 代替。
* `nodeDrainTimeout` - 使用`spec.machineTemplate.spec.deletion.nodeDrainTimeout` 代替。

=== 重新命名超时字段

`RKE2ControlPlaneMachineTemplate` 中的超时字段已移至`deletion` 子对象下并重新命名。它们的类型也从`metav1.Duration` 改为`int32` ，现在期望的值以秒为单位，而不是持续时间字符串。

|===
| v1beta1 | v1beta2

| `nodeDrainTimeout`
| `deletion.nodeDrainTimeoutSeconds`

| `nodeVolumeDetachTimeout`
| `deletion.nodeVolumeDetachTimeoutSeconds`

| `nodeDeletionTimeout`
| `deletion.nodeDeletionTimeoutSeconds`
|===

`RKE2ControlPlaneMachineTemplate` 对象现在也需要一个`spec` 字段。

=== v1alpha1 过时

`v1alpha1` API 在 CAPRKE2 v0.22.0 中被弃用。如果您仍在使用`v1alpha1` 资源，现在是至少迁移到`v1beta1` 的好时机。

== 升级前的建议步骤

. 更新您维护的任何`ClusterClass` 清单：将`ref` 重命名为`templateRef` ，移除工作者部分的`template` 封装程序，并将核心 CAPI`apiVersion` 字符串更新为`v1beta2` 。
. 更新`Cluster` 清单：将`spec.topology.class` 和`spec.topology.classNamespace` 替换为`spec.topology.classRef.name` 和`spec.topology.classRef.namespace` 。
. 如果使用 CAPRKE2`v1beta2` 资源，请删除`RKE2ControlPlane.spec` 中的任何直接`infrastructureRef` 或`nodeDrainTimeout` 字段，并将其移至`spec.machineTemplate.spec` 。
. 请查看`examples/clusterclasses/` 中的更新示例，了解每个经认证的 CAPI 提供商的完整有效清单。
