=== Gestionnaire de l'élevage

Il s'agit du composant central de Rancher et les utilisateurs peuvent exploiter la fonction d'exploration existante dans le tableau de bord pour accéder aux détails de la charge de travail du cluster.

=== Agent de cluster Rancher

L'agent est déployé dans les clusters enfants, ce qui permet à Rancher d'importer et d'établir une connexion avec ces clusters. Cette connexion permet à Rancher de gérer efficacement les clusters enfants depuis sa plateforme.

=== {product_name} - Extension CAPI Rancher

Il assure l'intégration entre CAPI et Rancher et prend actuellement en charge les fonctionnalités suivantes :

* *Importation des clusters CAPI dans Rancher :* installation de Rancher Cluster Agent dans les clusters provisionnés CAPI.
* *Configuration de l'opérateur CAPI :* Aide à la configuration de l'opérateur CAPI.

== Topologie de déploiement

=== Gestionnaire de l'élevage et gestion de l'ICPA combinées

Il s'agit de la seule topologie actuellement prise en charge, dans laquelle Rancher Manager et {product_name} sont déployés sur le même cluster Kubernetes, et qui agit comme un cluster de gestion centralisé.

Cette architecture permet un déploiement simplifié des composants et offre une vue unique de tous les clusters. D'un autre côté, il est important de considérer que le nombre de clusters qui peuvent être gérés efficacement par l'API Cluster (CAPI) est limité par les ressources disponibles au sein de l'unique cluster de gestion.
