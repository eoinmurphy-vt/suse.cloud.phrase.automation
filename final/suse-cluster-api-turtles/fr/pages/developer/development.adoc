= Développement de l'installation
:revdate: 2025-07-01
:page-revdate: {revdate}

== Prérequis :

* https://kind.sigs.k8s.io/[type]
* https://helm.sh/[barre]
* https://tilt.dev/[inclinaison]

== Créer un environnement de développement local

. Cloner le https://github.com/rancher/turtles[{product_name}] localement
. Créer *tilt-settings.yaml*:
+
[source,yaml]
----
{
    "k8s_context": "k3d-rancher-test",
    "default_registry": "ghcr.io/turtles-dev",
    "debug": {
        "turtles": {
            "continue": true,
            "port": 40000,
            "insecure_skip_verify": "true"
        }
    }
}
----
+
. Ouvrez un terminal à la racine du dépôt {product_name} 
. Exécutez les opérations suivantes :
+
[source,bash]
----
make dev-env

# Or if you want to use a custom hostname for Rancher and your ngrok credentials
NGROK_API_KEY=`key` NGROK_AUTHTOKEN=`token` RANCHER_HOSTNAME=my.customhost.dev make dev-env
----
+
. Si vous n'avez pas d'identifiants ngrok et que vous ne les avez pas fournis à l'étape précédente, vous pouvez également ouvrir un nouveau terminal et lancer ngrok ou inlets après le démarrage de tilt :
+
[source,bash]
----
kubectl port-forward --namespace cattle-system svc/rancher 10000:443
ngrok http https://localhost:10000
----

== Que se passe-t-il lorsque vous lancez `make dev-env`?

. Une grappe de https://kind.sigs.k8s.io/[type] sera créée à l'aide de la https://github.com/rancher/turtles/blob/main/scripts/kind-cluster-with-extramounts.yaml[] suivante.
. Les composants prérequis suivants seront installés à l'aide de Helm :
 ** Gestionnaire de l'élevage.
 ** Responsable du Cert.
 ** ngrok Opérateur Kubernetes.
. Des images Docker seront construites localement, et Helm déploiera les contrôleurs suivants dans le cluster :
 ** Contrôleur de l'API Core Cluster.
 ** Contrôleurs RKE2 Bootstrap/Control Plane Provider.
 ** Contrôleur du fournisseur d'infrastructure Docker.
 ** CAPI Add-on Provider pour le contrôleur de flotte.
 ** {product_name} contrôleur.
 ** {product_name} Contrôleur de l'opérateur de l'API Cluster.
 ** {product_name} Contrôleur ClusterClass.
 ** {product_name} Contrôleur des opérations du jour 2.
. L'environnement de développement sera lancé à l'aide de `tilt up`.
