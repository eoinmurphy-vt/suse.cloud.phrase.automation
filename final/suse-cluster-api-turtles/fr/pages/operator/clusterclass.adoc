= Activation de ClusterClass
:revdate: 2025-06-30
:page-revdate: {revdate}

Dans cette section, nous abordons l'activation de la classe https://cluster-api.sigs.k8s.io/tasks/experimental-features/cluster-class/[*ClusterClass*] pour un fournisseur CAPI.

[TIP]
====
Avant d'utiliser la fonction ClusterClass, étudiez la documentation du fournisseur et vérifiez que cette fonction est prise en charge. Cette documentation comprend une matrice dans la section xref:../overview/certified.adoc[Fournisseurs certifiés] avec l'xref:../overview/certified.adoc#_clusterclass_support_for_certified_providers[état de la prise en charge de la topologie de la grappe].
====

== Activer ClusterClass

Afin d'utiliser la fonctionnalité ClusterClass, elle doit être activée pour l'API Cluster de base et tout fournisseur qui la prend en charge. Pour ce faire, la variable `CLUSTER_TOPOLOGY` est remplacée par `true`.

La carte {product_name} Helm activera automatiquement la fonction lors de l'installation de l'ICPA. Cependant, lorsque vous activez des fournisseurs supplémentaires, assurez-vous que `CLUSTER_TOPOLOGY` est défini dans la configuration du fournisseur. La ressource xref:../reference/capiprovider.adoc[CAPIProvider] de Turtles permet de transmettre des paramètres d'installation au fournisseur via `variables` comme suit :

[source,yaml]
----
apiVersion: turtles-capi.cattle.io/v1alpha1
kind: CAPIProvider
metadata:
  name: azure
  namespace: capz-system
spec:
  type: infrastructure
  name: azure
  configSecret:
    name: azure-variables
  variables:
    CLUSTER_TOPOLOGY: "true"
    EXP_CLUSTER_RESOURCE_SET: "true"
    EXP_MACHINE_POOL: "true"
    EXP_AKS_RESOURCE_HEALTH: "true"
----

La ressource définie dans ce fichier YAML installera le CAPZ avec la prise en charge d'un certain nombre de fonctionnalités, notamment `CLUSTER_TOPOLOGY`.
