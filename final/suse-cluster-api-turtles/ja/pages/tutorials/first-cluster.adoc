= 最初のクラスタの作成とインポート
:revdate: 2025-11-20
:page-revdate: {revdate}

このセクションでは、最初のクラスタを作成し、Rancher Managerにインポートする手順を説明します。クラスタのプロビジョニングには、**Fleetを使ったGitOpsワークフローを使う**方法と、** `kubectl` を使って手動でマニフェストを適用**する方法の2つがあります。

== 前提条件

* Rancher Managerクラスタ
* シナリオに合わせてインストールされたクラスタAPIプロバイダ - この説明では、https://github.com/kubernetes-sigs/cluster-api/tree/main/test/infrastructure/docker[]Docker https://github.com/rancher/cluster-api-provider-rke2[インフラストラクチャと]RKE2使用します -xref:../reference/capiprovider.adoc[Turtles]'をxref:../reference/capiprovider.adoc[使用した一般的なプロバイダの初期化を]参照してください。xref:../reference/capiprovider.adoc[ `CAPIProvider`]

== CAPIワークロードクラスタのプロビジョニング

[tabs]
======
Fleetを使ったGitOps::
+
--
*フリートリポジトリの設定*

クラスタ・プロビジョニングのプロセスを簡略化するために、https://github.com/rancher/turtles/tree/main/examples.このリポジトリの中身を調べると、一連の設定済みサンプルを使用することがわかります：

. Fleet（CNI、CSI、CPI）を使用してデプロイ可能な追加アプリ用の*アプリケーションフォルダ*。
. ClusterClassのサンプルがたくさん入った*clusterclasses*フォルダ。
. *Clusters/docker/rke2*フォルダにDockerクラスタの定義例がある。
+
[NOTE]
お好みであれば、同じ基本構造を使って独自のフリートリポジトリを作成することもできます。

*Rancher UIを使ってFleetリポジトリを追加する*

これでクラスタ定義がgitリポジトリにコミットされ、クラスタのプロビジョニングに使えるようになった。これを行うには、*継続的デリバリー*機能（Fleetを使用）を使用してRancher Managerクラスタ（クラスタAPI管理クラスタとしても動作している）にインポートする必要があります。

. ランチャーマネージャーへ
. メニューから「*継続的デリバリー*」を選択：image:sidebar.png[サイドバー]
. 右上のimage:ns.png[]ネームスペースとして*fleet-localを*選択する。
. サイドバーから*Git Reposを*選択
. *リポジトリの追加を*クリックします。
. 名前に*クラスターと*入力
. git リポジトリから*HTTPS*クローン URL を取得image:gh_clone.png[clone url] git
. *リポジトリURL*フィールドにURLを追加する
. ブランチ名を*mainに*変更
. 以下のパスを追加して、デプロイするディレクトリを指定する：
+
image:paths.png[フリートパスの構成]
+
----
examples/clusterclasses/docker/rke2
examples/applications/cni/calico
examples/applications/lb/docker
examples/clusters/docker/rke2
----
. *次へ」を*クリック
. *作成を*クリック
. *クラスター*名をクリック
. リソースの準備が整うのを見る
. メニューから*クラスタ管理を*選択
. クラスタがインポートされていることを確認する
--

kubectlを手動で使用する::
+
--
*クラスタリソースを適用する*

kubectlを使ってクラスタを作成するには、以下のコマンドを実行する：

. ターミナルを開き、ClusterClassと関連テンプレートを適用します：
+
[source,bash]
----
kubectl apply -f https://raw.githubusercontent.com/rancher/turtles/refs/heads/main/examples/clusterclasses/docker/rke2/clusterclass-docker-rke2.yaml
----

. ロードバランサーの設定を適用する：
+
[source,bash]
----
kubectl apply -f https://raw.githubusercontent.com/rancher/turtles/refs/heads/main/examples/applications/lb/docker/configmap.yaml
----

. CNI（キャリコ）コンフィギュレーションを適用する：
+
[source,bash]
----
kubectl apply -f https://raw.githubusercontent.com/rancher/turtles/refs/heads/main/examples/applications/cni/calico/helm-chart.yaml
----

. クラスターを作成する：
+
[source,bash]
----
kubectl apply -f https://raw.githubusercontent.com/rancher/turtles/refs/heads/main/examples/clusters/docker/rke2/cluster.yaml
----
. リソースの準備が整うのを見守る：
+
[source,bash]
----
kubectl get clusters -w
----
. メニューから*クラスタ管理を*選択
. クラスタがインポートされていることを確認する
--
======

== 次のステップRancherダッシュボードにクラスタが表示されるのを待つ

このチュートリアルでプロビジョニングするクラスタには、`cluster-api.cattle.io/rancher-auto-import` のラベルが含まれています。
このラベルは、クラスタをRancherに自動的にインポートするようTurtlesに指示します。
詳しくは「xref:user/rancher-cluster-registration.adoc[ランチャークラスター登録]」をご覧ください。
