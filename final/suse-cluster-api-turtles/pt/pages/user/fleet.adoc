= Criar um cluster usando o Fleet
:revdate: 2025-08-01
:page-revdate: {revdate}

Esta seção o guiará na criação de um cluster que utiliza o ClusterClass usando um fluxo de trabalho do GitOps com o Fleet.

[NOTE]
====
Este guia usa o https://github.com/rancher/turtles/tree/main/examples[repositório de exemplos].
====


== Pré-requisitos

* Cluster do Rancher Manager com {product_name} instalado.
* Provedores de API de cluster instalados para o seu cenário - usaremos a do Docker https://github.com/kubernetes-sigs/cluster-api/tree/main/test/infrastructure/docker[infraestrutura] e os provedores de https://github.com/rancher/cluster-api-provider-rke2[plano de controle/bootstrap] RKE2 nestas instruções - consulte xref:../reference/capiprovider.adoc[Inicialização para provedores comuns usando o Turtles] xref:../reference/capiprovider.adoc[ `CAPIProvider`]

== Configurar o Rancher Manager

*Configure seu repositório do Fleet*

Para simplificar o processo de provisionamento de cluster, usaremos uma série de exemplos pré-configurados que podem ser encontrados no repositório https://github.com/rancher/turtles/tree/main/examples.. Ao inspecionar o conteúdo desse repositório, você encontrará:

. Uma pasta de *aplicativos* para aplicativos adicionais que podem ser implementados usando o Fleet (CNI, CSI, CPI).
. Uma pasta *clusterclasses* com vários exemplos de ClusterClass.
. Uma pasta *clusters/docker/rke2* com um exemplo de definição de cluster do Docker.

[NOTE]
Se preferir, você pode criar seu próprio repositório do Fleet usando a mesma estrutura básica.

As definições do cluster serão importadas para o cluster do Rancher Manager (que também está atuando como um cluster de gerenciamento da API do Cluster) usando o recurso *de Entrega Contínua* (que usa o Fleet).

Há duas opções para fornecer a configuração. A primeira é usando a interface do usuário do Rancher Manager e a segunda é aplicando algum YAML ao seu cluster. Ambos são abordados a seguir.

=== Importar definições de cluster e de classe de cluster

[tabs]
======
Usando a interface do usuário do Rancher Manager::
+
--
. Vá para o Rancher Manager.
. Selecione *Continuous Delivery (Entrega contínua* ) no menu:image:sidebar.png[sidebar (barra lateral])
. Selecione *fleet-local* como o espaço de nome no canto superior direito.image:ns.png[namespace]
. Selecione *Git Repos* na barra lateral.
. Clique em *Add Repository*.
. Digite *clusters* como o nome.
. Obtenha o URL do clone *HTTPS* de seu repositório git.image:gh_clone.png[clone url] git
. Adicione o URL no campo *URL do repositório*.
. Altere o nome da ramificação para *main*.
. Adicione os seguintes caminhos para especificar os diretórios a serem implantados:
+
image:paths.png[Configuração dos caminhos da frota]
+
----
examples/clusterclasses/docker/rke2
examples/applications/cni/calico
examples/applications/lb/docker
examples/clusters/docker/rke2
----
. Clique em *Avançar*
. Clique em *Criar*
. Clique no nome dos *clusters* 
. Veja os recursos ficarem prontos
. Selecione *Cluster Management* no menu
. Verifique se seu cluster foi importado
--

Uso manual do kubectl::
+
--
. Obtenha o URL do clone *HTTPS* de seu repositório git.
. Crie um novo arquivo chamado *repo.yaml*.
. Adicione o seguinte conteúdo ao novo arquivo:
+
[source,yaml]
----
apiVersion: fleet.cattle.io/v1alpha1
kind: GitRepo
metadata:
  name: clusters
  namespace: fleet-local
spec:
  branch: main
  repo: https://github.com/rancher/turtles.git
  paths:
    - examples/clusterclasses/docker/rke2
    - examples/applications/cni/calico
    - examples/applications/lb/docker
    - examples/clusters/docker/rke2
  targets: []
----

. Aplique o arquivo ao cluster do Rancher Manager usando *o kubectl*:
+
[source,bash]
----
kubectl apply -f repo.yaml
----

. Vá para o Rancher Manager.
. Selecione *Continuous Delivery (Entrega contínua* ) na barra lateral
. Selecione *fleet-local* como o namespace no canto superior direito.
. Selecione *Git Repos* na barra lateral.
. Clique no nome dos *clusters*.
. Observe os recursos ficarem prontos.
. Selecione *Cluster Management* no menu.
. Verifique se o cluster foi importado.
--
======

=== Importar definições de cluster

Agora que as classes foram importadas, é possível usá-las com definições de cluster.

[tabs]
======
Usando a interface do usuário do Rancher Manager::
+
--
. Vá para o Rancher Manager.
. Selecione *Entrega contínua* no menu.
. Selecione *fleet-local* como o namespace no canto superior direito.
. Selecione *Git Repos* na barra lateral.
. Clique em *Add Repository*.
. Digite *clusters* como o nome.
. Obtenha o URL do clone *HTTPS* de seu repositório git.
. Adicione o URL no campo *URL do repositório*.
. Altere o nome da ramificação para *clusterclass*.
. Clique em *Add Path (Adicionar caminho*).
. Acesse `/clusters`.
. Clique em *Next*.
. Clique em *Create*.
. Clique no nome dos *clusters*.
. Observe os recursos ficarem prontos.
. Selecione *Cluster Management* no menu.
. Verifique se o cluster foi importado.
--

Uso manual do kubectl::
+
--
. Obtenha o URL do clone *HTTPS* de seu repositório git.
. Crie um novo arquivo chamado *repo.yaml*.
. Adicione o seguinte conteúdo ao novo arquivo:
+
[source,yaml]
----
apiVersion: fleet.cattle.io/v1alpha1
kind: GitRepo
metadata:
  name: clusters
  namespace: fleet-local
spec:
  branch: clusterclass
  repo: https://github.com/rancher-sandbox/rancher-turtles-fleet-example.git
  paths:
    - /clusters
  targets: []
----

. Aplique o arquivo ao cluster do Rancher Manager usando *o kubectl*:
+
[source,bash]
----
kubectl apply -f repo.yaml
----

. Vá para o Rancher Manager.
. Selecione *Continuous Delivery (Entrega contínua* ) na barra lateral.
. Selecione *fleet-local* como o namespace no canto superior direito.
. Selecione *Git Repos* na barra lateral.
. Clique no nome das *classes*.
. Observe os recursos ficarem prontos.
. Selecione *Cluster Management* no menu.
. Verifique se o cluster foi importado.
--
