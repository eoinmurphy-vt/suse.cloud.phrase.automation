== Como registrar um cluster CAPI no Rancher usando a importação automática do Turtles

O único fluxo de trabalho compatível com o registro de clusters CAPI no Rancher usando o Turtles é o uso do rótulo `cluster-api.cattle.io/rancher-auto-import`:

. Rotular um namespace para que todos os clusters contidos nele sejam importados.
+
[source,bash]
----
kubectl label namespace capi-clusters cluster-api.cattle.io/rancher-auto-import=true
----
. Rotular uma definição de cluster individual para que ela seja importada.
+
[source,bash]
----
kubectl label clusters.cluster.x-k8s.io cluster1 cluster-api.cattle.io/rancher-auto-import=true
----

Quando o rótulo de importação automática for usado, o Turtles aguardará que o cluster CAPI tenha a condição `ControlPlaneAvailable` definida como `True`.
Essa condição destaca que o cluster está totalmente inicializado e disponível para processar solicitações.
Em seguida, o Turtles criará automaticamente o recurso Rancher `clusters.management.cattle.io` no cluster de gerenciamento, levando à instalação do `cattle-cluster-agent` no cluster CAPI downstream.
Se a implantação do agente for bem-sucedida, o cluster importado deverá aparecer no Rancher Dashboard.

[NOTE]
====
Um namespace (ou cluster) pode ser marcado para importação automática a qualquer momento: antes ou depois de o cluster ter sido criado.
====

[CAUTION]
====
A marcação de um cluster para importação automática aciona o registro automático pelo controlador do Turtles; não há suporte para a execução manual de comandos de registro.
====

== Adicionar uma descrição personalizada ao cluster do Rancher

A descrição padrão atribuída a um cluster importado do Rancher é "cluster CAPI importado para o Rancher".image:imported-cluster-default-description.png[Cluster importado do Rancher com descrição padrão]

Para adicionar uma descrição personalizada ao cluster importado do Rancher, você pode adicionar a anotação `cluster-api.cattle.io/cluster-description` ao seu cluster CAPI.

[source,yaml]
----
apiVersion: cluster.x-k8s.io/v1beta1
kind: Cluster
metadata:
  name: docker-kubeadm-quickstart
  labels:
    cni: calico
  annotations:
    cluster-api.cattle.io/cluster-description: "This is a custom description of Rancher Cluster"
----

image:imported-cluster-custom-description.png[Rancher Cluster importado com descrição personalizada]

Se um cluster CAPI já estiver importado, a adição da anotação não terá efeito algum. Talvez seja necessário excluir o cluster importado do Rancher e reimportá-lo.

== Excluir o cluster importado do Rancher

Os rótulos `cluster-api.cattle.io/capi-cluster-owner` e `cluster-api.cattle.io/capi-cluster-owner-ns` podem ser usados para excluir o cluster do Rancher:

[source,bash]
----
kubectl delete clusters.management.cattle.io -l cluster-api.cattle.io/capi-cluster-owner=cluster1 -l cluster-api.cattle.io/capi-cluster-owner-ns=capi-clusters
----

Quando o cluster importado estiver marcado para exclusão:

. O controlador Turtles anotará o cluster CAPI com `imported: true`.
. Essa anotação impede que o cluster seja reimportado automaticamente pelo Rancher.
. Verifique a anotação usando o seguinte comando:
+
[source,bash]
----
kubectl -n capi-clusters get clusters.cluster.x-k8s.io cluster1 -o json | jq .metadata.annotations
{
"imported": "true"
...
}
----

[NOTE]
====
A exclusão do cluster do Rancher remove apenas o recurso `clusters.management.cattle.io` e desativa a importação automática para o cluster CAPI associado. O próprio cluster CAPI subjacente permanece intacto.
====

== Reimportar o cluster

Se você precisar reimportar o mesmo cluster CAPI para o Rancher, remova a anotação `imported: true` com o comando abaixo:

[source, bash]
----
kubectl -n capi-clusters patch clusters.cluster.x-k8s.io <cluster-name> --type='json' -p='[{"op": "remove", "path": "/metadata/annotations/imported"}]'
----
