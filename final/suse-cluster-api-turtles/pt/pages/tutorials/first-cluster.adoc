= Criar e importar seu primeiro cluster
:revdate: 2025-11-20
:page-revdate: {revdate}

Esta seção o guiará na criação do seu primeiro cluster e na importação dele para o Rancher Manager. São apresentados dois métodos alternativos para o provisionamento de clusters: **usar um fluxo de trabalho GitOps com o Fleet** e **aplicar manualmente os manifestos por meio do** site ** `kubectl`.**

== Pré-requisitos

* Cluster do Rancher Manager
* Provedores de API de cluster instalados para o seu cenário - usaremos a do Docker https://github.com/kubernetes-sigs/cluster-api/tree/main/test/infrastructure/docker[infraestrutura] e os provedores de https://github.com/rancher/cluster-api-provider-rke2[plano de controle/bootstrap] RKE2 nestas instruções - consulte xref:../reference/capiprovider.adoc[Inicialização para provedores comuns usando o Turtles] xref:../reference/capiprovider.adoc[ `CAPIProvider`]

== Provisionar um cluster de carga de trabalho CAPI

[tabs]
======
GitOps usando o Fleet::
+
--
*Configure seu repositório do Fleet*

Para simplificar o processo de provisionamento de cluster, usaremos uma série de exemplos pré-configurados que podem ser encontrados no repositório https://github.com/rancher/turtles/tree/main/examples.. Ao inspecionar o conteúdo desse repositório, você encontrará:

. Uma pasta de *aplicativos* para aplicativos adicionais que podem ser implementados usando o Fleet (CNI, CSI, CPI).
. Uma pasta *clusterclasses* com vários exemplos de ClusterClass.
. Uma pasta *clusters/docker/rke2* com um exemplo de definição de cluster do Docker.
+
[NOTE]
Se preferir, você pode criar seu próprio repositório do Fleet usando a mesma estrutura básica.

*Use a interface do usuário do Rancher para adicionar seu repositório do Fleet*

Agora que as definições do cluster foram confirmadas em um repositório git, elas podem ser usadas para provisionar os clusters. Para fazer isso, eles precisarão ser importados para o cluster do Rancher Manager (que também está atuando como um cluster de gerenciamento da API de cluster) usando o recurso *de Entrega contínua* (que usa o Fleet).

. Ir para o Rancher Manager
. Selecione *Continuous Delivery (Entrega contínua* ) no menu:image:sidebar.png[sidebar (barra lateral])
. Selecione *fleet-local* como o espaço de nome no canto superior direitoimage:ns.png[namespace]
. Selecione *Git Repos* na barra lateral
. Clique em *Add Repository (Adicionar repositório)*
. Digite *clusters* como o nome
. Obtenha o URL do clone *HTTPS* de seu repositório gitimage:gh_clone.png[clone url] git
. Adicione a URL no campo *URL do repositório* 
. Alterar o nome da ramificação para *principal*
. Adicione os seguintes caminhos para especificar os diretórios a serem implantados:
+
image:paths.png[Configuração dos caminhos da frota]
+
----
examples/clusterclasses/docker/rke2
examples/applications/cni/calico
examples/applications/lb/docker
examples/clusters/docker/rke2
----
. Clique em *Avançar*
. Clique em *Criar*
. Clique no nome dos *clusters* 
. Veja os recursos ficarem prontos
. Selecione *Cluster Management* no menu
. Verifique se seu cluster foi importado
--

Uso manual do kubectl::
+
--
*Aplicar os recursos do cluster*

Para criar o cluster usando o kubectl, execute os seguintes comandos:

. Abra um terminal e aplique o ClusterClass e os modelos relacionados:
+
[source,bash]
----
kubectl apply -f https://raw.githubusercontent.com/rancher/turtles/refs/heads/main/examples/clusterclasses/docker/rke2/clusterclass-docker-rke2.yaml
----

. Aplique a configuração do balanceador de carga:
+
[source,bash]
----
kubectl apply -f https://raw.githubusercontent.com/rancher/turtles/refs/heads/main/examples/applications/lb/docker/configmap.yaml
----

. Aplique a configuração do CNI (Calico):
+
[source,bash]
----
kubectl apply -f https://raw.githubusercontent.com/rancher/turtles/refs/heads/main/examples/applications/cni/calico/helm-chart.yaml
----

. Crie o cluster:
+
[source,bash]
----
kubectl apply -f https://raw.githubusercontent.com/rancher/turtles/refs/heads/main/examples/clusters/docker/rke2/cluster.yaml
----
. Veja os recursos ficarem prontos:
+
[source,bash]
----
kubectl get clusters -w
----
. Selecione *Cluster Management* no menu
. Verifique se seu cluster foi importado
--

Esse rótulo instruirá o Turtles a importar automaticamente o cluster para o Rancher.
