= Configuração do desenvolvimento
:revdate: 2025-07-01
:page-revdate: {revdate}

== Pré-requisitos:

* https://kind.sigs.k8s.io/[gentil]
* https://helm.sh/[helm]
* https://tilt.dev/[tilt]

== Criar um ambiente de desenvolvimento local

. Clone o https://github.com/rancher/turtles[{product_name}] repositório localmente
. Crie *o arquivo tilt-settings.yaml*:
+
[source,yaml]
----
{
    "k8s_context": "k3d-rancher-test",
    "default_registry": "ghcr.io/turtles-dev",
    "debug": {
        "turtles": {
            "continue": true,
            "port": 40000,
            "insecure_skip_verify": "true"
        }
    }
}
----
+
. Abra um terminal na raiz do repositório {product_name} 
. Execute o seguinte:
+
[source,bash]
----
make dev-env

# Or if you want to use a custom hostname for Rancher and your ngrok credentials
NGROK_API_KEY=`key` NGROK_AUTHTOKEN=`token` RANCHER_HOSTNAME=my.customhost.dev make dev-env
----
+
. Se você não tiver credenciais do ngrok e não as tiver fornecido na etapa anterior, também poderá abrir um novo terminal e iniciar o ngrok ou os inlets depois que o tilt for iniciado:
+
[source,bash]
----
kubectl port-forward --namespace cattle-system svc/rancher 10000:443
ngrok http https://localhost:10000
----

== O que acontece quando você executa `make dev-env`?

. Um cluster https://kind.sigs.k8s.io/[do tipo] será criado usando a seguinte https://github.com/rancher/turtles/blob/main/scripts/kind-cluster-with-extramounts.yaml[].
. Os seguintes componentes de pré-requisito serão instalados usando o Helm:
 ** Gerente de fazendas.
 ** Gerente de certificação.
 ** ngrok Kubernetes Operator.
. As imagens do Docker serão criadas localmente, e o Helm implantará os seguintes controladores no cluster:
 ** Controlador da API do cluster principal.
 ** Controladores do provedor RKE2 Bootstrap/Control Plane.
 ** Controlador do provedor de infraestrutura do Docker.
 ** Provedor complementar de CAPI para o controlador Fleet.
 ** {product_name} controlador.
 ** {product_name} Controlador do operador da API do cluster.
 ** {product_name} Controlador ClusterClass.
 ** {product_name} Controlador de operações do dia 2.
. O ambiente de desenvolvimento será iniciado usando `tilt up`.
